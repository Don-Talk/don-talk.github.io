# 异常处理

## 一、异常概述

### 1.1 什么是异常？

异常是程序运行过程中发生的非正常情况，它会中断程序的正常执行流程。

**示例：**
```java
int result = 10 / 0;  // ArithmeticException：除以零
String str = null;
str.length();          // NullPointerException：空指针
int[] arr = new int[5];
arr[10] = 1;           // ArrayIndexOutOfBoundsException：数组越界
```

### 1.2 为什么要处理异常？

- 提高程序的健壮性
- 提供更好的用户体验
- 便于问题定位和调试
- 防止程序崩溃

## 二、异常体系

### 2.1 异常类继承结构

```
Throwable（所有异常的父类）
├── Error（错误，无法恢复）
│   ├── OutOfMemoryError
│   ├── StackOverflowError
│   └── VirtualMachineError
│
└── Exception（异常，可以处理）
    ├── RuntimeException（运行时异常，非受检异常）
    │   ├── NullPointerException
    │   ├── ArrayIndexOutOfBoundsException
    │   ├── ClassCastException
    │   ├── IllegalArgumentException
    │   └── ArithmeticException
    │
    └── 其他 Exception（受检异常）
        ├── IOException
        ├── SQLException
        ├── ClassNotFoundException
        └── FileNotFoundException
```

### 2.2 Error vs Exception

| 对比项 | Error | Exception |
|--------|-------|-----------|
| 含义 | 严重错误 | 一般异常 |
| 可恢复 | 否 | 是 |
| 是否需要处理 | 否 | 是 |
| 示例 | OutOfMemoryError | NullPointerException |

### 2.3 受检异常 vs 非受检异常

| 对比项 | 受检异常 | 非受检异常 |
|--------|---------|-----------|
| 别名 | Checked Exception | Unchecked Exception |
| 继承自 | Exception | RuntimeException |
| 编译期检查 | 是 | 否 |
| 必须处理 | 是 | 否 |
| 示例 | IOException | NullPointerException |

## 三、异常处理机制

### 3.1 try-catch-finally

```java
try {
    // 可能抛出异常的代码
    int result = 10 / 0;
} catch (ArithmeticException e) {
    // 捕获并处理异常
    System.out.println("除数不能为零");
    e.printStackTrace();  // 打印异常堆栈信息
} finally {
    // 无论是否发生异常都会执行
    System.out.println("finally块");
}
```

**执行流程：**
1. 执行 try 块
2. 如果发生异常，跳转到匹配的 catch 块
3. 最后执行 finally 块

### 3.2 多个 catch 块

```java
try {
    // 可能抛出多种异常的代码
} catch (NullPointerException e) {
    System.out.println("空指针异常");
} catch (ArrayIndexOutOfBoundsException e) {
    System.out.println("数组越界异常");
} catch (Exception e) {
    System.out.println("其他异常");
}
```

**注意事项：**
- catch 块从上到下，范围从小到大
- 父类异常要放在子类异常后面

### 3.3 JDK 7+ 多异常捕获

```java
try {
    // 代码
} catch (NullPointerException | ArrayIndexOutOfBoundsException e) {
    System.out.println("捕获多种异常");
    e.printStackTrace();
}
```

### 3.4 finally 的作用

```java
InputStream is = null;
try {
    is = new FileInputStream("file.txt");
    // 读取文件
} catch (IOException e) {
    e.printStackTrace();
} finally {
    // 确保资源被释放
    if (is != null) {
        try {
            is.close();
        } catch (IOException e) {
            e.printStackTrace();
        }
    }
}
```

**finally 一定会执行吗？**
- ✅ 正常情况下会执行
- ❌ 如果 JVM 退出（System.exit()），不会执行
- ❌ 如果线程被强制终止，不会执行

## 四、throws 和 throw

### 4.1 throws（声明异常）

在方法签名中声明可能抛出的异常

```java
public void readFile() throws IOException {
    FileReader reader = new FileReader("file.txt");
    // 读取文件
}
```

**规则：**
- 受检异常必须声明或处理
- 非受检异常可以不声明
- 可以声明多个异常

```java
public void method() throws IOException, SQLException {
    // 代码
}
```

### 4.2 throw（抛出异常）

手动抛出异常对象

```java
public void setAge(int age) {
    if (age < 0 || age > 150) {
        throw new IllegalArgumentException("年龄不合法：" + age);
    }
    this.age = age;
}
```

### 4.3 throws vs throw

| 对比项 | throws | throw |
|--------|--------|-------|
| 位置 | 方法签名 | 方法体内 |
| 作用 | 声明异常 | 抛出异常 |
| 后跟 | 异常类名 | 异常对象 |
| 数量 | 可以多个 | 只能一个 |

## 五、自定义异常

### 5.1 创建自定义异常

```java
// 自定义受检异常
public class BusinessException extends Exception {
    private String errorCode;
    
    public BusinessException(String message) {
        super(message);
    }
    
    public BusinessException(String errorCode, String message) {
        super(message);
        this.errorCode = errorCode;
    }
    
    public String getErrorCode() {
        return errorCode;
    }
}

// 自定义运行时异常
public class ValidationException extends RuntimeException {
    public ValidationException(String message) {
        super(message);
    }
    
    public ValidationException(String message, Throwable cause) {
        super(message, cause);
    }
}
```

### 5.2 使用自定义异常

```java
public class UserService {
    
    public void register(String username, String password) throws BusinessException {
        // 验证用户名
        if (username == null || username.isEmpty()) {
            throw new BusinessException("USER_001", "用户名不能为空");
        }
        
        // 验证密码
        if (password == null || password.length() < 6) {
            throw new BusinessException("USER_002", "密码长度不能小于6位");
        }
        
        // 注册逻辑
        System.out.println("用户注册成功");
    }
}

// 调用
public class Test {
    public static void main(String[] args) {
        UserService service = new UserService();
        try {
            service.register("", "123");
        } catch (BusinessException e) {
            System.out.println("错误码：" + e.getErrorCode());
            System.out.println("错误信息：" + e.getMessage());
        }
    }
}
```

### 5.3 最佳实践

```java
// 好的做法：提供有意义的异常信息
throw new BusinessException("ORDER_001", "订单不存在，订单ID：" + orderId);

// 不好的做法：异常信息不明确
throw new BusinessException("出错了");
```

## 六、异常处理最佳实践

### 6.1 不要吞掉异常

```java
// 错误做法
try {
    doSomething();
} catch (Exception e) {
    // 什么都不做，吞掉异常
}

// 正确做法
try {
    doSomething();
} catch (Exception e) {
    log.error("操作失败", e);  // 记录日志
    throw new BusinessException("操作失败", e);  // 或者重新抛出
}
```

### 6.2 不要捕获过于宽泛的异常

```java
// 错误做法
try {
    doSomething();
} catch (Exception e) {  // 太宽泛
    e.printStackTrace();
}

// 正确做法
try {
    doSomething();
} catch (IOException e) {  // 具体异常
    log.error("IO操作失败", e);
} catch (SQLException e) {
    log.error("数据库操作失败", e);
}
```

### 6.3 及时释放资源

#### 方式1：try-finally

```java
FileInputStream fis = null;
try {
    fis = new FileInputStream("file.txt");
    // 读取文件
} catch (IOException e) {
    e.printStackTrace();
} finally {
    if (fis != null) {
        try {
            fis.close();
        } catch (IOException e) {
            e.printStackTrace();
        }
    }
}
```

#### 方式2：try-with-resources（JDK 7+，推荐）

```java
// 自动关闭资源
try (FileInputStream fis = new FileInputStream("file.txt");
     BufferedReader br = new BufferedReader(new InputStreamReader(fis))) {
    
    String line;
    while ((line = br.readLine()) != null) {
        System.out.println(line);
    }
} catch (IOException e) {
    e.printStackTrace();
}
```

**要求：** 资源类必须实现 `AutoCloseable` 接口

### 6.4 异常转换

```java
public void loadData() throws BusinessException {
    try {
        // 数据库操作
        database.query();
    } catch (SQLException e) {
        // 将底层异常转换为业务异常
        throw new BusinessException("DATA_001", "数据加载失败", e);
    }
}
```

### 6.5 不要在循环中捕获异常

```java
// 低效做法
for (String item : items) {
    try {
        process(item);
    } catch (Exception e) {
        e.printStackTrace();
    }
}

// 高效做法
try {
    for (String item : items) {
        process(item);
    }
} catch (Exception e) {
    log.error("批量处理失败", e);
}
```

## 七、常见异常及解决方案

### 7.1 NullPointerException

```java
// 原因：对象为null时调用方法
String str = null;
str.length();  // NullPointerException

// 解决：
if (str != null) {
    System.out.println(str.length());
}

// 或使用Optional
Optional.ofNullable(str).ifPresent(s -> System.out.println(s.length()));
```

### 7.2 ArrayIndexOutOfBoundsException

```java
// 原因：访问数组越界
int[] arr = new int[5];
arr[10] = 1;  // ArrayIndexOutOfBoundsException

// 解决：检查索引范围
if (index >= 0 && index < arr.length) {
    arr[index] = 1;
}
```

### 7.3 ClassCastException

```java
// 原因：类型转换错误
Object obj = "Hello";
Integer num = (Integer)obj;  // ClassCastException

// 解决：先判断类型
if (obj instanceof Integer) {
    Integer num = (Integer)obj;
}
```

### 7.4 NumberFormatException

```java
// 原因：字符串格式不正确
int num = Integer.parseInt("abc");  // NumberFormatException

// 解决：验证格式或使用try-catch
try {
    int num = Integer.parseInt(str);
} catch (NumberFormatException e) {
    System.out.println("数字格式错误");
}
```

### 7.5 ConcurrentModificationException

```java
// 原因：遍历集合时修改集合
List<String> list = new ArrayList<>();
list.add("a");
list.add("b");

for (String item : list) {
    list.remove(item);  // ConcurrentModificationException
}

// 解决：使用Iterator
Iterator<String> iterator = list.iterator();
while (iterator.hasNext()) {
    String item = iterator.next();
    iterator.remove();  // 安全删除
}
```

## 八、异常链

### 8.1 什么是异常链？

保留原始异常信息，形成异常链条

```java
try {
    // 底层操作
    database.connect();
} catch (SQLException e) {
    // 包装为业务异常，保留原始异常
    throw new BusinessException("数据库连接失败", e);
}
```

### 8.2 查看异常链

```java
try {
    // 代码
} catch (Exception e) {
    e.printStackTrace();  // 打印完整异常链
    
    // 获取原始异常
    Throwable cause = e.getCause();
    while (cause != null) {
        System.out.println(cause.getMessage());
        cause = cause.getCause();
    }
}
```

## 十、实战案例

### 10.1 统一异常处理（Spring Boot）

```java
// 全局异常处理器
@RestControllerAdvice
public class GlobalExceptionHandler {
    
    @ExceptionHandler(BusinessException.class)
    public ResponseEntity<Result> handleBusinessException(BusinessException e) {
        Result result = Result.error(e.getErrorCode(), e.getMessage());
        return ResponseEntity.ok(result);
    }
    
    @ExceptionHandler(Exception.class)
    public ResponseEntity<Result> handleException(Exception e) {
        log.error("系统异常", e);
        Result result = Result.error("SYSTEM_ERROR", "系统异常");
        return ResponseEntity.status(500).body(result);
    }
}
```

### 10.2 资源管理工具类

```java
public class ResourceUtils {
    
    public static void closeQuietly(Closeable closeable) {
        if (closeable != null) {
            try {
                closeable.close();
            } catch (IOException e) {
                // 静默关闭，记录日志
                log.warn("关闭资源失败", e);
            }
        }
    }
    
    public static void closeQuietly(Connection conn, Statement stmt, ResultSet rs) {
        closeQuietly(rs);
        closeQuietly(stmt);
        closeQuietly(conn);
    }
}
```

### 10.3 重试机制

```java
public class RetryUtils {
    
    public static <T> T executeWithRetry(Supplier<T> action, int maxRetries) {
        Exception lastException = null;
        
        for (int i = 0; i < maxRetries; i++) {
            try {
                return action.get();
            } catch (Exception e) {
                lastException = e;
                log.warn("第{}次重试失败", i + 1, e);
                
                // 等待一段时间后重试
                try {
                    Thread.sleep(1000 * (i + 1));  // 递增等待时间
                } catch (InterruptedException ie) {
                    Thread.currentThread().interrupt();
                    throw new RuntimeException("重试被中断", ie);
                }
            }
        }
        
        throw new RuntimeException("重试" + maxRetries + "次后仍然失败", lastException);
    }
}

// 使用
String result = RetryUtils.executeWithRetry(() -> {
    return httpClient.get(url);
}, 3);
```

## 十一、总结

### 异常处理原则

1. **尽早抛出**：在发现问题时立即抛出异常
2. **晚些捕获**：在合适的层级捕获和处理异常
3. **具体捕获**：捕获具体的异常，而不是通用的 Exception
4. **不要吞掉**：要么处理，要么记录，要么重新抛出
5. **清理资源**：使用 try-with-resources 或 finally 确保资源释放
6. **提供信息**：异常消息要有意义，便于排查问题
7. **保持链式**：包装异常时保留原始异常

### 常用异常处理模式

```java
// 模式1：捕获并处理
try {
    // 代码
} catch (SpecificException e) {
    handle(e);
}

// 模式2：捕获并重新抛出
try {
    // 代码
} catch (SpecificException e) {
    log.error("错误", e);
    throw new BusinessException("业务错误", e);
}

// 模式3：自动资源管理
try (Resource resource = new Resource()) {
    // 使用资源
}

// 模式4：声明抛出
public void method() throws SpecificException {
    // 代码
}
```

掌握异常处理是编写健壮 Java 程序的关键，建议在实际开发中遵循最佳实践，养成良好的异常处理习惯。
