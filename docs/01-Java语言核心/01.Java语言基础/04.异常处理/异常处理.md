# 异常处理

## 一、异常概述

### 1.1 什么是异常？

异常是程序运行过程中发生的非正常情况，它会中断程序的正常执行流程。

**示例：**
```java
int result = 10 / 0;  // ArithmeticException：除以零
String str = null;
str.length();          // NullPointerException：空指针
int[] arr = new int[5];
arr[10] = 1;           // ArrayIndexOutOfBoundsException：数组越界
```

### 1.2 为什么要处理异常？

- 提高程序的健壮性
- 提供更好的用户体验
- 便于问题定位和调试
- 防止程序崩溃

## 二、异常体系

### 2.1 异常类继承结构

```
Throwable（所有异常的父类）
├── Error（错误，无法恢复）
│   ├── OutOfMemoryError
│   ├── StackOverflowError
│   └── VirtualMachineError
│
└── Exception（异常，可以处理）
    ├── RuntimeException（运行时异常，非受检异常）
    │   ├── NullPointerException
    │   ├── ArrayIndexOutOfBoundsException
    │   ├── ClassCastException
    │   ├── IllegalArgumentException
    │   └── ArithmeticException
    │
    └── 其他 Exception（受检异常）
        ├── IOException
        ├── SQLException
        ├── ClassNotFoundException
        └── FileNotFoundException
```

### 2.2 Error vs Exception

| 对比项 | Error | Exception |
|--------|-------|-----------|
| 含义 | 严重错误 | 一般异常 |
| 可恢复 | 否 | 是 |
| 是否需要处理 | 否 | 是 |
| 示例 | OutOfMemoryError | NullPointerException |

### 2.3 受检异常 vs 非受检异常

| 对比项 | 受检异常 | 非受检异常 |
|--------|---------|-----------|
| 别名 | Checked Exception | Unchecked Exception |
| 继承自 | Exception | RuntimeException |
| 编译期检查 | 是 | 否 |
| 必须处理 | 是 | 否 |
| 示例 | IOException | NullPointerException |

## 三、异常处理机制

### 3.1 try-catch-finally

```java
try {
    // 可能抛出异常的代码
    int result = 10 / 0;
} catch (ArithmeticException e) {
    // 捕获并处理异常
    System.out.println("除数不能为零");
    e.printStackTrace();  // 打印异常堆栈信息
} finally {
    // 无论是否发生异常都会执行
    System.out.println("finally块");
}
```

**执行流程：**
1. 执行 try 块
2. 如果发生异常，跳转到匹配的 catch 块
3. 最后执行 finally 块

### 3.2 多个 catch 块

```java
try {
    // 可能抛出多种异常的代码
} catch (NullPointerException e) {
    System.out.println("空指针异常");
} catch (ArrayIndexOutOfBoundsException e) {
    System.out.println("数组越界异常");
} catch (Exception e) {
    System.out.println("其他异常");
}
```

**注意事项：**
- catch 块从上到下，范围从小到大
- 父类异常要放在子类异常后面

### 3.3 JDK 7+ 多异常捕获

```java
try {
    // 代码
} catch (NullPointerException | ArrayIndexOutOfBoundsException e) {
    System.out.println("捕获多种异常");
    e.printStackTrace();
}
```

### 3.4 finally 的作用

```java
InputStream is = null;
try {
    is = new FileInputStream("file.txt");
    // 读取文件
} catch (IOException e) {
    e.printStackTrace();
} finally {
    // 确保资源被释放
    if (is != null) {
        try {
            is.close();
        } catch (IOException e) {
            e.printStackTrace();
        }
    }
}
```

**finally 一定会执行吗？**
- ✅ 正常情况下会执行
- ❌ 如果 JVM 退出（System.exit()），不会执行
- ❌ 如果线程被强制终止，不会执行

## 四、throws 和 throw

### 4.1 throws（声明异常）

在方法签名中声明可能抛出的异常

```java
public void readFile() throws IOException {
    FileReader reader = new FileReader("file.txt");
    // 读取文件
}
```

**规则：**
- 受检异常必须声明或处理
- 非受检异常可以不声明
- 可以声明多个异常

```java
public void method() throws IOException, SQLException {
    // 代码
}
```

### 4.2 throw（抛出异常）

手动抛出异常对象

```java
public void setAge(int age) {
    if (age < 0 || age > 150) {
        throw new IllegalArgumentException("年龄不合法：" + age);
    }
    this.age = age;
}
```

### 4.3 throws vs throw

| 对比项 | throws | throw |
|--------|--------|-------|
| 位置 | 方法签名 | 方法体内 |
| 作用 | 声明异常 | 抛出异常 |
| 后跟 | 异常类名 | 异常对象 |
| 数量 | 可以多个 | 只能一个 |

## 六、异常处理最佳实践

### 6.1 不要吞掉异常

```java
// 错误做法
try {
    doSomething();
} catch (Exception e) {
    // 什么都不做，吞掉异常
}

// 正确做法
try {
    doSomething();
} catch (Exception e) {
    log.error("操作失败", e);  // 记录日志
    throw new BusinessException("操作失败", e);  // 或者重新抛出
}
```

### 6.2 不要捕获过于宽泛的异常

```java
// 错误做法
try {
    doSomething();
} catch (Exception e) {  // 太宽泛
    e.printStackTrace();
}

// 正确做法
try {
    doSomething();
} catch (IOException e) {  // 具体异常
    log.error("IO操作失败", e);
} catch (SQLException e) {
    log.error("数据库操作失败", e);
}
```

### 6.3 及时释放资源

#### 方式1：try-finally

```java
FileInputStream fis = null;
try {
    fis = new FileInputStream("file.txt");
    // 读取文件
} catch (IOException e) {
    e.printStackTrace();
} finally {
    if (fis != null) {
        try {
            fis.close();
        } catch (IOException e) {
            e.printStackTrace();
        }
    }
}
```

#### 方式2：try-with-resources（JDK 7+，推荐）

```java
// 自动关闭资源
try (FileInputStream fis = new FileInputStream("file.txt");
     BufferedReader br = new BufferedReader(new InputStreamReader(fis))) {
    
    String line;
    while ((line = br.readLine()) != null) {
        System.out.println(line);
    }
} catch (IOException e) {
    e.printStackTrace();
}
```

**要求：** 资源类必须实现 `AutoCloseable` 接口

### 6.4 异常转换

```java
public void loadData() throws BusinessException {
    try {
        // 数据库操作
        database.query();
    } catch (SQLException e) {
        // 将底层异常转换为业务异常
        throw new BusinessException("DATA_001", "数据加载失败", e);
    }
}
```

### 6.5 不要在循环中捕获异常

```java
// 低效做法
for (String item : items) {
    try {
        process(item);
    } catch (Exception e) {
        e.printStackTrace();
    }
}

// 高效做法
try {
    for (String item : items) {
        process(item);
    }
} catch (Exception e) {
    log.error("批量处理失败", e);
}
```

## 七、常见异常及解决方案

### 7.1 NullPointerException

```java
// 原因：对象为null时调用方法
String str = null;
str.length();  // NullPointerException

// 解决：
if (str != null) {
    System.out.println(str.length());
}

// 或使用Optional
Optional.ofNullable(str).ifPresent(s -> System.out.println(s.length()));
```

### 7.2 ArrayIndexOutOfBoundsException

```java
// 原因：访问数组越界
int[] arr = new int[5];
arr[10] = 1;  // ArrayIndexOutOfBoundsException

// 解决：检查索引范围
if (index >= 0 && index < arr.length) {
    arr[index] = 1;
}
```

### 7.3 ClassCastException

```java
// 原因：类型转换错误
Object obj = "Hello";
Integer num = (Integer)obj;  // ClassCastException

// 解决：先判断类型
if (obj instanceof Integer) {
    Integer num = (Integer)obj;
}
```

### 7.4 NumberFormatException

```java
// 原因：字符串格式不正确
int num = Integer.parseInt("abc");  // NumberFormatException

// 解决：验证格式或使用try-catch
try {
    int num = Integer.parseInt(str);
} catch (NumberFormatException e) {
    System.out.println("数字格式错误");
}
```

### 7.5 ConcurrentModificationException

```java
// 原因：遍历集合时修改集合
List<String> list = new ArrayList<>();
list.add("a");
list.add("b");

for (String item : list) {
    list.remove(item);  // ConcurrentModificationException
}

// 解决：使用Iterator
Iterator<String> iterator = list.iterator();
while (iterator.hasNext()) {
    String item = iterator.next();
    iterator.remove();  // 安全删除
}
```

## 八、异常链

### 8.1 什么是异常链？

保留原始异常信息，形成异常链条

```java
try {
    // 底层操作
    database.connect();
} catch (SQLException e) {
    // 包装为业务异常，保留原始异常
    throw new BusinessException("数据库连接失败", e);
}
```

### 8.2 查看异常链

```java
try {
    // 代码
} catch (Exception e) {
    e.printStackTrace();  // 打印完整异常链
    
    // 获取原始异常
    Throwable cause = e.getCause();
    while (cause != null) {
        System.out.println(cause.getMessage());
        cause = cause.getCause();
    }
}
```

### 10.3 重试机制

```java
public class RetryUtils {
    
    public static <T> T executeWithRetry(Supplier<T> action, int maxRetries) {
        Exception lastException = null;
        
        for (int i = 0; i < maxRetries; i++) {
            try {
                return action.get();
            } catch (Exception e) {
                lastException = e;
                log.warn("第{}次重试失败", i + 1, e);
                
                // 等待一段时间后重试
                try {
                    Thread.sleep(1000 * (i + 1));  // 递增等待时间
                } catch (InterruptedException ie) {
                    Thread.currentThread().interrupt();
                    throw new RuntimeException("重试被中断", ie);
                }
            }
        }
        
        throw new RuntimeException("重试" + maxRetries + "次后仍然失败", lastException);
    }
}

// 使用
String result = RetryUtils.executeWithRetry(() -> {
    return httpClient.get(url);
}, 3);
```

## 十二、高级主题

### 12.1 异常性能影响

频繁抛出异常会影响性能：

```java
// 不好：用异常控制流程
public int findIndex(int[] arr, int target) {
    try {
        for (int i = 0; ; i++) {
            if (arr[i] == target) {
                return i;
            }
        }
    } catch (ArrayIndexOutOfBoundsException e) {
        return -1;
    }
}

// 好：正常判断
public int findIndex(int[] arr, int target) {
    for (int i = 0; i < arr.length; i++) {
        if (arr[i] == target) {
            return i;
        }
    }
    return -1;
}
```

**建议：**
- 异常应该用于异常情况，而不是正常流程控制
- 避免在循环中抛出异常
- 预检查条件，避免不必要的异常

### 12.2 自定义异常层次设计

### 12.3 异常翻译模式

将底层异常转换为业务异常：

```java
public class UserRepository {
    
    public User findById(Long id) throws UserNotFoundException {
        try {
            // 数据库操作
            return jdbcTemplate.queryForObject(
                "SELECT * FROM users WHERE id = ?", 
                new Object[]{id}, 
                new UserRowMapper()
            );
        } catch (EmptyResultDataAccessException e) {
            // 翻译为业务异常

        } catch (DataAccessException e) {
            // 翻译为通用数据异常
            throw new DataAccessException("查询用户失败", e);
        }
    }
}
```

### 12.4 全局异常处理器（Spring Boot）

```java
@RestControllerAdvice
@Slf4j
public class GlobalExceptionHandler {
    
    /**
     * 处理业务异常
     */
    @ExceptionHandler(BusinessException.class)
    public ResponseEntity<ApiResponse<?>> handleBusinessException(
            BusinessException e) {
        log.warn("业务异常: {}", e.getMessage());
        ApiResponse<?> response = ApiResponse.error(
            e.getErrorCode(), 
            e.getMessage()
        );
        return ResponseEntity.badRequest().body(response);
    }
    
    /**
     * 处理参数校验异常
     */
    @ExceptionHandler(MethodArgumentNotValidException.class)
    public ResponseEntity<ApiResponse<?>> handleValidationException(
            MethodArgumentNotValidException e) {
        String message = e.getBindingResult().getFieldErrors().stream()
            .map(error -> error.getField() + ": " + error.getDefaultMessage())
            .collect(Collectors.joining(", "));
        
        ApiResponse<?> response = ApiResponse.error("VALIDATION_ERROR", message);
        return ResponseEntity.badRequest().body(response);
    }
    
    /**
     * 处理资源未找到
     */
    @ExceptionHandler(NotFoundException.class)
    public ResponseEntity<ApiResponse<?>> handleNotFoundException(
            NotFoundException e) {
        ApiResponse<?> response = ApiResponse.error("NOT_FOUND", e.getMessage());
        return ResponseEntity.status(HttpStatus.NOT_FOUND).body(response);
    }
    
    /**
     * 处理系统异常
     */
    @ExceptionHandler(Exception.class)
    public ResponseEntity<ApiResponse<?>> handleException(Exception e) {
        log.error("系统异常", e);
        ApiResponse<?> response = ApiResponse.error(
            "SYSTEM_ERROR", 
            "系统繁忙，请稍后重试"
        );
        return ResponseEntity.status(HttpStatus.INTERNAL_SERVER_ERROR)
            .body(response);
    }
}
```

## 十三、常见陷阱与最佳实践

### 13.1 常见陷阱

#### 陷阱1：吞掉异常

```java
// ❌ 错误做法
try {
    doSomething();
} catch (Exception e) {
    // 什么都不做
}

// ✅ 正确做法
try {
    doSomething();
} catch (Exception e) {
    log.error("操作失败", e);
    throw new BusinessException("操作失败", e);
}
```

#### 陷阱2：捕获过于宽泛

```java
// ❌ 错误做法
try {
    doSomething();
} catch (Exception e) {  // 太宽泛
    e.printStackTrace();
}

// ✅ 正确做法
try {
    doSomething();
} catch (IOException e) {  // 具体异常
    log.error("IO操作失败", e);
}
```

#### 陷阱3：丢失原始异常

```java
// ❌ 错误做法
try {
    doSomething();
} catch (SQLException e) {
    throw new BusinessException("数据库错误");  // 丢失了原始异常
}

// ✅ 正确做法
try {
    doSomething();
} catch (SQLException e) {
    throw new BusinessException("数据库错误", e);  // 保留原始异常
}
```

#### 陷阱4：在finally中返回

```java
// ❌ 错误做法
public int method() {
    try {
        return 1;
    } finally {
        return 2;  // 会覆盖try中的返回值
    }
}

// ✅ 正确做法
public int method() {
    try {
        return 1;
    } finally {
        // 清理资源，不要返回
    }
}
```

### 13.2 最佳实践总结

#### 设计原则

1. **尽早抛出**：发现问题立即抛出异常
2. **晚些捕获**：在合适的层级处理异常
3. **具体捕获**：捕获具体的异常类型
4. **保持链式**：包装异常时保留原始异常

#### 编码规范

1. **不要吞掉异常**：要么处理，要么记录，要么重新抛出
2. **使用try-with-resources**：自动管理资源
3. **提供有意义的消息**：便于问题排查
4. **区分业务异常和系统异常**：不同处理方式

#### 性能考虑

1. **避免用异常控制流程**：异常应该用于异常情况
2. **减少不必要的异常**：预检查条件
3. **合理使用日志级别**：error用于异常，warn用于警告

#### 安全考虑

1. **不要暴露敏感信息**：异常消息不要包含密码、密钥等
2. **生产环境隐藏堆栈**：只记录日志，不返回给用户
3. **统一异常响应格式**：便于前端处理

```java
@Service
@Slf4j
public class OrderService {
    
    @Autowired
    private OrderRepository orderRepository;
    
    @Autowired
    private PaymentService paymentService;
    
    /**
     * 创建订单
     */
    @Transactional(rollbackFor = Exception.class)
    public Order createOrder(OrderRequest request) 
            throws BusinessException {
        
        // 1. 参数校验
        validateOrderRequest(request);
        
        try {
            // 2. 创建订单
            Order order = buildOrder(request);
            orderRepository.save(order);
            
            // 3. 扣减库存
            inventoryService.deductStock(request.getItems());
            
            // 4. 支付
            paymentService.pay(order.getId(), request.getPaymentInfo());
            
            // 5. 更新订单状态
            order.setStatus(OrderStatus.PAID);
            orderRepository.update(order);
            
            log.info("订单创建成功: {}", order.getId());
            return order;
            
        } catch (InsufficientStockException e) {
            // 库存不足，回滚
            log.warn("库存不足: {}", e.getMessage());
            throw new BusinessException("STOCK_001", "库存不足");
            
        } catch (PaymentException e) {
            // 支付失败，回滚
            log.error("支付失败: {}", e.getMessage(), e);
            throw new BusinessException("PAYMENT_001", "支付失败");
            
        } catch (Exception e) {
            // 其他异常，回滚
            log.error("订单创建失败", e);
            throw new BusinessException("ORDER_001", "订单创建失败");
        }
    }
    
    private void validateOrderRequest(OrderRequest request) {
        if (request == null) {
            throw new IllegalArgumentException("订单请求不能为空");
        }
        if (request.getItems() == null || request.getItems().isEmpty()) {
            throw new IllegalArgumentException("订单项不能为空");
        }
        // 更多校验...
    }
    
    private Order buildOrder(OrderRequest request) {
        Order order = new Order();
        order.setUserId(request.getUserId());
        order.setItems(request.getItems());
        order.setTotalAmount(calculateTotal(request.getItems()));
        order.setStatus(OrderStatus.CREATED);
        order.setCreateTime(LocalDateTime.now());
        return order;
    }
}
```

### 14.2 异常监控与告警

```java
@Component
@Aspect
@Slf4j
public class ExceptionMonitorAspect {
    
    @Autowired
    private AlertService alertService;
    
    /**
     * 监控Controller层异常
     */
    @AfterThrowing(
        pointcut = "@within(org.springframework.web.bind.annotation.RestController)",
        throwing = "ex"
    )
    public void monitorControllerException(JoinPoint joinPoint, Throwable ex) {
        String className = joinPoint.getTarget().getClass().getName();
        String methodName = joinPoint.getSignature().getName();
        
        log.error("Controller异常: {}.{}", className, methodName, ex);
        
        // 发送告警
        if (shouldAlert(ex)) {
            alertService.sendAlert(
                String.format("[%s.%s] %s", className, methodName, ex.getMessage()),
                ex
            );
        }
    }
    
    private boolean shouldAlert(Throwable ex) {
        // 根据异常类型决定是否告警
        return !(ex instanceof BusinessException);
    }
}
```

## 十五、总结

### 核心要点回顾

1. **异常体系**
   - Throwable → Error / Exception
   - 受检异常 vs 非受检异常
   - 合理选择异常类型

2. **处理机制**
   - try-catch-finally
   - throws 声明
   - throw 抛出
   - try-with-resources

3. **最佳实践**
   - 不要吞掉异常
   - 捕获具体异常
   - 保留异常链
   - 提供有意义消息

4. **高级应用**
   - 自定义异常层次
   - 全局异常处理
   - 异常监控告警
   - 异步异常处理

### 异常处理流程图

```
异常发生
  ↓
是否可恢复？
  ├─ 是 → 捕获并处理 → 继续执行
  └─ 否 → 向上抛出
           ↓
      是否有上层处理？
        ├─ 是 → 上层处理
        └─ 否 → 全局异常处理器
                 ↓
              记录日志 + 返回错误响应
```

掌握异常处理是编写健壮 Java 程序的关键。遵循最佳实践，养成良好的异常处理习惯，能大大提高代码质量和系统稳定性。

**记住：**
- 异常是朋友，不是敌人
- 正确处理异常，让程序更健壮
- 记录异常信息，便于问题排查
- 用户体验优先，技术细节隐藏
