# 汇丰银行回单处理模块

## 一、功能概述

汇丰银行（HSBC）回单处理模块实现了批量回单文件的自动拆分、信息提取和存储功能。系统接收一个包含多页回单的 PDF 文件，自动将每一页拆分为独立的回单文件，从页面中提取贷记账号和金额作为标识，并上传到 OBS 对象存储。

### 核心功能
- ✅ 接收批量回单 PDF 文件（格式：交易银行账号_日期_批次号.pdf）
- ✅ 自动拆分多页 PDF 为单页文件
- ✅ 从每页提取贷记账号和金额
- ✅ 生成唯一的流水号标识
- ✅ 并发上传到华为云 OBS
- ✅ 防重复处理机制
- ✅ 完善的异常处理和状态追踪

### 2.3 并发设计

**双线程池架构**：

1. **pdfSplitExecutor（PDF 拆分线程池）**
   - 核心线程数：10
   - 最大线程数：20
   - 队列容量：200
   - 用途：处理 PDF 拆分任务（IO 密集型）

2. **receiptTaskExecutor（回单处理线程池）**
   - 核心线程数：5
   - 最大线程数：10
   - 队列容量：100
   - 用途：处理单个页面的上传和入库

**为什么需要两个线程池？**
1. **职责分离**：PDF 拆分和回单处理是不同的 IO 操作
2. **资源隔离**：避免大量 PDF 拆分占用所有线程
3. **性能优化**：针对不同任务类型优化线程池参数
4. **优先级控制**：可以独立调整不同任务的优先级

### 3.1 表结构

```sql
-- 创建数据库
CREATE DATABASE IF NOT EXISTS dt_finance DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_general_ci;

USE dt_finance;

-- 流水号回单文件关系表
CREATE TABLE IF NOT EXISTS `receipt_relation` (
    `id` BIGINT NOT NULL AUTO_INCREMENT COMMENT '主键ID',
    `serial_number` VARCHAR(100) NOT NULL COMMENT '流水号（贷记账号_金额）',
    `channel` VARCHAR(20) NOT NULL COMMENT '回单渠道: PAP-平安付, PAB-平安银行, HSBC-汇丰银行',
    `document_id` VARCHAR(200) NOT NULL COMMENT 'OBS文档ID',
    `file_name` VARCHAR(200) DEFAULT NULL COMMENT '文件名',
    `file_size` BIGINT DEFAULT NULL COMMENT '文件大小(字节)',
    `status` TINYINT NOT NULL DEFAULT 0 COMMENT '状态: 0-待处理, 1-处理中, 2-成功, 3-失败, 4-已经关联回单文件',
    `error_msg` TEXT COMMENT '错误信息',
    `create_time` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    `update_time` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
    PRIMARY KEY (`id`),
    UNIQUE KEY `uk_serial_channel` (`serial_number`, `channel`),
    KEY `idx_channel` (`channel`),
    KEY `idx_status` (`status`),
    KEY `idx_create_time` (`create_time`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_general_ci COMMENT='流水号回单文件关系表';
```

### 3.2 流水号生成规则

**汇丰银行流水号格式**：`贷记账号_金额`

示例：
- 贷记账号：6222021234567890
- 金额：1000.50
- 流水号：`6222021234567890_1000.50`

**如果无法提取字段**：使用 `批次文件名_时间戳` 作为备选方案

### 3.3 索引说明

- **唯一索引 `uk_serial_channel`**：防止同一回单重复处理
- **普通索引 `idx_channel`**：按渠道查询优化
- **普通索引 `idx_status`**：按状态查询优化
- **普通索引 `idx_create_time`**：按时间排序优化

```java
package com.dt.finance.config;

import lombok.extern.slf4j.Slf4j;
import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;
import org.springframework.scheduling.annotation.EnableAsync;
import org.springframework.scheduling.concurrent.ThreadPoolTaskExecutor;

import java.util.concurrent.Executor;
import java.util.concurrent.ThreadPoolExecutor;

/**
 * 线程池配置
 */
@Slf4j
@Configuration
@EnableAsync
public class ThreadPoolConfig {

    /**
     * 回单处理线程池
     * 用于并发处理回单下载、上传等操作
     */
    @Bean("receiptTaskExecutor")
    public Executor receiptTaskExecutor() {
        ThreadPoolTaskExecutor executor = new ThreadPoolTaskExecutor();
        
        // 核心线程数
        executor.setCorePoolSize(5);
        // 最大线程数
        executor.setMaxPoolSize(10);
        // 队列容量
        executor.setQueueCapacity(100);
        // 线程空闲时间（秒）
        executor.setKeepAliveSeconds(60);
        // 线程名称前缀
        executor.setThreadNamePrefix("receipt-task-");
        // 拒绝策略：由调用线程处理
        executor.setRejectedExecutionHandler(new ThreadPoolExecutor.CallerRunsPolicy());
        // 等待所有任务结束后再关闭线程池
        executor.setWaitForTasksToCompleteOnShutdown(true);
        // 等待时间（秒）
        executor.setAwaitTerminationSeconds(60);
        
        executor.initialize();
        log.info("回单处理线程池初始化完成");
        
        return executor;
    }

    /**
     * PDF 拆分线程池
     * 专门用于汇丰银行批量回单拆分，IO 密集型任务
     */
    @Bean("pdfSplitExecutor")
    public Executor pdfSplitExecutor() {
        ThreadPoolTaskExecutor executor = new ThreadPoolTaskExecutor();
        
        // IO 密集型任务，设置较多线程
        executor.setCorePoolSize(10);
        executor.setMaxPoolSize(20);
        executor.setQueueCapacity(200);
        executor.setKeepAliveSeconds(60);
        executor.setThreadNamePrefix("pdf-split-");
        executor.setRejectedExecutionHandler(new ThreadPoolExecutor.CallerRunsPolicy());
        executor.setWaitForTasksToCompleteOnShutdown(true);
        executor.setAwaitTerminationSeconds(60);
        
        executor.initialize();
        log.info("PDF 拆分线程池初始化完成");
        
        return executor;
    }
}
```

```java
package com.dt.finance.common.util;

import lombok.extern.slf4j.Slf4j;
import org.apache.pdfbox.Loader;
import org.apache.pdfbox.pdmodel.PDDocument;
import org.apache.pdfbox.pdmodel.PDPage;
import org.apache.pdfbox.text.PDFTextStripper;

import java.io.ByteArrayOutputStream;
import java.util.ArrayList;
import java.util.List;

/**
 * PDF 工具类
 */
@Slf4j
public class PdfUtil {

    /**
     * 提取 PDF 文本内容
     */
    public static String extractText(byte[] pdfContent) throws Exception {
        try (PDDocument document = Loader.loadPDF(pdfContent)) {
            PDFTextStripper stripper = new PDFTextStripper();
            return stripper.getText(document);
        }
    }

    /**
     * 提取指定页的文本
     */
    public static String extractPageText(byte[] pdfContent, int pageNumber) throws Exception {
        try (PDDocument document = Loader.loadPDF(pdfContent)) {
            PDFTextStripper stripper = new PDFTextStripper();
            stripper.setStartPage(pageNumber);
            stripper.setEndPage(pageNumber);
            return stripper.getText(document);
        }
    }

    /**
     * 获取 PDF 页数
     */
    public static int getPageCount(byte[] pdfContent) throws Exception {
        try (PDDocument document = Loader.loadPDF(pdfContent)) {
            return document.getNumberOfPages();
        }
    }

    /**
     * 拆分 PDF，将每一页拆分为单独的 PDF
     *
     * @param pdfContent PDF 文件内容
     * @return 每页的 PDF 内容列表
     */
    public static List<byte[]> splitPdf(byte[] pdfContent) throws Exception {
        List<byte[]> pages = new ArrayList<>();
        
        try (PDDocument document = Loader.loadPDF(pdfContent)) {
            int pageCount = document.getNumberOfPages();
            log.info("PDF 总页数: {}", pageCount);

            for (int i = 0; i < pageCount; i++) {
                try (PDDocument singlePageDoc = new PDDocument()) {
                    PDPage page = document.getPage(i);
                    singlePageDoc.addPage(page);

                    ByteArrayOutputStream outputStream = new ByteArrayOutputStream();
                    singlePageDoc.save(outputStream);
                    pages.add(outputStream.toByteArray());
                }
            }
        }

        log.info("PDF 拆分完成，共 {} 页", pages.size());
        return pages;
    }

    /**
     * 从 PDF 文本中提取指定字段（使用正则表达式）
     *
     * @param text      PDF 文本内容
     * @param fieldName 字段名称
     * @return 字段值
     */
    public static String extractField(String text, String fieldName) {
        // 这里需要根据实际 PDF 格式调整正则表达式
        // 示例：查找 "贷记账号: 123456" 或 "贷记账号 123456"
        String regex = fieldName + "[\\s:]*([\\d]+)";
        java.util.regex.Pattern pattern = java.util.regex.Pattern.compile(regex);
        java.util.regex.Matcher matcher = pattern.matcher(text);
        
        if (matcher.find()) {
            return matcher.group(1);
        }
        return null;
    }

    /**
     * 从 PDF 文本中提取金额
     */
    public static String extractAmount(String text) {
        // 根据实际 PDF 格式调整正则表达式
        String regex = "[\\d,]+\\.\\d{2}";
        java.util.regex.Pattern pattern = java.util.regex.Pattern.compile(regex);
        java.util.regex.Matcher matcher = pattern.matcher(text);
        
        if (matcher.find()) {
            return matcher.group();
        }
        return null;
    }
}
```

```java
package com.dt.finance.common.util;

import com.obs.services.ObsClient;
import com.obs.services.model.PutObjectRequest;
import com.obs.services.model.PutObjectResult;
import lombok.extern.slf4j.Slf4j;

import java.io.ByteArrayInputStream;

/**
 * OBS 工具类（华为云对象存储）
 */
@Slf4j
public class ObsUtil {

    private ObsClient obsClient;
    private String bucketName;

    /**
     * 初始化 OBS 客户端
     */
    public void init(String endPoint, String ak, String sk, String bucketName) {
        this.obsClient = new ObsClient(ak, sk, endPoint);
        this.bucketName = bucketName;
        log.info("OBS 客户端初始化成功");
    }

    /**
     * 上传文件到 OBS
     *
     * @param fileName 文件名
     * @param content  文件内容
     * @return documentId (objectKey)
     */
    public String uploadFile(String fileName, byte[] content) {
        try {
            PutObjectRequest request = new PutObjectRequest(bucketName, fileName, new ByteArrayInputStream(content));
            PutObjectResult result = obsClient.putObject(request);
            
            log.info("文件上传成功: {}, ETag: {}", fileName, result.getEtag());
            return fileName; // 返回 objectKey 作为 documentId
        } catch (Exception e) {
            log.error("文件上传失败: {}", fileName, e);
            throw new RuntimeException("文件上传失败", e);
        }
    }

    /**
```

### 4.8 Service 层

**文件位置**：`src/main/java/com/dt/finance/service/receipt/HsbcReceiptService.java`

```java
package com.dt.finance.service.receipt;

import com.dt.finance.common.util.ObsUtil;
import com.dt.finance.common.util.PdfUtil;
import com.dt.finance.entity.ReceiptRelation;
import com.dt.finance.enums.ReceiptChannel;
import com.dt.finance.enums.ReceiptStatus;
import com.dt.finance.mapper.ReceiptRelationMapper;
import lombok.extern.slf4j.Slf4j;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.scheduling.annotation.Async;
import org.springframework.stereotype.Service;

import java.time.LocalDateTime;
import java.util.List;
import java.util.UUID;
import java.util.concurrent.CompletableFuture;
import java.util.stream.Collectors;

/**
 * 汇丰银行回单处理服务
 */
@Slf4j
@Service
public class HsbcReceiptService {
    
    @Autowired
    private ReceiptRelationMapper receiptRelationMapper;
    
    @Autowired
    private ObsUtil obsUtil;
    
    /**
     * 处理汇丰银行批量回单文件
     *
     * @param batchFileName 批量文件名格式: 交易银行账号_日期_批次号.pdf
     * @param pdfContent    PDF 文件内容
     */
    @Async("pdfSplitExecutor")
    public void processBatchReceipt(String batchFileName, byte[] pdfContent) {
        log.info("开始处理汇丰银行批量回单，文件: {}", batchFileName);
        
        try {
            // 1. 拆分 PDF
            List<byte[]> pages = PdfUtil.splitPdf(pdfContent);
            log.info("PDF 拆分为 {} 页", pages.size());
            
            // 2. 并发处理每一页
            List<CompletableFuture<Void>> futures = pages.stream()
                .map(pageContent -> processSinglePage(batchFileName, pageContent))
                .collect(Collectors.toList());
            
            // 3. 等待所有页面处理完成
            CompletableFuture.allOf(futures.toArray(new CompletableFuture[0])).join();
            
            log.info("汇丰银行批量回单处理完成，文件: {}", batchFileName);
            
        } catch (Exception e) {
            log.error("汇丰银行批量回单处理失败，文件: {}", batchFileName, e);
        }
    }
    
    /**
     * 处理单个页面（异步）
     */
    @Async("receiptTaskExecutor")
    private CompletableFuture<Void> processSinglePage(String batchFileName, byte[] pageContent) {
        return CompletableFuture.runAsync(() -> {
            ReceiptRelation relation = new ReceiptRelation();
            relation.setChannel(ReceiptChannel.HSBC.getCode());
            relation.setStatus(ReceiptStatus.PROCESSING.getCode());
            relation.setCreateTime(LocalDateTime.now());
            relation.setUpdateTime(LocalDateTime.now());
            
            try {
                // 1. 提取文本信息
                String text = PdfUtil.extractText(pageContent);
                
                // 2. 从 PDF 中提取贷记账号和金额作为流水号
                String accountNo = PdfUtil.extractField(text, "贷记账号");
                String amount = PdfUtil.extractAmount(text);
                
                // 如果无法提取，使用批次号+页码作为标识
                String serialNumber = (accountNo != null && amount != null) 
                    ? accountNo + "_" + amount 
                    : batchFileName + "_" + System.currentTimeMillis();
                
                relation.setSerialNumber(serialNumber);
                
                // 3. 检查是否已存在
                ReceiptRelation existing = receiptRelationMapper.findBySerialAndChannel(
                    serialNumber, ReceiptChannel.HSBC.getCode());
                if (existing != null) {
                    log.warn("回单已存在，跳过处理: {}", serialNumber);
                    return;
                }
                
                // 4. 插入记录
                receiptRelationMapper.insert(relation);
                
                // 5. 上传到 OBS
                String fileName = serialNumber + "_" + UUID.randomUUID().toString().substring(0, 8) + ".pdf";
                String documentId = obsUtil.uploadFile(fileName, pageContent);
                
                // 6. 更新记录
                relation.setDocumentId(documentId);
                relation.setFileName(fileName);
                relation.setFileSize((long) pageContent.length);
                relation.setStatus(ReceiptStatus.SUCCESS.getCode());
                receiptRelationMapper.updateStatus(relation);
                
                log.info("汇丰回单页面处理成功: {}", serialNumber);
                
            } catch (Exception e) {
                log.error("汇丰回单页面处理失败", e);
                relation.setStatus(ReceiptStatus.FAILED.getCode());
                relation.setErrorMsg(e.getMessage());
                receiptRelationMapper.updateStatus(relation);
            }
        });
    }
}
```

```java
package com.dt.finance.controller;

@Slf4j
@RestController
@RequestMapping("/api/receipts")

    private HsbcReceiptService hsbcReceiptService;
    
    @Autowired
    private ReceiptRelationMapper receiptRelationMapper;
    
    /**
     * 上传汇丰银行批量回单文件
     */
    @PostMapping("/hsbc/upload")
    public Map<String, Object> uploadHsbcBatchReceipt(@RequestParam("file") MultipartFile file) {
        Map<String, Object> result = new HashMap<>();
        try {
            byte[] content = file.getBytes();
            String fileName = file.getOriginalFilename();
            
            hsbcReceiptService.processBatchReceipt(fileName, content);

        }

    }

     * 查询回单列表
     */
    @GetMapping("/list")
    public Map<String, Object> listReceipts(@RequestParam(required = false) String channel,
                                            @RequestParam(defaultValue = "100") int limit) {
        Map<String, Object> result = new HashMap<>();
        try {
            List<ReceiptRelation> receipts;
            if (channel != null && !channel.isEmpty()) {
                receipts = receiptRelationMapper.findByChannel(channel, limit);
            } else {
                receipts = receiptRelationMapper.findByChannel("HSBC", limit);
            }
            
            result.put("code", 200);
            result.put("data", receipts);
        } catch (Exception e) {
            log.error("查询回单列表失败", e);
            result.put("code", 500);
            result.put("message", "查询失败: " + e.getMessage());
        }

    @GetMapping("/{serialNumber}")

                                          @RequestParam String channel) {
        Map<String, Object> result = new HashMap<>();

            result.put("code", 500);
            result.put("message", "查询失败: " + e.getMessage());
        }
        return result;
    }
```

## 六、PDF 文本提取规则定制

### 6.1 调整正则表达式

由于不同银行的回单 PDF 格式不同，需要根据实际情况调整 `PdfUtil.java` 中的正则表达式。

**示例 1：提取贷记账号**

如果 PDF 中格式为：
```
贷记账号：6222021234567890
```

正则表达式：
```java
String regex = "贷记账号[\\s:：]*([\\d]+)";
```

**示例 2：提取金额**

如果 PDF 中格式为：
```
金额：1,000.50
```

正则表达式：
```java
String regex = "金额[\\s:：]*([\\d,]+\\.\\d{2})";
```

### 6.2 调试技巧

1. **打印提取的文本**：
```java
String text = PdfUtil.extractText(pageContent);
log.debug("PDF 文本内容:\n{}", text);
```

2. **测试正则表达式**：
```java
Pattern pattern = Pattern.compile("你的正则表达式");
Matcher matcher = pattern.matcher(text);
if (matcher.find()) {
    log.info("匹配成功: {}", matcher.group(1));
}
```

3. **使用在线工具**：
   - [Regex101](https://regex101.com/) - 在线正则测试
   - [PDFBox Debugger](https://pdfbox.apache.org/) - PDF 结构分析

## 七、注意事项

### 7.1 并发安全

1. **唯一索引保护**：数据库唯一索引 `(serial_number, channel)` 防止重复处理
2. **先查后插**：处理前先检查是否已存在
3. **线程池隔离**：PDF 拆分和回单处理使用不同线程池

### 7.2 性能优化

1. **并行处理**：使用 `CompletableFuture` 并发处理每一页
2. **双线程池**：拆分和处理分离，提高吞吐量
3. **流式处理**：使用 Stream API 简化代码

### 7.3 异常处理

1. **单页失败不影响其他页**：每个页面独立处理
2. **详细错误日志**：记录每页的处理结果
3. **状态追踪**：失败记录保存到数据库

### 7.4 内存管理

1. **大文件处理**：如果 PDF 文件很大（几百页），建议分批处理
2. **及时释放资源**：使用 try-with-resources 自动关闭文档
3. **监控内存使用**：观察 JVM 内存使用情况

## 八、常见问题

### Q1: PDF 文本提取不准确怎么办？

**A**: 
1. 检查 PDF 是否是扫描件（图片型 PDF 需要 OCR）
2. 调整正则表达式匹配实际格式
3. 打印原始文本查看实际内容
4. 考虑使用更强大的 PDF 解析库（如 Adobe SDK）

### Q2: 如何处理超大批量文件（1000+ 页）？

**A**:
1. 增加线程池大小
2. 分批处理（每 100 页一批）
3. 使用消息队列异步处理
4. 监控内存使用，避免 OOM

### Q3: 如何验证拆分后的 PDF 是否正确？

**A**:
```java
// 验证拆分后的 PDF
for (int i = 0; i < pages.size(); i++) {

}
```

2. 实现特定的文本提取逻辑
3. 在 `ReceiptChannel` 枚举中添加新渠道
4. 添加对应的 API 接口

### Q5: 如何提高 PDF 处理速度？

**A**:
1. 使用更快的 PDF 库（如 PDFium）

3. 使用 SSD 存储临时文件
4. 优化正则表达式性能
