## 一、多线程基础概念



### 1. 线程与进程

在操作系统中，**进程**是资源分配的最小单位，而**线程**是程序执行的最小单位，也是CPU调度的基本单元。每个进程都有自己独立的内存空间，而同一进程内的多个线程则共享该进程的内存空间。

- 进程：指在系统中正在运行的一个应用程序，程序一旦运行就是进程。进程是资源分配的最小单位。
- 线程：系统分配处理器时间资源的基本单元，或者说进程之内独立执行的一个单元执行流。

Java的多线程编程模型允许一个程序同时执行多个线程，以提高程序的执行效率。线程比进程更轻量，创建和销毁线程的开销远小于进程，且线程间通信更为高效。

### 2. 并发与并行

**并发**和**并行**是多线程编程中的两个核心概念：

- 并发(Concurrency)：指在同一时间段内多个任务交替执行，从宏观上看像是同时进行，但实际上是通过时间片轮转实现的。
- 并行(Parallelism)：指多个任务真正同时执行，这需要多核处理器的支持，每个核心独立执行一个任务。

Java通过多线程实现了并发编程，而在多核处理器上，Java线程可以被分配到不同的核心上真正并行执行。

### 3. 为什么需要多线程

使用多线程主要基于以下几个原因：

1. **提高程序运行效率**：通过利用多核CPU资源，同时执行多个任务，缩短程序总执行时间。
2. **提高系统资源利用率**：当某一线程因I/O操作阻塞时，其他线程可以继续执行，避免CPU空闲。
3. **增强程序响应性**：在GUI应用中，使用多线程可以避免主线程被长时间任务阻塞，保持界面响应。
4. **更灵活的编程模型**：多线程允许以更自然的方式建模某些问题，如服务器同时处理多个客户端请求。

### 4.守护线程

“守护线程”（Daemon Thread）是多线程编程中的一个重要概念，主要出现在Java、Python等支持多线程的语言中。它的核心作用是**为其他线程（尤其是主线程）提供后台支持服务**，自身不直接影响程序的“存活状态”。

守护线程是一种**低优先级的后台线程**，其存在意义是为用户线程（非守护线程，如主线程或业务线程）提供服务。当所有用户线程执行完毕后，即使守护线程仍在运行，JVM或解释器也会直接终止程序，**不会等待守护线程完成任务**。

**关键特性**

- 依赖用户线程：守护线程的生命周期依赖于用户线程。若所有的用户线程结束，守护线程会被强制终止（无论是否执行完毕）
- 优先级低：守护线程的调度优先级通常低于用户线程，系统资源紧张时可能优先保证用户线程执行
- 无法阻止程序退出：即使守护线程未完成任务（如循环中），只要用户线程全部结束，程序会立即退出，不会抛异常或等待。

**典型应用场景**

守护线程适合执行辅助性、非关键性的后台任务，常见场景包括：

- 垃圾回收（GC）：Java的GC线程是典型的守护线程，负责自动回收内存，无需用户线程干预，用户线程结束后GC也无需继续工作
- 心跳检测：监控其他线程或服务的存活状态(如分布式系统中的节点探活)，若主业务线程结束，心跳检测失去意义
- 日志记录/统计：收集运行时日志或性能数据，若主程序已退出，这些数据的保存不再必要
- 缓存清理：定期清理临时缓存（如过期会话），但主业务停止后缓存清理无需继续

**Java 中的守护线程**

- **设置方式**：通过 `Thread.setDaemon(true)` 标记线程为守护线程（需在 `start()` 前调用，否则抛 `IllegalThreadStateException`）。
- **默认规则**：主线程（main 线程）是用户线程；通过 `new Thread()` 创建的线程默认继承父线程的守护属性（若父线程是守护线程，子线程也是）。

```java
public class DaemonDemo {
    public static void main(String[] args) {
        Thread daemonThread = new Thread(() -> {
            while (true) { // 无限循环
                System.out.println("守护线程运行中...");
                try { Thread.sleep(1000); } catch (InterruptedException e) {}
            }
        });
        daemonThread.setDaemon(true); // 标记为守护线程
        daemonThread.start();

        // 主线程（用户线程）执行3秒后结束
        try { Thread.sleep(3000); } catch (InterruptedException e) {}
        System.out.println("主线程结束");
        // 主线程结束后，守护线程被强制终止，不会继续打印
    }
}
```

## 二、线程的创建与管理

### 1. 线程创建方式

Java提供了三种创建线程的基本方式：

#### 1.1 继承Thread类

```java
public class MyThread extends Thread {
    @Override
    public void run() {
        // 线程执行的代码
        for (int i = 0; i < 10; i++) {
            System.out.println("Thread: " + i);
            try {
                Thread.sleep(100);
            } catch (InterruptedException e) {
                e.printStackTrace();
            }
        }
    }
    
    public static void main(String[] args) {
        MyThread thread = new MyThread();
        thread.start(); // 启动线程
    }
}
```

**优点**：代码简单直接。 **缺点**：由于Java是单继承，继承Thread类后不能再继承其他类。

#### 1.2 实现Runnable接口

```java
public class MyRunnable implements Runnable {
    @Override
    public void run() {
        // 线程执行的代码
        for (int i = 0; i < 10; i++) {
            System.out.println("Runnable: " + i);
            try {
                Thread.sleep(100);
            } catch (InterruptedException e) {
                e.printStackTrace();
            }
        }
    }
    
    public static void main(String[] args) {
        Thread thread = new Thread(new MyRunnable());
        thread.start(); // 启动线程
    }
}
```

**优点**：可以继承其他类，适合多线程共享资源。 **缺点**：无法直接返回执行结果。

#### 1.3 使用Callable和Future

```java
import java.util.concurrent.Callable;
import java.util.concurrent.ExecutionException;
import java.util.concurrent.FutureTask;

public class MyCallable implements Callable<Integer> {
    @Override
    public Integer call() throws Exception {
        int sum = 0;
        for (int i = 0; i <= 100; i++) {
            sum += i;
        }
        return sum;
    }
    
    public static void main(String[] args) {
        MyCallable callable = new MyCallable();
        FutureTask<Integer> futureTask = new FutureTask<>(callable);
        Thread thread = new Thread(futureTask);
        thread.start();
        
        try {
            // 获取线程执行结果
            Integer result = futureTask.get();
            System.out.println("1到100的和为: " + result);
        } catch (InterruptedException | ExecutionException e) {
            e.printStackTrace();
        }
    }
}
```

**优点**：可以返回执行结果，可以抛出异常。 **缺点**：代码相对复杂。

### 2. 线程生命周期

> 参考：https://www.cnblogs.com/vipstone/p/15907280.html

Java线程在其生命周期中会经历多种状态，Java 5之后线程状态被明确定义在Thread.State枚举中：

1. **NEW(新建)**：线程对象创建后，但尚未调用start()方法。
2. **RUNNABLE(可运行)**：调用start()方法后，线程等待CPU时间片的状态，包括就绪和运行中两种情况。
3. **BLOCKED(阻塞)**：线程因为等待锁而进入的状态，当锁可用时重新进入RUNNABLE状态。
4. **WAITING(等待)**：线程无限期等待其他线程执行特定操作，通过Object.wait()、Thread.join()或LockSupport.park()进入。
5. **TIMED_WAITING(定时等待)**：与WAITING类似，但只等待指定的时间。
6. **TERMINATED(终止)**：线程执行完毕或异常终止后的状态。

### 3. 线程控制方法

Java提供了一系列控制线程行为的方法：

- **start()**：启动线程，使其进入就绪状态。
- **run()**：线程执行的核心方法，通常需要重写。
- **sleep(long millis)**：让当前线程休眠指定毫秒数，不释放锁。
- **join()**：等待该线程执行完毕。
- **yield()**：提示调度器当前线程愿意让出CPU，但实际效果取决于JVM实现。
- **interrupt()**：中断线程，设置中断标志位。
- **setPriority(int)**：设置线程优先级(1-10)，但实际优先级取决于操作系统。
- **isAlive()**：判断线程是否仍在运行。

## 三、线程同步与并发控制

当多个线程访问共享资源时，可能会出现**竞态条件**(Race Condition)，导致数据不一致等问题。Java提供了多种机制来解决线程安全问题

### 1. synchronized关键字

`synchronized`是Java中最基本的同步机制，它可以用于方法或代码块：

```java
// 同步方法
public synchronized void method() {
    // 需要同步的代码
}

// 同步代码块
public void method() {
    synchronized (this) {
        // 需要同步的代码
    }
}
```

`synchronized`保证了：

1. **互斥性**：同一时刻只有一个线程能执行该代码块。
2. **可见性**：线程退出同步块时，对共享变量的修改会立即刷新到主内存。
3. **可重入性**：同一个线程可以多次获取同一个锁。

### 2. Lock接口与ReentrantLock

Java 5引入了更灵活的锁机制——`Lock`接口及其实现类`ReentrantLock`：

```java
import java.util.concurrent.locks.Lock;
import java.util.concurrent.locks.ReentrantLock;

public class LockExample {
    private Lock lock = new ReentrantLock();
    
    public void method() {
        lock.lock(); // 获取锁
        try {
            // 需要同步的代码
        } finally {
            lock.unlock(); // 释放锁
        }
    }
}
```

相比`synchronized`，`ReentrantLock`提供了更多高级功能：

- 尝试非阻塞获取锁：`tryLock()`
- 可中断的获取锁：`lockInterruptibly()`
- 公平锁与非公平锁
- 多个条件变量(Condition)

### 3. volatile关键字

`volatile`是一种轻量级的同步机制，它确保变量的**可见性**（一个线程的修改对其他线程立即可见），但不保证**原子性**：

```java
volatile boolean flag = false;
```

适用场景：

- 对变量的写入操作不依赖于当前值
- 变量不参与其他变量的不变式约束
- 访问变量时不需要加锁

### 4. 原子变量

Java的`java.util.concurrent.atomic`包提供了一系列原子变量类，如`AtomicInteger`、`AtomicLong`等，它们使用CAS(Compare-And-Swap)操作保证单个变量的原子性：

```java
import java.util.concurrent.atomic.AtomicInteger;

public class Counter {
    private AtomicInteger count = new AtomicInteger(0);
    
    public void increment() {
        count.incrementAndGet();
    }
    
    public int getCount() {
        return count.get();
    }
}
```

### 5. 线程通信

Java提供了`wait()`、`notify()`和`notifyAll()`方法实现线程间通信，这些方法必须在同步块或同步方法中使用：

```java
public class SharedObject {
    private boolean available = false;
    
    public synchronized void produce() {
        while (available) {
            try {
                wait(); // 等待消费者消费
            } catch (InterruptedException e) {
                Thread.currentThread().interrupt();
            }
        }
        available = true;
        notifyAll(); // 通知消费者
    }
    
    public synchronized void consume() {
        while (!available) {
            try {
                wait(); // 等待生产者生产
            } catch (InterruptedException e) {
                Thread.currentThread().interrupt();
            }
        }
        available = false;
        notifyAll(); // 通知生产者
    }
}
```



## 四、高级并发工具类

Java 5引入的`java.util.concurrent`包提供了一系列高级并发工具，大大简化了并发编程的复杂度。

### 1. CountDownLatch

允许一个或多个线程等待其他线程完成操作：

```java
import java.util.concurrent.CountDownLatch;

public class Example {
    public static void main(String[] args) throws InterruptedException {
        CountDownLatch latch = new CountDownLatch(3);
        
        new Thread(() -> {
            // do something
            latch.countDown();
        }).start();
        
        new Thread(() -> {
            // do something
            latch.countDown();
        }).start();
        
        new Thread(() -> {
            // do something
            latch.countDown();
        }).start();
        
        latch.await(); // 等待所有线程完成
        System.out.println("All threads completed");
    }
}
```

### 2. CyclicBarrier

让一组线程到达一个屏障时被阻塞，直到最后一个线程到达屏障时，屏障才会打开：

```java
import java.util.concurrent.CyclicBarrier;

public class Example {
    public static void main(String[] args) {
        CyclicBarrier barrier = new CyclicBarrier(3, () -> {
            System.out.println("All threads reached the barrier");
        });
        
        for (int i = 0; i < 3; i++) {
            new Thread(() -> {
                try {
                    // do something
                    barrier.await();
                } catch (Exception e) {
                    e.printStackTrace();
                }
            }).start();
        }
    }
}
```

### 3. Semaphore

控制同时访问特定资源的线程数量：

```java
import java.util.concurrent.Semaphore;

public class Example {
    public static void main(String[] args) {
        Semaphore semaphore = new Semaphore(3); // 允许3个线程同时访问
        
        for (int i = 0; i < 10; i++) {
            new Thread(() -> {
                try {
                    semaphore.acquire();
                    // 访问受限资源
                    Thread.sleep(1000);
                    semaphore.release();
                } catch (InterruptedException e) {
                    e.printStackTrace();
                }
            }).start();
        }
    }
}
```

### 4. Exchanger

用于两个线程间交换数据：

```java
import java.util.concurrent.Exchanger;

public class Example {
    public static void main(String[] args) {
        Exchanger<String> exchanger = new Exchanger<>();
        
        new Thread(() -> {
            try {
                String data = "Data from Thread A";
                String received = exchanger.exchange(data);
                System.out.println("Thread A received: " + received);
            } catch (InterruptedException e) {
                e.printStackTrace();
            }
        }).start();
        
        new Thread(() -> {
            try {
                String data = "Data from Thread B";
                String received = exchanger.exchange(data);
                System.out.println("Thread B received: " + received);
            } catch (InterruptedException e) {
                e.printStackTrace();
            }
        }).start();
    }
}
```

## 线程池

### 基础概念

#### 什么是线程池？

线程池是一种多线程处理形式，它维护了一个线程集合，用于执行提交的任务。线程池中的线程在初始化时创建，并在需要时重复使用，而不是为每个任务都新建一个线程。

简单来说，线程池就是预先创建一组线程，统一管理任务的执行，避免频繁创建和销毁线程的开销。

#### 为什么需要线程池？

1. 减少资源消耗
   - 每次创建新线程都需要分配内存、栈空间等系统资源，销毁时也要回收
   - 线程池复用已有线程，避免了频繁的线程创建于销毁，降低系统开销
2. 提高响应速度
   - 线程已提前创建好并处于就绪状态，任务到来时可立即执行，无需等待线程创建时间
3. 控制并发数量，防止系统崩溃
   - 无限制创建线程会导致系统资源耗尽（如内存溢出、CPU上下文切 换过多）
   - 线程池可设置最大线程数，限制并发线程数量，保护系统稳定性
4. 便于管理和监控
   - 线程池提供统一的任务队列、线程生命周期管理、异常处理机制
   - 可以统计任务执行情况、线程活跃度等，便于调优和监控
5. 支持任务排队和拒绝策略
   - 当任务过多时，线程池可将任务放入队列等待，或按策略拒绝新任务，增强系统弹性

#### 什么时候使用线程池？

| 场景               | 说明                                                 |
| ------------------ | ---------------------------------------------------- |
| 高并发任务处理     | 如Web服务器处理大量请求，每个请求可用线程池异步处理  |
| 大量短生命周期任务 | 如日志写入、消息推送、数据校验等，频繁创建线程不划算 |
| 需要控制资源使用   | 如移动端或嵌入式设备，需要限制线程数量避免OOM        |
| 异步操作耗时操作   | 如文件上传、图片处理、网络请求，避免阻塞主线程       |
| 定时或周期性任务   | 配合调度器实现定时任务（如每天清晨清理缓存）         |

> 不适合使用线程池的情况：
>
> - 任务数量极少且执行时间长（线程池优势不明显）
> - 需要精细控制每个线程行为的特殊场景（如实时系统）

#### 线程池需要配置哪些参数？

> 参数配置流程总结：
>
> 1. 分析任务类型：确定是CPU密集型还是I/O密集型
> 2. 确定核心线程数：根据任务类型和CPU核心数初步设定
> 3. 选择工作队列：根据响应速度、资源控制和可靠性的要求，选择有界队列或无界队列（慎用）
> 4. 设定最大线程数：对于有界队列，根据峰值流量和系统承受能力设定
> 5. 设定生存时间：根据任务到达的间歇性设定，避免资源浪费
> 6. 定义线程工厂：设置有意义的线程名，便于排查问题
> 7. 选择合适的拒绝策略：根据业务容忍选择，通常CallerRunsPolicy是一个稳妥的七点。
> 8. 压测与调优：以上只是理论指导，必须进行压力测试和线上监控，观察线程池的各项指标（活跃线程数、队列大小、拒绝次数等），反复调整参数以达到最优。



1. **核心线程数**
   - 含义：线程池中始终保持存活的线程数量，即使它们是空闲的（除非设置了allowCoreThreadTimeOut）
   - 配置考虑：
     - CPU密集型任务（如计算、图像处理）：核心线程数≈CPU核心数或CPU核心数+1。过多的线程会导致频繁的上下文切换，反而降低性能。
     - I/O密集型任务（如网络请求、数据库操作）：核心线程数可以设置的大一些，因为线程大部分时间在等待I/O，CPU是空闲的。通常可以设置为**CPU核心数 * (1+平均等待时间/平均计算时间)**。一个经验值可以是**CPU核心数 * 2**，或者更高，具体需要根据压测结果调整。
     - 混合型任务：可以将任务拆分为CPU密集型和I/O密集型，分别用不同的线程池处理，或者取一个折中值并经过充分测试。
2. **最大线程数**
   - 含义：线程池允许创建的最大线程数量。当工作队列已满，且当前运行的线程数小于最大线程数时，线程池会创建新的线程来处理任务。
   - 配置考虑：
     - 它定义了线程池处理突发流量的能力上限
     - 对于有界队列，最大线程数通常大于核心线程数，以应对瞬时高峰
     - 对于无界队列，由于队列永远不会满，所以最大线程数实际上无效，因为不会触发创建新线程的逻辑。
     - 需要根据系统的资源（内存、CPU）和任务的特性来设定，避免创建过多线程导致系统崩溃。
3. **生存时间**
   - 含义：当线程池中的线程数量大于核心线程数时，多余的空闲线程在终止前等待新任务的最长时间。
   - 配置考虑：
     - 这个参数只有在workQueue不为空且 线程数>corePoolSize时才有效
     - 如果任务是突发的，之后长时间空闲，设置一个合理的keepAliveTime可以避免浪费系统资源
     - 可以通过alloCoreThreadTimeOut方法让核心线程也使用此规则
4. **时间单位**
   - 含义：keepAliveTime参数的时间单位
   - 配置考虑：`TimeUnit.SECONDS`（秒）、`TimeUnit.MILLISECONDS`（毫秒）、`TimeUnit.MINUTES`（分钟）等。
5. **工作队列**
   - 含义：用于保存等待执行的任务的阻塞队列
   - 常见类型与选择：
     - SynchronousQueue：一个不存储元素的队列。每个插入操作必须等待另一个线程的对应移除操作。这相当于将任务直接交给线程执行，如果没有空闲线程，则尝试创建新线程（直到达到最大线程数）。适用于任务量很大且执行快的场景。配合较大的最大线程数。
     - LinkedBlockingQueue：一个基于链表的无界队列（如果不指定容量）。如果使用该队列，**最大线程数参数将失效**，因为队列永远不会满。风险在于可能导致任务无无限堆积，最终OOM。使用时务必谨慎，最好指定一个有界容量。
     - ArrayBlockQueue：一个基于数组的有界队列。可以防止资源耗尽，但需要合理设置队列大小。当队列满时，才会创建非核心线程。
     - PriorityBlockingQueue：一个具有优先级的无界队列。可以根据任务的优先级来执行。
   - 队列选择策略：
     - 希望快速响应，控制资源：使用有界队列（如ArrayBlockingQueue） + 合理的最大线程数。
     - 任务量大但执行快，不希望拒绝任务：使用SynchronousQueue + 较大最大线程数
     - 避免使用无界队列，除非你能确保任务生产速度不会持续超过消费速度
6. **线程工厂**
   - 含义：用于创建新线程的工厂。它是一个借口，可以自定义实现。
   - 作用：
     - 给线程设置有意义的名字，便于日志排查和监控
     - 设置线程的优先级、守护线程状态、异常处理器等
     - 例如：Executors.defaultThreadFactory()创建的线程是非守护线程，且优先级为正常。
7. **拒绝策略**
   - 含义：当线程池和工作队列都已满，无法处理新提交的任务时，才去的应对策略。
   - 内置策略：
     - AbortPolicy（默认）：直接抛出RejectedExecutionException异常。组织系统正常工作，但能明确感知到任务被拒绝了。
     - CallerRunsPolicy：有调用者线程自己执行该任务。这是一种简单的反馈机制，会降低新任务提交的速度，给线程池喘息的机会。通常是个不错的选择。
     - DiscadPolicy：默默丢弃无法处理的任务，不做任何通知。风险较大，可能丢失重要任务。
     - DiscardOldestPolicy：丢弃队列中最旧的一个任务，然后尝试重新提交当前任务。同样可能丢失任务。
   - 自定义策略：可以实现RejectExecutionHandler接口，例如将任务持久化到磁盘、记录日志、或由监控系统警告后人工介入。





#### 线程池的使用场景与实例

1. 场景1：Web服务器处理HTTP请求

   - 背景：Web服务器每收到一个请求，就可能需要执行数据库查询、业务逻辑、渲染页面等操作

   - 问题：如果每个请求都新建线程，高并发时会创建成千上万个线程，导致系统崩溃

   - 解决方案：使用线程池管理请求处理线程

     ```java
     // Java示例：模拟Web服务器使用线程池
     ExecutorService webPool = Executors.newFixedThreadPool(200); // 最多200个线程
     
     while (true) {
         Socket socket = serverSocket.accept(); // 接收客户端连接
         webPool.execute(() -> handleHttpRequest(socket)); // 提交任务到线程池
     }
     
     ```

2. 场景2：批量数据处理（如日志分析、ETL）

   - 背景：需要从多个文件中读取日志，解析后写入数据库

   - 问题：串行处理太慢；开太多线程又会压垮数据库或内存

   - 解决方案：使用线程池并行处理文件，控制并发数

     ```Java
     ExecutorService dataPool = Executors.newFixedThreadPool(10);
     
     List<File> logFiles = getLogFiles();
     for (File file : logFiles) {
         dataPool.submit(() -> processLogFile(file)); // 每个文件一个任务
     }
     dataPool.shutdown();
     ```

3. 场景3：异步通知客户（如短信、邮件发送）

   - 背景：用户注册后需发送欢迎邮件，但不应阻塞主流程
   - 解决方案：使用线程池异步发送

4. 场景4：定时任务调度（如每日报表生成）

   - 背景：每天凌晨2点生成销售报表

   - 解决方案：结合ScheduledThreadPoolExecutor实现定时执行

     ```Java
     ScheduledExecutorService scheduler = Executors.newScheduledThreadPool(1);
     
     // 每天凌晨2点执行
     LocalTime time = LocalTime.of(2, 0);
     long initialDelay = Duration.between(LocalTime.now(), time).toMillis();
     if (initialDelay < 0) initialDelay += TimeUnit.DAYS.toMillis(1);
     
     scheduler.scheduleAtFixedRate(
         this::generateDailyReport,
         initialDelay,
         TimeUnit.DAYS.toMillis(1),
         TimeUnit.MILLISECONDS
     );
     ```



### 线程池与Executor框架

创建和销毁线程会消耗大量系统资源，线程池通过重用已创建的线程来减少这种开销。Java提供了强大的Executor框架来管理线程池。

#### 1. 线程池类型

`Executors`类提供了几种常用的线程池工厂方法：

1. **newFixedThreadPool**：固定大小的线程池
2. **newCachedThreadPool**：可缓存的线程池，根据需要创建新线程
3. **newSingleThreadExecutor**：单线程的线程池，保证任务顺序执行
4. **newScheduledThreadPool**：支持定时及周期性任务执行的线程池

```java
import java.util.concurrent.ExecutorService;
import java.util.concurrent.Executors;

public class ThreadPoolExample {
    public static void main(String[] args) {
        // 创建固定大小的线程池
        ExecutorService executor = Executors.newFixedThreadPool(5);
        
        // 提交任务
        for (int i = 0; i < 10; i++) {
            executor.submit(() -> {
                System.out.println("Thread: " + Thread.currentThread().getId());
            });
        }
        
        // 关闭线程池
        executor.shutdown();
    }
}
```

#### 2. ThreadPoolExecutor

对于更精细的控制，可以直接使用`ThreadPoolExecutor`：

```java
import java.util.concurrent.LinkedBlockingQueue;
import java.util.concurrent.ThreadPoolExecutor;
import java.util.concurrent.TimeUnit;

public class CustomThreadPool {
    public static void main(String[] args) {
        ThreadPoolExecutor executor = new ThreadPoolExecutor(
            4, // 核心线程数
            8, // 最大线程数
            60, // 空闲线程存活时间
            TimeUnit.SECONDS, // 时间单位
            new LinkedBlockingQueue<>(100) // 工作队列
        );
        
        executor.submit(() -> {
            // 任务代码
        });
        
        executor.shutdown();
    }
}
```

#### 3. Fork/Join框架

Java 7引入的Fork/Join框架特别适合分而治之的并行任务：

```java
import java.util.concurrent.RecursiveTask;
import java.util.concurrent.ForkJoinPool;

public class FibonacciTask extends RecursiveTask<Long> {
    private final long n;
    
    public FibonacciTask(long n) {
        this.n = n;
    }
    
    @Override
    protected Long compute() {
        if (n <= 1) {
            return n;
        }
        FibonacciTask task1 = new FibonacciTask(n - 1);
        task1.fork();
        FibonacciTask task2 = new FibonacciTask(n - 2);
        return task2.compute() + task1.join();
    }
    
    public static void main(String[] args) {
        ForkJoinPool pool = new ForkJoinPool();
        FibonacciTask task = new FibonacciTask(10);
        System.out.println(pool.invoke(task));
    }
}
```

## 六、并发集合类

Java提供了一系列线程安全的并发集合类，它们在内部实现了高效的并发控制机制。

### 1. ConcurrentHashMap

高并发的HashMap实现，采用分段锁技术：

```java
import java.util.concurrent.ConcurrentHashMap;

public class Example {
    public static void main(String[] args) {
        ConcurrentHashMap<String, Integer> map = new ConcurrentHashMap<>();
        map.put("a", 1);
        map.put("b", 2);
        
        // 线程安全的操作
        map.compute("a", (k, v) -> v == null ? 0 : v + 1);
    }
}
```

### 2. CopyOnWriteArrayList

适合读多写少的场景，写操作时复制整个数组：

```java
import java.util.concurrent.CopyOnWriteArrayList;

public class Example {
    public static void main(String[] args) {
        CopyOnWriteArrayList<String> list = new CopyOnWriteArrayList<>();
        list.add("a");
        list.add("b");
        
        // 线程安全的遍历
        for (String s : list) {
            System.out.println(s);
        }
    }
}
```

### 3. BlockingQueue

阻塞队列，常用于生产者-消费者模式：

```java
import java.util.concurrent.ArrayBlockingQueue;
import java.util.concurrent.BlockingQueue;

public class Example {
    public static void main(String[] args) {
        BlockingQueue<String> queue = new ArrayBlockingQueue<>(10);
        
        // 生产者
        new Thread(() -> {
            try {
                queue.put("item");
            } catch (InterruptedException e) {
                e.printStackTrace();
            }
        }).start();
        
        // 消费者
        new Thread(() -> {
            try {
                String item = queue.take();
                System.out.println(item);
            } catch (InterruptedException e) {
                e.printStackTrace();
            }
        }).start();
    }
}
```

## 七、Java内存模型与并发问题

### 1. Java内存模型(JMM)

Java内存模型定义了线程如何与内存交互，规定了以下内容：

1. **可见性**：一个线程对共享变量的修改何时对其他线程可见。
2. **原子性**：哪些操作是不可分割的。
3. **有序性**：指令重排序的规则。

### 2. happens-before规则

JMM通过happens-before规则保证内存操作的可见性，主要包括：

- 程序顺序规则
- 监视器锁规则
- volatile变量规则
- 线程启动规则
- 线程终止规则
- 中断规则
- 终结器规则
- 传递性

### 3. 常见并发问题

1. **死锁**：多个线程互相持有对方需要的资源而无限等待。
   - 避免方法：按固定顺序获取锁，使用tryLock()设置超时。
2. **活锁**：线程不断改变状态而无法继续执行。
   - 避免方法：引入随机性，如随机退避。
3. **饥饿**：某些线程长时间得不到执行机会。
   - 避免方法：使用公平锁，合理设置线程优先级。

## 八、并行编程模型

Java支持多种并行编程模型，适用于不同的应用场景。

### 1. 共享内存模型

多线程共享同一内存空间，通过锁机制协调访问：

- 优点：通信开销低，性能较高。
- 缺点：同步复杂，容易出错。
- 应用场景：高性能计算、游戏开发等。

### 2. 消息传递模型

进程/线程通过发送消息进行通信：

- Java实现：Java Message Service(JMS)、Akka框架。
- 优点：松耦合，可扩展性好，容错性高。
- 应用场景：分布式系统、微服务架构。

### 3. 数据并行模型

对数据进行并行操作：

- Java实现：Java Parallel Processing Framework(JPPF)、Apache Spark。
- 优点：可扩展性好，易于编程。
- 应用场景：大数据处理、机器学习。

### 4. 任务并行模型

将任务分解为子任务并行执行：

- Java实现：Fork/Join框架、Java Concurrency Utilities。
- 优点：可扩展性好，易于编程。
- 应用场景：并行算法、图像处理。

### 5. 混合并行模型

同时使用多种并行编程模型：

- Java实现：Java Concurrency Utilities、Apache Spark。
- 优点：灵活性高，可根据问题特点选择最佳模型。
- 应用场景：复杂并行应用。

## 九、最佳实践与性能优化

### 1. 多线程编程最佳实践

1. **避免过早优化**：先保证正确性，再考虑性能优化。
2. **尽量使用高层抽象**：优先使用并发工具类而非低级的wait/notify。
3. **减少锁粒度**：缩小同步块范围，使用细粒度锁。
4. **避免锁嵌套**：容易导致死锁。
5. **使用不可变对象**：避免同步需求。
6. **注意异常处理**：确保锁能正确释放。

### 2. 性能优化技巧

1. **合理设置线程池大小**：
   - CPU密集型任务：线程数 ≈ CPU核心数
   - I/O密集型任务：线程数可以更多
2. **使用适当的并发集合**：根据读写比例选择合适实现。
3. **减少上下文切换**：
   - 避免创建过多线程
   - 使用协程(Java 19+虚拟线程)
4. **避免伪共享**：使用@Contended注解(Java 8+)填充缓存行。
5. **利用并行流**：Java 8+的parallelStream()简化并行处理。

```java
List<Integer> numbers = Arrays.asList(1, 2, 3, 4, 5);
int sum = numbers.parallelStream().reduce(0, Integer::sum);
```
