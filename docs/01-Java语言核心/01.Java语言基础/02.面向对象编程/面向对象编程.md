# 面向对象编程

## 一、面向对象概述

### 1.1 什么是面向对象？

面向对象（Object-Oriented Programming，OOP）是一种编程思想，它将现实世界的事物抽象为对象，通过对象之间的交互来完成程序功能。

**核心思想：**
- 把数据和对数据的操作封装在一起
- 以对象为中心组织代码
- 模拟现实世界的思维方式

### 1.2 面向对象 vs 面向过程

| 对比项 | 面向过程 | 面向对象 |
|--------|---------|---------|
| 核心 | 步骤、流程 | 对象、类 |
| 关注点 | 怎么做 | 谁来做 |
| 特点 | 线性思维 | 模块化思维 |
| 适用场景 | 简单任务 | 复杂系统 |
| 可维护性 | 较差 | 较好 |
| 可扩展性 | 较差 | 较好 |

**示例：大象装冰箱**

```java
// 面向过程
openDoor();      // 打开冰箱门
putElephant();   // 放入大象
closeDoor();     // 关闭冰箱门

// 面向对象
Refrigerator fridge = new Refrigerator();
Elephant elephant = new Elephant();
fridge.open();
fridge.put(elephant);
fridge.close();
```

### 1.3 面向对象的三大特征

1. **封装**：隐藏内部实现，暴露外部接口
2. **继承**：子类继承父类的属性和方法
3. **多态**：同一个方法调用，根据对象不同产生不同行为

## 二、类与对象

### 2.1 类的定义

类是对象的模板，定义了对象的属性和行为

```java
public class Student {
    // 属性（成员变量）
    private String name;
    private int age;
    private String gender;
    
    // 构造方法
    public Student() {
    }
    
    public Student(String name, int age, String gender) {
        this.name = name;
        this.age = age;
        this.gender = gender;
    }
    
    // 方法（成员方法）
    public void study() {
        System.out.println(name + "正在学习");
    }
    
    public void introduce() {
        System.out.println("我叫" + name + "，今年" + age + "岁");
    }
    
    // getter和setter方法
    public String getName() {
        return name;
    }
    
    public void setName(String name) {
        this.name = name;
    }
    
    public int getAge() {
        return age;
    }
    
    public void setAge(int age) {
        this.age = age;
    }
}
```

### 2.2 对象的创建与使用

```java
public class Test {
    public static void main(String[] args) {
        // 创建对象
        Student stu1 = new Student();
        
        // 设置属性
        stu1.setName("张三");
        stu1.setAge(20);
        
        // 调用方法
        stu1.study();
        stu1.introduce();
        
        // 使用构造方法创建对象
        Student stu2 = new Student("李四", 22, "男");
        stu2.introduce();
    }
}
```

### 2.3 对象在内存中的存储

```
栈内存                    堆内存
┌──────────┐            ┌──────────────┐
│ stu1 引用 │ ──────→   │ Student对象   │
│ stu2 引用 │ ──────→   │ name="李四"   │
└──────────┘            │ age=22       │
                        │ gender="男"   │
                        └──────────────┘
```

## 三、封装

### 3.1 什么是封装？

封装是将对象的属性和实现细节隐藏起来，只对外提供公共的访问方式。

**优点：**
- 提高安全性
- 提高复用性
- 降低耦合度

### 3.2 访问修饰符

| 修饰符 | 同类 | 同包 | 子类 | 其他 |
|--------|------|------|------|------|
| private | ✓ | ✗ | ✗ | ✗ |
| default（默认） | ✓ | ✓ | ✗ | ✗ |
| protected | ✓ | ✓ | ✓ | ✗ |
| public | ✓ | ✓ | ✓ | ✓ |

```java
public class Person {
    private String name;      // 私有，只能在本类访问
    int age;                  // 默认，同包可访问
    protected String address; // 保护，子类和同包可访问
    public String phone;      // 公开，任何地方都可访问
}
```

### 3.3 Getter 和 Setter 方法

```java
public class User {
    private String username;
    private int age;
    
    // Getter方法
    public String getUsername() {
        return username;
    }
    
    // Setter方法（可以加入验证逻辑）
    public void setUsername(String username) {
        if (username != null && !username.isEmpty()) {
            this.username = username;
        }
    }
    
    public int getAge() {
        return age;
    }
    
    public void setAge(int age) {
        if (age >= 0 && age <= 150) {
            this.age = age;
        } else {
            System.out.println("年龄不合法");
        }
    }
}
```

### 3.4 this 关键字

`this` 代表当前对象的引用

```java
public class Student {
    private String name;
    private int age;
    
    public Student(String name, int age) {
        this.name = name;  // this.name指成员变量，name指参数
        this.age = age;
    }
    
    public void study() {
        System.out.println(this.name + "在学习");
    }
    
    // this调用构造方法
    public Student() {
        this("未知", 0);  // 必须放在第一行
    }
}
```

## 四、继承

### 4.1 什么是继承？

继承是子类继承父类的属性和方法，实现代码复用。

**语法：**
```java
class 子类 extends 父类 {
    // 子类特有的属性和方法
}
```

### 4.2 继承示例

```java
// 父类
public class Animal {
    private String name;
    private int age;
    
    public void eat() {
        System.out.println(name + "在吃东西");
    }
    
    public void sleep() {
        System.out.println(name + "在睡觉");
    }
    
    // getter和setter省略
}

// 子类
public class Dog extends Animal {
    private String breed;  // 品种
    
    public void bark() {
        System.out.println(getName() + "汪汪叫");
    }
}

// 另一个子类
public class Cat extends Animal {
    public void meow() {
        System.out.println(getName() + "喵喵叫");
    }
}

// 测试
public class Test {
    public static void main(String[] args) {
        Dog dog = new Dog();
        dog.setName("旺财");
        dog.eat();   // 继承自Animal
        dog.bark();  // Dog特有
        
        Cat cat = new Cat();
        cat.setName("咪咪");
        cat.eat();   // 继承自Animal
        cat.meow();  // Cat特有
    }
}
```

### 4.3 继承的特点

1. **单继承**：Java中一个类只能有一个直接父类
2. **多层继承**：可以形成继承链 A → B → C
3. **传递性**：子类可以间接继承祖先类的成员

```java
class A { }
class B extends A { }
class C extends B { }  // C间接继承了A
```

### 4.4 super 关键字

`super` 代表父类对象的引用

```java
public class Animal {
    protected String name;
    
    public Animal(String name) {
        this.name = name;
    }
    
    public void eat() {
        System.out.println(name + "在吃东西");
    }
}

public class Dog extends Animal {
    private String breed;
    
    public Dog(String name, String breed) {
        super(name);  // 调用父类构造方法，必须在第一行
        this.breed = breed;
    }
    
    @Override
    public void eat() {
        super.eat();  // 调用父类方法
        System.out.println("吃骨头");
    }
}
```

### 4.5 方法重写（Override）

子类重新定义父类已有的方法

```java
public class Animal {
    public void speak() {
        System.out.println("动物叫声");
    }
}

public class Dog extends Animal {
    @Override  // 注解，表示重写父类方法
    public void speak() {
        System.out.println("汪汪汪");
    }
}

public class Cat extends Animal {
    @Override
    public void speak() {
        System.out.println("喵喵喵");
    }
}
```

**重写规则：**
- 方法名、参数列表必须相同
- 返回值类型可以相同或是其子类
- 访问权限不能比父类更严格
- 不能抛出新的或更大的异常

### 4.6 Object 类

所有类的根父类

```java
public class Student {
    // 隐式继承自Object类
}

// 等价于
public class Student extends Object {
}
```

**常用方法：**
```java
Student stu = new Student();

// toString()：返回对象的字符串表示
System.out.println(stu.toString());

// equals()：比较两个对象是否相等
stu.equals(otherStu);

// hashCode()：返回对象的哈希码
stu.hashCode();

// getClass()：返回对象的运行时类
stu.getClass().getName();
```

## 五、多态

### 5.1 什么是多态？

多态是同一个行为具有多个不同表现形式的能力。

**前提条件：**
1. 有继承关系
2. 有方法重写
3. 父类引用指向子类对象

### 5.2 多态示例

```java
// 父类
public class Animal {
    public void speak() {
        System.out.println("动物叫声");
    }
}

// 子类
public class Dog extends Animal {
    @Override
    public void speak() {
        System.out.println("汪汪汪");
    }
}

public class Cat extends Animal {
    @Override
    public void speak() {
        System.out.println("喵喵喵");
    }
}

// 测试多态
public class Test {
    public static void main(String[] args) {
        // 父类引用指向子类对象
        Animal animal1 = new Dog();
        Animal animal2 = new Cat();
        
        animal1.speak();  // 汪汪汪（实际调用Dog的方法）
        animal2.speak();  // 喵喵喵（实际调用Cat的方法）
    }
}
```

### 5.3 多态的好处

```java
public class AnimalCare {
    // 使用多态，可以接受任何Animal的子类
    public void feed(Animal animal) {
        System.out.println("喂食...");
        animal.speak();
        animal.eat();
    }
}

// 调用
AnimalCare care = new AnimalCare();
care.feed(new Dog());  // 喂狗
care.feed(new Cat());  // 喂猫
care.feed(new Bird()); // 喂鸟
```

**优点：**
- 提高代码扩展性
- 降低耦合度
- 便于维护

### 5.4 类型转换

#### 向上转型（自动）

```java
Animal animal = new Dog();  // 自动向上转型
```

#### 向下转型（强制）

```java
Animal animal = new Dog();
Dog dog = (Dog)animal;  // 强制向下转型
dog.bark();  // 可以调用Dog特有的方法
```

#### instanceof 关键字

判断对象是否是某个类的实例

```java
Animal animal = new Dog();

if (animal instanceof Dog) {
    Dog dog = (Dog)animal;
    dog.bark();
} else if (animal instanceof Cat) {
    Cat cat = (Cat)animal;
    cat.meow();
}
```

### 5.5 多态的注意事项

```java
public class Animal {
    public String type = "动物";
    
    public void speak() {
        System.out.println("动物叫声");
    }
}

public class Dog extends Animal {
    public String type = "狗";
    
    @Override
    public void speak() {
        System.out.println("汪汪汪");
    }
    
    public void bark() {
        System.out.println("大声汪汪叫");
    }
}

// 测试
Animal animal = new Dog();
System.out.println(animal.type);  // 动物（成员变量看编译时类型）
animal.speak();                   // 汪汪汪（成员方法看运行时类型）
// animal.bark();                 // 编译错误！父类引用不能调用子类特有方法
```

**总结：**
- 成员变量：编译看左边，运行看左边
- 成员方法：编译看左边，运行看右边
- 静态方法：编译看左边，运行看左边

## 六、抽象类

### 6.1 什么是抽象类？

抽象类是不能被实例化的类，用于作为其他类的基类。

**特点：**
- 用 `abstract` 修饰
- 可以有抽象方法（没有方法体）
- 可以有普通方法和成员变量
- 不能被实例化

### 6.2 抽象类示例

```java
// 抽象类
public abstract class Shape {
    protected String color;
    
    public Shape(String color) {
        this.color = color;
    }
    
    // 抽象方法（没有方法体）
    public abstract double getArea();
    public abstract double getPerimeter();
    
    // 普通方法
    public void display() {
        System.out.println("颜色：" + color);
        System.out.println("面积：" + getArea());
        System.out.println("周长：" + getPerimeter());
    }
}

// 子类必须实现抽象方法
public class Circle extends Shape {
    private double radius;
    
    public Circle(String color, double radius) {
        super(color);
        this.radius = radius;
    }
    
    @Override
    public double getArea() {
        return Math.PI * radius * radius;
    }
    
    @Override
    public double getPerimeter() {
        return 2 * Math.PI * radius;
    }
}

public class Rectangle extends Shape {
    private double width;
    private double height;
    
    public Rectangle(String color, double width, double height) {
        super(color);
        this.width = width;
        this.height = height;
    }
    
    @Override
    public double getArea() {
        return width * height;
    }
    
    @Override
    public double getPerimeter() {
        return 2 * (width + height);
    }
}

// 测试
public class Test {
    public static void main(String[] args) {
        // Shape shape = new Shape();  // 错误！抽象类不能实例化
        
        Shape circle = new Circle("红色", 5.0);
        circle.display();
        
        Shape rectangle = new Rectangle("蓝色", 4.0, 6.0);
        rectangle.display();
    }
}
```

### 6.3 抽象类的应用场景

- 定义模板方法
- 提供部分实现
- 强制子类实现某些方法

## 七、接口

### 7.1 什么是接口？

接口是完全抽象的类型，定义了一组行为规范。

**特点：**
- 用 `interface` 关键字定义
- 所有方法都是抽象的（JDK 8之前）
- 所有变量都是 `public static final`
- 一个类可以实现多个接口

### 7.2 接口定义与实现

```java
// 定义接口
public interface USB {
    // 常量（默认public static final）
    int MAX_SPEED = 480;
    
    // 抽象方法（默认public abstract）
    void connect();
    void disconnect();
    void transferData();
}

// 实现接口
public class Mouse implements USB {
    @Override
    public void connect() {
        System.out.println("鼠标连接");
    }
    
    @Override
    public void disconnect() {
        System.out.println("鼠标断开");
    }
    
    @Override
    public void transferData() {
        System.out.println("传输鼠标数据");
    }
}

public class Keyboard implements USB {
    @Override
    public void connect() {
        System.out.println("键盘连接");
    }
    
    @Override
    public void disconnect() {
        System.out.println("键盘断开");
    }
    
    @Override
    public void transferData() {
        System.out.println("传输键盘数据");
    }
}
```

### 7.3 接口的多实现

```java
public interface Flyable {
    void fly();
}

public interface Swimmable {
    void swim();
}

// 一个类可以实现多个接口
public class Duck implements Flyable, Swimmable {
    @Override
    public void fly() {
        System.out.println("鸭子在飞");
    }
    
    @Override
    public void swim() {
        System.out.println("鸭子在游泳");
    }
}
```

### 7.4 接口的继承

```java
public interface A {
    void methodA();
}

public interface B {
    void methodB();
}

// 接口可以多继承
public interface C extends A, B {
    void methodC();
}

// 实现类需要实现所有方法
public class Impl implements C {
    @Override
    public void methodA() { }
    
    @Override
    public void methodB() { }
    
    @Override
    public void methodC() { }
}
```

### 7.5 JDK 8+ 接口新特性

#### 默认方法

```java
public interface MyInterface {
    // 默认方法（有方法体）
    default void defaultMethod() {
        System.out.println("默认方法");
    }
}
```

#### 静态方法

```java
public interface MyInterface {
    // 静态方法
    static void staticMethod() {
        System.out.println("静态方法");
    }
}

// 调用
MyInterface.staticMethod();
```

### 7.6 抽象类 vs 接口

| 对比项 | 抽象类 | 接口 |
|--------|--------|------|
| 关键字 | abstract class | interface |
| 继承/实现 | 单继承 | 多实现 |
| 构造方法 | 有 | 无 |
| 成员变量 | 各种类型 | 只能是常量 |
| 成员方法 | 抽象和非抽象 | JDK8前只能抽象 |
| 设计理念 | is-a关系 | like-a关系 |
| 使用场景 | 模板设计 | 规范制定 |

## 八、内部类

### 8.1 成员内部类

```java
public class Outer {
    private String name = "外部类";
    
    // 成员内部类
    class Inner {
        public void show() {
            System.out.println(name);  // 可以访问外部类私有成员
        }
    }

    }

// 使用
Outer outer = new Outer();
Outer.Inner inner = outer.new Inner();

```

### 8.2 静态内部类

```java
public class Outer {

            System.out.println("静态内部类");

// 使用
Outer.StaticInner inner = new Outer.StaticInner();
inner.show();
```

### 8.3 局部内部类

```java
public class Outer {
    public void method() {
        // 局部内部类

            public void show() {
                System.out.println("局部内部类");

```

### 8.4 匿名内部类

## 九、代码块

### 9.1 代码块分类

```java
public class CodeBlockDemo {
    
    // 静态代码块（类加载时执行，只执行一次）
    static {
        System.out.println("静态代码块");
    }
    
    // 构造代码块（每次创建对象时执行，在构造方法之前）
    {
        System.out.println("构造代码块");
    }
    
    // 构造方法
    public CodeBlockDemo() {
        System.out.println("构造方法");
    }
    
    // 局部代码块（方法内部）
    public void method() {
        {
            int x = 10;
            System.out.println(x);
        }
        // System.out.println(x);  // 错误！x已超出作用域
    }
}

// 执行顺序：静态代码块 → 构造代码块 → 构造方法
```

### 9.2 执行顺序

```java
public class Parent {
    static {
        System.out.println("父类静态代码块");
    }
    
    {
        System.out.println("父类构造代码块");
    }
    
    public Parent() {
        System.out.println("父类构造方法");
    }
}

public class Child extends Parent {
    static {
        System.out.println("子类静态代码块");
    }
    
    {
        System.out.println("子类构造代码块");
    }
    
    public Child() {
        System.out.println("子类构造方法");
    }
}

// 测试
new Child();

// 输出：
// 父类静态代码块
// 子类静态代码块
// 父类构造代码块
// 父类构造方法
// 子类构造代码块
// 子类构造方法
```

## 十、final 关键字

### 10.1 final 修饰类

```java
// 最终类，不能被继承
public final class FinalClass {
    // ...
}

// 错误！不能继承final类
// class SubClass extends FinalClass { }
```

### 10.2 final 修饰方法

```java
public class Parent {
    // 最终方法，不能被重写
    public final void finalMethod() {
        System.out.println("最终方法");
    }
}

public class Child extends Parent {
    // 错误！不能重写final方法
    // @Override
    // public void finalMethod() { }
}
```

### 10.3 final 修饰变量

```java
// 基本类型：值不能改变
final int MAX = 100;
// MAX = 200;  // 错误！

// 引用类型：引用不能改变，但对象内容可以改变
final StringBuilder sb = new StringBuilder();
sb.append("Hello");  // 可以
// sb = new StringBuilder();  // 错误！
```

## 十一、综合案例

### 11.1 员工管理系统

```java
// 抽象员工类
public abstract class Employee {
    private String id;
    private String name;
    private double salary;
    
    public Employee(String id, String name, double salary) {
        this.id = id;
        this.name = name;
        this.salary = salary;
    }
    
    // 抽象方法：计算工资
    public abstract double calculateSalary();
    
    public void displayInfo() {
        System.out.println("工号：" + id);
        System.out.println("姓名：" + name);
        System.out.println("工资：" + calculateSalary());
    }
    
    // getter和setter省略
}

// 正式员工
public class FullTimeEmployee extends Employee {
    private double bonus;  // 奖金
    
    public FullTimeEmployee(String id, String name, double salary, double bonus) {
        super(id, name, salary);
        this.bonus = bonus;
    }
    
    @Override
    public double calculateSalary() {
        return getSalary() + bonus;
    }
}

// 兼职员工
public class PartTimeEmployee extends Employee {
    private int hours;      // 工作小时数
    private double hourlyRate;  // 时薪
    
    public PartTimeEmployee(String id, String name, int hours, double hourlyRate) {
        super(id, name, 0);
        this.hours = hours;
        this.hourlyRate = hourlyRate;
    }
    
    @Override
    public double calculateSalary() {
        return hours * hourlyRate;
    }
}
```

## 十二、设计原则

### SOLID 原则

1. **单一职责原则（SRP）**：一个类只负责一项职责
2. **开闭原则（OCP）**：对扩展开放，对修改关闭
3. **里氏替换原则（LSP）**：子类可以替换父类
4. **接口隔离原则（ISP）**：使用多个专门的接口
5. **依赖倒置原则（DIP）**：依赖抽象，不依赖具体
