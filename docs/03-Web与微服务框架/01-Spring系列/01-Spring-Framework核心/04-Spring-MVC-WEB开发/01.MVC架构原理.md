## 一、MVC设计模式

### 1.1 什么是MVC

**MVC (Model-View-Controller)** 是一种软件架构模式,将应用程序分为三个核心组件:

- **Model (模型)**: 负责数据和业务逻辑
- **View (视图)**: 负责展示数据(页面)
- **Controller (控制器)**: 负责接收请求、调用模型、返回视图

### 1.2 MVC的优势

1. **职责分离**: 各组件职责明确,便于维护
2. **松耦合**: 组件之间依赖关系清晰
3. **可复用**: 模型可以被多个视图复用
4. **易扩展**: 可以独立替换某个组件

### 1.3 Web MVC工作流程

典型的Web MVC处理流程:

```
用户请求 → 控制器 → 模型处理 → 返回数据 → 视图渲染 → 响应给用户
```

## 二、Spring MVC概述

### 2.1 Spring MVC简介

Spring MVC是Spring Framework的一部分,是基于Java实现的MVC轻量级Web框架。

**核心特点**:
- 清晰的角色划分
- 强大的注解支持
- 灵活的URL映射
- 支持多种视图技术(JSP、Thymeleaf、FreeMarker等)
-  RESTful风格支持
-  强大的数据绑定和验证

## 三、DispatcherServlet工作流程

### 3.1 核心流程

DispatcherServlet是Spring MVC的核心,所有请求都经过它来处理。

**完整工作流程**:

```
1. 用户发送请求 → DispatcherServlet
2. DispatcherServlet → HandlerMapping查找Controller
3. HandlerMapping → 返回执行链(Interceptor + Controller)
4. DispatcherServlet → HandlerAdapter执行Controller
5. Controller → 执行业务逻辑,返回ModelAndView
6. ModelAndView → ViewResolver解析视图
7. ViewResolver → 返回具体View
8. View → 渲染视图
9. DispatcherServlet → 响应给用户
```

### 3.2 核心组件详解

#### 1. DispatcherServlet (前端控制器)

- 整个流程控制的中心
- 由它调用其它组件处理用户的请求
- 降低了组件之间的耦合性

**配置示例** (web.xml):

```xml
<servlet>
    <servlet-name>dispatcher</servlet-name>
    <servlet-class>org.springframework.web.servlet.DispatcherServlet</servlet-class>
    <init-param>
        <param-name>contextConfigLocation</param-name>
        <param-value>classpath:spring-mvc.xml</param-value>
    </init-param>
    <load-on-startup>1</load-on-startup>
</servlet>

<servlet-mapping>
    <servlet-name>dispatcher</servlet-name>
    <url-pattern>/</url-pattern>
</servlet-mapping>
```

#### 2. HandlerMapping (处理器映射器)

- 根据请求的URL查找对应的Controller
- 支持多种映射策略(注解、XML配置等)

**常见实现**:
- `RequestMappingHandlerMapping`: 基于注解的映射

#### 3. HandlerAdapter (处理器适配器)

- 按照特定规则(Controller的实现方式)执行Controller
- 支持多种类型的Controller

**常见实现**:
- `RequestMappingHandlerAdapter`: 适配注解方式的Controller

#### 4. Controller (处理器)

- 处理用户请求的核心业务逻辑
- 返回ModelAndView给DispatcherServlet

**示例**:

```java
@Controller
public class HelloController {
    
    @RequestMapping("/hello")
    public ModelAndView hello() {
        ModelAndView mav = new ModelAndView();
        mav.addObject("message", "Hello Spring MVC!");
        mav.setViewName("hello");
        return mav;
    }
}
```

#### 5. ViewResolver (视图解析器)

- 将逻辑视图名解析为具体的View对象
- 支持多种视图技术

**配置示例**:

```xml
<!--配置视图解析器-->
<bean class="org.springframework.web.servlet.view.InternalResourceViewResolver">
    <property name="prefix" value="/WEB-INF/views/"/>
    <property name="suffix" value=".jsp"/>
</bean>
```

## 四、Spring MVC快速入门

### 4.1 环境搭建

#### 第一步: 导入依赖

```xml
<dependencies>
    <!--Spring MVC-->
    <dependency>
        <groupId>org.springframework</groupId>
        <artifactId>spring-webmvc</artifactId>
        <version>5.3.20</version>
    </dependency>
    
    <!--Servlet API-->
    <dependency>
        <groupId>javax.servlet</groupId>
        <artifactId>javax.servlet-api</artifactId>
        <version>4.0.1</version>
        <scope>provided</scope>
    </dependency>
    
    <!--JSP支持(可选)-->
    <dependency>
        <groupId>javax.servlet.jsp</groupId>
        <artifactId>javax.servlet.jsp-api</artifactId>
        <version>2.3.3</version>
        <scope>provided</scope>
    </dependency>
</dependencies>
```

#### 第二步: 配置web.xml

```xml
<?xml version="1.0" encoding="UTF-8"?>
<web-app xmlns="http://xmlns.jcp.org/xml/ns/javaee"
         xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
         xsi:schemaLocation="http://xmlns.jcp.org/xml/ns/javaee 
                             http://xmlns.jcp.org/xml/ns/javaee/web-app_4_0.xsd"
         version="4.0">
    
    <!--配置DispatcherServlet-->
    <servlet>
        <servlet-name>dispatcher</servlet-name>
        <servlet-class>org.springframework.web.servlet.DispatcherServlet</servlet-class>
        <init-param>
            <param-name>contextConfigLocation</param-name>
            <param-value>classpath:spring-mvc.xml</param-value>
        </init-param>
        <load-on-startup>1</load-on-startup>
    </servlet>
    
    <servlet-mapping>
        <servlet-name>dispatcher</servlet-name>
        <url-pattern>/</url-pattern>
    </servlet-mapping>
    
</web-app>
```

#### 第三步: 配置spring-mvc.xml

```xml
<?xml version="1.0" encoding="UTF-8"?>
<beans xmlns="http://www.springframework.org/schema/beans"
       xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
       xmlns:context="http://www.springframework.org/schema/context"
       xmlns:mvc="http://www.springframework.org/schema/mvc"
       xsi:schemaLocation="http://www.springframework.org/schema/beans
                           http://www.springframework.org/schema/beans/spring-beans.xsd
                           http://www.springframework.org/schema/context
                           http://www.springframework.org/schema/context/spring-context.xsd
                           http://www.springframework.org/schema/mvc
                           http://www.springframework.org/schema/mvc/spring-mvc.xsd">
    
    <!--开启组件扫描-->
    <context:component-scan base-package="com.example.controller"/>
    
    <!--配置视图解析器-->
    <bean class="org.springframework.web.servlet.view.InternalResourceViewResolver">
        <property name="prefix" value="/WEB-INF/views/"/>
        <property name="suffix" value=".jsp"/>
    </bean>
    
    <!--开启MVC注解驱动-->
    <mvc:annotation-driven/>
    
    <!--静态资源放行-->
    <mvc:default-servlet-handler/>
    
</beans>
```

#### 第四步: 创建Controller

```java
package com.example.controller;

import org.springframework.stereotype.Controller;
import org.springframework.web.bind.annotation.RequestMapping;
import org.springframework.web.servlet.ModelAndView;

@Controller
public class HelloController {
    
    @RequestMapping("/hello")
    public ModelAndView hello() {
        ModelAndView mav = new ModelAndView();
        // 添加模型数据
        mav.addObject("message", "Hello Spring MVC!");
        // 设置视图名称
        mav.setViewName("hello");
        return mav;
    }
}
```

#### 第五步: 创建视图页面

在`WEB-INF/views/`目录下创建`hello.jsp`:

```jsp
<%@ page contentType="text/html;charset=UTF-8" language="java" %>
<html>
<head>
    <title>Hello</title>
</head>
<body>
    <h1>${message}</h1>
</body>
</html>
```

#### 第六步: 部署测试

启动Tomcat,访问: `http://localhost:8080/项目名/hello`

## 五、请求映射与参数绑定

### 5.1 @RequestMapping注解

#### 基本用法

`@RequestMapping`用于建立请求URL和处理方法之间的对应关系。

**可以标注的位置**:
- 类级别: 定义请求的根路径
- 方法级别: 定义具体的请求路径

```java
@Controller
@RequestMapping("/user")
public class UserController {
    
    @RequestMapping("/list")
    public String list() {
        return "user-list";
    }
}
// 访问路径: /user/list
```

#### 请求方式限定

```java
// 只接受GET请求
@RequestMapping(value = "/get", method = RequestMethod.GET)
public String get() {
    return "get";
}

// 只接受POST请求
@RequestMapping(value = "/save", method = RequestMethod.POST)
public String save() {
    return "save";
}
```

**简化写法** (Spring 4.3+):

```java
@GetMapping("/get")    // 等同于 @RequestMapping(method = GET)
@PostMapping("/save")  // 等同于 @RequestMapping(method = POST)
@PutMapping("/update")
@DeleteMapping("/delete")
```

### 5.2 参数绑定

#### 1. 基本类型参数

```java
@RequestMapping("/param1")
public String param1(String name, int age) {
    System.out.println(name + ", " + age);
    return "success";
}
// 访问: /param1?name=张三&age=20
```

**注意**: 参数名必须与请求参数名一致

#### 2. POJO类型参数

```java
public class User {
    private String name;
    private int age;
    private String email;
    // getter/setter
}

@RequestMapping("/param2")
public String param2(User user) {
    System.out.println(user);
    return "success";
}
// 访问: /param2?name=李四&age=25&email=lisi@example.com
```

#### 3. 数组和集合参数

```java
// 数组参数
@RequestMapping("/param3")
public String param3(String[] hobbies) {
    for (String hobby : hobbies) {
        System.out.println(hobby);
    }
    return "success";
}
// 访问: /param3?hobbies=读书&hobbies=电影&hobbies=音乐

// List参数(需要封装在对象中)
public class QueryVO {
    private List<String> ids;
    // getter/setter
}

@RequestMapping("/param4")
public String param4(QueryVO vo) {
    System.out.println(vo.getIds());
    return "success";
}
// 访问: /param4?ids=1&ids=2&ids=3
```

#### 4. 路径变量@PathVariable

```java
@RequestMapping("/user/{id}")
public String pathVar(@PathVariable Integer id) {
    System.out.println("id: " + id);
    return "success";
}
// 访问: /user/123
```

**多个路径变量**:

```java
@RequestMapping("/user/{userId}/order/{orderId}")
public String multiPath(@PathVariable Integer userId, 
                        @PathVariable String orderId) {
    System.out.println(userId + ", " + orderId);
    return "success";
}
```

#### 5. 请求头@RequestParam

```java
@RequestMapping("/header")
public String header(@RequestParam("token") String token) {
    System.out.println("token: " + token);
    return "success";
}
// 访问: /header?token=abc123
```

**设置默认值和必填**:

```java
@RequestMapping("/header2")
public String header2(
    @RequestParam(value = "page", defaultValue = "1") Integer page,
    @RequestParam(required = false) String keyword
) {
    System.out.println(page + ", " + keyword);
    return "success";
}
```
