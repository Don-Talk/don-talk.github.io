## 一、拦截器Interceptor

### 1.1 什么是拦截器

**拦截器 (Interceptor)** 是Spring MVC提供的用于预处理和后处理请求的组件,在Controller执行前后进行拦截。

**典型应用场景**:
- 登录验证
- 权限检查
- 日志记录
- 性能监控
- 国际化处理

### 1.2 拦截器 vs 过滤器

| 特性 | 拦截器 (Interceptor) | 过滤器 (Filter) |
|------|---------------------|----------------|
| 规范 | Spring框架提供 | Servlet规范 |
| 依赖 | 依赖Spring容器 | 不依赖Spring |
| 使用范围 | 只能拦截Controller请求 | 可以拦截所有请求 |
| 执行顺序 | 在DispatcherServlet之后 | 在DispatcherServlet之前 |
| 注入Bean | ✅ 支持 | ❌ 不支持 |
| 获取Handler | ✅ 可以 | ❌ 不可以 |

**推荐**: 
- 业务逻辑相关(登录、权限) → 用拦截器
- 通用处理(编码、压缩) → 用过滤器

### 1.3 快速入门

#### 第一步: 创建拦截器

```java
package com.example.interceptor;

import org.springframework.stereotype.Component;
import org.springframework.web.servlet.HandlerInterceptor;
import org.springframework.web.servlet.ModelAndView;

import javax.servlet.http.HttpServletRequest;
import javax.servlet.http.HttpServletResponse;

@Component
public class LogInterceptor implements HandlerInterceptor {
    
    /**
     * 前置处理: Controller执行前调用
     * 返回true: 继续执行
     * 返回false: 中断执行
     */
    @Override
    public boolean preHandle(HttpServletRequest request, 
                            HttpServletResponse response, 
                            Object handler) throws Exception {
        System.out.println("【前置拦截】URL: " + request.getRequestURI());
        System.out.println("【前置拦截】方法: " + request.getMethod());
        
        // 记录开始时间
        request.setAttribute("startTime", System.currentTimeMillis());
        
        return true;  // 放行
    }
    
    /**
     * 后置处理: Controller执行后调用
     */
    @Override
    public void postHandle(HttpServletRequest request, 
                          HttpServletResponse response, 
                          Object handler, 
                          ModelAndView modelAndView) throws Exception {
        System.out.println("【后置拦截】视图: " + 
            (modelAndView != null ? modelAndView.getViewName() : "null"));
    }
    
    /**
     * 完成处理: 视图渲染后调用
     */
    @Override
    public void afterCompletion(HttpServletRequest request, 
                               HttpServletResponse response, 
                               Object handler, 
                               Exception ex) throws Exception {
        Long startTime = (Long) request.getAttribute("startTime");
        long cost = System.currentTimeMillis() - startTime;
        
        System.out.println("【完成拦截】耗时: " + cost + "ms");
        
        if (ex != null) {
            System.out.println("【完成拦截】异常: " + ex.getMessage());
        }
    }
}
```

#### 第二步: 配置拦截器

**方式一: 实现WebMvcConfigurer**

```java
package com.example.config;

import com.example.interceptor.LogInterceptor;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.context.annotation.Configuration;
import org.springframework.web.servlet.config.annotation.InterceptorRegistry;
import org.springframework.web.servlet.config.annotation.WebMvcConfigurer;

@Configuration
public class WebConfig implements WebMvcConfigurer {
    
    @Autowired
    private LogInterceptor logInterceptor;
    
    @Override
    public void addInterceptors(InterceptorRegistry registry) {
        // 添加拦截器
        registry.addInterceptor(logInterceptor)
            .addPathPatterns("/api/**")           // 拦截路径
            .excludePathPatterns("/api/login", "/api/register");  // 排除路径
    }
}
```

**方式二: XML配置**

```xml
<mvc:interceptors>
    <mvc:interceptor>
        <mvc:mapping path="/api/**"/>
        <mvc:exclude-mapping path="/api/login"/>
        <bean class="com.example.interceptor.LogInterceptor"/>
    </mvc:interceptor>
</mvc:interceptors>
```

### 1.4 多个拦截器执行顺序

```java
@Configuration
public class WebConfig implements WebMvcConfigurer {
    
    @Autowired
    private LogInterceptor logInterceptor;
    
    @Autowired
    private AuthInterceptor authInterceptor;
    
    @Override
    public void addInterceptors(InterceptorRegistry registry) {
        // 添加顺序决定执行顺序
        
        // 1. 日志拦截器
        registry.addInterceptor(logInterceptor)
            .addPathPatterns("/**")
            .order(1);  // 优先级1
        
        // 2. 认证拦截器
        registry.addInterceptor(authInterceptor)
            .addPathPatterns("/api/**")
            .order(2);  // 优先级2
    }
}
```

**执行流程**:

```
请求进入
  ↓
LogInterceptor.preHandle()
  ↓
AuthInterceptor.preHandle()
  ↓
Controller方法执行
  ↓
AuthInterceptor.postHandle()
  ↓
LogInterceptor.postHandle()
  ↓
视图渲染
  ↓
AuthInterceptor.afterCompletion()
  ↓
LogInterceptor.afterCompletion()
  ↓
响应返回
```

## 二、实战案例

### 2.1 登录验证拦截器

#### 第一步: 创建拦截器

```java
package com.example.interceptor;

import com.example.constant.Constants;
import com.example.entity.User;
import org.springframework.stereotype.Component;
import org.springframework.web.servlet.HandlerInterceptor;

import javax.servlet.http.HttpServletRequest;
import javax.servlet.http.HttpServletResponse;
import javax.servlet.http.HttpSession;

@Component
public class LoginInterceptor implements HandlerInterceptor {
    
    @Override
    public boolean preHandle(HttpServletRequest request, 
                            HttpServletResponse response, 
                            Object handler) throws Exception {
        
        // 1. 获取Session
        HttpSession session = request.getSession();
        User user = (User) session.getAttribute(Constants.USER_SESSION_KEY);
        
        // 2. 判断是否登录
        if (user == null) {
            // 未登录,返回401
            response.setStatus(HttpServletResponse.SC_UNAUTHORIZED);
            response.setContentType("application/json;charset=UTF-8");
            response.getWriter().write("{\"code\":401,\"message\":\"请先登录\"}");
            return false;  // 拦截
        }
        
        // 3. 已登录,放行
        return true;
    }
}
```

#### 第二步: 配置拦截路径

```java
@Configuration
public class WebConfig implements WebMvcConfigurer {
    
    @Autowired
    private LoginInterceptor loginInterceptor;
    
    @Override
    public void addInterceptors(InterceptorRegistry registry) {
        registry.addInterceptor(loginInterceptor)
            .addPathPatterns("/api/**")              // 拦截所有API
            .excludePathPatterns(                     // 排除以下路径
                "/api/login",                         // 登录接口
                "/api/register",                      // 注册接口
                "/api/public/**"                      // 公开接口
            );
    }
}
```

#### 第三步: 登录Controller

```java
@RestController
@RequestMapping("/api")
public class AuthController {
    
    @Autowired
    private UserService userService;
    
    @PostMapping("/login")
    public Result<User> login(@RequestBody LoginDTO loginDTO, 
                             HttpServletRequest request) {
        // 验证用户名密码
        User user = userService.login(loginDTO.getUsername(), loginDTO.getPassword());
        
        if (user != null) {
            // 保存用户信息到Session
            request.getSession().setAttribute(Constants.USER_SESSION_KEY, user);
            return Result.success(user);
        } else {
            return Result.error("用户名或密码错误");
        }
    }
    
    @PostMapping("/logout")
    public Result<Void> logout(HttpServletRequest request) {
        // 清除Session
        request.getSession().invalidate();
        return Result.success();
    }
}
```

### 2.2 权限验证拦截器

```java
@Component
public class PermissionInterceptor implements HandlerInterceptor {
    
    @Autowired
    private PermissionService permissionService;
    
    @Override
    public boolean preHandle(HttpServletRequest request, 
                            HttpServletResponse response, 
                            Object handler) throws Exception {
        
        // 1. 获取当前用户
        User user = (User) request.getSession()
            .getAttribute(Constants.USER_SESSION_KEY);
        
        if (user == null) {
            response.sendError(HttpServletResponse.SC_UNAUTHORIZED);
            return false;
        }
        
        // 2. 获取请求的URL和方法
        String uri = request.getRequestURI();
        String method = request.getMethod();
        
        // 3. 检查权限
        boolean hasPermission = permissionService.checkPermission(
            user.getId(), uri, method
        );
        
        if (!hasPermission) {
            response.setStatus(HttpServletResponse.SC_FORBIDDEN);
            response.getWriter().write("{\"code\":403,\"message\":\"没有权限\"}");
            return false;
        }
        
        return true;
    }
}
```

### 2.3 性能监控拦截器

```java
@Component
@Slf4j
public class PerformanceInterceptor implements HandlerInterceptor {
    
    private static final long SLOW_THRESHOLD = 1000;  // 慢请求阈值(毫秒)
    
    @Override
    public boolean preHandle(HttpServletRequest request, 
                            HttpServletResponse response, 
                            Object handler) throws Exception {
        request.setAttribute("startTime", System.currentTimeMillis());
        return true;
    }
    
    @Override
    public void afterCompletion(HttpServletRequest request, 
                               HttpServletResponse response, 
                               Object handler, 
                               Exception ex) throws Exception {
        
        Long startTime = (Long) request.getAttribute("startTime");
        long cost = System.currentTimeMillis() - startTime;
        
        String uri = request.getRequestURI();
        String method = request.getMethod();
        
        // 记录慢请求
        if (cost > SLOW_THRESHOLD) {
            log.warn("【慢请求】{} {} 耗时: {}ms", method, uri, cost);
        }
        
        // 记录所有请求
        log.info("【请求统计】{} {} 耗时: {}ms 状态: {}", 
                 method, uri, cost, response.getStatus());
    }
}
```

## 三、文件上传下载

### 3.1 文件上传

#### 配置上传参数

`application.yml`:

```yaml
spring:
  servlet:
    multipart:
      max-file-size: 10MB        # 单个文件最大大小
      max-request-size: 50MB     # 整个请求最大大小
      enabled: true              # 启用文件上传
      
# 自定义上传路径
file:
  upload-path: D:/uploads/
```

#### 单文件上传

```java
@RestController
@RequestMapping("/api/file")
public class FileController {
    
    @Value("${file.upload-path}")
    private String uploadPath;
    
    /**
     * 单文件上传
     */
    @PostMapping("/upload")
    public Result<String> upload(@RequestParam("file") MultipartFile file) {
        
        // 1. 校验文件
        if (file.isEmpty()) {
            return Result.error("文件不能为空");
        }
        
        // 2. 获取文件名
        String originalFilename = file.getOriginalFilename();
        String extension = originalFilename.substring(originalFilename.lastIndexOf("."));
        
        // 3. 生成新文件名(UUID避免重复)
        String newFilename = UUID.randomUUID().toString() + extension;
        
        // 4. 创建目录
        File dir = new File(uploadPath);
        if (!dir.exists()) {
            dir.mkdirs();
        }
        
        // 5. 保存文件
        try {
            File destFile = new File(dir, newFilename);
            file.transferTo(destFile);
            
            // 6. 返回文件访问URL
            String url = "/uploads/" + newFilename;
            return Result.success(url);
            
        } catch (IOException e) {
            log.error("文件上传失败", e);
            return Result.error("文件上传失败");
        }
    }
}
```

#### 多文件上传

```java
@PostMapping("/upload/batch")
public Result<List<String>> uploadBatch(@RequestParam("files") MultipartFile[] files) {
    
    List<String> urls = new ArrayList<>();
    
    for (MultipartFile file : files) {
        if (!file.isEmpty()) {
            String url = uploadSingle(file);
            if (url != null) {
                urls.add(url);
            }
        }
    }
    
    return Result.success(urls);
}
```

#### 前端表单

```html
<form action="/api/file/upload" method="post" enctype="multipart/form-data">
    <input type="file" name="file" />
    <button type="submit">上传</button>
</form>
```

**注意**: `enctype="multipart/form-data"` 是必须的!

### 3.2 文件下载

```java
@GetMapping("/download/{filename}")
public void download(@PathVariable String filename, 
                    HttpServletResponse response) throws IOException {
    
    // 1. 找到文件
    File file = new File(uploadPath, filename);
    if (!file.exists()) {
        response.sendError(HttpServletResponse.SC_NOT_FOUND);
        return;
    }
    
    // 2. 设置响应头
    response.setContentType("application/octet-stream");
    response.setHeader("Content-Disposition", 
        "attachment; filename=" + URLEncoder.encode(filename, "UTF-8"));
    response.setContentLength((int) file.length());
    
    // 3. 输出文件流
    try (InputStream in = new FileInputStream(file);
         OutputStream out = response.getOutputStream()) {
        
        byte[] buffer = new byte[4096];
        int len;
        while ((len = in.read(buffer)) != -1) {
            out.write(buffer, 0, len);
        }
        out.flush();
    }
}
```

## 四、跨域CORS配置

### 4.1 什么是跨域

**跨域 (Cross-Origin)**: 浏览器出于安全考虑,限制从一个源加载的文档或脚本与另一个源的资源进行交互。

**同源策略**: 协议、域名、端口都相同才算同源。

| URL | 是否跨域 | 原因 |
|-----|---------|------|
| http://example.com/api | ✅ 同源 | - |
| https://example.com/api | ❌ 跨域 | 协议不同 |
| http://api.example.com/api | ❌ 跨域 | 域名不同 |
| http://example.com:8080/api | ❌ 跨域 | 端口不同 |

### 4.2 CORS配置方式

#### 方式一: @CrossOrigin注解

```java
@RestController
@RequestMapping("/api")
@CrossOrigin(origins = {"http://localhost:3000", "https://example.com"})
public class UserController {
    
    @GetMapping("/users")
    public List<User> list() {
        // 允许跨域访问
        return userService.list();
    }
}
```

#### 方式二: 全局配置

```java
@Configuration
public class WebConfig implements WebMvcConfigurer {
    
    @Override
    public void addCorsMappings(CorsRegistry registry) {
        registry.addMapping("/api/**")          // 映射路径
            .allowedOrigins(                    // 允许的源
                "http://localhost:3000",
                "https://example.com"
            )
            .allowedMethods(                    // 允许的方法
                "GET", "POST", "PUT", "DELETE", "OPTIONS"
            )
            .allowedHeaders("*")                // 允许的请求头
            .allowCredentials(true)             // 允许携带Cookie
            .maxAge(3600);                      // 预检请求缓存时间(秒)
    }
}
```

#### 方式三: 过滤器

```java
@Component
@Order(Ordered.HIGHEST_PRECEDENCE)
public class CorsFilter implements Filter {
    
    @Override
    public void doFilter(ServletRequest req, ServletResponse res, 
                        FilterChain chain) throws IOException, ServletException {
        
        HttpServletResponse response = (HttpServletResponse) res;
        HttpServletRequest request = (HttpServletRequest) req;
        
        response.setHeader("Access-Control-Allow-Origin", "*");
        response.setHeader("Access-Control-Allow-Methods", 
            "GET, POST, PUT, DELETE, OPTIONS");
        response.setHeader("Access-Control-Allow-Headers", "*");
        response.setHeader("Access-Control-Max-Age", "3600");
        
        // 处理预检请求
        if ("OPTIONS".equalsIgnoreCase(request.getMethod())) {
            response.setStatus(HttpServletResponse.SC_OK);
        } else {
            chain.doFilter(req, res);
        }
    }
}
```

## 总结

本节讲解了Spring MVC的高级特性:

**核心要点**:
1. 拦截器在Controller执行前后进行拦截
2. 拦截器支持Spring Bean注入,过滤器不支持
3. 多个拦截器按order顺序执行
4. 文件上传使用MultipartFile接收
5. CORS配置解决跨域问题

**最佳实践**:
- 登录验证用拦截器,编码转换用过滤器
- 拦截器要合理设置排除路径
- 文件上传要校验文件大小和类型
- 生产环境CORS要指定具体的域名,不要用*
- 敏感操作要同时做前端和后端验证
