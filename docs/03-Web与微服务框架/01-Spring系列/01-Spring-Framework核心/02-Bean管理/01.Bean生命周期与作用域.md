## 一、Bean生命周期详解

### 1.1 什么是Bean生命周期

**Bean生命周期** 指的是Bean从创建到销毁的整个过程,Spring容器在这个过程中会执行一系列的操作。

### 1.2 Bean生命周期的7个阶段

```
1. 实例化(Instantiation)
   ↓
2. 属性赋值(Populate)
   ↓
3. 初始化前处理(BeanPostProcessor.postProcessBeforeInitialization)
   ↓
4. 初始化(Initialization)
   - @PostConstruct
   - InitializingBean.afterPropertiesSet()
   - init-method
   ↓
5. 初始化后处理(BeanPostProcessor.postProcessAfterInitialization)
   ↓
6. 使用(In Use)
   ↓
7. 销毁(Destruction)
   - @PreDestroy
   - DisposableBean.destroy()
   - destroy-method
```

### 1.3 完整生命周期代码演示

#### 第一步: 创建Bean类

```java
package com.example.bean;

import org.springframework.beans.factory.DisposableBean;
import org.springframework.beans.factory.InitializingBean;
import javax.annotation.PostConstruct;
import javax.annotation.PreDestroy;

public class UserBean implements InitializingBean, DisposableBean {
    
    private String name;
    private Integer age;
    
    public UserBean() {
        System.out.println("【1】调用构造方法实例化Bean");
    }
    
    public void setName(String name) {
        System.out.println("【2】调用setter方法注入属性");
        this.name = name;
    }
    
    public void setAge(Integer age) {
        this.age = age;
    }
    
    @PostConstruct
    public void postConstruct() {
        System.out.println("【3】@PostConstruct注解方法");
    }
    
    @Override
    public void afterPropertiesSet() throws Exception {
        System.out.println("【4】InitializingBean.afterPropertiesSet()方法");
    }
    
    public void initMethod() {
        System.out.println("【5】自定义init-method方法");
    }
    
    @PreDestroy
    public void preDestroy() {
        System.out.println("【6】@PreDestroy注解方法");
    }
    
    @Override
    public void destroy() throws Exception {
        System.out.println("【7】DisposableBean.destroy()方法");
    }
    
    public void destroyMethod() {
        System.out.println("【8】自定义destroy-method方法");
    }
    
    @Override
    public String toString() {
        return "UserBean{name='" + name + "', age=" + age + "}";
    }
}
```

#### 第二步: 配置Bean

**XML方式**:

```xml
<bean id="userBean" 
      class="com.example.bean.UserBean"
      init-method="initMethod"
      destroy-method="destroyMethod">
    <property name="name" value="张三"/>
    <property name="age" value="20"/>
</bean>
```

**注解方式**:

```java
@Configuration
public class BeanConfig {
    
    @Bean(initMethod = "initMethod", destroyMethod = "destroyMethod")
    public UserBean userBean() {
        UserBean userBean = new UserBean();
        userBean.setName("张三");
        userBean.setAge(20);
        return userBean;
    }
}
```

#### 第三步: 测试

```java
public class LifecycleTest {
    
    public static void main(String[] args) {
        ClassPathXmlApplicationContext ctx = 
            new ClassPathXmlApplicationContext("applicationContext.xml");
        
        UserBean userBean = (UserBean) ctx.getBean("userBean");
        System.out.println("【使用中】" + userBean);
        
        // 关闭容器,触发销毁
        ctx.close();
    }
}
```

**输出结果**:

```
【1】调用构造方法实例化Bean
【2】调用setter方法注入属性
【3】@PostConstruct注解方法
【4】InitializingBean.afterPropertiesSet()方法
【5】自定义init-method方法
【使用中】UserBean{name='张三', age=20}
【6】@PreDestroy注解方法
【7】DisposableBean.destroy()方法
【8】自定义destroy-method方法
```

### 1.4 BeanPostProcessor后置处理器

**作用**: 在Bean初始化前后进行额外的处理,是Spring扩展的核心机制。

```java
package com.example.processor;

import org.springframework.beans.BeansException;
import org.springframework.beans.factory.config.BeanPostProcessor;
import org.springframework.stereotype.Component;

@Component
public class MyBeanPostProcessor implements BeanPostProcessor {
    
    /**
     * 初始化之前执行
     */
    @Override
    public Object postProcessBeforeInitialization(Object bean, String beanName) 
            throws BeansException {
        if ("userBean".equals(beanName)) {
            System.out.println("【BPP】Bean初始化前: " + beanName);
        }
        return bean;
    }
    
    /**
     * 初始化之后执行
     */
    @Override
    public Object postProcessAfterInitialization(Object bean, String beanName) 
            throws BeansException {
        if ("userBean".equals(beanName)) {
            System.out.println("【BPP】Bean初始化后: " + beanName);
        }
        return bean;
    }
}
```

**典型应用场景**:
- AOP代理对象创建
- 属性校验
- 日志记录
- 缓存增强

## 二、Bean作用域

### 2.1 五种作用域对比

| 作用域 | 说明 | 使用场景 |
|--------|------|----------|
| singleton | 单例(默认) | 无状态服务、工具类 |
| prototype | 原型(多例) | 有状态的对象 |
| request | 请求级别 | Web应用,每个请求一个实例 |
| session | 会话级别 | Web应用,每个会话一个实例 |
| application | 应用级别 | Web应用,整个应用共享 |

### 2.2 singleton单例模式

**特点**:
- Spring默认作用域
- 整个容器中只有一个实例
- 线程不安全(需要注意)

**配置**:

```java
@Component
@Scope("singleton")  // 可以省略,默认就是singleton
public class UserService {
    // ...
}
```

**测试**:

```java
UserService service1 = ctx.getBean(UserService.class);
UserService service2 = ctx.getBean(UserService.class);
System.out.println(service1 == service2);  // true
```

### 2.3 prototype原型模式

**特点**:
- 每次获取都创建新实例
- Spring不管理prototype Bean的生命周期
- 不会调用destroy方法

**配置**:

```java
@Component
@Scope("prototype")
public class OrderService {
    private String orderId;
    
    public void setOrderId(String orderId) {
        this.orderId = orderId;
    }
}
```

**测试**:

```java
OrderService service1 = ctx.getBean(OrderService.class);
OrderService service2 = ctx.getBean(OrderService.class);
System.out.println(service1 == service2);  // false
```

**使用场景**:
- 有状态的对象(如订单、购物车)
- 需要隔离数据的场景

### 2.4 Web作用域

需要在Web环境中使用:

```java
// 请求级别
@Scope(value = WebApplicationContext.SCOPE_REQUEST, proxyMode = ScopedProxyMode.TARGET_CLASS)
public class RequestData {
    // 每个HTTP请求一个实例
}

// 会话级别
@Scope(value = WebApplicationContext.SCOPE_SESSION, proxyMode = ScopedProxyMode.TARGET_CLASS)
public class UserData {
    // 每个用户会话一个实例
}
```

## 三、Bean高级特性

### 3.1 FactoryBean工厂Bean

**作用**: 用于创建复杂对象,提供额外的定制逻辑。

**与普通Bean的区别**:
- 普通Bean: 直接返回对象本身
- FactoryBean: 通过getObject()方法返回对象

#### 示例: 创建MyBatis的SqlSessionFactory

```java
package com.example.factory;

import org.apache.ibatis.session.SqlSessionFactory;
import org.apache.ibatis.session.SqlSessionFactoryBuilder;
import org.springframework.beans.factory.FactoryBean;
import org.springframework.stereotype.Component;

import java.io.InputStream;

@Component
public class SqlSessionFactoryBean implements FactoryBean<SqlSessionFactory> {
    
    @Override
    public SqlSessionFactory getObject() throws Exception {
        // 复杂的创建逻辑
        InputStream inputStream = getClass().getClassLoader()
            .getResourceAsStream("mybatis-config.xml");
        return new SqlSessionFactoryBuilder().build(inputStream);
    }
    
    @Override
    public Class<?> getObjectType() {
        return SqlSessionFactory.class;
    }
    
    @Override
    public boolean isSingleton() {
        return true;
    }
}
```

**使用**:

```java
@Autowired
private SqlSessionFactory sqlSessionFactory;  // 自动获取getObject()返回的对象
```

**获取FactoryBean本身**:

```java
// 加&符号获取FactoryBean本身
SqlSessionFactoryBean factoryBean = (SqlSessionFactoryBean) ctx.getBean("&sqlSessionFactoryBean");
```

### 3.2 Bean的懒加载

**默认行为**: singleton Bean在容器启动时创建

**懒加载**: 第一次使用时才创建

**配置**:

```java
@Component
@Lazy
public class ExpensiveService {
    public ExpensiveService() {
        System.out.println("ExpensiveService被创建了");
    }
}
```

**全局懒加载**:

```yaml
spring:
  main:
    lazy-initialization: true
```

**适用场景**:
- 启动缓慢的服务
- 很少使用的功能模块
- 依赖外部资源的服务

### 3.3 Bean的条件装配

根据条件决定是否创建Bean:

```java
@Configuration
public class ConditionalConfig {
    
    /**
     * 仅在Windows系统创建
     */
    @Bean
    @ConditionalOnOs(OS.WINDOWS)
    public WindowsService windowsService() {
        return new WindowsService();
    }
    
    /**
     * 仅在Linux系统创建
     */
    @Bean
    @ConditionalOnOs(OS.LINUX)
    public LinuxService linuxService() {
        return new LinuxService();
    }
    
    /**
     * 仅当classpath中存在Redis类时创建
     */
    @Bean
    @ConditionalOnClass(RedisTemplate.class)
    public RedisService redisService() {
        return new RedisService();
    }
    
    /**
     * 仅当配置文件中存在指定属性时创建
     */
    @Bean
    @ConditionalOnProperty(name = "sms.enabled", havingValue = "true")
    public SmsService smsService() {
        return new SmsService();
    }
}
```

## 四、最佳实践

### 4.1 生命周期方法选择优先级

**初始化方法**:
1. ✅ **推荐**: `@PostConstruct` - JSR-250标准,与框架无关
2. ⚠️ 次选: `InitializingBean` - 侵入性强,耦合Spring
3. ⚠️ 备选: `init-method` - XML配置,不够直观

**销毁方法**:
1. ✅ **推荐**: `@PreDestroy` - JSR-250标准
2. ⚠️ 次选: `DisposableBean` - 侵入性强
3. ⚠️ 备选: `destroy-method` - XML配置

### 4.2 作用域选择原则

**singleton适用场景**:
- Controller、Service、Dao层
- 工具类、配置类
- 无状态的对象

**prototype适用场景**:
- 有状态的业务对象
- 需要隔离的数据容器
- 多线程环境下的非线程安全对象

### 4.3 线程安全问题

**singleton Bean不是线程安全的**!

**解决方案**:

```java
@Service
public class OrderService {
    
    // ❌ 错误: 成员变量在多线程下不安全
    // private int count = 0;
    
    // ✅ 正确: 使用局部变量
    public void createOrder() {
        int count = 0;  // 局部变量,线程安全
        // ...
    }
    
    // ✅ 正确: 使用ThreadLocal
    private ThreadLocal<Integer> threadLocalCount = ThreadLocal.withInitial(() -> 0);
    
    // ✅ 正确: 使用同步锁
    private final Object lock = new Object();
    private int safeCount = 0;
    
    public void increment() {
        synchronized (lock) {
            safeCount++;
        }
    }
}
```

### 4.4 避免循环依赖

**问题**: A依赖B,B又依赖A

**解决**:
1. 重新设计,消除循环依赖
2. 使用`@Lazy`延迟加载
3. 使用setter注入代替构造器注入

```java
@Service
public class ServiceA {
    
    @Autowired
    @Lazy  // 打破循环依赖
    private ServiceB serviceB;
}

@Service
public class ServiceB {
    
    @Autowired
    private ServiceA serviceA;
}
```

## 总结

本节深入讲解了Spring Bean的生命周期和作用域:

**核心要点**:
1. Bean生命周期包含7个阶段,可通过多种方式扩展
2. BeanPostProcessor是Spring扩展的核心机制
3. singleton是默认作用域,prototype用于有状态对象
4. FactoryBean用于创建复杂对象
5. 注意singleton Bean的线程安全问题

**最佳实践**:
- 优先使用@PostConstruct和@PreDestroy
- 无状态服务用singleton,有状态用prototype
- 避免在singleton Bean中使用可变成员变量
- 合理设计,避免循环依赖
- 使用@Lazy优化启动速度
