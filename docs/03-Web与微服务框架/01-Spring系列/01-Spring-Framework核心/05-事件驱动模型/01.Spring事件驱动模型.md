## 一、事件驱动模型概述

### 1.1 什么是事件驱动?

**事件驱动(Event-Driven)**是一种编程范式,核心思想是:

> **当某个动作发生时(事件),通知所有感兴趣的组件(监听器)做出响应。**

**生活中的例子**:
- 📱 手机收到短信 → 触发通知铃声震动
- 🔔 门铃被按下 → 主人去开门
- 📧 收到邮件 → 客户端弹出提醒

### 1.2 Spring为什么需要事件驱动?

**实际应用场景**:

| 场景 | 说明 |
|------|------|
| **用户注册** | 注册成功后发送欢迎邮件、初始化用户数据、记录日志 |
| **订单支付** | 支付成功后扣减库存、发送通知、更新积分 |
| **文章发布** | 发布后推送给订阅者、更新搜索引擎、清理缓存 |
| **系统启动** | 启动时加载配置、预热缓存、检查依赖 |

**核心优势**:
1. ✅ **解耦**: 发布者和订阅者不需要知道彼此的存在
2. ✅ **扩展性**: 新增监听器无需修改原有代码
3. ✅ **灵活性**: 可以动态添加/移除监听器
4. ✅ **异步处理**: 支持异步事件提升性能

## 二、Spring事件核心组件

### 2.1 三大核心接口

```
┌─────────────────────────────────────────────────────┐
│           Spring事件驱动三大核心组件                   │
├─────────────────────────────────────────────────────┤
│                                                     │
│  1. ApplicationEvent (事件)                          │
│     - 继承java.util.EventObject                      │
│     - 封装事件源和事件数据                            │
│                                                     │
│  2. ApplicationListener (监听器)                     │
│     - 继承java.util.EventListener                    │
│     - onApplicationEvent()方法处理事件                │
│                                                     │
│  3. ApplicationEventPublisher (发布者)               │
│     - publishEvent()方法发布事件                      │
│     - ApplicationContext默认实现                     │
│                                                     │
└─────────────────────────────────────────────────────┘
```

### 2.2 工作流程

```
发布者(Publisher)
    ↓ publishEvent(event)
事件广播器(ApplicationEventMulticaster)
    ↓ 遍历所有监听器
监听器1(Listener1) → onApplicationEvent(event)
监听器2(Listener2) → onApplicationEvent(event)
监听器3(Listener3) → onApplicationEvent(event)
```

## 三、快速入门案例

### 3.1 案例: 用户注册事件

#### 步骤1: 定义事件类

```java
package com.dontalk.event;

import org.springframework.context.ApplicationEvent;

/**
 * 用户注册事件
 */
public class UserRegisterEvent extends ApplicationEvent {
    
    private final Long userId;
    private final String username;
    private final String email;
    
    /**
     * 构造方法
     * @param source 事件源(通常是发布者的对象)
     * @param userId 用户ID
     * @param username 用户名
     * @param email 邮箱
     */
    public UserRegisterEvent(Object source, Long userId, String username, String email) {
        super(source);
        this.userId = userId;
        this.username = username;
        this.email = email;
    }
    
    // Getter方法
    public Long getUserId() { return userId; }
    public String getUsername() { return username; }
    public String getEmail() { return email; }
}
```

**关键点**:
- 必须继承`ApplicationEvent`
- 调用父类构造方法`super(source)`
- 封装事件相关的数据

#### 步骤2: 创建监听器

**方式1: 实现ApplicationListener接口**

```java

        log.info("【邮件监听器】收到用户注册事件: userId={}, username={}, email={}", 
                event.getUserId(), event.getUsername(), event.getEmail());
        
        // 模拟发送邮件
        sendWelcomeEmail(event.getEmail(), event.getUsername());
    }
    
    private void sendWelcomeEmail(String email, String username) {
        log.info("发送欢迎邮件到: {}", email);
        // TODO: 调用邮件服务API
    }
}
```

#### 步骤3: 发布事件

```java
package com.dontalk.service;

import com.dontalk.event.UserRegisterEvent;
import lombok.RequiredArgsConstructor;
import lombok.extern.slf4j.Slf4j;
import org.springframework.context.ApplicationEventPublisher;
import org.springframework.stereotype.Service;

@Slf4j
@Service
@RequiredArgsConstructor
public class UserService {
    
    private final ApplicationEventPublisher eventPublisher;
    
    /**
     * 用户注册
     */
    public void register(String username, String email) {
        log.info("开始注册用户: {}", username);
        
        // 1. 保存用户到数据库
        Long userId = saveUserToDatabase(username, email);
        
        // 2. 发布用户注册事件
        UserRegisterEvent event = new UserRegisterEvent(this, userId, username, email);
        eventPublisher.publishEvent(event);
        
        log.info("用户注册成功: userId={}", userId);
    }
    
    private Long saveUserToDatabase(String username, String email) {
        // 模拟保存用户
        log.info("保存用户到数据库: username={}, email={}", username, email);
        return 1L; // 返回用户ID
    }
}
```

**关键点**:
- 注入`ApplicationEventPublisher`
- 创建事件对象并传入事件源(this)
- 调用`publishEvent()`发布事件

## 四、高级特性

### 4.1 异步事件

**问题**: 默认情况下,事件发布是**同步阻塞**的,所有监听器执行完成后才返回。

**解决方案**: 使用`@Async`实现异步事件。

#### 配置异步支持

```java
@Configuration
@EnableAsync  // 启用异步支持
public class AsyncConfig {
    
    @Bean
    public Executor taskExecutor() {
        ThreadPoolTaskExecutor executor = new ThreadPoolTaskExecutor();
        executor.setCorePoolSize(5);
        executor.setMaxPoolSize(10);
        executor.setQueueCapacity(100);
        executor.setThreadNamePrefix("event-async-");
        return executor;
    }
}
```

#### 异步监听器

```java
@Slf4j
@Component
public class AsyncEmailListener {
    
    @Async  // 异步执行
    @EventListener
    public void handleUserRegister(UserRegisterEvent event) {
        log.info("【异步邮件监听器】线程: {}, 收到事件: {}", 
                Thread.currentThread().getName(), event.getUserId());
        
        // 模拟耗时操作
        try {
            Thread.sleep(2000);
        } catch (InterruptedException e) {
            e.printStackTrace();
        }
        
        sendWelcomeEmail(event.getEmail());
    }
}
```

**效果**: 
- 主线程不会等待监听器执行完成
- 监听器在独立线程池中执行
- 提升系统响应速度

### 4.2 事件顺序控制

**问题**: 多个监听器的执行顺序不确定。

**解决方案**: 使用`@Order`注解控制顺序。

```java
@Slf4j
@Component
@Order(1)  // 优先级最高,最先执行
public class FirstListener {
    
    @EventListener
    public void handleEvent(UserRegisterEvent event) {
        log.info("【监听器1】最先执行");
    }
}

@Slf4j
@Component
@Order(2)  // 优先级第二
public class SecondListener {
    
    @EventListener
    public void handleEvent(UserRegisterEvent event) {
        log.info("【监听器2】第二个执行");
    }
}

@Slf4j
@Component
@Order(3)  // 优先级最低,最后执行
public class ThirdListener {
    
    @EventListener
    public void handleEvent(UserRegisterEvent event) {
        log.info("【监听器3】最后执行");
    }
}
```

**注意**: `@Order`值越小,优先级越高。

### 4.3 条件监听

**场景**: 只在特定条件下才处理事件。

```java
@Component
public class ConditionalListener {
    
    @EventListener(condition = "#event.userId > 100")
    public void handleEvent(UserRegisterEvent event) {
        log.info("只处理userId大于100的事件: {}", event.getUserId());
    }
}
```

**SpEL表达式**:
- `#event`: 引用事件对象
- `#event.userId`: 访问事件属性
- 支持复杂的布尔表达式

### 4.4 泛型事件(Spring 4.2+)

**优势**: 简化事件定义,无需创建事件类。

```java
// 发布泛型事件
@Service
@RequiredArgsConstructor
public class OrderService {
    
    private final ApplicationEventPublisher publisher;
    
    public void payOrder(Long orderId) {
        // 直接发布任意对象作为事件
        publisher.publishEvent(new OrderPaidEvent(orderId));
    }
}

// 监听泛型事件
@Component
public class OrderListener {
    
    @EventListener
    public void handleOrderPaid(OrderPaidEvent event) {
        log.info("订单支付成功: orderId={}", event.getOrderId());
    }
}

// 事件类
@Data
@AllArgsConstructor
public class OrderPaidEvent {
    private Long orderId;
}
```

## 五、实战案例

### 5.1 订单支付完整流程

#### 事件定义

```java
/**
 * 订单支付事件
 */
@Data
public class OrderPaidEvent extends ApplicationEvent {
    private Long orderId;
    private Long userId;
    private BigDecimal amount;
    
    public OrderPaidEvent(Object source, Long orderId, Long userId, BigDecimal amount) {
        super(source);
        this.orderId = orderId;
        this.userId = userId;
        this.amount = amount;
    }
}
```

#### 监听器1: 扣减库存

```java
@Component
@Slf4j
public class InventoryListener {
    
    @Autowired
    private InventoryService inventoryService;
    
    @EventListener
    @Order(1)
    public void deductInventory(OrderPaidEvent event) {
        log.info("扣减库存: orderId={}", event.getOrderId());
        inventoryService.deductStock(event.getOrderId());
    }
}
```

#### 监听器2: 增加积分

```java
@Component
@Slf4j
public class PointsListener {
    
    @Autowired
    private PointsService pointsService;
    
    @Async  // 异步执行
    @EventListener
    @Order(2)
    public void addPoints(OrderPaidEvent event) {
        log.info("增加用户积分: userId={}, amount={}", event.getUserId(), event.getAmount());
        int points = event.getAmount().multiply(new BigDecimal("10")).intValue();
        pointsService.addPoints(event.getUserId(), points);
    }
}
```

#### 监听器3: 发送通知

```java
@Slf4j
public class NotificationListener {

```

#### 监听器4: 记录日志

```java
@Component
@Slf4j
public class AuditLogListener {
    
    @Autowired
    private AuditLogService auditLogService;
    
    @EventListener
    @Order(4)
    public void recordLog(OrderPaidEvent event) {
        log.info("记录审计日志: orderId={}, amount={}", event.getOrderId(), event.getAmount());
        auditLogService.saveLog("ORDER_PAID", event.getOrderId());
```

#### 发布事件

```java
@Service
@RequiredArgsConstructor
public class PaymentService {
    
    private final ApplicationEventPublisher eventPublisher;
    private final OrderMapper orderMapper;
    
    @Transactional
    public void payOrder(Long orderId) {
        // 1. 查询订单
        Order order = orderMapper.selectById(orderId);
        if (order == null) {
            throw new BusinessException("订单不存在");
        }
        
        // 2. 更新订单状态
        order.setStatus(OrderStatus.PAID);
        order.setPayTime(LocalDateTime.now());
        orderMapper.updateById(order);
        
        // 3. 发布支付成功事件
        OrderPaidEvent event = new OrderPaidEvent(
            this, 
            order.getId(), 
            order.getUserId(), 
            order.getAmount()
        );
        eventPublisher.publishEvent(event);
        
        log.info("订单支付成功: orderId={}", orderId);
    }
}
```

## 六、最佳实践

### 6.1 命名规范

| 类型 | 命名规范 | 示例 |
|------|---------|------|
| 事件类 | XxxEvent | UserRegisterEvent、OrderPaidEvent |
| 监听器类 | XxxListener | EmailSendListener、InventoryListener |
| 处理方法 | handleXxx / onXxx | handleUserRegister、onOrderPaid |

### 6.2 设计原则

1. **单一职责**: 一个监听器只做一件事
2. **轻量级**: 监听器中避免耗时操作,使用异步
3. **幂等性**: 监听器可能被多次调用,确保幂等
4. **异常处理**: 监听器中捕获异常,避免影响其他监听器
5. **事务边界**: 注意监听器是否在事务中执行

### 6.3 常见陷阱

#### 陷阱1: 循环依赖

```java
// ❌ 错误: A监听器发布B事件,B监听器发布A事件,形成循环
@Component
public class ListenerA {
    @EventListener
    public void handleB(BEvent event) {
        publisher.publishEvent(new AEvent());
    }
}

@Component
public class ListenerB {
    @EventListener
    public void handleA(AEvent event) {
        publisher.publishEvent(new BEvent());
    }
}
```

**解决**: 重新设计事件流,避免循环。

#### 陷阱2: 事务未提交就查询

```java
// ❌ 错误: 监听器在事务提交前执行,查不到数据
@Transactional
public void createUser() {
    userMapper.insert(user);
    publisher.publishEvent(new UserCreatedEvent(user.getId()));
}

@Component
public class UserListener {
    @EventListener
    public void handleUserCreated(UserCreatedEvent event) {
        // 此时事务可能还未提交,查询可能失败
        User user = userMapper.selectById(event.getUserId());
    }
}
```

**解决**: 使用`@TransactionalEventListener(phase = TransactionPhase.AFTER_COMMIT)`

```java
@Component
public class UserListener {
    
    @TransactionalEventListener(phase = TransactionPhase.AFTER_COMMIT)
    public void handleUserCreated(UserCreatedEvent event) {
        // 事务提交后才执行,能查到数据
        User user = userMapper.selectById(event.getUserId());
    }
}
```

#### 陷阱3: 忘记加@Component

```java
// ❌ 错误: 监听器没有被Spring管理,不会接收到事件
public class MyListener {
    @EventListener
    public void handleEvent(MyEvent event) {
        // ...
    }
}

// ✅ 正确: 加上@Component注解
@Component
public class MyListener {
    @EventListener
    public void handleEvent(MyEvent event) {
        // ...
    }
}
```

## 七、总结

### 7.1 核心要点

1. ✅ **三大组件**: ApplicationEvent、ApplicationListener、ApplicationEventPublisher
2. ✅ **两种监听方式**: 实现接口 vs @EventListener注解(推荐)
3. ✅ **异步支持**: @Async + @EnableAsync
4. ✅ **顺序控制**: @Order注解
5. ✅ **事务同步**: @TransactionalEventListener
6. ✅ **条件监听**: SpEL表达式

### 7.2 适用场景

| 场景 | 是否适合 |
|------|---------|
| 用户注册后多操作 | ✅ 非常适合 |
| 订单支付后处理 | ✅ 非常适合 |
| 系统启动初始化 | ✅ 适合 |
| 实时同步数据 | ⚠️ 考虑消息队列 |
| 跨服务通信 | ❌ 不适合,用MQ |

### 7.3 与消息队列对比

| 特性 | Spring事件 | 消息队列(RabbitMQ/Kafka) |
|------|-----------|------------------------|
| 作用域 | 单应用内 | 分布式系统 |
| 可靠性 | 低(内存) | 高(持久化) |
| 性能 | 高(无网络开销) | 中(网络传输) |
| 复杂度 | 低 | 高 |
| 适用场景 | 模块解耦 | 服务解耦 |

## 八、常见问题

### Q1: 事件监听器能否捕获异常?

**A**: 可以,但建议每个监听器自己处理异常:

```java
@EventListener
public void handleEvent(MyEvent event) {
    try {
        // 业务逻辑
    } catch (Exception e) {
        log.error("处理事件失败", e);
        // 不要抛出异常,避免影响其他监听器
    }
}
```

### Q2: 如何停止监听某个事件?

**A**: Spring没有直接移除监听器的API,可以通过条件判断:

```java
@Component
public class DynamicListener {
    
    private volatile boolean enabled = true;
    
    @EventListener
    public void handleEvent(MyEvent event) {
        if (!enabled) {
            return; // 禁用时不处理
        }
        // 正常处理
    }
    
    public void disable() {
        this.enabled = false;
    }
}
```

### Q3: 事件监听器能否返回值?

**A**: 不能,监听器方法必须是void类型。如果需要反馈,可以在事件中设置结果字段。
