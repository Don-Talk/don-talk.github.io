## 一、依赖注入方式

### 1.1 注入方式概述

**问题导入**: 依赖注入有几种方式?

**两种主要方式**:
- **setter注入**
  - 简单类型(String、int等)
  - 引用类型(对象,**很常用**)
  
- **构造器注入**
  - 简单类型
  - 引用类型

### 1.2 setter方式注入

#### 引用类型注入

**配置示例**:

```xml
<bean id="bookDao" class="com.itheima.dao.impl.BookDaoImpl"/>

<bean id="bookService" class="com.itheima.service.impl.BookServiceImpl">
    <!--使用property标签注入引用类型-->
    <property name="bookDao" ref="bookDao"/>

```

**关键点**:
- 使用`\<property>`标签
- `name`属性:对应setter方法名(去掉set,首字母小写)
- `ref`属性:引用另一个Bean的id

#### 简单类型注入

**配置示例**:

```xml
<bean id="bookService" class="com.itheima.service.impl.BookServiceImpl">
    <!--注入简单类型-->
    <property name="name" value="Spring教程"/>
    <property name="price" value="99.9"/>
</bean>
```

**关键点**:
- 使用`\<property>`标签
- `name`属性:对应setter方法名
- `value`属性:直接赋值(不是ref)

### 1.3 构造器方式注入

#### 引用类型注入

**配置示例**:

```xml
<bean id="bookDao" class="com.itheima.dao.impl.BookDaoImpl"/>

    <!--使用constructor-arg标签注入-->

**关键点**:
- 使用`\<constructor-arg>`标签
- `name`属性:对应构造器参数名
- `ref`属性:引用另一个Bean的id

    <constructor-arg name="name" value="Spring教程"/>
    <constructor-arg name="price" value="99.9"/>

```

### 1.4 依赖注入方式选择

**选择建议**:

1. **强制依赖使用构造器注入**
   - 使用setter注入有概率不进行注入导致null对象出现
   - 构造器注入能保证依赖一定被注入

2. **可选依赖使用setter注入**
   - 灵活性强,可以选择不注入

3. **Spring框架倡导使用构造器**
   - 第三方框架内部大多数采用构造器注入
   - 相对严谨,符合不可变对象设计理念

4. **混合使用**
   - 使用构造器注入完成强制依赖的注入
   - 使用setter注入完成可选依赖的注入

5. **实际情况分析**
   - 如果受控对象没有提供setter方法就必须使用构造器注入

6. **自己开发的模块推荐使用setter注入**
   - 便于测试和扩展

## 二、依赖自动装配

### 2.1 自动装配概念

**问题导入**: 如何配置按照类型自动装配?

**定义**: IoC容器根据bean所依赖的资源在容器中自动查找并注入到bean中的过程称为自动装配

**自动装配方式**:
- **按类型**(byType,**常用**)
- 按名称(byName)
- 按构造方法
- 不启用自动装配

### 2.2 自动装配配置

#### 按类型自动装配(byType)

配置中使用bean标签autowire属性设置自动装配的类型:

```xml
<bean id="bookDao" class="com.itheima.dao.impl.BookDaoImpl"/>

<!--自动按类型装配,Spring会查找类型为BookDao的bean并注入-->
<bean id="bookService" class="com.itheima.service.impl.BookServiceImpl" autowire="byType"/>
```

**工作原理**:
- Spring扫描BookService中的所有setter方法
- 对于每个setter方法,查找容器中与该参数类型匹配的Bean
- 如果找到唯一匹配的Bean,自动注入

#### 按名称自动装配(byName)

```xml
<bean id="bookDao" class="com.itheima.dao.impl.BookDaoImpl"/>

<!--自动按名称装配,Spring会查找id为bookDao的bean并注入-->
<bean id="bookService" class="com.itheima.service.impl.BookServiceImpl" autowire="byName"/>
```

**工作原理**:
- Spring扫描BookService中的所有setter方法
- 将setter方法名转换为属性名(去掉set,首字母小写)
- 查找容器中与该属性名相同的Bean

### 2.3 自动装配特征

**重要特性**:

1. 自动装配用于**引用类型**依赖注入,不能对简单类型进行操作

2. 使用按类型装配时(byType)**必须保障容器中相同类型的bean唯一**,推荐使用

3. 使用按名称装配时(byName)**必须保障容器中具有指定名称的bean**,因变量名与配置耦合,不推荐使用

4. 自动装配优先级**低于**setter注入与构造器注入,同时出现时自动装配配置失效

**示例**:

```xml
<bean id="bookDao" class="com.itheima.dao.impl.BookDaoImpl"/>

<!--手动配置的property优先级高于autowire-->
<bean id="bookService" class="com.itheima.service.impl.BookServiceImpl" autowire="byType">
    <property name="bookDao" ref="bookDao"/>  <!--这个配置会生效,autowire被忽略-->
</bean>
```

## 三、集合注入

### 3.1 注入数组类型数据

```xml
<bean id="myBean" class="com.example.MyBean">
    <property name="array">
        <array>
            <value>100</value>
            <value>200</value>
            <value>300</value>
        </array>
    </property>
</bean>
```

对应的Java类:

```java
public class MyBean {
    private int[] array;
    
    public void setArray(int[] array) {
        this.array = array;
    }
}
```

### 3.2 注入List类型数据

```xml
<property name="list">
    <list>
        <value>itcast</value>
        <value>itheima</value>
        <value>boxuegu</value>
        <value>chuanzhihui</value>
    </list>
</property>
```

对应的Java类:

```java
public class MyBean {
    private List<String> list;
    
    public void setList(List<String> list) {
        this.list = list;
    }
}
```

### 3.3 注入Set类型数据

```xml
<property name="set">
    <set>
        <value>itcast</value>
        <value>itheima</value>
        <value>boxuegu</value>
        <value>boxuegu</value>  <!--重复值会被自动去重-->
    </set>
</property>
```

对应的Java类:

```java
public class MyBean {
    private Set<String> set;
    
    public void setSet(Set<String> set) {
        this.set = set;
    }
}
```

### 3.4 注入Map类型数据

```xml
<property name="map">
    <map>
        <entry key="country" value="china"/>
        <entry key="province" value="henan"/>
        <entry key="city" value="kaifeng"/>
    </map>
</property>
```

对应的Java类:

```java
public class MyBean {
    private Map<String, String> map;
    
    public void setMap(Map<String, String> map) {
        this.map = map;
    }
}
```

### 3.5 注入Properties类型数据

```xml
<property name="properties">
    <props>
        <prop key="country">china</prop>
        <prop key="province">henan</prop>
        <prop key="city">kaifeng</prop>
    </props>
</property>
```

对应的Java类:

```java
public class MyBean {
    private Properties properties;
    
    public void setProperties(Properties properties) {
        this.properties = properties;
    }
}
```

### 3.6 集合注入注意事项

**重要说明**: 
- `\<property>`标签表示setter方式注入
- `\<constructor-arg>`标签表示构造器方式注入
- 两种方式都可以在内部使用`\<array>`、`\<list>`、`\<set>`、`\<map>`、`\<props>`标签

**构造器方式集合注入示例**:

```xml
<bean id="myBean" class="com.example.MyBean">
    <constructor-arg name="list">
        <list>
            <value>item1</value>
            <value>item2</value>
        </list>
    </constructor-arg>
</bean>
```

## 四、Bean的生命周期【了解】

### 4.1 生命周期相关概念

**核心概念**:
- **生命周期**: 从创建到消亡的完整过程
- **bean生命周期**: bean从创建到销毁的整体过程
- **bean生命周期控制**: 在bean创建后到销毁前做一些事情

### 4.2 生命周期控制方法

#### 方式一: 配置init-method和destroy-method

提供生命周期控制方法:

```java
public class BookDaoImpl implements BookDao {
    public void save() {
        System.out.println("book dao save ...");
    }
    
    //表示bean初始化对应的操作
    public void init(){
        System.out.println("init...");
    }
    
    //表示bean销毁前对应的操作
    public void destory(){
        System.out.println("destory...");
    }
}
```

applicationContext.xml配置:

```xml
<!--init-method:设置bean初始化生命周期回调函数-->
<!--destroy-method:设置bean销毁生命周期回调函数,仅适用于单例对象-->
<bean id="bookDao" class="com.itheima.dao.impl.BookDaoImpl" 
      init-method="init" 
      destroy-method="destory"/>
```

测试类:

```java
public class AppForLifeCycle {
    public static void main(String[] args) {
        //此处需要使用实现类类型,接口类型没有close方法
        ClassPathXmlApplicationContext ctx = new ClassPathXmlApplicationContext("applicationContext.xml");
        
        BookDao bookDao = (BookDao) ctx.getBean("bookDao");
        bookDao.save();
        
        //关闭容器,执行销毁的方法
        ctx.close();
    }
}
```

#### 方式二: 实现InitializingBean和DisposableBean接口

```java
public class BookServiceImpl implements BookService, InitializingBean, DisposableBean {
    private BookDao bookDao;
    
    public void setBookDao(BookDao bookDao) {
        System.out.println("set .....");
        this.bookDao = bookDao;
    }
    
    public void save() {
        System.out.println("book service save ...");
        bookDao.save();
    }
    
    //DisposableBean接口的方法,对应destroy-method
    public void destroy() throws Exception {
        System.out.println("service destroy");
    }
    
    //InitializingBean接口的方法,对应init-method
    public void afterPropertiesSet() throws Exception {
        System.out.println("service init");
    }
}
```

### 4.3 Bean销毁时机

**销毁触发条件**: 容器关闭前触发bean的销毁

**关闭容器方式**:

1. **手工关闭容器**
   ```java
   ConfigurableApplicationContext接口close()操作
   ```

2. **注册关闭钩子**
   ```java
   //在虚拟机退出前先关闭容器再退出虚拟机
   ctx.registerShutdownHook();
   ```

**示例**:

```java
public class AppForLifeCycle {
    public static void main(String[] args) {
        ClassPathXmlApplicationContext ctx = new ClassPathXmlApplicationContext("applicationContext.xml");
        
        BookDao bookDao = (BookDao) ctx.getBean("bookDao");
        bookDao.save();
        
        //注册关闭钩子函数,在虚拟机退出之前回调此函数,关闭容器
        ctx.registerShutdownHook();
        
        //或者手动关闭
        //ctx.close();
    }
}
```

**注意**: 多例(prototype)的Bean不会执行destroy-method,因为Spring容器不管理多例Bean的销毁。

## 总结

本节详细讲解了依赖注入的各种方式和高级特性:

**核心要点**:
1. setter注入和构造器注入各有优劣,根据实际情况选择
2. 自动装配可以简化配置,但要注意类型唯一性和命名规范
3. 集合注入支持array、list、set、map、properties五种类型
4. Bean生命周期可以通过init-method和destroy-method控制

**最佳实践**:
- 强制依赖用构造器注入,可选依赖用setter注入
- 推荐使用byType自动装配,确保类型唯一
- 集合注入时注意泛型类型的匹配
- 单例Bean才需要配置destroy-method
