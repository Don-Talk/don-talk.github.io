## 一、RBAC概述

### 1.1 什么是RBAC

**RBAC (Role-Based Access Control)**: 基于角色的访问控制,是一种广泛使用的权限管理模型。

**核心思想**: 
- 用户不直接拥有权限
- 用户通过角色间接获得权限
- 权限分配给角色,角色分配给用户

### 1.2 RBAC的优势

| 特性 | 说明 |
|------|------|
| **简化授权** | 只需管理角色权限,无需为每个用户单独配置 |
| **易于维护** | 用户离职只需删除角色关联,无需逐个撤销权限 |
| **职责清晰** | 角色对应岗位职责,便于理解和管理 |
| **灵活扩展** | 新增用户只需分配角色即可 |
| **符合审计** | 权限变更可追溯,满足合规要求 |

### 1.3 RBAC模型组件

#### 基本要素

1. **用户(User)**: 系统的使用者
2. **角色(Role)**: 权限的集合,如管理员、普通用户
3. **权限(Permission)**: 对资源的操作,如user:add、user:delete
4. **资源(Resource)**: 被操作的对象,如用户、订单、文章

#### 关系

```
用户 --(多对多)--> 角色 --(多对多)--> 权限
```

## 五、Service层实现

### 5.1 自定义UserDetailsService

```java
package com.example.service;

import com.example.entity.SysPermission;
import com.example.entity.SysRole;
import com.example.entity.SysUser;
import com.example.mapper.RoleMapper;
import com.example.mapper.UserMapper;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.security.core.GrantedAuthority;
import org.springframework.security.core.authority.SimpleGrantedAuthority;
import org.springframework.security.core.userdetails.User;
import org.springframework.security.core.userdetails.UserDetails;
import org.springframework.security.core.userdetails.UserDetailsService;
import org.springframework.security.core.userdetails.UsernameNotFoundException;
import org.springframework.stereotype.Service;

import java.util.ArrayList;
import java.util.List;
import java.util.Set;
import java.util.stream.Collectors;

@Service
public class CustomUserDetailsService implements UserDetailsService {
    
    @Autowired
    private UserMapper userMapper;
    
    @Autowired
    private RoleMapper roleMapper;
    
    @Override
    public UserDetails loadUserByUsername(String username) 
            throws UsernameNotFoundException {
        
        // 1. 查询用户信息
        SysUser user = userMapper.selectByUsername(username);
        if (user == null) {
            throw new UsernameNotFoundException("用户不存在: " + username);
        }
        
        // 2. 检查用户状态
        if (user.getStatus() == 0) {
            throw new UsernameNotFoundException("用户已被禁用: " + username);
        }
        
        // 3. 查询用户角色
        List<SysRole> roles = userMapper.selectRolesByUserId(user.getId());
        
        // 4. 查询用户权限
        Set<String> permissions = roles.stream()
            .flatMap(role -> roleMapper.selectPermissionsByRoleId(role.getId()).stream())
            .map(SysPermission::getPermissionCode)
            .collect(Collectors.toSet());
        
        // 5. 构建权限列表
        List<GrantedAuthority> authorities = new ArrayList<>();
        
        // 添加角色(需要加ROLE_前缀)
        roles.forEach(role -> {
            authorities.add(new SimpleGrantedAuthority("ROLE_" + role.getRoleCode()));
        });
        
        // 添加权限
        permissions.forEach(permission -> {
            authorities.add(new SimpleGrantedAuthority(permission));
        });
        
        // 6. 返回UserDetails对象
        return User.builder()
            .username(user.getUsername())
            .password(user.getPassword())
            .authorities(authorities)
            .accountExpired(false)
            .accountLocked(false)
            .credentialsExpired(false)
            .disabled(false)
            .build();
    }
}
```

```java
            .and()
            .addFilterBefore(jwtAuthenticationFilter, 
                           UsernamePasswordAuthenticationFilter.class);
        
        return http.build();
    }
```

### 6.2 方法级别权限控制

```java
package com.example.controller;

import com.example.common.Result;
import com.example.entity.SysUser;
import com.example.service.UserService;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.security.access.prepost.PreAuthorize;
import org.springframework.web.bind.annotation.*;

import java.util.List;

@RestController
@RequestMapping("/api/users")
public class UserController {
    
    @Autowired
    private UserService userService;
    
    /**
     * 查询用户列表 - 需要user:query权限
     */
    @GetMapping
    @PreAuthorize("hasAuthority('user:query')")
    public Result<List<SysUser>> list() {
        return Result.success(userService.list());
    }
    
    /**
     * 根据ID查询用户 - 需要user:query权限
     */
    @GetMapping("/{id}")
    @PreAuthorize("hasAuthority('user:query')")
    public Result<SysUser> getById(@PathVariable Long id) {
        return Result.success(userService.getById(id));
    }
    
    /**
     * 新增用户 - 需要user:add权限
     */
    @PostMapping
    @PreAuthorize("hasAuthority('user:add')")
    public Result<Void> add(@RequestBody SysUser user) {
        userService.save(user);
        return Result.success();
    }
    
    /**
     * 修改用户 - 需要user:update权限
     */
    @PutMapping("/{id}")
    @PreAuthorize("hasAuthority('user:update')")
    public Result<Void> update(@PathVariable Long id, @RequestBody SysUser user) {
        user.setId(id);
        userService.updateById(user);
        return Result.success();
    }
    
    /**
     * 删除用户 - 需要user:delete权限
     */
    @DeleteMapping("/{id}")
    @PreAuthorize("hasAuthority('user:delete')")
    public Result<Void> delete(@PathVariable Long id) {
        userService.removeById(id);
        return Result.success();
    }
}
```

## 七、动态权限配置

### 7.1 基于数据库的动态权限

**需求**: 权限规则存储在数据库中,可以动态调整,无需重启应用

**实现思路**:
1. 将所有URL和所需权限存储在数据库
2. 自定义FilterInvocationSecurityMetadataSource加载权限配置
3. 每次请求时动态匹配URL和权限

### 7.2 自定义安全元数据源

```java
package com.example.config;

import com.example.entity.SysPermission;
import com.example.mapper.PermissionMapper;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.security.access.ConfigAttribute;
import org.springframework.security.access.SecurityConfig;
import org.springframework.security.web.FilterInvocation;
import org.springframework.security.web.access.intercept.FilterInvocationSecurityMetadataSource;
import org.springframework.stereotype.Component;

import javax.annotation.PostConstruct;
import java.util.Collection;
import java.util.HashMap;
import java.util.List;
import java.util.Map;

@Component
public class CustomSecurityMetadataSource implements FilterInvocationSecurityMetadataSource {
    
    @Autowired
    private PermissionMapper permissionMapper;
    
    /**
     * URL与权限的映射关系
     * key: URL pattern
     * value: 需要的权限列表
     */
    private Map<String, Collection<ConfigAttribute>> urlPermissionMap = new HashMap<>();
    
    @PostConstruct
    public void init() {
        // 从数据库加载权限配置
        loadUrlPermissions();
    }
    
    /**
     * 加载URL权限配置
     */
    private void loadUrlPermissions() {
        List<SysPermission> permissions = permissionMapper.selectAll();
        
        for (SysPermission permission : permissions) {
            String url = permission.getPath();
            String code = permission.getPermissionCode();
            
            urlPermissionMap.put(url, 
                SecurityConfig.createList(code)
            );
        }
    }
    
    @Override
    public Collection<ConfigAttribute> getAttributes(Object object) {
        FilterInvocation fi = (FilterInvocation) object;
        String requestUrl = fi.getRequestUrl();
        String method = fi.getRequest().getMethod();
        
        // 精确匹配
        if (urlPermissionMap.containsKey(requestUrl)) {
            return urlPermissionMap.get(requestUrl);
        }
        
        // Ant风格匹配(简化实现,实际应该使用AntPathMatcher)
        for (Map.Entry<String, Collection<ConfigAttribute>> entry : urlPermissionMap.entrySet()) {
            if (match(entry.getKey(), requestUrl)) {
                return entry.getValue();
            }
        }
        
        return null;
    }
    
    /**
     * 简单的URL匹配
     */
    private boolean match(String pattern, String url) {
        // 实际应该使用AntPathMatcher
        return pattern.equals(url);
    }
    
    @Override
    public Collection<ConfigAttribute> getAllConfigAttributes() {
        return null;
    }
    
    @Override
    public boolean supports(Class<?> clazz) {
        return FilterInvocation.class.isAssignableFrom(clazz);
    }
}
```

## 总结

本节详细介绍了RBAC权限模型的完整实现:

**核心要点**:
1. RBAC通过角色间接分配权限,简化管理
2. 标准的五张表结构:用户、角色、权限、用户角色、角色权限
3. UserDetailsService从数据库加载用户和权限信息
4. 使用@PreAuthorize实现方法级别的细粒度权限控制
5. 可以实现基于数据库的动态权限配置

**最佳实践**:
- 权限编码采用"资源:操作"格式,如user:add
- 角色编码使用大写,如ADMIN、USER
- 频繁查询的权限数据应该加入缓存(Redis)
- 定期审计权限分配,避免权限过度集中
- 敏感操作需要二次验证
