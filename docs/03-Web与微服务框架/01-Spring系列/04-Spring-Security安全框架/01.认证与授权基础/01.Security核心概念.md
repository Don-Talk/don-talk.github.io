## 一、Spring Security概述

### 1.1 什么是Spring Security

Spring Security是一个功能强大且高度可定制的身份验证和访问控制框架,是保护基于Spring的企业应用程序的事实标准。

**核心功能**:
- **认证(Authentication)**: 验证用户身份(你是谁)
- **授权(Authorization)**: 控制用户能做什么(你能干什么)
- **攻击防护**: 防止会话固定、CSRF、XSS等常见攻击

### 1.2 为什么需要Spring Security

#### 传统安全方案的痛点

1. **重复代码多**: 每个项目都要写登录、权限判断逻辑
2. **安全性差**: 容易出现安全漏洞
3. **维护成本高**: 安全策略变更需要修改大量代码
4. **功能不完善**: 缺少会话管理、Remember-Me等高级功能

#### Spring Security的优势

1. **开箱即用**: 默认提供完整的安全解决方案
2. **高度可定制**: 可以灵活配置认证和授权策略
3. **生态完善**: 与Spring家族无缝集成
4. **企业级**: 经过大量生产环境验证

### 1.3 Spring Security核心概念

#### 认证相关概念

| 概念 | 说明 |
|------|------|
| Authentication | 认证对象,包含用户信息和认证状态 |
| AuthenticationManager | 认证管理器,处理认证请求 |
| AuthenticationProvider | 认证提供者,执行具体的认证逻辑 |
| UserDetailsService | 用户详情服务,加载用户数据 |
| UserDetails | 用户详情接口,封装用户信息 |

#### 授权相关概念

| 概念 | 说明 |
|------|------|
| AccessDecisionManager | 访问决策管理器,决定用户是否有权限 |
| SecurityMetadataSource | 安全元数据源,提供资源的权限要求 |
| Role/Voter | 角色投票器,参与权限决策 |

## 三、基于内存的认证

### 3.1 配置类基础结构

创建Security配置类,继承`WebSecurityConfigurerAdapter`:

```java
package com.example.config;

import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;
import org.springframework.security.config.annotation.authentication.builders.AuthenticationManagerBuilder;
import org.springframework.security.config.annotation.web.builders.HttpSecurity;
import org.springframework.security.config.annotation.web.configuration.EnableWebSecurity;
import org.springframework.security.config.annotation.web.configuration.WebSecurityConfigurerAdapter;
import org.springframework.security.crypto.password.PasswordEncoder;

@Configuration
@EnableWebSecurity
public class SecurityConfig extends WebSecurityConfigurerAdapter {
    
    /**
     * 配置密码编码器
     */
    @Bean
    public PasswordEncoder passwordEncoder() {
        // BCrypt强哈希加密
        return new BCryptPasswordEncoder();
    }
    
    /**
     * 配置认证信息
     */
    @Override
    protected void configure(AuthenticationManagerBuilder auth) throws Exception {
        // 在此配置用户信息
    }
    
    /**
     * 配置授权规则
     */
    @Override
    protected void configure(HttpSecurity http) throws Exception {
        // 在此配置访问规则
    }
}
```

### 3.2 配置内存用户

```java
@Override
protected void configure(AuthenticationManagerBuilder auth) throws Exception {
    auth.inMemoryAuthentication()
        .withUser("admin")
        .password(passwordEncoder().encode("123456"))
        .roles("ADMIN", "USER")
        .and()
        .withUser("user")
        .password(passwordEncoder().encode("123456"))
        .roles("USER");
}
```

**说明**:
- `withUser()`: 设置用户名
- `password()`: 设置加密后的密码
- `roles()`: 设置用户角色
- `and()`: 链式调用,继续配置下一个用户

### 3.3 配置访问规则

```java
@Override
protected void configure(HttpSecurity http) throws Exception {
    http
        // 授权规则
        .authorizeRequests()
            // /admin路径需要ADMIN角色
            .antMatchers("/admin").hasRole("ADMIN")
            // /user路径需要USER或ADMIN角色
            .antMatchers("/user").hasAnyRole("USER", "ADMIN")
            // 其他路径允许匿名访问
            .anyRequest().permitAll()
        .and()
        
        // 表单登录配置
        .formLogin()
            // 自定义登录页面
            .loginPage("/login.html")
            // 登录处理URL
            .loginProcessingUrl("/login")
            // 登录成功跳转页面
            .defaultSuccessUrl("/hello")
            // 登录失败跳转页面
            .failureUrl("/login.html?error")
            // 允许所有用户访问登录相关页面
            .permitAll()
        .and()
        
        // 登出配置
        .logout()
            // 登出URL
            .logoutUrl("/logout")
            // 登出成功跳转页面
            .logoutSuccessUrl("/login.html")
            // 清除会话
            .invalidateHttpSession(true)
            // 允许所有用户访问登出URL
            .permitAll()
        .and()
        
        // 关闭CSRF(开发环境方便测试)
        .csrf().disable();
}
```

## 四、UserDetailsService详解

### 4.1 什么是UserDetailsService

`UserDetailsService`是Spring Security的核心接口,用于加载用户特定数据。

**核心方法**:

```java
public interface UserDetailsService {
    /**
     * 根据用户名加载用户信息
     * @param username 用户名
     * @return 用户详情对象
     * @throws UsernameNotFoundException 用户不存在时抛出
     */
    UserDetails loadUserByUsername(String username) 
        throws UsernameNotFoundException;
}
```

**工作流程**:

```
用户登录 → AuthenticationManager → UserDetailsService → 返回UserDetails → 验证密码
```

### 4.2 自定义UserDetailsService

#### 第一步: 实现UserDetailsService接口

```java
package com.example.service;

import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.security.core.authority.AuthorityUtils;
import org.springframework.security.core.userdetails.User;
import org.springframework.security.core.userdetails.UserDetails;
import org.springframework.security.core.userdetails.UserDetailsService;
import org.springframework.security.core.userdetails.UsernameNotFoundException;
import org.springframework.stereotype.Service;

@Service
public class CustomUserDetailsService implements UserDetailsService {
    
    @Autowired
    private PasswordEncoder passwordEncoder;
    
    @Override
    public UserDetails loadUserByUsername(String username) 
            throws UsernameNotFoundException {
        
        // 模拟从数据库查询用户
        // 实际项目中应该查询数据库
        if ("admin".equals(username)) {
            return User.builder()
                .username("admin")
                .password(passwordEncoder.encode("123456"))
                .authorities(AuthorityUtils.createAuthorityList(
                    "ROLE_ADMIN", "ROLE_USER"
                ))
                .build();
        } else if ("user".equals(username)) {
            return User.builder()
                .username("user")
                .password(passwordEncoder.encode("123456"))
                .authorities(AuthorityUtils.createAuthorityList(
                    "ROLE_USER"
                ))
                .build();
        } else {
            throw new UsernameNotFoundException("用户不存在: " + username);
        }
    }
}
```

#### 第二步: 在配置类中使用

```java
@Autowired
private CustomUserDetailsService userDetailsService;

@Override
protected void configure(AuthenticationManagerBuilder auth) throws Exception {
    auth.userDetailsService(userDetailsService);
}
```

### 4.3 UserDetails接口

`UserDetails`是Spring Security中表示用户信息的接口:

```java
public interface UserDetails extends Serializable {
    // 获取用户权限列表
    Collection<? extends GrantedAuthority> getAuthorities();
    
    // 获取密码
    String getPassword();
    
    // 获取用户名
    String getUsername();
    
    // 账户是否未过期
    boolean isAccountNonExpired();
    
    // 账户是否未锁定
    boolean isAccountNonLocked();
    
    // 凭证是否未过期
    boolean isCredentialsNonExpired();
    
    // 账户是否启用
    boolean isEnabled();
}
```

**使用Spring提供的默认实现**:

```java
// 方式一: User.builder()
UserDetails user = User.builder()
    .username("admin")
    .password("$2a$10$...")  // BCrypt加密后的密码
    .roles("ADMIN", "USER")
    .accountExpired(false)
    .accountLocked(false)
    .credentialsExpired(false)
    .disabled(false)
    .build();

// 方式二: new User()
UserDetails user = new User(
    "admin",
    "$2a$10$...",
    AuthorityUtils.createAuthorityList("ROLE_ADMIN", "ROLE_USER")
);
```

## 五、角色与权限控制

### 5.1 角色(Role) vs 权限(Permission)

**区别**:
- **角色**: 用户的身份标识,如ADMIN、USER
- **权限**: 具体的操作权限,如user:add、user:delete

**关系**: 一个角色可以包含多个权限

### 5.2 基于角色的访问控制

#### 配置类中配置

```java
@Override
protected void configure(HttpSecurity http) throws Exception {
    http.authorizeRequests()
        // 需要ADMIN角色
        .antMatchers("/admin/**").hasRole("ADMIN")
        // 需要USER或ADMIN角色
        .antMatchers("/user/**").hasAnyRole("USER", "ADMIN")
        // 允许所有用户访问
        .antMatchers("/public/**").permitAll()
        // 需要认证才能访问
        .anyRequest().authenticated();
}
```

#### 方法级别注解

```java
@RestController
@RequestMapping("/admin")
public class AdminController {
    
    @GetMapping("/users")
    @PreAuthorize("hasRole('ADMIN')")
    public List<User> getUsers() {
        // 只有ADMIN角色可以访问
        return userService.list();
    }
    
    @PostMapping("/users")
    @PreAuthorize("hasRole('ADMIN')")
    public void addUser(@RequestBody User user) {
        userService.save(user);
    }
}
```

**启用方法级别安全**:

```java
@Configuration
@EnableWebSecurity
@EnableGlobalMethodSecurity(prePostEnabled = true)  // 启用@PreAuthorize
public class SecurityConfig extends WebSecurityConfigurerAdapter {
    // ...
}
```

### 5.3 基于权限的访问控制

#### 配置类中配置

```java
@Override
protected void configure(HttpSecurity http) throws Exception {
    http.authorizeRequests()
        // 需要user:add权限
        .antMatchers("/users").hasAuthority("user:add")
        // 需要user:delete或user:update权限
        .antMatchers("/users/*").hasAnyAuthority("user:delete", "user:update")
        .anyRequest().authenticated();
}
```

#### 方法级别注解

```java
@RestController
@RequestMapping("/users")
public class UserController {
    
    @PostMapping
    @PreAuthorize("hasAuthority('user:add')")
    public void add(@RequestBody User user) {
        userService.save(user);
    }
    
    @DeleteMapping("/{id}")
    @PreAuthorize("hasAuthority('user:delete')")
    public void delete(@PathVariable Long id) {
        userService.removeById(id);
    }
    
    @PutMapping("/{id}")
    @PreAuthorize("hasAuthority('user:update')")
    public void update(@PathVariable Long id, @RequestBody User user) {
        user.setId(id);
        userService.updateById(user);
    }
}
```

### 5.4 常用表达式

| 表达式 | 说明 | 示例 |
|--------|------|------|
| hasRole | 检查是否有指定角色 | `hasRole('ADMIN')` |
| hasAnyRole | 检查是否有任一角色 | `hasAnyRole('ADMIN', 'USER')` |
| hasAuthority | 检查是否有指定权限 | `hasAuthority('user:add')` |
| hasAnyAuthority | 检查是否有任一权限 | `hasAnyAuthority('user:add', 'user:delete')` |
| permitAll | 允许所有 | `permitAll()` |
| denyAll | 拒绝所有 | `denyAll()` |
| authenticated | 需要认证 | `authenticated()` |
| anonymous | 匿名用户 | `anonymous()` |
| hasIpAddress | IP地址匹配 | `hasIpAddress('192.168.1.0/24')` |

## 六、密码加密

### 6.1 为什么需要密码加密

**安全问题**:
- 明文存储密码极易泄露
- 数据库被攻破后所有密码暴露
- 用户可能在多个网站使用相同密码

**解决方案**: 使用单向哈希算法加密存储

### 6.2 PasswordEncoder接口

Spring Security提供了`PasswordEncoder`接口:

```java
public interface PasswordEncoder {
    // 加密密码
    String encode(CharSequence rawPassword);
    
    // 验证密码
    boolean matches(CharSequence rawPassword, String encodedPassword);
}
```

### 6.3 常用实现类

#### BCryptPasswordEncoder (推荐)

```java
@Bean
public PasswordEncoder passwordEncoder() {
    // strength参数: 4-31,值越大越安全但性能越低,默认10
    return new BCryptPasswordEncoder(10);
}
```

**特点**:
- 自动加盐(Salt)
- 每次加密结果不同
- 强度高,推荐使用

**使用示例**:

```java
@Autowired
private PasswordEncoder passwordEncoder;

// 加密
String rawPassword = "123456";
String encodedPassword = passwordEncoder.encode(rawPassword);
// 输出: $2a$10$N9qo8uLOickgx2ZMRZoMyeIjZAgcfl7p92ldGxad68LJZdL17lhWy

// 验证
boolean matches = passwordEncoder.matches("123456", encodedPassword);
// 输出: true
```

#### 其他实现

```java
// SHA-256加密(不推荐,强度较低)
new StandardPasswordEncoder();

// 无加密(仅用于测试,生产环境禁止使用)
NoOpPasswordEncoder.getInstance();
```

### 6.4 历史密码升级

如果数据库中有旧格式的密码,可以使用`DelegatingPasswordEncoder`:

```java
@Bean
public PasswordEncoder passwordEncoder() {
    // 默认就是DelegatingPasswordEncoder
    // 支持多种加密格式,自动识别前缀
    // {bcrypt}$2a$10$...  -> BCrypt
    // {noop}123456         -> 明文
    // {sha256}...          -> SHA-256
    return PasswordEncoderFactories.createDelegatingPasswordEncoder();
}
```

**核心要点**:

- 遵循最小权限原则,只授予必要的权限
- 启用CSRF保护,防止跨站请求伪造
