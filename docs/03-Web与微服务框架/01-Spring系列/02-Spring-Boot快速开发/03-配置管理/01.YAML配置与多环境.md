## 一、配置文件概述

### 1.1 Spring Boot支持的配置文件

Spring Boot支持多种格式的配置文件:

| 文件名 | 格式 | 说明 |
|--------|------|------|
| application.properties | Properties | 传统键值对格式 |
| application.yml | YAML | 层次化结构,更直观 |
| application.yaml | YAML | 与.yml相同 |

**推荐使用YAML**,原因:
- 层次清晰,易于阅读
- 支持列表和复杂数据结构
- 减少重复配置

### 1.2 配置文件位置

Spring Boot按以下顺序搜索配置文件(优先级从高到低):

```
1. file:./config/                    # 当前目录的config子目录
2. file:./                           # 当前目录
3. classpath:/config/                # classpath的config目录
4. classpath:/                       # classpath根目录
```

**打包后的应用**:
```
java -jar app.jar                    # 使用jar包内的配置
java -jar app.jar --spring.config.location=file:./config/  # 使用外部配置
```

## 二、YAML配置语法

### 2.1 基本语法

#### 简单键值对

```yaml
server:
  port: 8080
  servlet:
    context-path: /api
```

**等价于Properties**:
```properties
server.port=8080
server.servlet.context-path=/api
```

**注意事项**:
- 冒号后面必须有空格: `key: value` ✅, `key:value` ❌
- 大小写敏感
- 使用缩进表示层级关系(通常2个空格)
- 不能使用Tab键缩进

### 2.2 数据类型

#### 字符串

```yaml
# 普通字符串
name: Hello World

# 包含特殊字符需要引号
message: "Hello: World"
path: 'C:\Users\test'

# 多行字符串
description: |
  这是第一行
  这是第二行
  这是第三行

# 折叠换行(替换为空格)
summary: >
  这是第一行
  这是第二行
```

#### 数字

```yaml
# 整数
port: 8080
timeout: 30

# 浮点数
rate: 3.14
price: 99.9

# 科学计数法
large: 1.23e10
```

#### 布尔值

```yaml
enabled: true
debug: false
```

#### 日期时间

```yaml
# ISO-8601格式
create-time: 2024-01-15T10:30:00

# 仅日期
birth-date: 1990-01-01
```

### 2.3 列表(List)

```yaml
# 方式一: 短横线
servers:
  - host: server1.example.com
    port: 8080
  - host: server2.example.com
    port: 8081

# 方式二: 方括号(内联)
ports: [8080, 8081, 8082]

# 简单列表
tags:
  - java
  - spring
  - boot
```

**对应的Java类**:

```java
@Data
@ConfigurationProperties(prefix = "app")
public class AppProperties {
    
    private List<Server> servers;
    private List<Integer> ports;
    private List<String> tags;
    
    @Data
    public static class Server {
        private String host;
        private Integer port;
    }
}
```

### 2.4 Map映射

```yaml
# 方式一: 标准写法
metadata:
  author: DonTalk
  version: 1.0.0
  description: Spring Boot教程

# 方式二: 内联写法
metadata2: {author: DonTalk, version: 1.0.0}
```

**对应的Java类**:

```java
@Data
@ConfigurationProperties(prefix = "app")
public class AppProperties {
    
    private Map<String, String> metadata;
}
```

### 2.5 null值

```yaml
# 显式null
name: null

# 或者不赋值
name:
```

### 2.6 锚点和引用

**避免重复配置**:

```yaml
# 定义锚点
default-server: &default
  host: localhost
  port: 8080
  timeout: 3000

# 引用锚点
server1:
  <<: *default
  host: server1.example.com

server2:
  <<: *default
  host: server2.example.com
```

## 三、配置文件加载顺序

### 3.1 多配置文件激活

Spring Boot支持通过Profile激活不同的配置:

```
application.yml              # 主配置文件(始终加载)
application-dev.yml          # 开发环境
application-test.yml         # 测试环境
application-prod.yml         # 生产环境
```

**激活方式**:

```yaml
# application.yml
spring:
  profiles:
    active: dev  # 激活dev配置
```

**命令行激活**:

```bash
java -jar app.jar --spring.profiles.active=prod
```

**环境变量激活**:

```bash
export SPRING_PROFILES_ACTIVE=prod
java -jar app.jar
```

### 3.2 配置加载优先级

Spring Boot的配置来源非常多,优先级从高到低:

```
1. 命令行参数
   java -jar app.jar --server.port=9090

2. JNDI属性

3. Java系统属性(System.getProperties())

4. 操作系统环境变量

5. RandomValuePropertySource (random.*)

6. jar包外部的application-{profile}.yml

7. jar包内部的application-{profile}.yml

8. jar包外部的application.yml

9. jar包内部的application.yml

10. @PropertySource注解

11. SpringApplication.setDefaultProperties()
```

**重要原则**:
- **高优先级配置覆盖低优先级配置**
- **后加载的配置合并先加载的配置**(不是完全替换)

### 3.3 配置合并规则

**示例**:

`application.yml`:
```yaml
server:
  port: 8080
  
app:
  name: MyApp
  version: 1.0.0
```

`application-prod.yml`:
```yaml
server:
  port: 80
  
app:
  version: 2.0.0
```

**最终结果** (激活prod时):
```yaml
server:
  port: 80            # prod覆盖
  
app:
  name: MyApp         # 保留主配置
  version: 2.0.0      # prod覆盖
```

## 四、读取配置值

### 4.1 @Value注解

#### 基本用法

```java
@Component
public class MyComponent {
    
    @Value("${server.port}")
    private Integer port;
    
    @Value("${app.name:DefaultApp}")  // 默认值
    private String appName;
    
    @Value("${app.description:}")     // 空字符串默认值
    private String description;
}
```

#### 读取列表

```java
@Component
public class MyComponent {
    
    @Value("${app.tags:java,spring,boot}")
    private List<String> tags;
}
```

#### 读取Map

```java
@Component
public class MyComponent {
    
    @Value("#{${app.metadata}}")
    private Map<String, String> metadata;
}
```

**配置**:
```yaml
app:
  metadata: "{author: 'DonTalk', version: '1.0.0'}"
```

### 4.2 @ConfigurationProperties

#### 基本用法

```java
@Data
@Component
@ConfigurationProperties(prefix = "app")
public class AppProperties {
    
    private String name;
    private String version;
    private Integer timeout;
    private Boolean enabled;
}
```

**配置**:
```yaml
app:
  name: MyApp
  version: 1.0.0
  timeout: 3000
  enabled: true
```

#### 嵌套对象

```java
@Data
@Component
@ConfigurationProperties(prefix = "app")
public class AppProperties {
    
    private String name;
    private Server server;
    private List<String> tags;
    
    @Data
    public static class Server {
        private String host;
        private Integer port;
    }
}
```

**配置**:
```yaml
app:
  name: MyApp
  server:
    host: localhost
    port: 8080
  tags:
    - java
    - spring
```

#### 验证配置

```java
@Data
@Component
@ConfigurationProperties(prefix = "app")
@Validated
public class AppProperties {
    
    @NotBlank(message = "应用名称不能为空")
    private String name;
    
    @Min(value = 1, message = "端口号必须大于0")
    @Max(value = 65535, message = "端口号必须小于65535")
    private Integer port;
    
    @Email(message = "邮箱格式不正确")
    private String adminEmail;
}
```

**启用验证**:

```java
@SpringBootApplication
@EnableConfigurationProperties
public class Application {
    public static void main(String[] args) {
        SpringApplication.run(Application.class, args);
    }
}
```

### 4.3 @Value vs @ConfigurationProperties

| 特性 | @Value | @ConfigurationProperties |
|------|--------|-------------------------|
| 绑定方式 | 单个属性 | 批量绑定 |
| 松散绑定 | ❌ 不支持 | ✅ 支持(appName = app-name) |
| JSR-303验证 | ❌ 不支持 | ✅ 支持 |
| 复杂类型 | ❌ 困难 | ✅ 轻松处理 |
| SpEL表达式 | ✅ 支持 | ❌ 不支持 |
| 元数据生成 | ❌ 无提示 | ✅ IDE自动提示 |

**推荐**: 
- 少量配置用`@Value`
- 大量相关配置用`@ConfigurationProperties`

## 五、多环境配置实战

### 5.1 环境划分

典型的环境划分:

- **dev**: 开发环境
- **test**: 测试环境
- **staging**: 预发布环境
- **prod**: 生产环境

### 5.2 配置文件组织

**项目结构**:

```
src/main/resources/
├── application.yml              # 公共配置
├── application-dev.yml          # 开发环境
├── application-test.yml         # 测试环境
└── application-prod.yml         # 生产环境
```

**application.yml** (公共配置):

```yaml
spring:
  application:
    name: my-app
    
# 默认激活dev环境
spring:
  profiles:
    active: ${SPRING_PROFILES_ACTIVE:dev}

# 公共配置
logging:
  level:
    root: INFO
```

**application-dev.yml**:

```yaml
server:
  port: 8080

spring:
  datasource:
    url: jdbc:mysql://localhost:3306/dev_db
    username: root
    password: root
    
logging:
  level:
    com.example: DEBUG
```

**application-prod.yml**:

```yaml
server:
  port: 80

spring:
  datasource:
    url: jdbc:mysql://prod-server:3306/prod_db
    username: ${DB_USERNAME}
    password: ${DB_PASSWORD}
    
logging:
  level:
    com.example: WARN
  file:
    name: /var/log/my-app/app.log
```

### 5.3 启动时指定环境

**开发环境**:
```bash
java -jar app.jar
# 或使用
java -jar app.jar --spring.profiles.active=dev
```

**生产环境**:
```bash
java -jar app.jar --spring.profiles.active=prod
```

**Docker环境**:
```dockerfile
ENV SPRING_PROFILES_ACTIVE=prod
```

## 六、外部化配置最佳实践

### 6.1 敏感信息处理

**❌ 错误做法**: 硬编码密码

```yaml
spring:
  datasource:
    password: mySecretPassword123
```

**✅ 正确做法**: 使用环境变量

```yaml
spring:
  datasource:
    password: ${DB_PASSWORD}
```

**启动时传入**:
```bash
export DB_PASSWORD=mySecretPassword123
java -jar app.jar
```

### 6.2 配置中心集成

对于微服务架构,建议使用配置中心:

- **Spring Cloud Config**: Spring官方方案
- **Nacos**: 阿里巴巴开源,功能强大
- **Apollo**: 携程开源,适合大型项目
- **Consul**: HashiCorp出品,生态完善

### 6.3 配置文件版本控制

**.gitignore**:

```gitignore
# 忽略包含敏感信息的配置文件
application-local.yml
application-prod.yml

# 保留模板文件
application-template.yml
```

**模板文件** `application-template.yml`:

```yaml
spring:
  datasource:
    url: jdbc:mysql://HOST:PORT/DATABASE
    username: YOUR_USERNAME
    password: YOUR_PASSWORD
```

### 6.4 配置热刷新

**需求**: 修改配置后无需重启应用

**方案**: 使用Spring Cloud Context

```xml
<dependency>
    <groupId>org.springframework.cloud</groupId>
    <artifactId>spring-cloud-starter-bootstrap</artifactId>
</dependency>
```

**刷新端点**:

```bash
# 刷新所有配置
POST http://localhost:8080/actuator/refresh

# 刷新指定配置
POST http://localhost:8080/actuator/refresh?keys=spring.datasource.url
```

## 七、高级配置技巧

### 7.1 随机值

```yaml
app:
  secret: ${random.value}        # UUID
  number: ${random.int}          # int随机数
  long-number: ${random.long}    # long随机数
  uuid: ${random.uuid}           # UUID
  port: ${random.int[1024,65535]} # 范围内的随机数
```

### 7.2 占位符引用

```yaml
app:
  name: MyApp
  description: ${app.name} is a great application
  full-name: ${app.name}-${spring.profiles.active}
```

### 7.3 条件配置

```yaml
# 仅在dev环境生效
debug: true

# 根据操作系统
os-name: ${os.name}

# 根据Java版本
java-version: ${java.version}
```

### 7.4 自定义配置文件

**非标准名称的配置文件**:

```java
@SpringBootApplication
@PropertySource("classpath:custom.yml")
public class Application {
    public static void main(String[] args) {
        SpringApplication.run(Application.class, args);
    }
}
```

**多个配置文件**:

```java
@PropertySources({
    @PropertySource("classpath:config1.yml"),
    @PropertySource("classpath:config2.yml")
})
```

## 总结

本节全面讲解了Spring Boot的配置管理:

**核心要点**:
1. YAML格式更直观,推荐使用
2. 配置文件按优先级加载,高优先级覆盖低优先级
3. 通过Profile实现多环境配置
4. @ConfigurationProperties比@Value更强大
5. 敏感信息应该外部化,不要硬编码

**最佳实践**:
- 使用YAML格式,层次清晰
- 合理划分环境(dev/test/prod)
- 敏感信息使用环境变量或配置中心
- 大量配置使用@ConfigurationProperties
- 添加JSR-303验证保证配置正确性
