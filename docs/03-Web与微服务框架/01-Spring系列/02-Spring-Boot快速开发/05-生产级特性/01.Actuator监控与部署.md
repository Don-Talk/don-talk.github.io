## 一、Actuator监控端点

### 1.1 什么是Actuator

**Actuator** 是Spring Boot提供的生产级监控和管理功能,通过HTTP端点或JMX暴露应用的健康状况、指标、日志等信息。

**核心价值**:
- **健康检查**: 实时监控应用状态
- **性能监控**: CPU、内存、线程等指标
- **环境信息**: 配置、Bean、映射等
- **运维友好**: 便于自动化运维

### 1.2 启用Actuator

#### 第一步: 导入依赖

```xml
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-actuator</artifactId>
</dependency>
```

#### 第二步: 配置端点

`application.yml`:

```yaml
management:
  endpoints:
    web:
      exposure:
        include: "*"  # 暴露所有端点(开发环境)
        # include: health,info,metrics  # 生产环境只暴露必要的
  endpoint:
    health:
      show-details: always  # 显示详细健康信息
```

**安全警告**: 
- ❌ 生产环境不要暴露所有端点
- ✅ 只暴露health、info、metrics等安全端点
- ✅ 敏感端点(如env、beans)需要认证

### 1.3 常用端点详解

#### 1. health - 健康检查

**访问**: `GET http://localhost:8080/actuator/health`

**响应**:

```json
{
  "status": "UP",
  "components": {
    "db": {
      "status": "UP",
      "details": {
        "database": "MySQL",
        "validationQuery": "isValid()"
      }
    },
    "diskSpace": {
      "status": "UP",
      "details": {
        "total": 500000000000,
        "free": 300000000000,
        "threshold": 10485760
      }
    },
    "redis": {
      "status": "UP",
      "details": {
        "version": "6.2.6"
      }
    }
  }
}
```

**状态说明**:
- `UP`: 正常
- `DOWN`: 异常
- `OUT_OF_SERVICE`: 服务暂停
- `UNKNOWN`: 未知状态

#### 2. info - 应用信息

**配置**:

```yaml
info:
  app:
    name: My Application
    version: 1.0.0
    description: Spring Boot应用
  author:
    name: DonTalk
    email: dontalk@example.com
```

**访问**: `GET http://localhost:8080/actuator/info`

**响应**:

```json
{
  "app": {
    "name": "My Application",
    "version": "1.0.0",
    "description": "Spring Boot应用"
  },
  "author": {
    "name": "DonTalk",
    "email": "dontalk@example.com"
  }
}
```

#### 3. metrics - 性能指标

**查看所有可用指标**:

`GET http://localhost:8080/actuator/metrics`

**查看具体指标**:

```bash
# JVM内存
GET /actuator/metrics/jvm.memory.used

# HTTP请求数
GET /actuator/metrics/http.server.requests

# 系统CPU
GET /actuator/metrics/system.cpu.usage
```

**响应示例**:

```json
{
  "name": "jvm.memory.used",
  "measurements": [
    {
      "statistic": "VALUE",
      "value": 157286400
    }
  ],
  "availableTags": [
    {
      "tag": "area",
      "values": ["heap", "nonheap"]
    },
    {
      "tag": "id",
      "values": ["G1 Survivor Space", "G1 Old Gen", "G1 Eden Space"]
    }
  ]
}
```

#### 4. env - 环境变量

**访问**: `GET http://localhost:8080/actuator/env`

**注意**: 包含敏感信息,生产环境必须保护!

#### 5. beans - Spring Bean列表

**访问**: `GET http://localhost:8080/actuator/beans`

显示所有Spring容器中的Bean。

#### 6. mappings - URL映射

**访问**: `GET http://localhost:8080/actuator/mappings`

显示所有Controller的URL映射关系。

#### 7. loggers - 日志级别

**查看日志级别**:

`GET http://localhost:8080/actuator/loggers`

**动态修改日志级别**:

```bash
POST http://localhost:8080/actuator/loggers/com.example
Content-Type: application/json

{
  "configuredLevel": "DEBUG"
}
```

无需重启应用,即时生效!

## 二、自定义Health Indicator

### 2.1 实现接口

```java
package com.example.health;

import org.springframework.boot.actuate.health.Health;
import org.springframework.boot.actuate.health.HealthIndicator;
import org.springframework.stereotype.Component;

@Component
public class CustomHealthIndicator implements HealthIndicator {
    
    @Override
    public Health health() {
        try {
            // 执行健康检查逻辑
            boolean success = checkDatabase();
            
            if (success) {
                return Health.up()
                    .withDetail("database", "connected")
                    .withDetail("message", "数据库连接正常")
                    .build();
            } else {
                return Health.down()
                    .withDetail("database", "disconnected")
                    .withDetail("message", "数据库连接失败")
                    .build();
            }
        } catch (Exception e) {
            return Health.down(e)
                .withDetail("message", "健康检查异常")
                .build();
        }
    }
    
    private boolean checkDatabase() {
        // 实际检查逻辑
        return true;
    }
}
```

## 三、日志配置最佳实践

### 3.1 Logback配置

Spring Boot默认使用Logback作为日志框架。

**基础配置** `application.yml`:

```yaml
logging:
  level:
    root: INFO
    com.example: DEBUG
  pattern:
    console: "%d{yyyy-MM-dd HH:mm:ss} [%thread] %-5level %logger{36} - %msg%n"
    file: "%d{yyyy-MM-dd HH:mm:ss} [%thread] %-5level %logger{36} - %msg%n"
  file:
    name: logs/app.log
```

### 3.2 高级配置 logback-spring.xml

在`src/main/resources/`下创建`logback-spring.xml`:

```xml
<?xml version="1.0" encoding="UTF-8"?>
<configuration>
    
    <!-- 引入Spring Boot默认配置 -->
    <include resource="org/springframework/boot/logging/logback/defaults.xml"/>
    
    <!-- 控制台输出 -->
    <appender name="CONSOLE" class="ch.qos.logback.core.ConsoleAppender">
        <encoder>
            <pattern>%d{yyyy-MM-dd HH:mm:ss.SSS} [%thread] %-5level %logger{50} - %msg%n</pattern>
            <charset>UTF-8</charset>
        </encoder>
    </appender>
    
    <!-- 文件输出 - 所有日志 -->
    <appender name="FILE" class="ch.qos.logback.core.rolling.RollingFileAppender">
        <file>logs/app.log</file>
        <rollingPolicy class="ch.qos.logback.core.rolling.SizeAndTimeBasedRollingPolicy">
            <fileNamePattern>logs/app.%d{yyyy-MM-dd}.%i.log</fileNamePattern>
            <maxFileSize>100MB</maxFileSize>
            <maxHistory>30</maxHistory>
            <totalSizeCap>10GB</totalSizeCap>
        </rollingPolicy>
        <encoder>
            <pattern>%d{yyyy-MM-dd HH:mm:ss.SSS} [%thread] %-5level %logger{50} - %msg%n</pattern>
            <charset>UTF-8</charset>
        </encoder>
    </appender>
    
    <!-- 错误日志单独输出 -->
    <appender name="ERROR_FILE" class="ch.qos.logback.core.rolling.RollingFileAppender">
        <file>logs/error.log</file>
        <filter class="ch.qos.logback.classic.filter.ThresholdFilter">
            <level>ERROR</level>
        </filter>
        <rollingPolicy class="ch.qos.logback.core.rolling.SizeAndTimeBasedRollingPolicy">
            <fileNamePattern>logs/error.%d{yyyy-MM-dd}.%i.log</fileNamePattern>
            <maxFileSize>100MB</maxFileSize>
            <maxHistory>30</maxHistory>
        </rollingPolicy>
        <encoder>
            <pattern>%d{yyyy-MM-dd HH:mm:ss.SSS} [%thread] %-5level %logger{50} - %msg%n</pattern>
            <charset>UTF-8</charset>
        </encoder>
    </appender>
    
    <!-- 根日志级别 -->
    <root level="INFO">
        <appender-ref ref="CONSOLE"/>
        <appender-ref ref="FILE"/>
        <appender-ref ref="ERROR_FILE"/>
    </root>
    
    <!-- 自定义包日志级别 -->
    <logger name="com.example" level="DEBUG"/>
    
</configuration>
```

## 四、打包与部署

### 4.1 Maven打包

**pom.xml配置**:

```xml
<build>
    <plugins>
        <plugin>
            <groupId>org.springframework.boot</groupId>
            <artifactId>spring-boot-maven-plugin</artifactId>
            <configuration>
                <excludes>
                    <exclude>
                        <groupId>org.projectlombok</groupId>
                        <artifactId>lombok</artifactId>
                    </exclude>
                </excludes>
            </configuration>
        </plugin>
    </plugins>
</build>
```

**打包命令**:

```bash
mvn clean package -DskipTests
```

**生成的jar包**: `target/myapp-1.0.0.jar`

### 4.2 运行jar包

```bash
# 基本运行
java -jar myapp-1.0.0.jar

# 指定端口
java -jar myapp-1.0.0.jar --server.port=9090

# 指定配置文件
java -jar myapp-1.0.0.jar --spring.profiles.active=prod

# 后台运行
nohup java -jar myapp-1.0.0.jar > /dev/null 2>&1 &

# 设置JVM参数
java -Xms512m -Xmx1024m -jar myapp-1.0.0.jar
```

### 4.3 Docker部署

**Dockerfile**:

```dockerfile
FROM openjdk:11-jre-slim

WORKDIR /app

COPY target/myapp-1.0.0.jar app.jar

EXPOSE 8080

ENTRYPOINT ["java", "-jar", "app.jar"]
```

**构建镜像**:

```bash
docker build -t myapp:1.0.0 .
```

**运行容器**:

```bash
docker run -d \
  --name myapp \
  -p 8080:8080 \
  -e SPRING_PROFILES_ACTIVE=prod \
  myapp:1.0.0
```

### 4.4 docker-compose部署

**docker-compose.yml**:

```yaml
version: '3'

services:
  app:
    image: myapp:1.0.0
    ports:
      - "8080:8080"
    environment:
      - SPRING_PROFILES_ACTIVE=prod
      - DB_HOST=mysql
      - DB_PASSWORD=root
    depends_on:
      - mysql
    restart: always
    
  mysql:
    image: mysql:8.0
    environment:
      MYSQL_ROOT_PASSWORD: root
      MYSQL_DATABASE: mydb
    volumes:
      - mysql_data:/var/lib/mysql
    restart: always

volumes:
  mysql_data:
```

**启动**:

```bash
docker-compose up -d
```
