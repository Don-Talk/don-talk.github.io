# 博客系统 - 文件上传模块开发(图片、附件)

## 一、存储方案设计

### 1.1 存储方式对比

| 方案 | 优点 | 缺点 | 适用场景 |
|------|------|------|----------|
| **本地存储** | 简单、快速 | 占用服务器空间、不支持CDN | 小型项目、测试环境 |
| **对象存储(OSS)** | 高可用、CDN加速、便宜 | 需要配置密钥 | 生产环境推荐 ⭐ |
| **分布式文件系统(FastDFS)** | 自建、可控 | 运维成本高 | 大型企业 |

**本教程采用**: 阿里云OSS (也可切换到腾讯云COS/七牛云)

## 二、阿里云OSS集成

### 2.1 添加依赖

```xml
<!-- pom.xml -->
<dependency>
    <groupId>com.aliyun.oss</groupId>
    <artifactId>aliyun-sdk-oss</artifactId>
    <version>3.15.1</version>
</dependency>
```

### 2.2 配置application.yml

```yaml
# OSS配置
oss:
  aliyun:
    endpoint: oss-cn-hangzhou.aliyuncs.com  # OSS Endpoint
    access-key-id: ${ALIYUN_ACCESS_KEY_ID}   # 从环境变量读取
    access-key-secret: ${ALIYUN_ACCESS_KEY_SECRET}
    bucket-name: dontalk-blog
    domain: https://blog.dontalk.com  # CDN域名(可选)
    base-path: blog/                  # 文件存储路径前缀
```

**安全提示**: 
- ❌ 不要将AccessKey硬编码在代码中
- ✅ 使用环境变量或配置中心管理敏感信息

package com.dontalk.blog.config;

import com.aliyun.oss.OSS;
import com.aliyun.oss.OSSClientBuilder;
import lombok.Data;
import org.springframework.boot.context.properties.ConfigurationProperties;
import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;

@Data
@Configuration
@ConfigurationProperties(prefix = "oss.aliyun")
public class OssConfig {
    
    /**
     * OSS Endpoint
     */
    private String endpoint;
    
    /**
     * Access Key ID
     */
    private String accessKeyId;
    
    /**
     * Access Key Secret
     */
    private String accessKeySecret;
    
    /**
     * Bucket名称
     */
    private String bucketName;
    
    /**
     * CDN域名
     */
    private String domain;
    
    /**
     * 基础路径
     */
    private String basePath;
    
    /**
     * 创建OSS客户端
     */
    @Bean
    public OSS ossClient() {
        return new OSSClientBuilder().build(endpoint, accessKeyId, accessKeySecret);
    }
}

### 4.2 FileUploadServiceImpl.java

```java
package com.dontalk.blog.module.file.service.impl;

import com.aliyun.oss.OSS;
import com.aliyun.oss.model.ObjectMetadata;
import com.dontalk.blog.common.exception.BusinessException;
import com.dontalk.blog.config.OssConfig;
import com.dontalk.blog.module.file.service.FileUploadService;
import lombok.RequiredArgsConstructor;
import lombok.extern.slf4j.Slf4j;
import org.springframework.stereotype.Service;
import org.springframework.util.StringUtils;
import org.springframework.web.multipart.MultipartFile;

import java.io.IOException;
import java.time.LocalDate;
import java.time.format.DateTimeFormatter;
import java.util.Arrays;
import java.util.List;
import java.util.UUID;

@Slf4j
@Service
@RequiredArgsConstructor
public class FileUploadServiceImpl implements FileUploadService {
    
    private final OSS ossClient;
    private final OssConfig ossConfig;
    
    // 允许的图片类型
    private static final List<String> IMAGE_TYPES = Arrays.asList(
            "image/jpeg", "image/jpg", "image/png", "image/gif", "image/webp"
    );
    
    // 最大文件大小: 5MB
    private static final long MAX_FILE_SIZE = 5 * 1024 * 1024;
    
    @Override
    public String uploadImage(MultipartFile file, String directory) {
        // 1. 验证文件类型
        String contentType = file.getContentType();
        if (!IMAGE_TYPES.contains(contentType)) {
            throw new BusinessException("只支持jpg、png、gif、webp格式的图片");
        }
        
        // 2. 验证文件大小
        if (file.getSize() > MAX_FILE_SIZE) {
            throw new BusinessException("图片大小不能超过5MB");
        }
        
        // 3. 上传到OSS
        return uploadToOss(file, directory);
    }
    
    @Override
    public String uploadFile(MultipartFile file, String directory) {
        // 1. 验证文件大小
        if (file.getSize() > MAX_FILE_SIZE) {
            throw new BusinessException("文件大小不能超过5MB");
        }
        
        // 2. 上传到OSS
        return uploadToOss(file, directory);
    }
    
    @Override
    public void deleteFile(String url) {
        try {
            // 1. 从URL中提取objectName
            String objectName = extractObjectName(url);
            
            // 2. 删除OSS文件
            ossClient.deleteObject(ossConfig.getBucketName(), objectName);
            
            log.info("删除文件成功: {}", url);
        } catch (Exception e) {
            log.error("删除文件失败: {}", url, e);
            throw new BusinessException("删除文件失败");
        }
    }
    
    // ==================== 私有方法 ====================
    
    /**
     * 上传文件到OSS
     */
    private String uploadToOss(MultipartFile file, String directory) {
        try {
            // 1. 生成文件名
            String originalFilename = file.getOriginalFilename();
            String extension = StringUtils.getFilenameExtension(originalFilename);
            String fileName = generateFileName(extension);
            
            // 2. 构建OSS对象名称(按日期分目录)
            String datePath = LocalDate.now().format(DateTimeFormatter.ofPattern("yyyy/MM/dd"));
            String objectName = ossConfig.getBasePath() + directory + "/" + datePath + "/" + fileName;
            
            // 3. 设置元数据
            ObjectMetadata metadata = new ObjectMetadata();
            metadata.setContentType(file.getContentType());
            metadata.setContentLength(file.getSize());
            
            // 4. 上传文件
            ossClient.putObject(ossConfig.getBucketName(), objectName, file.getInputStream(), metadata);
            
            // 5. 返回访问URL
            String url = buildFileUrl(objectName);
            
            log.info("上传文件成功: {}, size: {}", url, file.getSize());
            return url;
            
        } catch (IOException e) {
            log.error("上传文件失败", e);
            throw new BusinessException("上传文件失败: " + e.getMessage());
        }
    }
    
    /**
     * 生成唯一文件名
     */
    private String generateFileName(String extension) {
        return UUID.randomUUID().toString().replace("-", "") + "." + extension;
    }
    
    /**
     * 构建文件访问URL
     */
    private String buildFileUrl(String objectName) {
        if (StringUtils.hasText(ossConfig.getDomain())) {
            // 使用CDN域名
            return ossConfig.getDomain() + "/" + objectName;
        } else {
            // 使用OSS默认域名
            return "https://" + ossConfig.getBucketName() + "." + 
                   ossConfig.getEndpoint() + "/" + objectName;
        }
    }
    
    /**
     * 从URL中提取objectName
     */
    private String extractObjectName(String url) {
        // https://blog.dontalk.com/blog/articles/2024/01/01/abc123.jpg
        // -> blog/articles/2024/01/01/abc123.jpg
        
        String baseUrl = ossConfig.getDomain();
        if (!StringUtils.hasText(baseUrl)) {
            baseUrl = "https://" + ossConfig.getBucketName() + "." + ossConfig.getEndpoint();
        }
        
        return url.replace(baseUrl + "/", "");
    }
}
```

**关键逻辑说明**:
1. **文件验证**: 类型和大小校验，防止恶意上传
2. **文件名生成**: UUID确保唯一性，避免中文文件名问题
3. **目录组织**: 按日期分目录(`yyyy/MM/dd`)，便于管理
4. **CDN加速**: 支持配置CDN域名提升访问速度
5. **异常处理**: IO异常统一转换为BusinessException

## 五、Controller层

### 5.1 FileUploadController.java

```java
package com.dontalk.blog.module.file.controller;

import com.dontalk.blog.common.result.Result;
import com.dontalk.blog.module.file.service.FileUploadService;
import io.swagger.annotations.Api;
import io.swagger.annotations.ApiOperation;
import lombok.RequiredArgsConstructor;
import org.springframework.web.bind.annotation.*;
import org.springframework.web.multipart.MultipartFile;

@RestController
@RequestMapping("/api/files")
@RequiredArgsConstructor
@Api(tags = "文件上传")
public class FileUploadController {
    
    private final FileUploadService fileUploadService;
    
    @PostMapping("/image")
    @ApiOperation("上传图片")
    public Result<String> uploadImage(
            @RequestParam("file") MultipartFile file,
            @RequestParam(value = "directory", defaultValue = "articles") String directory) {
        String url = fileUploadService.uploadImage(file, directory);
        return Result.success(url);
    }
    
    @PostMapping("/avatar")
    @ApiOperation("上传头像")
    public Result<String> uploadAvatar(@RequestParam("file") MultipartFile file) {
        String url = fileUploadService.uploadImage(file, "avatars");
        return Result.success(url);
    }
    
    @PostMapping("/cover")
    @ApiOperation("上传封面图")
    public Result<String> uploadCover(@RequestParam("file") MultipartFile file) {
        String url = fileUploadService.uploadImage(file, "covers");
        return Result.success(url);
    }
    
    @DeleteMapping
    @ApiOperation("删除文件")
    public Result<Void> deleteFile(@RequestParam("url") String url) {
        fileUploadService.deleteFile(url);
        return Result.success();
    }
}
```

**API设计要点**:
- 不同场景使用不同接口(文章图片/头像/封面)
- `directory`参数控制存储目录
- 支持DELETE方法删除文件

## 六、前端集成示例

### 6.1 Vue.js图片上传组件

```vue
<template>
  <div class="image-upload">
    <el-upload
      action="/api/files/image"
      :headers="headers"
      :on-success="handleSuccess"
      :on-error="handleError"
      :before-upload="beforeUpload"
      list-type="picture-card"
      accept="image/*"
    >
      <i class="el-icon-plus"></i>
    </el-upload>
  </div>
</template>

<script>
export default {
  name: 'ImageUpload',
  data() {
    return {
      headers: {
        Authorization: `Bearer ${this.$store.state.user.token}`
      }
    }
  },
  methods: {
    beforeUpload(file) {
      // 验证文件类型
      const isImage = file.type.startsWith('image/')
      if (!isImage) {
        this.$message.error('只能上传图片文件!')
        return false
      }
      
      // 验证文件大小
      const isLt5M = file.size / 1024 / 1024 < 5
      if (!isLt5M) {
        this.$message.error('图片大小不能超过5MB!')
        return false
      }
      
      return true
    },
    handleSuccess(response) {
      if (response.code === 200) {
        this.$message.success('上传成功')
        this.$emit('success', response.data) // 返回图片URL
      } else {
        this.$message.error(response.message)
      }
    },
    handleError(error) {
      this.$message.error('上传失败: ' + error.message)
    }
  }
}
</script>
```

## 七、本地存储方案(备选)

如果不想使用OSS，可以使用本地存储：

### 7.1 LocalFileStorageService.java

```java
package com.dontalk.blog.module.file.service.impl;

import com.dontalk.blog.common.exception.BusinessException;
import com.dontalk.blog.module.file.service.FileUploadService;
import lombok.extern.slf4j.Slf4j;
import org.springframework.beans.factory.annotation.Value;
import org.springframework.stereotype.Service;
import org.springframework.util.StringUtils;
import org.springframework.web.multipart.MultipartFile;

import java.io.IOException;
import java.nio.file.Files;
import java.nio.file.Path;
import java.nio.file.Paths;
import java.time.LocalDate;
import java.time.format.DateTimeFormatter;
import java.util.UUID;

@Slf4j
@Service("localFileUploadService")
public class LocalFileStorageService implements FileUploadService {
    
    @Value("${file.upload.path:/uploads}")
    private String uploadPath;
    
    @Value("${file.access.url:http://localhost:8080/files}")
    private String accessUrl;
    
    @Override
    public String uploadImage(MultipartFile file, String directory) {
        return uploadLocal(file, directory);
    }
    
    @Override
    public String uploadFile(MultipartFile file, String directory) {
        return uploadLocal(file, directory);
    }
    
    @Override
    public void deleteFile(String url) {
        // 提取文件路径并删除
        String filePath = url.replace(accessUrl + "/", "");
        try {
            Files.deleteIfExists(Paths.get(uploadPath, filePath));
            log.info("删除文件成功: {}", url);
        } catch (IOException e) {
            log.error("删除文件失败: {}", url, e);
        }
    }
    
    private String uploadLocal(MultipartFile file, String directory) {
        try {
            // 1. 生成文件名
            String originalFilename = file.getOriginalFilename();
            String extension = StringUtils.getFilenameExtension(originalFilename);
            String fileName = UUID.randomUUID().toString().replace("-", "") + "." + extension;
            
            // 2. 构建存储路径(按日期分目录)
            String datePath = LocalDate.now().format(DateTimeFormatter.ofPattern("yyyy/MM/dd"));
            Path dirPath = Paths.get(uploadPath, directory, datePath);
            Files.createDirectories(dirPath);
            
            // 3. 保存文件
            Path filePath = dirPath.resolve(fileName);
            Files.write(filePath, file.getBytes());
            
            // 4. 返回访问URL
            String url = accessUrl + "/" + directory + "/" + datePath + "/" + fileName;
            
            log.info("上传文件成功: {}", url);
            return url;
            
        } catch (IOException e) {
            log.error("上传文件失败", e);
            throw new BusinessException("上传文件失败: " + e.getMessage());
        }
    }
}
```

### 7.2 静态资源映射配置

```java
package com.dontalk.blog.config;

import org.springframework.beans.factory.annotation.Value;
import org.springframework.context.annotation.Configuration;
import org.springframework.web.servlet.config.annotation.ResourceHandlerRegistry;
import org.springframework.web.servlet.config.annotation.WebMvcConfigurer;

@Configuration
public class WebConfig implements WebMvcConfigurer {
    
    @Value("${file.upload.path:/uploads}")
    private String uploadPath;
    
    @Override
    public void addResourceHandlers(ResourceHandlerRegistry registry) {
        // 映射 /files/** 到本地目录
        registry.addResourceHandler("/files/**")
                .addResourceLocations("file:" + uploadPath + "/");
    }
}
```

## 十、常见问题

### Q1: 如何限制用户上传文件大小?

**A**: 在application.yml中配置:
```yaml
spring:
  servlet:
    multipart:
      max-file-size: 10MB      # 单个文件最大
      max-request-size: 50MB   # 整个请求最大
```

### Q2: 如何实现图片压缩?

**A**: 使用Thumbnailator库:
```xml
<dependency>
    <groupId>net.coobird</groupId>
    <artifactId>thumbnailator</artifactId>
    <version>0.4.16</version>
</dependency>
```

```java
BufferedImage image = Thumbnails.of(file.getInputStream())
    .size(800, 600)  // 压缩到800x600
    .outputQuality(0.8)  // 质量80%
    .asBufferedImage();
```

### Q3: 如何防止盗链?

**A**: 
1. **OSS Referer白名单**: 只允许自己的域名访问
2. **签名URL**: 生成带过期时间的签名URL
3. **Nginx防盗链**: 
```nginx
location ~ .*\.(gif|jpg|jpeg|png)$ {
    valid_referers none blocked *.dontalk.com;
    if ($invalid_referer) {
        return 403;
    }
}
```
