## 📝 代码规范总结

### 1. 分层架构

```
Controller层 → Service层 → Mapper层 → Database
     ↓              ↓            ↓
   DTO/VO       Entity       Entity
```

**职责划分**:
- **Controller**: 接收请求、参数校验、返回响应
- **Service**: 业务逻辑、事务管理
- **Mapper**: 数据库操作(CRUD)
- **Entity**: 数据库表映射
- **DTO**: 数据传输对象(请求参数)
- **VO**: 视图对象(响应数据)

### 2. 命名规范

- **类名**: 大驼峰(UserService、ArticleController)
- **方法名**: 小驼峰(getUserById、createArticle)
- **常量**: 全大写+下划线(MAX_FILE_SIZE)
- **包名**: 全小写(com.dontalk.blog.module.user)
- **变量**: 小驼峰(userName、createTime)

### 3. 注释规范

```java
/**
 * 创建文章
 * 
 * @param request 创建文章请求
 * @param authorId 作者ID
 * @return 文章ID
 */
Long createArticle(ArticleCreateRequest request, Long authorId);
```

### 4. 异常处理

- 业务异常使用BusinessException
- Controller层不捕获异常,交给全局处理器
- 关键操作记录日志(log.info/log.error)

## 💡 常见问题FAQ

### Q1: 如何实现文章的定时发布?

**A**: 使用Spring的@Scheduled定时任务:
```java
@Scheduled(fixedRate = 60000) // 每分钟执行一次
public void publishScheduledArticles() {
    List<Article> articles = articleMapper.selectList(
        new LambdaQueryWrapper<Article>()
            .eq(Article::getStatus, 0)
            .le(Article::getPublishTime, LocalDateTime.now())
    );
    
    articles.forEach(article -> {
        article.setStatus(1);
        articleMapper.updateById(article);
    });
}
```

### Q2: 如何防止恶意刷评论?

**A**: 
1. **频率限制**: 同一用户每分钟最多发5条评论
2. **内容审核**: 敏感词过滤
3. **验证码**: 提交评论前需要滑块验证
4. **IP限流**: 同一IP每小时最多发50条评论

### Q3: 如何实现文章草稿自动保存?

**A**: 
- **前端**: 每隔30秒将草稿保存到localStorage
- **后端**: 提供自动保存接口,保存到数据库(draft表)
- **定时任务**: 每天凌晨清理7天前的草稿

### Q4: 如何优化大量文章的查询性能?

**A**: 
1. **索引优化**: 在category_id、status、create_time建立联合索引
2. **缓存**: Redis缓存热门文章(hot_articles)
3. **分页**: 避免一次性加载所有数据
4. **读写分离**: 主库写、从库读
5. **搜索引擎**: 引入Elasticsearch做全文检索

### Q5: 如何实现RBAC权限控制?

**A**: 
1. **角色表**: sys_role(id, role_name)
2. **权限表**: sys_permission(id, permission_name, url)
3. **用户角色关联表**: sys_user_role(user_id, role_id)
4. **角色权限关联表**: sys_role_permission(role_id, permission_id)
5. **Security配置**: 根据用户角色判断是否有权限访问
