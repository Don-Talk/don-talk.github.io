## 一、AOP核心概念回顾

### 1.1 关键术语

| 术语 | 说明 | 对应类 |
|------|------|--------|
| JoinPoint | 连接点,方法执行的某个点 | JoinPoint |
| Pointcut | 切入点,匹配连接点的表达式 | Pointcut |
| Advice | 通知,在连接点执行的动作 | Advice |
| Advisor | 通知器,Pointcut + Advice | Advisor |
| Aspect | 切面,跨越多个类的关注点 | @Aspect |
| Proxy | 代理对象 | AopProxy |

### 1.2 AOP实现方式

```
Spring AOP
  ├── JDK动态代理 (基于接口)
  │     └── java.lang.reflect.Proxy
  │
  └── CGLIB代理 (基于继承)
        └── net.sf.cglib.proxy.Enhancer
```

```java
public Object postProcessAfterInitialization(Object bean, String beanName) {
    
    if (bean != null) {
        Object cacheKey = getCacheKey(bean.getClass(), beanName);
        
        // 1. 检查是否需要代理
        if (!this.earlyProxyReferences.contains(cacheKey)) {
            
            // 2. 创建代理
            return wrapIfNecessary(bean, beanName, cacheKey);

    return bean;
```

### 3.2 wrapIfNecessary()方法

```java
// AbstractAutoProxyCreator.wrapIfNecessary
protected Object wrapIfNecessary(Object bean, String beanName, Object cacheKey) {
    
    // 1. 检查是否已经处理过
    if (StringUtils.hasLength(beanName) && this.targetSourcedBeans.contains(beanName)) {
        return bean;
    }
    
    // 2. 检查是否需要代理
    if (isInfrastructureClass(bean.getClass()) || shouldSkip(bean.getClass(), beanName)) {
        return bean;
    }
    
    // 3. 获取适用于该Bean的Advisor列表
    Object[] specificInterceptors = getAdvicesAndAdvisorsForBean(
        bean.getClass(), beanName, null);
    
    if (specificInterceptors != DO_NOT_PROXY) {
        
        // 4. 创建代理
        Object proxy = createProxy(
            bean.getClass(), beanName, specificInterceptors, 
            new SingletonTargetSource(bean));
        
        this.proxyTypes.put(cacheKey, proxy.getClass());
        
        return proxy;
    }
    
    return bean;
}
```

## 四、Advisor查找

### 4.1 getAdvicesAndAdvisorsForBean()

```java
// AbstractAdvisorAutoProxyCreator.getAdvicesAndAdvisorsForBean
protected Object[] getAdvicesAndAdvisorsForBean(Class<?> beanClass, 
                                                String beanName, 
                                                TargetSource targetSource) {
    
    // 1. 查找所有Advisor
    List<Advisor> advisors = findEligibleAdvisors(beanClass, beanName);
    
    if (advisors.isEmpty()) {
        return DO_NOT_PROXY;
    }
    
    return advisors.toArray();
}

protected List<Advisor> findEligibleAdvisors(Class<?> beanClass, String beanName) {
    
    // 2. 查找所有候选Advisor
    List<Advisor> candidateAdvisors = findCandidateAdvisors();
    
    // 3. 筛选适用的Advisor
    List<Advisor> eligibleAdvisors = findAdvisorsThatCanApply(candidateAdvisors, beanClass, beanName);
    
    // 4. 扩展(添加ExposeInvocationInterceptor等)
    extendAdvisors(eligibleAdvisors);
    
    if (!eligibleAdvisors.isEmpty()) {
        // 5. 排序
        eligibleAdvisors = sortAdvisors(eligibleAdvisors);
    }
    
    return eligibleAdvisors;
}
```

### 4.2 查找候选Advisor

```java
// AnnotationAwareAspectJAutoProxyCreator.findCandidateAdvisors
protected List<Advisor> findCandidateAdvisors() {
    
    // 1. 调用父类查找@Aspect注解的Bean
    List<Advisor> advisors = super.findCandidateAdvisors();
    
    // 2. 解析@Aspect注解,创建Advisor
    advisors.addAll(this.aspectJAdvisorsBuilder.buildAspectJAdvisors());
    
    return advisors;
}
```

### 4.3 解析@Aspect注解

```java
// BeanFactoryAspectJAdvisorsBuilder.buildAspectJAdvisors
public List<Advisor> buildAspectJAdvisors() {
    
    List<String> aspectNames = this.aspectBeanNames;
    
    if (aspectNames == null) {
        
        // 1. 查找所有@Aspect注解的Bean
        String[] beanNames = BeanFactoryUtils.beanNamesForTypeIncludingAncestors(
            this.beanFactory, Object.class, true, false);
        
        for (String beanName : beanNames) {
            
            Class<?> beanType = this.beanFactory.getType(beanName);
            
            // 2. 检查是否有@Aspect注解
            if (this.advisorFactory.isAspect(beanType)) {
                
                // 3. 解析@Aspect,创建Advisor
                List<Advisor> advisors = this.advisorFactory.getAdvisors(
                    new SingletonMetadataAwareAspectInstanceFactory(
                        this.beanFactory.getBean(beanName), beanName));
                
                advisors.addAll(advisors);
            }
        }
    }
    
    return advisors;
}
```

## 五、代理创建

### 5.1 createProxy()方法

```java
// AbstractAutoProxyCreator.createProxy
protected Object createProxy(Class<?> beanClass, String beanName, 
                            Object[] specificInterceptors, 
                            TargetSource targetSource) {
    
    // 1. 创建ProxyFactory
    ProxyFactory proxyFactory = new ProxyFactory();
    
    // 2. 复制配置
    proxyFactory.copyFrom(this);
    
    // 3. 确定代理方式(JDK or CGLIB)
    if (!proxyFactory.isProxyTargetClass()) {
        if (shouldProxyTargetClass(beanClass, beanName)) {
            proxyFactory.setProxyTargetClass(true);
        } else {
            evaluateProxyInterfaces(beanClass, proxyFactory);
        }
    }
    
    // 4. 添加Advisor
    Advisor[] advisors = buildAdvisors(beanName, specificInterceptors);
    proxyFactory.addAdvisors(advisors);
    
    // 5. 设置目标源
    proxyFactory.setTargetSource(targetSource);
    
    // 6. 自定义配置
    customizeProxyFactory(proxyFactory);
    
    // 7. 创建代理
    return proxyFactory.getProxy(getProxyClassLoader());
}
```

### 5.2 选择代理方式

```java
// DefaultAopProxyFactory.createAopProxy
public AopProxy createAopProxy(AdvisedSupport config) throws AopConfigException {
    
    // 1. 如果强制使用CGLIB或没有接口
    if (config.isOptimize() || config.isProxyTargetClass() || 
        hasNoUserSuppliedProxyInterfaces(config)) {
        
        Class<?> targetClass = config.getTargetClass();
        
        // 2. 目标是接口 → 用JDK代理
        if (targetClass.isInterface()) {
            return new JdkDynamicAopProxy(config);
        }
        
        // 3. 否则用CGLIB
        return new ObjenesisCglibAopProxy(config);
    }
    
    // 4. 默认用JDK代理
    return new JdkDynamicAopProxy(config);
}
```

**决策逻辑**:
- 有`proxyTargetClass=true` → CGLIB
- 目标是接口 → JDK动态代理
- 其他情况 → JDK动态代理

## 六、JDK动态代理

### 6.1 JdkDynamicAopProxy

```java
final class JdkDynamicAopProxy implements AopProxy, InvocationHandler, Serializable {
    
    private final AdvisedSupport advised;
    
    @Override
    public Object getProxy(ClassLoader classLoader) {
        
        // 1. 获取代理接口
        Class<?>[] proxiedInterfaces = AopProxyUtils.completeProxiedInterfaces(this.advised);
        
        // 2. 创建JDK代理
        return Proxy.newProxyInstance(classLoader, proxiedInterfaces, this);
    }
    
    @Override
    public Object invoke(Object proxy, Method method, Object[] args) throws Throwable {
        
        MethodInvocation invocation;
        Object oldProxy = null;
        boolean setProxyContext = false;
        
        TargetSource targetSource = this.advised.targetSource;
        Object target = null;
        
        try {
            
            // 1. 特殊方法处理(equals、hashCode)
            if (!this.equalsDefined && AopUtils.isEqualsMethod(method)) {
                return equals(args[0]);
            }
            if (!this.hashCodeDefined && AopUtils.isHashCodeMethod(method)) {
                return hashCode();
            }
            
            // 2. 创建MethodInvocation
            List<Object> chain = this.advised.getInterceptorsAndDynamicInterceptionAdvice(method, targetClass);
            
            if (chain.isEmpty()) {
                // 3. 没有拦截器,直接调用目标方法
                Object[] argsToUse = AopProxyUtils.adaptArgumentsIfNecessary(method, args);
                retVal = AopUtils.invokeJoinpointUsingReflection(target, method, argsToUse);
            } else {
                
                // 4. 创建ReflectiveMethodInvocation
                invocation = new ReflectiveMethodInvocation(
                    proxy, target, method, args, targetClass, chain);
                
                // 5. 执行拦截器链
                retVal = invocation.proceed();
            }
            
        } finally {
            // 清理
        }
        
        return retVal;
    }
}
```

### 6.2 拦截器链执行

```java
// ReflectiveMethodInvocation.proceed
public Object proceed() throws Throwable {
    
    // 1. 如果所有拦截器都执行完了,调用目标方法
    if (this.currentInterceptorIndex == this.interceptorsAndDynamicMethodMatchers.size() - 1) {
        return invokeJoinpoint();
    }
    
    // 2. 获取下一个拦截器
    Object interceptorOrInterceptionAdvice = 
        this.interceptorsAndDynamicMethodMatchers.get(++this.currentInterceptorIndex);
    
    if (interceptorOrInterceptionAdvice instanceof InterceptorAndDynamicMethodMatcher) {
        
        InterceptorAndDynamicMethodMatcher dm = 
            (InterceptorAndDynamicMethodMatcher) interceptorOrInterceptionAdvice;
        
        // 3. 动态匹配
        if (dm.methodMatcher.matches(this.method, this.targetClass, this.arguments)) {
            return dm.interceptor.invoke(this);
        } else {
            // 不匹配,跳过
            return proceed();
        }
    } else {
        
        // 4. 调用拦截器
        return ((MethodInterceptor) interceptorOrInterceptionAdvice).invoke(this);
    }
}
```

### 7.1 CglibAopProxy

```java
class CglibAopProxy implements AopProxy {
    
    @Override
    public Object getProxy(ClassLoader classLoader) {
        
        // 1. 创建Enhancer
        Enhancer enhancer = createEnhancer();
        
        // 2. 设置超类(目标类)
        enhancer.setSuperclass(proxySuperClass);
        
        // 3. 设置接口
        enhancer.setInterfaces(AopProxyUtils.completeProxiedInterfaces(this.advised));
        
        // 4. 设置回调(拦截器)
        enhancer.setCallbackFilter(new AcceptAllCallbackFilter());
        enhancer.setCallbacks(this.callbacks);
        
        // 5. 创建代理
        Object proxy = enhancer.create();
        
        return proxy;
    }
}
```

### 7.2 DynamicAdvisedInterceptor

```java
private static class DynamicAdvisedInterceptor implements MethodInterceptor, Serializable {
    
    @Override
    public Object intercept(Object proxy, Method method, Object[] args, 
                           MethodProxy methodProxy) throws Throwable {
        
        Object oldProxy = null;
        boolean setProxyContext = false;
        Object target = null;
        TargetSource targetSource = this.advised.getTargetSource();
        
        try {
            
            // 1. 获取目标对象
            target = targetSource.getTarget();
            
            // 2. 获取拦截器链
            List<Object> chain = this.advised.getInterceptorsAndDynamicInterceptionAdvice(
                method, targetClass);
            
            Object retVal;
            
            if (chain.isEmpty() && Modifier.isPublic(method.getModifiers())) {
                
                // 3. 没有拦截器,直接调用
                Object[] argsToUse = AopProxyUtils.adaptArgumentsIfNecessary(method, args);
                retVal = methodProxy.invoke(target, argsToUse);
            } else {
                
                // 4. 创建CglibMethodInvocation
                retVal = new CglibMethodInvocation(
                    proxy, target, method, args, targetClass, chain, methodProxy).proceed();
            }
            
            return retVal;
            
        } finally {
            // 清理
        }
    }
}
```

## 八、通知执行顺序

### 8.1 五种通知的执行顺序

```
@Around前置
  ↓
@Before
  ↓
执行目标方法
  ↓
@AfterReturning / @AfterThrowing
  ↓
@After
  ↓
@Around后置
```

### 8.2 源码实现

```java
// AspectJAfterAdvice.invoke
public Object invoke(MethodInvocation mi) throws Throwable {
    
    try {
        // 1. 执行后续拦截器(包括目标方法)
        return mi.proceed();
    } finally {
        // 2. 执行@After通知
        invokeAdviceMethod(mi.getThis(), null, null);
    }
}

// AfterReturningAdviceInterceptor.invoke
public Object invoke(MethodInvocation mi) throws Throwable {
    
    // 1. 执行后续拦截器
    Object retVal = mi.proceed();
    
    // 2. 执行@AfterReturning通知
    this.afterReturningAdvice.afterReturning(retVal, mi.getMethod(), 
                                             mi.getArguments(), mi.getThis());
    
    return retVal;
}

// AspectJAroundAdvice.invoke
public Object invoke(MethodInvocation mi) throws Throwable {
    
    if (!(mi instanceof ProxyMethodInvocation)) {
        throw new IllegalStateException("...");
    }
    
    ProxyMethodInvocation pmi = (ProxyMethodInvocation) mi;
    
    // 1. 创建ProceedingJoinPoint
    ProceedingJoinPoint pjp = lazyGetProceedingJoinPoint(pmi);
    
    // 2. 执行@Around通知
    return this.aspectInstanceFactory.getAspectInstance().invoke(pjp);
}
```

## 总结

本节深入分析了Spring AOP的源码实现:

**核心要点**:
1. @EnableAspectJAutoProxy注册AnnotationAwareAspectJAutoProxyCreator
2. BeanPostProcessor在Bean初始化后创建代理
3. 根据是否有接口选择JDK动态代理或CGLIB
4. 拦截器链按顺序执行五种通知
5. Around通知包裹整个调用过程

**设计模式应用**:
- **模板方法**: AbstractAutoProxyCreator定义流程
- **策略模式**: JdkDynamicAopProxy vs CglibAopProxy
- **责任链**: 拦截器链模式
- **工厂方法**: ProxyFactory创建代理
