## 一、JPA缓存概述

### 1.1 为什么需要缓存?

**数据库访问的性能瓶颈**:
- 🐌 网络延迟: 每次查询都需要网络往返
- 🐌 磁盘IO: 数据库需要从磁盘读取数据
- 🐌 锁竞争: 高并发下数据库锁竞争激烈

**缓存的优势**:
- ✅ **提升性能**: 内存访问速度是磁盘的10万倍
- ✅ **减少负载**: 降低数据库压力
- ✅ **提高并发**: 缓存可以承受更高的QPS

### 1.2 JPA两级缓存

```
┌─────────────────────────────────────────────────────┐
│              JPA 两级缓存架构                         │
├─────────────────────────────────────────────────────┤
│                                                     │
│  第一级缓存(EntityManager级别)                        │
│  ├── 默认开启,无法关闭                                │
│  ├── 生命周期: EntityManager实例                     │
│  └── 作用域: 单个事务内                              │
│                                                     │
│  第二级缓存(SessionFactory级别)                       │
│  ├── 需要手动配置                                    │
│  ├── 生命周期: 应用启动到关闭                         │
│  └── 作用域: 跨事务、跨EntityManager                 │
│                                                     │
└─────────────────────────────────────────────────────┘
```

#### 一级缓存 vs 二级缓存对比

| 特性 | 一级缓存 | 二级缓存 |
|------|---------|---------|
| **开启方式** | 自动开启 | 手动配置 |
| **作用域** | EntityManager | SessionFactory |
| **生命周期** | 事务级别 | 应用级别 |
| **线程安全** | 天然安全 | 需要保证 |
| **适用场景** | 同一事务内重复查询 | 跨事务共享数据 |
| **缓存提供者** | 无需 | Ehcache/Redis等 |

## 二、一级缓存(默认开启)

### 2.1 一级缓存示例

```java
@Service
@RequiredArgsConstructor
public class UserService {
    
    private final UserRepository userRepository;
    
    @Transactional
    public void testL1Cache() {
        // 第一次查询: 发送SQL到数据库
        User user1 = userRepository.findById(1L).orElse(null);
        System.out.println("user1: " + user1.getUsername());
        
        // 第二次查询: 从一级缓存获取,不发送SQL
        User user2 = userRepository.findById(1L).orElse(null);
        System.out.println("user2: " + user2.getUsername());
        
        // 验证: 两个对象是同一个引用
        System.out.println(user1 == user2); // true
    }
}
```

**输出日志**:
```sql
-- 第一次查询执行SQL
SELECT u.id, u.username, u.email FROM user u WHERE u.id = ?

user1: 张三
user2: 张三
true
```

**关键点**:
- 一级缓存在**同一个EntityManager**(事务)内有效
- 第二次查询不会发送SQL
- 返回的是同一个对象引用

### 2.2 一级缓存失效场景

#### 场景1: 不同事务

```java
@Transactional
public User getUserInTransaction1() {
    return userRepository.findById(1L).orElse(null);
}

@Transactional
public User getUserInTransaction2() {
    return userRepository.findById(1L).orElse(null);
}

// 测试
@Test
public void testDifferentTransactions() {
    User user1 = getUserInTransaction1(); // 执行SQL
    User user2 = getUserInTransaction2(); // 再次执行SQL(一级缓存失效)
}
```

#### 场景2: clear()清空缓存

```java
@Transactional
public void testClearCache() {
    User user1 = userRepository.findById(1L).orElse(null); // 执行SQL
    
    entityManager.clear(); // 清空一级缓存
    
    User user2 = userRepository.findById(1L).orElse(null); // 再次执行SQL
}
```

## 三、二级缓存配置

### 3.1 添加依赖

```xml
<!-- pom.xml -->
<dependencies>
    <!-- Spring Data JPA -->
    <dependency>
        <groupId>org.springframework.boot</groupId>
        <artifactId>spring-boot-starter-data-jpa</artifactId>
    </dependency>
    
    <!-- Hibernate Ehcache(二级缓存提供者) -->
    <dependency>
        <groupId>org.hibernate</groupId>
        <artifactId>hibernate-ehcache</artifactId>
        <version>5.6.15.Final</version>
    </dependency>
    
    <!-- Ehcache核心 -->
    <dependency>
        <groupId>net.sf.ehcache</groupId>
        <artifactId>ehcache</artifactId>
        <version>2.10.9.2</version>
    </dependency>
</dependencies>
```

### 3.2 配置文件

#### application.yml

```yaml
spring:
  jpa:
    # 启用二级缓存
    properties:
      hibernate:
        cache:
          use_second_level_cache: true          # 启用二级缓存
          use_query_cache: true                 # 启用查询缓存
          region_prefix: myapp                  # 缓存区域前缀
          provider_class: org.hibernate.cache.ehcache.EhCacheRegionFactory
        
        # 二级缓存策略配置
        entity:
          com.dontalk.entity.User:              # 实体类全限定名
            cache_concurrency_strategy: READ_WRITE  # 缓存策略
          
        # 查询缓存配置
        query:
          cache_region: query_cache             # 查询缓存区域名称
```

#### ehcache.xml (可选,自定义缓存配置)

```xml
<?xml version="1.0" encoding="UTF-8"?>
<ehcache xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
         xsi:noNamespaceSchemaLocation="http://ehcache.org/ehcache.xsd">
    
    <!-- 磁盘存储路径 -->
    <diskStore path="java.io.tmpdir/ehcache"/>
    
    <!-- 默认缓存配置 -->
    <defaultCache
        maxElementsInMemory="10000"           # 内存中最大元素数
        eternal="false"                       # 是否永久有效
        timeToIdleSeconds="120"               # 空闲超时时间(秒)
        timeToLiveSeconds="300"               # 存活时间(秒)
        overflowToDisk="true"                 # 溢出到磁盘
        diskPersistent="false"                # 磁盘持久化
        memoryStoreEvictionPolicy="LRU"/>     # 淘汰策略(LRU/FIFO/LFU)
    
    <!-- 用户实体缓存 -->
    <cache name="com.dontalk.entity.User"
        maxElementsInMemory="5000"
        eternal="false"
        timeToIdleSeconds="300"
        timeToLiveSeconds="600"
        overflowToDisk="true"/>
    
    <!-- 查询缓存 -->
    <cache name="query_cache"
        maxElementsInMemory="1000"
        eternal="false"
        timeToIdleSeconds="120"
        timeToLiveSeconds="300"
        overflowToDisk="true"/>
    
    <!-- 集合缓存 -->
    <cache name="com.dontalk.entity.User.orders"
        maxElementsInMemory="5000"
        eternal="false"
        timeToIdleSeconds="300"
        timeToLiveSeconds="600"
        overflowToDisk="true"/>
</ehcache>
```

### 3.3 实体类配置

#### 方式1: 注解配置(推荐)

```java
package com.dontalk.entity;

import lombok.Data;
import org.hibernate.annotations.Cache;
import org.hibernate.annotations.CacheConcurrencyStrategy;

import javax.persistence.*;

@Entity
@Table(name = "user")
@Data
@Cacheable  // 标记为可缓存
@Cache(usage = CacheConcurrencyStrategy.READ_WRITE)  // 设置缓存策略
public class User {
    
    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;
    
    @Column(name = "username")
    private String username;
    
    @Column(name = "email")
    private String email;
    
    @Column(name = "age")
    private Integer age;
}
```

#### 方式2: XML配置

```xml
<!-- persistence.xml -->
<persistence-unit name="myPersistenceUnit">
    <class>com.dontalk.entity.User</class>
    
    <shared-cache-mode>ENABLE_SELECTIVE</shared-cache-mode>
    
    <properties>
        <property name="javax.persistence.sharedCache.mode" 
                  value="ENABLE_SELECTIVE"/>
    </properties>
</persistence-unit>
```

### 3.4 缓存策略说明

| 策略 | 说明 | 适用场景 |
|------|------|---------|
| **READ_ONLY** | 只读缓存,不允许修改 | 字典表、配置表等不变数据 |
| **READ_WRITE** | 读写缓存,支持更新 | 频繁读写的业务数据 ⭐推荐 |
| **NONSTRICT_READ_WRITE** | 非严格读写,不保证一致性 | 对一致性要求不高的数据 |
| **TRANSACTIONAL** | 事务性缓存,需要JTA支持 | 分布式事务场景 |

### 4.1 基础缓存测试

@SpringBootTest
@Slf4j
public class SecondLevelCacheTest {

### 4.2 查询缓存

**问题**: 二级缓存默认只缓存`findById()`等根据ID查询的方法,不缓存JPQL/HQL查询。

**解决方案**: 启用查询缓存。

```java
@Repository
public interface UserRepository extends JpaRepository<User, Long> {
    
    /**
     * 根据用户名查询(启用查询缓存)
     */
    @Query("SELECT u FROM User u WHERE u.username = :username")
    @QueryHint(name = "org.hibernate.cacheable", value = "true")
    User findByUsername(@Param("username") String username);
    
    /**
     * 根据年龄范围查询(启用查询缓存)
     */
    @Query("SELECT u FROM User u WHERE u.age BETWEEN :minAge AND :maxAge")
    @QueryHint(name = "org.hibernate.cacheable", value = "true")
    List<User> findByAgeRange(@Param("minAge") int minAge, @Param("maxAge") int maxAge);
}
```

**测试查询缓存**:

```java
@Test
public void testQueryCache() {
    log.info("========== 第一次查询 ==========");
    List<User> users1 = userRepository.findByAgeRange(18, 30);
    log.info("查询结果数量: {}", users1.size());
    
    log.info("========== 第二次查询 ==========");
    List<User> users2 = userRepository.findByAgeRange(18, 30);
    log.info("查询结果数量: {}", users2.size());
    // 第二次查询不会执行SQL,从查询缓存获取
}
```

### 4.3 集合缓存

**场景**: 一对多关系中,缓存集合属性。

```java
@Entity
@Table(name = "user")
@Data
@Cacheable
@Cache(usage = CacheConcurrencyStrategy.READ_WRITE)
public class User {
    
    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;
    
    private String username;
    
    // 一对多: 用户的订单列表
    @OneToMany(mappedBy = "user", fetch = FetchType.LAZY)
    @Cache(usage = CacheConcurrencyStrategy.READ_WRITE)  // 缓存集合
    private List<Order> orders = new ArrayList<>();
}

@Entity
@Table(name = "orders")
@Data
@Cacheable
@Cache(usage = CacheConcurrencyStrategy.READ_WRITE)
public class Order {
    
    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;
    
    @Column(name = "order_no")
    private String orderNo;
    
    @ManyToOne(fetch = FetchType.LAZY)
    @JoinColumn(name = "user_id")
    private User user;
}
```

## 六、缓存管理

### 6.1 编程式清除缓存

```java
@Service
@RequiredArgsConstructor
public class CacheService {
    
    private final EntityManagerFactory entityManagerFactory;
    
    /**
     * 清除指定实体的缓存
     */
    public void evictEntityCache(Class<?> entityClass, Long id) {
        entityManagerFactory.getCache().evict(entityClass, id);
    }
    
    /**
     * 清除所有实体缓存
     */
    public void evictAllEntityCache() {
        entityManagerFactory.getCache().evictAll();
    }
    
    /**
     * 清除查询缓存
     */
    public void evictQueryCache() {
        entityManagerFactory.getCache().evictQueryRegions();
    }
    
    /**
     * 清除集合缓存
     */
    public void evictCollectionCache(String roleName) {
        entityManagerFactory.getCache().evictCollection(roleName);
    }
}
```

## 七、最佳实践

### 7.1 缓存设计原则

1. ✅ **选择合适的缓存策略**: 读多写少用READ_WRITE,只读用READ_ONLY
2. ✅ **设置合理的过期时间**: 避免缓存过大或过旧
3. ✅ **控制缓存粒度**: 只缓存热点数据
4. ✅ **监控缓存命中率**: 及时调整缓存策略
5. ✅ **注意缓存一致性**: 数据更新时及时清除缓存

### 7.2 性能优化技巧

#### 技巧1: 批量加载

```java
// ❌ 错误: N+1问题
List<User> users = userRepository.findAll();
for (User user : users) {
    List<Order> orders = user.getOrders(); // 每次都会查询数据库
}

// ✅ 正确: 使用JOIN FETCH
@Query("SELECT DISTINCT u FROM User u LEFT JOIN FETCH u.orders")
List<User> findAllWithOrders();
```

#### 技巧2: 分页查询不缓存

```java
// ❌ 不建议缓存分页查询
@Query("SELECT u FROM User u ORDER BY u.createTime DESC")
Page<User> findAll(Pageable pageable);

// ✅ 建议只缓存单条记录
User findById(Long id);
```

#### 技巧3: 使用缓存预热

```java
@Component
public class CacheWarmer implements ApplicationListener<ApplicationReadyEvent> {
    
    @Autowired
    private UserRepository userRepository;
    
    @Override
    public void onApplicationEvent(ApplicationReadyEvent event) {
        log.info("开始缓存预热...");
        
        // 预加载热点数据
        List<User> hotUsers = userRepository.findTop100ByOrderByLastLoginTimeDesc();
        for (User user : hotUsers) {
            // 触发缓存加载
            userRepository.findById(user.getId());
        }
        
        log.info("缓存预热完成");
    }
}
```

### 7.3 常见陷阱

#### 陷阱1: 缓存穿透

**问题**: 查询不存在的数据,缓存无法生效。

**解决**: 缓存空值或使用布隆过滤器。

```java
public User findById(Long id) {
    User user = cache.get(id);
    if (user == null) {
        user = userRepository.findById(id).orElse(null);
        if (user != null) {
            cache.put(id, user);
        } else {
            // 缓存空值,防止穿透
            cache.put(id, EMPTY_USER);
        }
    }
    return user == EMPTY_USER ? null : user;
}
```

#### 陷阱2: 缓存雪崩

**问题**: 大量缓存同时过期,导致数据库压力激增。

**解决**: 设置随机过期时间。

```java
// 基础过期时间 + 随机偏移
int expireTime = 3600 + new Random().nextInt(300); // 3600~3900秒
```

#### 陷阱3: 缓存击穿

**问题**: 热点key过期,大量请求同时查询数据库。

**解决**: 使用互斥锁。

```java
private final ConcurrentHashMap<Long, ReentrantLock> locks = new ConcurrentHashMap<>();

public User findById(Long id) {
    User user = cache.get(id);
    if (user == null) {
        ReentrantLock lock = locks.computeIfAbsent(id, k -> new ReentrantLock());
        lock.lock();
        try {
            // 双重检查
            user = cache.get(id);
            if (user == null) {
                user = userRepository.findById(id).orElse(null);
                if (user != null) {
                    cache.put(id, user);
                }
            }
        } finally {
            lock.unlock();
        }
    }
    return user;
}
```

## 八、总结

### 8.1 核心要点

1. ✅ **一级缓存**: EntityManager级别,默认开启
2. ✅ **二级缓存**: SessionFactory级别,需要配置
3. ✅ **缓存策略**: READ_ONLY、READ_WRITE、NONSTRICT_READ_WRITE
4. ✅ **查询缓存**: 需要手动启用(@QueryHint)
5. ✅ **集合缓存**: 一对多关系需要单独配置
6. ✅ **Redis缓存**: 生产环境推荐

### 8.2 配置清单

```yaml

spring:
  jpa:
    properties:
      hibernate:
        cache:
          use_second_level_cache: true    # 启用二级缓存
          use_query_cache: true           # 启用查询缓存
          region.factory_class: xxx       # 缓存提供者

# 实体类注解
@Cacheable

```

### 8.3 监控指标

- **命中率**: > 80% 为优秀
- **未命中次数**: 持续增高需要优化
- **缓存大小**: 避免超过内存限制
- **过期率**: 过高说明过期时间设置不合理

## 九、常见问题

### Q1: 二级缓存什么时候失效?

**A**: 
1. 调用`entityManagerFactory.getCache().evict()`清除
2. 实体被更新/删除时自动失效
3. 缓存过期时间到达
4. 缓存空间满,触发淘汰策略

### Q2: 如何判断是否需要二级缓存?

**A**: 
- ✅ 读多写少的数据
- ✅ 数据量不大(< 10万条)
- ✅ 对一致性要求不高
- ❌ 频繁更新的数据不用缓存

### Q3: Ehcache和Redis如何选择?

**A**: 
- **单体应用**: Ehcache(简单、快速)
- **微服务**: Redis(分布式、共享)
- **混合方案**: 本地缓存 + Redis(多级缓存)
