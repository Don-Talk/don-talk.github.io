## 一、JPA概述

### 1.1 什么是JPA

**JPA (Java Persistence API)** 是Java EE规范中的持久层API,定义了ORM(对象关系映射)的标准接口。

**核心理念**: 
- 将Java对象映射到数据库表
- 通过操作对象来操作数据库
- 屏蔽底层SQL差异,实现跨数据库移植

### 1.2 JPA vs Hibernate vs Spring Data JPA

| 技术 | 说明 | 关系 |
|------|------|------|
| JPA | 规范/标准 | 定义接口,没有实现 |
| Hibernate | JPA的实现 | 最流行的JPA实现框架 |
| Spring Data JPA | 对JPA的封装 | 简化JPA开发,基于Hibernate |

**通俗理解**:
- JPA = 接口规范(像JDBC)
- Hibernate = 具体实现(像MySQL驱动)
- Spring Data JPA = 工具类(简化Hibernate使用)

### 1.3 Spring Data JPA的优势

**传统JPA/Hibernate的痛点**:
```java
// 繁琐的代码
EntityManagerFactory emf = Persistence.createEntityManagerFactory("myUnit");
EntityManager em = emf.createEntityManager();
EntityTransaction tx = em.getTransaction();

try {
    tx.begin();
    User user = new User();
    user.setName("张三");
    em.persist(user);
    tx.commit();
} catch (Exception e) {
    tx.rollback();
} finally {
    em.close();
    emf.close();
}
```

**Spring Data JPA的简洁**:
```java
// 只需一行代码
userRepository.save(new User("张三"));
```

**优势**:
1. **零实现**: 只需定义接口,无需编写实现类
2. **方法名查询**: 根据方法名自动生成SQL
3. **分页排序**: 内置Pageable支持
4. **审计功能**: 自动记录创建时间、更新时间
5. **事务管理**: 默认开启事务

```xml
<dependencies>

        <groupId>org.springframework.boot</groupId>

        <artifactId>mysql-connector-java</artifactId>
        <scope>runtime</scope>
    </dependency>
    
    <!--Lombok-->
    <dependency>
        <groupId>org.projectlombok</groupId>
        <artifactId>lombok</artifactId>
        <optional>true</optional>
    </dependency>
</dependencies>
```

### 2.2 配置数据源

`application.yml`:

```yaml
spring:
  datasource:
    driver-class-name: com.mysql.cj.jdbc.Driver
    url: jdbc:mysql://localhost:3306/jpa_db?useUnicode=true&characterEncoding=utf8&serverTimezone=Asia/Shanghai
    username: root
    password: root
    
  jpa:
    # 数据库平台
    database: mysql
    # 是否显示SQL
    show-sql: true
    # SQL格式化
    properties:
      hibernate:
        format_sql: true
    # DDL策略
    hibernate:
      ddl-auto: update
      # create: 每次启动删除旧表,创建新表
      # create-drop: 关闭时删除表
      # update: 自动更新表结构(推荐开发环境)
      # validate: 验证表结构,不匹配则报错
      # none: 不做任何操作(推荐生产环境)
```

### 2.3 创建数据库表

```sql
CREATE DATABASE jpa_db CHARACTER SET utf8mb4;

USE jpa_db;

CREATE TABLE `user` (
  `id` BIGINT NOT NULL AUTO_INCREMENT COMMENT '用户ID',
  `username` VARCHAR(50) NOT NULL COMMENT '用户名',
  `password` VARCHAR(100) NOT NULL COMMENT '密码',
  `email` VARCHAR(100) COMMENT '邮箱',
  `age` INT COMMENT '年龄',
  `create_time` DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  `update_time` DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  PRIMARY KEY (`id`),
  UNIQUE KEY `uk_username` (`username`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='用户表';
```

## 三、实体映射

### 3.1 基本实体类

```java
package com.example.entity;

import lombok.Data;
import javax.persistence.*;
import java.time.LocalDateTime;

@Data
@Entity  // 标识为JPA实体
@Table(name = "user")  // 映射的表名
public class User {
    
    @Id  // 主键
    @GeneratedValue(strategy = GenerationType.IDENTITY)  // 自增策略
    private Long id;
    
    @Column(name = "username", nullable = false, length = 50)
    private String username;
    
    @Column(name = "password", nullable = false, length = 100)
    private String password;
    
    @Column(name = "email", length = 100)
    private String email;
    
    private Integer age;
    
    @Column(name = "create_time")
    private LocalDateTime createTime;
    
    @Column(name = "update_time")
    private LocalDateTime updateTime;
}
```

### 3.2 常用注解详解

#### @Entity

标识该类是一个JPA实体,必须标注在主类上。

```java
@Entity
public class User {
    // ...
}
```

#### @Table

指定映射的数据库表名。

```java
@Table(name = "sys_user")  // 表名与类名不同时使用
public class User {
    // ...
}
```

#### @Id

标识主键字段。

```java
@Id
private Long id;
```

#### @GeneratedValue

主键生成策略。

```java
// 策略1: IDENTITY - 数据库自增(MySQL)
@GeneratedValue(strategy = GenerationType.IDENTITY)

// 策略2: SEQUENCE - 序列(Oracle)
@GeneratedValue(strategy = GenerationType.SEQUENCE, generator = "user_seq")
@SequenceGenerator(name = "user_seq", sequenceName = "seq_user_id")

// 策略3: TABLE - 使用表模拟序列(通用)
@GeneratedValue(strategy = GenerationType.TABLE, generator = "user_gen")
@TableGenerator(name = "user_gen", table = "hibernate_sequences")

// 策略4: AUTO - 自动选择(默认)
@GeneratedValue(strategy = GenerationType.AUTO)
```

#### @Column

列属性配置。

```java
@Column(
    name = "user_name",      // 列名
    nullable = false,         // 是否允许为空
    unique = true,            // 是否唯一
    length = 50,              // 字符串长度
    precision = 10,           // 数字精度
    scale = 2                 // 小数位数
)
private String username;
```

#### @Transient

标识该字段不映射到数据库。

```java
@Transient
private String confirmPassword;  // 不会保存到数据库
```

## 四、Repository接口

### 4.1 Repository层次结构

```
Repository (顶层接口)
  └── CrudRepository (CRUD操作)
        └── PagingAndSortingRepository (分页排序)
              └── JpaRepository (JPA特有功能)
```

### 4.2 创建Repository接口

```java
package com.example.repository;

import com.example.entity.User;
import org.springframework.data.jpa.repository.JpaRepository;
import org.springframework.stereotype.Repository;

@Repository
public interface UserRepository extends JpaRepository<User, Long> {
    // 继承JpaRepository后,自动拥有以下方法:
    // save(), findById(), findAll(), deleteById(), count()等
}
```

### 4.3 基础CRUD操作

```java
@Service
public class UserService {
    
    @Autowired
    private UserRepository userRepository;
    
    /**
     * 保存用户
     */
    public User saveUser(User user) {
        return userRepository.save(user);
    }
    
    /**
     * 批量保存
     */
    public List<User> saveUsers(List<User> users) {
        return userRepository.saveAll(users);
    }
    
    /**
     * 根据ID查询
     */
    public Optional<User> findById(Long id) {
        return userRepository.findById(id);
    }
    
    /**
     * 查询所有
     */
    public List<User> findAll() {
        return userRepository.findAll();
    }
    
    /**
     * 根据ID删除
     */
    public void deleteById(Long id) {
        userRepository.deleteById(id);
    }
    
    /**
     * 统计总数
     */
    public long count() {
        return userRepository.count();
    }
    
    /**
     * 判断是否存在
     */
    public boolean existsById(Long id) {
        return userRepository.existsById(id);
    }
}
```

### 4.4 方法名查询(Query By Method Name)

Spring Data JPA可以根据方法名自动生成SQL!

```java
public interface UserRepository extends JpaRepository<User, Long> {
    
    // 根据用户名查询
    User findByUsername(String username);
    
    // 根据邮箱查询
    User findByEmail(String email);
    
    // 根据年龄查询
    List<User> findByAge(Integer age);
    
    // 根据年龄大于查询
    List<User> findByAgeGreaterThan(Integer age);
    
    // 根据年龄在范围内查询
    List<User> findByAgeBetween(Integer min, Integer max);
    
    // 根据用户名模糊查询
    List<User> findByUsernameContaining(String keyword);
    
    // 根据用户名开头查询
    List<User> findByUsernameStartingWith(String prefix);
    
    // 多条件查询(AND)
    List<User> findByUsernameAndAge(String username, Integer age);
    
    // 多条件查询(OR)
    List<User> findByUsernameOrEmail(String username, String email);
    
    // 排序
    List<User> findByAgeOrderByCreateTimeDesc(Integer age);
    
    // 前N条记录
    List<User> findTop3ByAgeOrderByCreateTimeDesc(Integer age);
}
```

**方法名命名规则**:

| 关键字 | 示例 | SQL片段 |
|--------|------|---------|
| And | findByUsernameAndAge | WHERE username = ? AND age = ? |
| Or | findByUsernameOrEmail | WHERE username = ? OR email = ? |
| Between | findByAgeBetween | WHERE age BETWEEN ? AND ? |
| LessThan | findByAgeLessThan | WHERE age < ? |
| GreaterThan | findByAgeGreaterThan | WHERE age > ? |
| Like | findByUsernameLike | WHERE username LIKE ? |
| Containing | findByUsernameContaining | WHERE username LIKE %?% |
| StartsWith | findByUsernameStartingWith | WHERE username LIKE ?% |
| EndsWith | findByUsernameEndingWith | WHERE username LIKE %? |
| IsNull | findByEmailIsNull | WHERE email IS NULL |
| IsNotNull | findByEmailIsNotNull | WHERE email IS NOT NULL |
| OrderBy | findByAgeOrderByCreateTimeDesc | ORDER BY create_time DESC |
| Top | findTop3ByAge | LIMIT 3 |

## 五、JPQL查询

### 5.1 什么是JPQL

**JPQL (Java Persistence Query Language)** 是面向对象的查询语言,语法类似SQL,但操作的是实体对象而非数据库表。

**特点**:
- 跨数据库,可移植性强
- 面向对象,更符合Java思维
- 支持多态查询

### 5.2 @Query注解

```java
public interface UserRepository extends JpaRepository<User, Long> {
    
    /**
     * 基本JPQL查询
     */
    @Query("SELECT u FROM User u WHERE u.username = ?1")
    User findByUsername(String username);
    
    /**
     * 命名参数
     */
    @Query("SELECT u FROM User u WHERE u.username = :username")
    User findByUsername(@Param("username") String username);
    
    /**
     * 模糊查询
     */
    @Query("SELECT u FROM User u WHERE u.username LIKE %:keyword%")
    List<User> searchByUsername(@Param("keyword") String keyword);
    
    /**
     * 范围查询
     */
    @Query("SELECT u FROM User u WHERE u.age BETWEEN :min AND :max")
    List<User> findByAgeRange(@Param("min") Integer min, @Param("max") Integer max);
    
    /**
     * 聚合查询
     */
    @Query("SELECT COUNT(u) FROM User u WHERE u.age > :age")
    long countByAgeGreaterThan(@Param("age") Integer age);
    
    /**
     * 更新操作(需要@Transactional)
     */
    @Modifying
    @Transactional
    @Query("UPDATE User u SET u.email = :email WHERE u.id = :id")
    int updateEmail(@Param("id") Long id, @Param("email") String email);
    
    /**
     * 删除操作
     */
    @Modifying
    @Transactional
    @Query("DELETE FROM User u WHERE u.age < :age")
    int deleteByAgeLessThan(@Param("age") Integer age);
}
```

### 5.3 原生SQL查询

```java
public interface UserRepository extends JpaRepository<User, Long> {
    
    /**
     * 原生SQL查询
     */
    @Query(value = "SELECT * FROM user WHERE username = ?1", nativeQuery = true)
    User findByUsernameNative(String username);
    
    /**
     * 复杂JOIN查询
     */
    @Query(value = "SELECT u.* FROM user u INNER JOIN order o ON u.id = o.user_id WHERE o.amount > ?1", 
           nativeQuery = true)
    List<User> findUsersWithLargeOrders(BigDecimal minAmount);
}
```

## 六、分页与排序

### 6.1 分页查询

```java
public interface UserRepository extends JpaRepository<User, Long> {
    
    /**
     * 分页查询
     */
    Page<User> findAll(Pageable pageable);
    
    /**
     * 条件分页查询
     */
    Page<User> findByAge(Integer age, Pageable pageable);
}
```

**Service层调用**:

```java
@Service
public class UserService {
    
    @Autowired
    private UserRepository userRepository;
    
    public Page<User> getUsers(int page, int size) {
        // 创建分页对象(页码从0开始)
        Pageable pageable = PageRequest.of(page, size);
        
        return userRepository.findAll(pageable);
    }
    
    public Page<User> getUsersByAge(Integer age, int page, int size) {
        // 分页 + 排序
        Pageable pageable = PageRequest.of(page, size, Sort.by("createTime").descending());
        
        return userRepository.findByAge(age, pageable);
    }
}
```

**返回结果**:

```json
{
  "content": [...],      // 数据列表
  "totalElements": 100,  // 总记录数
  "totalPages": 10,      // 总页数
  "size": 10,            // 每页大小
  "number": 0,           // 当前页码
  "first": true,         // 是否第一页
  "last": false          // 是否最后一页
}
```

### 6.2 排序

```java
// 单字段排序
Sort sort = Sort.by("createTime").descending();

// 多字段排序
Sort sort = Sort.by("age").ascending().and(Sort.by("createTime").descending());

// 组合使用
Pageable pageable = PageRequest.of(0, 10, sort);
List<User> users = userRepository.findAll(pageable).getContent();
```

## 总结

本节介绍了Spring Data JPA的基础知识:

**核心要点**:
1. JPA是规范,Hibernate是实现,Spring Data JPA是封装
2. 只需定义Repository接口,无需编写实现类
3. 方法名查询可以自动生成SQL
4. JPQL是面向对象的查询语言
5. 内置分页排序支持

**最佳实践**:
- 开发环境使用ddl-auto=update,生产环境用validate或none
- 简单查询使用方法名查询,复杂查询用JPQL
- 分页查询使用Pageable,避免一次性加载大量数据
- 更新和删除操作需要@Transactional注解
- 优先使用JPQL,必要时才用原生SQL
