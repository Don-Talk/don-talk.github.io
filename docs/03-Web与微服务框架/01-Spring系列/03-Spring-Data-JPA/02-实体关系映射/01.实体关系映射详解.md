## 一、实体关系概述

### 1.1 四种关系类型

| 关系类型 | 注解 | 示例 |
|---------|------|------|
| 一对一 | @OneToOne | 用户 ↔ 身份证 |
| 一对多 | @OneToMany | 部门 → 员工 |
| 多对一 | @ManyToOne | 订单 → 用户 |
| 多对多 | @ManyToMany | 学生 ↔ 课程 |

### 1.2 核心概念

**拥有方 (Owner)**: 
- 负责维护关系的一方
- 数据库中存储外键的表
- 在双向关系中,没有mappedBy的一方

**被拥有方 (Inverse)**:
- 不维护关系的一方
- 使用mappedBy指向拥有方

## 二、一对一映射 (@OneToOne)

### 2.1 单向一对一

**场景**: 用户有一个身份证

#### 数据库设计

```sql
CREATE TABLE `user` (
  `id` BIGINT PRIMARY KEY AUTO_INCREMENT,
  `username` VARCHAR(50) NOT NULL
);

CREATE TABLE `id_card` (
  `id` BIGINT PRIMARY KEY AUTO_INCREMENT,
  `card_no` VARCHAR(18) NOT NULL,
  `user_id` BIGINT UNIQUE,  -- 外键
  FOREIGN KEY (`user_id`) REFERENCES `user`(`id`)
);
```

#### 实体类

```java
// User实体(拥有方)
@Entity
@Table(name = "user")
public class User {
    
    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;
    
    private String username;
    
    // 一对一映射
    @OneToOne
    @JoinColumn(name = "id", referencedColumnName = "user_id")
    private IdCard idCard;
}

// IdCard实体
@Entity
@Table(name = "id_card")
public class IdCard {
    
    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;
    
    @Column(name = "card_no")
    private String cardNo;
}
```

### 2.2 双向一对一

#### 实体类

```java
// User实体(拥有方)
@Entity
@Table(name = "user")
public class User {
    
    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;
    
    private String username;
    
    @OneToOne(cascade = CascadeType.ALL)
    @JoinColumn(name = "id", referencedColumnName = "user_id")
    private IdCard idCard;
}

// IdCard实体(被拥有方)
@Entity
@Table(name = "id_card")
public class IdCard {
    
    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;
    
    @Column(name = "card_no")
    private String cardNo;
    
    // mappedBy指向User中的idCard字段
    @OneToOne(mappedBy = "idCard")
    private User user;
}
```

**关键点**:
- `@JoinColumn` 在拥有方(User)
- `mappedBy` 在被拥有方(IdCard)

## 三、一对多/多对一映射 (@OneToMany / @ManyToOne)

### 3.1 单向多对一

**场景**: 多个订单属于一个用户

#### 数据库设计

```sql
CREATE TABLE `user` (
  `id` BIGINT PRIMARY KEY AUTO_INCREMENT,
  `username` VARCHAR(50)
);

CREATE TABLE `orders` (
  `id` BIGINT PRIMARY KEY AUTO_INCREMENT,
  `order_no` VARCHAR(50),
  `amount` DECIMAL(10,2),
  `user_id` BIGINT,  -- 外键
  FOREIGN KEY (`user_id`) REFERENCES `user`(`id`)
);
```

#### 实体类

```java
// Order实体(拥有方)
@Entity
@Table(name = "orders")
public class Order {
    
    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;
    
    @Column(name = "order_no")
    private String orderNo;
    
    private BigDecimal amount;
    
    // 多对一
    @ManyToOne
    @JoinColumn(name = "user_id")
    private User user;
}

// User实体
@Entity
@Table(name = "user")
public class User {
    
    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;
    
    private String username;
}
```

### 3.2 双向一对多(推荐)

#### 实体类

```java
// User实体(被拥有方)
@Entity
@Table(name = "user")
public class User {
    
    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;
    
    private String username;
    
    // 一对多,mappedBy指向Order中的user字段
    @OneToMany(mappedBy = "user", cascade = CascadeType.ALL)
    private List<Order> orders = new ArrayList<>();
    
    // 便捷方法
    public void addOrder(Order order) {
        orders.add(order);
        order.setUser(this);
    }
    
    public void removeOrder(Order order) {
        orders.remove(order);
        order.setUser(null);
    }
}

// Order实体(拥有方)
@Entity
@Table(name = "orders")
public class Order {
    
    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;
    
    @Column(name = "order_no")
    private String orderNo;
    
    private BigDecimal amount;
    
    // 多对一
    @ManyToOne(fetch = FetchType.LAZY)  // 延迟加载
    @JoinColumn(name = "user_id")
    private User user;
}
```

**关键点**:
- `@ManyToOne` 在拥有方(Order),有外键
- `@OneToMany` 在被拥有方(User),使用mappedBy
- 建议设置`fetch = FetchType.LAZY`避免性能问题

### 3.3 级联操作

**CascadeType枚举**:

| 类型 | 说明 |
|------|------|
| ALL | 所有操作都级联 |
| PERSIST | 保存时级联 |
| MERGE | 更新时级联 |
| REMOVE | 删除时级联 |
| REFRESH | 刷新时级联 |
| DETACH | 分离时级联 |

**示例**:

```java
@OneToMany(mappedBy = "user", cascade = {CascadeType.PERSIST, CascadeType.MERGE})
private List<Order> orders;
```

**使用场景**:

```java
@Service
public class UserService {
    
    @Autowired
    private UserRepository userRepository;
    
    public void createUserWithOrders() {
        User user = new User();
        user.setUsername("张三");
        
        Order order1 = new Order();
        order1.setOrderNo("ORD001");
        order1.setAmount(new BigDecimal("100"));
        
        Order order2 = new Order();
        order2.setOrderNo("ORD002");
        order2.setAmount(new BigDecimal("200"));
        
        // 添加订单到用户
        user.addOrder(order1);
        user.addOrder(order2);
        
        // 只需保存用户,订单会自动保存(因为cascade=PERSIST)
        userRepository.save(user);
    }
}
```

## 四、多对多映射 (@ManyToMany)

### 4.1 场景分析

**场景**: 学生可以选多门课程,一门课程可以被多个学生选择

#### 数据库设计

```sql
-- 学生表
CREATE TABLE `student` (
  `id` BIGINT PRIMARY KEY AUTO_INCREMENT,
  `name` VARCHAR(50)
);

-- 课程表
CREATE TABLE `course` (
  `id` BIGINT PRIMARY KEY AUTO_INCREMENT,
  `course_name` VARCHAR(100)
);

-- 中间表
CREATE TABLE `student_course` (
  `student_id` BIGINT,
  `course_id` BIGINT,
  PRIMARY KEY (`student_id`, `course_id`),
  FOREIGN KEY (`student_id`) REFERENCES `student`(`id`),
  FOREIGN KEY (`course_id`) REFERENCES `course`(`id`)
);
```

### 4.2 实体类

```java
// Student实体(拥有方)
@Entity
@Table(name = "student")
public class Student {
    
    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;
    
    private String name;
    
    // 多对多
    @ManyToMany(cascade = CascadeType.PERSIST)
    @JoinTable(
        name = "student_course",  // 中间表名
        joinColumns = @JoinColumn(name = "student_id"),  // 当前表外键
        inverseJoinColumns = @JoinColumn(name = "course_id")  // 对方表外键
    )
    private Set<Course> courses = new HashSet<>();
    
    // 便捷方法
    public void addCourse(Course course) {
        courses.add(course);
    }
}

// Course实体(被拥有方)
@Entity
@Table(name = "course")
public class Course {
    
    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;
    
    @Column(name = "course_name")
    private String courseName;
    
    // 多对多,mappedBy指向Student中的courses字段
    @ManyToMany(mappedBy = "courses")
    private Set<Student> students = new HashSet<>();
}
```

**关键点**:
- `@JoinTable` 在拥有方(Student)
- 指定中间表名和两个外键
- 被拥有方(Course)使用mappedBy

### 4.3 使用示例

```java
@Service
public class StudentService {
    
    @Autowired
    private StudentRepository studentRepository;
    
    @Autowired
    private CourseRepository courseRepository;
    
    public void enrollStudent() {
        // 创建学生
        Student student = new Student();
        student.setName("李四");
        
        // 查询课程
        Course math = courseRepository.findById(1L).orElseThrow();
        Course english = courseRepository.findById(2L).orElseThrow();
        
        // 选课
        student.addCourse(math);
        student.addCourse(english);
        
        // 保存(级联保存课程关联)
        studentRepository.save(student);
    }
}
```

## 五、延迟加载与即时加载

### 5.1 Fetch类型

| 类型 | 说明 | 适用场景 |
|------|------|----------|
| EAGER | 即时加载,立即查询关联对象 | 一对一、多对一(默认) |
| LAZY | 延迟加载,使用时才查询 | 一对多、多对多(默认) |

### 5.2 N+1问题

**问题描述**:

```java
// 查询10个用户
List<User> users = userRepository.findAll();

// 访问每个用户的订单(触发10次额外查询)
for (User user : users) {
    System.out.println(user.getOrders().size());  // N+1问题!
}
```

**解决方案1: JOIN FETCH**

```java
public interface UserRepository extends JpaRepository<User, Long> {
    
    @Query("SELECT DISTINCT u FROM User u LEFT JOIN FETCH u.orders")
    List<User> findAllWithOrders();
}
```

**解决方案2: @EntityGraph**

```java
public interface UserRepository extends JpaRepository<User, Long> {
    
    @EntityGraph(attributePaths = {"orders"})
    List<User> findAll();
}
```

**解决方案3: Batch Size**

```java
@OneToMany(mappedBy = "user", fetch = FetchType.LAZY)
@BatchSize(size = 10)  // 批量加载,每次10个
private List<Order> orders;
```

## 六、最佳实践

### 6.1 关系设计原则

1. **优先使用单向关系**: 简单清晰,避免复杂性
2. **双向关系谨慎使用**: 只在确实需要时才用
3. **设置合理的Fetch策略**: 避免N+1问题
4. **级联操作要小心**: 防止误删数据

### 6.2 常见陷阱

#### 陷阱1: 循环引用导致JSON序列化失败

**问题**:

```java
// User引用Order,Order又引用User,形成循环
public class User {
    private List<Order> orders;
}

public class Order {
    private User user;
}
```

**解决**:

```java
// 方案1: @JsonIgnore
@JsonIgnore
private User user;

// 方案2: @JsonBackReference / @JsonManagedReference
@JsonBackReference
private User user;

@JsonManagedReference
private List<Order> orders;
```

#### 陷阱2: LazyInitializationException

**问题**: Session关闭后访问延迟加载的对象

```java
@Transactional
public User getUser(Long id) {
    return userRepository.findById(id).orElseThrow();
}

// 在Controller中访问orders会报错
User user = userService.getUser(1L);
user.getOrders();  // LazyInitializationException!
```

**解决**:

```java
// 方案1: 在Service层初始化
@Transactional
public User getUserWithOrders(Long id) {
    User user = userRepository.findById(id).orElseThrow();
    Hibernate.initialize(user.getOrders());  // 强制加载
    return user;
}

// 方案2: 使用JOIN FETCH
@Query("SELECT u FROM User u LEFT JOIN FETCH u.orders WHERE u.id = :id")
User findByIdWithOrders(@Param("id") Long id);
```

### 6.3 性能优化建议

1. **合理使用懒加载**: 大数据量集合用LAZY
2. **避免N+1查询**: 使用JOIN FETCH或@BatchSize
3. **只查询需要的字段**: 使用投影(Projection)
4. **分页查询**: 避免一次性加载大量数据
5. **缓存热点数据**: 使用二级缓存

## 总结

本节深入讲解了JPA的实体关系映射:

**核心要点**:
1. 一对一使用@OneToOne,注意区分拥有方和被拥有方
2. 一对多/多对一是最常用的关系,@ManyToOne在拥有方
3. 多对多需要中间表,使用@JoinTable配置
4. 级联操作简化代码,但要谨慎使用
5. 延迟加载避免性能问题,注意N+1陷阱

**最佳实践**:
- 优先使用单向关系,简化设计
- 一对多集合默认使用LAZY加载
- 使用JOIN FETCH解决N+1问题
- 双向关系要维护两端的一致性
- 注意JSON序列化时的循环引用
