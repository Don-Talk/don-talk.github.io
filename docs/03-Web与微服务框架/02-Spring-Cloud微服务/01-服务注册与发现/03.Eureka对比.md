## 二、核心差异对比

### 2.1 CAP理论差异

#### Eureka: AP模式(高可用)

```
特点:
✅ 保证可用性: 即使部分节点宕机,仍能提供服务
✅ 分区容错性: 网络分区时系统仍可用
❌ 不保证强一致性: 节点间数据可能短暂不一致

适用场景:
- 服务注册发现(允许短暂不一致)
- 对可用性要求高的场景
```

**工作原理**:
```
Client → Node1: 注册服务
Node1 → 本地存储: 保存
Node1 → Client: 立即返回成功
Node1 → Node2: 异步同步(可能延迟)
```

#### Nacos: CP + AP可切换

```
CP模式(配置中心):
✅ 强一致性: 所有节点数据一致
❌ 写入性能较低
❌ Leader选举期间不可用

AP模式(服务发现,默认):
✅ 高可用性
✅ 写入性能高
⚠️ 短暂不一致(最终一致)
```

**切换方式**:
```yaml
spring:
  cloud:
    nacos:
      discovery:
        # true=AP模式(默认), false=CP模式
        ephemeral: true
```

### 2.2 功能对比

#### Eureka的功能

```
✅ 服务注册
✅ 服务发现
✅ 健康检查(TCP)
✅ 自我保护机制
❌ 配置管理(需额外引入Spring Cloud Config)
❌ 服务元数据管理(功能弱)
❌ 跨数据中心支持
❌ 权限控制
```

#### Nacos的功能

```
✅ 服务注册
✅ 服务发现
✅ 健康检查(TCP/HTTP/MySQL/自定义)
✅ 配置管理(内置)
✅ 服务元数据管理(强大)
✅ 跨数据中心支持
✅ 权限控制(RBAC)
✅ 服务权重调整
✅ 灰度发布支持
✅ 流量保护(Sentinel整合)
```

**结论**: Nacos功能更全面,一站式解决方案。

### 2.3 性能对比

#### 基准测试数据

| 指标 | Eureka | Nacos | 提升 |
|------|--------|-------|------|
| **注册QPS** | ~5000 | ~10000 | 2x |
| **查询QPS** | ~8000 | ~15000 | 1.87x |
| **平均响应时间** | 50ms | 20ms | 60%↓ |
| **内存占用** | 512MB | 256MB | 50%↓ |
| **启动时间** | 30s | 10s | 66%↓ |

**测试环境**:
- CPU: 4核
- 内存: 8GB
- 服务数量: 1000个
- 实例数量: 5000个

#### 性能优化技巧

**Eureka优化**:
```yaml
eureka:
  server:
    # 关闭自我保护(开发环境)
    enable-self-preservation: false
    # 清理间隔(毫秒)
    eviction-interval-timer-in-ms: 5000
  client:
    # 心跳间隔(秒)
    lease-renewal-interval-in-seconds: 5
    # 过期时间(秒)

```

**Nacos优化**:
```yaml
spring:
  cloud:
    nacos:

        # 心跳间隔(毫秒)
        heart-beat-interval: 5000
        # 心跳超时(毫秒)
        heart-beat-timeout: 15000
```

## 三、代码对比

### 3.1 依赖对比

#### Eureka依赖

```xml
<!-- Eureka Client -->
<dependency>
    <groupId>org.springframework.cloud</groupId>
    <artifactId>spring-cloud-starter-netflix-eureka-client</artifactId>
</dependency>

<!-- 配置中心(需要额外引入) -->
<dependency>
    <groupId>org.springframework.cloud</groupId>
    <artifactId>spring-cloud-config-client</artifactId>
</dependency>
```

#### Nacos依赖

```xml
<!-- Nacos服务发现(一个依赖搞定) -->
<dependency>
    <groupId>com.alibaba.cloud</groupId>
    <artifactId>spring-cloud-starter-alibaba-nacos-discovery</artifactId>
</dependency>

<!-- Nacos配置中心(可选) -->
<dependency>
    <groupId>com.alibaba.cloud</groupId>
    <artifactId>spring-cloud-starter-alibaba-nacos-config</artifactId>
</dependency>
```

**优势**: Nacos一个依赖替代Eureka两个依赖。

### 3.2 配置对比

#### Eureka配置

```yaml
server:
  port: 8081

spring:
  application:
    name: user-service

eureka:
  client:
    service-url:
      defaultZone: http://localhost:8761/eureka/
    # 心跳间隔
    lease-renewal-interval-in-seconds: 5
    # 过期时间
    lease-expiration-duration-in-seconds: 15
  instance:
    # 实例ID
    instance-id: ${spring.application.name}:${server.port}
    # 显示IP
    prefer-ip-address: true
```

#### Nacos配置

```yaml
server:
  port: 8081

spring:
  application:
    name: user-service
  cloud:
    nacos:
      discovery:
        server-addr: 127.0.0.1:8848
        namespace: public
        group: DEFAULT_GROUP
        weight: 1
        metadata:
          version: 1.0.0
```

**对比**:
- ✅ Nacos配置更简洁
- ✅ Nacos支持命名空间、分组、元数据
- ✅ Nacos支持权重配置

### 3.3 注解对比

#### Eureka注解

```java
@SpringBootApplication
@EnableEurekaClient  // 启用Eureka客户端
public class UserServiceApplication {
    public static void main(String[] args) {
        SpringApplication.run(UserServiceApplication.class, args);
    }
}
```

#### Nacos注解

```java
@SpringBootApplication
@EnableDiscoveryClient  // 通用服务发现注解
public class UserServiceApplication {
    public static void main(String[] args) {
        SpringApplication.run(UserServiceApplication.class, args);
    }
}
```

**注意**: Spring Cloud 2020.x后,`@EnableDiscoveryClient`可以省略。

### 3.4 服务调用对比

#### Eureka + Ribbon

```java
@Configuration
public class RestTemplateConfig {
    
    @Bean
    @LoadBalanced
    public RestTemplate restTemplate() {
        return new RestTemplate();
    }
}

@RestController
public class OrderController {
    
    @Autowired
    private RestTemplate restTemplate;
    
    @GetMapping("/order/{userId}")
    public User getUser(@PathVariable Long userId) {
        // 使用服务名调用
        return restTemplate.getForObject(
            "http://user-service/api/users/" + userId, 
            User.class
        );
    }
}
```

#### Nacos + Ribbon/OpenFeign

```java
// 完全相同的代码!
@Configuration
public class RestTemplateConfig {
    
    @Bean
    @LoadBalanced
    public RestTemplate restTemplate() {
        return new RestTemplate();
    }
}

@RestController
public class OrderController {
    
    @Autowired
    private RestTemplate restTemplate;
    
    @GetMapping("/order/{userId}")
    public User getUser(@PathVariable Long userId) {
        return restTemplate.getForObject(
            "http://user-service/api/users/" + userId, 
            User.class
        );
    }
}
```

**优势**: Nacos完全兼容Ribbon,迁移无需修改业务代码!

## 四、迁移指南

### 4.1 迁移步骤

#### 步骤1: 替换依赖

**删除Eureka依赖**:
```xml
<!-- 删除这段 -->
<dependency>
    <groupId>org.springframework.cloud</groupId>
    <artifactId>spring-cloud-starter-netflix-eureka-client</artifactId>
</dependency>
```

**添加Nacos依赖**:
```xml
<!-- 添加这段 -->
<dependency>
    <groupId>com.alibaba.cloud</groupId>
    <artifactId>spring-cloud-starter-alibaba-nacos-discovery</artifactId>
</dependency>
```

#### 步骤2: 修改配置

**删除Eureka配置**:
```yaml
# 删除这段
eureka:
  client:
    service-url:
      defaultZone: http://localhost:8761/eureka/
```

**添加Nacos配置**:
```yaml
# 添加这段
spring:
  cloud:
    nacos:
      discovery:
        server-addr: 127.0.0.1:8848
```

#### 步骤3: 修改注解

**替换注解**:
```java
// 删除
import org.springframework.cloud.netflix.eureka.EnableEurekaClient;
@EnableEurekaClient

// 替换为
import org.springframework.cloud.client.discovery.EnableDiscoveryClient;
@EnableDiscoveryClient

// 或者直接删除(Spring Cloud 2020.x后可省略)
```

#### 步骤4: 安装Nacos Server

参考上一章:Nacos Server安装

#### 步骤5: 测试验证

1. 启动Nacos Server
2. 启动微服务应用
3. 访问Nacos控制台,确认服务已注册
4. 测试服务调用是否正常

### 4.2 迁移注意事项

#### 注意事项1: 版本兼容性

| Spring Cloud版本 | Spring Cloud Alibaba版本 | Nacos版本 |
|-----------------|------------------------|----------|
| Hoxton | 2.2.x | 1.x |
| 2020.x | 2021.0.x | 2.x |
| 2021.x | 2021.0.x | 2.x |

**推荐组合**:
- Spring Boot 2.7.x
- Spring Cloud 2021.0.8
- Spring Cloud Alibaba 2021.0.5.0
- Nacos 2.2.3

#### 注意事项2: 配置中心迁移

如果使用了Eureka + Spring Cloud Config,需要同时迁移配置中心。

**Eureka + Config**:
```xml
<dependency>
    <groupId>org.springframework.cloud</groupId>
    <artifactId>spring-cloud-config-client</artifactId>
</dependency>
```

```yaml
spring:
  cloud:
    config:
      uri: http://config-server:8888
```

**Nacos Config**:
```xml
<dependency>
    <groupId>com.alibaba.cloud</groupId>
    <artifactId>spring-cloud-starter-alibaba-nacos-config</artifactId>
</dependency>
```

```yaml
spring:
  cloud:
    nacos:
      config:
        server-addr: 127.0.0.1:8848
        file-extension: yaml
```

#### 注意事项3: 健康检查差异

**Eureka健康检查**:
```yaml
eureka:
  client:
    healthcheck:
      enabled: true
```

**Nacos健康检查**:
```yaml
# Nacos默认启用健康检查,无需额外配置
# 如需自定义:
management:
  endpoints:
    web:
      exposure:
        include: health
```

## 五、选型建议

### 5.1 选择Nacos的场景

✅ **强烈推荐Nacos**:

1. **新项目**: 直接使用Nacos
2. **Spring Cloud Alibaba技术栈**: 天然集成
3. **需要配置中心**: Nacos一体化解决方案
4. **高并发场景**: Nacos性能更优
5. **多云部署**: Nacos支持跨数据中心
6. **灰度发布**: Nacos元数据管理强大
7. **国内项目**: 中文文档完善,社区活跃

### 5.2 选择Eureka的场景

⚠️ **仅在以下情况考虑Eureka**:

1. **老项目维护**: 已有Eureka,迁移成本高
2. **团队熟悉Eureka**: 学习成本低
3. **简单场景**: 只需要服务注册发现,不需要配置中心
4. **海外项目**: Netflix生态更成熟

**但要注意**: Eureka 2.x已停止更新,长期来看不推荐。

### 5.3 选择Consul的场景

✅ **考虑Consul**:

1. **多数据中心**: Consul原生支持
2. **非Java技术栈**: Consul提供HTTP API,语言无关
3. **强一致性要求**: Consul CP模式
4. **Service Mesh**: Consul Connect集成

### 5.4 选择Zookeeper的场景

✅ **考虑Zookeeper**:

1. **强一致性要求**: 金融系统等
2. **已有Zookeeper集群**: 复用基础设施
3. **分布式锁**: Zookeeper擅长
4. **大数据生态**: Hadoop/Kafka等已使用

## 七、常见问题

### Q1: Eureka和Nacos可以同时使用吗?

**A**: 可以,但不推荐。会导致:
- ❌ 维护成本高
- ❌ 资源浪费
- ❌ 数据不一致风险

**建议**: 逐步迁移,最终只保留Nacos。

### Q2: 迁移过程中如何保证服务不中断?

**A**: 采用灰度迁移策略:

1. **阶段1**: 新实例注册到Nacos,旧实例仍在Eureka
2. **阶段2**: 消费者同时从Eureka和Nacos获取服务列表
3. **阶段3**: 逐步将旧实例迁移到Nacos
4. **阶段4**: 下线Eureka

### Q3: Nacos的性能瓶颈在哪里?

**A**: 
- **CPU**: 大量服务注册/发现时
- **内存**: 存储大量服务实例信息
- **网络**: 集群间数据同步
- **数据库**: MySQL持久化时的IO

**优化**: 
- 增加Nacos节点数量
- 优化JVM参数
- 使用SSD硬盘
- 定期清理过期数据

### Q4: Nacos安全如何保障?

**A**: 
1. **启用认证**: `nacos.core.auth.enabled=true`
2. **配置RBAC**: 角色权限控制
3. **HTTPS**: 加密通信
4. **防火墙**: 限制访问IP
5. **审计日志**: 记录操作日志
