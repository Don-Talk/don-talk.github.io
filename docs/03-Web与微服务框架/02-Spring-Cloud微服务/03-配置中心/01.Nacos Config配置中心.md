## 一、配置中心概述

### 1.1 为什么需要配置中心?

**传统配置文件的问题**:

```
项目结构:
├── user-service/
│   └── application.yml  ← 配置文件
├── order-service/
│   └── application.yml  ← 配置文件
└── product-service/
    └── application.yml  ← 配置文件

问题:
❌ 配置分散: 每个服务都有自己的配置文件
❌ 修改困难: 改配置需要重启服务
❌ 环境切换: dev/test/prod需要维护多套配置
❌ 权限控制: 谁都可以修改配置
❌ 版本管理: 配置变更没有历史记录
```

**配置中心的优势**:

```
架构:
                    ┌──────────────┐
                    │  Nacos Config │
                    │  (配置中心)    │
                    └──────┬───────┘
                           │
              ┌────────────┼────────────┐
              │            │            │
      ┌───────▼───┐ ┌─────▼────┐ ┌────▼──────┐
      │  User     │ │ Order    │ │ Product   │
      │ Service   │ │ Service  │ │ Service   │
      └───────────┘ └──────────┘ └───────────┘

优势:
✅ 集中管理: 所有配置统一管理
✅ 动态刷新: 修改配置无需重启
✅ 环境隔离: dev/test/prod分开管理
✅ 权限控制: RBAC角色权限
✅ 版本管理: 配置变更历史可追溯
✅ 配置共享: 多个服务共享同一配置
```

## 二、快速入门

### 2.1 添加依赖

**order-service/pom.xml**

```xml
<dependencies>
    <!-- Nacos配置中心 -->
    <dependency>
        <groupId>com.alibaba.cloud</groupId>
        <artifactId>spring-cloud-starter-alibaba-nacos-config</artifactId>
    </dependency>
    
    <!-- Nacos服务发现(可选,如果同时使用) -->
    <dependency>
        <groupId>com.alibaba.cloud</groupId>
        <artifactId>spring-cloud-starter-alibaba-nacos-discovery</artifactId>
    </dependency>
    
    <!-- Spring Boot Actuator(用于刷新配置) -->
    <dependency>
        <groupId>org.springframework.boot</groupId>
        <artifactId>spring-boot-starter-actuator</artifactId>
    </dependency>
</dependencies>
```

### 2.2 配置文件

**注意**: Nacos Config需要使用`bootstrap.yml`(优先级高于`application.yml`)

**order-service/src/main/resources/bootstrap.yml**

```yaml
spring:
  application:
    name: order-service
  
  cloud:
    nacos:
      config:
        server-addr: 127.0.0.1:8848  # Nacos Server地址
        file-extension: yaml          # 配置文件格式(yaml/properties)
        namespace: public             # 命名空间
        group: DEFAULT_GROUP          # 分组
        prefix: order-service         # 配置前缀(默认使用spring.application.name)
        
        # 共享配置
        shared-configs:
          - data-id: common.yaml
            group: DEFAULT_GROUP
            refresh: true  # 是否自动刷新
        
        # 扩展配置
        extension-configs:
          - data-id: redis.yaml
            group: DEFAULT_GROUP
            refresh: true
          - data-id: datasource.yaml
            group: DEFAULT_GROUP
            refresh: false
```

**order-service/src/main/resources/application.yml**

```yaml
server:
  port: 8082

# 暴露刷新端点
management:
  endpoints:
    web:
      exposure:
        include: refresh
```

### 2.4 读取配置

**order-service/src/main/java/com/dontalk/order/config/OrderConfig.java**

```java
package com.dontalk.order.config;

import lombok.Data;
import org.springframework.boot.context.properties.ConfigurationProperties;
import org.springframework.cloud.context.config.annotation.RefreshScope;
import org.springframework.stereotype.Component;

@Data
@Component
@RefreshScope  // 支持动态刷新
@ConfigurationProperties(prefix = "order")
public class OrderConfig {
    
    private Integer timeout;
    private Integer maxRetry;
}
```

**关键点**: 
- `@RefreshScope`: 使Bean支持动态刷新
- `@ConfigurationProperties`: 绑定配置属性

**order-service/src/main/java/com/dontalk/order/controller/OrderController.java**

```java
package com.dontalk.order.controller;

import com.dontalk.order.config.OrderConfig;
import lombok.RequiredArgsConstructor;
import lombok.extern.slf4j.Slf4j;
import org.springframework.beans.factory.annotation.Value;
import org.springframework.web.bind.annotation.*;

import java.util.HashMap;
import java.util.Map;

@Slf4j
@RestController
@RequestMapping("/api/orders")
@RequiredArgsConstructor
public class OrderController {
    
    private final OrderConfig orderConfig;
    
    @Value("${redis.host}")
    private String redisHost;
    
    @Value("${redis.port}")
    private Integer redisPort;
    
    /**
     * 获取配置信息
     */
    @GetMapping("/config")
    public Map<String, Object> getConfig() {
        Map<String, Object> config = new HashMap<>();
        config.put("order.timeout", orderConfig.getTimeout());
        config.put("order.maxRetry", orderConfig.getMaxRetry());
        config.put("redis.host", redisHost);
        config.put("redis.port", redisPort);
        
        return config;
    }
    
    /**
     * 测试订单超时配置
     */
    @PostMapping
    public Map<String, Object> createOrder(@RequestBody Map<String, Object> params) {
        log.info("订单超时配置: {}ms", orderConfig.getTimeout());
        log.info("订单最大重试次数: {}", orderConfig.getMaxRetry());
        
        // 业务逻辑...
        
        Map<String, Object> result = new HashMap<>();
        result.put("code", 200);
        result.put("message", "订单创建成功");
        return result;
    }
}
```

## 三、配置加载规则

### 3.1 Data ID命名规则

**默认规则**:
```
${prefix}-${spring.profiles.active}.${file-extension}
```

**示例**:
- `order-service.yaml` (无profile)
- `order-service-dev.yaml` (dev环境)
- `order-service-test.yaml` (test环境)
- `order-service-prod.yaml` (prod环境)

### 3.2 配置加载优先级

**从高到低**:

```
1. profile配置(order-service-dev.yaml)
   ↓
2. 默认配置(order-service.yaml)
   ↓
3. 共享配置(shared-configs)
   ↓
4. 扩展配置(extension-configs)
   ↓
5. 本地配置(application.yml)
```

**示例**:

```yaml
# bootstrap.yml
spring:
  cloud:
    nacos:
      config:
        # 1. 加载order-service-dev.yaml(最高优先级)
        # 2. 加载order-service.yaml
        # 3. 加载common.yaml(共享配置)
        shared-configs:
          - data-id: common.yaml
            refresh: true
        
        # 4. 加载redis.yaml、datasource.yaml(扩展配置)
        extension-configs:
          - data-id: redis.yaml
            refresh: true
          - data-id: datasource.yaml
            refresh: false
```

### 3.3 Profile配置

**激活Profile**:

```yaml
# bootstrap.yml
spring:
  profiles:
    active: dev  # 激活dev环境
```

**创建不同环境的配置**:

在Nacos控制台创建:
- `order-service-dev.yaml`
- `order-service-test.yaml`
- `order-service-prod.yaml`

**效果**: 启动时自动加载对应环境的配置。

## 四、配置共享

### 4.1 共享配置场景

**场景**: 多个服务共用相同的配置(如Redis、MySQL)。

**传统方式**: 每个服务都配置一遍,重复且难以维护。

**共享配置方式**: 配置一次,所有服务共享。

### 4.2 配置共享实现

#### 步骤1: 创建共享配置

在Nacos控制台创建`common.yaml`:

```yaml
# 公共配置
redis:
  host: localhost
  port: 6379
  timeout: 3000
  
mysql:
  host: localhost
  port: 3306
  database: mydb
  username: root
  password: root123
  
logging:
  level:
    root: INFO
```

#### 步骤2: 引用共享配置

**bootstrap.yml**

```yaml
spring:
  cloud:
    nacos:
      config:
        shared-configs:
          - data-id: common.yaml
            group: DEFAULT_GROUP
            refresh: true  # 支持动态刷新
```

#### 步骤3: 使用共享配置

```java
@Value("${redis.host}")
private String redisHost;

@Value("${mysql.host}")
private String mysqlHost;
```

**效果**: 所有引用`common.yaml`的服务都能读取到这些配置。

### 4.3 配置隔离

**场景**: 不同环境(dev/test/prod)使用不同的配置。

**方式1: 使用Profile**

```yaml
# bootstrap.yml
spring:
  profiles:
    active: ${spring.profiles.active:dev}  # 默认dev环境
```

**启动时指定环境**:

```bash
# 开发环境
java -jar order-service.jar --spring.profiles.active=dev

# 测试环境
java -jar order-service.jar --spring.profiles.active=test

# 生产环境
java -jar order-service.jar --spring.profiles.active=prod
```

**方式2: 使用命名空间**

在Nacos控制台创建命名空间:
- `dev`: 开发环境
- `test`: 测试环境
- `prod`: 生产环境

**配置**:

```yaml
spring:
  cloud:
    nacos:
      config:
        namespace: dev  # 指定命名空间ID
```

**效果**: 不同命名空间的配置完全隔离,互不可见。

## 五、配置加密

### 5.1 为什么需要加密?

**问题**: 数据库密码、API密钥等敏感信息明文存储在配置中心,存在安全风险。

**解决方案**: 使用Jasypt加密敏感配置。

### 5.2 Jasypt加密配置

#### 步骤1: 添加依赖

```xml
<dependency>
    <groupId>com.github.ulisesbocchio</groupId>
    <artifactId>jasypt-spring-boot-starter</artifactId>
    <version>3.0.5</version>
</dependency>
```

#### 步骤2: 配置加密密钥

**bootstrap.yml**

```yaml
jasypt:
  encryptor:
    password: mySecretKey123  # 加密密钥(建议从环境变量读取)
    algorithm: PBEWithMD5AndDES
```

**安全提示**: 不要将密钥硬编码在代码中,应该从环境变量或KMS获取。

#### 步骤3: 加密敏感配置

**生成密文**:

```java
import org.jasypt.encryptor.StringEncryptor;

@Autowired
private StringEncryptor encryptor;

@Test
public void testEncrypt() {
    String password = "root123";
    String encrypted = encryptor.encrypt(password);
    System.out.println("加密后: " + encrypted);
    // 输出: ENC(xxx...)
}
```

或使用命令行工具:

```bash
java -cp jasypt-1.9.4.jar org.jasypt.intf.cli.JasyptPBEStringEncryptionCLI \
  input="root123" \
  password=mySecretKey123 \
  algorithm=PBEWithMD5AndDES
```

#### 步骤4: 使用密文配置

**Nacos配置**:

```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/mydb
    username: root
    password: ENC(WZ5+VJQzXxXxXxXxXxXxXx==)  # 加密后的密码
```

**自动解密**: Jasypt会自动解密`ENC(...)`包裹的密文。

**场景**: 配置变更时执行自定义逻辑。

```java
@Component
@Slf4j
public class ConfigChangeListener {
    
    @EventListener
    public void onConfigChange(RefreshScopeRefreshedEvent event) {

        // 例如: 重新初始化连接池、刷新缓存等
```

## 七、最佳实践

### 7.1 配置设计原则

1. **单一职责**: 一个配置文件只负责一个领域
2. **合理拆分**: 按功能拆分为多个配置文件
3. **命名规范**: 使用清晰的命名(`service-name-env.yaml`)
4. **版本管理**: 重要配置变更记录变更原因
5. **权限控制**: 生产环境配置限制修改权限

### 7.2 配置分类

```
配置类型:
├── 应用配置(application.yml)
│   ├── 端口
│   ├── 上下文路径
│   └── ...
├── 基础设施配置(common.yaml)
│   ├── Redis
│   ├── MySQL
│   ├── RabbitMQ
│   └── ...
├── 业务配置(service-name.yaml)
│   ├── 业务参数
│   ├── 开关配置
│   └── ...
└── 环境配置(service-name-env.yaml)
    ├── dev
    ├── test
    └── prod
```

### 7.3 性能优化

1. **减少配置大小**: 避免在配置中存储大量数据
2. **合理设置刷新**: 不是所有配置都需要动态刷新
3. **使用本地缓存**: 频繁读取的配置缓存到本地
4. **长轮询优化**: 调整长轮询间隔

```yaml
spring:
  cloud:
    nacos:
      config:
        # 长轮询间隔(毫秒),默认30秒
        config-long-poll-timeout: 30000
        # 重试时间(毫秒),默认2秒
        config-retry-time: 2000
```

## 八、常见问题

### Q1: 配置不刷新怎么办?

**A**: 
1. 检查是否添加`@RefreshScope`注解
2. 检查是否暴露`refresh`端点
3. 查看日志是否有错误
4. 手动触发刷新: `POST http://localhost:8082/actuator/refresh`

### Q2: 如何回滚配置?

**A**: 
1. 登录Nacos控制台
2. 进入"配置管理" → "配置列表"
3. 找到配置,点击"历史版本"
4. 选择要回滚的版本,点击"回滚"

### Q3: 配置中心挂了怎么办?

**A**: 
1. 使用集群模式部署Nacos
2. 客户端会缓存配置,短时间可用
3. 配置本地兜底配置文件

```yaml
spring:
  cloud:
    nacos:
      config:
        # 启用本地缓存
        enable-remote-sync-config: true
```

### Q4: 如何保证配置安全性?

**A**: 
1. 启用Nacos认证:`nacos.core.auth.enabled=true`
2. 配置RBAC权限
3. 敏感配置使用Jasypt加密
4. HTTPS通信
5. 审计日志记录
