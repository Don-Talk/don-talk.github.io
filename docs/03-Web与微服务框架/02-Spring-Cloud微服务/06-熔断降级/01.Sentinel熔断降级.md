## 一、熔断降级概述

### 1.1 为什么需要熔断降级?

**雪崩效应**:
```
用户请求 → API网关
           ↓
       order-service (正常)
           ↓ 调用
       user-service (响应慢)
           ↓ 调用
       pay-service (宕机!) ❌
           
结果:
❌ pay-service宕机
❌ user-service等待超时,线程池耗尽
❌ order-service等待超时,线程池耗尽
❌ 整个系统瘫痪!
```

TODO: [此处需要添加雪崩效应示意图]

**解决方案**:
```
用户请求 → API网关
           ↓
       order-service (正常)
           ↓ 调用
       user-service (正常)
           ↓ 调用 [熔断器打开]
       pay-service (宕机!) ❌
           ↓ 快速失败
       返回默认值/错误提示 ✅
           
结果:
✅ pay-service故障被隔离
✅ user-service不受影响
✅ order-service返回降级数据
✅ 系统整体可用
```

### 1.2 核心概念

#### 熔断(Circuit Breaker)

```
状态机:
┌──────────┐
│  关闭    │ ← 初始状态,请求正常通过
└────┬─────┘
     │ 错误率达到阈值
     ↓
┌──────────┐
│  打开    │ ← 拒绝所有请求,快速失败
└────┬─────┘
     │ 等待一段时间
     ↓
┌──────────┐
│  半开    │ ← 允许少量请求试探
└────┬─────┘
     │ 成功 → 关闭
     │ 失败 → 打开
```

#### 降级(Fallback)

```
正常情况:
客户端 → 服务A → 服务B → 返回真实数据

降级情况:
客户端 → 服务A → [服务B故障] → 返回默认数据/缓存数据
```

#### 限流(Rate Limiting)

```
请求流量:
████████████████████████████████ (1000 QPS)
            ↓ 限流阈值: 500 QPS
允许通过:
████████████ (500 QPS)
拒绝请求:
            ████████████████████ (500 QPS)
```

### 1.3 Hystrix vs Sentinel

| 特性 | Hystrix | Sentinel |
|------|---------|----------|
| **维护状态** | 停更 | 活跃 ⭐ |
| **流控规则** | 简单 | 丰富 |
| **实时监控** | 弱 | 强 |
| **动态配置** | 不支持 | 支持 |
| **集群限流** | 不支持 | 支持 |
| **热点参数** | 不支持 | 支持 |
| **推荐程度** | 不推荐 | ⭐⭐⭐⭐⭐ |

## 二、Sentinel快速开始

### 2.1 安装Sentinel Dashboard

#### Docker安装

```bash
# 拉取镜像
docker pull bladex/sentinel-dashboard:latest

# 启动容器
docker run -d \
  --name sentinel-dashboard \
  -p 8858:8858 \
  -e JAVA_OPTS="-Dserver.port=8858" \
  bladex/sentinel-dashboard:latest

# 访问控制台
# http://localhost:8858
# 用户名/密码: sentinel/sentinel
```

#### 源码安装

```bash
# 克隆代码
git clone https://github.com/alibaba/Sentinel.git

# 编译
cd Sentinel
mvn clean package -DskipTests

# 启动
java -jar sentinel-dashboard/target/sentinel-dashboard.jar

# 访问
# http://localhost:8858
```

### 2.2 集成Spring Cloud

#### 步骤1: 添加依赖

```xml
<dependency>
    <groupId>com.alibaba.cloud</groupId>
    <artifactId>spring-cloud-starter-alibaba-sentinel</artifactId>
</dependency>

<!-- 可选: 持久化规则到Nacos -->
<dependency>
    <groupId>com.alibaba.csp</groupId>
    <artifactId>sentinel-datasource-nacos</artifactId>
</dependency>
```

#### 步骤2: 配置连接

```yaml
# application.yml
spring:
  cloud:
    sentinel:
      transport:
        dashboard: localhost:8858  # Dashboard地址
        port: 8719  # 本地端口
      datasource:
        ds1:
          nacos:
            server-addr: localhost:8848
            data-id: ${spring.application.name}-sentinel-rules
            group-id: DEFAULT_GROUP
            rule-type: flow
```

#### 步骤3: 启用注解支持

```java
@Configuration
public class SentinelConfig {
    
    @Bean
    public SentinelResourceAspect sentinelResourceAspect() {
        return new SentinelResourceAspect();
    }
}
```

## 三、流控规则

### 3.1 基本概念

**流控规则要素**:
```
资源名: 保护的目标(方法/API接口)
阈值类型: QPS / 线程数
单机阈值: 每秒允许的请求数
流控模式: 直接 / 关联 / 链路
流控效果: 快速失败 / Warm Up / 排队等待
```

### 3.2 QPS流控

#### 快速失败

```java
@RestController
public class OrderController {
    
    @GetMapping("/order/create")
    @SentinelResource(value = "createOrder", blockHandler = "handleBlock")
    public Result createOrder(@RequestBody OrderDTO orderDTO) {
        // 业务逻辑
        return Result.success();
    }
    
    // 流控处理方法
    public Result handleBlock(OrderDTO orderDTO, BlockException ex) {
        return Result.error("请求过于频繁,请稍后重试");
    }
}
```

**Dashboard配置**:
```
资源名: createOrder
阈值类型: QPS
单机阈值: 100
流控模式: 直接
流控效果: 快速失败

含义: 每秒最多100个请求,超过直接拒绝
```

#### Warm Up (预热)

```java
@GetMapping("/product/detail/{id}")
@SentinelResource(value = "productDetail", blockHandler = "handleBlock")
public Result productDetail(@PathVariable Long id) {
    Product product = productService.getById(id);
    return Result.success(product);
}

public Result handleBlock(Long id, BlockException ex) {
    return Result.error("系统繁忙,请稍后重试");
}
```

**Dashboard配置**:
```
资源名: productDetail
阈值类型: QPS
单机阈值: 100
预热时长: 10秒
流控模式: 直接
流控效果: Warm Up

含义: 
- 初始阈值: 100/3 ≈ 33 QPS
- 10秒后逐渐增加到100 QPS
- 避免冷启动时流量突增
```

#### 排队等待

```java
@PostMapping("/order/pay")
@SentinelResource(value = "orderPay", blockHandler = "handleBlock")
public Result payOrder(@RequestBody PayDTO payDTO) {
    // 支付逻辑
    return Result.success();
}

public Result handleBlock(PayDTO payDTO, BlockException ex) {
    return Result.error("系统繁忙");
}
```

**Dashboard配置**:
```
资源名: orderPay
阈值类型: QPS
单机阈值: 50
超时时间: 3000ms
流控模式: 直接
流控效果: 排队等待

含义: 
- 每秒最多50个请求
- 超出的请求排队等待,最多等3秒
- 适合秒杀场景
```

### 3.3 线程数流控

```java
@GetMapping("/export/excel")
@SentinelResource(value = "exportExcel", blockHandler = "handleBlock")
public Result exportExcel() {
    // 导出Excel,耗时操作
    excelService.export();
    return Result.success();
}

public Result handleBlock(BlockException ex) {
    return Result.error("系统繁忙,导出任务较多");
}
```

**Dashboard配置**:
```
资源名: exportExcel
阈值类型: 线程数
单机阈值: 10

含义: 最多10个线程同时执行导出,超过直接拒绝
```

### 3.4 关联流控

```java
// 写操作
@PostMapping("/order/create")
@SentinelResource(value = "createOrder", blockHandler = "handleBlock")
public Result createOrder() {
    return Result.success();
}

// 读操作
@GetMapping("/order/list")
@SentinelResource(value = "listOrder", blockHandler = "handleBlockList")
public Result listOrder() {
    return Result.success();
}
```

**Dashboard配置**:
```
资源名: listOrder
阈值类型: QPS
单机阈值: 100
流控模式: 关联
关联资源: createOrder

含义: 当createOrder的QPS超过100时,限制listOrder
应用场景: 写多读少,优先保证写操作
```

### 3.5 链路流控

```java
@Service
public class OrderService {
    
    @SentinelResource(value = "queryDB")
    public Order queryFromDB(Long orderId) {
        // 查询数据库
        return orderMapper.selectById(orderId);
    }
    
    @SentinelResource(value = "queryCache")
    public Order queryFromCache(Long orderId) {
        // 查询缓存
        return cacheService.get(orderId);
    }
}
```

**Dashboard配置**:
```
资源名: queryDB
阈值类型: QPS
单机阈值: 50
流控模式: 链路
入口资源: OrderController.queryOrder

含义: 只限制从OrderController调用queryDB的流量
```

## 四、降级规则

### 4.1 RT(平均响应时间)

```java
@GetMapping("/user/info/{id}")
@SentinelResource(value = "userInfo", fallback = "fallbackUserInfo")
public Result userInfo(@PathVariable Long id) {
    User user = userService.getById(id);
    return Result.success(user);
}

public Result fallbackUserInfo(Long id, Throwable ex) {
    // 返回缓存数据或默认数据
    return Result.success(User.getDefaultUser());
}
```

**Dashboard配置**:
```
资源名: userInfo
RT: 100ms
时间窗口: 10s
最小请求数: 5
统计窗口时长: 1s

含义: 
- 1秒内至少5个请求
- 平均RT超过100ms
- 触发降级,10秒内直接调用fallback
```

### 4.2 异常比例

```java
@PostMapping("/order/submit")
@SentinelResource(value = "submitOrder", fallback = "fallbackSubmit")
public Result submitOrder(@RequestBody OrderDTO orderDTO) {
    // 提交订单
    return Result.success();
}

public Result fallbackSubmit(OrderDTO orderDTO, Throwable ex) {
    log.error("订单提交失败", ex);
    return Result.error("系统繁忙,请稍后重试");
}
```

**Dashboard配置**:
```
资源名: submitOrder
异常比例: 0.5 (50%)
时间窗口: 10s
最小请求数: 5

含义: 
- 1秒内至少5个请求
- 异常比例超过50%
- 触发降级,10秒内直接调用fallback
```

### 4.3 异常数

```java
@GetMapping("/payment/callback")
@SentinelResource(value = "payCallback", fallback = "fallbackCallback")
public Result callback(@RequestParam String orderId) {
    // 支付回调处理
    return Result.success();
}

public Result fallbackCallback(String orderId, Throwable ex) {
    log.error("支付回调失败: {}", orderId, ex);
    return Result.success(); // 返回成功,避免重复回调
}
```

**Dashboard配置**:
```
资源名: payCallback
异常数: 10
时间窗口: 60s

含义: 
- 60秒内异常数超过10个
- 触发降级,60秒内直接调用fallback
```

## 五、热点参数限流

### 5.1 基本概念

**普通限流**: 限制整个接口的QPS  
**热点参数限流**: 限制某个参数值的QPS

### 5.2 使用示例

```java
@GetMapping("/product/detail")
@SentinelResource(
    value = "productDetail", 
    blockHandler = "handleBlock",
    paramFlowItem = true  // 启用热点参数限流
)
public Result productDetail(@RequestParam Long productId) {
    Product product = productService.getById(productId);
    return Result.success(product);
}

public Result handleBlock(Long productId, ParamFlowException ex) {
    return Result.error("商品访问过于频繁");
}
```

**Dashboard配置**:
```
资源名: productDetail
参数索引: 0 (第一个参数productId)
参数类型: java.lang.Long
限流模式: QPS
单机阈值: 100

特殊配置:
参数值: 1001 (爆款商品)
阈值: 10

含义: 
- 普通商品: 100 QPS
- 商品1001: 10 QPS (单独限制)
```

## 十一、最佳实践

### 11.1 规则配置建议

| 接口类型 | QPS阈值 | 降级策略 | 说明 |
|----------|---------|----------|------|
| **核心接口** | 1000+ | RT 200ms | 高性能要求 |
| **普通接口** | 100-500 | RT 500ms | 一般要求 |
| **耗时接口** | 10-50 | 线程数 20 | 防止资源耗尽 |
| **外部调用** | 50-100 | 异常比例 50% | 防止第三方故障 |

### 11.2 Fallback设计原则

```java
// ✅ 好的Fallback
public Result fallbackUserInfo(Long id, Throwable ex) {
    // 1. 返回缓存数据
    User cachedUser = cacheService.get(id);
    if (cachedUser != null) {
        return Result.success(cachedUser);
    }
    
    // 2. 返回默认数据
    return Result.success(User.getDefaultUser());
}

// ❌ 坏的Fallback
public Result fallbackUserInfo(Long id, Throwable ex) {
    // 只是简单返回错误,没有降级
    return Result.error("系统错误");
}
```

### 11.3 日志记录

```java
@Component
public class SentinelExceptionHandler {
    
    private static final Logger log = LoggerFactory.getLogger(SentinelExceptionHandler.class);
    
    @ExceptionHandler(BlockException.class)
    public Result handleBlockException(BlockException ex) {
        log.warn("触发限流: {}", ex.getRuleLimitApp());
        return Result.error("请求过于频繁");
    }
    
    @ExceptionHandler(DegradeException.class) {
        public Result handleDegradeException(DegradeException ex) {
            log.warn("触发降级: {}", ex.getRule().getResource());
            return Result.error("系统繁忙");
        }
    }
}
```

## 十二、常见问题

### 12.1 规则不生效

**问题**: 配置了规则但没有生效

**解决**:
```java
// ❌ 忘记添加@SentinelResource
@GetMapping("/order/create")
public Result createOrder() {
    return Result.success();
}

// ✅ 添加@SentinelResource
@GetMapping("/order/create")
@SentinelResource(value = "createOrder", blockHandler = "handleBlock")
public Result createOrder() {
    return Result.success();
}
```

### 12.2 Dashboard看不到应用

**问题**: 启动应用后,Dashboard中看不到

**解决**:
```yaml
# 检查配置
spring:
  cloud:
    sentinel:
      transport:
        dashboard: localhost:8858  # 地址是否正确
        port: 8719  # 端口是否被占用
      eager: true  # 立即注册
```

### 12.3 重启后规则丢失

**问题**: 应用重启后,配置的规则全部丢失

**解决**: 使用Nacos持久化规则
```yaml
spring:
  cloud:
    sentinel:
      datasource:
        flow:
          nacos:
            server-addr: localhost:8848
            data-id: ${spring.application.name}-flow-rules
```

✅ **熔断降级**: 防止雪崩效应

✅ **集群限流**: Token Server

    ↓
快速开始 (Dashboard + 集成)
    ↓
流控规则 (QPS/线程数)
