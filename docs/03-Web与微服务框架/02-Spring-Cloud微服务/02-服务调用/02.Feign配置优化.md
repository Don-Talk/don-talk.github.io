## 一、Feign工作原理

### 1.1 核心组件

```
┌─────────────────────────────────────────────────────┐
│              OpenFeign 工作流程                      │
├─────────────────────────────────────────────────────┤
│                                                     │
│  1. @FeignClient注解扫描                             │
│     ↓                                               │
│  2. 生成动态代理对象(JDK Proxy)                      │
│     ↓                                               │
│  3. 方法调用 → RequestTemplate                       │
│     ↓                                               │
│  4. RequestInterceptor拦截                           │
│     ↓                                               │
│  5. Encoder编码(Java Object → HTTP Request)         │
│     ↓                                               │
│  6. Client发送请求(HttpClient/OkHttp)               │
│     ↓                                               │
│  7. Decoder解码(HTTP Response → Java Object)        │
│     ↓                                               │
│  8. ErrorDecoder处理错误                             │
│     ↓                                               │
│  9. 返回结果                                         │
│                                                     │
└─────────────────────────────────────────────────────┘
```

### 1.2 动态代理

**Feign接口**:
```java
@FeignClient(name = "user-service")
public interface UserFeignClient {
    @GetMapping("/api/users/{id}")
    UserDTO getUserById(@PathVariable("id") Long id);
}
```

**生成的代理类**(简化版):
```java
public class UserFeignClientProxy implements UserFeignClient {
    
    private final Client client;
    private final Encoder encoder;
    private final Decoder decoder;
    
    @Override
    public UserDTO getUserById(Long id) {
        // 1. 构建RequestTemplate
        RequestTemplate template = new RequestTemplate();
        template.method("GET");
        template.uri("/api/users/" + id);
        
        // 2. 应用拦截器
        for (RequestInterceptor interceptor : interceptors) {
            interceptor.apply(template);
        }
        
        // 3. 发送请求
        Request request = template.request();
        Response response = client.execute(request);
        
        // 4. 解码响应
        return decoder.decode(response, UserDTO.class);
    }
}
```

## 二、自定义Encoder和Decoder

### 2.1 自定义Encoder

**场景**: 需要对请求参数进行特殊处理(如加密、签名)。

#### 实现自定义Encoder

```java
@Component
public class CustomEncoder implements Encoder {
    
    private final Encoder defaultEncoder = new SpringEncoder();
    
    @Override
    public void encode(Object object, Type bodyType, RequestTemplate template) throws EncodeException {
        // 1. 如果是Map类型,添加签名
        if (object instanceof Map) {
            Map<String, Object> map = (Map<String, Object>) object;
            
            // 添加时间戳
            map.put("timestamp", System.currentTimeMillis());
            
            // 添加签名
            String sign = generateSign(map);
            map.put("sign", sign);
        }
        
        // 2. 使用默认Encoder编码
        defaultEncoder.encode(object, bodyType, template);
    }
    
    private String generateSign(Map<String, Object> params) {
        // 生成签名逻辑
        String sortedParams = params.entrySet().stream()
            .sorted(Map.Entry.comparingByKey())
            .map(e -> e.getKey() + "=" + e.getValue())
            .collect(Collectors.joining("&"));
        
        return DigestUtils.md5Hex(sortedParams + "secret_key");
    }
}
```

**配置**:

```java
@Configuration
public class FeignEncoderConfig {
    
    @Bean
    public Encoder feignEncoder() {
        return new CustomEncoder();
    }
}
```

### 2.2 自定义Decoder

**场景**: 需要对响应数据进行统一处理(如解密、验签)。

#### 实现自定义Decoder

```java
@Component
public class CustomDecoder implements Decoder {
    
    private final Decoder defaultDecoder = new SpringDecoder();
    
    @Override
    public Object decode(Response response, Type type) throws IOException, DecodeException {
        // 1. 读取响应体
        byte[] bodyData = Util.toByteArray(response.body().asInputStream());
        
        // 2. 检查响应状态
        if (response.status() >= 400) {
            throw new DecodeException(response.status(), "HTTP错误", response.request());
        }
        
        // 3. 验证签名
        String sign = response.headers().get("X-Sign").iterator().next();
        if (!verifySign(bodyData, sign)) {
            throw new DecodeException(response.status(), "签名验证失败", response.request());
        }
        
        // 4. 使用默认Decoder解码
        Response newResponse = response.toBuilder().body(bodyData).build();
        return defaultDecoder.decode(newResponse, type);
    }
    
    private boolean verifySign(byte[] data, String sign) {
        // 验证签名逻辑
        String expectedSign = DigestUtils.md5Hex(new String(data) + "secret_key");
        return expectedSign.equals(sign);
    }
}
```

**配置**:

```java
@Configuration
public class FeignDecoderConfig {
    
    @Bean
    public Decoder feignDecoder() {
        return new CustomDecoder();
    }
}
```

## 三、Contract配置

### 3.1 什么是Contract?

**Contract**(契约)负责解析Feign接口上的注解,生成`MethodMetadata`。

**默认Contract**: `SpringMvcContract`,支持Spring MVC注解。

### 3.2 自定义Contract

**场景**: 需要使用非Spring MVC注解(如JAX-RS)。

#### 使用JAX-RS注解

**添加依赖**:

```xml
<dependency>
    <groupId>io.github.openfeign</groupId>
    <artifactId>feign-jaxrs</artifactId>
</dependency>
```

**定义接口**:

```java
@FeignClient(name = "user-service", configuration = JaxRsConfig.class)
public interface UserFeignClient {
    
    @GET
    @Path("/api/users/{id}")
    @Produces(MediaType.APPLICATION_JSON)
    UserDTO getUserById(@PathParam("id") Long id);
}
```

**配置**:

```java
@Configuration
public class JaxRsConfig {
    
    @Bean
    public Contract feignContract() {
        return new JAXRSContract();
    }
}
```

## 四、性能调优

### 4.1 连接池调优

#### Apache HttpClient调优

```yaml
feign:
  httpclient:
    enabled: true
    max-connections: 200           # 最大连接数
    max-connections-per-route: 50  # 每个路由最大连接数
    time-to-live: 900              # 连接存活时间(秒)
    connection-timer-repeat: 3000  # 连接清理间隔(毫秒)
```

**参数说明**:
- `max-connections`: 根据QPS和平均响应时间计算
  - 公式: `maxConnections = QPS * avgResponseTime / 1000 * 1.5`
  - 例如: QPS=1000, avgResponseTime=50ms → maxConnections = 75
- `max-connections-per-route`: 单个服务的最大连接数
- `time-to-live`: 连接最大存活时间,避免长连接失效

#### OkHttp调优

```yaml
feign:
  okhttp:
    enabled: true

okhttp:
  connection-pool:
    max-idle-connections: 50      # 最大空闲连接数
    keep-alive-duration: 300      # Keep-Alive时间(秒)
```

### 4.2 超时时间调优

**原则**: 
- 连接超时: 短(1-5秒),网络问题快速失败
- 读取超时: 根据业务场景(5-30秒)

**推荐配置**:

```yaml
feign:
  client:
    config:
      default:
        connect-timeout: 3000   # 连接超时3秒
        read-timeout: 10000     # 读取超时10秒
      
      # 查询服务(快速响应)
      user-service:
        connect-timeout: 2000
        read-timeout: 5000
      
      # 报表服务(慢查询)
      report-service:
        connect-timeout: 5000
        read-timeout: 30000
      
      # 文件上传服务
      file-service:
        connect-timeout: 5000
        read-timeout: 60000
```

### 4.3 线程池调优

**场景**: 高并发场景,Feign默认使用主线程,可能阻塞。

**配置Hystrix线程池**(如果集成了Hystrix):

```yaml
hystrix:
  threadpool:
    default:
      coreSize: 10              # 核心线程数
      maximumSize: 50           # 最大线程数
      maxQueueSize: 100         # 队列大小
      queueSizeRejectionThreshold: 50  # 队列拒绝阈值
```

**或使用Sentinel**(推荐):

```yaml
spring:
  cloud:
    sentinel:
      datasource:
        ds1:
          file:
            file: classpath:flow-rule.json
            rule-type: flow
```

### 4.4 缓存调优

#### 一级缓存(本地缓存)

```java
@Service
@RequiredArgsConstructor
public class UserService {
    
    private final UserFeignClient userFeignClient;
    
    // Guava Cache
    private final Cache<Long, UserDTO> cache = CacheBuilder.newBuilder()
        .maximumSize(1000)
        .expireAfterWrite(60, TimeUnit.SECONDS)
        .build();
    
    public UserDTO getUserById(Long id) {
        // 先从缓存获取
        UserDTO user = cache.getIfPresent(id);
        if (user != null) {
            return user;
        }
        
        // 缓存未命中,调用Feign
        user = userFeignClient.getUserById(id);
        
        // 放入缓存
        cache.put(id, user);
        
        return user;
    }
}
```

#### 二级缓存(Redis)

```java
@Service
@RequiredArgsConstructor
public class UserService {
    
    private final UserFeignClient userFeignClient;
    private final RedisTemplate<String, Object> redisTemplate;
    
    public UserDTO getUserById(Long id) {
        String key = "user:" + id;
        
        // 1. 从Redis获取
        UserDTO user = (UserDTO) redisTemplate.opsForValue().get(key);
        if (user != null) {
            return user;
        }
        
        // 2. Redis未命中,调用Feign
        user = userFeignClient.getUserById(id);
        
        // 3. 放入Redis(过期时间60秒)
        redisTemplate.opsForValue().set(key, user, 60, TimeUnit.SECONDS);
        
        return user;
    }
}
```

**效果**: 
- 缓存命中率: 90%+
- 响应时间: 从50ms降低到5ms
- 下游服务压力: 降低90%

## 六、常见问题

### Q1: Feign如何传递Header?

**A**: 

**方式1: 请求拦截器**
```java
@Component
public class HeaderInterceptor implements RequestInterceptor {
    @Override
    public void apply(RequestTemplate template) {
        template.header("Authorization", "Bearer token123");
    }
}
```

**方式2: 注解方式**
```java
@GetMapping("/api/users/{id}")
UserDTO getUserById(@PathVariable("id") Long id, 
                    @RequestHeader("Authorization") String token);
```

### Q2: Feign如何处理大文件上传?

**A**: 不推荐用Feign上传大文件,建议使用:
1. 直接HTTP客户端(RestTemplate/HttpClient)
2. 分片上传
3. 使用对象存储(OSS/S3)预签名URL

### Q3: Feign调用失败如何降级?

**A**: 集成Sentinel或Hystrix(下一章详细讲解)

```java
@FeignClient(name = "user-service", fallback = UserFeignFallback.class)
public interface UserFeignClient {
    @GetMapping("/api/users/{id}")
    UserDTO getUserById(@PathVariable("id") Long id);
}

@Component
public class UserFeignFallback implements UserFeignClient {
    @Override
    public UserDTO getUserById(Long id) {
        // 降级逻辑: 返回默认值或缓存数据
        return new UserDTO();
    }
}
```

### Q4: Feign性能瓶颈在哪里?

**A**: 
1. **序列化/反序列化**: 使用更快的JSON库(Jackson → Fastjson)
2. **网络连接**: 启用连接池
3. **DNS解析**: 使用IP直连或DNS缓存
4. **GC**: 减少对象创建,复用RequestTemplate
