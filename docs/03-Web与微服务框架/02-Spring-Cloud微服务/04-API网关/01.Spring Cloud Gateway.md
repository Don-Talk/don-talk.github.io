## 一、API网关概述

### 1.1 为什么需要API网关?

**没有网关的架构**:

```
客户端 → user-service:8081
客户端 → order-service:8082
客户端 → product-service:8083
客户端 → pay-service:8084

问题:
❌ 客户端需要知道所有服务的地址
❌ 每个服务都要处理认证、限流
❌ 跨域问题难以统一处理
❌ 服务地址变化,客户端要跟着改
❌ 无法统一监控和日志
```

**有网关的架构**:

```
                    ┌──────────────┐
客户端 → 网关:8080  │ API Gateway  │
                    └──────┬───────┘
                           │
              ┌────────────┼────────────┐
              │            │            │
      ┌───────▼───┐ ┌─────▼────┐ ┌────▼──────┐
      │  User     │ │ Order    │ │ Product   │
      │ Service   │ │ Service  │ │ Service   │
      └───────────┘ └──────────┘ └───────────┘

优势:
✅ 统一入口: 客户端只需知道网关地址
✅ 统一认证: 网关层处理JWT验证
✅ 统一限流: 防止下游服务被压垮
✅ 统一跨域: 集中配置CORS
✅ 负载均衡: 自动分发请求
✅ 监控日志: 统一记录访问日志
✅ 灰度发布: 流量控制和路由规则
```

## 二、快速入门

### 2.1 创建网关服务

#### 步骤1: 创建gateway-service模块

**gateway-service/pom.xml**

```xml
<?xml version="1.0" encoding="UTF-8"?>
<project xmlns="http://maven.apache.org/POM/4.0.0"
         xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
         xsi:schemaLocation="http://maven.apache.org/POM/4.0.0 
         http://maven.apache.org/xsd/maven-4.0.0.xsd">
    <parent>
        <artifactId>nacos-demo</artifactId>
        <groupId>com.dontalk.cloud</groupId>
        <version>1.0.0</version>
    </parent>
    <modelVersion>4.0.0</modelVersion>
    
    <artifactId>gateway-service</artifactId>
    
    <dependencies>
        <!-- Spring Cloud Gateway -->
        <dependency>
            <groupId>org.springframework.cloud</groupId>
            <artifactId>spring-cloud-starter-gateway</artifactId>
        </dependency>
        
        <!-- Nacos服务发现 -->
        <dependency>
            <groupId>com.alibaba.cloud</groupId>
            <artifactId>spring-cloud-starter-alibaba-nacos-discovery</artifactId>
        </dependency>
        
        <!-- Lombok -->
        <dependency>
            <groupId>org.projectlombok</groupId>
            <artifactId>lombok</artifactId>
            <optional>true</optional>
        </dependency>
    </dependencies>
</project>
```

**注意**: Gateway基于WebFlux,**不能**引入`spring-boot-starter-web`依赖!

#### 步骤2: 配置文件

**gateway-service/src/main/resources/application.yml**

```yaml
server:
  port: 8080  # 网关端口

spring:
  application:
    name: gateway-service
  
  cloud:
    nacos:
      discovery:
        server-addr: 127.0.0.1:8848
    
    gateway:
      # 路由配置
      routes:
        # 用户服务路由
        - id: user-service
          uri: lb://user-service  # lb=负载均衡
          predicates:
            - Path=/api/users/**  # 路径匹配
          
        # 订单服务路由
        - id: order-service
          uri: lb://order-service
          predicates:
            - Path=/api/orders/**
          
        # 商品服务路由
        - id: product-service
          uri: lb://product-service
          predicates:
            - Path=/api/products/**
      
      # 全局跨域配置
      globalcors:
        cors-configurations:
          '[/**]':
            allowedOrigins: "*"
            allowedMethods:
              - GET
              - POST
              - PUT
              - DELETE
            allowedHeaders: "*"
```

**关键配置说明**:
- `uri: lb://user-service`: `lb`表示使用负载均衡,从Nacos获取服务实例
- `predicates`: 路由断言,决定请求是否匹配该路由
- `Path=/api/users/**`: 路径匹配规则

## 三、路由断言(Predicates)

### 3.1 什么是断言?

**断言**: 判断请求是否匹配某个路由的规则。

**常用断言**:

| 断言 | 说明 | 示例 |
|------|------|------|
| **Path** | 路径匹配 | `- Path=/api/users/**` |
| **Method** | 请求方法 | `- Method=GET,POST` |
| **Header** | 请求头 | `- Header=X-Request-Id, \d+` |
| **Query** | 查询参数 | `- Query=name,张三` |
| **Cookie** | Cookie | `- Cookie=chocolate, ch.p` |
| **Host** | 域名 | `- Host=**.example.com` |
| **RemoteAddr** | IP地址 | `- RemoteAddr=192.168.1.0/24` |
| **After** | 时间之后 | `- After=2024-01-01T00:00:00+08:00` |
| **Before** | 时间之前 | `- Before=2024-12-31T23:59:59+08:00` |
| **Between** | 时间之间 | `- Between=xxx, yyy` |

### 3.2 组合断言

**场景**: 多个条件同时满足才匹配。

```yaml
spring:
  cloud:
    gateway:
      routes:
        - id: user-service
          uri: lb://user-service
          predicates:
            - Path=/api/users/**        # 路径匹配
            - Method=GET,POST           # 方法匹配
            - Header=X-Request-Id, \d+  # 请求头匹配
```

**效果**: 只有同时满足3个条件的请求才会路由到user-service。

### 3.3 自定义断言

**场景**: 根据业务逻辑自定义匹配规则。

#### 实现自定义断言

```java
@Component
public class AgeRoutePredicateFactory extends AbstractRoutePredicateFactory<AgeRoutePredicateFactory.Config> {
    
    public AgeRoutePredicateFactory() {
        super(Config.class);
    }
    
    @Override
    public Predicate<ServerWebExchange> apply(Config config) {
        return exchange -> {
            // 从请求参数获取age
            String age = exchange.getRequest().getQueryParams().getFirst("age");
            
            if (age != null) {
                int ageValue = Integer.parseInt(age);
                return ageValue >= config.getMinAge() && ageValue <= config.getMaxAge();
            }
            
            return false;
        };
    }
    
    @Data
    public static class Config {
        private Integer minAge;
        private Integer maxAge;
    }
}
```

**使用自定义断言**:

```yaml
spring:
  cloud:
    gateway:
      routes:
        - id: user-service
          uri: lb://user-service
          predicates:
            - Path=/api/users/**
            - Age=18, 60  # 年龄在18-60之间
```

## 四、过滤器(Filters)

### 4.1 过滤器类型

| 类型 | 作用域 | 说明 |
|------|--------|------|
| **GatewayFilter** | 单个路由 | 只对特定路由生效 |
| **GlobalFilter** | 全局 | 对所有路由生效 |

### 4.2 内置过滤器

**常用内置过滤器**:

| 过滤器 | 说明 | 示例 |
|--------|------|------|
| **AddRequestHeader** | 添加请求头 | `AddRequestHeader=X-Request-Id, 123` |
| **AddRequestParameter** | 添加请求参数 | `AddRequestParameter=name,张三` |
| **AddResponseHeader** | 添加响应头 | `AddResponseHeader=X-Response-Id, 456` |
| **RewritePath** | 重写路径 | `RewritePath=/api/(?<segment>.*), /$\{segment}` |
| **StripPrefix** | 去除前缀 | `StripPrefix=1` (/api/users → /users) |
| **PrefixPath** | 添加前缀 | `PrefixPath=/api` |
| **SetStatus** | 设置状态码 | `SetStatus=404` |
| **RedirectTo** | 重定向 | `RedirectTo=302, https://example.com` |

#### 使用内置过滤器

```yaml
spring:
  cloud:
    gateway:
      routes:
        - id: user-service
          uri: lb://user-service
          predicates:
            - Path=/api/users/**
          filters:
            - StripPrefix=1          # 去除/api前缀
            - AddRequestHeader=X-Gateway, true  # 添加请求头
            - AddResponseHeader=X-Response-Time, #{T(java.lang.System).currentTimeMillis()}  # 添加响应头
```

**效果**:
- 请求: `GET http://localhost:8080/api/users/1`
- 转发: `GET http://user-service/users/1` (去除了/api)
- 请求头: `X-Gateway: true`
- 响应头: `X-Response-Time: 1704067200000`

### 4.3 自定义全局过滤器

**场景**: 统一处理认证、日志、限流等。

#### 实现认证过滤器

```java
@Component
@Slf4j
public class AuthGlobalFilter implements GlobalFilter, Ordered {
    
    @Override
    public Mono<Void> filter(ServerWebExchange exchange, GatewayFilterChain chain) {
        ServerHttpRequest request = exchange.getRequest();
        ServerHttpResponse response = exchange.getResponse();
        
        // 1. 获取Token
        String token = request.getHeaders().getFirst("Authorization");
        
        // 2. 检查Token是否存在
        if (!StringUtils.hasText(token) || !token.startsWith("Bearer ")) {
            log.warn("请求未携带Token: {}", request.getPath());
            
            // 返回401
            response.setStatusCode(HttpStatus.UNAUTHORIZED);
            response.getHeaders().setContentType(MediaType.APPLICATION_JSON);
            
            Map<String, Object> result = new HashMap<>();
            result.put("code", 401);
            result.put("message", "未授权");
            
            DataBuffer buffer = response.bufferFactory().wrap(
                JSON.toJSONString(result).getBytes()
            );
            
            return response.writeWith(Mono.just(buffer));
        }
        
        // 3. 验证Token(简化版,实际应该调用JWT工具类)
        token = token.substring(7);  // 去除"Bearer "
        if (!validateToken(token)) {
            log.warn("Token无效: {}", token);
            
            response.setStatusCode(HttpStatus.UNAUTHORIZED);
            response.getHeaders().setContentType(MediaType.APPLICATION_JSON);
            
            Map<String, Object> result = new HashMap<>();
            result.put("code", 401);
            result.put("message", "Token无效");
            
            DataBuffer buffer = response.bufferFactory().wrap(
                JSON.toJSONString(result).getBytes()
            );
            
            return response.writeWith(Mono.just(buffer));
        }
        
        // 4. Token有效,放行
        log.info("Token验证通过: {}", request.getPath());
        return chain.filter(exchange);
    }
    
    @Override
    public int getOrder() {
        return -1;  // 优先级,值越小越先执行
    }
    
    private boolean validateToken(String token) {
        // TODO: 调用JWT工具类验证Token
        return "valid_token".equals(token);  // 简化版
    }
}
```

**测试**:

**不带Token**:
```
GET http://localhost:8080/api/users/1
```

**响应**:
```json
{
  "code": 401,
  "message": "未授权"
}
```

**带Token**:
```
GET http://localhost:8080/api/users/1
Authorization: Bearer valid_token
```

**响应**: 正常返回用户信息

### 4.4 自定义局部过滤器

**场景**: 对特定路由添加特殊逻辑。

#### 实现日志过滤器

```java
@Component
@Slf4j
public class LoggingGatewayFilterFactory extends AbstractGatewayFilterFactory<LoggingGatewayFilterFactory.Config> {
    
    public LoggingGatewayFilterFactory() {
        super(Config.class);
    }
    
    @Override
    public GatewayFilter apply(Config config) {
        return (exchange, chain) -> {
            long startTime = System.currentTimeMillis();
            
            ServerHttpRequest request = exchange.getRequest();
            log.info("请求开始: {} {}", request.getMethod(), request.getPath());
            
            return chain.filter(exchange).then(Mono.fromRunnable(() -> {
                long endTime = System.currentTimeMillis();
                log.info("请求结束: {} {}, 耗时: {}ms", 
                    request.getMethod(), request.getPath(), endTime - startTime);
            }));
        };
    }
    
    @Data
    public static class Config {
        private Boolean enabled = true;
    }
}
```

**使用**:

```yaml
spring:
  cloud:
    gateway:
      routes:
        - id: user-service
          uri: lb://user-service
          predicates:
            - Path=/api/users/**
          filters:
            - Logging=true  # 启用日志过滤器
```

## 五、限流熔断

### 5.1 基于Redis的限流

**场景**: 防止恶意刷接口,保护下游服务。

#### 添加依赖

```xml
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-data-redis-reactive</artifactId>
</dependency>
```

#### 配置限流

```yaml
spring:
  cloud:
    gateway:
      routes:
        - id: user-service
          uri: lb://user-service
          predicates:
            - Path=/api/users/**
          filters:
            - name: RequestRateLimiter
              args:
                redis-rate-limiter.replenishRate: 10   # 每秒允许10个请求
                redis-rate-limiter.burstCapacity: 20   # 桶容量20
                key-resolver: "#{@ipKeyResolver}"      # 按IP限流
```

#### 配置Key解析器

```java
@Configuration
public class RateLimiterConfig {
    
    /**
     * 按IP限流
     */
    @Bean
    public KeyResolver ipKeyResolver() {
        return exchange -> Mono.just(
            exchange.getRequest().getRemoteAddress().getHostName()
        );
    }
    
    /**
     * 按用户ID限流
     */
    @Bean
    public KeyResolver userKeyResolver() {
        return exchange -> {
            String userId = exchange.getRequest().getQueryParams().getFirst("userId");
            return Mono.just(userId != null ? userId : "anonymous");
        };
    }
}
```

**效果**: 
- 每个IP每秒最多10个请求
- 超过限制返回`429 Too Many Requests`

**JwtUtils.java**:

```java
@Component
public class JwtUtils {
    
    private static final String SECRET_KEY = "your-secret-key-at-least-256-bits-long";
    private static final long EXPIRATION_TIME = 7 * 24 * 60 * 60 * 1000; // 7天
    
    /**
     * 生成Token
     */
    public String generateToken(Long userId, String username) {
        Date now = new Date();
        Date expiryDate = new Date(now.getTime() + EXPIRATION_TIME);
        
        return Jwts.builder()
            .setSubject(String.valueOf(userId))
            .claim("username", username)
            .setIssuedAt(now)
            .setExpiration(expiryDate)
            .signWith(Keys.hmacShaKeyFor(SECRET_KEY.getBytes()), SignatureAlgorithm.HS256)
            .compact();
    }
    
    /**
     * 验证Token
     */
    public Claims validateToken(String token) {
        try {
            return Jwts.parserBuilder()
                .setSigningKey(Keys.hmacShaKeyFor(SECRET_KEY.getBytes()))
                .build()
                .parseClaimsJws(token)
                .getBody();
        } catch (Exception e) {
            return null;
        }
    }
}
```

### 6.2 鉴权过滤器

```java
@Component
@RequiredArgsConstructor
@Slf4j
public class JwtAuthGlobalFilter implements GlobalFilter, Ordered {
    
    private final JwtUtils jwtUtils;
    
    @Override
    public Mono<Void> filter(ServerWebExchange exchange, GatewayFilterChain chain) {
        ServerHttpRequest request = exchange.getRequest();
        String path = request.getPath().toString();
        
        // 1. 白名单路径直接放行
        if (isWhiteList(path)) {
            return chain.filter(exchange);
        }
        
        // 2. 获取Token
        String token = request.getHeaders().getFirst("Authorization");
        if (!StringUtils.hasText(token) || !token.startsWith("Bearer ")) {
            return unauthorized(exchange, "未授权");
        }
        
        token = token.substring(7);
        
        // 3. 验证Token
        Claims claims = jwtUtils.validateToken(token);
        if (claims == null) {
            return unauthorized(exchange, "Token无效或已过期");
        }
        
        // 4. 将用户信息传递给下游服务
        String userId = claims.getSubject();
        String username = claims.get("username", String.class);
        
        ServerHttpRequest mutatedRequest = request.mutate()
            .header("X-User-Id", userId)
            .header("X-Username", username)
            .build();
        
        ServerWebExchange mutatedExchange = exchange.mutate()
            .request(mutatedRequest)
            .build();
        
        log.info("用户鉴权通过: userId={}, username={}", userId, username);
        
        return chain.filter(mutatedExchange);
    }
    
    @Override
    public int getOrder() {
        return -100;
    }
    
    private boolean isWhiteList(String path) {
        return path.startsWith("/api/auth/") || 
               path.startsWith("/api/public/");
    }
    
    private Mono<Void> unauthorized(ServerWebExchange exchange, String message) {
        ServerHttpResponse response = exchange.getResponse();
        response.setStatusCode(HttpStatus.UNAUTHORIZED);
        response.getHeaders().setContentType(MediaType.APPLICATION_JSON);
        
        Map<String, Object> result = new HashMap<>();
        result.put("code", 401);
        result.put("message", message);
        
        DataBuffer buffer = response.bufferFactory().wrap(
            JSON.toJSONString(result).getBytes()
        );
        
        return response.writeWith(Mono.just(buffer));
    }
}
```

## 七、总结

### 7.1 核心要点

1. ✅ **路由配置**: uri、predicates、filters
2. ✅ **路由断言**: Path、Method、Header等
3. ✅ **过滤器**: 全局过滤器、局部过滤器
4. ✅ **限流**: Redis限流、Sentinel限流
5. ✅ **JWT鉴权**: 统一认证、白名单
6. ✅ **跨域**: 全局CORS配置

### 7.2 最佳实践

1. **路由规则**: 清晰的命名和路径规划
2. **过滤器顺序**: 合理设置getOrder()优先级
3. **限流策略**: 根据业务场景设置QPS
4. **异常处理**: 统一的错误响应格式
5. **监控日志**: 记录请求日志和耗时
6. **性能优化**: 避免在过滤器中执行耗时操作
