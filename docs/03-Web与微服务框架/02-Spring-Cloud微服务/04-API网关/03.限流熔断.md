## 一、为什么需要限流?

### 1.1 限流场景

```
场景1: 秒杀活动
- 预期流量: 1000 QPS
- 实际流量: 100000 QPS
- 结果: 系统崩溃 ❌

场景2: 恶意攻击
- 正常用户: 10 QPS
- 攻击者: 5000 QPS
- 结果: 服务不可用 ❌

场景3: 依赖服务故障
- order-service调用user-service
- user-service响应变慢(从50ms到5s)
- 结果: order-service线程池耗尽 ❌
```

### 1.2 限流算法

#### 计数器算法

```java
// 简单计数器
public class CounterRateLimiter {
    
    private AtomicInteger count = new AtomicInteger(0);
    private long startTime = System.currentTimeMillis();
    private final int limit;
    
    public boolean tryAcquire() {
        long now = System.currentTimeMillis();
        
        // 重置计数器(每秒)
        if (now - startTime > 1000) {
            count.set(0);
            startTime = now;
        }
        
        // 检查是否超限
        return count.incrementAndGet() <= limit;
    }
}

缺点:
❌ 临界问题(第59秒999次 + 第60秒999次 = 1998次/秒)
```

#### 滑动窗口算法

```
时间轴:
|--窗口1--|--窗口2--|--窗口3--|
   100次     150次     50次

优势:
✅ 避免临界问题
✅ 更平滑的限流
```

#### 令牌桶算法 (Sentinel使用)

```
桶容量: 100个令牌
生成速率: 10个/秒

请求到来:
- 有令牌 → 获取令牌,通过 ✅
- 无令牌 → 拒绝 ❌

特点:
✅ 允许突发流量(桶中有缓存令牌)
✅ 平滑限流
```

#### 漏桶算法

```
桶容量: 不限
流出速率: 固定10个/秒

请求到来:
- 放入桶中
- 以固定速率处理

特点:
✅ 强制平滑输出
❌ 无法应对突发流量
```

## 二、Gateway集成Sentinel

### 2.1 添加依赖

```xml
<!-- Sentinel核心 -->
<dependency>
    <groupId>com.alibaba.cloud</groupId>
    <artifactId>spring-cloud-starter-alibaba-sentinel</artifactId>
</dependency>

<!-- Gateway适配器 -->
<dependency>
    <groupId>com.alibaba.csp</groupId>
    <artifactId>sentinel-spring-cloud-gateway-adapter</artifactId>
</dependency>
```

### 2.2 配置Sentinel

```yaml
spring:
  cloud:
    sentinel:
      transport:
        dashboard: localhost:8080  # Sentinel Dashboard地址
      datasource:
        ds1:
          nacos:
            server-addr: localhost:8848
            data-id: gateway-flow-rules
            group-id: DEFAULT_GROUP
            rule-type: flow
```

### 2.3 配置类

```java
@Configuration
public class GatewayConfig {
    
    private final List<ViewResolver> viewResolvers;
    private final ServerCodecConfigurer serverCodecConfigurer;
    
    public GatewayConfig(List<ViewResolver> viewResolvers,
                        ServerCodecConfigurer serverCodecConfigurer) {
        this.viewResolvers = viewResolvers;
        this.serverCodecConfigurer = serverCodecConfigurer;
    }
    
    /**
     * Sentinel全局异常处理器
     */
    @Bean
    @Order(Ordered.HIGHEST_PRECEDENCE)
    public SentinelGatewayBlockExceptionHandler sentinelGatewayBlockExceptionHandler() {
        return new SentinelGatewayBlockExceptionHandler(viewResolvers, serverCodecConfigurer);
    }
    
    /**
     * Sentinel全局过滤器
     */
    @Bean
    @Order(-1)
    public GlobalFilter sentinelGatewayFilter() {
        return new SentinelGatewayFilter();
    }
    
    @PostConstruct
    public void doInit() {
        initGatewayRules();
    }
    
    /**
     * 初始化网关限流规则
     */
    private void initGatewayRules() {
        Set<GatewayFlowRule> rules = new HashSet<>();
        
        // 针对路由限流
        rules.add(new GatewayFlowRule("order-service")
            .setCount(100)           // 限流阈值
            .setIntervalSec(1)       // 统计窗口(秒)
        );
        
        // 针对API分组限流
        rules.add(new GatewayFlowRule("api-group")
            .setCount(50)
            .setIntervalSec(1)
            .setResourceMode(GatewayFlowRule.RESOURCE_MODE_CUSTOM_API_NAME)
        );
        
        GatewayRuleManager.loadRules(rules);
    }
}
```

## 三、API级别限流

### 3.1 定义API分组

```java
@Component
public class ApiDefinitionConfig {
    
    @PostConstruct
    public void initApiDefinitions() {
        Set<ApiDefinition> definitions = new HashSet<>();
        
        // 订单相关API
        ApiDefinition orderApi = new ApiDefinition("order-api")
            .setPredicateItems(new HashSet<ApiPredicateItem>() {{
                add(new ApiPathPredicateItem().setPattern("/api/order/**"));
            }});
        
        // 用户相关API
        ApiDefinition userApi = new ApiDefinition("user-api")
            .setPredicateItems(new HashSet<ApiPredicateItem>() {{
                add(new ApiPathPredicateItem().setPattern("/api/user/**"));
            }});
        
        definitions.add(orderApi);
        definitions.add(userApi);
        
        GatewayApiDefinitionManager.loadApiDefinitions(definitions);
    }
}
```

## 四、路由级别限流

### 4.1 基础限流

```java
Set<GatewayFlowRule> rules = new HashSet<>();

// 针对order-service路由限流
rules.add(new GatewayFlowRule("order-service")
    .setCount(100)              // QPS阈值
    .setIntervalSec(1)          // 统计窗口
    .setControlBehavior(RuleConstant.CONTROL_BEHAVIOR_DEFAULT)  // 快速失败
);

GatewayRuleManager.loadRules(rules);
```

### 4.2 限流模式

#### 快速失败 (默认)

```java
rules.add(new GatewayFlowRule("order-service")
    .setCount(100)
    .setControlBehavior(RuleConstant.CONTROL_BEHAVIOR_DEFAULT)
);

效果:
超过100 QPS → 立即返回429 Too Many Requests
```

#### Warm Up (预热)

```java
rules.add(new GatewayFlowRule("order-service")
    .setCount(100)
    .setControlBehavior(RuleConstant.CONTROL_BEHAVIOR_WARM_UP)
    .setWarmUpPeriodSec(10)  // 预热期10秒
);

效果:
初始阈值: 100/3 ≈ 33 QPS
10秒后逐渐增加到100 QPS
避免冷启动时流量突增
```

#### 排队等待

```java
rules.add(new GatewayFlowRule("order-service")
    .setCount(100)
    .setControlBehavior(RuleConstant.CONTROL_BEHAVIOR_RATE_LIMITER)
    .setMaxQueueingTimeoutMs(3000)  // 最多等待3秒
);

效果:
超过100 QPS的请求排队等待
等待超过3秒则拒绝
适合秒杀场景
```

## 五、热点参数限流

### 5.1 配置热点参数

```java
Set<GatewayParamFlowItem> paramItems = new HashSet<>();

// 根据userId限流
GatewayParamFlowItem item = new GatewayParamFlowItem()
    .setParseStrategy(SentinelGatewayConstants.PARAM_PARSE_STRATEGY_URL_PARAM)
    .setFieldName("userId");

paramItems.add(item);

Set<GatewayFlowRule> rules = new HashSet<>();
rules.add(new GatewayFlowRule("order-api")
    .setResourceMode(GatewayFlowRule.RESOURCE_MODE_CUSTOM_API_NAME)
    .setCount(10)  // 每个userId最多10 QPS
    .setIntervalSec(1)
    .setParamItems(paramItems)
);

GatewayRuleManager.loadRules(rules);
```

**效果**:
```
用户A: /api/order/list?userId=1  → 10 QPS限制
用户B: /api/order/list?userId=2  → 10 QPS限制

防止单个用户刷接口!
```

### 5.2 从Header获取参数

## 六、熔断降级

### 6.1 熔断策略

```
熔断器状态机:

关闭(Closed) → 打开(Open) → 半开(Half-Open)
   ↑                                    |
   └────────────────────────────────────┘
```

**状态说明**:
```
关闭: 正常状态,请求通过
打开: 熔断状态,直接拒绝
半开: 试探状态,允许少量请求
```

### 6.2 配置熔断规则

```java
Set<DegradeRule> rules = new HashSet<>();

// 基于响应时间熔断
DegradeRule rtRule = new DegradeRule("order-service")
    .setGrade(RuleConstant.DEGRADE_GRADE_RT)
    .setCount(500)         // 平均RT超过500ms
    .setTimeWindow(10)     // 熔断10秒
    .setMinRequestAmount(5) // 最小请求数
    .setStatIntervalMs(1000); // 统计窗口1秒

rules.add(rtRule);

// 基于异常比例熔断
DegradeRule errorRateRule = new DegradeRule("order-service")
    .setGrade(RuleConstant.DEGRADE_GRADE_EXCEPTION_RATIO)
    .setCount(0.5)         // 异常比例超过50%
    .setTimeWindow(10)     // 熔断10秒
    .setMinRequestAmount(5)
    .setStatIntervalMs(1000);

rules.add(errorRateRule);

DegradeRuleManager.loadRules(rules);
```

### 6.3 熔断降级处理

```java
@Component
public class FallbackController {
    
    /**
     * order-service降级处理
     */

    }
    
    /**
     * user-service降级处理
     */
    @GetMapping("/fallback/user")
    public Result<?> userFallback() {
        // 返回缓存数据

    }
}
```

## 七、自定义限流异常处理

### 7.1 默认异常响应

```json
{
  "code": 429,
  "message": "Blocked by Sentinel: FlowException"
}
```

### 7.2 自定义异常处理器

```java
@Component
public class CustomBlockExceptionHandler implements BlockRequestHandler {
    
    @Override
    public Mono<ServerResponse> handleRequest(ServerWebExchange exchange, Throwable t) {
        Map<String, Object> result = new HashMap<>();
        
        if (t instanceof FlowException) {
            result.put("code", 429);
            result.put("message", "请求过于频繁,请稍后重试");
            result.put("data", null);
            
        } else if (t instanceof DegradeException) {
            result.put("code", 503);
            result.put("message", "服务降级,请稍后重试");
            result.put("data", null);
            
        } else if (t instanceof ParamFlowException) {
            result.put("code", 429);
            result.put("message", "热点参数限流");
            result.put("data", null);
            
        } else {
            result.put("code", 500);
            result.put("message", "系统繁忙");
            result.put("data", null);
        }
        
        return ServerResponse.status(HttpStatus.TOO_MANY_REQUESTS)
            .contentType(MediaType.APPLICATION_JSON)
            .body(BodyInserters.fromValue(result));
    }
}
```

**注册异常处理器**:
```java
@Configuration
public class SentinelConfig {
    
    @PostConstruct
    public void init() {
        GatewayCallbackManager.setBlockHandler(new CustomBlockExceptionHandler());
    }
}
```

**自定义响应**:
```json
{
  "code": 429,
  "message": "请求过于频繁,请稍后重试",
  "data": null
}
```

## 九、最佳实践

### 9.1 限流阈值设置

| API类型 | QPS阈值 | 说明 |
|---------|---------|------|
| **核心接口** | 1000+ | 下单/支付 |
| **普通接口** | 100-500 | 查询列表 |
| **耗时接口** | 10-50 | 导出Excel |
| **公开接口** | 50-100 | 登录/注册 |

### 9.2 分级限流

```
第一层: Nginx限流 (IP级别)
  ↓
第二层: Gateway限流 (API级别)
  ↓
第三层: 服务限流 (方法级别)
  ↓
第四层: 数据库限流 (连接池)
```

### 9.3 动态调整

```java
// 活动期间临时提升限流阈值
@GetMapping("/admin/update-limit")
public Result updateLimit(@RequestParam int limit) {
    Set<GatewayFlowRule> rules = new HashSet<>();
    
    rules.add(new GatewayFlowRule("order-api")
        .setCount(limit)
        .setIntervalSec(1)
    );
    
    GatewayRuleManager.loadRules(rules);
    
    return Result.success("限流阈值已更新为: " + limit);
}
```
