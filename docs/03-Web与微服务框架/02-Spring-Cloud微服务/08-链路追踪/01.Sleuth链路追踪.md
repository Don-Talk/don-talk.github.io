## 一、链路追踪概述

### 1.1 为什么需要链路追踪?

**微服务调用链复杂**:
```
用户请求 → API Gateway
           ↓
       order-service
           ↓
       ├── user-service (验证用户)
       ├── product-service (查询商品)
       ├── inventory-service (扣减库存)
       ├── payment-service (处理支付)
       └── notification-service (发送通知)

问题:
❌ 哪个服务慢了?
❌ 哪个服务报错了?
❌ 请求经过了哪些服务?
❌ 如何优化性能瓶颈?
```

TODO: [此处需要添加链路追踪示意图]

**链路追踪的解决方案**:
```
Trace ID: abc123 (整个请求链路)
    ↓
Span ID: span1 → API Gateway (耗时10ms)
    ↓
Span ID: span2 → order-service (耗时50ms)
    ↓
    ├── Span ID: span3 → user-service (耗时20ms)
    ├── Span ID: span4 → product-service (耗时30ms)
    ├── Span ID: span5 → inventory-service (耗时40ms)
    ├── Span ID: span6 → payment-service (耗时100ms) ← 瓶颈!
    └── Span ID: span7 → notification-service (耗时15ms)

结果:
✅ 清晰看到每个服务的耗时
✅ 快速定位性能瓶颈(payment-service)
✅ 可视化调用链路
```

### 1.2 核心概念

| 概念 | 说明 | 示例 |
|------|------|------|
| **Trace** | 完整的请求链路 | 用户下单的整个过程 |
| **Span** | 链路中的一个节点 | order-service处理 |
| **Trace ID** | 链路唯一标识 | abc123 |
| **Span ID** | 节点唯一标识 | span1, span2 |
| **Parent Span ID** | 父节点标识 | span2是span3的父节点 |
| **Annotation** | 事件标记 | CS/CR/SS/SR |

### 1.3 Zipkin注解

```
CS (Client Send): 客户端发起请求
SR (Server Receive): 服务端收到请求
SS (Server Send): 服务端返回响应
CR (Client Receive): 客户端收到响应

时间线:
CS ────────────────────── CR
    SR ─────────── SS
    
客户端耗时 = CR - CS
服务端耗时 = SS - SR
网络耗时 = (SR - CS) + (CR - SS)
```

## 三、Sleuth工作原理

### 3.1 Trace ID传递

```
用户请求
  ↓
  Header: X-B3-TraceId: abc123
  Header: X-B3-SpanId: span1
  ↓
API Gateway
  ↓
  Header: X-B3-TraceId: abc123 (保持不变)
  Header: X-B3-SpanId: span2 (生成新的Span)
  Header: X-B3-ParentSpanId: span1 (父Span)
  ↓
order-service
  ↓
  Header: X-B3-TraceId: abc123
  Header: X-B3-SpanId: span3
  Header: X-B3-ParentSpanId: span2
  ↓
user-service
```

### 3.2 采样率配置

```yaml
# 开发环境: 100%采样
spring:
  sleuth:
    sampler:
      probability: 1.0

# 测试环境: 50%采样
spring:
  sleuth:
    sampler:
      probability: 0.5

# 生产环境: 10%采样(降低开销)
spring:
  sleuth:
    sampler:
      probability: 0.1
```

## 四、自定义链路数据

### 4.1 添加Tag

```java
@RestController
@Slf4j
public class OrderController {
    
    @Autowired
    private Tracer tracer;
    
    @GetMapping("/order/{id}")
    public Result getOrder(@PathVariable Long id) {
        // 添加自定义Tag
        Span currentSpan = tracer.currentSpan();
        currentSpan.tag("orderId", String.valueOf(id));
        currentSpan.tag("userId", "user123");
        currentSpan.tag("orderAmount", "99.99");
        
        log.info("查询订单: {}", id);
        
        Order order = orderService.getById(id);
        return Result.success(order);
    }
}
```

**Zipkin中显示**:
```
Tags:
  orderId: 1001
  userId: user123
  orderAmount: 99.99
```

### 4.2 添加Event

```java
@GetMapping("/order/create")
public Result createOrder(@RequestBody OrderDTO orderDTO) {
    Span currentSpan = tracer.currentSpan();
    
    // 添加事件标记
    currentSpan.event("order-validation-started");
    
    // 验证订单
    validateOrder(orderDTO);
    
    currentSpan.event("order-validation-completed");
    currentSpan.event("order-saved");
    
    Order order = orderService.save(orderDTO);
    
    currentSpan.event("order-created");
    
    return Result.success(order);
}
```

**Zipkin中显示**:
```
Events:
  10:00:00.100 - order-validation-started
  10:00:00.150 - order-validation-completed
  10:00:00.200 - order-saved
  10:00:00.250 - order-created
```

### 4.3 创建子Span

```java
@Service
@Slf4j
public class OrderService {
    
    @Autowired
    private Tracer tracer;
    
    public Order createOrder(OrderDTO orderDTO) {
        // 创建子Span
        Span childSpan = tracer.nextSpan().name("validateOrder").start();
        
        try (Tracer.SpanInScope ws = tracer.withSpan(childSpan.start())) {
            // 验证逻辑
            validateOrder(orderDTO);
            
            // 添加Tag
            childSpan.tag("validation.result", "passed");
            
        } finally {
            childSpan.end();
        }
        
        // 保存订单
        return saveOrder(orderDTO);
    }
    
    private void validateOrder(OrderDTO orderDTO) {
        // 验证逻辑
    }
}
```

## 五、Feign链路传递

### 5.1 自动传递

**Sleuth自动在Feign请求中添加Trace Header**:

```java
@FeignClient(name = "user-service")
public interface UserFeignClient {
    
    @GetMapping("/api/users/{id}")
    User getUserById(@PathVariable("id") Long id);
}
```

**无需额外配置,自动传递**:
```
order-service (TraceId: abc123, SpanId: span2)
    ↓ Feign调用
    Header: X-B3-TraceId: abc123
    Header: X-B3-SpanId: span3 (新Span)
    Header: X-B3-ParentSpanId: span2
    ↓
user-service (TraceId: abc123, SpanId: span3)
```

### 5.2 自定义Header传递

```java
@Configuration
public class FeignConfig {
    
    @Bean
    public RequestInterceptor requestInterceptor(Tracer tracer) {
        return template -> {
            Span currentSpan = tracer.currentSpan();
            if (currentSpan != null) {
                // 添加自定义Header
                template.header("X-Custom-Trace", "my-trace");
            }
        };
    }
}
```

## 七、异步链路传递

### 7.1 ThreadLocal问题

```java
// ❌ 问题: 子线程无法获取父线程的Span
@GetMapping("/async/order")
public Result asyncOrder() {
    Span currentSpan = tracer.currentSpan();
    System.out.println("主线程Span: " + currentSpan); // 有值
    
    CompletableFuture.runAsync(() -> {
        Span span = tracer.currentSpan();
        System.out.println("子线程Span: " + span); // null! ❌
    });
    
    return Result.success();
}
```

### 7.2 解决方案

```java
@GetMapping("/async/order")
public Result asyncOrder() {
    Span currentSpan = tracer.currentSpan();
    
    // ✅ 使用Sleuth提供的包装器
    CompletableFuture.runAsync(tracer.wrap(() -> {
        Span span = tracer.currentSpan();
        System.out.println("子线程Span: " + span); // 有值! ✅
        
        // 业务逻辑
        processOrder();
    }));
    
    return Result.success();
}
```

### 7.3 @Async支持

```java
@Configuration
public class AsyncConfig implements AsyncConfigurer {
    
    @Autowired
    private Tracer tracer;
    
    @Override
    public Executor getAsyncExecutor() {
        ThreadPoolTaskExecutor executor = new ThreadPoolTaskExecutor();
        executor.setCorePoolSize(5);
        executor.setMaxPoolSize(10);
        executor.initialize();
        
        // ✅ 使用Sleuth包装
        return new LazyTraceExecutor(executor, tracer);
    }
}
```

**使用**:
```java
@Service
public class OrderService {
    
    @Async
    public void asyncProcessOrder(Order order) {
        // 自动传递Span
        log.info("异步处理订单: {}", order.getOrderId());
    }
}
```

@Component
@EnableBinding(OrderChannels.class)
public class OrderMessageProducer {

## 十、性能监控

### 10.1 慢调用检测

```java
@Component
public class SlowCallDetector {
    
    @Autowired
    private Tracer tracer;
    
    @EventListener
    public void onSpanEnded(SpanEndedEvent event) {
        Span span = event.getSpan();
        long duration = span.getDurationMillis();
        
        // 检测超过1秒的调用
        if (duration > 1000) {
            log.warn("慢调用检测: {} 耗时 {}ms", 
                span.name(), duration);
            
            // 记录到监控系统
            metricsService.recordSlowCall(span);
        }
    }
}
```

### 10.2 错误率监控

```java
@Component
public class ErrorRateMonitor {
    
    @Autowired
    private Tracer tracer;
    
    private final ConcurrentHashMap<String, AtomicInteger> errorCount = new ConcurrentHashMap<>();
    
    @EventListener
    public void onSpanEnded(SpanEndedEvent event) {
        Span span = event.getSpan();
        
        // 检查是否有错误Tag
        String error = span.tags().get("error");
        if (error != null) {
            String serviceName = span.remoteServiceName();
            errorCount.computeIfAbsent(serviceName, k -> new AtomicInteger(0))
                .incrementAndGet();
            
            log.error("服务 {} 发生错误: {}", serviceName, error);
        }
    }
}
```
