## 一、负载均衡概述

### 1.1 为什么需要负载均衡?

**单实例的问题**:
```
客户端 → user-service:8081 (单实例)

问题:
❌ 单点故障: 服务宕机,整个系统不可用
❌ 性能瓶颈: 大量请求时响应慢
❌ 无法水平扩展: 只能升级硬件配置
```

**多实例 + 负载均衡**:
```
                    ┌→ user-service:8081
客户端 → 负载均衡器 ─┼→ user-service:8082
                    └→ user-service:8083

优势:
✅ 高可用: 一个实例宕机,其他实例继续工作
✅ 高性能: 请求分散到多个实例
✅ 水平扩展: 随时增加实例数量
```

### 1.2 负载均衡分类

#### 客户端负载均衡

```
架构:
客户端内置负载均衡器
    ↓
查询注册中心获取服务列表
    ↓
根据策略选择实例
    ↓
直接调用目标实例

代表技术:
- Spring Cloud LoadBalancer
- Netflix Ribbon (已停更)

优势:
✅ 减少网络跳数
✅ 避免单点故障
✅ 支持本地策略定制
```

#### 服务端负载均衡

```
架构:
客户端 → 负载均衡器(Nginx/HAProxy)
           ↓
      后端服务集群

代表技术:
- Nginx
- HAProxy
- LVS

优势:
✅ 客户端无需关心负载均衡
✅ 统一管理
✅ 支持SSL终止
```

### 1.3 常见负载均衡策略

| 策略 | 说明 | 适用场景 |
|------|------|----------|
| **轮询(Round Robin)** | 按顺序依次分配 | 实例性能相近 |
| **随机(Random)** | 随机选择一个实例 | 简单场景 |
| **加权轮询(Weighted)** | 根据权重分配请求 | 实例性能不均 |
| **最少连接(Least Connections)** | 选择当前连接数最少的实例 | 长连接场景 |
| **一致性哈希(Consistent Hash)** | 相同请求路由到同一实例 | 缓存场景 |
| **响应时间(Response Time)** | 选择响应最快的实例 | 对延迟敏感 |

## 二、Spring Cloud LoadBalancer

### 2.1 LoadBalancer vs Ribbon

**Ribbon的问题**:
```
2012年: Netflix开源Ribbon
2018年: 进入维护模式
2020年: 停止更新

问题:
❌ 代码老旧,难以维护
❌ 不支持响应式编程
❌ 与新版Spring Cloud兼容性差
```

**LoadBalancer的优势**:
```
2019年: Spring官方推出LoadBalancer

优势:
✅ 官方支持,持续更新
✅ 支持响应式编程(WebFlux)
✅ 更好的性能
✅ 与Spring Cloud无缝集成
```

**对比表**:

| 特性 | Ribbon | LoadBalancer |
|------|--------|--------------|
| **维护状态** | 停更 | 活跃 |
| **响应式支持** | ❌ | ✅ |
| **性能** | 一般 | 优秀 |
| **学习成本** | 中 | 低 |
| **推荐程度** | 不推荐 | ⭐⭐⭐⭐⭐ |

### 2.2 快速开始

#### 步骤1: 添加依赖

```xml
<!-- Spring Cloud 2020+ 已内置LoadBalancer -->
<dependency>
    <groupId>org.springframework.cloud</groupId>
    <artifactId>spring-cloud-starter-loadbalancer</artifactId>
</dependency>
```

#### 步骤2: 启用负载均衡

```java
@Configuration
public class LoadBalancerConfig {
    
    @Bean
    @LoadBalanced  // 关键注解
    public RestTemplate restTemplate() {
        return new RestTemplate();
    }
}
```

#### 步骤3: 使用服务名调用

```java
@RestController
public class OrderController {
    
    @Autowired
    private RestTemplate restTemplate;
    
    @GetMapping("/order/{userId}")
    public User getUser(@PathVariable Long userId) {
        // ✅ 使用服务名,自动负载均衡
        String url = "http://user-service/api/users/" + userId;
        return restTemplate.getForObject(url, User.class);
    }
}
```

## 三、自定义负载均衡策略

### 3.1 内置策略

**RoundRobinLoadBalancer (默认)**:
```java
@Configuration
public class LoadBalancerConfig {
    
    @Bean
    public ReactorLoadBalancer<ServiceInstance> randomLoadBalancer(
            Environment environment,
            LoadBalancerClientFactory loadBalancerClientFactory) {
        
        String name = environment.getProperty(LoadBalancerClientFactory.PROPERTY_NAME);
        
        // 轮询策略(默认)
        return new RoundRobinLoadBalancer(
            loadBalancerClientFactory.getLazyProvider(name, ServiceInstanceListSupplier.class),
            name
        );
    }
}
```

**RandomLoadBalancer**:
```java
@Configuration
public class LoadBalancerConfig {
    
    @Bean
    public ReactorLoadBalancer<ServiceInstance> randomLoadBalancer(
            Environment environment,
            LoadBalancerClientFactory loadBalancerClientFactory) {
        
        String name = environment.getProperty(LoadBalancerClientFactory.PROPERTY_NAME);
        
        // 随机策略
        return new RandomLoadBalancer(
            loadBalancerClientFactory.getLazyProvider(name, ServiceInstanceListSupplier.class),
            name
        );
    }
}
```

### 3.2 自定义策略

#### 基于权重的负载均衡

```java
@Component
public class WeightedLoadBalancer implements ReactorServiceInstanceLoadBalancer {
    
    private static final Log log = LogFactory.getLog(WeightedLoadBalancer.class);
    
    private final ObjectProvider<ServiceInstanceListSupplier> serviceInstanceListSupplierProvider;
    private final String serviceId;
    
    public WeightedLoadBalancer(
            ObjectProvider<ServiceInstanceListSupplier> serviceInstanceListSupplierProvider,
            String serviceId) {
        this.serviceInstanceListSupplierProvider = serviceInstanceListSupplierProvider;
        this.serviceId = serviceId;
    }
    
    @Override
    public Mono<Response<ServiceInstance>> choose(Request request) {
        ServiceInstanceListSupplier supplier = serviceInstanceListSupplierProvider.getIfAvailable();
        
        return supplier.get().next()
            .map(serviceInstances -> {
                if (serviceInstances.isEmpty()) {
                    log.warn("No servers available for service: " + serviceId);
                    return new EmptyResponse();
                }
                
                // 根据元数据中的权重选择实例
                List<ServiceInstance> instances = serviceInstances.stream()
                    .sorted((a, b) -> {
                        int weightA = getWeight(a);
                        int weightB = getWeight(b);
                        return weightB - weightA; // 权重高的优先
                    })
                    .collect(Collectors.toList());
                
                // 简单的加权轮询
                ServiceInstance instance = instances.get(0);
                log.info("Selected instance: {} with weight: {}", 
                    instance.getHost(), getWeight(instance));
                
                return new DefaultResponse(instance);
            });
    }
    
    private int getWeight(ServiceInstance instance) {
        // 从元数据中获取权重
        String weight = instance.getMetadata().getOrDefault("weight", "1");
        return Integer.parseInt(weight);
    }
}
```

**配置权重**:
```yaml
# application.yml
spring:
  cloud:
    nacos:
      discovery:
        metadata:
          weight: 5  # 设置权重为5
```

#### 基于区域亲和性的负载均衡

```java
@Component
public class ZoneAwareLoadBalancer implements ReactorServiceInstanceLoadBalancer {
    
    private final ObjectProvider<ServiceInstanceListSupplier> serviceInstanceListSupplierProvider;
    private final String serviceId;
    private final String currentZone;
    
    public ZoneAwareLoadBalancer(
            ObjectProvider<ServiceInstanceListSupplier> serviceInstanceListSupplierProvider,
            @Value("${spring.cloud.nacos.discovery.metadata.zone:default}") String currentZone) {
        this.serviceInstanceListSupplierProvider = serviceInstanceListSupplierProvider;
        this.serviceId = serviceId;
        this.currentZone = currentZone;
    }
    
    @Override
    public Mono<Response<ServiceInstance>> choose(Request request) {
        ServiceInstanceListSupplier supplier = serviceInstanceListSupplierProvider.getIfAvailable();
        
        return supplier.get().next()
            .map(serviceInstances -> {
                if (serviceInstances.isEmpty()) {
                    return new EmptyResponse();
                }
                
                // 优先选择同区域的实例
                List<ServiceInstance> sameZoneInstances = serviceInstances.stream()
                    .filter(instance -> currentZone.equals(getZone(instance)))
                    .collect(Collectors.toList());
                
                ServiceInstance selectedInstance;
                if (!sameZoneInstances.isEmpty()) {
                    // 同区域有实例,随机选择一个
                    selectedInstance = sameZoneInstances.get(
                        ThreadLocalRandom.current().nextInt(sameZoneInstances.size())
                    );
                } else {
                    // 同区域没有实例,从所有实例中随机选择
                    selectedInstance = serviceInstances.get(
                        ThreadLocalRandom.current().nextInt(serviceInstances.size())
                    );
                }
                
                return new DefaultResponse(selectedInstance);
            });
    }
    
    private String getZone(ServiceInstance instance) {
        return instance.getMetadata().getOrDefault("zone", "default");
    }
}
```

**配置区域**:
```yaml
# application.yml
spring:
  cloud:
    nacos:
      discovery:
        metadata:
          zone: beijing  # 北京区域
```

## 七、常见问题

### 7.1 负载均衡不生效

**问题**:
```
报错: java.net.UnknownHostException: user-service
```

**解决**:
```java
// ❌ 错误: 忘记添加@LoadBalanced
@Bean
public RestTemplate restTemplate() {
    return new RestTemplate();
}

// ✅ 正确: 添加@LoadBalanced
@Bean
@LoadBalanced
public RestTemplate restTemplate() {
    return new RestTemplate();
}
```

### 7.2 实例选择不均匀

**问题**:
```
实例1: 80% 流量
实例2: 15% 流量
实例3: 5% 流量
```

**原因**:
- 实例启动时间不同,缓存未刷新
- 健康检查失败,部分实例被剔除

**解决**:
```yaml
# 调整缓存过期时间
spring:
  cloud:
    loadbalancer:
      cache:
        ttl: 10s  # 缩短缓存时间
```

### 7.3 Ribbon迁移问题

**问题**:
```
老项目使用Ribbon,如何迁移到LoadBalancer?
```

**解决步骤**:
```xml
<!-- 1. 移除Ribbon依赖 -->
<!-- 删除以下依赖 -->
<dependency>
    <groupId>org.springframework.cloud</groupId>
    <artifactId>spring-cloud-starter-netflix-ribbon</artifactId>
</dependency>

<!-- 2. 添加LoadBalancer依赖 -->
<dependency>
    <groupId>org.springframework.cloud</groupId>
    <artifactId>spring-cloud-starter-loadbalancer</artifactId>
</dependency>

<!-- 3. 代码无需修改,@LoadBalanced注解保持不变 -->
```

## 八、最佳实践

### 8.1 策略选择建议

| 场景 | 推荐策略 | 说明 |
|------|----------|------|
| **通用场景** | RoundRobin | 简单高效 |
| **实例性能不均** | Weighted | 根据性能分配权重 |
| **跨区域部署** | ZoneAware | 优先同区域调用 |
| **缓存场景** | ConsistentHash | 相同请求路由到同一实例 |
| **压力测试** | Random | 均匀分布请求 |

### 8.2 健康检查配置

```yaml
# 生产环境推荐配置
spring:
  cloud:
    loadbalancer:
      health-check:
        interval: 10s  # 每10秒检查一次
        initial-delay: 30s  # 启动30秒后开始检查
        path: /actuator/health
      cache:
        enabled: true
        ttl: 35s  # 略大于健康检查间隔
      retry:
        enabled: true
        max-retries: 3
```
