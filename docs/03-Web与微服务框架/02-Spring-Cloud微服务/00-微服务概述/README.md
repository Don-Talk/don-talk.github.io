## 💡 常见问题

### Q1: 我的项目需要微服务吗?

**判断标准**:
```
✅ 团队规模 > 20人
✅ 代码行数 > 50万行
✅ QPS > 10000
✅ 需要快速迭代

如果以上都是YES → 考虑微服务
否则 → 单体架构可能更合适
```

### Q2: 如何确定服务粒度?

**原则**:
```
- 单一职责: 一个服务只做一件事
- 独立部署: 可以单独发布
- 团队结构: 一个团队负责2-3个服务
- 调用链深度: ≤ 3层
```

### Q3: 微服务的最大挑战是什么?

**答案**:
```
1. 分布式复杂性 (网络延迟/部分失败)
2. 数据一致性 (分布式事务)
3. 运维成本高 (需要DevOps自动化)
4. 测试复杂 (需要Mock和集成测试)
```
