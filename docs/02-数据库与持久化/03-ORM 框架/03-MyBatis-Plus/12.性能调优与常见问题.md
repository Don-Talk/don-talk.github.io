## 1. N+1 查询问题

### 1.1 问题描述

```java
// 查询所有用户
List<User> users = userMapper.selectList(null);  // 1次查询

// 遍历用户，访问订单
for (User user : users) {
    List<Order> orders = orderMapper.selectList(
        new LambdaQueryWrapper<Order>().eq(Order::getUserId, user.getId())
    );  // N次查询
}

// 总共执行：1 + N 次查询
```

### 1.2 解决方案一：JOIN 查询

```xml
<select id="selectUserWithOrders" resultType="UserVO">
    SELECT 
        u.id,
        u.username,
        o.id as orderId,
        o.amount
    FROM user u
    LEFT JOIN orders o ON u.id = o.user_id
</select>
```

### 1.3 解决方案二：批量查询

```java
// 先查询所有用户
List<User> users = userMapper.selectList(null);

// 提取所有用户ID
List<Long> userIds = users.stream()
    .map(User::getId)
    .collect(Collectors.toList());

// 批量查询订单
List<Order> orders = orderMapper.selectList(
    new LambdaQueryWrapper<Order>().in(Order::getUserId, userIds)
);

// 在内存中组装
Map<Long, List<Order>> orderMap = orders.stream()
    .collect(Collectors.groupingBy(Order::getUserId));

users.forEach(user -> {
    user.setOrders(orderMap.getOrDefault(user.getId(), new ArrayList<>()));
});

// 只执行 2 次查询
```

## 2. 批量操作优化

### 2.1 批量插入

```java
@Service
public class UserService {
    
    @Autowired
    private UserServiceImpl userService;  // IService
    
    public void batchInsert(List<User> users) {
        // MP 内置批量插入（默认每批 1000 条）
        userService.saveBatch(users);
        
        // 指定批次大小
        userService.saveBatch(users, 500);
    }
}
```

**底层实现**：
```sql
INSERT INTO user (username, email) VALUES (?, ?), (?, ?), ...;
```

### 2.2 批量更新

```java
@Test
public void testBatchUpdate() {
    List<User> users = new ArrayList<>();
    // ... 准备数据
    
    // 批量更新
    userService.updateBatchById(users);
}
```

### 2.3 配置批量大小

```yaml
mybatis-plus:
  configuration:
    # JDBC 批量大小
    default-executor-type: batch
```

## 3. 缓存策略

### 3.1 一级缓存（SqlSession 级别）

* 默认开启
* 同一个 SqlSession 内有效
* 自动管理

```java
@Test
public void testFirstLevelCache() {
    // 第一次查询
    User user1 = userMapper.selectById(1L);
    
    // 第二次查询（从缓存获取）
    User user2 = userMapper.selectById(1L);
    
    System.out.println(user1 == user2);  // true
}
```

### 3.2 二级缓存（Mapper 级别）

#### 启用二级缓存

```yaml
mybatis-plus:
  configuration:
    cache-enabled: true
```

#### Mapper 接口标注

```java
@CacheNamespace
public interface UserMapper extends BaseMapper<User> {
}
```

#### 实体类实现 Serializable

```java
@Data
public class User implements Serializable {
    private static final long serialVersionUID = 1L;
    // ...
}
```

### 3.3 Redis 缓存

```xml
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-data-redis</artifactId>
</dependency>
```

```java
@Service
public class UserService {
    
    @Autowired
    private RedisTemplate<String, Object> redisTemplate;
    
    @Autowired
    private UserMapper userMapper;
    
    public User getUserById(Long id) {
        String key = "user:" + id;
        
        // 先从 Redis 查询
        User user = (User) redisTemplate.opsForValue().get(key);
        
        if (user != null) {
            return user;
        }
        
        // Redis 没有，从数据库查询
        user = userMapper.selectById(id);
        
        // 写入 Redis（过期时间 1 小时）
        redisTemplate.opsForValue().set(key, user, 1, TimeUnit.HOURS);
        
        return user;
    }
    
    public void updateUser(User user) {
        userMapper.updateById(user);
        
        // 删除缓存
        String key = "user:" + user.getId();
        redisTemplate.delete(key);
    }
}
```


```yaml

  datasource:
    driver-class-name: com.p6spy.engine.spy.P6SpyDriver
    url: jdbc:p6spy:mysql://localhost:3306/mp_demo
```

输出示例：
```

```

### 4.3 索引优化

```sql
-- 为常用查询条件添加索引
CREATE INDEX idx_username ON user(username);
CREATE INDEX idx_age_email ON user(age, email);
CREATE INDEX idx_create_time ON user(create_time);
```

### 4.4 EXPLAIN 分析

```sql
EXPLAIN SELECT * FROM user WHERE age > 20 AND username LIKE '%张%';
```

关注指标：
* `type`：访问类型（ALL、index、range、ref、const）
* `key`：实际使用的索引
* `rows`：扫描行数
* `Extra`：额外信息

## 5. 常见陷阱

### 5.1 空指针异常

```java
// ❌ 错误
User user = userMapper.selectById(999L);
System.out.println(user.getUsername());  // NullPointerException

// ✅ 正确
User user = userMapper.selectById(999L);
if (user != null) {
    System.out.println(user.getUsername());
}
```

### 5.2 逻辑删除失效

```java
// ❌ 错误：自定义 SQL 未加逻辑删除条件
@Select("SELECT * FROM user")
List<User> selectAll();

// ✅ 正确
@Select("SELECT * FROM user WHERE deleted = 0")
List<User> selectAll();
```

### 5.3 事务失效

```java
// ❌ 错误：private 方法
@Transactional
private void method() {
    // 事务不生效
}

// ✅ 正确
@Transactional
public void method() {
    // 事务生效
}
```

### 5.4 分页参数错误

```java
// ❌ 错误：页码从 0 开始
Page<User> page = new Page<>(0, 10);

// ✅ 正确：页码从 1 开始
Page<User> page = new Page<>(1, 10);
```

### 5.5 Wrapper 条件为空

```java
// ❌ 错误：可能生成 WHERE 1=1
LambdaQueryWrapper<User> wrapper = new LambdaQueryWrapper<>();
if (StringUtils.isNotBlank(username)) {
    wrapper.eq(User::getUsername, username);
}
// 如果 username 为空，wrapper 没有任何条件

// ✅ 正确：使用 condition 参数
LambdaQueryWrapper<User> wrapper = new LambdaQueryWrapper<>();
wrapper.eq(StringUtils.isNotBlank(username), User::getUsername, username);
```
