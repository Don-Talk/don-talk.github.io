## 1. 什么是 MyBatis-Plus？

### 1.1 MP 简介

MyBatis-Plus（简称 MP）是一个 MyBatis 的**增强工具**，在 MyBatis 的基础上只做增强不做改变，为简化开发、提高效率而生。

* **官网**：https://baomidou.com/
* **GitHub**：https://github.com/baomidou/mybatis-plus
* **当前版本**：3.5.x
* **愿景**：成为 MyBatis 最好的搭档，就像魂斗罗中的 1P、2P，基友搭配，效率翻倍

### 1.2 核心特性

✅ **无侵入**：只做增强不做改变，引入它不会对现有工程产生影响  
✅ **损耗小**：启动即会自动注入基本 CURD，性能基本无损耗  
✅ **强大的 CRUD 操作**：内置通用 Mapper、Service，仅需少量配置即可实现单表大部分 CRUD 操作  
✅ **支持 Lambda 形式调用**：通过 Lambda 表达式，方便的编写各类查询条件，无需再担心字段写错  
✅ **支持主键自动生成**：支持多达 4 种主键策略（内含分布式唯一 ID 生成器 - Sequence），可自由配置  
✅ **支持 ActiveRecord 模式**：支持 ActiveRecord 形式调用，实体类只需继承 Model 类即可进行强大的 CRUD 操作  
✅ **支持自定义全局通用操作**：支持全局通用方法注入（Write once, use anywhere）  
✅ **内置代码生成器**：采用代码或者 Maven 插件可快速生成 Mapper、Model、Service、Controller 层代码  
✅ **内置分页插件**：基于 MyBatis 物理分页，开发者无需关心具体操作  
✅ **分页插件支持多种数据库**：支持 MySQL、MariaDB、Oracle、DB2、H2、HSQL、SQLite、Postgre、SQLServer 等多种数据库  
✅ **内置性能分析插件**：可输出 Sql 语句以及其执行时间，建议开发测试时启用该功能  
✅ **内置全局拦截插件**：提供全表 delete、update 操作智能分析阻断，也可自定义拦截规则

## 2. MP vs MyBatis vs JPA 对比

### 2.1 功能对比

| 特性 | MyBatis | MyBatis-Plus | Spring Data JPA |
|------|---------|--------------|-----------------|
| **CRUD** | 需手写 SQL | ✅ 内置通用 CRUD | ✅ 内置 Repository |
| **分页** | 需配置插件 | ✅ 内置分页插件 | ✅ 内置分页 |
| **条件构造** | 手动拼接 | ✅ Wrapper 构造器 | ✅ Specification |
| **代码生成** | 无 | ✅ 内置生成器 | ❌ 无 |
| **学习成本** | 中等 | 低 | 高 |
| **灵活性** | 高 | 高 | 中等 |
| **SQL 控制** | 完全控制 | 完全控制 | 自动生成 |
| **适用场景** | 灵活查询 | 大多数项目 | 复杂领域模型 |

### 2.2 代码量对比

#### MyBatis 方式

```xml
<!-- UserMapper.xml -->
<select id="selectList" resultType="User">
    SELECT * FROM user WHERE age &gt; #{age}
</select>

<insert id="insert">
    INSERT INTO user (username, age) VALUES (#{username}, #{age})
</insert>
```

```java
// UserMapper.java
List<User> selectList(@Param("age") Integer age);
int insert(User user);
```

#### MyBatis-Plus 方式

```java
// UserMapper.java - 无需 XML
List<User> users = userMapper.selectList(
    new QueryWrapper<User>().gt("age", 20)
);

userMapper.insert(user);
```

**代码量减少约 70%！**

### 2.3 选择建议

**使用 MyBatis-Plus 的场景**：
* 快速开发 CRUD 功能
* 团队熟悉 MyBatis
* 需要灵活的 SQL 控制
* 国内项目（MP 在国内非常流行）

**使用原生 MyBatis 的场景**：
* 遗留系统维护
* 对框架有严格控制要求

**使用 JPA 的场景**：
* 国际化项目
* 复杂领域模型驱动设计
* 团队熟悉 JPA 规范

## 4. 第一个 MP 程序

### 4.1 创建实体类

```java
package com.example.entity;

import com.baomidou.mybatisplus.annotation.*;
import lombok.Data;
import java.io.Serializable;
import java.time.LocalDateTime;

/**
 * 用户实体类
 */
@Data
@TableName("user")  // 指定表名（可省略，默认类名转下划线）
public class User implements Serializable {
    
    private static final long serialVersionUID = 1L;
    
    @TableId(type = IdType.AUTO)  // 主键自增
    private Long id;
    
    @TableField("username")  // 字段名（可省略，默认驼峰转下划线）
    private String username;
    
    private String email;
    
    private Integer age;
    
    private Integer gender;
    
    @TableLogic  // 逻辑删除
    private Integer deleted;
    
    @TableField(fill = FieldFill.INSERT)  // 插入时自动填充
    private LocalDateTime createTime;
    
    @TableField(fill = FieldFill.INSERT_UPDATE)  // 插入和更新时自动填充
    private LocalDateTime updateTime;
}
```

**常用注解说明**：

| 注解 | 作用 | 示例 |
|------|------|------|
| `@TableName` | 指定表名 | `@TableName("sys_user")` |
| `@TableId` | 主键字段 | `@TableId(type = IdType.AUTO)` |
| `@TableField` | 字段映射 | `@TableField("user_name")` |
| `@TableLogic` | 逻辑删除 | `@TableLogic` |

### 4.2 创建 Mapper 接口

```java
package com.example.mapper;

import com.baomidou.mybatisplus.core.mapper.BaseMapper;
import com.example.entity.User;
import org.apache.ibatis.annotations.Mapper;

/**
 * 用户 Mapper 接口
 * 继承 BaseMapper 后，无需编写任何方法即可使用 CRUD
 */
@Mapper
public interface UserMapper extends BaseMapper<User> {
    // 无需编写任何代码！
}
```

**关键点**：
* 继承 `BaseMapper<T>` 接口
* 泛型 `T` 是实体类类型
* 无需编写任何方法，已拥有 CRUD 功能

### 4.3 启动类配置

```java

```

## 5. 代码生成器快速体验

### 5.1 添加依赖

```xml
<dependency>
    <groupId>com.baomidou</groupId>
    <artifactId>mybatis-plus-generator</artifactId>
    <version>3.5.3.1</version>
</dependency>

<dependency>
    <groupId>org.apache.velocity</groupId>
    <artifactId>velocity-engine-core</artifactId>
    <version>2.3</version>
</dependency>
```

### 5.2 生成代码

```java
package com.example.generator;

import com.baomidou.mybatisplus.generator.FastAutoGenerator;
import com.baomidou.mybatisplus.generator.config.OutputFile;

import java.util.Collections;

public class CodeGenerator {
    
    public static void main(String[] args) {
        FastAutoGenerator.create(
            "jdbc:mysql://localhost:3306/mp_demo?useSSL=false&serverTimezone=Asia/Shanghai",
            "root",
            "123456"
        )
        .globalConfig(builder -> {
            builder.author("Your Name")           // 作者
                   .outputDir("src/main/java");   // 输出目录
        })
        .packageConfig(builder -> {
            builder.parent("com.example")         // 父包名
                   .moduleName("system")          // 模块名
                   .entity("entity")
                   .mapper("mapper")
                   .service("service")
                   .controller("controller")
                   .pathInfo(Collections.singletonMap(
                       OutputFile.xml, "src/main/resources/mapper"));
        })
        .strategyConfig(builder -> {
            builder.addInclude("user")            // 需要生成的表
                   .entityBuilder()
                   .enableLombok()                // 启用 Lombok
                   .enableTableFieldAnnotation()  // 启用字段注解
                   .controllerBuilder()
                   .enableRestStyle();            // 启用 REST 风格
        })
        .execute();
    }
}
```

运行后自动生成：
* Entity 实体类
* Mapper 接口
* Service 接口和实现类
* Controller 控制器
* Mapper XML 文件

## 6. 常见问题

### 6.1 Mapper 找不到

**错误**：
```
Consider defining a bean of type 'com.example.mapper.UserMapper' in your configuration.
```

**解决**：
* 检查是否添加了 `@MapperScan` 或 `@Mapper` 注解
* 检查包路径是否正确

### 6.2 表名不匹配

**错误**：
```
Table 'mp_demo.user' doesn't exist
```

**解决**：
* 使用 `@TableName` 指定正确的表名
* 检查数据库连接配置

### 6.3 主键策略问题

**错误**：
```
Field 'id' doesn't have a default value
```

**解决**：
* 配置正确的主键策略：`@TableId(type = IdType.AUTO)`
* 确保数据库字段设置了自增
