## 1. 逻辑删除概述

### 1.1 什么是逻辑删除？

逻辑删除不是真正从数据库中删除数据，而是通过标记字段标识数据已删除。

**对比**：

| 特性 | 物理删除 | 逻辑删除 |
|------|---------|---------|
| **操作** | DELETE FROM table | UPDATE table SET deleted=1 |
| **数据恢复** | ❌ 不可恢复 | ✅ 可恢复 |
| **数据安全** | 低 | 高 |
| **查询性能** | 高 | 略低（需加条件） |
| **存储空间** | 小 | 大 |

### 1.2 适用场景

✅ **推荐逻辑删除**：
* 用户数据（需要保留历史记录）
* 订单数据（需要审计）
* 重要业务数据

❌ **可以物理删除**：
* 临时数据
* 日志数据
* 缓存数据

## 2. 配置逻辑删除

### 2.1 数据库添加字段

```sql
ALTER TABLE user ADD COLUMN deleted TINYINT DEFAULT 0 COMMENT '逻辑删除：0-未删除，1-已删除';
```

### 2.2 全局配置

```yaml
mybatis-plus:
  global-config:
    db-config:
      logic-delete-field: deleted      # 全局逻辑删除字段名
      logic-delete-value: 1            # 已删除值
      logic-not-delete-value: 0        # 未删除值
```

### 2.3 实体类标注

```java
@Data
@TableName("user")
public class User {
    
    @TableId(type = IdType.AUTO)
    private Long id;
    
    private String username;
    
    private String email;
    
    @TableLogic  // 标注逻辑删除字段
    private Integer deleted;
}
```

## 3. 使用示例

### 3.1 删除操作

```java
@Test
public void testDelete() {
    // 执行删除
    int rows = userMapper.deleteById(1L);
    System.out.println("影响行数：" + rows);
}
```

**实际执行的 SQL**：
```sql
-- 不是 DELETE，而是 UPDATE
UPDATE user SET deleted=1 WHERE id=1 AND deleted=0
```

### 3.2 查询操作

```java
@Test
public void testSelect() {
    // 自动添加 WHERE deleted=0 条件
    List<User> users = userMapper.selectList(null);
}
```

**实际执行的 SQL**：
```sql
SELECT id,username,email,deleted FROM user WHERE deleted=0
```

### 3.3 更新操作

```java
@Test
public void testUpdate() {
    User user = new User();
    user.setId(1L);
    user.setUsername("新名字");
    
    userMapper.updateById(user);
}
```

**实际执行的 SQL**：
```sql
UPDATE user SET username='新名字' WHERE id=1 AND deleted=0
```

## 5. 局部配置

如果不同表的逻辑删除字段不同，可以使用局部配置。

### 5.1 注解配置

```java
@Data
@TableName("article")
public class Article {
    
    @TableId
    private Long id;
    
    private String title;
    
    @TableLogic(value = "0", delval = "1")
    private Integer isDeleted;
}
```

### 5.2 不同表不同字段

```java
// 用户表
@Data
public class User {
    @TableLogic
    private Integer deleted;  // deleted 字段
}

// 文章表
@Data
public class Article {
    @TableLogic(value = "0", delval = "1")
    private Integer isDeleted;  // is_deleted 字段
}
```

## 6. 数据权限

### 6.1 什么是数据权限？

数据权限是指控制用户能访问哪些数据，如：
* 只能查看本部门数据
* 只能查看自己创建的数据
* 管理员可以查看所有数据

### 6.2 实现方案一：拦截器

```java
@Component
@Intercepts({
    @Signature(type = StatementHandler.class, 
               method = "prepare", 
               args = {Connection.class, Integer.class})
})
public class DataPermissionInterceptor implements Interceptor {
    
    @Override
    public Object intercept(Invocation invocation) throws Throwable {
        StatementHandler statementHandler = 
            (StatementHandler) invocation.getTarget();
        
        MetaObject metaObject = SystemMetaObject.forObject(statementHandler);
        
        // 获取 SQL
        String sql = (String) metaObject.getValue("delegate.boundSql.sql");
        
        // 获取当前用户
        Long userId = UserContext.getCurrentUserId();
        Long deptId = UserContext.getCurrentDeptId();
        
        // 判断是否需要添加数据权限
        if (needDataPermission(sql)) {
            // 添加权限条件
            String permissionSql = addPermissionCondition(sql, userId, deptId);
            metaObject.setValue("delegate.boundSql.sql", permissionSql);
        }
        
        return invocation.proceed();
    }
    
    private boolean needDataPermission(String sql) {
        // 判断是否是 SELECT 语句
        return sql.trim().toUpperCase().startsWith("SELECT");
    }
    
    private String addPermissionCondition(String sql, Long userId, Long deptId) {
        // 根据用户角色添加不同的条件
        UserRole role = UserContext.getCurrentUserRole();
        
        if (role == UserRole.ADMIN) {
            return sql;  // 管理员不限制
        } else if (role == UserRole.DEPT_MANAGER) {
            return sql + " AND dept_id = " + deptId;
        } else {
            return sql + " AND create_by = " + userId;
        }
    }
}
```

### 6.3 实现方案二：注解 + AOP

```java
@Target({ElementType.METHOD})
@Retention(RetentionPolicy.RUNTIME)
public @interface DataScope {
    String deptAlias() default "";
    String userAlias() default "";
}
```

```java
@Aspect
@Component
public class DataScopeAspect {
    
    @Before("@annotation(dataScope)")
    public void doBefore(DataScope dataScope) {
        // 获取当前用户信息
        Long userId = UserContext.getCurrentUserId();
        Long deptId = UserContext.getCurrentDeptId();
        
        // 将权限参数存入 ThreadLocal
        DataScopeContext.setUserId(userId);
        DataScopeContext.setDeptId(deptId);
    }
}
```

```java
@Service
public class UserService {
    
    @Autowired
    private UserMapper userMapper;
    
    @DataScope(deptAlias = "u", userAlias = "u")
    public List<User> getUserList() {
        return userMapper.selectList(null);
    }
}
```

## 7. 多租户支持

### 7.1 什么是多租户？

多租户是指一套系统服务于多个客户（租户），每个租户的数据相互隔离。

**隔离级别**：
* 数据库隔离：每个租户独立数据库
* Schema 隔离：每个租户独立 Schema
* 数据隔离：同一张表，通过 tenant_id 区分

### 7.2 MP 多租户插件

#### 配置插件

```java
@Bean
public MybatisPlusInterceptor mybatisPlusInterceptor() {
    MybatisPlusInterceptor interceptor = new MybatisPlusInterceptor();
    
    // 添加多租户插件
    TenantLineInnerInterceptor tenantInterceptor = 
        new TenantLineInnerInterceptor(new TenantLineHandler() {
            
            @Override
            public Expression getTenantId() {
                // 返回当前租户 ID
                Long tenantId = TenantContext.getCurrentTenantId();
                return new LongValue(tenantId);
            }
            
            @Override
            public String getTenantIdColumn() {
                return "tenant_id";
            }
            
            @Override
            public boolean ignoreTable(String tableName) {
                // 某些表不需要租户隔离（如字典表）
                return "dict".equals(tableName);
            }
        });
    
    interceptor.addInnerInterceptor(tenantInterceptor);
    
    return interceptor;
}
```

#### 实体类

```java
@Data
@TableName("order")
public class Order {
    
    @TableId
    private Long id;
    
    private Long tenantId;  // 租户 ID
    
    private String orderNo;
    
    private BigDecimal amount;
}
```

#### 使用

```java
@Test
public void testTenant() {
    // 设置当前租户
    TenantContext.setCurrentTenantId(1L);
    
    // 查询时自动添加 tenant_id 条件
    List<Order> orders = orderMapper.selectList(null);
    
    // 生成的 SQL：
    // SELECT * FROM order WHERE tenant_id = 1
}
```

## 9. 常见问题

### 9.1 逻辑删除失效

**问题**：deleteById 仍然执行 DELETE 语句

**原因**：
* 未配置全局逻辑删除
* 未添加 @TableLogic 注解

**解决**：检查配置和注解

### 9.2 查询包含已删除数据

**问题**：selectList 返回了 deleted=1 的数据

**原因**：手动编写 SQL 未加条件

**解决**：
```xml
<!-- ❌ 错误 -->
<select id="selectCustom" resultType="User">
    SELECT * FROM user
</select>

<!-- ✅ 正确 -->
<select id="selectCustom" resultType="User">
    SELECT * FROM user WHERE deleted = 0
</select>
```

### 9.3 性能问题

**问题**：逻辑删除导致查询变慢

**优化**：
* 为 deleted 字段添加索引
* 定期归档历史数据

```sql
CREATE INDEX idx_deleted ON user(deleted);
```
