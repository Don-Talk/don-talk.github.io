## 1. BaseMapper 接口解析

### 1.1 BaseMapper 方法总览

`BaseMapper<T>` 是 MP 的核心接口，提供了完整的 CRUD 方法。

<!-- TODO: 插入图片：BaseMapper 方法结构图 -->

```java
public interface BaseMapper<T> extends Mapper<T> {
    
    // ========== Insert ==========
    int insert(T entity);
    
    // ========== Delete ==========
    int deleteById(Serializable id);
    int deleteByMap(@Param("cm") Map<String, Object> columnMap);
    int delete(@Param("ew") Wrapper<T> wrapper);
    int deleteBatchIds(@Param("coll") Collection<? extends Serializable> idList);
    
    // ========== Update ==========
    int updateById(@Param("et") T entity);
    int update(@Param("et") T entity, @Param("ew") Wrapper<T> updateWrapper);
    
    // ========== Select ==========
    T selectById(Serializable id);
    List<T> selectBatchIds(@Param("coll") Collection<? extends Serializable> idList);
    List<T> selectByMap(@Param("cm") Map<String, Object> columnMap);
    T selectOne(@Param("ew") Wrapper<T> queryWrapper);
    Integer selectCount(@Param("ew") Wrapper<T> queryWrapper);
    List<T> selectList(@Param("ew") Wrapper<T> queryWrapper);
    List<Map<String, Object>> selectMaps(@Param("ew") Wrapper<T> queryWrapper);
    List<Object> selectObjs(@Param("ew") Wrapper<T> queryWrapper);
    <E extends IPage<T>> E selectPage(E page, @Param("ew") Wrapper<T> queryWrapper);
    <E extends IPage<Map<String, Object>>> E selectMapsPage(E page, @Param("ew") Wrapper<T> queryWrapper);
}
```

**统计**：共 17 个方法，覆盖所有 CRUD 场景！

## 2. Insert 操作

### 2.1 基本插入

```java
@Test
public void testInsert() {
    User user = new User();
    user.setUsername("新用户");
    user.setEmail("new@example.com");
    user.setAge(25);
    user.setGender(1);
    
    int rows = userMapper.insert(user);
    System.out.println("影响行数：" + rows);
    System.out.println("生成的ID：" + user.getId());  // 自动回填主键
}
```

**输出**：
```sql
==>  Preparing: INSERT INTO user ( username, email, age, gender ) VALUES ( ?, ?, ?, ? )
==> Parameters: 新用户(String), new@example.com(String), 25(Integer), 1(Integer)
<==    Updates: 1

影响行数：1
生成的ID：5
```

### 2.2 主键策略

MP 支持多种主键生成策略：

| 策略 | 说明 | 适用数据库 |
|------|------|-----------|
| `AUTO` | 数据库自增 | MySQL、SQL Server |
| `INPUT` | 手动输入 | 所有 |
| `ASSIGN_ID` | 雪花算法（默认） | 所有（分布式推荐） |
| `ASSIGN_UUID` | UUID | 所有 |

#### 配置方式

```java
// 方式一：注解配置
@TableId(type = IdType.ASSIGN_ID)
private Long id;

// 方式二：全局配置（application.yml）
mybatis-plus:
  global-config:
    db-config:
      id-type: assign_id
```

#### 雪花算法示例

```java
@Test
public void testAssignId() {
    User user = new User();
    user.setUsername("测试用户");
    
    // ID 由 MP 自动生成（19位数字）
    userMapper.insert(user);
    System.out.println("生成的ID：" + user.getId());
    // 输出：生成的ID：1234567890123456789
}
```

**优势**：
* 分布式唯一 ID
* 趋势递增
* 高性能

### 2.3 批量插入

```java
@Test
public void testBatchInsert() {
    List<User> users = new ArrayList<>();
    
    for (int i = 0; i < 10; i++) {
        User user = new User();
        user.setUsername("用户" + i);
        user.setEmail("user" + i + "@example.com");
        user.setAge(20 + i);
        users.add(user);
    }
    
    // MP 没有内置批量插入，需要自定义 SQL
    // 或使用 Service 层的 saveBatch 方法（见后文）
}
```

## 3. Delete 操作

### 3.1 根据 ID 删除

```java
@Test
public void testDeleteById() {
    int rows = userMapper.deleteById(1L);
    System.out.println("影响行数：" + rows);
}
```

**输出**：
```sql
==>  Preparing: UPDATE user SET deleted=1 WHERE id=? AND deleted=0
==> Parameters: 1(Long)
<==    Updates: 1
```

**注意**：如果配置了逻辑删除，实际执行的是 UPDATE 而非 DELETE

### 3.2 批量删除

```java
@Test
public void testDeleteBatchIds() {
    List<Long> ids = Arrays.asList(1L, 2L, 3L);
    int rows = userMapper.deleteBatchIds(ids);
    System.out.println("影响行数：" + rows);
}
```

**输出**：
```sql
==>  Preparing: UPDATE user SET deleted=1 WHERE id IN (?,?,?) AND deleted=0
==> Parameters: 1(Long), 2(Long), 3(Long)
<==    Updates: 3
```

### 3.3 条件删除

```java
@Test
public void testDeleteByWrapper() {
    // 删除年龄小于 18 的用户
    int rows = userMapper.delete(
        new QueryWrapper<User>().lt("age", 18)
    );
    System.out.println("影响行数：" + rows);
}
```

**输出**：
```sql
==>  Preparing: UPDATE user SET deleted=1 WHERE (age < ?) AND deleted=0
==> Parameters: 18(Integer)
<==    Updates: 2
```

### 3.4 Map 条件删除

```java
@Test
public void testDeleteByMap() {
    Map<String, Object> columnMap = new HashMap<>();
    columnMap.put("gender", 0);
    columnMap.put("age", 25);
    
    int rows = userMapper.deleteByMap(columnMap);
    System.out.println("影响行数：" + rows);
}
```

**输出**：
```sql
==>  Preparing: UPDATE user SET deleted=1 WHERE gender = ? AND age = ? AND deleted=0
==> Parameters: 0(Integer), 25(Integer)
<==    Updates: 1
```

## 4. Update 操作

### 4.1 根据 ID 更新

```java
@Test
public void testUpdateById() {
    User user = new User();
    user.setId(1L);
    user.setUsername("新名字");
    user.setAge(26);
    
    int rows = userMapper.updateById(user);
    System.out.println("影响行数：" + rows);
}
```

**输出**：
```sql
==>  Preparing: UPDATE user SET username=?, age=? WHERE id=? AND deleted=0
==> Parameters: 新名字(String), 26(Integer), 1(Long)
<==    Updates: 1
```

**特点**：只更新非 null 字段

### 4.2 选择性更新

```java
@Test
public void testSelectiveUpdate() {
    User user = new User();
    user.setId(1L);
    // 只设置要更新的字段
    user.setEmail("newemail@example.com");
    
    int rows = userMapper.updateById(user);
}
```

**输出**：
```sql
==>  Preparing: UPDATE user SET email=? WHERE id=? AND deleted=0
==> Parameters: newemail@example.com(String), 1(Long)
```

### 4.3 条件更新

```java
@Test
public void testUpdateByWrapper() {
    // 将所有 25 岁用户的年龄改为 26
    User user = new User();
    user.setAge(26);
    
    int rows = userMapper.update(user, 
        new UpdateWrapper<User>().eq("age", 25)
    );
    System.out.println("影响行数：" + rows);
}
```

**输出**：
```sql
==>  Preparing: UPDATE user SET age=? WHERE (age = ?) AND deleted=0
==> Parameters: 26(Integer), 25(Integer)
<==    Updates: 3
```

### 4.4 LambdaUpdateWrapper

```java
@Test
public void testLambdaUpdate() {
    int rows = userMapper.update(null,
        new LambdaUpdateWrapper<User>()
            .set(User::getAge, 30)
            .set(User::getEmail, "updated@example.com")
            .eq(User::getUsername, "张三")
    );
}
```

**优势**：类型安全，避免字段名写错

## 5. Select 操作

### 5.1 根据 ID 查询

```java
@Test
public void testSelectById() {
    User user = userMapper.selectById(1L);
    System.out.println(user);
}
```

**返回 null**：如果记录不存在或已被逻辑删除

### 5.2 批量查询

```java
@Test
public void testSelectBatchIds() {
    List<Long> ids = Arrays.asList(1L, 2L, 3L);
    List<User> users = userMapper.selectBatchIds(ids);
    users.forEach(System.out::println);
}
```

**输出**：
```sql
==>  Preparing: SELECT id,username,email,age,gender,deleted,create_time,update_time 
                FROM user WHERE id IN (?,?,?) AND deleted=0
==> Parameters: 1(Long), 2(Long), 3(Long)
<==    Columns: id, username, email, age, gender, ...
<==        Row: 1, 张三, zhangsan@example.com, 25, 1, ...
<==        Row: 2, 李四, lisi@example.com, 28, 1, ...
<==        Row: 3, 王五, wangwu@example.com, 22, 2, ...
<==      Total: 3
```

### 5.3 Map 条件查询

```java
@Test
public void testSelectByMap() {
    Map<String, Object> columnMap = new HashMap<>();
    columnMap.put("age", 25);
    columnMap.put("gender", 1);
    
    List<User> users = userMapper.selectByMap(columnMap);
    users.forEach(System.out::println);
}
```

**输出**：
```sql
==>  Preparing: SELECT ... FROM user WHERE age = ? AND gender = ? AND deleted=0
==> Parameters: 25(Integer), 1(Integer)
```

### 5.4 条件查询（Wrapper）

```java
@Test
public void testSelectByWrapper() {
    // 查询年龄大于 20 的男性用户
    List<User> users = userMapper.selectList(
        new QueryWrapper<User>()
            .gt("age", 20)
            .eq("gender", 1)
            .orderByDesc("age")
    );
    users.forEach(System.out::println);
}
```

**输出**：
```sql
==>  Preparing: SELECT ... FROM user WHERE (age > ? AND gender = ?) AND deleted=0 ORDER BY age DESC
==> Parameters: 20(Integer), 1(Integer)
```

### 5.5 查询单个对象

```java
@Test
public void testSelectOne() {
    User user = userMapper.selectOne(
        new QueryWrapper<User>().eq("username", "张三")
    );
    System.out.println(user);
}
```

**注意**：如果查询结果多于一条，会抛出异常

### 5.6 统计数量

```java
@Test
public void testSelectCount() {
    Integer count = userMapper.selectCount(
        new QueryWrapper<User>().gt("age", 25)
    );
    System.out.println("数量：" + count);
}
```

**输出**：
```sql
==>  Preparing: SELECT COUNT(*) FROM user WHERE (age > ?) AND deleted=0
==> Parameters: 25(Integer)
<==    Columns: COUNT(*)
<==        Row: 2
```

### 5.7 查询部分字段

```java
@Test
public void testSelectMaps() {
    List<Map<String, Object>> maps = userMapper.selectMaps(
        new QueryWrapper<User>()
            .select("id", "username", "age")  // 只查询指定字段
            .gt("age", 20)
    );
    maps.forEach(System.out::println);
}
```

**输出**：
```
{id=1, username=张三, age=25}
{id=2, username=李四, age=28}
```

**优势**：减少数据传输量，提高性能

### 5.8 查询第一个字段

```java
@Test
public void testSelectObjs() {
    List<Object> objs = userMapper.selectObjs(
        new QueryWrapper<User>()
            .select("username")
            .gt("age", 20)
    );
    objs.forEach(System.out::println);
}
```

**输出**：
```
张三
李四
王五
```

## 6. IService 服务层封装

### 6.1 什么是 IService？

`IService` 是 MP 提供的服务层接口，在 `BaseMapper` 基础上进一步封装，提供更便捷的方法。

### 6.2 创建 Service

#### 接口

```java
package com.example.service;

import com.baomidou.mybatisplus.extension.service.IService;
import com.example.entity.User;

public interface UserService extends IService<User> {
    // 可以添加自定义方法
}
```

#### 实现类

```java
package com.example.service.impl;

import com.baomidou.mybatisplus.extension.service.impl.ServiceImpl;
import com.example.entity.User;
import com.example.mapper.UserMapper;
import com.example.service.UserService;
import org.springframework.stereotype.Service;

@Service
public class UserServiceImpl extends ServiceImpl<UserMapper, User> 
                              implements UserService {
    // 无需编写任何代码，已拥有所有 CRUD 方法
}
```

### 6.3 IService 常用方法

```java
@Service
public class UserServiceDemo {
    
    @Autowired
    private UserService userService;
    
    /**
     * 保存或更新
     */
    public void testSaveOrUpdate() {
        User user = new User();
        user.setUsername("测试");
        
        // ID 为空则插入，否则更新
        userService.saveOrUpdate(user);
    }
    
    /**
     * 批量保存
     */
    public void testSaveBatch() {
        List<User> users = new ArrayList<>();
        // ... 添加数据
        
        // 批量插入（默认每批 1000 条）
        userService.saveBatch(users);
        
        // 指定批次大小
        userService.saveBatch(users, 500);
    }
    
    /**
     * 批量更新
     */
    public void testUpdateBatchById() {
        List<User> users = new ArrayList<>();
        // ... 添加数据
        
        userService.updateBatchById(users);
    }
    
    /**
     * 链式查询
     */
    public void testChainQuery() {
        List<User> users = userService.lambdaQuery()
            .gt(User::getAge, 20)
            .eq(User::getGender, 1)
            .orderByDesc(User::getAge)
            .list();
    }
    
    /**
     * 分页查询
     */
    public void testPage() {
        Page<User> page = new Page<>(1, 10);
        Page<User> result = userService.page(page,
            new QueryWrapper<User>().gt("age", 20)
        );
        
        System.out.println("总数：" + result.getTotal());
        System.out.println("当前页数据：" + result.getRecords());
    }
}
```

### 6.4 IService vs BaseMapper

| 特性 | BaseMapper | IService |
|------|-----------|----------|
| **位置** | Mapper 层 | Service 层 |
| **批量操作** | ❌ 无 | ✅ saveBatch、updateBatchById |
| **链式调用** | ❌ 无 | ✅ lambdaQuery、lambdaUpdate |
| **保存或更新** | ❌ 无 | ✅ saveOrUpdate |
| **事务管理** | 需手动 | 可在 Service 层统一处理 |

**建议**：Service 层优先使用 IService

## 8. 常见问题

### 8.1 主键未回填

**问题**：insert 后 getId() 为 null

**原因**：主键策略配置错误

**解决**：
```java
// 确保配置了正确的主键策略
@TableId(type = IdType.AUTO)  // 或 ASSIGN_ID
private Long id;
```

### 8.2 更新无效

**问题**：updateById 返回 0

**原因**：
* ID 不存在
* 所有字段都是 null
* 记录已被逻辑删除

**解决**：检查数据和日志

### 8.3 查询结果为空

**问题**：selectList 返回空列表

**原因**：
* 条件过于严格
* 逻辑删除字段问题

**解决**：打印 SQL 日志排查

## 9. 总结

### 9.1 核心知识点

✅ **BaseMapper**：17 个方法覆盖所有 CRUD  
✅ **Insert**：主键策略、自动回填  
✅ **Delete**：逻辑删除、批量删除  
✅ **Update**：选择性更新、条件更新  
✅ **Select**：多种查询方式、部分字段查询  
✅ **IService**：服务层封装、批量操作、链式调用  

### 9.2 最佳实践

📌 优先使用 IService 而非直接使用 BaseMapper  
📌 批量操作使用 saveBatch/updateBatchById  
📌 复杂查询使用 LambdaWrapper 保证类型安全  
📌 合理配置主键策略（分布式用 ASSIGN_ID）  
📌 开启逻辑删除保护数据
