## 1. 插件机制概述

### 1.1 MybatisPlusInterceptor

MP 使用 `MybatisPlusInterceptor` 管理所有插件。

```java
@Bean
public MybatisPlusInterceptor mybatisPlusInterceptor() {
    MybatisPlusInterceptor interceptor = new MybatisPlusInterceptor();
    
    // 添加各种插件
    interceptor.addInnerInterceptor(new PaginationInnerInterceptor());
    interceptor.addInnerInterceptor(new OptimisticLockerInnerInterceptor());
    
    return interceptor;
}
```

## 3. 性能分析插件

### 3.1 作用

监控 SQL 执行时间，超过阈值则警告或阻断。

### 3.2 配置

```java
@Bean
public MybatisPlusInterceptor mybatisPlusInterceptor() {
    MybatisPlusInterceptor interceptor = new MybatisPlusInterceptor();
    
    // 性能分析插件
    PerformanceAnalysisInterceptor performanceInterceptor = 
        new PerformanceAnalysisInterceptor();
    
    // 设置最大执行时间（毫秒）
    performanceInterceptor.setMaxTime(1000L);
    
    // 超过时间是否格式化 SQL
    performanceInterceptor.setFormat(true);
    
    interceptor.addInnerInterceptor(performanceInterceptor);
    
    return interceptor;
}
```

### 3.3 使用效果

```sql
-- 慢 SQL 警告
[WARN] SQL execution time: 1500ms > maxTime: 1000ms
SELECT * FROM user WHERE age > 20
```

## 4. 非法 SQL 阻断插件

### 4.1 作用

阻止全表更新、删除等危险操作。

### 4.2 配置

```java
@Bean
public MybatisPlusInterceptor mybatisPlusInterceptor() {
    MybatisPlusInterceptor interceptor = new MybatisPlusInterceptor();
    
    // 非法 SQL 阻断插件
    IllegalSQLInnerInterceptor illegalSQLInterceptor = 
        new IllegalSQLInnerInterceptor();
    
    interceptor.addInnerInterceptor(illegalSQLInterceptor);
    
    return interceptor;
}
```

### 4.3 阻断示例

```java
@Test
public void testBlockFullUpdate() {
    // 尝试全表更新
    User user = new User();
    user.setAge(30);
    
    // ❌ 抛出异常：禁止全表更新
    userMapper.update(user, null);
}
```

**异常信息**：
```
com.baomidou.mybatisplus.core.exceptions.MybatisPlusException: 
Prohibition of full table updates
```

### 4.4 允许的操作

```java
// ✅ 带条件的更新
userMapper.update(user, 
    new LambdaUpdateWrapper<User>().eq(User::getStatus, 0));

// ✅ 根据 ID 更新
userMapper.updateById(user);
```

## 5. 自定义插件

### 5.1 实现 InnerInterceptor 接口

```java
package com.example.interceptor;

import com.baomidou.mybatisplus.extension.plugins.inner.InnerInterceptor;
import org.apache.ibatis.executor.Executor;
import org.apache.ibatis.mapping.BoundSql;
import org.apache.ibatis.mapping.MappedStatement;
import org.apache.ibatis.session.ResultHandler;
import org.apache.ibatis.session.RowBounds;

import java.sql.SQLException;
import java.util.List;

/**
 * 自定义数据权限插件
 */
public class DataPermissionInterceptor implements InnerInterceptor {
    
    @Override
    public void beforeQuery(Executor executor, MappedStatement ms, 
                           Object parameter, RowBounds rowBounds, 
                           ResultHandler resultHandler, BoundSql boundSql) 
                           throws SQLException {
        
        // 获取 SQL
        String sql = boundSql.getSql();
        
        // 判断是否需要添加权限条件
        if (needDataPermission(sql)) {
            // 修改 SQL，添加权限条件
            String newSql = addPermissionCondition(sql);
            
            // 反射修改 SQL
            ReflectUtils.setFieldValue(boundSql, "sql", newSql);
        }
    }
    
    private boolean needDataPermission(String sql) {
        // 只处理 SELECT 语句
        return sql.trim().toUpperCase().startsWith("SELECT");
    }
    
    private String addPermissionCondition(String sql) {
        Long deptId = UserContext.getCurrentDeptId();
        
        // 在 WHERE 后添加条件
        if (sql.contains("WHERE")) {
            return sql + " AND dept_id = " + deptId;
        } else {
            return sql + " WHERE dept_id = " + deptId;
        }
    }
}
```

### 5.2 注册插件

```java
@Bean
public MybatisPlusInterceptor mybatisPlusInterceptor() {
    MybatisPlusInterceptor interceptor = new MybatisPlusInterceptor();
    
    // 添加自定义插件
    interceptor.addInnerInterceptor(new DataPermissionInterceptor());
    
    return interceptor;
}
```

## 6. 拦截器链配置

### 6.1 推荐顺序

```java
@Bean
public MybatisPlusInterceptor mybatisPlusInterceptor() {
    MybatisPlusInterceptor interceptor = new MybatisPlusInterceptor();
    
    // 1. 多租户插件（最先执行）
    interceptor.addInnerInterceptor(new TenantLineInnerInterceptor(handler));
    
    // 2. 动态表名插件
    interceptor.addInnerInterceptor(new DynamicTableNameInnerInterceptor(handler));
    
    // 3. 分页插件
    interceptor.addInnerInterceptor(new PaginationInnerInterceptor(DbType.MYSQL));
    
    // 4. 乐观锁插件
    interceptor.addInnerInterceptor(new OptimisticLockerInnerInterceptor());
    
    // 5. 性能分析插件（开发环境）
    if (isDevEnvironment()) {
        interceptor.addInnerInterceptor(new PerformanceAnalysisInterceptor());
    }
    
    // 6. 非法 SQL 阻断插件（最后执行）
    interceptor.addInnerInterceptor(new IllegalSQLInnerInterceptor());
    
    return interceptor;
}
```

### 6.2 注意事项

* 多租户插件应放在最前面
* 分页插件应在查询之前
* 性能分析插件只在开发环境启用
* 非法 SQL 阻断插件应放在最后

## 7. 实战：审计日志插件

### 7.1 需求

记录所有 SQL 操作到审计日志表。

### 7.2 实现

```java
@Component
public class AuditLogInterceptor implements InnerInterceptor {
    
    @Autowired
    private AuditLogMapper auditLogMapper;
    
    @Override
    public void afterQuery(Executor executor, MappedStatement ms, 
                          Object parameter, RowBounds rowBounds, 
                          ResultHandler resultHandler, BoundSql boundSql) {
        
        // 记录查询日志
        AuditLog log = new AuditLog();
        log.setOperationType("QUERY");
        log.setSql(boundSql.getSql());
        log.setExecuteTime(System.currentTimeMillis());
        log.setUserId(UserContext.getCurrentUserId());
        
        auditLogMapper.insert(log);
    }
    
    @Override
    public void beforeUpdate(Executor executor, MappedStatement ms, 
                            Object parameter) throws SQLException {
        
        // 记录更新日志
        AuditLog log = new AuditLog();
        log.setOperationType("UPDATE");
        log.setSql(ms.getBoundSql(parameter).getSql());
        log.setExecuteTime(System.currentTimeMillis());
        log.setUserId(UserContext.getCurrentUserId());
        
        auditLogMapper.insert(log);
    }
}
```

## 8. 常见问题

### 8.1 插件冲突

**问题**：多个插件同时修改 SQL，导致错误

**解决**：调整插件顺序

### 8.2 性能影响

**问题**：插件过多影响性能

**优化**：
* 生产环境禁用性能分析插件
* 减少不必要的插件
* 优化插件逻辑

### 8.3 SQL 解析失败

**问题**：复杂 SQL 无法正确解析

**解决**：
* 简化 SQL
* 使用原生 MyBatis XML
* 自定义解析逻辑
