## 1. 分页插件配置

### 1.1 添加配置类

```java
package com.example.config;

import com.baomidou.mybatisplus.annotation.DbType;
import com.baomidou.mybatisplus.extension.plugins.MybatisPlusInterceptor;
import com.baomidou.mybatisplus.extension.plugins.inner.PaginationInnerInterceptor;
import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;

@Configuration
public class MybatisPlusConfig {
    
    /**
     * 分页插件配置
     */
    @Bean
    public MybatisPlusInterceptor mybatisPlusInterceptor() {
        MybatisPlusInterceptor interceptor = new MybatisPlusInterceptor();
        
        // 添加分页插件
        PaginationInnerInterceptor paginationInterceptor = 
            new PaginationInnerInterceptor(DbType.MYSQL);
        
        // 设置最大单页限制数量，默认 500 条，-1 不受限制
        paginationInterceptor.setMaxLimit(500L);
        
        // 溢出总页数后是否进行处理（默认不处理）
        paginationInterceptor.setOverflow(false);
        
        interceptor.addInnerInterceptor(paginationInterceptor);
        
        return interceptor;
    }
}
```

### 1.2 多数据库支持

```java
@Bean
public MybatisPlusInterceptor mybatisPlusInterceptor() {
    MybatisPlusInterceptor interceptor = new MybatisPlusInterceptor();
```

## 2. Page 对象使用

### 2.1 基本分页查询

```java
@Test
public void testBasicPage() {
    // 第 1 页，每页 10 条
    Page<User> page = new Page<>(1, 10);
    
    // 执行分页查询
    Page<User> result = userMapper.selectPage(page, null);
    
    System.out.println("总记录数：" + result.getTotal());
    System.out.println("总页数：" + result.getPages());
    System.out.println("当前页：" + result.getCurrent());
    System.out.println("每页大小：" + result.getSize());
    System.out.println("当前页数据：" + result.getRecords());
}
```

**输出**：
```sql
==>  Preparing: SELECT COUNT(*) FROM user WHERE deleted=0
==> Parameters: 
<==    Columns: COUNT(*)
<==        Row: 100

==>  Preparing: SELECT id,username,email,age,... FROM user WHERE deleted=0 LIMIT ?
==> Parameters: 10(Long)
<==      Total: 10

总记录数：100
总页数：10
当前页：1
每页大小：10
当前页数据：[User(...), User(...), ...]
```

### 2.2 带条件的分页查询

```java
@Test
public void testPageWithCondition() {
    Page<User> page = new Page<>(1, 10);
    
    LambdaQueryWrapper<User> wrapper = new LambdaQueryWrapper<>();
    wrapper.gt(User::getAge, 20)
           .orderByDesc(User::getCreateTime);
    
    Page<User> result = userMapper.selectPage(page, wrapper);
    
    System.out.println("符合条件的总数：" + result.getTotal());
    result.getRecords().forEach(System.out::println);
}
```

**输出 SQL**：
```sql
-- 先查询总数
SELECT COUNT(*) FROM user WHERE age > 20 AND deleted=0

-- 再查询数据
SELECT id,username,... FROM user 
WHERE age > 20 AND deleted=0 
ORDER BY create_time DESC 
LIMIT 10
```

### 2.3 IService 分页

```java
@Service
public class UserServiceDemo {
    
    @Autowired
    private UserService userService;
    
    public Page<User> getUserPage(int pageNum, int pageSize) {
        Page<User> page = new Page<>(pageNum, pageSize);
        
        return userService.page(page, 
            new LambdaQueryWrapper<User>()
                .gt(User::getAge, 20)
        );
    }
}
```

## 3. IPage 返回结果

### 3.1 IPage 接口

`IPage<T>` 是分页结果的接口，常用实现类是 `Page<T>`。

**主要方法**：

| 方法 | 说明 |
|------|------|
| `getTotal()` | 总记录数 |
| `getPages()` | 总页数 |
| `getCurrent()` | 当前页 |
| `getSize()` | 每页大小 |
| `getRecords()` | 数据列表 |
| `hasNext()` | 是否有下一页 |
| `hasPrevious()` | 是否有上一页 |

### 3.2 Controller 层封装

```java
@RestController
@RequestMapping("/api/users")
public class UserController {
    
    @Autowired
    private UserService userService;
    
    @GetMapping
    public Result<IPage<User>> list(
            @RequestParam(defaultValue = "1") int pageNum,
            @RequestParam(defaultValue = "10") int pageSize,
            @RequestParam(required = false) String username) {
        
        Page<User> page = new Page<>(pageNum, pageSize);
        
        LambdaQueryWrapper<User> wrapper = new LambdaQueryWrapper<>();
        wrapper.like(StringUtils.isNotBlank(username), 
                     User::getUsername, username)
               .orderByDesc(User::getCreateTime);
        
        IPage<User> result = userService.page(page, wrapper);
        
        return Result.success(result);
    }
}
```

**返回 JSON**：
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "records": [...],
    "total": 100,
    "size": 10,
    "current": 1,
    "pages": 10
  }
}
```

## 4. 自定义分页查询

```java

     * 自定义分页查询

                               @Param("minAge") Integer minAge,
                               @Param("maxAge") Integer maxAge);

```

### 4.2 XML 映射

```xml
<!-- UserMapper.xml -->
<select id="selectUserPage" resultType="com.example.entity.User">
    SELECT id, username, email, age, gender, create_time
    FROM user
    <where>
        deleted = 0
        <if test="minAge != null">
            AND age &gt;= #{minAge}
        </if>
        <if test="maxAge != null">
            AND age &lt;= #{maxAge}
        </if>
    </where>
    ORDER BY create_time DESC
</select>
```

## 5. 分页性能优化

### 5.1 深分页问题

**问题**：当页码很大时，性能急剧下降

```sql
-- 第 10000 页，每页 10 条
SELECT * FROM user LIMIT 100000, 10;
-- 需要扫描前 100010 条记录，然后丢弃前 100000 条
```

### 5.2 优化方案一：子查询优化

```xml
<select id="selectUserPageOptimized" resultType="User">
    SELECT u.*
    FROM user u
    INNER JOIN (
        SELECT id FROM user
        WHERE deleted = 0
        ORDER BY create_time DESC
        LIMIT #{offset}, #{limit}
    ) tmp ON u.id = tmp.id
</select>
```

**原理**：子查询只查 ID，减少数据传输

### 5.3 优化方案二：游标分页（推荐）

```java
@GetMapping
public List<User> cursorPage(
        @RequestParam(required = false) Long lastId,
        @RequestParam(defaultValue = "10") int size) {
    
    LambdaQueryWrapper<User> wrapper = new LambdaQueryWrapper<>();
    
    if (lastId != null) {
        wrapper.lt(User::getId, lastId);  // ID < lastId
    }
    
    wrapper.orderByDesc(User::getId)
           .last("LIMIT " + size);
    
    return userMapper.selectList(wrapper);
}
```

**优势**：
* 利用主键索引
* 性能稳定
* 适合无限滚动场景

**前端传参**：
```javascript
// 第一页
GET /api/users?size=10

// 第二页（传入最后一条记录的 ID）
GET /api/users?lastId=100&size=10
```

### 5.4 优化方案三：禁止 count 查询

如果不需要总记录数，可以跳过 count 查询：

```java
@Test
public void testNoCount() {
    Page<User> page = new Page<>(1, 10, false);  // 第三个参数：是否查询总数
    
    Page<User> result = userMapper.selectPage(page, null);
    
    System.out.println("总数：" + result.getTotal());  // 0
    System.out.println("数据：" + result.getRecords());
}
```

**优势**：减少一次 COUNT 查询，提高性能

### 6.3 XML 映射

```xml
<select id="selectUserOrders" resultType="com.example.vo.UserOrderVO">
    SELECT 
        u.id as userId,
        u.username,
        o.id as orderId,
        o.amount as orderAmount,
        o.create_time as orderTime
    FROM user u
    LEFT JOIN orders o ON u.id = o.user_id
```

## 7. 常见问题

### 7.1 分页无效

**问题**：返回所有数据，未分页

**原因**：
* 未配置分页插件
* 未使用 Page 对象

**解决**：
```java
// ✅ 正确：传入 Page 对象
Page<User> page = new Page<>(1, 10);
userMapper.selectPage(page, wrapper);

// ❌ 错误：直接查询
userMapper.selectList(wrapper);  // 不分页
```

### 7.2 总数不准确

**问题**：totalCount 与实际不符

**原因**：逻辑删除配置问题

**解决**：检查 `@TableLogic` 注解和全局配置

### 7.3 内存溢出

**问题**：一次性加载大量数据

**解决**：
* 限制每页大小（setMaxLimit）
* 使用流式查询
* 分批处理
