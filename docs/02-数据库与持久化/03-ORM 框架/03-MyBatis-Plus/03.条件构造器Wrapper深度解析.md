## 1. Wrapper 概述

### 1.1 什么是 Wrapper？

Wrapper 是 MP 的条件构造器，用于生成 SQL 的 WHERE 条件部分。

**优势**：
* ✅ 无需手写 SQL
* ✅ 支持链式调用
* ✅ 类型安全（Lambda 方式）
* ✅ 动态条件构建

### 1.2 Wrapper 分类

| 类名 | 作用 | 使用场景 |
|------|------|---------|
| **QueryWrapper** | 查询条件 | SELECT 查询 |
| **LambdaQueryWrapper** | 查询条件（Lambda） | 类型安全的查询 |
| **UpdateWrapper** | 更新条件 | UPDATE 操作 |
| **LambdaUpdateWrapper** | 更新条件（Lambda） | 类型安全的更新 |

### 1.3 基本用法

```java
// QueryWrapper 方式
QueryWrapper<User> wrapper = new QueryWrapper<>();
wrapper.eq("username", "张三");
List<User> users = userMapper.selectList(wrapper);

// LambdaQueryWrapper 方式（推荐）
LambdaQueryWrapper<User> wrapper = new LambdaQueryWrapper<>();
wrapper.eq(User::getUsername, "张三");
List<User> users = userMapper.selectList(wrapper);
```

## 2. 常用条件方法

### 2.1 等值条件

#### eq - 等于

```java
@Test
public void testEq() {
    // WHERE username = '张三'
    List<User> users = userMapper.selectList(
        new LambdaQueryWrapper<User>()
            .eq(User::getUsername, "张三")
    );
}
```

#### ne - 不等于

```java
@Test
public void testNe() {
    // WHERE age <> 25
    List<User> users = userMapper.selectList(
        new LambdaQueryWrapper<User>()
            .ne(User::getAge, 25)
    );
}
```

### 2.2 比较条件

#### gt / ge - 大于 / 大于等于

```java
@Test
public void testGt() {
    // WHERE age > 20
    List<User> users = userMapper.selectList(
        new LambdaQueryWrapper<User>()
            .gt(User::getAge, 20)
    );
    
    // WHERE age >= 20
    List<User> users2 = userMapper.selectList(
        new LambdaQueryWrapper<User>()
            .ge(User::getAge, 20)
    );
}
```

#### lt / le - 小于 / 小于等于

```java
@Test
public void testLt() {
    // WHERE age < 30
    List<User> users = userMapper.selectList(
        new LambdaQueryWrapper<User>()
            .lt(User::getAge, 30)
    );
    
    // WHERE age <= 30
    List<User> users2 = userMapper.selectList(
        new LambdaQueryWrapper<User>()
            .le(User::getAge, 30)
    );
}
```

#### between - 介于

```java
@Test
public void testBetween() {
    // WHERE age BETWEEN 20 AND 30
    List<User> users = userMapper.selectList(
        new LambdaQueryWrapper<User>()
            .between(User::getAge, 20, 30)
    );
}
```

#### like - 包含

```java
@Test
public void testLike() {

    List<User> users = userMapper.selectList(
        new LambdaQueryWrapper<User>()
            .like(User::getUsername, "张")

```

#### likeLeft - 左模糊

```java
@Test

        new LambdaQueryWrapper<User>()

    );
```

#### likeRight - 右模糊

```java
@Test
```

**性能提示**：右模糊可以利用索引，左模糊和全模糊无法利用索引

### 2.4 NULL 判断

#### isNull / isNotNull

```java
@Test
public void testNull() {
    // WHERE email IS NULL
    List<User> users1 = userMapper.selectList(
        new LambdaQueryWrapper<User>()
            .isNull(User::getEmail)
    );
    
    // WHERE email IS NOT NULL
    List<User> users2 = userMapper.selectList(
        new LambdaQueryWrapper<User>()
            .isNotNull(User::getEmail)
    );
}
```

### 2.5 IN 查询

#### in / notIn

```java

        new LambdaQueryWrapper<User>()

    );
    
    // WHERE age NOT IN (20, 25, 30)

        new LambdaQueryWrapper<User>()
            .notIn(User::getAge, 20, 25, 30)
    );

```

#### in 集合

```java
@Test
public void testInCollection() {

    // WHERE age IN (20, 25, 30)
    List<User> users = userMapper.selectList(
        new LambdaQueryWrapper<User>()
            .in(User::getAge, ages)
    );
}
```

## 3. 逻辑组合

### 3.1 and - 并且

```java
@Test
public void testAnd() {
    // WHERE age > 20 AND gender = 1
    List<User> users = userMapper.selectList(
        new LambdaQueryWrapper<User>()
            .gt(User::getAge, 20)
            .eq(User::getGender, 1)
    );
}
```

**注意**：默认就是 AND 连接，无需显式调用 and()

### 3.2 or - 或者

```java
@Test
public void testOr() {
    // WHERE age > 30 OR gender = 2
    List<User> users = userMapper.selectList(
        new LambdaQueryWrapper<User>()
            .gt(User::getAge, 30)
            .or()
            .eq(User::getGender, 2)
    );
}
```

### 3.3 嵌套条件

```java
@Test
public void testNested() {
    // WHERE age > 20 AND (gender = 1 OR email IS NOT NULL)
    List<User> users = userMapper.selectList(
        new LambdaQueryWrapper<User>()
            .gt(User::getAge, 20)
            .and(wrapper -> 
                wrapper.eq(User::getGender, 1)
                       .or()
                       .isNotNull(User::getEmail)
            )
    );
}
```

**输出 SQL**：
```sql
WHERE age > 20 AND (gender = 1 OR email IS NOT NULL)
```

### 3.4 复杂嵌套示例

```java
@Test
public void testComplexNested() {
    // WHERE (age BETWEEN 20 AND 30 AND gender = 1) 
    //    OR (username LIKE '%张%' AND email IS NOT NULL)
    List<User> users = userMapper.selectList(
        new LambdaQueryWrapper<User>()
            .and(w1 -> w1.between(User::getAge, 20, 30)
                         .eq(User::getGender, 1))
            .or(w2 -> w2.like(User::getUsername, "张")
                        .isNotNull(User::getEmail))
    );
}
```

## 4. 排序与分组

### 4.1 orderByAsc / orderByDesc

```java
@Test
public void testOrderBy() {
    // ORDER BY age ASC
    List<User> users1 = userMapper.selectList(
        new LambdaQueryWrapper<User>()

        new LambdaQueryWrapper<User>()
            .orderByDesc(User::getAge)
    );

        new LambdaQueryWrapper<User>()
            .orderByDesc(User::getAge)
            .orderByAsc(User::getCreateTime)
    );
}
```

### 4.2 groupBy + having

```java
@Test
public void testGroupBy() {
    // GROUP BY gender HAVING COUNT(*) > 5
    List<Map<String, Object>> result = userMapper.selectMaps(
        new QueryWrapper<User>()
            .select("gender, COUNT(*) as count")
            .groupBy("gender")
            .having("COUNT(*) > {0}", 5)
    );
}
```

## 5. 字段选择

### 5.1 select - 指定字段

```java
@Test
public void testSelect() {

        new LambdaQueryWrapper<User>()
            .select(User::getId, User::getUsername, User::getAge)
```

**优势**：减少数据传输量，提高性能

### 5.2 排除字段

```java
public void testSelectExclude() {
    // 查询所有字段，排除 email 和 create_time
    List<User> users = userMapper.selectList(
        new QueryWrapper<User>()
            .select(User.class, info -> {
                String columnName = info.getColumn();
                return !"email".equals(columnName) && 
                       !"create_time".equals(columnName);
            })
    );
}
```

## 6. 动态条件构建

### 6.1 条件判断

```java
@Test
public void testDynamicCondition() {
    String username = "张";
    Integer minAge = 20;
    Integer maxAge = 30;
    
    LambdaQueryWrapper<User> wrapper = new LambdaQueryWrapper<>();
    
    // 只有 username 不为空时才添加条件
    if (StringUtils.isNotBlank(username)) {
        wrapper.like(User::getUsername, username);
    }
    
    // 只有 minAge 不为 null 时才添加条件
    if (minAge != null) {
        wrapper.ge(User::getAge, minAge);
    }
    
    // 只有 maxAge 不为 null 时才添加条件
    if (maxAge != null) {
        wrapper.le(User::getAge, maxAge);
    }
    
    List<User> users = userMapper.selectList(wrapper);
}
```

### 6.2 condition 参数（推荐）

MP 提供了更优雅的方式：

```java
@Test
public void testConditionParam() {
    String username = "张";
    Integer minAge = 20;
    Integer maxAge = 30;
    
    List<User> users = userMapper.selectList(
        new LambdaQueryWrapper<User>()
            .like(StringUtils.isNotBlank(username), 
                  User::getUsername, username)
            .ge(minAge != null, User::getAge, minAge)
            .le(maxAge != null, User::getAge, maxAge)
    );
}
```

**语法**：`方法名(boolean condition, 参数...)`

* `condition` 为 true 时才添加该条件
* 代码更简洁，可读性更好

### 6.3 实战：搜索功能

```java
@Service
public class UserService {
    
    @Autowired
    private UserMapper userMapper;
    
    public List<User> search(String username, Integer minAge, 
                            Integer maxAge, Integer gender) {
        return userMapper.selectList(
            new LambdaQueryWrapper<User>()
                .like(StringUtils.isNotBlank(username), 
                      User::getUsername, username)
                .ge(minAge != null, User::getAge, minAge)
                .le(maxAge != null, User::getAge, maxAge)
                .eq(gender != null, User::getGender, gender)
                .orderByDesc(User::getCreateTime)
        );
    }
}
```

## 7. UpdateWrapper 详解

### 7.1 set - 设置字段值

```java
@Test
public void testUpdateWrapper() {
    // UPDATE user SET age = 30, email = 'new@example.com' 
    // WHERE username = '张三'
    int rows = userMapper.update(null,
        new LambdaUpdateWrapper<User>()
            .set(User::getAge, 30)
            .set(User::getEmail, "new@example.com")
            .eq(User::getUsername, "张三")
    );
}
```

### 7.2 setSql - 自定义 SQL

```java
@Test
public void testSetSql() {
    // UPDATE user SET age = age + 1 WHERE id = 1
    int rows = userMapper.update(null,
        new LambdaUpdateWrapper<User>()
            .setSql("age = age + 1")
            .eq(User::getId, 1L)
    );
}
```

**应用场景**：计数器、累加等操作

### 7.3 批量更新

```java
@Test
public void testBatchUpdate() {
    // 将所有 25 岁用户的年龄改为 26
    int rows = userMapper.update(null,
        new LambdaUpdateWrapper<User>()
            .set(User::getAge, 26)
            .eq(User::getAge, 25)
    );
}
```

## 8. 高级用法

### 8.1 apply - 拼接 SQL

```java
@Test
public void testApply() {
    // WHERE DATE(create_time) = '2024-01-01'
    List<User> users = userMapper.selectList(
        new LambdaQueryWrapper<User>()
            .apply("DATE(create_time) = {0}", "2024-01-01")
    );
}
```

**注意**：{0} 会自动转义，防止 SQL 注入

### 8.2 exists - 子查询

```java
@Test
public void testExists() {
    // WHERE EXISTS (SELECT 1 FROM order WHERE order.user_id = user.id)
    List<User> users = userMapper.selectList(
        new LambdaQueryWrapper<User>()
            .exists("SELECT 1 FROM orders WHERE orders.user_id = user.id")
    );
}
```

### 8.3 nested - 嵌套查询

```java
@Test
public void testNestedQuery() {
    // WHERE age > (SELECT AVG(age) FROM user)
    List<User> users = userMapper.selectList(
        new LambdaQueryWrapper<User>()
            .gt(User::getAge, 
                new QueryWrapper<User>().select("AVG(age)").last("FROM user"))
    );
}
```

## 9. 常见陷阱

### 9.1 空字符串问题

```java
// ❌ 错误：空字符串也会作为条件
wrapper.eq(User::getUsername, "");

// ✅ 正确：先判断
wrapper.eq(StringUtils.isNotBlank(username), User::getUsername, username);
```

### 9.2 OR 优先级问题

```java
// ❌ 错误：WHERE age > 20 AND gender = 1 OR status = 1
// 实际执行：WHERE (age > 20 AND gender = 1) OR status = 1
wrapper.gt(User::getAge, 20)
       .eq(User::getGender, 1)
       .or()
       .eq(User::getStatus, 1);

// ✅ 正确：使用嵌套
wrapper.gt(User::getAge, 20)
       .and(w -> w.eq(User::getGender, 1)
                  .or()
                  .eq(User::getStatus, 1));
```

### 9.3 Lambda 表达式限制

```java
// ❌ 错误：不能使用方法引用
wrapper.eq(User::getUsername(), "张三");

// ✅ 正确：使用字段引用
wrapper.eq(User::getUsername, "张三");
```

## 11. 总结

### 11.1 核心知识点

✅ **Wrapper 分类**：QueryWrapper、LambdaQueryWrapper、UpdateWrapper  
✅ **常用条件**：eq、ne、gt、lt、like、in、between  
✅ **逻辑组合**：and、or、nested  
✅ **排序分组**：orderBy、groupBy、having  
✅ **动态条件**：condition 参数  
✅ **UpdateWrapper**：set、setSql  

### 11.2 最佳实践

📌 优先使用 LambdaQueryWrapper（类型安全）  
📌 动态条件使用 condition 参数  
📌 复杂条件使用 nested 嵌套  
📌 只查询需要的字段（select）  
📌 注意 OR 的优先级问题
