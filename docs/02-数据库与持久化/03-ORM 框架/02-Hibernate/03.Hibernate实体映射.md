## 1. 实体类基础

### 1.1 @Entity 注解

标记一个类为 JPA 实体，表示该类需要映射到数据库表。

```java
import javax.persistence.Entity;
import javax.persistence.Table;

@Entity
@Table(name = "user", schema = "public")
public class User {
    // ...
}
```

**@Table 属性**：

| 属性 | 说明 | 示例 |
|------|------|------|
| name | 表名 | `@Table(name = "sys_user")` |
| schema | 数据库模式/Schema | `@Table(schema = "public")` |
| catalog | 数据库目录 | `@Table(catalog = "mydb")` |
| uniqueConstraints | 唯一约束 | 见下文 |

**唯一约束示例**：

```java
@Table(
    name = "user",
    uniqueConstraints = {
        @UniqueConstraint(columnNames = {"username"}),
        @UniqueConstraint(columnNames = {"email"})
    }
)
public class User {
    // ...
}
```

### 1.2 实体类要求

1. 必须有 **无参构造函数**（可以是 protected）
2. 不能是 final 类
3. 必须有一个字段标注为 `@Id`
4. 建议使用包装类型（Integer、Long）而非基本类型

```java
@Entity
@Table(name = "user")
public class User {
    
    // ✅ 正确：使用包装类型
    private Integer age;
    
    // ❌ 避免：基本类型，无法表示 null
    // private int age;
    
    // 无参构造函数（Lombok @Data 会自动生成）
    public User() {}
}
```

## 2. 主键映射

### 2.1 @Id 和 @GeneratedValue

```java
@Id
@GeneratedValue(strategy = GenerationType.IDENTITY)
private Long id;
```

### 2.2 主键生成策略

#### （1）IDENTITY（自增）

```java
@Id
@GeneratedValue(strategy = GenerationType.IDENTITY)
private Long id;
```

* **适用**：MySQL、SQL Server
* **特点**：数据库自动生成 ID
* **SQL**：`AUTO_INCREMENT`

#### （2）SEQUENCE（序列）

```java
@Id
@GeneratedValue(strategy = GenerationType.SEQUENCE, generator = "user_seq")
@SequenceGenerator(
    name = "user_seq",
    sequenceName = "seq_user_id",
    initialValue = 1,
    allocationSize = 50
)
private Long id;
```

* **适用**：Oracle、PostgreSQL
* **特点**：使用数据库序列
* **优势**：支持批量插入优化

#### （3）TABLE（表生成）

```java
@Id
@GeneratedValue(strategy = GenerationType.TABLE, generator = "user_gen")
@TableGenerator(
    name = "user_gen",
    table = "hibernate_sequences",
    pkColumnName = "seq_name",
    valueColumnName = "seq_value",
    pkColumnValue = "user_seq",
    initialValue = 1,
    allocationSize = 50
)
private Long id;
```

* **适用**：所有数据库
* **特点**：使用单独的表存储序列值
* **缺点**：性能较差

#### （4）AUTO（自动选择）

```java
@Id
@GeneratedValue(strategy = GenerationType.AUTO)
private Long id;
```

* **特点**：由 Hibernate 根据数据库方言自动选择
* **推荐**：跨数据库应用

### 2.3 复合主键

#### 方式一：@EmbeddedId

**步骤 1**：创建复合主键类

```java
package com.example.entity;

import lombok.Data;
import lombok.EqualsAndHashCode;

import javax.persistence.Embeddable;
import java.io.Serializable;

@Embeddable
@Data
@EqualsAndHashCode
public class OrderItemId implements Serializable {
    
    private static final long serialVersionUID = 1L;
    
    private Long orderId;
    private Long productId;
}
```

**步骤 2**：在实体中使用

```java
package com.example.entity;

import lombok.Data;
import javax.persistence.*;

@Entity
@Table(name = "order_item")
@Data
public class OrderItem {
    
    @EmbeddedId
    private OrderItemId id;
    
    private Integer quantity;
    private Double price;
}
```

#### 方式二：@IdClass

**步骤 1**：创建主键类

```java
package com.example.entity;

import lombok.Data;
import java.io.Serializable;

@Data
public class OrderItemId implements Serializable {
    
    private static final long serialVersionUID = 1L;
    
    private Long orderId;
    private Long productId;
}
```

**步骤 2**：在实体中标注

```java
package com.example.entity;

import lombok.Data;
import javax.persistence.*;

@Entity
@Table(name = "order_item")
@Data
@IdClass(OrderItemId.class)
public class OrderItem {
    
    @Id
    private Long orderId;
    
    @Id
    private Long productId;
    
    private Integer quantity;
    private Double price;
}
```

**两种方式对比**：

| 特性 | @EmbeddedId | @IdClass |
|------|-------------|----------|
| 代码复杂度 | 较简洁 | 较繁琐 |
| 查询语法 | `item.getId().getOrderId()` | `item.getOrderId()` |
| 适用场景 | 推荐大多数情况 | 需要单独访问主键字段 |

## 3. 字段映射

### 3.1 @Column 注解

```java
@Column(
    name = "user_name",           // 列名
    nullable = false,              // 是否允许为空
    unique = true,                 // 是否唯一
    length = 50,                   // 字符串长度
    precision = 10,                // 数字精度
    scale = 2,                     // 小数位数
    insertable = true,             // 是否可插入
    updatable = true,              // 是否可更新
    columnDefinition = "TEXT"      // 列定义
)
private String username;
```

### 3.2 常用类型映射

| Java 类型 | JDBC 类型 | 说明 |
|-----------|-----------|------|
| String | VARCHAR | 字符串 |
| int/Integer | INTEGER | 整数 |
| long/Long | BIGINT | 长整数 |
| double/Double | DOUBLE | 双精度浮点数 |
| boolean/Boolean | BOOLEAN | 布尔值 |
| Date | DATE/TIMESTAMP | 日期时间 |
| LocalDate | DATE | 本地日期 |
| LocalDateTime | TIMESTAMP | 本地日期时间 |
| byte[] | BLOB | 二进制数据 |
| char/Character | CHAR | 单字符 |

### 3.3 特殊类型处理

#### （1）枚举类型

```java
public enum Gender {
    MALE("男"),
    FEMALE("女");
    
    private String description;
    
    Gender(String description) {
        this.description = description;
    }
}

@Entity
public class User {
    
    // 方式一：存储枚举名称（默认）
    @Enumerated(EnumType.STRING)
    private Gender gender;
    
    // 方式二：存储枚举序号
    @Enumerated(EnumType.ORDINAL)
    private Gender gender;
}
```

#### （2）日期时间类型

```java
import java.util.Date;
import java.time.LocalDate;
import java.time.LocalDateTime;

@Entity
public class User {
    
    // Java util.Date
    @Temporal(TemporalType.DATE)         // 只存日期
    private Date birthDate;
    
    @Temporal(TemporalType.TIME)         // 只存时间
    private Date birthTime;
    
    @Temporal(TemporalType.TIMESTAMP)    // 日期+时间
    private Date createTime;
    
    // Java 8 时间 API（推荐）
    private LocalDate birthDate;         // 自动映射为 DATE
    private LocalDateTime createTime;    // 自动映射为 TIMESTAMP
}
```

#### （3）大文本和大对象

```java
@Entity
public class Article {
    
    // 大文本
    @Column(columnDefinition = "TEXT")
    private String content;
    
    // 或者使用 @Lob
    @Lob
    private String content;
    
    // 二进制大对象
    @Lob
    private byte[] attachment;
}
```

#### （4）忽略字段

```java
@Entity
public class User {
    
    private String username;
    
    // 不映射到数据库
    @Transient
    private String confirmPassword;
    
    // 或者使用 transient 关键字
    private transient String tempData;
}
```

## 4. 自定义类型转换器

### 4.1 @Convert 注解

**场景**：将 JSON 字符串转换为 Java 对象

**步骤 1**：创建转换器

```java
package com.example.converter;

import com.fasterxml.jackson.core.JsonProcessingException;
import com.fasterxml.jackson.databind.ObjectMapper;

import javax.persistence.AttributeConverter;
import javax.persistence.Converter;

@Converter(autoApply = false)
public class JsonConverter implements AttributeConverter<Object, String> {
    
    private static final ObjectMapper objectMapper = new ObjectMapper();
    
    @Override
    public String convertToDatabaseColumn(Object attribute) {
        try {
            return objectMapper.writeValueAsString(attribute);
        } catch (JsonProcessingException e) {
            throw new RuntimeException("JSON 序列化失败", e);
        }
    }
    
    @Override
    public Object convertToEntityAttribute(String dbData) {
        try {
            return objectMapper.readValue(dbData, Object.class);
        } catch (JsonProcessingException e) {
            throw new RuntimeException("JSON 反序列化失败", e);
        }
    }
}
```

**步骤 2**：使用转换器

```java
@Entity
public class User {
    
    @Column(columnDefinition = "TEXT")
    @Convert(converter = JsonConverter.class)
    private Map<String, Object> preferences;
}
```

### 4.2 全局自动应用

```java
@Converter(autoApply = true)
public class StringTrimConverter implements AttributeConverter<String, String> {
    
    @Override
    public String convertToDatabaseColumn(String attribute) {
        return attribute == null ? null : attribute.trim();
    }
    
    @Override
    public String convertToEntityAttribute(String dbData) {
        return dbData == null ? null : dbData.trim();
    }
}
```

## 5. 实体继承映射

### 5.1 SINGLE_TABLE（单表策略）

所有子类存储在一张表中，使用鉴别列区分。

```java
@Entity
@Inheritance(strategy = InheritanceType.SINGLE_TABLE)
@DiscriminatorColumn(name = "payment_type", discriminatorType = DiscriminatorType.STRING)
public abstract class Payment {
    
    @Id
    @GeneratedValue
    private Long id;
    
    private BigDecimal amount;
}

@Entity
@DiscriminatorValue("CREDIT_CARD")
public class CreditCardPayment extends Payment {
    private String cardNumber;
    private String expiryDate;
}

@Entity
@DiscriminatorValue("PAYPAL")
public class PayPalPayment extends Payment {
    private String paypalEmail;
}
```

**生成的表结构**：

```sql
CREATE TABLE payment (
    id BIGINT PRIMARY KEY,
    payment_type VARCHAR(31),  -- 鉴别列
    amount DECIMAL(19,2),
    card_number VARCHAR(255),   -- CreditCardPayment 专用
    expiry_date VARCHAR(255),   -- CreditCardPayment 专用
    paypal_email VARCHAR(255)   -- PayPalPayment 专用
);
```

### 5.2 JOINED（ Joined 策略）

每个类一张表，通过 JOIN 关联。

```java
@Entity
@Inheritance(strategy = InheritanceType.JOINED)
public abstract class Payment {
    @Id
    @GeneratedValue
    private Long id;
    private BigDecimal amount;
}

@Entity
@PrimaryKeyJoinColumn(name = "payment_id")
public class CreditCardPayment extends Payment {
    private String cardNumber;
}
```

**生成的表结构**：

```sql
-- 父类表
CREATE TABLE payment (
    id BIGINT PRIMARY KEY,
    amount DECIMAL(19,2)
);

-- 子类表
CREATE TABLE credit_card_payment (
    payment_id BIGINT PRIMARY KEY,
    card_number VARCHAR(255),
    FOREIGN KEY (payment_id) REFERENCES payment(id)
);
```

### 5.3 TABLE_PER_CLASS（每类一表）

每个类独立一张表，包含所有字段。

```java
@Entity
@Inheritance(strategy = InheritanceType.TABLE_PER_CLASS)
public abstract class Payment {
    @Id
    @GeneratedValue
    private Long id;
    private BigDecimal amount;
}

@Entity
public class CreditCardPayment extends Payment {
    private String cardNumber;
}
```

**生成的表结构**：

```sql
-- 每张表都包含父类和子类的所有字段
CREATE TABLE payment (
    id BIGINT PRIMARY KEY,
    amount DECIMAL(19,2)
);

CREATE TABLE credit_card_payment (
    id BIGINT PRIMARY KEY,
    amount DECIMAL(19,2),
    card_number VARCHAR(255)
);
```

### 5.4 策略对比

| 策略 | 优点 | 缺点 |
|------|------|------|
| SINGLE_TABLE | 性能好，简单 | 浪费空间，不支持非空约束 |
| JOINED | 规范化，节省空间 | 查询需要 JOIN，性能较差 |
| TABLE_PER_CLASS | 查询快 | 多态查询困难，重复列 |

**推荐**：SINGLE_TABLE（最常用）

## 6. 实体生命周期

### 6.1 四种状态

<!-- TODO: 插入图片：Hibernate 实体生命周期状态图 -->

1. **New（瞬时态）**
   - 刚创建的对象
   - 未与 Session 关联
   - 数据库中没有对应记录

2. **Managed（持久态）**
   - 已与 Session 关联
   - 数据库中有对应记录
   - 属性变化会自动同步

3. **Detached（游离态）**
   - Session 已关闭
   - 数据库中有对应记录
   - 属性变化不会同步

4. **Removed（删除态）**
   - 已标记删除
   - 事务提交后从数据库删除

### 6.2 状态转换

```java
// New → Managed
User user = new User();           // New
session.save(user);               // Managed

// Managed → Detached
session.close();                  // Detached

// Detached → Managed
session.update(user);             // Managed

// Managed → Removed
session.delete(user);             // Removed
```

### 6.3 脏检查（Dirty Checking）

```java
Transaction tx = session.beginTransaction();

User user = session.get(User.class, 1L);  // 持久态
user.setUsername("新名字");                // 修改属性

tx.commit();  // Hibernate 自动检测变化并执行 UPDATE
```

**原理**：Session 维护了持久化对象的快照，提交时比较差异。

## 7. 完整示例

### 7.1 用户实体

```java
package com.example.entity;

import lombok.Data;
import lombok.NoArgsConstructor;
import lombok.AllArgsConstructor;

import javax.persistence.*;
import java.io.Serializable;
import java.time.LocalDateTime;

@Entity
@Table(name = "user")
@Data
@NoArgsConstructor
@AllArgsConstructor
public class User implements Serializable {
    
    private static final long serialVersionUID = 1L;
    
    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;
    
    @Column(name = "username", nullable = false, unique = true, length = 50)
    private String username;
    
    @Column(name = "email", nullable = false, unique = true, length = 100)
    private String email;
    
    @Column(name = "age")
    private Integer age;
    
    @Enumerated(EnumType.STRING)
    @Column(name = "gender")
    private Gender gender;
    
    @Column(name = "phone", length = 20)
    private String phone;
    
    @Column(name = "address", length = 200)
    private String address;
    
    @Column(name = "status")
    private Integer status;  // 0-禁用，1-启用
    
    @Column(name = "create_time", updatable = false)
    private LocalDateTime createTime;
    
    @Column(name = "update_time")
    private LocalDateTime updateTime;
    
    @PrePersist
    protected void onCreate() {
        createTime = LocalDateTime.now();
        updateTime = LocalDateTime.now();
    }
    
    @PreUpdate
    protected void onUpdate() {
        updateTime = LocalDateTime.now();
    }
}
```

### 7.2 测试代码

```java
@Test
public void testUserCRUD() {
    Session session = sessionFactory.openSession();
    Transaction tx = session.beginTransaction();
    
    try {
        // 创建
        User user = new User();
        user.setUsername("张三");
        user.setEmail("zhangsan@example.com");
        user.setAge(25);
        user.setGender(Gender.MALE);
        user.setPhone("13800138000");
        
        session.save(user);
        System.out.println("保存成功，ID: " + user.getId());
        
        // 查询
        User found = session.get(User.class, user.getId());
        System.out.println("查询结果: " + found);
        
        // 更新
        found.setAge(26);
        // 不需要调用 update，脏检查会自动同步
        
        tx.commit();
        System.out.println("事务提交成功");
        
    } catch (Exception e) {
        tx.rollback();
        e.printStackTrace();
    } finally {
        session.close();
    }
}
```

## 9. 总结

### 9.1 核心知识点

✅ **实体注解**：@Entity、@Table、@Id、@Column  
✅ **主键策略**：IDENTITY、SEQUENCE、TABLE、AUTO  
✅ **复合主键**：@EmbeddedId、@IdClass  
✅ **类型映射**：枚举、日期、大对象  
✅ **自定义转换器**：@Convert  
✅ **继承映射**：SINGLE_TABLE、JOINED、TABLE_PER_CLASS  
✅ **实体状态**：New、Managed、Detached、Removed  

### 9.2 最佳实践

📌 优先使用注解而非 XML 配置  
📌 使用包装类型而非基本类型  
📌 合理选择主键生成策略  
📌 使用 @Enumerated(EnumType.STRING) 存储枚举  
📌 使用 Java 8 时间 API（LocalDateTime）  
📌 注意实体生命周期的状态转换
