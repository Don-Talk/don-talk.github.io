## 1. 缓存架构

### 1.1 三级缓存

<!-- TODO: 插入图片：Hibernate 三级缓存架构图 -->

| 缓存级别 | 范围 | 生命周期 | 默认开启 |
|---------|------|---------|---------|
| **一级缓存** | Session | Session 级别 | ✅ 是 |
| **二级缓存** | SessionFactory | 应用级别 | ❌ 否 |
| **查询缓存** | SessionFactory | 应用级别 | ❌ 否 |

### 1.2 缓存工作流程

```
查询请求
  ↓
一级缓存命中？ → 是 → 返回结果
  ↓ 否
二级缓存命中？ → 是 → 返回结果
  ↓ 否
查询缓存命中？ → 是 → 返回结果
  ↓ 否
数据库查询 → 写入各级缓存
```

## 2. 一级缓存（Session 缓存）

### 2.1 基本概念

* **作用域**：单个 Session
* **生命周期**：Session 创建到关闭
* **自动管理**：无需配置，默认开启
* **线程安全**：否（Session 非线程安全）

### 2.2 缓存测试

```java
@Test
public void testFirstLevelCache() {
    Session session = sessionFactory.openSession();
    
    // 第一次查询：从数据库加载
    User user1 = session.get(User.class, 1L);
    System.out.println("第一次查询：" + user1.getUsername());
    
    // 第二次查询：从一级缓存获取
    User user2 = session.get(User.class, 1L);
    System.out.println("第二次查询：" + user2.getUsername());
    
    // 验证是同一个对象
    System.out.println("是否同一对象：" + (user1 == user2));  // true
    
    session.close();
}
```

**输出**：
```
Hibernate: SELECT ... FROM user WHERE id = ?  -- 只执行一次 SQL
第一次查询：张三
第二次查询：张三
是否同一对象：true
```

### 2.3 清空一级缓存

#### （1）clear() - 清空所有

```java
@Test
public void testClear() {
    Session session = sessionFactory.openSession();
    
    User user1 = session.get(User.class, 1L);
    User user2 = session.get(User.class, 2L);
    
    // 清空缓存
    session.clear();
    
    // 再次查询会执行 SQL
    User user3 = session.get(User.class, 1L);
    
    session.close();
}
```

#### （2）evict() - 移除单个对象

```java
@Test
public void testEvict() {
    Session session = sessionFactory.openSession();
    
    User user1 = session.get(User.class, 1L);
    
    // 从缓存中移除
    session.evict(user1);
    
    // 再次查询会执行 SQL
    User user2 = session.get(User.class, 1L);
    
    session.close();
}
```

### 2.4 批量操作优化

```java
@Test
public void testBatchOperation() {
    Session session = sessionFactory.openSession();
    Transaction tx = session.beginTransaction();
    
    for (int i = 0; i < 1000; i++) {
        User user = new User();
        user.setUsername("用户" + i);
        session.save(user);
        
        // 每 50 条刷新并清空缓存
        if (i % 50 == 0) {
            session.flush();
            session.clear();
        }
    }
    
    tx.commit();
    session.close();
}
```

**原因**：避免内存溢出，及时释放缓存空间。

## 3. 二级缓存（SessionFactory 缓存）

### 3.1 基本概念

* **作用域**：整个应用（所有 Session 共享）
* **生命周期**：SessionFactory 创建到关闭
* **需要配置**：默认不开启
* **线程安全**：是

### 3.2 启用二级缓存

#### 步骤 1：添加依赖

```xml
<dependency>
    <groupId>org.hibernate.orm</groupId>
    <artifactId>hibernate-ehcache</artifactId>
    <version>6.2.7.Final</version>
</dependency>

<dependency>
    <groupId>net.sf.ehcache</groupId>
    <artifactId>ehcache</artifactId>
    <version>2.10.9.2</version>
</dependency>
```

#### 步骤 2：配置文件

```xml
<!-- hibernate.cfg.xml -->
<property name="hibernate.cache.use_second_level_cache">true</property>
<property name="hibernate.cache.region.factory_class">
    org.hibernate.cache.ehcache.EhCacheRegionFactory
</property>
```

#### 步骤 3：实体类标注

```java
@Entity
@Cacheable  // 启用二级缓存
@org.hibernate.annotations.Cache(
    usage = CacheConcurrencyStrategy.READ_WRITE
)
public class User {
    @Id
    @GeneratedValue
    private Long id;
    
    private String username;
}
```

### 3.3 缓存并发策略

| 策略 | 说明 | 适用场景 |
|------|------|---------|
| READ_ONLY | 只读，不可修改 | 字典表、配置表 |
| NONSTRICT_READ_WRITE | 非严格读写 | 偶尔更新的数据 |
| READ_WRITE | 读写，使用锁 | 频繁更新的数据 |
| TRANSACTIONAL | 事务性缓存 | 需要事务支持 |

### 3.4 Ehcache 配置

创建 `ehcache.xml`：

```xml
<ehcache>
    <!-- 默认缓存配置 -->
    <defaultCache
        maxElementsInMemory="10000"
        eternal="false"
        timeToIdleSeconds="120"
        timeToLiveSeconds="120"
        overflowToDisk="false"/>
    
    <!-- User 实体缓存 -->
    <cache name="com.example.entity.User"
           maxElementsInMemory="5000"
           eternal="false"
           timeToIdleSeconds="300"
           timeToLiveSeconds="600"
           overflowToDisk="false"/>
    
    <!-- Order 实体缓存 -->
    <cache name="com.example.entity.Order"
           maxElementsInMemory="10000"
           eternal="false"
           timeToIdleSeconds="600"
           timeToLiveSeconds="1200"
           overflowToDisk="false"/>
</ehcache>
```

**参数说明**：
* `maxElementsInMemory`：内存中最大元素数
* `eternal`：是否永久有效
* `timeToIdleSeconds`：最大空闲时间（秒）
* `timeToLiveSeconds`：最大存活时间（秒）
* `overflowToDisk`：是否溢出到磁盘

### 3.5 测试二级缓存

```java
@Test
public void testSecondLevelCache() {
    // 第一个 Session
    Session session1 = sessionFactory.openSession();
    User user1 = session1.get(User.class, 1L);
    System.out.println("第一次查询：" + user1.getUsername());
    session1.close();  // 关闭 Session，数据写入二级缓存
    
    // 第二个 Session
    Session session2 = sessionFactory.openSession();
    User user2 = session2.get(User.class, 1L);
    System.out.println("第二次查询：" + user2.getUsername());
    session2.close();
    
    // 两个不同 Session，但共享二级缓存
    System.out.println("是否同一对象：" + (user1 == user2));  // false（不同对象）
    System.out.println("ID 是否相同：" + user1.getId().equals(user2.getId()));  // true
}
```

### 3.6 管理二级缓存

```java
// 清空某个实体的缓存
sessionFactory.getCache().evictEntityRegion(User.class);

// 清空某个对象的缓存
sessionFactory.getCache().evictEntity(User.class, 1L);

// 清空所有缓存
sessionFactory.getCache().evictAllRegions();
```

## 4. 查询缓存

### 4.1 基本概念

* **作用**：缓存 HQL/Criteria 查询的结果
* **依赖**：需要二级缓存支持
* **适用**：相同查询条件频繁执行

### 4.2 启用查询缓存

```xml
<!-- hibernate.cfg.xml -->
<property name="hibernate.cache.use_query_cache">true</property>
```

### 4.3 使用查询缓存

```java
@Test
public void testQueryCache() {
    Session session = sessionFactory.openSession();
    
    // 第一次查询
    List<User> users1 = session.createQuery("FROM User WHERE age > :age", User.class)
            .setParameter("age", 20)
            .setCacheable(true)  // 启用查询缓存
            .list();
    
    System.out.println("第一次查询结果数：" + users1.size());
    session.close();
    
    // 第二次查询（相同条件）
    Session session2 = sessionFactory.openSession();
    List<User> users2 = session2.createQuery("FROM User WHERE age > :age", User.class)
            .setParameter("age", 20)
            .setCacheable(true)
            .list();
    
    System.out.println("第二次查询结果数：" + users2.size());
    session2.close();
}
```

### 4.4 注意事项

⚠️ **查询缓存的局限性**：
* 只缓存 ID 列表，不缓存实体对象
* 实体对象仍需从二级缓存或数据库加载
* 表数据变化时，相关查询缓存会失效
* 适合读多写少、查询条件固定的场景

## 5. Redis 集成（高级）

### 5.1 为什么使用 Redis？

* **分布式缓存**：支持集群部署
* **高性能**：内存存储，速度快
* **持久化**：支持数据持久化
* **灵活性**：丰富的数据结构

### 5.2 添加依赖

```xml
<dependency>
    <groupId>org.redisson</groupId>
    <artifactId>redisson-hibernate-6</artifactId>
    <version>3.23.5</version>
</dependency>
```

### 5.3 配置 Redis

```xml
<property name="hibernate.cache.use_second_level_cache">true</property>
<property name="hibernate.cache.region.factory_class">
    org.redisson.hibernate.RedissonRegionFactory
</property>

<!-- Redisson 配置文件 -->
<property name="hibernate.cache.redisson.config">/redisson.yaml</property>
```

创建 `redisson.yaml`：

```yaml
singleServerConfig:
  address: "redis://localhost:6379"
  password: null
  database: 0
  connectionPoolSize: 64
  connectionMinimumIdleSize: 24
```

## 6. 缓存性能优化

### 6.1 缓存命中率监控

```java
Statistics stats = sessionFactory.getStatistics();

// 二级缓存命中率
long hitCount = stats.getSecondLevelCacheHitCount();
long missCount = stats.getSecondLevelCacheMissCount();
double hitRate = (double) hitCount / (hitCount + missCount);

System.out.println("二级缓存命中率：" + String.format("%.2f%%", hitRate * 100));
```

### 6.2 优化建议

📌 **合理设置缓存大小**：根据内存情况调整  
📌 **选择合适的过期策略**：根据数据更新频率  
📌 **优先缓存热点数据**：访问频繁的实体  
📌 **避免缓存大对象**：占用过多内存  
📌 **定期清理无效缓存**：防止内存泄漏  

### 6.3 缓存适用场景

✅ **适合缓存**：
* 字典表、配置表（几乎不变）
* 热点数据（频繁读取）
* 计算成本高的数据

❌ **不适合缓存**：
* 频繁更新的数据
* 大数据量集合
* 时效性要求高的数据

## 7. 完整示例

### 7.1 商品缓存

```java
@Entity
@Cacheable
@org.hibernate.annotations.Cache(usage = CacheConcurrencyStrategy.READ_WRITE)
public class Product {
    
    @Id
    @GeneratedValue
    private Long id;
    
    private String name;
    
    private BigDecimal price;
    
    private Integer stock;
}
```

### 7.2 缓存服务

```java
@Service
public class ProductService {
    
    @Autowired
    private SessionFactory sessionFactory;
    
    public Product findById(Long id) {
        Session session = sessionFactory.openSession();
        try {
            return session.get(Product.class, id);
        } finally {
            session.close();
        }
    }
    
    public void clearProductCache(Long id) {
        sessionFactory.getCache().evictEntity(Product.class, id);
    }
    
    public void clearAllProductCache() {
        sessionFactory.getCache().evictEntityRegion(Product.class);
    }
}
```

## 8. 总结

### 8.1 核心知识点

✅ **一级缓存**：Session 级别，自动管理  
✅ **二级缓存**：SessionFactory 级别，需配置  
✅ **查询缓存**：缓存查询结果，依赖二级缓存  
✅ **缓存策略**：READ_ONLY、READ_WRITE 等  
✅ **缓存提供者**：Ehcache、Redis  
✅ **性能监控**：命中率统计  

### 8.2 最佳实践

📌 一级缓存注意及时清空（批量操作）  
📌 二级缓存适合读多写少场景  
📌 选择合适的缓存并发策略  
📌 监控缓存命中率，持续优化  
📌 分布式环境使用 Redis
