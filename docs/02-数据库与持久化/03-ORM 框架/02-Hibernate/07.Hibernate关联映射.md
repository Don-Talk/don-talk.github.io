# 07Hibernate 关联映射

**学习目标：**
> * 掌握一对一、一对多、多对一、多对多映射
> * 理解级联操作（CascadeType）
> * 掌握延迟加载与立即加载（FetchType）
> * 理解双向关联的维护方
> * 能够设计合理的实体关系

---

## 1. 关联映射概述

### 1.1 关联类型

| 关联类型 | 注解 | 示例 |
|---------|------|------|
| 一对一 | @OneToOne | 用户 - 身份证 |
| 一对多 | @OneToMany | 用户 - 订单 |
| 多对一 | @ManyToOne | 订单 - 用户 |
| 多对多 | @ManyToMany | 学生 - 课程 |

### 1.2 加载策略

* **EAGER（立即加载）**：查询主对象时同时加载关联对象
* **LAZY（延迟加载）**：访问关联对象时才加载

**默认值**：
* `@OneToOne`、`@ManyToOne`：EAGER
* `@OneToMany`、`@ManyToMany`：LAZY

## 2. 一对一映射（@OneToOne）

### 2.1 单向关联

```java
// 用户实体
@Entity
@Table(name = "user")
public class User {
    
    @Id
    @GeneratedValue
    private Long id;
    
    private String username;
    
    // 一对一：用户 -> 身份证
    @OneToOne
    @JoinColumn(name = "id_card_id")  // 外键列
    private IdCard idCard;
}

// 身份证实体
@Entity
@Table(name = "id_card")
public class IdCard {
    
    @Id
    @GeneratedValue
    private Long id;
    
    private String cardNo;
}
```

### 2.2 双向关联

```java
// 用户实体（维护方）
@Entity
public class User {
    
    @Id
    @GeneratedValue
    private Long id;
    
    private String username;
    
    @OneToOne(mappedBy = "user", cascade = CascadeType.ALL, fetch = FetchType.LAZY)
    private IdCard idCard;
}

// 身份证实体（被维护方）
@Entity
public class IdCard {
    
    @Id
    @GeneratedValue
    private Long id;
    
    private String cardNo;
    
    @OneToOne
    @JoinColumn(name = "user_id")
    private User user;
}
```

**关键点**：
* `mappedBy`：指定由哪一方维护关系
* `cascade`：级联操作类型
* `fetch`：加载策略

## 3. 一对多/多对一映射

### 3.1 单向一对多

```java
@Entity
public class User {
    
    @Id
    @GeneratedValue
    private Long id;
    
    private String username;
    
    // 一对多：用户 -> 订单
    @OneToMany
    @JoinColumn(name = "user_id")  // 在 Order 表中添加外键
    private List<Order> orders;
}
```

### 3.2 双向一对多（推荐）

```java
// 用户实体（一方）
@Entity
public class User {
    
    @Id
    @GeneratedValue
    private Long id;
    
    private String username;
    
    // 一对多
    @OneToMany(mappedBy = "user", cascade = CascadeType.ALL, fetch = FetchType.LAZY)
    private List<Order> orders = new ArrayList<>();
    
    // 辅助方法
    public void addOrder(Order order) {
        orders.add(order);
        order.setUser(this);
    }
    
    public void removeOrder(Order order) {
        orders.remove(order);
        order.setUser(null);
    }
}

// 订单实体（多方）
@Entity
public class Order {
    
    @Id
    @GeneratedValue
    private Long id;
    
    private String orderNo;
    
    private BigDecimal amount;
    
    // 多对一
    @ManyToOne(fetch = FetchType.LAZY)
    @JoinColumn(name = "user_id")
    private User user;
}
```

**数据库表结构**：

```sql
CREATE TABLE user (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    username VARCHAR(50)
);

CREATE TABLE orders (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    order_no VARCHAR(50),
    amount DECIMAL(10,2),
    user_id BIGINT,  -- 外键
    FOREIGN KEY (user_id) REFERENCES user(id)
);
```

### 3.3 最佳实践

✅ **在"多"方维护关系**（使用 @ManyToOne）  
✅ **一方使用 mappedBy**  
✅ **设置 fetch = LAZY** 避免性能问题  
✅ **提供辅助方法**管理关联

## 4. 多对多映射（@ManyToMany）

### 4.1 基本映射

```java
// 学生实体
@Entity
public class Student {
    
    @Id
    @GeneratedValue
    private Long id;
    
    private String name;
    
    // 多对多：学生 <-> 课程
    @ManyToMany
    @JoinTable(
        name = "student_course",
        joinColumns = @JoinColumn(name = "student_id"),
        inverseJoinColumns = @JoinColumn(name = "course_id")
    )
    private List<Course> courses = new ArrayList<>();
}

// 课程实体
@Entity
public class Course {
    
    @Id
    @GeneratedValue
    private Long id;
    
    private String courseName;
    
    @ManyToMany(mappedBy = "courses")
    private List<Student> students = new ArrayList<>();
}
```

**数据库表结构**：

```sql
CREATE TABLE student (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    name VARCHAR(50)
);

CREATE TABLE course (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    course_name VARCHAR(100)
);

-- 中间表
CREATE TABLE student_course (
    student_id BIGINT,
    course_id BIGINT,
    PRIMARY KEY (student_id, course_id),
    FOREIGN KEY (student_id) REFERENCES student(id),
    FOREIGN KEY (course_id) REFERENCES course(id)
);
```

### 4.2 双向多对多

```java
// 学生实体（维护方）
@Entity
public class Student {
    
    @Id
    @GeneratedValue
    private Long id;
    
    private String name;
    
    @ManyToMany(cascade = {CascadeType.PERSIST, CascadeType.MERGE})
    @JoinTable(
        name = "student_course",
        joinColumns = @JoinColumn(name = "student_id"),
        inverseJoinColumns = @JoinColumn(name = "course_id")
    )
    private List<Course> courses = new ArrayList<>();
    
    public void addCourse(Course course) {
        courses.add(course);
        course.getStudents().add(this);
    }
    
    public void removeCourse(Course course) {
        courses.remove(course);
        course.getStudents().remove(this);
    }
}

// 课程实体（被维护方）
@Entity
public class Course {
    
    @Id
    @GeneratedValue
    private Long id;
    
    private String courseName;
    
    @ManyToMany(mappedBy = "courses")
    private List<Student> students = new ArrayList<>();
}
```

## 5. 级联操作（CascadeType）

### 5.1 级联类型

| 类型 | 说明 |
|------|------|
| ALL | 所有操作 |
| PERSIST | 保存时级联 |
| MERGE | 更新时级联 |
| REMOVE | 删除时级联 |
| REFRESH | 刷新时级联 |
| DETACH | 分离时级联 |

### 5.2 使用示例

```java
@Entity
public class User {
    
    @Id
    @GeneratedValue
    private Long id;
    
    // 级联保存、更新、删除
    @OneToMany(mappedBy = "user", cascade = {
        CascadeType.PERSIST,
        CascadeType.MERGE,
        CascadeType.REMOVE
    })
    private List<Order> orders;
}
```

### 5.3 级联测试

```java
@Test
public void testCascade() {
    Session session = sessionFactory.openSession();
    Transaction tx = session.beginTransaction();
    
    // 创建用户和订单
    User user = new User();
    user.setUsername("张三");
    
    Order order1 = new Order();
    order1.setOrderNo("ORD001");
    order1.setAmount(new BigDecimal("100"));
    
    Order order2 = new Order();
    order2.setOrderNo("ORD002");
    order2.setAmount(new BigDecimal("200"));
    
    // 添加订单到用户
    user.addOrder(order1);
    user.addOrder(order2);
    
    // 只保存用户，订单会自动保存（级联 PERSIST）
    session.save(user);
    
    tx.commit();
    session.close();
}
```

## 6. 延迟加载与立即加载

### 6.1 FetchType.EAGER

```java
@ManyToOne(fetch = FetchType.EAGER)
@JoinColumn(name = "user_id")
private User user;
```

**特点**：
* ✅ 查询时立即加载关联对象
* ❌ 可能产生 N+1 问题
* ❌ 性能较差

### 6.2 FetchType.LAZY

```java
@ManyToOne(fetch = FetchType.LAZY)
@JoinColumn(name = "user_id")
private User user;
```

**特点**：
* ✅ 按需加载，性能好
* ⚠️ 必须在 Session 关闭前访问
* ⚠️ 可能抛出 LazyInitializationException

### 6.3 解决懒加载异常

**方式一：在 Session 内访问**

```java
Session session = sessionFactory.openSession();
User user = session.get(User.class, 1L);

// 在 Session 关闭前访问
List<Order> orders = user.getOrders();  // 触发加载

session.close();
```

**方式二：使用 JOIN FETCH**

```java
List<User> users = session.createQuery(
    "FROM User u LEFT JOIN FETCH u.orders", User.class)
    .list();
```

**方式三：Open Session In View（不推荐）**

```xml
<!-- web.xml -->
<filter>
    <filter-name>OpenSessionInViewFilter</filter-name>
    <filter-class>org.springframework.orm.hibernate5.support.OpenSessionInViewFilter</filter-class>
</filter>
```

## 7. 双向关联维护

### 7.1 维护方选择原则

* **一对多/多对一**：在"多"方维护（@ManyToOne）
* **一对一**：任意一方，通常在外键持有方
* **多对多**：任意一方，通常在使用频率高的一方

### 7.2 mappedBy 的作用

```java
// User 端（不维护）
@OneToMany(mappedBy = "user")
private List<Order> orders;

// Order 端（维护）
@ManyToOne
@JoinColumn(name = "user_id")
private User user;
```

**mappedBy** 表示：
* 该字段不维护关系
* 关系由对方的 `user` 字段维护
* 不会生成额外的外键列

### 7.3 常见错误

❌ **两端都维护**：会产生两个外键列  
❌ **两端都不维护**：无法建立关联  
✅ **一端维护，一端 mappedBy**：正确做法

## 8. 完整示例

### 8.1 博客系统

```java
// 用户
@Entity
public class User {
    
    @Id
    @GeneratedValue
    private Long id;
    
    private String username;
    
    // 一对多：用户 -> 文章
    @OneToMany(mappedBy = "author", cascade = CascadeType.ALL, orphanRemoval = true)
    private List<Article> articles = new ArrayList<>();
    
    public void addArticle(Article article) {
        articles.add(article);
        article.setAuthor(this);
    }
}

// 文章
@Entity
public class Article {
    
    @Id
    @GeneratedValue
    private Long id;
    
    private String title;
    
    private String content;
    
    // 多对一：文章 -> 用户
    @ManyToOne(fetch = FetchType.LAZY)
    @JoinColumn(name = "author_id")
    private User author;
    
    // 一对多：文章 -> 评论
    @OneToMany(mappedBy = "article", cascade = CascadeType.ALL, orphanRemoval = true)
    private List<Comment> comments = new ArrayList<>();
}

// 评论
@Entity
public class Comment {
    
    @Id
    @GeneratedValue
    private Long id;
    
    private String content;
    
    // 多对一：评论 -> 文章
    @ManyToOne(fetch = FetchType.LAZY)
    @JoinColumn(name = "article_id")
    private Article article;
}
```

### 8.2 测试代码

```java
@Test
public void testBlogSystem() {
    Session session = sessionFactory.openSession();
    Transaction tx = session.beginTransaction();
    
    // 创建用户
    User user = new User();
    user.setUsername("张三");
    
    // 创建文章
    Article article = new Article();
    article.setTitle("第一篇文章");
    article.setContent("文章内容...");
    
    // 创建评论
    Comment comment = new Comment();
    comment.setContent("很好的文章！");
    
    // 建立关联
    user.addArticle(article);
    article.getComments().add(comment);
    comment.setArticle(article);
    
    // 保存用户（级联保存文章和评论）
    session.save(user);
    
    tx.commit();
    session.close();
}
```

## 10. 总结

### 10.1 核心知识点

✅ **一对一**：@OneToOne、@JoinColumn  
✅ **一对多/多对一**：@OneToMany、@ManyToOne、mappedBy  
✅ **多对多**：@ManyToMany、@JoinTable  
✅ **级联操作**：CascadeType  
✅ **加载策略**：FetchType.EAGER、FetchType.LAZY  
✅ **双向关联**：维护方选择、mappedBy  

### 10.2 最佳实践

📌 优先使用 LAZY 加载  
📌 在"多"方维护关系  
📌 谨慎使用级联删除  
📌 使用 JOIN FETCH 解决 N+1  
📌 提供辅助方法管理关联
