## 1. 什么是 Hibernate？

### 1.1 Hibernate 简介

Hibernate 是一个开源的 **ORM（Object-Relational Mapping，对象关系映射）** 框架，它将 Java 对象自动映射到数据库表中。

* **官网**：https://hibernate.org/
* **GitHub**：https://github.com/hibernate/hibernate-orm
* **当前版本**：Hibernate 6.x
* **JPA 实现**：Hibernate 是 JPA（Java Persistence API）的最流行实现

<!-- TODO: 插入图片：Hibernate 架构图 -->

### 1.2 ORM 思想

**ORM（对象关系映射）** 的核心思想：

* **对象**：Java 中的实体类（Entity）
* **关系**：数据库中的表（Table）
* **映射**：自动将对象操作转换为 SQL 语句

```java
// Java 对象
User user = new User();
user.setUsername("张三");
user.setAge(25);

// Hibernate 自动转换为 SQL
// INSERT INTO user (username, age) VALUES ('张三', 25);
```

### 1.3 JPA 与 Hibernate 的关系

**JPA（Java Persistence API）**：
* Java EE 规范，定义了一套 ORM 标准接口
* 不是具体实现，而是一套规范

**Hibernate**：
* JPA 的具体实现之一
* 提供了 JPA 规范的所有功能
* 还扩展了许多额外特性

**其他 JPA 实现**：
* EclipseLink
* OpenJPA
* TopLink

**关系图**：
```
JPA（规范/接口）
  ↓
Hibernate（实现）
EclipseLink（实现）
OpenJPA（实现）
```

## 2. Hibernate vs MyBatis 对比

### 2.1 设计理念对比

| 特性 | Hibernate | MyBatis |
|------|-----------|---------|
| **类型** | 全自动 ORM | 半自动 SQL 映射 |
| **SQL 控制** | 自动生成 SQL | 手动编写 SQL |
| **学习曲线** | 较陡峭 | 平缓 |
| **开发效率** | 高（CRUD 无需写 SQL） | 中等（需写 SQL） |
| **灵活性** | 中等 | 高 |
| **性能优化** | 需要理解缓存机制 | 直接优化 SQL |
| **适用场景** | 复杂业务逻辑、领域模型 | 灵活查询、报表统计 |

### 2.2 代码对比

#### Hibernate 方式

```java
// 保存用户（无需写 SQL）
User user = new User();
user.setUsername("张三");
user.setAge(25);
session.save(user);

// 查询用户（HQL 类似 SQL）
List<User> users = session.createQuery("FROM User WHERE age > :age")
    .setParameter("age", 18)
    .list();
```

#### MyBatis 方式

```xml
<!-- 需要编写 SQL -->
<insert id="insert">
    INSERT INTO user (username, age) VALUES (#{username}, #{age})
</insert>

<select id="findByAge" resultType="User">
    SELECT * FROM user WHERE age &gt; #{age}
</select>
```

### 2.3 选择建议

**使用 Hibernate 的场景**：
* 复杂的业务逻辑和领域模型
* 快速开发 CRUD 功能
* 团队熟悉面向对象设计
* 需要跨数据库移植

**使用 MyBatis 的场景**：
* 需要精细控制 SQL
* 复杂的报表和统计查询
* 对性能要求极高
* 遗留系统改造

**混合使用**：
* 简单 CRUD 用 Hibernate
* 复杂查询用 MyBatis
* Spring Data JPA + MyBatis

## 3. Hibernate 环境搭建

### 3.1 创建 Maven 项目

创建 Maven 项目，在 `pom.xml` 中添加依赖：

```xml
<?xml version="1.0" encoding="UTF-8"?>
<project xmlns="http://maven.apache.org/POM/4.0.0"
         xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
         xsi:schemaLocation="http://maven.apache.org/POM/4.0.0 
         http://maven.apache.org/xsd/maven-4.0.0.xsd">
    <modelVersion>4.0.0</modelVersion>

    <groupId>com.example</groupId>
    <artifactId>hibernate-demo</artifactId>
    <version>1.0-SNAPSHOT</version>

    <properties>
        <maven.compiler.source>8</maven.compiler.source>
        <maven.compiler.target>8</maven.compiler.target>
        <project.build.sourceEncoding>UTF-8</project.build.sourceEncoding>
    </properties>

    <dependencies>
        <!-- Hibernate Core -->
        <dependency>
            <groupId>org.hibernate.orm</groupId>
            <artifactId>hibernate-core</artifactId>
            <version>6.2.7.Final</version>
        </dependency>

        <!-- MySQL 驱动 -->
        <dependency>
            <groupId>mysql</groupId>
            <artifactId>mysql-connector-java</artifactId>

        <!-- Lombok（简化实体类） -->

            <scope>provided</scope>
        </dependency>

        <dependency>
            <groupId>junit</groupId>

            <version>4.13.2</version>
            <scope>test</scope>
        </dependency>
    </dependencies>
</project>
```

### 3.2 准备数据库表

在 MySQL 中创建测试数据库和表：

```sql
-- 创建数据库

-- 使用数据库
USE hibernate_demo;

-- 创建用户表
CREATE TABLE `user` (
  `id` BIGINT PRIMARY KEY AUTO_INCREMENT COMMENT '用户ID',
  `username` VARCHAR(50) NOT NULL COMMENT '用户名',

  `age` INT COMMENT '年龄',

-- 插入测试数据
INSERT INTO `user` (`username`, `email`, `age`) VALUES
('张三', 'zhangsan@example.com', 25),
('李四', 'lisi@example.com', 28),

```

### 3.3 创建实体类

创建与数据库表对应的 Java 实体类：

```java
package com.example.entity;

import lombok.Data;
import javax.persistence.*;
import java.util.Date;

/**
 * 用户实体类
 */
@Entity
@Table(name = "user")
@Data
public class User {
    
    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;
    
    @Column(name = "username", nullable = false, length = 50)
    private String username;
    
    @Column(name = "email", length = 100)
    private String email;
    
    @Column(name = "age")
    private Integer age;
    
    @Column(name = "create_time")
    @Temporal(TemporalType.TIMESTAMP)
    private Date createTime;
}
```

**注解说明**：
* `@Entity`：标记为 JPA 实体类
* `@Table`：指定数据库表名
* `@Id`：主键字段
* `@GeneratedValue`：主键生成策略
* `@Column`：字段映射配置

## 4. 第一个 Hibernate 程序

### 4.1 创建 Hibernate 配置文件

在 `src/main/resources` 目录下创建 `hibernate.cfg.xml`：

```xml
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE hibernate-configuration PUBLIC
        "-//Hibernate/Hibernate Configuration DTD 3.0//EN"
        "http://www.hibernate.org/dtd/hibernate-configuration-3.0.dtd">

<hibernate-configuration>
    <session-factory>
        <!-- 数据库连接配置 -->
        <property name="hibernate.connection.driver_class">com.mysql.cj.jdbc.Driver</property>
        <property name="hibernate.connection.url">
            jdbc:mysql://localhost:3306/hibernate_demo?useSSL=false&amp;serverTimezone=Asia/Shanghai&amp;characterEncoding=utf8
        </property>
        <property name="hibernate.connection.username">root</property>
        <property name="hibernate.connection.password">123456</property>

        <!-- 数据库方言 -->
        <property name="hibernate.dialect">org.hibernate.dialect.MySQLDialect</property>

        <!-- 显示 SQL -->
        <property name="hibernate.show_sql">true</property>
        <property name="hibernate.format_sql">true</property>

        <!-- 自动更新表结构 -->
        <property name="hibernate.hbm2ddl.auto">update</property>

        <!-- 注册实体类 -->
        <mapping class="com.example.entity.User"/>
    </session-factory>
</hibernate-configuration>
```

**关键配置说明**：

| 配置项 | 说明 | 可选值 |
|--------|------|--------|
| `hibernate.dialect` | 数据库方言 | MySQLDialect、OracleDialect 等 |
| `hibernate.show_sql` | 是否显示 SQL | true/false |
| `hibernate.format_sql` | 格式化 SQL | true/false |
| `hibernate.hbm2ddl.auto` | 自动建表策略 | create、update、validate、none |

**hbm2ddl.auto 选项**：
* `create`：每次运行都删除旧表，创建新表
* `update`：如果表不存在则创建，存在则更新
* `validate`：验证表结构，不匹配则报错
* `none`：不做任何操作（生产环境推荐）

### 4.2 编写测试代码

创建测试类 `HibernateTest.java`：

```java
package com.example.test;

import com.example.entity.User;
import org.hibernate.Session;
import org.hibernate.SessionFactory;
import org.hibernate.Transaction;
import org.hibernate.cfg.Configuration;
import org.junit.After;
import org.junit.Before;
import org.junit.Test;

import java.util.Date;
import java.util.List;

/**
 * Hibernate 测试类
 */
public class HibernateTest {
    
    private SessionFactory sessionFactory;
    private Session session;
    
    /**
     * 初始化：创建 SessionFactory
     */
    @Before
    public void init() {
        // 1. 读取配置文件
        Configuration configuration = new Configuration().configure();
        
        // 2. 创建 SessionFactory
        sessionFactory = configuration.buildSessionFactory();
        
        // 3. 打开 Session
        session = sessionFactory.openSession();
    }
    
    /**
     * 测试保存用户
     */
    @Test
    public void testSave() {
        // 4. 开启事务
        Transaction transaction = session.beginTransaction();
        
        try {
            // 5. 创建用户对象
            User user = new User();
            user.setUsername("测试用户");
            user.setEmail("test@example.com");
            user.setAge(20);
            user.setCreateTime(new Date());
            
            // 6. 保存用户
            session.save(user);
            
            // 7. 提交事务
            transaction.commit();
            
            System.out.println("保存成功，生成的ID：" + user.getId());
        } catch (Exception e) {
            // 回滚事务
            transaction.rollback();
            e.printStackTrace();
        }
    }
    
    /**
     * 测试查询所有用户
     */
    @Test
    public void testFindAll() {
        // HQL 查询
        List<User> users = session.createQuery("FROM User", User.class).list();
        
        for (User user : users) {
            System.out.println(user);
        }
    }
    
    /**
     * 测试根据 ID 查询
     */
    @Test
    public void testFindById() {
        User user = session.get(User.class, 1L);
        System.out.println(user);
    }
    
    /**
     * 测试更新用户
     */
    @Test
    public void testUpdate() {
        Transaction transaction = session.beginTransaction();
        
        try {
            // 先查询
            User user = session.get(User.class, 1L);
            
            // 修改属性
            user.setUsername("新名字");
            user.setAge(26);
            
            // 更新（Hibernate 自动检测变化）
            session.update(user);
            
            transaction.commit();
            System.out.println("更新成功");
        } catch (Exception e) {
            transaction.rollback();
            e.printStackTrace();
        }
    }
    
    /**
     * 测试删除用户
     */
    @Test
    public void testDelete() {
        Transaction transaction = session.beginTransaction();
        
        try {
            User user = session.get(User.class, 5L);
            if (user != null) {
                session.delete(user);
            }
            
            transaction.commit();
            System.out.println("删除成功");
        } catch (Exception e) {
            transaction.rollback();
            e.printStackTrace();
        }
    }
    
    /**
     * 清理资源
     */
    @After
    public void destroy() {
        if (session != null && session.isOpen()) {
            session.close();
        }
        if (sessionFactory != null) {
            sessionFactory.close();
        }
    }
}
```

### 5.2 组件详解

#### 5.2.1 Configuration

* **作用**：读取配置文件，创建 SessionFactory
* **生命周期**：应用启动时创建一次
* **使用方式**：

```java
Configuration configuration = new Configuration().configure();
SessionFactory sessionFactory = configuration.buildSessionFactory();
```

#### 5.2.2 SessionFactory

* **作用**：创建 Session 的工厂
* **生命周期**：应用级别，整个应用只创建一个实例
* **特点**：线程安全，重量级对象
* **使用方式**：

```java
SessionFactory sessionFactory = configuration.buildSessionFactory();
Session session = sessionFactory.openSession();
```

#### 5.2.3 Session

* **作用**：与数据库交互的主要接口
* **生命周期**：请求级别，每次操作创建一个新的 Session
* **特点**：**非线程安全**，不能共享
* **主要方法**：
  - `save()`：保存对象
  - `get()` / `load()`：查询对象
  - `update()`：更新对象
  - `delete()`：删除对象
  - `createQuery()`：创建 HQL 查询

```java
Session session = sessionFactory.openSession();
User user = session.get(User.class, 1L);
session.close();
```

#### 5.2.4 Transaction

* **作用**：管理事务
* **主要方法**：
  - `begin()`：开始事务
  - `commit()`：提交事务
  - `rollback()`：回滚事务

```java
Transaction transaction = session.beginTransaction();
try {
    // 执行数据库操作
    session.save(user);
    transaction.commit();
} catch (Exception e) {
    transaction.rollback();
}
```

#### 5.2.5 Query

* **作用**：执行 HQL 或 SQL 查询
* **使用方式**：

```java
// HQL 查询
Query<User> query = session.createQuery("FROM User WHERE age > :age", User.class);
query.setParameter("age", 18);
List<User> users = query.list();

// 分页查询
query.setFirstResult(0);
query.setMaxResults(10);
```

## 6. Hibernate 工作流程

### 6.1 工作流程步骤

1. **读取配置**：加载 `hibernate.cfg.xml` 配置文件
2. **创建 SessionFactory**：解析配置，构建工厂对象
3. **打开 Session**：通过工厂创建会话
4. **开启事务**：开始数据库事务
5. **执行操作**：保存、查询、更新、删除
6. **提交事务**：提交或回滚事务
7. **关闭 Session**：释放资源

### 6.2 对象状态转换

<!-- TODO: 插入图片：Hibernate 对象状态转换图 -->

**三种状态**：

1. **瞬时态（Transient）**
   - 对象刚创建，未与 Session 关联
   - 数据库中没有对应记录
   
2. **持久态（Persistent）**
   - 对象与 Session 关联
   - 数据库中有对应记录
   - 属性变化会自动同步到数据库

3. **游离态（Detached）**
   - Session 关闭后，对象变为游离态
   - 数据库中有对应记录
   - 属性变化不会同步到数据库

**状态转换**：

```java
// 瞬时态
User user = new User();
user.setUsername("张三");

// 持久态（调用 save 后）
session.save(user);

// 游离态（Session 关闭后）
session.close();
```

## 7. 常见问题与解决方案

### 7.1 找不到配置文件

**错误信息**：
```
Could not locate cfg.xml resource [hibernate.cfg.xml]
```

**解决方案**：
确保 `hibernate.cfg.xml` 放在 `src/main/resources` 目录下。

### 7.2 数据库连接失败

**错误信息**：
```
Could not open connection
Communications link failure
```

**解决方案**：
1. 检查 MySQL 服务是否启动
2. 检查数据库 URL、用户名、密码是否正确
3. 检查 MySQL 驱动版本是否与数据库版本匹配

### 7.3 表不存在

**错误信息**：
```
Table 'hibernate_demo.user' doesn't exist
```

**解决方案**：
设置 `hibernate.hbm2ddl.auto` 为 `update` 或 `create`：

```xml
<property name="hibernate.hbm2ddl.auto">update</property>
```

### 7.4 事务未提交

**问题**：数据未保存到数据库

**原因**：忘记提交事务

**解决方案**：
```java
Transaction transaction = session.beginTransaction();
// 执行操作
session.save(user);
transaction.commit();  // 必须提交
```

## 9. 总结

### 9.1 核心知识点

✅ **Hibernate 是什么**：JPA 实现的 ORM 框架  
✅ **与 MyBatis 对比**：全自动 vs 半自动，开发效率高 vs 灵活性强  
✅ **环境搭建**：Maven 依赖、配置文件、实体类注解  
✅ **核心组件**：Configuration、SessionFactory、Session、Transaction  
✅ **工作流程**：配置 → 工厂 → 会话 → 事务 → 操作 → 提交  

### 9.2 关键配置

* **hibernate.cfg.xml**：核心配置文件
* **数据库方言**：根据不同数据库选择
* **hbm2ddl.auto**：自动建表策略
* **show_sql**：开发时开启，生产时关闭

### 9.3 注意事项

⚠️ Session 是非线程安全的，不能共享  
⚠️ 必须手动管理事务（commit/rollback）  
⚠️ Session 使用后要及时关闭  
⚠️ 生产环境不要使用 `create` 或 `update` 自动建表
