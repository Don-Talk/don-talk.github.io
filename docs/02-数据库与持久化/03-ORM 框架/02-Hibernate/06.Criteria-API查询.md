## 1. Criteria API 简介

### 1.1 什么是 Criteria API？

Criteria API 是 JPA 2.0 引入的类型安全的查询 API，通过 Java 代码构建查询，而非字符串拼接。

**优势**：
* ✅ **类型安全**：编译时检查，避免运行时错误
* ✅ **动态查询**：易于构建动态条件
* ✅ **重构友好**：IDE 支持自动补全和重构

**劣势**：
* ❌ 代码较冗长
* ❌ 复杂查询可读性较差

### 1.2 核心组件

* **CriteriaBuilder**：构建查询的工厂
* **CriteriaQuery**：查询对象
* **Root**：查询根实体
* **Predicate**：查询条件

## 2. 基础查询

### 2.1 查询所有

```java
@Test
public void testFindAll() {
    Session session = sessionFactory.openSession();
    
    // 获取 CriteriaBuilder
    CriteriaBuilder cb = session.getCriteriaBuilder();
    
    // 创建查询
    CriteriaQuery<User> query = cb.createQuery(User.class);
    
    // 设置查询根
    Root<User> root = query.from(User.class);
    
    // 执行查询
    List<User> users = session.createQuery(query).list();
    
    users.forEach(System.out::println);
    
    session.close();
}
```

### 2.2 简化写法

```java
@Test
public void testSimpleQuery() {
    Session session = sessionFactory.openSession();
    
    CriteriaBuilder cb = session.getCriteriaBuilder();
    CriteriaQuery<User> query = cb.createQuery(User.class);
    query.from(User.class);  // 默认选择所有字段
    
    List<User> users = session.createQuery(query).list();
    
    session.close();
}
```

### 3.2 Lambda 表达式（推荐）


### 3.3 比较条件

```java
@Test
public void testComparison() {
    Session session = sessionFactory.openSession();
    
    CriteriaBuilder cb = session.getCriteriaBuilder();
    CriteriaQuery<User> query = cb.createQuery(User.class);
    Root<User> root = query.from(User.class);
    
    // age > 20
    Predicate predicate = cb.gt(root.get("age"), 20);
    query.where(predicate);
    
    List<User> users = session.createQuery(query).list();
    
    session.close();
}
```

**常用比较方法**：

| 方法 | 说明 | SQL |
|------|------|-----|
| `cb.equal()` | 等于 | = |
| `cb.notEqual()` | 不等于 | <> |
| `cb.gt()` | 大于 | > |
| `cb.ge()` | 大于等于 | >= |
| `cb.lt()` | 小于 | < |
| `cb.le()` | 小于等于 | <= |
| `cb.between()` | BETWEEN | BETWEEN |
| `cb.like()` | LIKE | LIKE |
| `cb.in()` | IN | IN |
| `cb.isNull()` | IS NULL | IS NULL |
| `cb.isNotNull()` | IS NOT NULL | IS NOT NULL |

## 4. 动态查询

### 4.1 动态条件构建

```java
public List<User> findByCondition(String username, Integer minAge, Integer maxAge) {
    Session session = sessionFactory.openSession();
    
    try {
        CriteriaBuilder cb = session.getCriteriaBuilder();
        CriteriaQuery<User> query = cb.createQuery(User.class);
        Root<User> root = query.from(User.class);
        
        // 动态构建条件列表
        List<Predicate> predicates = new ArrayList<>();
        
        if (username != null && !username.isEmpty()) {
            predicates.add(cb.like(root.get("username"), "%" + username + "%"));
        }
        
        if (minAge != null) {
            predicates.add(cb.ge(root.get("age"), minAge));
        }
        
        if (maxAge != null) {
            predicates.add(cb.le(root.get("age"), maxAge));
        }
        
        // 应用条件
        if (!predicates.isEmpty()) {
            query.where(cb.and(predicates.toArray(new Predicate[0])));
        }
        
        return session.createQuery(query).list();
        
    } finally {
        session.close();
    }
}
```

### 4.2 复杂动态查询

```java
public List<User> advancedSearch(String keyword, Integer gender, 
                                  Date startDate, Date endDate) {
    Session session = sessionFactory.openSession();
    
    try {
        CriteriaBuilder cb = session.getCriteriaBuilder();
        CriteriaQuery<User> query = cb.createQuery(User.class);
        Root<User> root = query.from(User.class);
        
        List<Predicate> predicates = new ArrayList<>();
        
        // 关键字搜索（用户名或邮箱）
        if (keyword != null && !keyword.isEmpty()) {
            Predicate p1 = cb.like(root.get("username"), "%" + keyword + "%");
            Predicate p2 = cb.like(root.get("email"), "%" + keyword + "%");
            predicates.add(cb.or(p1, p2));
        }
        
        // 性别
        if (gender != null) {
            predicates.add(cb.equal(root.get("gender"), gender));
        }
        
        // 注册时间范围
        if (startDate != null) {
            predicates.add(cb.ge(root.get("createTime"), startDate));
        }
        if (endDate != null) {
            predicates.add(cb.le(root.get("createTime"), endDate));
        }
        
        if (!predicates.isEmpty()) {
            query.where(cb.and(predicates.toArray(new Predicate[0])));
        }
        
        // 按创建时间降序
        query.orderBy(cb.desc(root.get("createTime")));
        
        return session.createQuery(query).list();
        
    } finally {
        session.close();
    }
}
```

## 7. 投影查询（Projection）

### 7.1 选择特定字段

```java
public void testSelectFields() {
```

### 7.2 使用 Tuple

```java
@Test
public void testTuple() {
    Session session = sessionFactory.openSession();
    
    CriteriaBuilder cb = session.getCriteriaBuilder();
    CriteriaQuery<Tuple> query = cb.createTupleQuery();
    Root<User> root = query.from(User.class);

        System.out.println("邮箱: " + tuple.get("email"));
    }
    
    session.close();
```

## 8. 关联查询

### 8.1 JOIN

```java
@Test
public void testJoin() {
    Session session = sessionFactory.openSession();
    
    CriteriaBuilder cb = session.getCriteriaBuilder();
    CriteriaQuery<Order> query = cb.createQuery(Order.class);
    
    Root<Order> orderRoot = query.from(Order.class);
    
    // JOIN User
    Join<Order, User> userJoin = orderRoot.join("user", JoinType.INNER);

    List<Order> orders = session.createQuery(query).list();
    
    session.close();
}
```

### 8.2 FETCH JOIN

```java
@Test
public void testFetchJoin() {
    Session session = sessionFactory.openSession();
    
    CriteriaBuilder cb = session.getCriteriaBuilder();
    CriteriaQuery<User> query = cb.createQuery(User.class);
    
    Root<User> root = query.from(User.class);
    
    // FETCH JOIN 加载订单

        System.out.println("用户: " + user.getUsername());
        System.out.println("订单数: " + user.getOrders().size());
    }
    
    session.close();
}
```

## 9. 子查询

```java

public void testSubquery() {

    session.close();
}
```

## 10. 完整示例

### 10.1 通用查询工具类

```java
public class CriteriaHelper {
    
    /**
     * 分页查询
     */
    public static <T> List<T> findPage(Session session, Class<T> entityClass,
                                        List<Predicate> predicates,
                                        List<Order> orders,
                                        int pageNum, int pageSize) {
        CriteriaBuilder cb = session.getCriteriaBuilder();
        CriteriaQuery<T> query = cb.createQuery(entityClass);
        Root<T> root = query.from(entityClass);
        
        if (predicates != null && !predicates.isEmpty()) {
            query.where(cb.and(predicates.toArray(new Predicate[0])));
        }
        
        if (orders != null && !orders.isEmpty()) {
            query.orderBy(orders);
        }
        
        return session.createQuery(query)
                .setFirstResult((pageNum - 1) * pageSize)
                .setMaxResults(pageSize)
                .list();
    }
    
    /**
     * 统计总数
     */
    public static <T> Long count(Session session, Class<T> entityClass,
                                  List<Predicate> predicates) {
        CriteriaBuilder cb = session.getCriteriaBuilder();
        CriteriaQuery<Long> query = cb.createQuery(Long.class);
        Root<T> root = query.from(entityClass);
        query.select(cb.count(root));
        
        if (predicates != null && !predicates.isEmpty()) {
            query.where(cb.and(predicates.toArray(new Predicate[0])));
        }
        
        return session.createQuery(query).getSingleResult();
    }
}
```

## 11. Criteria vs HQL 对比

| 特性 | Criteria API | HQL |
|------|-------------|-----|
| **类型安全** | ✅ 是 | ❌ 否 |
| **动态查询** | ✅ 容易 | ⚠️ 需拼接字符串 |
| **可读性** | ⚠️ 代码冗长 | ✅ 类似 SQL |
| **复杂查询** | ⚠️ 较复杂 | ✅ 更直观 |
| **IDE 支持** | ✅ 自动补全 | ⚠️ 有限 |
| **学习曲线** | ⚠️ 较陡峭 | ✅ 平缓 |

**选择建议**：
* **简单查询**：HQL 更简洁
* **动态查询**：Criteria API 更安全
* **复杂查询**：HQL 更直观
* **团队熟悉度**：根据团队技术栈选择

## 13. 总结

### 13.1 核心知识点

✅ **CriteriaBuilder**：构建查询  
✅ **条件查询**：equal、gt、lt、like、in  
✅ **动态查询**：`List<Predicate>`  
✅ **排序分页**：orderBy、setFirstResult、setMaxResults  
✅ **投影查询**：select、tuple、array  
✅ **聚合函数**：count、avg、sum、max、min  
✅ **关联查询**：join、fetch  
✅ **子查询**：subquery  

### 13.2 最佳实践

📌 动态查询优先使用 Criteria API  
📌 复杂查询使用 HQL 更直观  
📌 使用 FETCH JOIN 解决 N+1 问题  
📌 封装通用查询工具类  
📌 合理使用投影减少数据传输
