## 1. 动态 SQL 概述

### 1.1 什么是动态 SQL？

动态 SQL 是 MyBatis 的强大特性之一，可以根据不同的条件动态生成 SQL 语句。

**传统 JDBC 的问题**：
```java
// 需要手动拼接 SQL，容易出错
String sql = "SELECT * FROM user WHERE 1=1";
if (username != null) {
    sql += " AND username LIKE '%" + username + "%'";
}
if (age != null) {
    sql += " AND age = " + age;
}
```

**MyBatis 动态 SQL**：
```xml
<select id="findByCondition" resultType="User">
    SELECT * FROM user
    <where>
        <if test="username != null and username != ''">
            AND username LIKE CONCAT('%', #{username}, '%')
        </if>
        <if test="age != null">
            AND age = #{age}
        </if>
    </where>
</select>
```

### 1.2 动态 SQL 标签总览

| 标签 | 作用 | 类似 Java |
|------|------|----------|
| `<if>` | 条件判断 | if |
| `<choose>` | 多选一 | switch-case |
| `<when>` | 条件分支 | case |
| `<otherwise>` | 默认分支 | default |
| `<where>` | 智能 WHERE | - |
| `<set>` | 智能 SET | - |
| `<foreach>` | 循环遍历 | for-each |
| `<trim>` | 自定义处理 | - |
| `<bind>` | 创建变量 | - |

## 2. 条件查询实战

### 2.1 单条件查询

```xml
<select id="findByUsername" resultType="User">
    SELECT * FROM user
    <where>
        <if test="username != null and username != ''">
            AND username = #{username}
        </if>
    </where>
</select>
```

### 2.2 多条件组合查询

#### 场景：用户搜索功能

支持按用户名、邮箱、年龄范围、创建时间等多个条件组合查询。

```xml
<select id="searchUsers" resultType="User">
    SELECT * FROM user
    <where>
        <!-- 用户名模糊查询 -->
        <if test="username != null and username != ''">
            AND username LIKE CONCAT('%', #{username}, '%')
        </if>
        
        <!-- 邮箱精确查询 -->
        <if test="email != null and email != ''">
            AND email = #{email}
        </if>
        
        <!-- 最小年龄 -->
        <if test="minAge != null">
            AND age &gt;= #{minAge}
        </if>
        
        <!-- 最大年龄 -->
        <if test="maxAge != null">
            AND age &lt;= #{maxAge}
        </if>
        
        <!-- 开始时间 -->
        <if test="startTime != null">
            AND create_time &gt;= #{startTime}
        </if>
        
        <!-- 结束时间 -->
        <if test="endTime != null">
            AND create_time &lt;= #{endTime}
        </if>
    </where>
    ORDER BY create_time DESC
</select>
```

**Mapper 接口**：
```java
List<User> searchUsers(@Param("username") String username,
                       @Param("email") String email,
                       @Param("minAge") Integer minAge,
                       @Param("maxAge") Integer maxAge,
                       @Param("startTime") Date startTime,
                       @Param("endTime") Date endTime);
```

**使用示例**：
```java
// 只按用户名查询
List<User> users = mapper.searchUsers("张", null, null, null, null, null);

// 按年龄范围查询
List<User> users = mapper.searchUsers(null, null, 20, 30, null, null);

// 多条件组合
Date start = new SimpleDateFormat("yyyy-MM-dd").parse("2024-01-01");
Date end = new SimpleDateFormat("yyyy-MM-dd").parse("2024-12-31");
List<User> users = mapper.searchUsers("张", null, 20, 30, start, end);
```

### 2.3 选择性查询（多选一）

#### 场景：优先按用户名查，其次按邮箱查，最后按年龄查

```xml
<select id="searchByPriority" resultType="User">
    SELECT * FROM user
    <where>
        <choose>
            <when test="username != null and username != ''">
                AND username = #{username}
            </when>
            <when test="email != null and email != ''">
                AND email = #{email}
            </when>
            <when test="age != null">
                AND age = #{age}
            </when>
            <otherwise>
                AND 1 = 0  <!-- 无条件时返回空结果 -->
            </otherwise>
        </choose>
    </where>
</select>
```

## 3. 模糊查询最佳实践

### 3.1 方式一：CONCAT 函数（推荐）

```xml
<select id="findByKeyword" resultType="User">
    SELECT * FROM user
    WHERE username LIKE CONCAT('%', #{keyword}, '%')
</select>
```

**优点**：
* 防止 SQL 注入
* 兼容性好

### 3.2 方式二：bind 标签

```xml
<select id="findByKeyword" resultType="User">
    <bind name="pattern" value="'%' + keyword + '%'"/>
    SELECT * FROM user
    WHERE username LIKE #{pattern}
</select>
```

**优点**：
* 可以在多处复用
* 更灵活

### 3.3 方式三：${} 拼接（不推荐）

```xml
<select id="findByKeyword" resultType="User">
    SELECT * FROM user
    WHERE username LIKE '%${keyword}%'
</select>
```

**缺点**：
* 存在 SQL 注入风险
* 不安全

### 3.4 多字段模糊查询

```xml
<select id="searchByKeyword" resultType="User">
    SELECT * FROM user
    <where>
        <if test="keyword != null and keyword != ''">
            AND (
                username LIKE CONCAT('%', #{keyword}, '%')
                OR email LIKE CONCAT('%', #{keyword}, '%')
                OR phone LIKE CONCAT('%', #{keyword}, '%')
            )
        </if>
    </where>
</select>
```

## 4. 分页查询

### 4.1 LIMIT 分页（MySQL）

```xml
<select id="findByPage" resultType="User">
    SELECT * FROM user
    <where>
        <if test="username != null and username != ''">
            AND username LIKE CONCAT('%', #{username}, '%')
        </if>
    </where>
    ORDER BY id DESC
    LIMIT #{offset}, #{pageSize}
</select>
```

**使用**：
```java
// 第 1 页，每页 10 条
int pageNum = 1;
int pageSize = 10;
int offset = (pageNum - 1) * pageSize;

List<User> users = mapper.findByPage(null, offset, pageSize);
```

### 4.2 分页插件（推荐）

使用 PageHelper 插件简化分页。

**步骤 1**：添加依赖

```xml
<dependency>
    <groupId>com.github.pagehelper</groupId>
    <artifactId>pagehelper-spring-boot-starter</artifactId>
    <version>1.4.7</version>
</dependency>
```

**步骤 2**：配置（Spring Boot）

```yaml
pagehelper:
  helper-dialect: mysql
  reasonable: true
  support-methods-arguments: true
```

**步骤 3**：使用

```java
// 不需要修改 Mapper XML
PageHelper.startPage(1, 10);  // 第1页，每页10条
List<User> users = mapper.findAll();
PageInfo<User> pageInfo = new PageInfo<>(users);

System.out.println("总记录数：" + pageInfo.getTotal());
System.out.println("总页数：" + pageInfo.getPages());
System.out.println("当前页：" + pageInfo.getPageNum());
```

## 5. 批量操作

### 5.1 批量插入

#### 方式一：VALUES 批量插入（推荐）

```xml
<insert id="batchInsert">
    INSERT INTO user (username, email, age) VALUES
    <foreach collection="list" item="user" separator=",">
        (#{user.username}, #{user.email}, #{user.age})
    </foreach>
</insert>
```

**使用**：
```java
List<User> users = Arrays.asList(
    new User("张三", "zhangsan@example.com", 25),
    new User("李四", "lisi@example.com", 28),
    new User("王五", "wangwu@example.com", 22)
);

mapper.batchInsert(users);
```

**生成 SQL**：
```sql
INSERT INTO user (username, email, age) VALUES
('张三', 'zhangsan@example.com', 25),
('李四', 'lisi@example.com', 28),
('王五', 'wangwu@example.com', 22)
```

#### 方式二：逐条插入（性能较差）

```xml
<insert id="batchInsertOneByOne">
    <foreach collection="list" item="user" separator=";">
        INSERT INTO user (username, email, age) 
        VALUES (#{user.username}, #{user.email}, #{user.age})
    </foreach>
</insert>
```

**注意**：需要在 JDBC URL 中添加 `allowMultiQueries=true`

### 5.2 批量更新

#### 方式一：CASE WHEN 批量更新

```xml
<update id="batchUpdate">
    UPDATE user
    <set>
        username = CASE id
        <foreach collection="list" item="user">
            WHEN #{user.id} THEN #{user.username}
        </foreach>
        END,
        email = CASE id
        <foreach collection="list" item="user">
            WHEN #{user.id} THEN #{user.email}
        </foreach>
        END
    </set>
    WHERE id IN
    <foreach collection="list" item="user" open="(" separator="," close=")">
        #{user.id}
    </foreach>
</update>
```

**生成 SQL**：
```sql
UPDATE user
SET username = CASE id
    WHEN 1 THEN '张三'
    WHEN 2 THEN '李四'
END,
email = CASE id
    WHEN 1 THEN 'zhangsan@example.com'
    WHEN 2 THEN 'lisi@example.com'
END
WHERE id IN (1, 2)
```

#### 方式二：多条 UPDATE 语句

```xml
<update id="batchUpdateMulti">
    <foreach collection="list" item="user" separator=";">
        UPDATE user 
        SET username = #{user.username}, email = #{user.email}
        WHERE id = #{user.id}
    </foreach>
</update>
```

### 5.3 批量删除

```xml
<delete id="batchDelete">
    DELETE FROM user WHERE id IN
    <foreach collection="list" item="id" open="(" separator="," close=")">
        #{id}
    </foreach>
</delete>
```

**使用**：
```java
List<Integer> ids = Arrays.asList(1, 2, 3, 4, 5);
mapper.batchDelete(ids);
```

## 6. 复杂动态 SQL 案例

### 6.1 案例一：高级搜索

**需求**：支持多条件、多排序、分页的复杂搜索。

```xml
<select id="advancedSearch" resultType="User">
    SELECT * FROM user
    <where>
        <!-- 关键字搜索 -->
        <if test="keyword != null and keyword != ''">
            AND (
                username LIKE CONCAT('%', #{keyword}, '%')
                OR email LIKE CONCAT('%', #{keyword}, '%')
            )
        </if>
        
        <!-- 年龄范围 -->
        <if test="minAge != null">
            AND age &gt;= #{minAge}
        </if>
        <if test="maxAge != null">
            AND age &lt;= #{maxAge}
        </if>
        
        <!-- 注册时间范围 -->
        <if test="startDate != null">
            AND create_time &gt;= #{startDate}
        </if>
        <if test="endDate != null">
            AND create_time &lt;= #{endDate}
        </if>
        
        <!-- 状态筛选 -->
        <if test="status != null">
            AND status = #{status}
        </if>
    </where>
    
    <!-- 动态排序 -->
    ORDER BY
    <choose>
        <when test="orderBy == 'age'">
            age
        </when>
        <when test="orderBy == 'createTime'">
            create_time
        </when>
        <otherwise>
            id
        </otherwise>
    </choose>
    <choose>
        <when test="orderType == 'asc'">
            ASC
        </when>
        <otherwise>
            DESC
        </otherwise>
    </choose>
    
    <!-- 分页 -->
    LIMIT #{offset}, #{pageSize}
</select>
```

### 6.2 案例二：报表统计

**需求**：根据条件统计用户数量、平均年龄等。

```xml
<select id="userStatistics" resultType="map">
    SELECT 
        COUNT(*) AS totalCount,
        AVG(age) AS avgAge,
        MAX(age) AS maxAge,
        MIN(age) AS minAge,
        SUM(CASE WHEN age &lt; 18 THEN 1 ELSE 0 END) AS underageCount,
        SUM(CASE WHEN age &gt;= 18 AND age &lt; 30 THEN 1 ELSE 0 END) AS youngCount,
        SUM(CASE WHEN age &gt;= 30 AND age &lt; 50 THEN 1 ELSE 0 END) AS middleCount,
        SUM(CASE WHEN age &gt;= 50 THEN 1 ELSE 0 END) AS seniorCount
    FROM user
    <where>
        <if test="startDate != null">
            AND create_time &gt;= #{startDate}
        </if>
        <if test="endDate != null">
            AND create_time &lt;= #{endDate}
        </if>
        <if test="gender != null">
            AND gender = #{gender}
        </if>
    </where>
</select>
```

**使用**：
```java
Map<String, Object> stats = mapper.userStatistics(startDate, endDate, null);
System.out.println("总人数：" + stats.get("totalCount"));
System.out.println("平均年龄：" + stats.get("avgAge"));
```

### 6.3 案例三：数据导入

**需求**：从 Excel 导入用户数据，如果用户已存在则更新，不存在则插入。

```xml
<!-- 检查用户是否存在 -->
<select id="countByUsername" resultType="int">
    SELECT COUNT(*) FROM user WHERE username = #{username}
</select>

<!-- 插入新用户 -->
<insert id="insert">
    INSERT INTO user (username, email, age) 
    VALUES (#{username}, #{email}, #{age})
</insert>

<!-- 更新已有用户 -->
<update id="updateByEmail">
    UPDATE user SET email = #{email}, age = #{age}
    WHERE username = #{username}
</update>
```

**Service 层逻辑**：
```java
public void importUsers(List<User> users) {
    for (User user : users) {
        int count = mapper.countByUsername(user.getUsername());
        if (count > 0) {
            // 用户存在，更新
            mapper.updateByEmail(user);
        } else {
            // 用户不存在，插入
            mapper.insert(user);
        }
    }
}
```

## 7. 性能优化

### 7.1 避免 N+1 查询问题

**问题**：在循环中执行查询

```java
// 错误示例：N+1 查询
List<User> users = mapper.findAll();
for (User user : users) {
    List<Order> orders = mapper.findOrdersByUserId(user.getId());
    user.setOrders(orders);
}
```

**解决**：使用 JOIN 一次性查询

```xml
<select id="findAllWithOrders" resultMap="UserWithOrdersResultMap">
    SELECT u.*, o.*
    FROM user u
    LEFT JOIN orders o ON u.id = o.user_id
</select>
```

### 7.2 合理使用索引

```xml
<!-- 确保查询条件使用索引字段 -->
<select id="findByUsername" resultType="User">
    SELECT * FROM user WHERE username = #{username}
</select>
```

**建议**：
* 为常用查询条件创建索引
* 避免在索引列上使用函数
* 避免使用 `LIKE '%xxx%'`（无法使用索引）

### 7.3 限制查询结果集

```xml
<!-- 只查询需要的字段 -->
<select id="findUsernames" resultType="string">
    SELECT username FROM user WHERE age &gt;= #{age}
</select>

<!-- 使用 LIMIT 限制返回数量 -->
<select id="findTop10" resultType="User">
    SELECT * FROM user ORDER BY create_time DESC LIMIT 10
</select>
```

### 7.4 批量操作优化

```java
// 分批处理大数据量
int batchSize = 1000;
for (int i = 0; i < allUsers.size(); i += batchSize) {
    List<User> batch = allUsers.subList(i, Math.min(i + batchSize, allUsers.size()));
    mapper.batchInsert(batch);
}
```

## 8. 常见问题与解决方案

### 8.1 动态 SQL 不生效

**问题**：条件判断始终为 false

**原因**：参数传递错误

**解决**：
```java
// 错误：单个基本类型参数不能使用属性名
List<User> findByAge(int age);

// 正确：使用 @Param 或包装类
List<User> findByAge(@Param("age") int age);
List<User> findByAge(Integer age);
```

### 8.2 foreach 遍历失败

**问题**：collection 属性指定错误

**解决**：
```java
// 传入 List
List<User> batchInsert(List<User> users);
// XML: collection="list"

// 传入数组
List<User> batchInsert(User[] users);
// XML: collection="array"

// 传入 Map
List<User> batchInsert(@Param("userList") List<User> users);
// XML: collection="userList"
```

### 8.3 特殊字符转义

**问题**：XML 解析错误

**解决**：
```xml
<!-- 方式一：使用实体引用 -->
&lt; &gt; &amp; &quot; &apos;

<!-- 方式二：使用 CDATA -->
<![CDATA[
    SELECT * FROM user WHERE age > 18 AND age < 60
]]>
```

✅ **条件查询**：if、choose、where 标签
