> * 能够根据场景选择合适的开发方式

## 1. Mapper 接口动态代理

### 1.1 工作原理

MyBatis 通过 JDK 动态代理为 Mapper 接口生成代理对象，无需编写实现类。

```java
// 传统方式：需要编写实现类
UserDao userDao = new UserDaoImpl();
List<User> users = userDao.findAll();

// MyBatis 方式：自动生成代理对象
UserMapper mapper = session.getMapper(UserMapper.class);
List<User> users = mapper.findAll();
```

### 1.2 代理规则

要让 MyBatis 正确生成代理对象，必须满足以下条件：

1. **namespace 必须与接口全限定名一致**
   ```xml
   <mapper namespace="com.example.mapper.UserMapper">
   ```

2. **SQL 标签的 id 必须与接口方法名一致**
   ```xml
   <select id="findAll" resultType="User">
   ```
   ```java
   List<User> findAll();
   ```

3. **参数类型必须匹配**
   ```xml
   <select id="findById" parameterType="int" resultType="User">
   ```
   ```java
   User findById(Integer id);
   ```

4. **返回值类型必须匹配**
   ```xml
   <select id="findAll" resultType="User">
   ```
   ```java
   List<User> findAll();
   ```

### 1.3 底层原理

```
session.getMapper(UserMapper.class)
    ↓
MapperProxy (JDK 动态代理)
    ↓
MapperMethod (解析方法签名)
    ↓
Executor (执行 SQL)
    ↓
返回结果
```

## 2. 注解开发基础

### 2.1 基本 CRUD 注解

#### 2.1.1 @Select（查询）

```java
public interface UserMapper {
    
    // 查询所有用户
    @Select("SELECT * FROM user")
    List<User> findAll();
    
    // 根据 ID 查询
    @Select("SELECT * FROM user WHERE id = #{id}")
    User findById(Integer id);
    
    // 条件查询
    @Select("SELECT * FROM user WHERE username = #{username}")
    User findByUsername(String username);
}
```

#### 2.1.2 @Insert（插入）

```java
public interface UserMapper {
    
    // 插入用户
    @Insert("INSERT INTO user (username, email, age) VALUES (#{username}, #{email}, #{age})")
    int insert(User user);
    
    // 插入并返回自增主键
    @Insert("INSERT INTO user (username, email, age) VALUES (#{username}, #{email}, #{age})")
    @Options(useGeneratedKeys = true, keyProperty = "id")
    int insertWithKey(User user);
}
```

**使用**：
```java
User user = new User();
user.setUsername("张三");
user.setEmail("zhangsan@example.com");
user.setAge(25);

mapper.insertWithKey(user);
System.out.println("生成的ID：" + user.getId());
```

#### 2.1.3 @Update（更新）

```java
public interface UserMapper {
    
    // 更新用户
    @Update("UPDATE user SET username = #{username}, email = #{email}, age = #{age} WHERE id = #{id}")
    int update(User user);
    
    // 只更新用户名
    @Update("UPDATE user SET username = #{username} WHERE id = #{id}")
    int updateUsername(@Param("id") Integer id, @Param("username") String username);
}
```

#### 2.1.4 @Delete（删除）

```java
public interface UserMapper {
    
    // 根据 ID 删除
    @Delete("DELETE FROM user WHERE id = #{id}")
    int delete(Integer id);
    
    // 批量删除
    @Delete("<script>" +
            "DELETE FROM user WHERE id IN " +
            "<foreach collection='ids' item='id' open='(' separator=',' close=')'>" +
            "#{id}" +
            "</foreach>" +
            "</script>")
    int batchDelete(@Param("ids") List<Integer> ids);
}
```

**注意**：动态 SQL 需要使用 `<script>` 标签包裹。

### 2.2 参数注解 @Param

当方法有多个参数时，需要使用 `@Param` 指定参数名称。

```java
public interface UserMapper {
    
    // 单个参数可以不加 @Param
    User findById(Integer id);
    
    // 多个参数必须加 @Param
    @Select("SELECT * FROM user WHERE username = #{username} AND age = #{age}")
    User findByCondition(@Param("username") String username, 
                         @Param("age") Integer age);
}
```

**不使用 @Param 的情况**：
* 只有一个参数
* 参数是 JavaBean 对象

**必须使用 @Param 的情况**：
* 多个基本类型参数
* 需要在 SQL 中引用参数名

## 3. 复杂映射注解

### 3.1 @Results（结果映射）

类似于 XML 中的 `<resultMap>`。

```java
public interface UserMapper {
    
    @Select("SELECT user_id, user_name, user_email FROM user WHERE id = #{id}")
    @Results({
        @Result(property = "id", column = "user_id", id = true),
        @Result(property = "username", column = "user_name"),
        @Result(property = "email", column = "user_email")
    })
    User findById(Integer id);
}
```

**属性说明**：
* `property`：Java 属性名
* `column`：数据库列名
* `id`：是否为主键

### 3.2 @ResultMap（引用结果映射）

如果多个方法使用相同的映射规则，可以定义公共的 ResultMap。

```java
public interface UserMapper {
    
    // 定义 ResultMap
    @Results(id = "userResultMap", value = {
        @Result(property = "id", column = "user_id", id = true),
        @Result(property = "username", column = "user_name"),
        @Result(property = "email", column = "user_email")
    })
    @Select("SELECT user_id, user_name, user_email FROM user WHERE id = #{id}")
    User findById(Integer id);
    
    // 引用 ResultMap
    @ResultMap("userResultMap")
    @Select("SELECT user_id, user_name, user_email FROM user WHERE username = #{username}")
    User findByUsername(String username);
    
    // 再次引用
    @ResultMap("userResultMap")
    @Select("SELECT user_id, user_name, user_email FROM user")
    List<User> findAll();
}
```

### 3.3 一对一映射 @One

```java
public interface OrderMapper {
    
    @Select("SELECT * FROM orders WHERE id = #{id}")
    @Results({
        @Result(property = "id", column = "id", id = true),
        @Result(property = "orderNo", column = "order_no"),
        @Result(property = "amount", column = "amount"),
        @Result(property = "userId", column = "user_id"),
        @Result(property = "user", column = "user_id", 
                one = @One(select = "com.example.mapper.UserMapper.findById"))
    })
    Order findOrderWithUser(Integer id);
}
```

**对应的 UserMapper**：
```java
public interface UserMapper {
    
    @Select("SELECT * FROM user WHERE id = #{id}")
    User findById(Integer id);
}
```

**说明**：
* `column = "user_id"`：将 user_id 作为参数传递给 select 方法
* `one = @One(...)`：一对一关联
* `select`：指定调用的查询方法

### 3.4 一对多映射 @Many

```java
public interface UserMapper {
    
    @Select("SELECT * FROM user WHERE id = #{id}")
    @Results({
        @Result(property = "id", column = "id", id = true),
        @Result(property = "username", column = "username"),
        @Result(property = "orders", column = "id", 
                many = @Many(select = "com.example.mapper.OrderMapper.findByUserId"))
    })
    User findUserWithOrders(Integer id);
}
```

**对应的 OrderMapper**：
```java
public interface OrderMapper {
    
    @Select("SELECT * FROM orders WHERE user_id = #{userId}")
    List<Order> findByUserId(Integer userId);
}
```

**说明**：
* `column = "id"`：将用户 ID 作为参数传递
* `many = @Many(...)`：一对多关联

## 4. 动态 SQL 注解

### 4.1 使用 `<script>` 标签

在注解中使用动态 SQL，需要用 `<script>` 包裹。

```java
public interface UserMapper {
    
    @Select("<script>" +
            "SELECT * FROM user" +
            "<where>" +
            "  <if test='username != null and username != \"\"'>" +
            "    AND username LIKE CONCAT('%', #{username}, '%')" +
            "  </if>" +
            "  <if test='age != null'>" +
            "    AND age = #{age}" +
            "  </if>" +
            "</where>" +
            "</script>")
    List<User> findByCondition(User user);
}
```

### 4.2 使用 Provider 类（推荐）

对于复杂的动态 SQL，建议使用 Provider 类。

#### 步骤 1：创建 Provider 类

```java
package com.example.provider;

import com.example.entity.User;
import org.apache.ibatis.jdbc.SQL;

public class UserSqlProvider {
    
    /**
     * 动态查询 SQL
     */
    public String findByCondition(User user) {
        return new SQL() {{
            SELECT("*");
            FROM("user");
            
            if (user.getUsername() != null && !user.getUsername().isEmpty()) {
                WHERE("username LIKE CONCAT('%', #{username}, '%')");
            }
            if (user.getAge() != null) {
                WHERE("age = #{age}");
            }
            if (user.getEmail() != null && !user.getEmail().isEmpty()) {
                WHERE("email = #{email}");
            }
        }}.toString();
    }
    
    /**
     * 动态更新 SQL
     */
    public String updateSelective(User user) {
        return new SQL() {{
            UPDATE("user");
            
            if (user.getUsername() != null) {
                SET("username = #{username}");
            }
            if (user.getEmail() != null) {
                SET("email = #{email}");
            }
            if (user.getAge() != null) {
                SET("age = #{age}");
            }
            
            WHERE("id = #{id}");
        }}.toString();
    }
}
```

#### 步骤 2：在 Mapper 中使用

```java
public interface UserMapper {
    
    @SelectProvider(type = UserSqlProvider.class, method = "findByCondition")
    List<User> findByCondition(User user);
    
    @UpdateProvider(type = UserSqlProvider.class, method = "updateSelective")
    int updateSelective(User user);
}
```

**优点**：
* SQL 构建逻辑清晰
* 易于维护和测试
* 避免字符串拼接错误

## 5. XML vs 注解对比

### 5.1 对比表格

| 特性 | XML 方式 | 注解方式 |
|------|---------|---------|
| **简单 SQL** | 较繁琐 | ✅ 简洁明了 |
| **复杂 SQL** | ✅ 清晰易读 | 字符串拼接混乱 |
| **动态 SQL** | ✅ 标签化，易维护 | 需要 script 或 Provider |
| **结果映射** | ✅ resultMap 强大 | 注解较繁琐 |
| **SQL 复用** | ✅ sql 片段 | ❌ 难以复用 |
| **维护性** | SQL 集中管理 | 分散在代码中 |
| **修改 SQL** | 无需编译 | 需要重新编译 |
| **适用场景** | 复杂业务 | 简单 CRUD |

### 5.2 选择建议

**使用注解的场景**：
* 简单的 CRUD 操作
* SQL 语句较短
* 项目规模较小
* 快速原型开发

**使用 XML 的场景**：
* 复杂的 SQL 查询
* 大量的动态 SQL
* 企业级项目
* 需要频繁调整 SQL

**混合使用（推荐）**：
* 简单操作用注解
* 复杂操作用 XML
* 保持项目一致性

### 6.2 Mapper 接口（注解方式）

```java
package com.example.mapper;

import com.example.entity.User;
import org.apache.ibatis.annotations.*;
import java.util.List;

public interface UserMapper {
    
    // 查询所有
    @Select("SELECT * FROM user")
    @Results(id = "userResultMap", value = {
        @Result(property = "id", column = "id", id = true),
        @Result(property = "username", column = "username"),
        @Result(property = "email", column = "email"),
        @Result(property = "age", column = "age"),
        @Result(property = "createTime", column = "create_time"),
        @Result(property = "orders", column = "id",
                many = @Many(select = "com.example.mapper.OrderMapper.findByUserId"))
    })
    List<User> findAll();
    
    // 根据 ID 查询
    @Select("SELECT * FROM user WHERE id = #{id}")
    @ResultMap("userResultMap")
    User findById(Integer id);
    
    // 插入
    @Insert("INSERT INTO user (username, email, age) VALUES (#{username}, #{email}, #{age})")
    @Options(useGeneratedKeys = true, keyProperty = "id")
    int insert(User user);
    
    // 更新
    @Update("UPDATE user SET username = #{username}, email = #{email}, age = #{age} WHERE id = #{id}")
    int update(User user);
    
    // 删除
    @Delete("DELETE FROM user WHERE id = #{id}")
    int delete(Integer id);
    
    // 动态查询
    @Select("<script>" +
            "SELECT * FROM user" +
            "<where>" +
            "  <if test='username != null and username != \"\"'>" +
            "    AND username LIKE CONCAT('%', #{username}, '%')" +
            "  </if>" +
            "  <if test='age != null'>" +
            "    AND age = #{age}" +
            "  </if>" +
            "</where>" +
            "</script>")
    List<User> findByCondition(User user);
}
```


```java
package com.example.mapper;

import com.example.entity.User;
import java.util.List;

public interface UserMapperXml {
    
    List<User> findAll();

```

```xml
<?xml version="1.0" encoding="UTF-8" ?>
<!DOCTYPE mapper
        PUBLIC "-//mybatis.org//DTD Mapper 3.0//EN"
        "http://mybatis.org/dtd/mybatis-3-mapper.dtd">

<mapper namespace="com.example.mapper.UserMapperXml">
    
    <resultMap id="userResultMap" type="User">
        <id property="id" column="id"/>
        <result property="username" column="username"/>
        <result property="email" column="email"/>
        <result property="age" column="age"/>
        <result property="createTime" column="create_time"/>
        <collection property="orders" ofType="Order" 
                    select="com.example.mapper.OrderMapperXml.findByUserId"
                    column="id"/>
    </resultMap>
    
    <select id="findAll" resultMap="userResultMap">
        SELECT * FROM user
    </select>
    
    <select id="findById" resultMap="userResultMap">
        SELECT * FROM user WHERE id = #{id}
    </select>
    
    <insert id="insert" useGeneratedKeys="true" keyProperty="id">
        INSERT INTO user (username, email, age) 
        VALUES (#{username}, #{email}, #{age})
    </insert>
    
    <update id="update">
        UPDATE user 
        SET username = #{username}, email = #{email}, age = #{age}
        WHERE id = #{id}
    </update>
    
    <delete id="delete">
        DELETE FROM user WHERE id = #{id}
    </delete>
    
    <select id="findByCondition" resultType="User">
        SELECT * FROM user

            <if test="username != null and username != ''">
                AND username LIKE CONCAT('%', #{username}, '%')
            </if>
            <if test="age != null">
                AND age = #{age}
            </if>
        </where>
    </select>
    
</mapper>
```

## 7. 常见问题

### 7.1 找不到 Mapper

**错误**：
```
Invalid bound statement (not found): com.example.mapper.UserMapper.findById
```

**原因**：
1. namespace 不正确
2. 方法名不匹配
3. XML 文件未被扫描到

**解决**：
* 检查 namespace 是否与接口全限定名一致
* 检查方法名是否与 SQL 标签 id 一致
* 确认 mybatis-config.xml 中配置了 mapper 扫描

### 7.2 参数绑定失败

**错误**：
```
Parameter 'username' not found. Available parameters are [arg1, arg0, param1, param2]
```

**原因**：多参数未使用 @Param 注解

**解决**：
```java
// 错误
User findByCondition(String username, Integer age);

// 正确
User findByCondition(@Param("username") String username, 
                     @Param("age") Integer age);
```

### 7.3 注解中的特殊字符

**问题**：注解中不能直接使用 `<`、`>` 等字符

**解决**：
```java
// 错误
@Select("SELECT * FROM user WHERE age > #{age}")

// 正确：使用 CDATA 或转义
@Select("<script>SELECT * FROM user WHERE age &gt; #{age}</script>")
```

## 9. 总结

### 9.1 核心知识点

✅ **动态代理**：MyBatis 自动生成 Mapper 实现  
✅ **基本注解**：@Select、@Insert、@Update、@Delete  
✅ **结果映射**：@Results、@ResultMap  
✅ **关联映射**：@One（一对一）、@Many（一对多）  
✅ **动态 SQL**：script 标签、Provider 类  

### 9.2 最佳实践

📌 简单 SQL 用注解，复杂 SQL 用 XML  
📌 多参数必须使用 @Param 注解  
📌 使用 Provider 类构建复杂动态 SQL  
📌 保持项目中开发方式的一致性  
📌 合理使用 @ResultMap 避免重复配置
