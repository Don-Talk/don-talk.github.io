# MyBatis 核心配置详解

**学习目标：**
> * 掌握 mybatis-config.xml 的完整结构
> * 理解 properties、settings、typeAliases 等核心配置
> * 能够配置多环境和数据源
> * 了解类型处理器和插件的使用

---

## 1. mybatis-config.xml 完整结构

MyBatis 的核心配置文件 `mybatis-config.xml` 包含了影响 MyBatis 行为的设置和属性信息。

### 1.1 配置文件的 DTD 约束

```xml
<?xml version="1.0" encoding="UTF-8" ?>
<!DOCTYPE configuration
        PUBLIC "-//mybatis.org//DTD Config 3.0//EN"
        "http://mybatis.org/dtd/mybatis-3-config.dtd">
```

### 1.2 配置项的顺序

**重要**：MyBatis 配置项必须按照以下顺序声明，否则会出现解析错误：

```xml
<configuration>
    <!-- 1. properties（属性） -->
    <!-- 2. settings（设置） -->
    <!-- 3. typeAliases（类型别名） -->
    <!-- 4. typeHandlers（类型处理器） -->
    <!-- 5. objectFactory（对象工厂） -->
    <!-- 6. plugins（插件） -->
    <!-- 7. environments（环境配置） -->
    <!-- 8. databaseIdProvider（数据库厂商标识） -->
    <!-- 9. mappers（映射器） -->
</configuration>
```

## 2. properties（属性）

### 2.1 作用

用于配置外部化属性，可以将配置信息提取到单独的配置文件中，便于维护。

### 2.2 使用方式

#### 方式一：在 XML 中直接配置

```xml
<configuration>
    <properties>
        <property name="driver" value="com.mysql.cj.jdbc.Driver"/>
        <property name="url" value="jdbc:mysql://localhost:3306/mybatis_demo"/>
        <property name="username" value="root"/>
        <property name="password" value="123456"/>
    </properties>
    
    <environments default="development">
        <environment id="development">
            <transactionManager type="JDBC"/>
            <dataSource type="POOLED">
                <property name="driver" value="${driver}"/>
                <property name="url" value="${url}"/>
                <property name="username" value="${username}"/>
                <property name="password" value="${password}"/>
            </dataSource>
        </environment>
    </environments>
</configuration>
```

#### 方式二：引用外部 properties 文件（推荐）

**步骤 1**：创建 `db.properties` 文件（放在 `src/main/resources` 目录）

```properties
# 数据库配置
jdbc.driver=com.mysql.cj.jdbc.Driver
jdbc.url=jdbc:mysql://localhost:3306/mybatis_demo?useSSL=false&serverTimezone=Asia/Shanghai&characterEncoding=utf8
jdbc.username=root
jdbc.password=123456

# 连接池配置
pool.maxActive=20
pool.maxIdle=10
pool.minIdle=5
```

**步骤 2**：在 `mybatis-config.xml` 中引用

```xml
<configuration>
    <!-- 引用外部属性文件 -->
    <properties resource="db.properties"/>
    
    <environments default="development">
        <environment id="development">
            <transactionManager type="JDBC"/>
            <dataSource type="POOLED">
                <property name="driver" value="${jdbc.driver}"/>
                <property name="url" value="${jdbc.url}"/>
                <property name="username" value="${jdbc.username}"/>
                <property name="password" value="${jdbc.password}"/>
                <property name="maxActive" value="${pool.maxActive}"/>
                <property name="maxIdle" value="${pool.maxIdle}"/>
                <property name="minIdle" value="${pool.minIdle}"/>
            </dataSource>
        </environment>
    </environments>
</configuration>
```

#### 方式三：通过程序参数传递

```java
// 在构建 SqlSessionFactory 时传入属性
Properties props = new Properties();
props.setProperty("jdbc.username", "root");
props.setProperty("jdbc.password", "123456");

SqlSessionFactory factory = new SqlSessionFactoryBuilder()
    .build(inputStream, props);
```

### 2.3 属性优先级

如果同一个属性在多个地方配置，优先级如下：

1. **方法参数传递**（最高优先级）
2. **properties 元素体内配置**
3. **外部 properties 文件**（最低优先级）

## 3. settings（设置）

### 3.1 作用

settings 是 MyBatis 中最重要的调整设置，它们会改变 MyBatis 的运行时行为。

### 3.2 常用配置项

```xml
<configuration>
    <settings>
        <!-- 全局性地开启或关闭所有映射器配置文件中已配置的任何缓存 -->
        <setting name="cacheEnabled" value="true"/>
        
        <!-- 延迟加载的全局开关 -->
        <setting name="lazyLoadingEnabled" value="true"/>
        
        <!-- 当启用时，对任意延迟属性的调用会使带有延迟加载属性的对象完整加载 -->
        <setting name="aggressiveLazyLoading" value="false"/>
        
        <!-- 是否允许单一语句返回多结果集 -->
        <setting name="multipleResultSetsEnabled" value="true"/>
        
        <!-- 使用列标签代替列名 -->
        <setting name="useColumnLabel" value="true"/>
        
        <!-- 允许 JDBC 支持自动生成主键 -->
        <setting name="useGeneratedKeys" value="false"/>
        
        <!-- 指定 MyBatis 应如何自动映射列到字段或属性 -->
        <!-- NONE: 取消自动映射 -->
        <!-- PARTIAL: 只会自动映射没有定义嵌套结果映射的字段 -->
        <!-- FULL: 会自动映射任何复杂的结果集 -->
        <setting name="autoMappingBehavior" value="PARTIAL"/>
        
        <!-- 指定发现自动映射目标未知列（或未知属性类型）的行为 -->
        <!-- NONE: 不做任何反应 -->
        <!-- WARNING: 输出警告日志 -->
        <!-- FAILING: 映射失败 (抛出 SqlSessionException) -->
        <setting name="autoMappingUnknownColumnBehavior" value="WARNING"/>
        
        <!-- 配置默认的执行器 -->
        <!-- SIMPLE: 普通执行器 -->
        <!-- REUSE: 重用预处理语句的执行器 -->
        <!-- BATCH: 批量执行器 -->
        <setting name="defaultExecutorType" value="SIMPLE"/>
        
        <!-- 设置超时时间，它决定驱动等待一个数据库响应的时间 -->
        <setting name="defaultStatementTimeout" value="25"/>
        
        <!-- 为驱动的结果集获取数量（fetchSize）设置一个建议值 -->
        <setting name="defaultFetchSize" value="100"/>
        
        <!-- 是否允许在嵌套语句中使用分页（RowBounds） -->
        <setting name="safeRowBoundsEnabled" value="false"/>
        
        <!-- 是否开启驼峰命名自动映射 -->
        <!-- 即从经典数据库列名 A_COLUMN 映射到经典 Java 属性名 aColumn -->
        <setting name="mapUnderscoreToCamelCase" value="true"/>
        
        <!-- MyBatis 利用本地缓存机制防止循环引用和加速重复的嵌套查询 -->
        <!-- SESSION: 会话级别缓存 -->
        <!-- STATEMENT: 语句级别缓存，仅对当前执行的语句有效 -->
        <setting name="localCacheScope" value="SESSION"/>
        
        <!-- 当没有为参数指定特定的 JDBC 类型时，空值的默认 JDBC 类型 -->
        <setting name="jdbcTypeForNull" value="OTHER"/>
        
        <!-- 指定哪个对象的方法触发一次延迟加载 -->
        <setting name="lazyLoadTriggerMethods" value="equals,clone,hashCode,toString"/>
        
        <!-- 允许使用方法签名中的名称作为语句参数名称 -->
        <setting name="useActualParamName" value="true"/>
        
        <!-- 指定提供 Configuration 实例或者其子类实例的对象工厂 -->
        <setting name="configurationFactory" value=""/>
    </settings>
</configuration>
```

### 3.3 重点配置说明

#### 3.3.1 驼峰命名自动映射

```xml
<setting name="mapUnderscoreToCamelCase" value="true"/>
```

**作用**：自动将数据库的下划线命名转换为 Java 的驼峰命名。

**示例**：
* 数据库字段：`user_name` → Java 属性：`userName`
* 数据库字段：`create_time` → Java 属性：`createTime`

**好处**：无需手动配置 resultMap，简化开发。

#### 3.3.2 延迟加载

```xml
<setting name="lazyLoadingEnabled" value="true"/>
<setting name="aggressiveLazyLoading" value="false"/>
```

**作用**：在关联查询时，只有真正使用关联对象时才执行查询，提高性能。

**示例**：查询订单时，不立即查询用户信息，只有调用 `order.getUser()` 时才查询。

#### 3.3.3 缓存配置

```xml
<setting name="cacheEnabled" value="true"/>
```

**作用**：开启二级缓存，提高查询性能。

## 4. typeAliases（类型别名）

### 4.1 作用

为 Java 类型设置一个短的名字，减少类完全限定名的冗余。

### 4.2 配置方式

#### 方式一：单个类型别名

```xml
<typeAliases>
    <typeAlias type="com.example.entity.User" alias="User"/>
    <typeAlias type="com.example.entity.Product" alias="Product"/>
    <typeAlias type="com.example.entity.Order" alias="Order"/>
</typeAliases>
```

**使用**：
```xml
<!-- 之前 -->
<select id="findAll" resultType="com.example.entity.User">
    SELECT * FROM user
</select>

<!-- 之后 -->
<select id="findAll" resultType="User">
    SELECT * FROM user
</select>
```

#### 方式二：批量扫描包（推荐）

```xml
<typeAliases>
    <!-- 扫描指定包下的所有类，默认别名为类名（首字母小写或不区分大小写） -->
    <package name="com.example.entity"/>
</typeAliases>
```

**扫描规则**：
* `User` 类的别名可以是 `User` 或 `user`（不区分大小写）
* `UserInfo` 类的别名可以是 `UserInfo` 或 `userinfo`

#### 方式三：使用 @Alias 注解

```java
package com.example.entity;

import org.apache.ibatis.type.Alias;

@Alias("MyUser")
public class User {
    private Integer id;
    private String username;
    // ...
}
```

| `_long` | long |
| `_short` | short |
| `_int` | int |
| `_integer` | int |
| `_double` | double |
| `_float` | float |
| `_boolean` | boolean |
| `string` | String |

| `integer` | Integer |

## 5. typeHandlers（类型处理器）

### 5.1 作用

无论是 MyBatis 在预处理语句中设置一个参数，还是从结果集中取出一个值时，都会用类型处理器将获取的值以合适的方式转换成 Java 类型。

### 5.2 MyBatis 内置类型处理器

MyBatis 已经提供了大量的类型处理器，通常不需要自定义：

| 类型处理器 | Java 类型 | JDBC 类型 |
|-----------|----------|----------|
| BooleanTypeHandler | Boolean, boolean | BOOLEAN |
| ByteTypeHandler | Byte, byte | TINYINT |
| ShortTypeHandler | Short, short | SMALLINT |
| IntegerTypeHandler | Integer, int | INTEGER |
| LongTypeHandler | Long, long | BIGINT |
| FloatTypeHandler | Float, float | FLOAT |
| DoubleTypeHandler | Double, double | DOUBLE |
| StringTypeHandler | String | CHAR, VARCHAR |
| DateTypeHandler | Date | DATE |
| TimeTypeHandler | Time | TIME |
| TimestampTypeHandler | Timestamp | TIMESTAMP |

### 5.3 自定义类型处理器

#### 场景：枚举类型处理

**步骤 1**：创建枚举类

```java
package com.example.enums;

public enum Gender {
    MALE("男"),
    FEMALE("女");
    
    private String description;
    
    Gender(String description) {
        this.description = description;
    }
    
    public String getDescription() {
        return description;
    }
}
```

**步骤 2**：创建类型处理器

```java
package com.example.handler;

import com.example.enums.Gender;
import org.apache.ibatis.type.BaseTypeHandler;
import org.apache.ibatis.type.JdbcType;

import java.sql.CallableStatement;
import java.sql.PreparedStatement;
import java.sql.ResultSet;
import java.sql.SQLException;

/**
 * 性别枚举类型处理器
 */
public class GenderTypeHandler extends BaseTypeHandler<Gender> {
    
    /**
     * 设置参数
     */
    @Override
    public void setNonNullParameter(PreparedStatement ps, int i, 
                                     Gender parameter, JdbcType jdbcType) 
                                     throws SQLException {
        ps.setString(i, parameter.name());
    }
    
    /**
     * 根据列名获取结果
     */
    @Override
    public Gender getNullableResult(ResultSet rs, String columnName) 
                                     throws SQLException {
        String value = rs.getString(columnName);
        return value == null ? null : Gender.valueOf(value);
    }
    
    /**
     * 根据列索引获取结果
     */
    @Override
    public Gender getNullableResult(ResultSet rs, int columnIndex) 
                                     throws SQLException {
        String value = rs.getString(columnIndex);
        return value == null ? null : Gender.valueOf(value);
    }
    
    /**
     * 存储过程获取结果
     */
    @Override
    public Gender getNullableResult(CallableStatement cs, int columnIndex) 
                                     throws SQLException {
        String value = cs.getString(columnIndex);
        return value == null ? null : Gender.valueOf(value);
    }
}
```

**步骤 3**：注册类型处理器

```xml
<typeHandlers>
    <!-- 注册自定义类型处理器 -->
    <typeHandler handler="com.example.handler.GenderTypeHandler"/>
    
    <!-- 或者指定 Java 类型和 JDBC 类型 -->
    <typeHandler handler="com.example.handler.GenderTypeHandler" 
                 javaType="com.example.enums.Gender" 
                 jdbcType="VARCHAR"/>
</typeHandlers>
```

**步骤 4**：在实体类中使用

```java
@Data
public class User {
    private Integer id;
    private String username;
    private Gender gender;  // 使用枚举类型
    // ...
}
```

## 6. environments（环境配置）

### 6.1 作用

MyBatis 可以配置多种环境，比如开发、测试和生产环境需要有不同的配置。

### 6.2 配置示例

```xml
<environments default="development">
    <!-- 开发环境 -->
    <environment id="development">
        <transactionManager type="JDBC">
            <property name="..." value="..."/>
        </transactionManager>
        <dataSource type="POOLED">
            <property name="driver" value="${jdbc.driver}"/>
            <property name="url" value="${jdbc.url}"/>
            <property name="username" value="${jdbc.username}"/>
            <property name="password" value="${jdbc.password}"/>
        </dataSource>
    </environment>
    
    <!-- 测试环境 -->
    <environment id="test">
        <transactionManager type="JDBC"/>
        <dataSource type="POOLED">
            <property name="driver" value="com.mysql.cj.jdbc.Driver"/>
            <property name="url" value="jdbc:mysql://localhost:3306/test_db"/>
            <property name="username" value="test_user"/>
            <property name="password" value="test_pass"/>
        </dataSource>
    </environment>
    
    <!-- 生产环境 -->
    <environment id="production">
        <transactionManager type="MANAGED"/>
        <dataSource type="JNDI">
            <property name="data_source" value="java:comp/jdbc/MyDataSource"/>
        </dataSource>
    </environment>
</environments>
```

**切换环境**：修改 `default` 属性即可切换环境。

### 6.3 事务管理器（transactionManager）

MyBatis 支持两种类型的事务管理器：

#### JDBC

* 直接使用 JDBC 的提交和回滚功能
* 依赖于从数据源得到的连接来管理事务范围

```xml
<transactionManager type="JDBC"/>
```

#### MANAGED

* 不提交或回滚一个连接
* 让容器来管理事务的整个生命周期（如 Spring）

```xml
<transactionManager type="MANAGED">
    <property name="closeConnection" value="false"/>
</transactionManager>
```

### 6.4 数据源（dataSource）

MyBatis 内置了三种数据源类型：

#### UNPOOLED

* 不使用连接池
* 每次请求都打开和关闭连接
* 适用于简单的应用场景

```xml
<dataSource type="UNPOOLED">
    <property name="driver" value="com.mysql.cj.jdbc.Driver"/>
    <property name="url" value="jdbc:mysql://localhost:3306/mybatis_demo"/>
    <property name="username" value="root"/>
    <property name="password" value="123456"/>
</dataSource>
```

#### POOLED（推荐）

* 使用数据库连接池
* 避免创建新连接实例时必要的初始化和认证时间
* 适用于并发 Web 应用

```xml
<dataSource type="POOLED">
    <property name="driver" value="com.mysql.cj.jdbc.Driver"/>
    <property name="url" value="jdbc:mysql://localhost:3306/mybatis_demo"/>
    <property name="username" value="root"/>
    <property name="password" value="123456"/>
    
    <!-- 连接池配置 -->
    <property name="poolMaximumActiveConnections" value="20"/>
    <property name="poolMaximumIdleConnections" value="10"/>
    <property name="poolMaximumCheckoutTime" value="20000"/>
    <property name="poolTimeToWait" value="20000"/>
    <property name="poolPingQuery" value="SELECT 1"/>
    <property name="poolPingEnabled" value="true"/>
    <property name="poolPingConnectionsNotUsedFor" value="3600000"/>
</dataSource>
```

**常用连接池属性**：

| 属性 | 说明 | 默认值 |
|------|------|--------|
| poolMaximumActiveConnections | 最大活动连接数 | 10 |
| poolMaximumIdleConnections | 最大空闲连接数 | 5 |
| poolMaximumCheckoutTime | 连接最大checkout时间(ms) | 20000 |
| poolTimeToWait | 获取连接等待时间(ms) | 20000 |
| poolPingQuery | 探测查询SQL | NO PING QUERY SET |
| poolPingEnabled | 是否启用探测查询 | false |
| poolPingConnectionsNotUsedFor | 多久检测一次未使用的连接(ms) | 0 |

#### JNDI

* 为了能在如 EJB 或应用服务器这类容器中使用
* 容器可以集中或在外部配置数据源

```xml
<dataSource type="JNDI">
    <property name="data_source" value="java:comp/jdbc/MyDataSource"/>
</dataSource>
```

## 7. mappers（映射器）

### 7.1 作用

告诉 MyBatis 到哪里去找到 SQL 映射语句。

### 7.2 配置方式

#### 方式一：使用相对于类路径的资源引用

```xml
<mappers>
    <mapper resource="mapper/UserMapper.xml"/>
    <mapper resource="mapper/ProductMapper.xml"/>
    <mapper resource="mapper/OrderMapper.xml"/>
</mappers>
```

#### 方式二：使用完全限定资源定位符（URL）

```xml
<mappers>
    <mapper url="file:///var/mappers/UserMapper.xml"/>
</mappers>
```

#### 方式三：使用 Mapper 接口类名

```xml
<mappers>
    <mapper class="com.example.mapper.UserMapper"/>
    <mapper class="com.example.mapper.ProductMapper"/>
</mappers>
```

**要求**：
* XML 文件和 Mapper 接口必须在同一目录下
* XML 文件名必须与接口名相同

#### 方式四：批量扫描包（推荐）

```xml
<mappers>
    <package name="com.example.mapper"/>
</mappers>
```

**要求**：
* 所有 Mapper 接口都在该包下
* XML 文件和接口在同一目录下，且文件名相同

## 8. 完整配置示例

```xml
<?xml version="1.0" encoding="UTF-8" ?>
<!DOCTYPE configuration
        PUBLIC "-//mybatis.org//DTD Config 3.0//EN"
        "http://mybatis.org/dtd/mybatis-3-config.dtd">

<configuration>
    <!-- 1. 引入外部属性文件 -->
    <properties resource="db.properties"/>
    
    <!-- 2. 全局设置 -->
    <settings>
        <setting name="mapUnderscoreToCamelCase" value="true"/>
        <setting name="lazyLoadingEnabled" value="true"/>
        <setting name="aggressiveLazyLoading" value="false"/>
        <setting name="cacheEnabled" value="true"/>
        <setting name="logImpl" value="STDOUT_LOGGING"/>
    </settings>
    
    <!-- 3. 类型别名 -->
    <typeAliases>
        <package name="com.example.entity"/>
    </typeAliases>
    
    <!-- 4. 类型处理器 -->
    <typeHandlers>
        <typeHandler handler="com.example.handler.GenderTypeHandler"/>
    </typeHandlers>
    
    <!-- 5. 环境配置 -->
    <environments default="development">
        <environment id="development">
            <transactionManager type="JDBC"/>
            <dataSource type="POOLED">
                <property name="driver" value="${jdbc.driver}"/>
                <property name="url" value="${jdbc.url}"/>
                <property name="username" value="${jdbc.username}"/>
                <property name="password" value="${jdbc.password}"/>
                <property name="poolMaximumActiveConnections" value="20"/>
                <property name="poolMaximumIdleConnections" value="10"/>
            </dataSource>
        </environment>
    </environments>
    
    <!-- 6. 映射器 -->
    <mappers>
        <package name="com.example.mapper"/>
    </mappers>
</configuration>
```

## 10. 总结

### 10.1 核心知识点

✅ **properties**：外部化配置，支持多种方式加载  
✅ **settings**：调整 MyBatis 运行时行为  
✅ **typeAliases**：简化类型名称，减少冗余  
✅ **typeHandlers**：Java 类型与 JDBC 类型转换  
✅ **environments**：多环境配置（开发、测试、生产）  
✅ **mappers**：注册 SQL 映射文件  

### 10.2 最佳实践

📌 使用外部 properties 文件管理数据库配置  
📌 开启驼峰命名自动映射，减少手动配置
