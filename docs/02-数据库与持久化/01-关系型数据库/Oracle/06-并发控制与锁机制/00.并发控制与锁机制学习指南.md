### 01. 事务ACID特性实现

**核心内容**：
- Atomicity（原子性）：Undo Log 实现
- Consistency（一致性）：约束和触发器
- Isolation（隔离性）：锁和 MVCC
- Durability（持久性）：Redo Log 实现

**关键知识点**：
```sql
-- 事务控制
BEGIN TRANSACTION;
    UPDATE accounts SET balance = balance - 100 WHERE id = 1;
    UPDATE accounts SET balance = balance + 100 WHERE id = 2;
COMMIT;

-- 保存点
SAVEPOINT sp1;
ROLLBACK TO SAVEPOINT sp1;

-- 查看活动事务
SELECT * FROM v$transaction;

-- Redo Log 确保持久性
ALTER SYSTEM SWITCH LOGFILE;
```

### 02. 锁类型详解

**核心内容**：
- DML 锁（TM、TX）
- DDL 锁
- 内部锁和闩锁
- 行级锁机制
- 表级锁模式

**关键知识点**：
```sql
-- 查看当前锁
SELECT 
    s.sid,
    s.username,
    l.type,
    l.lmode,
    l.request,
    o.object_name
FROM v$lock l
JOIN v$session s ON l.sid = s.sid
LEFT JOIN dba_objects o ON l.id1 = o.object_id
WHERE l.type IN ('TM', 'TX');

-- 手动加锁
LOCK TABLE employees IN EXCLUSIVE MODE;
LOCK TABLE employees IN SHARE MODE;

-- 锁模式
-- 0: None
-- 1: Null
-- 2: Row-S (SS)
-- 3: Row-X (SX)
-- 4: Share (S)
-- 5: S/Row-X (SSX)
-- 6: Exclusive (X)
```

### 03. 死锁检测与解决

**核心内容**：
- 死锁形成条件
- Oracle 自动死锁检测
- 死锁诊断方法
- 死锁预防策略
- ORA-00060 错误处理

**关键知识点**：
```sql
-- 查看死锁信息
SELECT 
    s1.username || '@' || s1.machine || ' (SID=' || s1.sid || ') is blocking ' ||
    s2.username || '@' || s2.machine || ' (SID=' || s2.sid || ')' AS deadlock_info
FROM v$lock l1, v$session s1, v$lock l2, v$session s2
WHERE s1.sid = l1.sid
AND s2.sid = l2.sid
AND l1.BLOCK = 1
AND l2.request > 0
AND l1.id1 = l2.id1
AND l2.id2 = l2.id2;

-- 查找阻塞会话
SELECT 
    blocker.sid as blocker_sid,
    blocker.username as blocker_user,
    waiter.sid as waiter_sid,
    waiter.username as waiter_user
FROM v$session blocker
JOIN v$session waiter ON blocker.sid = waiter.blocking_session
WHERE waiter.blocking_session IS NOT NULL;

-- 终止阻塞会话（谨慎使用）
ALTER SYSTEM KILL SESSION 'sid,serial#' IMMEDIATE;
```

```sql
-- 查看 Undo 表空间
SELECT tablespace_name, status, contents
FROM dba_tablespaces
WHERE contents = 'UNDO';

-- 查看 Undo 使用情况
SELECT 
    tablespace_name,
    ROUND(used_ublk * 8 / 1024, 2) as used_mb
FROM v$undostat;

-- 配置 Undo 保留时间
ALTER SYSTEM SET undo_retention=900 SCOPE=BOTH;  -- 15分钟

-- 启用 Retention Guarantee
ALTER TABLESPACE undotbs1 RETENTION GUARANTEE;

-- 避免 ORA-01555
-- 1. 增加 Undo 表空间大小
-- 2. 增加 undo_retention
-- 3. 避免长时间运行的查询
```

### 05. 隔离级别与读一致性

**核心内容**：
- READ COMMITTED（默认）
- SERIALIZABLE
- READ ONLY
- 语句级读一致性
- 事务级读一致性

**关键知识点**：
```sql
-- 设置隔离级别
SET TRANSACTION ISOLATION LEVEL READ COMMITTED;  -- 默认
SET TRANSACTION ISOLATION LEVEL SERIALIZABLE;
SET TRANSACTION READ ONLY;

-- READ COMMITTED：每次读取看到最新已提交数据
-- SERIALIZABLE：事务中看到的数据保持一致

-- 示例：SERIALIZABLE 隔离
SET TRANSACTION ISOLATION LEVEL SERIALIZABLE;
SELECT COUNT(*) FROM employees;  -- 看到 100 条
-- 其他事务插入数据并提交
SELECT COUNT(*) FROM employees;  -- 仍然看到 100 条
COMMIT;

-- 查询一致性读统计
SELECT name, value
FROM v$sysstat
WHERE name IN ('consistent gets', 'db block gets');
```

### 06. 阻塞会话诊断

**核心内容**：
- 识别阻塞会话
- 分析锁等待链
- 自动化监控脚本
- 解决阻塞问题
- 预防阻塞的最佳实践

**关键知识点**：
```sql
-- 完整的阻塞链查询
WITH lock_tree AS (
    SELECT 
        sid,
        blocking_session,
        username,
        sql_id,
        LEVEL as level_num,
        SYS_CONNECT_BY_PATH(sid, ' -> ') as chain
    FROM v$session
    START WITH blocking_session IS NULL
    CONNECT BY PRIOR sid = blocking_session
)
SELECT * FROM lock_tree
WHERE blocking_session IS NOT NULL
ORDER BY level_num;

-- 查找长时间持有的锁
SELECT 
    s.sid,
    s.username,
    s.program,
    l.type,
    l.lmode,
    ROUND(s.last_call_et/60, 2) as minutes_active
FROM v$session s
JOIN v$lock l ON s.sid = l.sid
WHERE l.block = 1
AND s.last_call_et > 300;  -- 超过5分钟

-- 生成解锁脚本
SELECT 
    'ALTER SYSTEM KILL SESSION ''' || s.sid || ',' || s.serial# || ''' IMMEDIATE;' as kill_cmd
FROM v$session s
WHERE s.sid IN (
    SELECT blocking_session 
    FROM v$session 
    WHERE blocking_session IS NOT NULL
);
```

## 🔍 实战案例预告

### 案例 1：死锁分析
```sql
-- 模拟死锁场景
-- Session 1:
UPDATE employees SET salary = 5000 WHERE emp_id = 1;
UPDATE departments SET budget = 100000 WHERE dept_id = 10;

-- Session 2:
UPDATE departments SET budget = 100000 WHERE dept_id = 10;
UPDATE employees SET salary = 5000 WHERE emp_id = 1;
-- ORA-00060: deadlock detected
```

### 案例 2：长时间运行事务
```sql
-- 查找长时间未提交的事务
SELECT 
    t.start_time,
    s.username,
    s.sid,
    t.used_ublk * 8 / 1024 as undo_mb,
    ROUND((SYSDATE - TO_DATE(t.start_time, 'MM/DD/YY HH24:MI:SS')) * 24 * 60, 2) as minutes
FROM v$transaction t
JOIN v$session s ON t.ses_addr = s.saddr
ORDER BY t.start_time;
```
