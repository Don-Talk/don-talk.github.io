Oracle 数据库由以下物理文件组成：

| 文件类型 | 扩展名 | 作用 | 必需性 |

| **跟踪文件** | .trc | 诊断信息 | 自动生成 |

## 2. 数据文件（Data Files）

### 2.1 基本概念

数据文件是存储数据库实际数据的操作系统文件。

**特点**：
- 一个表空间可以包含多个数据文件
- 一个数据文件只能属于一个表空间
- 数据以块（Block）为单位存储
- 文件大小可以自动扩展

### 2.2 查看数据文件

```sql
-- 查看所有数据文件
SELECT 
    file_name,
    tablespace_name,
    bytes/1024/1024 as size_mb,
    maxbytes/1024/1024 as max_size_mb,
    autoextensible,
    increment_by * (SELECT value FROM v$parameter WHERE name = 'db_block_size') / 1024 as increment_kb,
    status
FROM dba_data_files
ORDER BY tablespace_name, file_name;

-- 查看文件使用率
SELECT 
    df.tablespace_name,
    df.file_name,
    df.bytes/1024/1024 as allocated_mb,
    (df.bytes - NVL(fs.free_bytes, 0))/1024/1024 as used_mb,
    ROUND((df.bytes - NVL(fs.free_bytes, 0)) / df.bytes * 100, 2) as used_pct
FROM dba_data_files df
LEFT JOIN (
    SELECT file_id, SUM(bytes) as free_bytes
    FROM dba_free_space
    GROUP BY file_id
) fs ON df.file_id = fs.file_id
ORDER BY used_pct DESC;
```

### 2.3 创建数据文件

#### （1）创建表空间时指定

```sql
-- 单个数据文件
CREATE TABLESPACE users_ts
DATAFILE '/u01/app/oracle/oradata/ORCL/users01.dbf'
SIZE 1G
AUTOEXTEND ON NEXT 100M MAXSIZE 10G
EXTENT MANAGEMENT LOCAL
SEGMENT SPACE MANAGEMENT AUTO;

-- 多个数据文件
CREATE TABLESPACE large_ts
DATAFILE 
    '/u01/app/oracle/oradata/ORCL/large01.dbf' SIZE 2G,
    '/u02/app/oracle/oradata/ORCL/large02.dbf' SIZE 2G
AUTOEXTEND ON NEXT 200M MAXSIZE UNLIMITED;
```

#### （2）为现有表空间添加数据文件

```sql
ALTER TABLESPACE users_ts
ADD DATAFILE '/u01/app/oracle/oradata/ORCL/users02.dbf'
SIZE 1G
AUTOEXTEND ON NEXT 100M MAXSIZE 10G;
```

#### （3）调整数据文件大小

```sql
-- 手动调整大小
ALTER DATABASE DATAFILE '/u01/app/oracle/oradata/ORCL/users01.dbf'
RESIZE 2G;

-- 启用自动扩展
ALTER DATABASE DATAFILE '/u01/app/oracle/oradata/ORCL/users01.dbf'
AUTOEXTEND ON NEXT 100M MAXSIZE 10G;

-- 禁用自动扩展
ALTER DATABASE DATAFILE '/u01/app/oracle/oradata/ORCL/users01.dbf'
AUTOEXTEND OFF;
```

### 2.4 数据文件管理

#### 重命名数据文件

```sql
-- 1. 使表空间离线
ALTER TABLESPACE users_ts OFFLINE NORMAL;

-- 2. 操作系统层面移动文件
-- mv /old/path/users01.dbf /new/path/users01.dbf

-- 3. 更新控制文件中的路径
ALTER DATABASE RENAME FILE 
    '/old/path/users01.dbf' 
    TO '/new/path/users01.dbf';

-- 4. 使表空间在线
ALTER TABLESPACE users_ts ONLINE;
```

#### 删除数据文件

```sql
-- Oracle 10g+ 支持
ALTER TABLESPACE users_ts
DROP DATAFILE '/u01/app/oracle/oradata/ORCL/users02.dbf';

-- 注意：只能删除空的数据文件
```

#### 脱机/联机数据文件

```sql
-- 脱机
ALTER DATABASE DATAFILE '/u01/app/oracle/oradata/ORCL/users01.dbf' OFFLINE;

-- 联机
ALTER DATABASE DATAFILE '/u01/app/oracle/oradata/ORCL/users01.dbf' ONLINE;

-- 如果需要恢复
RECOVER DATAFILE '/u01/app/oracle/oradata/ORCL/users01.dbf';
```

### 2.5 OMF（Oracle Managed Files）

OMF 让 Oracle 自动管理文件命名和位置。

```sql
-- 配置 OMF
ALTER SYSTEM SET db_create_file_dest='/u01/app/oracle/oradata' SCOPE=BOTH;
ALTER SYSTEM SET db_create_online_log_dest_1='/u01/app/oracle/oradata' SCOPE=BOTH;

-- 创建表空间（Oracle 自动命名和管理文件）
CREATE TABLESPACE users_ts DATAFILE SIZE 1G;

-- 添加数据文件
ALTER TABLESPACE users_ts ADD DATAFILE SIZE 1G;

-- 删除表空间时自动删除文件
DROP TABLESPACE users_ts INCLUDING CONTENTS AND DATAFILES;
```

**优势**：
- 简化管理
- 避免文件名冲突
- 自动清理

## 3. 控制文件（Control Files）

### 3.1 作用和重要性

控制文件是 Oracle 数据库的"大脑"，包含：
- 数据库名称和 ID
- 数据文件和重做日志文件的位置
- 数据库创建时间戳
- 当前日志序列号
- 检查点信息
- RMAN 备份信息

**关键特性**：
- 二进制文件，不能手动编辑
- 必须多路复用（至少 2 份，建议 3 份）
- 数据库启动时必须可访问
- 损坏会导致数据库无法启动

### 3.2 查看控制文件

```sql
-- 查看控制文件位置
SELECT name FROM v$controlfile;

-- 查看控制文件内容摘要
SELECT 
    database_name,
    created,
    checkpoint_change#,
    checkpoint_time
FROM v$database;

-- 查看控制文件记录的部分
SELECT type, record_size, records_total, records_used
FROM v$controlfile_record_section
ORDER BY type;
```

### 3.3 配置多路复用

```sql
-- 查看当前配置
SHOW PARAMETER control_files;

-- 修改控制文件位置（需要重启）
-- 1. 关闭数据库
SHUTDOWN IMMEDIATE;

-- 2. 复制控制文件到新的位置（操作系统命令）
-- cp /u01/control01.ctl /u02/control02.ctl
-- cp /u01/control01.ctl /u03/control03.ctl

-- 3. 修改参数文件
ALTER SYSTEM SET control_files=
    '/u01/app/oracle/oradata/ORCL/control01.ctl',
    '/u02/app/oracle/oradata/ORCL/control02.ctl',
    '/u03/app/oracle/oradata/ORCL/control03.ctl'
SCOPE=SPFILE;

-- 4. 启动数据库
STARTUP;
```

### 3.4 备份控制文件

```sql
-- 备份到跟踪文件（文本格式）
ALTER DATABASE BACKUP CONTROLFILE TO TRACE;

-- 备份到二进制文件
ALTER DATABASE BACKUP CONTROLFILE TO '/backup/control_backup.ctl';

-- 在 RMAN 中备份
RMAN> BACKUP CURRENT CONTROLFILE;
```

**从跟踪文件重建控制文件**：
```sql
-- 1. 找到跟踪文件
SELECT value FROM v$parameter WHERE name = 'user_dump_dest';

-- 2. 提取 CREATE CONTROLFILE 语句
-- 3. 关闭数据库
SHUTDOWN ABORT;

-- 4. 删除损坏的控制文件

-- 5. 执行 CREATE CONTROLFILE 语句
STARTUP NOMOUNT;
CREATE CONTROLFILE REUSE DATABASE "ORCL" NORESETLOGS...

-- 6. 打开数据库
ALTER DATABASE OPEN;
```

### 3.5 控制文件管理最佳实践

✅ **多路复用**：至少 3 份，放在不同磁盘  
✅ **定期备份**：每次结构变更后备份  
✅ **监控大小**：避免达到 MAXLOGFILES 限制  
✅ **保护权限**：只有 oracle 用户可读写

## 4. 重做日志文件（Redo Log Files）

### 4.1 作用和结构

重做日志记录所有数据变更，用于：
- 实例恢复
- 介质恢复
-  standby 数据库同步
- LogMiner 分析

**组织结构**：
- **日志组（Group）**：逻辑单元
- **日志成员（Member）**：物理文件，每组可有多个成员（镜像）
- **循环使用**：LGWR 依次写入各组

<!-- TODO: 插入图片：重做日志循环使用图 -->

### 4.2 查看重做日志

```sql
-- 查看日志组信息
SELECT 
    group#,
    sequence#,
    bytes/1024/1024 as size_mb,
    members,
    status,
    archived
FROM v$log
ORDER BY group#;

-- 查看日志成员
SELECT 
    group#,
    member,
    type
FROM v$logfile
ORDER BY group#, member;

-- 查看日志切换历史
SELECT 
    sequence#,
    first_time,
    next_time,
    ROUND((next_time - first_time) * 24 * 60, 2) as minutes
FROM v$log_history
ORDER BY sequence# DESC
FETCH FIRST 20 ROWS ONLY;
```

**日志组状态**：
- **CURRENT**：当前正在写入
- **ACTIVE**：活动状态，实例恢复需要
- **INACTIVE**：非活动，可以覆盖
- **UNUSED**：从未使用过

### 4.3 创建和管理重做日志

#### 添加日志组

```sql
-- 添加新日志组
ALTER DATABASE ADD LOGFILE GROUP 4 
    ('/u01/app/oracle/oradata/ORCL/redo04a.log',
     '/u02/app/oracle/oradata/ORCL/redo04b.log')
SIZE 500M;

-- 添加日志成员到现有组
ALTER DATABASE ADD LOGFILE MEMBER 
    '/u03/app/oracle/oradata/ORCL/redo01c.log'
TO GROUP 1;
```

#### 删除日志组和成员

```sql
-- 删除日志成员
ALTER DATABASE DROP LOGFILE MEMBER 
    '/u01/app/oracle/oradata/ORCL/redo01c.log';

-- 删除日志组（必须是 INACTIVE 或 UNUSED 状态）
ALTER DATABASE DROP LOGFILE GROUP 4;

-- 如果状态是 ACTIVE，先切换日志
ALTER SYSTEM SWITCH LOGFILE;
ALTER SYSTEM CHECKPOINT;
ALTER DATABASE DROP LOGFILE GROUP 4;
```

#### 调整日志文件大小

```sql
-- Oracle 不支持直接调整大小，需要重建
-- 1. 创建新的更大的日志组
ALTER DATABASE ADD LOGFILE GROUP 4 SIZE 1G;
ALTER DATABASE ADD LOGFILE GROUP 5 SIZE 1G;
ALTER DATABASE ADD LOGFILE GROUP 6 SIZE 1G;

-- 2. 切换日志直到旧组变为 INACTIVE
ALTER SYSTEM SWITCH LOGFILE;
ALTER SYSTEM CHECKPOINT;

-- 3. 删除旧的日志组
ALTER DATABASE DROP LOGFILE GROUP 1;
ALTER DATABASE DROP LOGFILE GROUP 2;
ALTER DATABASE DROP LOGFILE GROUP 3;
```

### 4.4 日志切换优化

**理想切换频率**：每 15-20 分钟一次

```sql
-- 计算平均切换间隔
SELECT 
    AVG(minutes) as avg_minutes,
    MIN(minutes) as min_minutes,
    MAX(minutes) as max_minutes
FROM (
    SELECT 
        sequence#,
        ROUND((first_time - LAG(first_time) OVER (ORDER BY sequence#)) * 24 * 60, 2) as minutes
    FROM v$log_history
    WHERE first_time > SYSDATE - 1
);
```

**调整建议**：
- 切换太频繁（< 15 分钟）→ 增大日志文件
- 切换太少（> 1 小时）→ 减小日志文件或增加组数

### 4.5 强制日志模式

确保所有操作都生成 redo，用于 Data Guard。

```sql
-- 启用强制日志
ALTER DATABASE FORCE LOGGING;

-- 检查状态
SELECT force_logging FROM v$database;
```

## 5. 归档日志（Archive Logs）

### 5.1 归档模式 vs 非归档模式

| 特性 | ARCHIVELOG | NOARCHIVELOG |
|------|-----------|--------------|
| **介质恢复** | ✅ 支持 | ❌ 不支持 |
| **热备份** | ✅ 支持 | ❌ 不支持 |
| **Data Guard** | ✅ 支持 | ❌ 不支持 |
| **性能** | 略低 | 略高 |
| **存储空间** | 需要更多 | 较少 |

**生产环境强烈建议使用 ARCHIVELOG 模式**

### 5.2 启用归档模式

```sql
-- 1. 关闭数据库
SHUTDOWN IMMEDIATE;

-- 2. 启动到 MOUNT 状态
STARTUP MOUNT;

-- 3. 启用归档
ALTER DATABASE ARCHIVELOG;

-- 4. 打开数据库
ALTER DATABASE OPEN;

-- 5. 验证
ARCHIVE LOG LIST;
```

### 5.3 配置归档目标

```sql
-- 查看归档配置
ARCHIVE LOG LIST;

-- 设置归档目标
ALTER SYSTEM SET log_archive_dest_1='LOCATION=/u01/archivelogs' SCOPE=BOTH;
ALTER SYSTEM SET log_archive_dest_2='LOCATION=/u02/archivelogs' SCOPE=BOTH;

-- 设置归档格式
ALTER SYSTEM SET log_archive_format='arch_%t_%s_%r.arc' SCOPE=SPFILE;

-- 启用归档目标
ALTER SYSTEM SET log_archive_dest_state_1=ENABLE SCOPE=BOTH;
```

**归档格式参数**：
- `%t`：线程号
- `%s`：日志序列号
- `%r`：resetlogs ID

### 5.4 管理归档日志

```sql
-- 查看归档日志
SELECT 
    name,
    sequence#,
    first_time,
    next_time,
    blocks * block_size / 1024 / 1024 as size_mb,
    archived,
    status
FROM v$archived_log
WHERE completion_time > SYSDATE - 7
ORDER BY sequence# DESC;

-- 查看归档空间使用
SELECT 
    *
FROM v$recovery_file_dest;
```

### 5.5 清理归档日志

#### 使用 RMAN（推荐）

```bash
# 连接到 RMAN
rman target /

# 删除 7 天前的归档日志
RMAN> DELETE ARCHIVELOG ALL COMPLETED BEFORE 'SYSDATE-7';

# 或者基于保留策略
RMAN> DELETE ARCHIVELOG UNTIL TIME 'SYSDATE-7';

# 交叉检查并删除过期的归档日志
RMAN> CROSSCHECK ARCHIVELOG ALL;
RMAN> DELETE EXPIRED ARCHIVELOG ALL;
```

#### 手动删除（不推荐）

```bash
# 仅当 RMAN 不可用时使用
rm /u01/archivelogs/arch_*.arc
```

**警告**：手动删除后必须在 RMAN 中执行 `CROSSCHECK`

### 5.6 归档空间满的处理

**症状**：
- 数据库挂起
- ORA-00257: archiver error

**紧急处理**：
```bash
# 1. 连接到 RMAN
rman target /

# 2. 删除旧归档日志
RMAN> DELETE ARCHIVELOG ALL COMPLETED BEFORE 'SYSDATE-1';

# 3. 如果还不行，临时增加归档空间
ALTER SYSTEM SET db_recovery_file_dest_size=50G SCOPE=BOTH;
```

**预防措施**：
- 监控归档空间使用率
- 设置合理的保留策略
- 定期备份并删除归档日志

## 6. 表空间管理

### 6.1 表空间类型

| 类型 | 用途 | 示例 |
|------|------|------|
| **SYSTEM** | 数据字典 | 核心系统表空间 |
| **SYSAUX** | 辅助系统组件 | AWR、Statspack |
| **UNDOTBS** | 撤销数据 | 事务回滚、读一致性 |
| **TEMP** | 临时数据 | 排序、哈希 |
| **USERS** | 用户数据 | 应用数据 |
| **INDEX** | 索引数据 | 单独存放索引 |

### 6.2 创建表空间

```sql
-- 永久表空间
CREATE TABLESPACE app_data
DATAFILE '/u01/app/oracle/oradata/ORCL/app_data01.dbf'
SIZE 1G
AUTOEXTEND ON NEXT 100M MAXSIZE 10G
EXTENT MANAGEMENT LOCAL
SEGMENT SPACE MANAGEMENT AUTO;

-- 临时表空间
CREATE TEMPORARY TABLESPACE temp_large
TEMPFILE '/u01/app/oracle/oradata/ORCL/temp_large01.dbf'
SIZE 2G
AUTOEXTEND ON NEXT 200M MAXSIZE 10G
EXTENT MANAGEMENT LOCAL UNIFORM SIZE 1M;

-- 撤销表空间
CREATE UNDO TABLESPACE undotbs2
DATAFILE '/u01/app/oracle/oradata/ORCL/undotbs2_01.dbf'
SIZE 2G
AUTOEXTEND ON NEXT 100M MAXSIZE 5G;
```

### 6.3 表空间管理

#### 修改表空间

```sql
-- 添加数据文件
ALTER TABLESPACE app_data
ADD DATAFILE '/u01/app/oracle/oradata/ORCL/app_data02.dbf'
SIZE 1G AUTOEXTEND ON;

-- 设置为只读
ALTER TABLESPACE app_data READ ONLY;

-- 设置为读写
ALTER TABLESPACE app_data READ WRITE;

-- 离线/在线
ALTER TABLESPACE app_data OFFLINE NORMAL;
ALTER TABLESPACE app_data ONLINE;

-- 重命名表空间
ALTER TABLESPACE app_data RENAME TO app_data_new;
```

#### 删除表空间

```sql
-- 删除空表空间
DROP TABLESPACE app_data;

-- 删除包含数据的表空间
DROP TABLESPACE app_data INCLUDING CONTENTS;

-- 删除表空间及其数据文件
DROP TABLESPACE app_data INCLUDING CONTENTS AND DATAFILES;

-- 同时删除引用完整性约束
DROP TABLESPACE app_data INCLUDING CONTENTS AND DATAFILES CASCADE CONSTRAINTS;
```

### 6.4 表空间使用监控

```sql
-- 表空间使用率
SELECT 
    ts.tablespace_name,
    ROUND(SUM(df.bytes)/1024/1024, 2) as allocated_mb,
    ROUND(SUM(NVL(fs.bytes, 0))/1024/1024, 2) as free_mb,
    ROUND((SUM(df.bytes) - SUM(NVL(fs.bytes, 0))) / SUM(df.bytes) * 100, 2) as used_pct,
    COUNT(df.file_name) as file_count
FROM dba_tablespaces ts
JOIN dba_data_files df ON ts.tablespace_name = df.tablespace_name
LEFT JOIN dba_free_space fs ON df.file_id = fs.file_id
GROUP BY ts.tablespace_name
ORDER BY used_pct DESC;

-- 临时表空间使用
SELECT 
    tablespace_name,
    bytes_used/1024/1024 as used_mb,
    bytes_free/1024/1024 as free_mb
FROM v$temp_space_header;

-- 查找大对象
SELECT 
    owner,
    segment_name,
    segment_type,
    tablespace_name,
    bytes/1024/1024 as size_mb
FROM dba_segments
ORDER BY bytes DESC
FETCH FIRST 20 ROWS ONLY;
```

### 6.5 表空间最佳实践

✅ **分离数据和索引**：不同的表空间  
✅ **使用本地管理**：EXTENT MANAGEMENT LOCAL  
✅ **启用自动段空间管理**：SEGMENT SPACE MANAGEMENT AUTO  
✅ **合理设置自动扩展**：避免频繁扩展  
✅ **监控使用率**：超过 80% 时预警  
✅ **定期整理碎片**：对于字典管理表空间

## 7. 参数文件和密码文件

### 7.1 参数文件类型

| 类型 | 名称            | 特点 |
|------|---------------|------|
| **PFILE** | `init<SID>.ora`              | 文本文件，可手动编辑 |
| **SPFILE** | `spfile<SID>.ora` | 二进制文件，动态修改 |

### 7.2 查看和管理参数

```sql
-- 查看当前参数
SHOW PARAMETER db_cache_size;

-- 查看所有参数
SELECT name, value, isdefault, issys_modifiable
FROM v$parameter
ORDER BY name;

-- 修改参数
ALTER SYSTEM SET db_cache_size=2G SCOPE=BOTH;

-- 修改 SPFILE（需重启）
ALTER SYSTEM SET processes=300 SCOPE=SPFILE;

-- 从 SPFILE 创建 PFILE
CREATE PFILE='/tmp/init.ora' FROM SPFILE;

-- 从 PFILE 创建 SPFILE
CREATE SPFILE FROM PFILE='/tmp/init.ora';
```

### 7.3 密码文件

```bash
# 创建密码文件
orapwd file=$ORACLE_HOME/dbs/orapwORCL password=oracle entries=10

# 允许远程 SYSDBA 连接
ALTER SYSTEM SET remote_login_passwordfile=EXCLUSIVE SCOPE=SPFILE;
```

## 9. 存储规划最佳实践

### 9.1 文件系统布局

```
/u01 - Oracle 软件
/u02 - 数据文件
/u03 - 重做日志
/u04 - 控制文件
/u05 - 归档日志
/u06 - 备份
```

### 9.2 存储建议

✅ **RAID 配置**：
- RAID 1+0：数据文件和日志文件
- RAID 5：归档日志和备份

✅ **分离 I/O**：
- 数据文件和日志文件分开磁盘
- 控制文件多路复用到不同控制器

✅ **容量规划**：
- 预留 20-30% 空闲空间
- 监控增长趋势

---

## 10. 总结

### 10.1 核心知识点

✅ **数据文件**：存储实际数据，支持自动扩展  
✅ **控制文件**：数据库元数据，必须多路复用  
✅ **重做日志**：变更记录，循环使用  
✅ **归档日志**：历史 redo，用于恢复  
✅ **表空间**：逻辑存储单元，多种类型  
✅ **参数文件**：PFILE vs SPFILE  

### 10.2 最佳实践

📌 控制文件至少 3 份，分布在不同磁盘  
📌 重做日志每组至少 2 个成员  
📌 生产环境使用 ARCHIVELOG 模式  
📌 定期备份控制文件和参数文件  
📌 监控表空间和归档空间使用率  
📌 使用 OMF 简化管理  

### 10.3 常见问题

- 控制文件损坏 → 从备份恢复
- 归档空间满 → 清理旧归档日志
- 表空间满 → 添加数据文件或调整大小
- 日志切换频繁 → 增大日志文件
