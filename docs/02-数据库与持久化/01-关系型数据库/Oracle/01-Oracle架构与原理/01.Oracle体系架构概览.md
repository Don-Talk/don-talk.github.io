## 1. Oracle 架构核心概念

### 1.1 实例 vs 数据库

这是 Oracle 最重要的概念区分！

<!-- TODO: 插入图片：Oracle 实例与数据库关系图 -->

**Oracle 实例（Instance）**：
* 内存结构（SGA + PGA）
* 后台进程（PMON、SMON、DBWR 等）
* **特点**：动态的、临时的，启动时创建，关闭时销毁

**Oracle 数据库（Database）**：
* 物理文件（数据文件、控制文件、日志文件）
* **特点**：静态的、持久的，存储在磁盘上

**关系**：
```
一个实例可以挂载并打开一个数据库
一个数据库可以被多个实例访问（RAC 环境）
```

### 1.2 架构总览

```
┌─────────────────────────────────────────────┐
│           Oracle 实例 (Instance)             │
│  ┌──────────────┐  ┌─────────────────────┐  │
│  │   SGA 内存   │  │   后台进程          │  │
│  │  (共享区)    │  │  PMON/SMON/DBWR...  │  │
│  └──────────────┘  └─────────────────────┘  │
└─────────────────────────────────────────────┘
                    ↓ 读写
┌─────────────────────────────────────────────┐
│         Oracle 数据库 (Database)             │
│  ┌────────┐ ┌────────┐ ┌────────────────┐  │
│  │数据文件│ │控制文件│ │ 重做日志文件    │  │
│  │.dbf    │ │.ctl    │ │ .log            │  │
│  └────────┘ └────────┘ └────────────────┘  │
└─────────────────────────────────────────────┘
```

## 2. 内存结构详解

### 2.1 SGA（System Global Area）

系统全局区，所有用户进程共享的内存区域。

<!-- TODO: 插入图片：SGA 组件结构图 -->

**主要组件**：

| 组件 | 作用 | 大小建议 |
|------|------|---------|
| **Database Buffer Cache** | 缓存数据块 | SGA 的 60-70% |
| **Shared Pool** | 缓存 SQL 和执行计划 | SGA 的 20-30% |
| **Redo Log Buffer** | 缓存重做日志条目 | 固定较小 |
| **Large Pool** | RMAN、并行查询 | 可选 |
| **Java Pool** | Java 对象缓存 | 可选 |
| **Streams Pool** | Streams 功能 | 可选 |

#### Database Buffer Cache

```sql
-- 查看缓冲区命中率
SELECT 
    name,
    value
FROM v$sysstat
WHERE name IN ('db block gets', 'consistent gets', 'physical reads');

-- 命中率计算
-- Hit Ratio = 1 - (physical reads / (db block gets + consistent gets))
```

**最佳实践**：命中率应 > 90%

#### Shared Pool

包含两个重要子组件：

1. **Library Cache**：缓存 SQL 语句和执行计划
2. **Dictionary Cache**：缓存数据字典信息

```sql
-- 查看 Library Cache 命中率
SELECT 
    namespace,
    gets,
    gethits,
    ROUND(gethits/gets*100, 2) as hit_ratio
FROM v$librarycache;
```

### 2.2 PGA（Program Global Area）

程序全局区，每个服务器进程私有的内存区域。

**包含内容**：
* 会话变量
* 排序区（Sort Area）
* 哈希区（Hash Area）
* 游标状态

```sql
-- 查看 PGA 使用情况
SELECT 
    program,
    pga_used_mem,
    pga_alloc_mem,
    pga_max_mem
FROM v$process
ORDER BY pga_used_mem DESC;
```

### 2.3 UGA（User Global Area）

用户全局区，存储会话相关信息。

* **专用服务器模式**：UGA 在 PGA 中
* **共享服务器模式**：UGA 在 SGA 的 Large Pool 中

## 3. 后台进程详解

Oracle 启动时会自动启动多个后台进程，各司其职。

<!-- TODO: 插入图片：后台进程工作流程图 -->

### 3.1 核心进程（必须）

#### DBWR（Database Writer）- 数据库写入进程

**职责**：
* 将脏数据块（Modified Blocks）从 Buffer Cache 写入数据文件
* 保证数据持久化

**触发条件**：
* Checkpoint 发生时
* Buffer Cache 空间不足
* 超时（每 3 秒）
* 表空间 offline

```sql
-- 查看 DBWR 统计信息
SELECT name, value
FROM v$sysstat
WHERE name LIKE 'DBWR%';
```

#### LGWR（Log Writer）- 日志写入进程

**职责**：
* 将 Redo Log Buffer 中的日志条目写入联机重做日志文件
* 保证事务的可恢复性

**触发条件**：
* 用户提交事务（COMMIT）
* Redo Log Buffer 满 1/3
* 每秒定时写入
* DBWR 写入前

```sql
-- 查看 LGWR 统计
SELECT name, value
FROM v$sysstat
WHERE name LIKE 'redo%';
```

#### CKPT（Checkpoint）- 检查点进程

**职责**：
* 更新控制文件和数据文件头，记录检查点位置
* 协调 DBWR 写入

**检查点类型**：
* 完全检查点（Full Checkpoint）
* 增量检查点（Incremental Checkpoint）

```sql
-- 手动触发检查点
ALTER SYSTEM CHECKPOINT;
```

#### SMON（System Monitor）- 系统监控进程

**职责**：
* 实例恢复（Instance Recovery）
* 清理临时段
* 合并空闲空间
* 监控其他进程

#### PMON（Process Monitor）- 进程监控进程

**职责**：
* 清理失败的进程
* 释放锁和資源
* 注册服务到监听器

### 3.2 可选进程

| 进程 | 作用 |
|------|------|
| **ARCn** | 归档日志进程 |
| **RECO** | 分布式事务恢复 |
| **MMON** | 自动负载信息库监控 |
| **CJQ0** | 作业队列进程 |
| **QMNC** | 高级队列监控 |

## 4. 物理存储结构

### 4.1 三大类文件

<!-- TODO: 插入图片：物理文件关系图 -->

#### 数据文件（Data Files）

* **扩展名**：`.dbf`
* **作用**：存储实际数据（表、索引等）
* **特点**：一个表空间可以包含多个数据文件

```sql
-- 查看数据文件
SELECT 
    file_name,
    tablespace_name,
    bytes/1024/1024 as size_mb,
    autoextensible
FROM dba_data_files;
```

#### 控制文件（Control Files）

* **扩展名**：`.ctl`
* **作用**：记录数据库的物理结构
* **内容**：
  * 数据库名称
  * 数据文件和日志文件的位置
  * 检查点信息
  * SCN（系统变更号）

```sql
-- 查看控制文件
SELECT name FROM v$controlfile;

-- 备份控制文件
ALTER DATABASE BACKUP CONTROLFILE TO TRACE;
```

**最佳实践**：至少镜像 3 份控制文件到不同磁盘

#### 重做日志文件（Redo Log Files）

* **扩展名**：`.log`
* **作用**：记录所有数据变更，用于恢复
* **组织方式**：日志组（Group），每组多个成员（Member）

```sql
-- 查看日志文件
SELECT 
    group#,
    member,
    bytes/1024/1024 as size_mb,
    status
FROM v$logfile;

-- 查看日志组状态
SELECT 
    group#,
    sequence#,
    bytes/1024/1024 as size_mb,
    status,
    archived
FROM v$log;
```

**日志切换**：
* 当前日志组写满后，切换到下一组
* 触发 LGWR 写入
* 可能触发 Checkpoint

```sql
-- 手动切换日志
ALTER SYSTEM SWITCH LOGFILE;
```

### 4.2 其他重要文件

| 文件类型 | 作用 |
|---------|------|
| **参数文件** | pfile/spfile，存储启动参数 |
| **密码文件** | orapw，存储 SYS 密码 |
| **告警日志** | alert.log，记录重要事件 |
| **跟踪文件** | trace，诊断信息 |

```sql
-- 查看告警日志位置
SELECT value FROM v$diag_info WHERE name = 'Diag Trace';
```

## 5. 逻辑存储结构

### 5.1 层次关系

```
数据库（Database）
  └─ 表空间（Tablespace）
      └─ 段（Segment）
          └─ 区（Extent）
              └─ 块（Block）
```

<!-- TODO: 插入图片：逻辑存储层次图 -->

### 5.2 表空间（Tablespace）

**类型**：
* **SYSTEM**：系统表空间，存储数据字典
* **SYSAUX**：辅助系统表空间
* **UNDOTBS**：撤销表空间
* **TEMP**：临时表空间
* **USERS**：用户表空间

```sql
-- 查看所有表空间
SELECT 
    tablespace_name,
    status,
    contents,
    extent_management
FROM dba_tablespaces;

-- 创建表空间
CREATE TABLESPACE my_ts
DATAFILE '/u01/app/oracle/oradata/my_ts01.dbf'
SIZE 100M
AUTOEXTEND ON NEXT 10M MAXSIZE 1G;
```

### 5.3 段（Segment）

**类型**：
* 表段（Table Segment）
* 索引段（Index Segment）
* 回滚段（Rollback Segment）
* 临时段（Temporary Segment）

### 5.4 区（Extent）

* 一组连续的数据块
* 段的空间分配单位

### 5.5 块（Block）

* 最小的 I/O 单位
* 默认大小：8KB
* 由 `DB_BLOCK_SIZE` 参数决定

```sql
-- 查看块大小
SHOW PARAMETER db_block_size;
```

## 6. Oracle 启动与关闭流程

### 6.1 启动三阶段

<!-- TODO: 插入图片：启动流程图 -->

#### NOMOUNT 阶段

* 读取参数文件（spfile/pfile）
* 分配 SGA 内存
* 启动后台进程
* **此时**：实例已启动，数据库未加载

```sql
STARTUP NOMOUNT;
```

#### MOUNT 阶段

* 读取控制文件
* 获取数据文件和日志文件信息
* **此时**：实例已启动，数据库已加载但未打开

```sql
ALTER DATABASE MOUNT;
```

#### OPEN 阶段

* 打开数据文件和日志文件
* 进行实例恢复（如果需要）
* **此时**：数据库完全可用

```sql
ALTER DATABASE OPEN;
```

**一键启动**：
```sql
STARTUP;  -- 等价于 STARTUP NOMOUNT + MOUNT + OPEN
```

### 6.2 关闭四种模式

| 模式 | 命令 | 特点 |
|------|------|------|
| **NORMAL** | `SHUTDOWN NORMAL` | 等待所有用户断开连接（默认） |
| **TRANSACTIONAL** | `SHUTDOWN TRANSACTIONAL` | 等待事务完成 |
| **IMMEDIATE** | `SHUTDOWN IMMEDIATE` | 回滚未提交事务，立即关闭（推荐） |
| **ABORT** | `SHUTDOWN ABORT` | 强制终止，下次启动需要恢复（危险） |

```sql
-- 推荐关闭方式
SHUTDOWN IMMEDIATE;
```

## 7. 实战练习

### 练习 1：查看实例状态

```sql
-- 查看实例状态
SELECT instance_name, status, startup_time
FROM v$instance;

-- 查看数据库状态
SELECT name, open_mode, log_mode
FROM v$database;
```

### 练习 2：查看内存使用

```sql
-- 查看 SGA 各组件大小
SELECT component, current_size/1024/1024 as size_mb
FROM v$sga_dynamic_components;

-- 查看 PGA 使用
SELECT 
    SUM(pga_used_mem)/1024/1024 as used_mb,
    SUM(pga_alloc_mem)/1024/1024 as alloc_mb
FROM v$process;
```

### 练习 3：查看后台进程

```sql
-- 查看所有后台进程
SELECT name, description
FROM v$bgprocess
WHERE paddr != '00';
```

### 练习 4：查看物理文件

```sql
-- 统计各类文件数量
SELECT 'Data Files' as type, COUNT(*) as count FROM dba_data_files
UNION ALL
SELECT 'Control Files', COUNT(*) FROM v$controlfile
UNION ALL
SELECT 'Log Files', COUNT(*) FROM v$logfile;
```

## 8. 常见问题

### 8.1 ORA-01034: ORACLE not available

**原因**：实例未启动

**解决**：
```bash
sqlplus / as sysdba
STARTUP;
```

### 8.2 ORA-00205: error in identifying control file

**原因**：控制文件损坏或路径错误

**解决**：
1. 检查 `CONTROL_FILES` 参数
2. 从备份恢复控制文件

### 8.3 SGA 内存不足

**症状**：ORA-04031

**解决**：
```sql
-- 调整 SGA 大小
ALTER SYSTEM SET sga_target=2G SCOPE=SPFILE;
-- 重启生效
```

## 9. 总结

### 9.1 核心知识点

✅ **实例 vs 数据库**：内存+进程 vs 物理文件  
✅ **SGA 组件**：Buffer Cache、Shared Pool、Redo Log Buffer  
✅ **后台进程**：DBWR、LGWR、CKPT、SMON、PMON  
✅ **物理文件**：数据文件、控制文件、重做日志文件  
✅ **逻辑结构**：表空间 → 段 → 区 → 块  
✅ **启动流程**：NOMOUNT → MOUNT → OPEN  

### 9.2 最佳实践

📌 多路复用控制文件和日志文件  
📌 合理配置 SGA 和 PGA 大小  
📌 定期监控后台进程状态  
📌 使用 SHUTDOWN IMMEDIATE 关闭数据库  
📌 保持告警日志清洁
