## 1. ASM 概述

### 1.1 什么是 ASM？

ASM（Automatic Storage Management，自动存储管理）是 Oracle 提供的卷管理器和文件系统。

**特点**：
- 简化存储管理
- 自动负载均衡
- 支持条带化和镜像
- 集成在 Oracle 生态中

<!-- TODO: 插入图片：ASM 架构示意图 -->

### 1.2 ASM vs 传统文件系统

| 特性 | ASM | 传统文件系统 |
|------|-----|------------|
| **条带化** | ✅ 自动 | ❌ 需手动配置 |
| **镜像** | ✅ 内置 | ❌ 需 RAID |
| **在线扩展** | ✅ 支持 | ⚠️ 复杂 |
| **负载均衡** | ✅ 自动 | ❌ 无 |
| **管理复杂度** | 低 | 高 |
| **适用场景** | Oracle 数据库 | 通用 |

### 1.3 ASM 架构

```
ASM Instance (内存结构)
    ↓
Disk Groups (磁盘组)
    ↓
Failure Groups (故障组)
    ↓
Disks (物理磁盘)
```

## 2. ASM 实例

### 2.1 ASM 实例组成

- **SGA**：共享池、大池
- **后台进程**：RBAL、ARBn、GMON、PSP0 等
- **参数文件**：+ASM 实例的参数

### 2.2 启动 ASM 实例

```bash
# 切换到 grid 用户
su - grid

# 启动 ASM 实例
sqlplus / as sysasm

SQL> STARTUP;

# 查看 ASM 实例状态
SQL> SELECT instance_name, status FROM v$instance;
```

### 2.3 ASM 参数

```sql
-- 查看 ASM 参数
SHOW PARAMETER asm;

-- 关键参数
asm_diskgroups      -- 自动挂载的磁盘组
asm_diskstring      -- 磁盘搜索路径
asm_power_limit     -- rebalance 功率限制
```

## 3. ASM 磁盘组

### 3.1 磁盘组冗余类型

#### （1）EXTERNAL REDUNDANCY（外部冗余）

- 依赖外部 RAID
- ASM 不做镜像
- 需要硬件 RAID 保护

```sql
CREATE DISKGROUP data EXTERNAL REDUNDANCY
DISK '/dev/oracleasm/disks/DISK1';
```

#### （2）NORMAL REDUNDANCY（普通冗余）

- 双向镜像
- 至少需要 2 个故障组
- 容忍 1 个故障组失败

```sql
CREATE DISKGROUP data NORMAL REDUNDANCY
FAILGROUP fg1 DISK '/dev/oracleasm/disks/DISK1'
FAILGROUP fg2 DISK '/dev/oracleasm/disks/DISK2';
```

#### （3）HIGH REDUNDANCY（高冗余）

- 三向镜像
- 至少需要 3 个故障组
- 容忍 2 个故障组失败

```sql
CREATE DISKGROUP data HIGH REDUNDANCY
FAILGROUP fg1 DISK '/dev/oracleasm/disks/DISK1'
FAILGROUP fg2 DISK '/dev/oracleasm/disks/DISK2'
FAILGROUP fg3 DISK '/dev/oracleasm/disks/DISK3';
```

### 3.2 创建磁盘组

```sql
-- 基本语法
CREATE DISKGROUP diskgroup_name
[REDUNDANCY_TYPE]
[FAILGROUP ...]
DISK 'disk_path' [...]
ATTRIBUTE 'attribute_name' = 'value';

-- 示例 1：外部冗余
CREATE DISKGROUP DATA EXTERNAL REDUNDANCY
DISK '/dev/oracleasm/disks/DISK1',
     '/dev/oracleasm/disks/DISK2',
     '/dev/oracleasm/disks/DISK3'
ATTRIBUTE 'au_size' = '4M',
          'compatible.asm' = '19.0.0',
          'compatible.rdbms' = '19.0.0';

-- 示例 2：普通冗余
CREATE DISKGROUP FRA NORMAL REDUNDANCY
FAILGROUP FG1 DISK '/dev/oracleasm/disks/FRA1',
              '/dev/oracleasm/disks/FRA2'
FAILGROUP FG2 DISK '/dev/oracleasm/disks/FRA3',
              '/dev/oracleasm/disks/FRA4'
ATTRIBUTE 'au_size' = '1M';
```

### 3.3 删除磁盘组

```sql
-- 删除空磁盘组
DROP DISKGROUP DATA;

-- 删除包含内容的磁盘组
DROP DISKGROUP DATA INCLUDING CONTENTS;

-- 强制删除
DROP DISKGROUP DATA FORCE INCLUDING CONTENTS;
```

### 3.4 挂载和卸载磁盘组

```sql
-- 挂载磁盘组
ALTER DISKGROUP DATA MOUNT;

-- 卸载磁盘组
ALTER DISKGROUP DATA DISMOUNT;

-- 挂载所有磁盘组
ALTER DISKGROUP ALL MOUNT;
```

## 4. ASM 磁盘管理

### 4.1 添加磁盘

```sql
-- 向磁盘组添加磁盘
ALTER DISKGROUP DATA ADD DISK
'/dev/oracleasm/disks/DISK4' NAME disk4,
'/dev/oracleasm/disks/DISK5' NAME disk5;

-- 指定故障组
ALTER DISKGROUP DATA ADD DISK
FAILGROUP FG1 '/dev/oracleasm/disks/DISK6';
```

### 4.2 删除磁盘

```sql
-- 删除磁盘
ALTER DISKGROUP DATA DROP DISK disk4;

-- 删除多个磁盘
ALTER DISKGROUP DATA DROP DISK disk4, disk5;

-- 取消删除（在 rebalance 完成前）
ALTER DISKGROUP DATA UNDROP DISKS;
```

### 4.3 调整磁盘大小

```sql
-- 调整磁盘大小
ALTER DISKGROUP DATA RESIZE DISK disk4 SIZE 100G;

-- 调整所有磁盘
ALTER DISKGROUP DATA RESIZE ALL;
```

### 4.4 查看磁盘信息

```sql
-- 查看磁盘组
SELECT name, state, type, total_mb, free_mb
FROM v$asm_diskgroup;

-- 查看磁盘
SELECT 
    dg.name as diskgroup,
    d.name as disk_name,
    d.path,
    d.total_mb,
    d.free_mb,
    d.state
FROM v$asm_disk d, v$asm_diskgroup dg
WHERE d.group_number = dg.group_number
ORDER BY dg.name, d.name;
```

## 5. Rebalance 操作

### 5.1 什么是 Rebalance？

当添加或删除磁盘时，ASM 自动重新分布数据以保持均衡。

### 5.2 监控 Rebalance

```sql
-- 查看 rebalance 进度
SELECT 
    operation,
    state,
    power,
    est_minutes,
    sofar,
    est_work
FROM v$asm_operation;

-- 查看历史 rebalance
SELECT * FROM v$asm_operation_history;
```

### 5.3 调整 Rebalance 功率

```sql
-- 设置功率（0-11，默认 1）
ALTER DISKGROUP DATA REBALANCE POWER 4;

-- 系统级设置
ALTER SYSTEM SET asm_power_limit = 4;
```

**功率说明**：
- 0：暂停 rebalance
- 1：默认，较慢
- 11：最快，占用资源多

## 6. ASM 文件管理

### 6.1 ASM 文件名格式

```
+DISKGROUP/db_name/file_type/tag.file_number.incarnation

示例：
+DATA/ORCL/DATAFILE/users.256.123456789
+FRA/ORCL/ARCHIVELOG/2026_05_19/thread_1_seq_100.257.123456790
```

### 6.2 创建 ASM 目录

```bash
# 使用 ASMCMD
asmcmd

ASMCMD> cd +DATA
ASMCMD> mkdir ORCL
ASMCMD> mkdir ORCL/DATAFILE
```

### 6.3 复制文件到 ASM

```bash
# 从文件系统复制到 ASM
ASMCMD> cp /tmp/datafile.dbf +DATA/ORCL/DATAFILE/

# 从 ASM 复制到文件系统
ASMCMD> cp +DATA/ORCL/DATAFILE/users.256.123456789 /tmp/users.dbf
```

### 6.4 删除 ASM 文件

```bash
ASMCMD> rm +DATA/ORCL/DATAFILE/old_file.dbf
```

## 7. RMAN 与 ASM

### 7.1 备份到 ASM

```sql
-- 备份数据库到 ASM
RMAN> BACKUP DATABASE FORMAT '+FRA';

-- 备份归档日志
RMAN> BACKUP ARCHIVELOG ALL FORMAT '+FRA';
```

### 7.2 从 ASM 恢复

```sql
-- 从 ASM 恢复
RMAN> RESTORE DATABASE FROM '+FRA';
RMAN> RECOVER DATABASE;
```

### 7.3 移动数据文件到 ASM

```sql
-- 方法 1：RMAN
RMAN> COPY DATAFILE '/u01/oradata/users01.dbf' TO '+DATA';

-- 方法 2：ALTER DATABASE
ALTER DATABASE MOVE DATAFILE '/u01/oradata/users01.dbf' 
TO '+DATA';
```

## 8. ACFS（ASM Cluster File System）

### 8.1 什么是 ACFS？

ACFS 是基于 ASM 的通用文件系统，可以存储非数据库文件。

**用途**：
- 存储 Oracle Home
- 存储备份文件
- 存储应用程序文件

### 8.2 创建 ACFS

```bash
# 创建 ACFS 卷
asmca -createVolume -volumeName acfs_vol -diskGroup DATA -sizeGB 100

# 创建 ACFS 文件系统
mkfs -t acfs /dev/asm/acfs_vol-123

# 挂载
mount -t acfs /dev/asm/acfs_vol-123 /u01/acfs
```

## 9. 监控和维护

### 9.1 监控脚本

```sql
-- 磁盘组使用情况
SELECT 
    name,
    ROUND(total_mb/1024, 2) as total_gb,
    ROUND(free_mb/1024, 2) as free_gb,
    ROUND((total_mb - free_mb) / total_mb * 100, 2) as used_pct
FROM v$asm_diskgroup;

-- 磁盘 I/O 统计
SELECT 
    dg.name as diskgroup,
    d.name as disk,
    d.reads,
    d.writes,
    d.read_time,
    d.write_time
FROM v$asm_disk d, v$asm_diskgroup dg
WHERE d.group_number = dg.group_number;

-- 告警：使用率超过 85%
SELECT name, used_pct
FROM (
    SELECT 
        name,
        ROUND((total_mb - free_mb) / total_mb * 100, 2) as used_pct
    FROM v$asm_diskgroup
)
WHERE used_pct > 85;
```

### 9.2 常见问题处理

**问题 1：磁盘组无法挂载**
```sql
-- 检查磁盘状态
SELECT path, header_status, mode_status
FROM v$asm_disk;

-- 修复磁盘头
-- 需要谨慎操作，可能需要重建磁盘组
```

**问题 2：Rebalance 太慢**
```sql
-- 增加功率
ALTER DISKGROUP DATA REBALANCE POWER 8;
```

**问题 3：空间不足**
```sql
-- 添加磁盘
ALTER DISKGROUP DATA ADD DISK '/dev/oracleasm/disks/NEW_DISK';
```

## 10. 最佳实践

### 10.1 设计原则

✅ **选择合适的冗余级别**
- 开发环境：EXTERNAL
- 生产环境：NORMAL 或 HIGH

✅ **合理设置 AU 大小**
- OLTP：1M 或 4M
- 数据仓库：4M 或更大

✅ **均匀分布磁盘**
- 相同大小的磁盘
- 相同的性能特性

✅ **监控 rebalance**
- 避免在业务高峰期 rebalance
- 适当设置功率

### 10.2 运维建议

📌 **定期检查磁盘健康**  
📌 **监控空间使用率**  
📌 **保持 firmware 更新**  
📌 **测试故障恢复**  
📌 **记录配置变更**
