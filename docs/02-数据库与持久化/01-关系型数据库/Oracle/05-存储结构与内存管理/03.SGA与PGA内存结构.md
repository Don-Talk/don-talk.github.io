## 1. Oracle 内存架构概述

### 1.1 内存组成

```
Oracle 内存
    ├── SGA（System Global Area，系统全局区）
    │   ├── Shared Pool
    │   ├── Buffer Cache
    │   ├── Redo Log Buffer
    │   ├── Large Pool
    │   ├── Java Pool
    │   └── Streams Pool
    └── PGA（Program Global Area，程序全局区）
        ├── Sort Area
        ├── Hash Area
        ├── Bitmap Merge Area
        └── Session Memory
```

### 1.2 SGA vs PGA

| 特性 | SGA | PGA |
|------|-----|-----|
| **共享性** | 所有进程共享 | 每个进程独立 |
| **分配时间** | 实例启动时 | 会话创建时 |
| **主要用途** | 缓存、共享数据 | 排序、哈希、会话信息 |
| **大小控制** | sga_target | pga_aggregate_target |
| **持久性** | 实例级 | 会话级 |

## 2. SGA 组件详解

### 2.1 Shared Pool（共享池）

**作用**：存储 SQL 语句、执行计划、数据字典缓存。

#### （1）Library Cache（库缓存）

- 存储 SQL 文本和执行计划
- 支持软解析（Soft Parse）
- LRU 算法管理

```sql
-- 查看 Library Cache 命中率
SELECT 
    namespace,
    gets,
    gethits,
    ROUND(gethitratio * 100, 2) as hit_ratio_pct
FROM v$librarycache;

-- 理想值：> 90%
```

#### （2）Dictionary Cache（数据字典缓存）

- 存储表、列、用户等元数据
- 减少数据字典查询

```sql
-- 查看 Dictionary Cache 命中率
SELECT 
    parameter,
    gets,
    getmisses,
    ROUND((1 - getmisses/NULLIF(gets,0)) * 100, 2) as hit_ratio_pct
FROM v$rowcache
WHERE gets > 0
ORDER BY hit_ratio_pct;

-- 理想值：> 95%
```

#### （3）Shared Pool 调优

```sql
-- 查看 Shared Pool 大小
SHOW PARAMETER shared_pool_size;

-- 查看使用率
SELECT 
    pool,
    name,
    bytes/1024/1024 as size_mb
FROM v$sgastat
WHERE pool = 'shared pool'
ORDER BY bytes DESC;

-- 刷新 Shared Pool（清除缓存）
ALTER SYSTEM FLUSH SHARED_POOL;
```

### 2.2 Buffer Cache（缓冲区缓存）

**作用**：缓存从数据文件读取的数据块。

#### （1）工作原理

```
用户请求数据
    ↓
检查 Buffer Cache
    ↓
命中 → 直接返回（Logical I/O）
    ↓
未命中 → 从磁盘读取 → 放入 Buffer Cache（Physical I/O）
```

#### （2）Buffer Cache 命中率

```sql
-- 计算 Buffer Cache 命中率
SELECT 
    ROUND((1 - (physical_reads / NULLIF(db_block_gets + consistent_gets, 0))) * 100, 2) as hit_ratio_pct
FROM v$sysstat
WHERE name IN ('db block gets', 'consistent gets', 'physical reads');

-- 理想值：> 90%
```

#### （3）多缓冲池

```sql
-- DEFAULT 池：正常数据
-- KEEP 池：频繁访问的小表
-- RECYCLE 池：很少访问的大表

-- 将表分配到 KEEP 池
ALTER TABLE employees STORAGE (BUFFER_POOL KEEP);

-- 将表分配到 RECYCLE 池
ALTER TABLE large_history STORAGE (BUFFER_POOL RECYCLE);
```

#### （4）调优建议

```sql
-- 查看 Buffer Cache 建议
SELECT 
    size_for_estimate,
    estd_physical_read_factor,
    estd_physical_reads
FROM v$db_cache_advice
WHERE name = 'DEFAULT';

-- 调整 Buffer Cache 大小
ALTER SYSTEM SET db_cache_size = 2G;
```

### 2.3 Redo Log Buffer（重做日志缓冲区）

**作用**：临时存储 redo 条目，然后写入 redo log 文件。

```sql
-- 查看 Redo Log Buffer 大小
SHOW PARAMETER log_buffer;

-- 查看 redo 生成量
SELECT name, value
FROM v$sysstat
WHERE name LIKE 'redo size%';
```

**特点**：
- 固定大小（通常 1-3 MB）
- LGWR 进程定期写入磁盘
- 不能直接调整大小（由 Oracle 自动管理）

### 2.4 Large Pool（大池）

**作用**：为大型内存分配提供空间，避免与 Shared Pool 竞争。

**用途**：
- RMAN 备份和恢复
- 并行查询
- Shared Server 会话
- I/O 奴隶进程

```sql
-- 查看 Large Pool 大小
SHOW PARAMETER large_pool_size;

-- 查看使用情况
SELECT * FROM v$sgastat WHERE pool = 'large pool';
```

### 2.5 Java Pool（Java 池）

**作用**：存储 Java 代码和对象。

```sql
-- 查看 Java Pool 大小
SHOW PARAMETER java_pool_size;
```

**注意**：如果不使用 Java，可以设置为 0。

### 2.6 Streams Pool（流池）

**作用**：用于 Oracle Streams 和 GoldenGate。

```sql
-- 查看 Streams Pool 大小
SHOW PARAMETER streams_pool_size;
```

## 3. PGA 详解

### 3.1 PGA 组成

#### （1）Sort Area（排序区）

- ORDER BY 操作
- GROUP BY 操作
- DISTINCT 操作
- CREATE INDEX

#### （2）Hash Area（哈希区）

- Hash Join
- Hash Aggregation

#### （3）Bitmap Merge Area（位图合并区）

- Bitmap 索引操作

#### （4）Session Memory（会话内存）

- 会话变量
- 游标状态
- 权限信息

### 3.2 PGA 管理模式

#### （1）自动 PGA 管理（推荐）

```sql
-- 设置目标大小
ALTER SYSTEM SET pga_aggregate_target = 2G;

-- Oracle 自动分配给各个会话
```

#### （2）手动 PGA 管理（已过时）

```sql
-- 不推荐使用
ALTER SYSTEM SET workarea_size_policy = MANUAL;
ALTER SYSTEM SET sort_area_size = 10M;
ALTER SYSTEM SET hash_area_size = 10M;
```

### 3.3 监控 PGA

```sql
-- 查看 PGA 使用情况
SELECT 
    name,
    value/1024/1024 as size_mb
FROM v$pgastat;

-- 查看各会话 PGA 使用
SELECT 
    s.sid,
    s.username,
    p.program,
    ROUND(pga_used_mem/1024/1024, 2) as used_mb,
    ROUND(pga_alloc_mem/1024/1024, 2) as alloc_mb
FROM v$session s, v$process p
WHERE s.paddr = p.addr
ORDER BY pga_used_mem DESC;

-- 查看 PGA 建议
SELECT 
    pga_target_for_estimate/1024/1024 as target_mb,
    estd_pga_cache_hit_percentage,
    estd_overalloc_count
FROM v$pga_target_advice;
```

## 4. 自动内存管理（AMM）

### 4.1 什么是 AMM？

Oracle 11g+ 提供的自动内存管理，同时管理 SGA 和 PGA。

### 4.2 启用 AMM

```sql
-- 设置目标内存
ALTER SYSTEM SET memory_target = 4G SCOPE=SPFILE;

-- 设置最大内存
ALTER SYSTEM SET memory_max_target = 6G SCOPE=SPFILE;

-- 重启实例
SHUTDOWN IMMEDIATE;
STARTUP;
```

### 4.3 禁用 AMM

```sql
-- 如果需要使用 HugePages，需要禁用 AMM
ALTER SYSTEM SET memory_target = 0 SCOPE=SPFILE;
ALTER SYSTEM SET sga_target = 3G SCOPE=SPFILE;
ALTER SYSTEM SET pga_aggregate_target = 1G SCOPE=SPFILE;

SHUTDOWN IMMEDIATE;
STARTUP;
```

## 5. 内存调优

### 5.1 SGA 调优

#### （1）Shared Pool 调优

```sql
-- 问题：Library Cache 命中率低
-- 解决：增加 Shared Pool 大小
ALTER SYSTEM SET shared_pool_size = 1G;

-- 或使用绑定变量减少硬解析
```

#### （2）Buffer Cache 调优

```sql
-- 问题：Buffer Cache 命中率低
-- 解决：增加 Buffer Cache 大小
ALTER SYSTEM SET db_cache_size = 4G;

-- 或使用 KEEP/RECYCLE 池优化
```

### 5.2 PGA 调优

```sql
-- 问题：大量磁盘排序
-- 解决：增加 PGA 大小
ALTER SYSTEM SET pga_aggregate_target = 4G;

-- 或优化 SQL 减少排序
```

### 5.3 内存泄漏检测

```sql
-- 查看内存使用趋势
SELECT 
    snap_id,
    begin_interval_time,
    sgasize/1024/1024 as sga_mb,
    pga_target/1024/1024 as pga_mb
FROM dba_hist_sgastat, dba_hist_snapshot
WHERE snap_id = snap_id
ORDER BY begin_interval_time;
```

## 6. 常见问题诊断

### 6.1 ORA-04031：无法分配共享内存

**原因**：Shared Pool 碎片化或不足。

**解决方案**：
```sql
-- 1. 刷新 Shared Pool
ALTER SYSTEM FLUSH SHARED_POOL;

-- 2. 增加 Shared Pool 大小
ALTER SYSTEM SET shared_pool_size = 2G;

-- 3. 使用绑定变量减少硬解析
```

### 6.2 ORA-04030：进程内存不足

**原因**：PGA 不足或内存泄漏。

**解决方案**：
```sql
-- 1. 增加 PGA 大小
ALTER SYSTEM SET pga_aggregate_target = 4G;

-- 2. 查找内存泄漏的会话
SELECT sid, program, pga_used_mem
FROM v$session s, v$process p
WHERE s.paddr = p.addr
ORDER BY pga_used_mem DESC;

-- 3. 杀死异常会话
ALTER SYSTEM KILL SESSION 'sid,serial#';
```

### 6.3 高 Physical I/O

**原因**：Buffer Cache 命中率低。

**解决方案**：
```sql
-- 1. 增加 Buffer Cache
ALTER SYSTEM SET db_cache_size = 8G;

-- 2. 优化 SQL 减少全表扫描

-- 3. 使用 KEEP 池缓存热点表
ALTER TABLE hot_table STORAGE (BUFFER_POOL KEEP);
```

## 7. 最佳实践

### 7.1 内存配置原则

✅ **OLTP 系统**
- 较大的 Buffer Cache
- 适中的 Shared Pool
- 较小的 PGA

✅ **数据仓库**
- 适中的 Buffer Cache
- 较大的 PGA（排序、哈希）
- 使用并行查询

✅ **混合负载**
- 平衡配置
- 使用 AMM 自动管理

### 7.2 监控建议

📌 **定期检查内存命中率**  
📌 **监控 PGA 使用峰值**  
📌 **分析 AWR 报告中的内存部分**  
📌 **设置内存告警阈值**  
📌 **记录内存配置变更**  

### 7.3 性能指标

| 指标 | 目标值 |
|------|--------|
| Buffer Cache 命中率 | > 90% |
| Library Cache 命中率 | > 90% |
| Dictionary Cache 命中率 | > 95% |
| PGA 缓存命中率 | > 90% |
| 硬解析比例 | < 10% |
