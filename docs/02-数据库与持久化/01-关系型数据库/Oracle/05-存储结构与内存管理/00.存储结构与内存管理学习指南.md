### 01. 表空间管理

**核心内容**：
- 表空间类型（SYSTEM、SYSAUX、UNDO、TEMP、USERS）
- 本地管理 vs 字典管理
- 段空间管理（AUTO vs MANUAL）
- 表空间操作（创建、扩展、收缩、删除）
- 大文件表空间（Bigfile Tablespace）

**关键知识点**：
```sql
-- 创建表空间
CREATE TABLESPACE app_data
DATAFILE '/u01/app/oracle/oradata/ORCL/app_data01.dbf'
SIZE 1G
AUTOEXTEND ON NEXT 100M MAXSIZE 10G
EXTENT MANAGEMENT LOCAL
SEGMENT SPACE MANAGEMENT AUTO;

-- 创建临时表空间
CREATE TEMPORARY TABLESPACE temp_large
TEMPFILE '/u01/app/oracle/oradata/ORCL/temp_large01.dbf'
SIZE 2G
AUTOEXTEND ON NEXT 200M MAXSIZE 10G
EXTENT MANAGEMENT LOCAL UNIFORM SIZE 1M;

-- 创建撤销表空间
CREATE UNDO TABLESPACE undotbs2
DATAFILE '/u01/app/oracle/oradata/ORCL/undotbs2_01.dbf'
SIZE 2G
AUTOEXTEND ON NEXT 100M MAXSIZE 5G;

-- 查看表空间使用率
SELECT 
    tablespace_name,
    ROUND(SUM(bytes)/1024/1024, 2) as total_mb,
    ROUND(SUM(NVL(free_bytes, 0))/1024/1024, 2) as free_mb,
    ROUND((SUM(bytes) - SUM(NVL(free_bytes, 0))) / SUM(bytes) * 100, 2) as used_pct
FROM dba_data_files df
LEFT JOIN (
    SELECT file_id, SUM(bytes) as free_bytes
    FROM dba_free_space
    GROUP BY file_id
) fs ON df.file_id = fs.file_id
GROUP BY tablespace_name;

-- 添加数据文件
ALTER TABLESPACE app_data
ADD DATAFILE '/u01/app/oracle/oradata/ORCL/app_data02.dbf'
SIZE 1G AUTOEXTEND ON NEXT 100M MAXSIZE 10G;

-- 调整数据文件大小
ALTER DATABASE DATAFILE '/u01/app/oracle/oradata/ORCL/app_data01.dbf'
RESIZE 2G;

-- 删除表空间
DROP TABLESPACE old_tablespace INCLUDING CONTENTS AND DATAFILES;
```

### 02. 自动存储管理ASM

**核心内容**：
- ASM 架构和原理
- 磁盘组管理
- ASM 实例配置
- 文件管理和迁移
- ASM 监控和维护

**关键知识点**：
```bash
# 查看 ASM 磁盘组
asmcmd lsdg

# 查看 ASM 磁盘
asmcmd lsdsk

# 创建磁盘组
CREATE DISKGROUP data NORMAL REDUNDANCY
FAILGROUP fg1 DISK '/dev/sdb1', '/dev/sdb2'
FAILGROUP fg2 DISK '/dev/sdc1', '/dev/sdc2';
```

```sql
-- 查看磁盘组信息
SELECT name, state, type, total_mb, free_mb
FROM v$asm_diskgroup;

-- 查看 ASM 文件
SELECT name, bytes/1024/1024 as size_mb, type
FROM v$asm_file;

-- 在 ASM 中创建数据文件
CREATE TABLESPACE asm_ts
DATAFILE '+DATA/ORCL/asm_ts01.dbf'
SIZE 1G AUTOEXTEND ON;
```

### 03. SGA组件详解

**核心内容**：
- Database Buffer Cache 深度解析
- Shared Pool 内部结构
- Redo Log Buffer 工作机制
- Large Pool 用途
- Java Pool 和 Streams Pool

**关键知识点**：
```sql
-- 查看 SGA 各组件大小
SELECT component, current_size/1024/1024 as size_mb
FROM v$sga_dynamic_components
ORDER BY current_size DESC;

-- Buffer Cache 命中率
SELECT 
    1 - (phy.value / NULLIF((cur.value + con.value), 0)) as hit_ratio
FROM v$sysstat cur, v$sysstat con, v$sysstat phy
WHERE cur.name = 'db block gets'
AND con.name = 'consistent gets'
AND phy.name = 'physical reads';

-- Shared Pool 使用情况
SELECT 
    pool,
    name,
    bytes/1024/1024 as mb
FROM v$sgastat
WHERE pool = 'shared pool'
ORDER BY bytes DESC;

-- Library Cache 命中率
SELECT 
    namespace,
    gethitratio,
    pinhitratio
FROM v$librarycache;

-- 调整 Buffer Cache
ALTER SYSTEM SET db_cache_size=2G SCOPE=BOTH;

-- 调整 Shared Pool
ALTER SYSTEM SET shared_pool_size=1G SCOPE=BOTH;
```

### 04. PGA与工作区管理

**核心内容**：
- PGA 组成结构
- 工作区大小管理
- 排序操作优化
- 哈希连接优化
- PGA 溢出处理

**关键知识点**：
```sql
-- 查看 PGA 统计
SELECT * FROM v$pgastat;

-- PGA 顾问建议
SELECT 
    pga_target_for_estimate/1024/1024 as estimate_mb,
    estd_pga_cache_hit_percentage as hit_pct,
    estd_overalloc_count as overalloc
FROM v$pga_target_advice;

-- 查看会话 PGA 使用
SELECT 
    s.sid,
    s.username,
    p.pga_used_mem/1024/1024 as used_mb,
    p.pga_alloc_mem/1024/1024 as alloc_mb
FROM v$session s
JOIN v$process p ON s.paddr = p.addr
ORDER BY p.pga_used_mem DESC;

-- 调整 PGA
ALTER SYSTEM SET pga_aggregate_target=4G SCOPE=BOTH;

-- 监控临时表空间使用（PGA 溢出）
SELECT 
    tablespace_name,
    bytes_used/1024/1024 as used_mb,
    bytes_free/1024/1024 as free_mb
FROM v$temp_space_header;
```

### 05. 内存参数调优

**核心内容**：
- AMM（自动内存管理）
- ASMM（自动共享内存管理）
- 手动内存管理
- 内存参数调优策略
- 内存诊断和监控

**关键知识点**：
```sql
-- AMM 配置（Oracle 11g+）
ALTER SYSTEM SET memory_target=6G SCOPE=BOTH;
ALTER SYSTEM SET memory_max_target=8G SCOPE=SPFILE;

-- ASMM 配置
ALTER SYSTEM SET sga_target=4G SCOPE=BOTH;
ALTER SYSTEM SET pga_aggregate_target=2G SCOPE=BOTH;

-- 手动管理
ALTER SYSTEM SET db_cache_size=2G SCOPE=BOTH;
ALTER SYSTEM SET shared_pool_size=1G SCOPE=BOTH;
ALTER SYSTEM SET large_pool_size=256M SCOPE=BOTH;
ALTER SYSTEM SET java_pool_size=128M SCOPE=BOTH;
ALTER SYSTEM SET pga_aggregate_target=2G SCOPE=BOTH;

-- SGA 顾问
SELECT 
    sga_size/1024/1024 as sga_mb,
    estd_db_time,
    estd_physical_reads
FROM v$sga_target_advice;

-- 内存使用监控
SELECT 
    component,
    current_size/1024/1024 as current_mb,
    min_size/1024/1024 as min_mb,
    max_size/1024/1024 as max_mb
FROM v$sga_dynamic_components;
```

### 06. 大页内存HugePages

**核心内容**：
- HugePages 原理和优势
- Linux HugePages 配置
- Oracle HugePages 配置
- Transparent HugePages（THP）
- 性能测试和验证

**关键知识点**：
```bash
# 查看系统 HugePages 配置
grep HugePages /proc/meminfo

# 计算所需 HugePages 数量
# HugePages = SGA大小 / HugePages大小
# 例如：SGA=4GB, HugePages=2MB → 需要 2048 页

# 配置 /etc/sysctl.conf
vm.nr_hugepages = 2048
vm.hugetlb_shm_group = 54321  # oracle 用户组 ID

# 应用配置
sysctl -p

# 禁用 Transparent HugePages（推荐）
echo never > /sys/kernel/mm/transparent_hugepage/enabled
echo never > /sys/kernel/mm/transparent_hugepage/defrag
```

```sql
-- 检查 Oracle 是否使用 HugePages
SHOW PARAMETER use_large_pages;

-- 启用 HugePages
ALTER SYSTEM SET use_large_pages=ONLY SCOPE=SPFILE;
```

**性能提升**：
- 减少 TLB miss
- 降低页表开销
- 提升内存访问效率 10-30%

## 🔍 实战案例预告

### 案例 1：表空间扩容
```sql
-- 监控表空间使用率并自动扩容
DECLARE
    v_used_pct NUMBER;
BEGIN
    SELECT ROUND((SUM(bytes) - SUM(NVL(fs.bytes, 0))) / SUM(bytes) * 100, 2)
    INTO v_used_pct
    FROM dba_data_files df
    LEFT JOIN dba_free_space fs ON df.file_id = fs.file_id
    WHERE df.tablespace_name = 'APP_DATA';
    
    IF v_used_pct > 85 THEN
        EXECUTE IMMEDIATE 'ALTER TABLESPACE app_data ADD DATAFILE ...';
    END IF;
END;
```

### 案例 2：内存调优
```sql
-- 根据 AWR 建议调整内存
SELECT * FROM v$memory_target_advice ORDER BY memory_size;
SELECT * FROM v$sga_target_advice ORDER BY sga_size;
SELECT * FROM v$pga_target_advice ORDER BY pga_target_for_estimate;
```
