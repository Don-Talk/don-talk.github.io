## 1. Oracle 存储层次结构

### 1.1 存储层级

```
数据库 (Database)
    ↓
表空间 (Tablespace)
    ↓
段 (Segment)
    ↓
区 (Extent)
    ↓
数据块 (Data Block)
    ↓
操作系统块 (OS Block)
```

### 1.2 各层级说明

| 层级 | 说明 | 大小 |
|------|------|------|
| **数据库** | 物理文件的集合 | - |
| **表空间** | 逻辑存储单元 | 多个数据文件 |
| **段** | 表、索引等对象 | 多个区 |
| **区** | 连续的数据块集合 | 多个数据块 |
| **数据块** | 最小 I/O 单位 | 2K-32K |
| **OS 块** | 操作系统块 | 通常 512B-4K |

## 2. 表空间类型

### 2.1 SYSTEM 表空间

- 存储数据字典
- 存储 SYS 用户的对象
- 必须 ONLINE
- 不能删除或重命名

```sql
-- 查看 SYSTEM 表空间
SELECT tablespace_name, status, contents
FROM dba_tablespaces
WHERE tablespace_name = 'SYSTEM';
```

### 2.2 SYSAUX 表空间

- SYSTEM 的辅助表空间
- 存储 AWR、Statspack 等组件数据
- Oracle 10g+ 引入

```sql
-- 查看 SYSAUX 使用情况
SELECT occupant_name, space_usage_kbytes/1024 as usage_mb
FROM v$sysaux_occupants
ORDER BY space_usage_kbytes DESC;
```

### 2.3 UNDO 表空间

- 存储回滚信息
- 支持事务回滚
- 支持读一致性
- 支持闪回查询

```sql
-- 查看 UNDO 表空间
SELECT tablespace_name, status, contents
FROM dba_tablespaces
WHERE contents = 'UNDO';

-- 查看 UNDO 使用情况
SELECT 
    tablespace_name,
    ROUND(used_ublk * 8192 / 1024 / 1024, 2) as used_mb,
    ROUND(total_ublk * 8192 / 1024 / 1024, 2) as total_mb
FROM v$undostat;
```

### 2.4 TEMP 表空间

- 存储临时数据
- 排序操作
- 哈希连接
- 全局临时表

```sql
-- 查看临时表空间
SELECT tablespace_name, status, contents
FROM dba_tablespaces
WHERE contents = 'TEMPORARY';

-- 查看临时表空间使用
SELECT 
    tablespace_name,
    ROUND(bytes_used / 1024 / 1024, 2) as used_mb,
    ROUND(bytes_free / 1024 / 1024, 2) as free_mb
FROM v$temp_space_header;
```

### 2.5 用户自定义表空间

- 存储用户数据
- 可以自定义属性
- 推荐按业务划分

## 3. 表空间管理方式

### 3.1 本地管理表空间（LMT）

**推荐方式**，Oracle 9i+ 默认。

**特点**：
- 使用位图管理空闲空间
- 自动管理区的分配
- 性能更好
- 减少碎片

```sql
-- 创建本地管理表空间
CREATE TABLESPACE users_lmt
DATAFILE '/u01/app/oracle/oradata/ORCL/users01.dbf'
SIZE 100M
AUTOEXTEND ON NEXT 10M MAXSIZE 1G
EXTENT MANAGEMENT LOCAL
UNIFORM SIZE 1M;
```

### 3.2 字典管理表空间（DMT）

**已过时**，不推荐使用。

**特点**：
- 使用数据字典管理
- 需要频繁更新数据字典
- 性能较差
- Oracle 早期版本使用

```sql
-- 创建字典管理表空间（不推荐）
CREATE TABLESPACE users_dmt
DATAFILE '/u01/app/oracle/oradata/ORCL/users_dmt01.dbf'
SIZE 100M
EXTENT MANAGEMENT DICTIONARY;
```

## 4. 创建表空间

### 4.1 基本语法

```sql
CREATE TABLESPACE tablespace_name
DATAFILE 'file_path'
SIZE size
[AUTOEXTEND ON|OFF]
[NEXT increment_size]
[MAXSIZE max_size|UNLIMITED]
[EXTENT MANAGEMENT LOCAL|DICTIONARY]
[SEGMENT SPACE MANAGEMENT AUTO|MANUAL]
[BLOCKSIZE block_size];
```

### 4.2 创建永久表空间

```sql
-- 示例 1：基本表空间
CREATE TABLESPACE app_data
DATAFILE '/u01/app/oracle/oradata/ORCL/app_data01.dbf'
SIZE 500M
AUTOEXTEND ON NEXT 50M MAXSIZE 10G
EXTENT MANAGEMENT LOCAL
SEGMENT SPACE MANAGEMENT AUTO;

-- 示例 2：多数据文件
CREATE TABLESPACE app_index
DATAFILE 
    '/u01/app/oracle/oradata/ORCL/app_idx01.dbf' SIZE 200M,
    '/u02/app/oracle/oradata/ORCL/app_idx02.dbf' SIZE 200M
AUTOEXTEND ON NEXT 20M MAXSIZE 5G;

-- 示例 3：指定块大小
CREATE TABLESPACE app_large_block
DATAFILE '/u01/app/oracle/oradata/ORCL/app_large01.dbf'
SIZE 100M
BLOCKSIZE 16K;
```

### 4.3 创建临时表空间

```sql
-- 创建临时表空间
CREATE TEMPORARY TABLESPACE temp_app
TEMPFILE '/u01/app/oracle/oradata/ORCL/temp_app01.dbf'
SIZE 100M
AUTOEXTEND ON NEXT 10M MAXSIZE 2G
EXTENT MANAGEMENT LOCAL UNIFORM SIZE 1M;

-- 设置为默认临时表空间
ALTER DATABASE DEFAULT TEMPORARY TABLESPACE temp_app;
```

### 4.4 创建 UNDO 表空间

```sql
-- 创建 UNDO 表空间
CREATE UNDO TABLESPACE undo_app
DATAFILE '/u01/app/oracle/oradata/ORCL/undo_app01.dbf'
SIZE 500M
AUTOEXTEND ON NEXT 50M MAXSIZE 10G;

-- 切换 UNDO 表空间
ALTER SYSTEM SET UNDO_TABLESPACE = undo_app;
```

## 5. 大文件表空间（Bigfile Tablespace）

### 5.1 什么是大文件表空间？

- 只包含一个数据文件
- 最大可达 128TB（取决于块大小）
- 简化管理
- 适合大型数据库

### 5.2 创建大文件表空间

```sql
-- 创建大文件表空间
CREATE BIGFILE TABLESPACE big_data
DATAFILE '/u01/app/oracle/oradata/ORCL/big_data01.dbf'
SIZE 10G
AUTOEXTEND ON NEXT 1G MAXSIZE UNLIMITED;

-- 调整大小
ALTER TABLESPACE big_data RESIZE 20G;

-- 启用自动扩展
ALTER DATABASE DATAFILE '/u01/app/oracle/oradata/ORCL/big_data01.dbf'
AUTOEXTEND ON NEXT 1G MAXSIZE UNLIMITED;
```

### 5.3 适用场景

✅ **数据仓库**  
✅ **VLDB（超大型数据库）**  
✅ **简化文件管理**  
❌ **不适合小数据库**

## 6. 表空间维护

### 6.1 添加数据文件

```sql
-- 添加数据文件到表空间
ALTER TABLESPACE app_data
ADD DATAFILE '/u01/app/oracle/oradata/ORCL/app_data02.dbf'
SIZE 500M
AUTOEXTEND ON NEXT 50M MAXSIZE 10G;
```

### 6.2 扩展数据文件

```sql
-- 手动扩展
ALTER DATABASE DATAFILE '/u01/app/oracle/oradata/ORCL/app_data01.dbf'
RESIZE 1G;

-- 启用自动扩展
ALTER DATABASE DATAFILE '/u01/app/oracle/oradata/ORCL/app_data01.dbf'
AUTOEXTEND ON NEXT 50M MAXSIZE 10G;
```

### 6.3 重命名数据文件

```sql
-- 1. 离线表空间
ALTER TABLESPACE app_data OFFLINE NORMAL;

-- 2. 操作系统层面移动文件
-- mv /old/path/file.dbf /new/path/file.dbf

-- 3. 重命名数据库中的文件
ALTER DATABASE RENAME FILE
'/old/path/file.dbf' TO '/new/path/file.dbf';

-- 4. 在线表空间
ALTER TABLESPACE app_data ONLINE;
```

### 6.4 删除表空间

```sql
-- 删除空表空间
DROP TABLESPACE app_data;

-- 删除表空间及内容
DROP TABLESPACE app_data INCLUDING CONTENTS;

-- 删除表空间、内容及数据文件
DROP TABLESPACE app_data INCLUDING CONTENTS AND DATAFILES;

-- 级联删除约束
DROP TABLESPACE app_data INCLUDING CONTENTS AND DATAFILES CASCADE CONSTRAINTS;
```

### 6.5 表空间脱机/联机

```sql
-- 脱机
ALTER TABLESPACE app_data OFFLINE NORMAL;

-- 联机
ALTER TABLESPACE app_data ONLINE;

-- 紧急脱机（可能有未提交事务）
ALTER TABLESPACE app_data OFFLINE IMMEDIATE;
```

## 7. 监控表空间

### 7.1 查看表空间使用情况

```sql
-- 表空间使用率
SELECT 
    df.tablespace_name,
    ROUND(df.total_bytes / 1024 / 1024, 2) as total_mb,
    ROUND((df.total_bytes - fs.free_bytes) / 1024 / 1024, 2) as used_mb,
    ROUND(fs.free_bytes / 1024 / 1024, 2) as free_mb,
    ROUND((df.total_bytes - fs.free_bytes) / df.total_bytes * 100, 2) as used_pct
FROM 
    (SELECT tablespace_name, SUM(bytes) as total_bytes
     FROM dba_data_files
     GROUP BY tablespace_name) df,
    (SELECT tablespace_name, SUM(bytes) as free_bytes
     FROM dba_free_space
     GROUP BY tablespace_name) fs
WHERE df.tablespace_name = fs.tablespace_name
ORDER BY used_pct DESC;
```

### 7.2 查看数据文件信息

```sql
SELECT 
    file_name,
    tablespace_name,
    bytes/1024/1024 as size_mb,
    autoextensible,
    maxbytes/1024/1024 as max_mb,
    increment_by * 8192 / 1024 / 1024 as next_mb
FROM dba_data_files
ORDER BY tablespace_name, file_name;
```

### 7.3 查找高水位表空间

```sql
-- 使用率超过 85% 的表空间
SELECT 
    tablespace_name,
    ROUND(used_percent, 2) as used_pct
FROM dba_tablespace_usage_metrics
WHERE used_percent > 85
ORDER BY used_percent DESC;
```

## 8. 表空间迁移

### 8.1 传输表空间（Transportable Tablespace）

快速移动表空间到另一个数据库。

```sql
-- 源数据库
-- 1. 设置为只读
ALTER TABLESPACE app_data READ ONLY;

-- 2. 导出元数据
EXPDP system/password DIRECTORY=dp_dir 
TRANSPORT_TABLESPACES=app_data 
DUMPFILE=app_data.dmp;

-- 3. 复制数据文件到目标数据库

-- 目标数据库
-- 4. 导入元数据
IMPDP system/password DIRECTORY=dp_dir 
TRANSPORT_DATAFILES='/path/to/app_data01.dbf'
DUMPFILE=app_data.dmp;

-- 5. 设置回读写
ALTER TABLESPACE app_data READ WRITE;
```

### 8.2 跨平台传输

```sql
-- 检查平台兼容性
SELECT * FROM V$TRANSPORTABLE_PLATFORM;

-- 转换字节序（如需要）
CONVERT TABLESPACE app_data
TO PLATFORM='Linux x86 64-bit'
FORMAT='/path/to/converted_file.dbf';
```

## 9. 最佳实践

### 9.1 表空间设计原则

✅ **按业务划分表空间**
```sql
-- 不同业务模块使用不同表空间
CREATE TABLESPACE sales_data ...;
CREATE TABLESPACE hr_data ...;
CREATE TABLESPACE inventory_data ...;
```

✅ **分离表和索引**
```sql
CREATE TABLESPACE app_data ...;   -- 表数据
CREATE TABLESPACE app_index ...;  -- 索引
```

✅ **使用 LMT + ASSM**
```sql
EXTENT MANAGEMENT LOCAL
SEGMENT SPACE MANAGEMENT AUTO
```

✅ **合理设置 AUTOEXTEND**
```sql
AUTOEXTEND ON NEXT 50M MAXSIZE 10G
```

### 9.2 监控建议

📌 **定期检查表空间使用率**  
📌 **设置告警阈值（85%、90%、95%）**  
📌 **监控 UNDO 表空间使用**  
📌 **检查临时表空间峰值使用**  
📌 **定期清理过期数据**  

### 9.3 常见问题处理

**问题 1：表空间已满**
```sql
-- 解决方案：添加数据文件或扩展
ALTER TABLESPACE app_data ADD DATAFILE '...' SIZE 500M;
```

**问题 2：UNDO 表空间不足**
```sql
-- 增加 UNDO 保留时间
ALTER SYSTEM SET UNDO_RETENTION = 1800;

-- 或增加 UNDO 表空间大小
ALTER DATABASE DATAFILE 'undo_file.dbf' RESIZE 2G;
```

**问题 3：临时表空间不足**
```sql
-- 增加临时文件
ALTER TABLESPACE temp ADD TEMPFILE '...' SIZE 1G;
```

## 10. 总结

### 10.1 核心知识点

✅ **存储层次**：数据库 → 表空间 → 段 → 区 → 块  
✅ **表空间类型**：SYSTEM、SYSAUX、UNDO、TEMP、用户  
✅ **管理方式**：LMT（推荐）、DMT（过时）  
✅ **大文件表空间**：简化管理，适合 VLDB  
✅ **维护操作**：添加、扩展、删除、迁移  

### 10.2 关键命令

📌 `CREATE TABLESPACE` - 创建表空间  
📌 `ALTER TABLESPACE` - 修改表空间  
📌 `DROP TABLESPACE` - 删除表空间  
📌 `ALTER DATABASE DATAFILE` - 管理数据文件
