## 1. HugePages 概述

### 1.1 什么是 HugePages？

HugePages（大页内存）是使用更大页面尺寸的内存管理机制。

**标准页 vs 大页**：
- 标准页：4 KB
- 大页：2 MB 或 1 GB（x86_64）

<!-- TODO: 插入图片：标准页与大页对比图 -->

### 1.2 为什么需要 HugePages？

#### （1）减少页表大小

```
假设 SGA = 100 GB

标准页（4KB）：
- 页数 = 100GB / 4KB = 26,214,400 页
- 页表项占用大量内存

大页（2MB）：
- 页数 = 100GB / 2MB = 51,200 页
- 页表显著减小
```

#### （2）减少 TLB Miss

- TLB（Translation Lookaside Buffer）缓存页表项
- 大页减少 TLB 压力
- 提高地址转换效率

#### （3）避免 Swap

- HugePages 不会被 swap
- 保证关键内存常驻物理内存

### 1.3 HugePages 的优势

✅ **性能提升**：减少页表查找开销  
✅ **降低 CPU 使用**：减少 TLB miss  
✅ **提高可扩展性**：支持更大的 SGA  
✅ **避免 Swap**：关键内存不交换

## 2. HugePages 配置

### 2.1 检查系统支持

```bash
# 查看支持的页大小
grep -i huge /proc/meminfo

# 输出示例：
# AnonHugePages:         0 kB
# ShmemHugePages:        0 kB
# HugePages_Total:       0
# HugePages_Free:        0
# HugePages_Rsvd:        0
# HugePages_Surp:        0
# Hugepagesize:       2048 kB
```

### 2.2 计算所需大页数量

```bash
#!/bin/bash
# 计算 HugePages 数量

# 获取 SGA 大小（字节）
SGA_BYTES=$(sqlplus -s / as sysdba << EOF
SET PAGESIZE 0 FEEDBACK OFF VERIFY OFF
SELECT SUM(value) FROM v\$sga;
EXIT;
EOF
)

# 转换为 MB
SGA_MB=$((SGA_BYTES / 1024 / 1024))

# 大页大小（MB）
HUGE_PAGE_SIZE=2

# 计算所需大页数（增加 10% 余量）
NUM_HUGEPAGES=$(( (SGA_MB / HUGE_PAGE_SIZE) * 110 / 100 ))

echo "SGA Size: ${SGA_MB} MB"
echo "Recommended HugePages: ${NUM_HUGEPAGES}"
```

### 2.3 配置 HugePages

#### （1）修改 sysctl.conf

```bash
# 编辑 /etc/sysctl.conf
vi /etc/sysctl.conf

# 添加以下行
vm.nr_hugepages = 51200
vm.hugetlb_shm_group = 54321  # oracle 用户的组 ID

# 应用配置
sysctl -p
```

#### （2）设置用户限制

```bash
# 编辑 /etc/security/limits.conf
vi /etc/security/limits.conf

# 添加以下行
oracle soft memlock unlimited
oracle hard memlock unlimited
```

#### （3）禁用 Transparent HugePages

```bash
# 编辑 /etc/default/grub
GRUB_CMDLINE_LINUX="... transparent_hugepage=never"

# 更新 GRUB
grub2-mkconfig -o /boot/grub2/grub.cfg

# 重启系统
reboot
```

或者运行时禁用：
```bash
echo never > /sys/kernel/mm/transparent_hugepage/enabled
echo never > /sys/kernel/mm/transparent_hugepage/defrag
```

### 2.4 配置 Oracle 使用 HugePages

```sql
-- 禁用 AMM（必须）
ALTER SYSTEM SET memory_target = 0 SCOPE=SPFILE;
ALTER SYSTEM SET memory_max_target = 0 SCOPE=SPFILE;

-- 设置 SGA 目标
ALTER SYSTEM SET sga_target = 10G SCOPE=SPFILE;
ALTER SYSTEM SET sga_max_size = 10G SCOPE=SPFILE;

-- 启用 HugePages
ALTER SYSTEM SET use_large_pages = ONLY SCOPE=SPFILE;
-- 选项：TRUE、FALSE、ONLY

-- 重启实例
SHUTDOWN IMMEDIATE;
STARTUP;
```

## 3. 验证 HugePages

### 3.1 检查大页使用情况

```bash
# 查看大页统计
grep -i huge /proc/meminfo

# 输出示例：
# HugePages_Total:     51200
# HugePages_Free:      1024
# HugePages_Rsvd:       512
# HugePages_Surp:         0
# Hugepagesize:        2048 kB
```

### 3.2 检查 Oracle 是否使用大页

```bash
# 查看 Oracle 进程的内存映射
cat /proc/<pid>/smaps | grep -i huge

# 或使用脚本
#!/bin/bash
for pid in $(pgrep -u oracle ora_); do
    echo "PID: $pid"
    grep -i huge /proc/$pid/smaps | head -5
done
```

### 3.3 查看数据库参数

```sql
-- 检查参数
SHOW PARAMETER use_large_pages;
SHOW PARAMETER sga_target;
SHOW PARAMETER memory_target;

-- 确认 AMM 已禁用
SELECT name, value 
FROM v$parameter 
WHERE name IN ('memory_target', 'sga_target', 'use_large_pages');
```

## 4. Transparent HugePages（THP）

### 4.1 什么是 THP？

THP 是 Linux 内核自动管理的大页机制，无需手动配置。

**问题**：
- Oracle 不推荐使用 THP
- 可能导致性能问题
- 可能引起内存分配延迟

### 4.2 禁用 THP

```bash
# 方法 1：GRUB 配置（推荐）
GRUB_CMDLINE_LINUX="... transparent_hugepage=never"

# 方法 2：rc.local
echo never > /sys/kernel/mm/transparent_hugepage/enabled
echo never > /sys/kernel/mm/transparent_hugepage/defrag

# 验证
cat /sys/kernel/mm/transparent_hugepage/enabled
# 应该显示：[never] madvise always
```

### 4.3 Oracle 建议

⚠️ **Oracle 官方建议**：
- 禁用 THP
- 使用标准 HugePages
- 特别是 RAC 环境

## 5. HugePages 故障排查

### 5.1 常见问题

#### 问题 1：无法启动实例

**错误**：ORA-04031 或内存不足

**原因**：大页数量不足

**解决**：
```bash
# 增加大页数量
sysctl -w vm.nr_hugepages=60000

# 或在 /etc/sysctl.conf 中修改后执行 sysctl -p
```

#### 问题 2：大页未使用

**检查**：
```bash
# 查看是否有进程使用大页
grep -i huge /proc/*/smaps

# 如果没有输出，说明未使用
```

**解决**：
```sql
-- 确认参数设置
SHOW PARAMETER use_large_pages;

-- 如果是 TRUE，改为 ONLY
ALTER SYSTEM SET use_large_pages = ONLY SCOPE=SPFILE;

-- 重启实例
```

#### 问题 3：THP 未禁用

**检查**：
```bash
cat /sys/kernel/mm/transparent_hugepage/enabled
# 如果显示 [always]，说明 THP 启用
```

**解决**：按照前面步骤禁用 THP

### 5.2 诊断脚本

```bash
#!/bin/bash
# HugePages 诊断脚本

echo "=== System HugePages Info ==="
grep -i huge /proc/meminfo

echo ""
echo "=== THP Status ==="
cat /sys/kernel/mm/transparent_hugepage/enabled

echo ""
echo "=== Oracle Parameters ==="
sqlplus -s / as sysdba << EOF
SET PAGESIZE 0 FEEDBACK OFF
SHOW PARAMETER use_large_pages;
SHOW PARAMETER sga_target;
SHOW PARAMETER memory_target;
EXIT;
EOF

echo ""
echo "=== Oracle Process HugePages Usage ==="
for pid in $(pgrep -u oracle ora_pmon); do
    echo "PMON PID: $pid"
    grep Huge /proc/$pid/status
done
```

## 6. RAC 环境的 HugePages

### 6.1 RAC 特殊考虑

- 每个节点独立配置
- 确保所有节点配置一致
- 考虑共享内存段

### 6.2 RAC 配置步骤

```bash
# 在所有节点执行

# 1. 计算大页数量（基于单实例 SGA）
# 2. 配置 sysctl.conf
# 3. 配置 limits.conf
# 4. 禁用 THP
# 5. 重启所有节点
```

### 6.3 验证 RAC HugePages

```bash
# 在每个节点检查
grep -i huge /proc/meminfo

# 确保所有节点的大页都被使用
crsctl stat res -t
```

## 7. 最佳实践

### 7.1 配置原则

✅ **始终禁用 THP**  
✅ **使用 use_large_pages = ONLY**  
✅ **预留 10% 余量**  
✅ **定期监控使用情况**  
✅ **RAC 环境所有节点一致**  

### 7.2 性能优化

📌 **OLTP 系统**：强烈推荐使用 HugePages  
📌 **数据仓库**：根据 SGA 大小决定  
📌 **小内存系统**（< 8GB）：可能不需要  
📌 **大内存系统**（> 32GB）：必须使用  

### 7.3 监控建议

```bash
# 添加到 cron，定期检查
0 * * * * /opt/scripts/check_hugepages.sh >> /var/log/hugepages.log

# 告警阈值
# - HugePages_Free < 10%：警告
# - HugePages_Free = 0：严重
```
