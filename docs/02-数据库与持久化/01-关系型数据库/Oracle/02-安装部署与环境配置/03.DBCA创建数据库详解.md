## 1. DBCA 简介

DBCA（Database Configuration Assistant）是 Oracle 提供的数据库创建和管理工具。

**功能**：
- 创建新数据库
- 配置数据库选项
- 克隆现有数据库
- 删除数据库
- 管理模板

**启动方式**：
```bash
# Linux
dbca

# Windows
dbca.bat
```

## 2. 图形化创建数据库

### 2.1 启动 DBCA

```bash
# 切换到 oracle 用户
su - oracle

# 启动 DBCA
dbca
```

### 2.2 创建步骤

#### 步骤 1：操作类型
- 选择 "Create a database"
- 点击 Next

#### 步骤 2：创建模式
- **Advanced mode**：高级配置（推荐）
- **Typical mode**：典型配置
- 点击 Next

#### 步骤 3：数据库模板
- **General Purpose or Transaction Processing**：通用/OLTP
- **Data Warehouse**：数据仓库
- **Custom Database**：自定义
- 点击 Next

#### 步骤 4：数据库标识
- **Global database name**: `ORCL`
- **SID**: `ORCL`
- 勾选 "Create as Container database"（多租户，可选）
- 点击 Next

#### 步骤 5：存储选项
- **Storage type**: File System / ASM
- **Use Oracle Managed Files (OMF)**: 推荐
- **Database file location**: `/u01/app/oracle/oradata`
- 点击 Next

#### 步骤 6：快速恢复选项
- 勾选 "Specify Fast Recovery Area"
- **Fast Recovery Area**: `/u01/app/oracle/fast_recovery_area`
- **Fast Recovery Area Size**: 10 GB
- 勾选 "Enable archiving"
- 点击 Next

#### 步骤 7：监听器配置
- 选择已存在的监听器
- 或创建新监听器
- 点击 Next

#### 步骤 8：Vault 配置（可选）
- 配置 Database Vault（安全选项）
- 通常跳过
- 点击 Next

#### 步骤 9：配置选项
- **Memory**: 
  - 使用 AMM
  - SGA Target: 2 GB
  - PGA Target: 1 GB
- **Processes**: 300
- **Character sets**: 
  - Use Unicode (AL32UTF8) - 推荐
- **Connection mode**: Dedicated server
- 点击 Next

#### 步骤 10：管理选项
- 配置 Enterprise Manager
- 注册到 Cloud Control（可选）
- 点击 Next

#### 步骤 11：用户口令
- 为所有账户使用相同口令
- 或为不同账户设置不同口令
- SYS、SYSTEM、DBSNMP、SYSMAN
- 点击 Next

#### 步骤 12：创建选项
- **Create database**: 立即创建
- **Save as a Database Template**: 保存为模板
- **Generate Database Creation Scripts**: 生成脚本
- 点击 Finish

#### 步骤 13：摘要
- 确认所有配置
- 点击 OK 开始创建

#### 步骤 14：进度监控
- 等待创建完成（约 15-30 分钟）
- 查看日志了解详细信息

#### 步骤 15：完成
- 记录密码和连接信息
- 点击 Close

## 3. 静默创建数据库

### 3.1 使用响应文件

创建 `dbca_response.rsp`：

```ini
[GENERAL]
RESPONSEFILE_VERSION = "19.0"
OPERATION_TYPE = "createDatabase"

[CREATEDATABASE]
GDBNAME = "ORCL"
SID = "ORCL"
TEMPLATENAME = "General_Purpose.dbc"
SYSPASSWORD = "Oracle123"
SYSTEMPASSWORD = "Oracle123"
EMCONFIGURATION = "NONE"
STORAGETYPE = "FS"
DATAFILEDESTINATION = "/u01/app/oracle/oradata"
RECOVERYAREADESTINATION = "/u01/app/oracle/fast_recovery_area"
CHARACTERSET = "AL32UTF8"
TOTALMEMORY = "4096"
DATABASETYPE = "OLTP"
AUTOMATICMEMORYMANAGEMENT = "TRUE"
ENABLEARCHIVE = "TRUE"
```

### 3.2 执行静默创建

```bash
dbca -silent \
  -responseFile /path/to/dbca_response.rsp
```

### 3.3 命令行参数方式

```bash
dbca -silent \
  -createDatabase \
  -templateName General_Purpose.dbc \
  -gdbname ORCL \
  -sid ORCL \
  -characterSet AL32UTF8 \
  -memoryPercentage 40 \
  -databaseType OLTP \
  -emConfiguration NONE \
  -datafileDestination /u01/app/oracle/oradata \
  -recoveryAreaDestination /u01/app/oracle/fast_recovery_area \
  -storageType FS \
  -automaticMemoryManagement true \
  -totalMemory 4096 \
  -sampleSchema false \
  -enableArchive true \
  -sysPassword Oracle123 \
  -systemPassword Oracle123
```

## 4. 数据库模板

### 4.1 内置模板

| 模板 | 用途 | 特点 |
|------|------|------|
| **General_Purpose.dbc** | 通用/OLTP | 平衡配置 |
| **Data_Warehouse.dbc** | 数据仓库 | 优化分析查询 |
| **New_Database.dbt** | 空数据库 | 最小配置 |

### 4.2 查看模板

```bash
# 模板位置
ls $ORACLE_HOME/assistants/dbca/templates/

# 查看模板内容
cat $ORACLE_HOME/assistants/dbca/templates/General_Purpose.dbc
```

### 4.3 创建自定义模板

```bash
# 从现有数据库创建模板
dbca -silent \
  -createTemplateFromDB \
  -sourceDB ORCL \
  -templateName MyTemplate \
  -maintainFileLocations false \
  -sysDBAUserName sys \
  -sysDBAPassword Oracle123
```

### 4.4 使用自定义模板

```bash
dbca -silent \
  -createDatabase \
  -templateName MyTemplate.dbc \
  -gdbname NEWDB \
  -sid NEWDB
```

## 5. 多租户数据库（CDB/PDB）

### 5.1 创建 CDB

```bash
dbca -silent \
  -createDatabase \
  -templateName General_Purpose.dbc \
  -gdbname CDB1 \
  -sid CDB1 \
  -createAsContainerDatabase true \
  -numberOfPDBs 1 \
  -pdbName PDB1 \
  -characterSet AL32UTF8 \
  -memoryPercentage 40 \
  -sysPassword Oracle123 \
  -systemPassword Oracle123
```

### 5.2 创建 PDB

```sql
-- 连接到 CDB
sqlplus / as sysdba

-- 创建新的 PDB
CREATE PLUGGABLE DATABASE pdb2 
ADMIN USER pdb_admin IDENTIFIED BY password
FILE_NAME_CONVERT = ('/u01/app/oracle/oradata/CDB1/pdbseed/',
                     '/u01/app/oracle/oradata/CDB1/pdb2/');

-- 打开 PDB
ALTER PLUGGABLE DATABASE pdb2 OPEN;
```

### 5.3 管理 PDB

```sql
-- 查看所有 PDB
SELECT name, open_mode FROM v$pdbs;

-- 切换容器
ALTER SESSION SET CONTAINER = pdb2;

-- 关闭 PDB
ALTER PLUGGABLE DATABASE pdb2 CLOSE IMMEDIATE;

-- 删除 PDB
DROP PLUGGABLE DATABASE pdb2 INCLUDING DATAFILES;
```

## 6. 克隆数据库

### 6.1 使用 DBCA 克隆

```bash
dbca -silent \
  -createDatabase \
  -templateName existing_db_template.dbc \
  -gdbname CLONEDB \
  -sid CLONEDB \
  -datafileDestination /u01/app/oracle/oradata/CLONEDB
```

### 6.2 RMAN 克隆

```bash
# 使用 RMAN 进行活跃数据库克隆
rman TARGET sys/password@SOURCEDB AUXILIARY sys/password@CLONEDB

RMAN> DUPLICATE TARGET DATABASE TO CLONEDB FROM ACTIVE DATABASE;
```

## 7. 删除数据库

### 7.1 使用 DBCA 删除

```bash
# 图形化
dbca

# 选择 "Delete a database"
# 选择要删除的数据库
# 确认删除
```

### 7.2 静默删除

```bash
dbca -silent \
  -deleteDatabase \
  -sourceDB ORCL \
  -sysDBAUserName sys \
  -sysDBAPassword Oracle123
```

### 7.3 手动删除

```sql
-- 1. 关闭数据库
SHUTDOWN IMMEDIATE;

-- 2. 启动到 MOUNT 独占模式
STARTUP MOUNT EXCLUSIVE RESTRICT;

-- 3. 删除数据库
DROP DATABASE;
```

```bash
# 4. 手动清理文件（谨慎操作）
rm -rf /u01/app/oracle/oradata/ORCL
rm -rf /u01/app/oracle/fast_recovery_area/ORCL
```

## 8. 数据库配置修改

### 8.1 修改字符集

```sql
-- 查看当前字符集
SELECT parameter, value 
FROM nls_database_parameters 
WHERE parameter LIKE '%CHARACTERSET%';

-- 修改字符集（需要重启）
-- 注意：只能从子集改到超集
SHUTDOWN IMMEDIATE;
STARTUP MOUNT;
ALTER SYSTEM ENABLE RESTRICTED SESSION;
ALTER DATABASE CHARACTER SET AL32UTF8;
ALTER DATABASE NATIONAL CHARACTER SET AL16UTF16;
SHUTDOWN IMMEDIATE;
STARTUP;
```

### 8.2 调整内存

```sql
-- 查看当前内存配置
SHOW PARAMETER memory_target;
SHOW PARAMETER sga_target;
SHOW PARAMETER pga_aggregate_target;

-- 修改内存（AMM）
ALTER SYSTEM SET memory_target=6G SCOPE=BOTH;

-- 修改内存（ASMM）
ALTER SYSTEM SET sga_target=4G SCOPE=BOTH;
ALTER SYSTEM SET pga_aggregate_target=2G SCOPE=BOTH;
```

### 8.3 启用归档

```sql
-- 查看归档状态
ARCHIVE LOG LIST;

-- 启用归档（需要重启）
SHUTDOWN IMMEDIATE;
STARTUP MOUNT;
ALTER DATABASE ARCHIVELOG;
ALTER DATABASE OPEN;

-- 验证
ARCHIVE LOG LIST;
```

## 9. 验证数据库

### 9.1 基本检查

```sql
-- 实例状态
SELECT instance_name, status, version FROM v$instance;

-- 数据库状态
SELECT name, open_mode, log_mode FROM v$database;

-- 控制文件
SELECT name FROM v$controlfile;

-- 数据文件
SELECT file_name, tablespace_name, bytes/1024/1024 as size_mb
FROM dba_data_files;

-- 日志文件
SELECT group#, member, bytes/1024/1024 as size_mb
FROM v$logfile;
```

### 9.2 性能检查

```sql
-- SGA 使用
SELECT * FROM v$sga;

-- PGA 使用
SELECT * FROM v$pgastat;

-- 会话数
SELECT COUNT(*) FROM v$session;

-- 表空间使用
SELECT tablespace_name, 
       ROUND(used_percent, 2) as used_pct
FROM dba_tablespace_usage_metrics;
```

### 9.3 监听器检查

```bash
lsnrctl status

# 应该看到服务 ORCL 已注册
```

## 10. 常见问题

### 10.1 DBCA 无法启动

**错误**：
```
DISPLAY not set
```

**解决**：
```bash
export DISPLAY=localhost:0.0
dbca
```

### 10.2 创建失败 - 空间不足

**错误**：
```
ORA-01119: error in creating database file
```

**解决**：
```bash
# 检查磁盘空间
df -h /u01

# 清理空间或更改路径
```

### 10.3 字符集不匹配

**问题**：客户端和服务器字符集不一致

**解决**：
```sql
-- 确保使用 AL32UTF8
SELECT parameter, value 
FROM nls_database_parameters 
WHERE parameter = 'NLS_CHARACTERSET';
```

### 10.4 监听器未注册

**问题**：数据库创建后监听器看不到服务

**解决**：
```sql
-- 手动注册
ALTER SYSTEM REGISTER;

-- 检查 local_listener 参数
SHOW PARAMETER local_listener;
```

## 11. 最佳实践

### 11.1 创建建议

✅ **字符集**：始终使用 AL32UTF8  
✅ **归档模式**：生产环境必须启用  
✅ **内存配置**：根据实际负载调整  
✅ **存储规划**：分离数据文件和日志文件  
✅ **备份策略**：创建后立即备份  

### 11.2 命名规范

📌 SID：大写字母，不超过 8 字符  
📌 全局数据库名：包含域名，如 `ORCL.COMPANY.COM`  
📌 数据文件：使用 OMF 或统一命名规则  

### 11.3 安全配置

📌 修改默认密码  
📌 锁定不必要的账户  
📌 启用审计  
📌 配置防火墙
