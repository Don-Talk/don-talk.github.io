## 1. 监听器简介

### 1.1 作用

Oracle 监听器（Listener）是数据库的网络入口，负责：
- 接收客户端连接请求
- 验证用户身份
- 建立数据库会话
- 负载均衡（RAC 环境）

### 1.2 架构

```
客户端应用
    ↓
网络（TCP/IP）
    ↓
监听器（端口 1521）
    ↓
数据库实例
    ↓
服务器进程
```

## 2. 监听器配置文件

### 2.1 listener.ora

位置：`$ORACLE_HOME/network/admin/listener.ora`

**基本配置**：
```ini
LISTENER =
  (DESCRIPTION_LIST =
    (DESCRIPTION =
      (ADDRESS = (PROTOCOL = TCP)(HOST = localhost)(PORT = 1521))
    )
  )

SID_LIST_LISTENER =
  (SID_LIST =
    (SID_DESC =
      (GLOBAL_DBNAME = ORCL)
      (ORACLE_HOME = /u01/app/oracle/product/19.0.0/dbhome_1)
      (SID_NAME = ORCL)
    )
  )
```

**参数说明**：
- `LISTENER`：监听器名称
- `PROTOCOL`：通信协议（TCP、IPC、TCPS）
- `HOST`：主机名或 IP
- `PORT`：端口号（默认 1521）
- `SID_LIST`：静态注册的数据库列表

### 2.2 多监听器配置

```ini
# 主监听器
LISTENER =
  (DESCRIPTION_LIST =
    (DESCRIPTION =
      (ADDRESS = (PROTOCOL = TCP)(HOST = dbserver)(PORT = 1521))
    )
  )

# 备用监听器
LISTENER_BACKUP =
  (DESCRIPTION_LIST =
    (DESCRIPTION =
      (ADDRESS = (PROTOCOL = TCP)(HOST = dbserver)(PORT = 1522))
    )
  )
```

### 2.3 多网卡配置

```ini
LISTENER =
  (DESCRIPTION_LIST =
    (DESCRIPTION =
      (ADDRESS = (PROTOCOL = TCP)(HOST = 192.168.1.100)(PORT = 1521))
      (ADDRESS = (PROTOCOL = TCP)(HOST = 10.0.0.100)(PORT = 1521))
    )
  )
```

### 2.4 IPC 协议（本地连接）

```ini
LISTENER =
  (DESCRIPTION_LIST =
    (DESCRIPTION =
      (ADDRESS = (PROTOCOL = IPC)(KEY = EXTPROC1521))
      (ADDRESS = (PROTOCOL = TCP)(HOST = localhost)(PORT = 1521))
    )
  )
```

## 3. 监听器管理命令

### 3.1 lsnrctl 工具

```bash
# 启动监听器
lsnrctl start

# 停止监听器
lsnrctl stop

# 查看状态
lsnrctl status

# 重新加载配置
lsnrctl reload

# 查看服务
lsnrctl services
```

```bash
# 如果监听器名称不是默认的 LISTENER
lsnrctl start LISTENER_BACKUP
lsnrctl status LISTENER_BACKUP
```

## 4. 服务注册

### 4.1 动态注册（推荐）

PMON 进程自动向监听器注册数据库服务。

**优点**：
- 自动管理
- 无需手动配置
- 支持负载均衡

**配置**：
```sql
-- 查看 local_listener 参数
SHOW PARAMETER local_listener;

-- 设置 local_listener（非标准端口时需要）
ALTER SYSTEM SET local_listener='(ADDRESS=(PROTOCOL=TCP)(HOST=localhost)(PORT=1521))';

-- 手动触发注册
ALTER SYSTEM REGISTER;
```

**验证**：
```bash
lsnrctl status

# 输出应包含：
# Service "ORCL" has 1 instance(s).
#   Instance "ORCL", status READY, has 1 handler(s) for this service...
```

### 4.2 静态注册

在 `listener.ora` 中手动配置。

**使用场景**：
- 远程启动数据库
- Data Guard 环境
- 动态注册失败时

**配置**：
```ini
SID_LIST_LISTENER =
  (SID_LIST =
    (SID_DESC =
      (GLOBAL_DBNAME = ORCL)
      (ORACLE_HOME = /u01/app/oracle/product/19.0.0/dbhome_1)
      (SID_NAME = ORCL)
    )
    (SID_DESC =
      (GLOBAL_DBNAME = ORCL_DGMGRL)
      (ORACLE_HOME = /u01/app/oracle/product/19.0.0/dbhome_1)
      (SID_NAME = ORCL)
    )
  )
```

**验证**：
```bash
lsnrctl status

# 输出应包含：
# Service "ORCL" has 1 instance(s).
#   Instance "ORCL", status UNKNOWN, has 1 handler(s) for this service...
```

**注意**：状态为 `UNKNOWN` 表示静态注册

### 4.3 动态 vs 静态对比

| 特性 | 动态注册 | 静态注册 |
|------|---------|---------|
| **配置** | 自动 | 手动 |
| **状态** | READY | UNKNOWN |
| **适用场景** | 正常连接 | 远程启动、DG |
| **维护** | 无需 | 需同步更新 |

## 5. tnsnames.ora 配置

位置：`$ORACLE_HOME/network/admin/tnsnames.ora`

### 5.1 基本配置

```ini
ORCL =
  (DESCRIPTION =
    (ADDRESS = (PROTOCOL = TCP)(HOST = dbserver)(PORT = 1521))
    (CONNECT_DATA =
      (SERVER = DEDICATED)
      (SERVICE_NAME = ORCL)
    )
  )
```

### 5.2 多地址配置（故障转移）

```ini
ORCL =
  (DESCRIPTION =
    (ADDRESS_LIST =
      (ADDRESS = (PROTOCOL = TCP)(HOST = primary)(PORT = 1521))
      (ADDRESS = (PROTOCOL = TCP)(HOST = standby)(PORT = 1521))
      (LOAD_BALANCE = OFF)
      (FAILOVER = ON)
    )
    (CONNECT_DATA =
      (SERVICE_NAME = ORCL)
    )
  )
```

### 5.3 负载均衡配置

```ini
ORCL =
  (DESCRIPTION =
    (ADDRESS_LIST =
      (LOAD_BALANCE = ON)
      (FAILOVER = ON)
      (ADDRESS = (PROTOCOL = TCP)(HOST = node1)(PORT = 1521))
      (ADDRESS = (PROTOCOL = TCP)(HOST = node2)(PORT = 1521))
    )
    (CONNECT_DATA =
      (SERVICE_NAME = ORCL)
    )
  )
```

### 5.4 EZCONNECT（免配置）

```bash
# 无需 tnsnames.ora，直接使用
sqlplus user/password@host:port/service_name

# 示例
sqlplus sys/Oracle123@dbserver:1521/ORCL as sysdba
```

## 6. sqlnet.ora 配置

位置：`$ORACLE_HOME/network/admin/sqlnet.ora`

### 6.1 常用参数

```ini
# 命名方法优先级
NAMES.DIRECTORY_PATH = (TNSNAMES, EZCONNECT, LDAP)

# 域名解析
NAMES.DEFAULT_DOMAIN = company.com

# 连接超时
SQLNET.EXPIRE_TIME = 10

# 认证服务
SQLNET.AUTHENTICATION_SERVICES = (NTS)

# 加密（可选）
SQLNET.ENCRYPTION_SERVER = REQUIRED
SQLNET.ENCRYPTION_TYPES_SERVER = (AES256)
```

### 6.2 连接超时配置

```ini
# 客户端空闲超时（分钟）
SQLNET.EXPIRE_TIME = 10

# 连接超时（秒）
TCP.CONNECT_TIMEOUT = 10
```

## 7. 监听器安全

### 7.1 密码保护

```bash
# 设置监听器密码
lsnrctl change_password

# 保存配置
lsnrctl save_config
```

在 `listener.ora` 中添加：
```ini
LOCAL_OS_AUTHENTICATION_LISTENER = OFF
PASSWORDS_LISTENER = <encrypted_password>
```

### 7.2 限制访问

```ini
# 只允许特定 IP 连接
VALID_NODE_CHECKING_REGISTRATION_LISTENER = ON
REGISTRATION_INVITED_NODES_LISTENER = (192.168.1.100, 192.168.1.101)
```

### 7.3 日志和跟踪

```ini
# 日志配置
LOG_DIRECTORY_LISTENER = /u01/app/oracle/diag/tnslsnr
LOG_FILE_LISTENER = listener.log
LOGGING_LISTENER = ON

# 跟踪配置（调试用）
TRACE_DIRECTORY_LISTENER = /u01/app/oracle/diag/tnslsnr
TRACE_FILE_LISTENER = listener.trc
TRACE_LEVEL_LISTENER = SUPPORT
```

## 8. 监控和诊断

### 8.1 查看监听器状态

```bash
# 详细状态
lsnrctl status

# 输出示例：
# Listening Endpoints Summary...
#   (DESCRIPTION=(ADDRESS=(PROTOCOL=tcp)(HOST=dbserver)(PORT=1521)))
# Services Summary...
# Service "ORCL" has 1 instance(s).
#   Instance "ORCL", status READY, has 1 handler(s) for this service...
```

### 8.2 查看服务详情

```bash
lsnrctl services

# 输出示例：
# Service "ORCL" has 1 instance(s).
#   Instance "ORCL", status READY, has 1 handler(s) for this service...
#     Handler(s):
#       "DEDICATED" established:10 refused:0 state:ready
#          LOCAL SERVER
```

### 8.3 查看日志

```bash
# 监听器日志位置
echo $ORACLE_BASE/diag/tnslsnr/$(hostname)/listener/alert/log.xml

# 查看最新日志
tail -f $ORACLE_BASE/diag/tnslsnr/$(hostname)/listener/alert/log.xml

# 文本格式日志
tail -f $ORACLE_BASE/diag/tnslsnr/$(hostname)/listener/trace/listener.log
```

### 8.4 常见问题诊断

#### 问题 1：TNS-12541 No listener

**原因**：监听器未启动

**解决**：
```bash
lsnrctl start
```

#### 问题 2：TNS-12514 Listener does not know of service

**原因**：服务未注册

**解决**：
```sql
-- 检查数据库是否打开
SELECT status FROM v$instance;

-- 手动注册
ALTER SYSTEM REGISTER;

-- 检查 local_listener
SHOW PARAMETER local_listener;
```

#### 问题 3：TNS-12560 Protocol adapter error

**原因**：环境变量问题

**解决**：
```bash
export ORACLE_SID=ORCL
export ORACLE_HOME=/u01/app/oracle/product/19.0.0/dbhome_1
```

#### 问题 4：连接被拒绝

**原因**：防火墙阻止

**解决**：
```bash
# Linux
firewall-cmd --permanent --add-port=1521/tcp
firewall-cmd --reload

# Windows
New-NetFirewallRule -DisplayName "Oracle" -Direction Inbound -LocalPort 1521 -Protocol TCP -Action Allow
```

## 9. 高级配置

### 9.1 共享服务器模式

```ini
# listener.ora
LISTENER =
  (DESCRIPTION =
    (ADDRESS = (PROTOCOL = TCP)(HOST = dbserver)(PORT = 1521))
  )

SID_LIST_LISTENER =
  (SID_LIST =
    (SID_DESC =
      (GLOBAL_DBNAME = ORCL)
      (ORACLE_HOME = /u01/app/oracle/product/19.0.0/dbhome_1)
      (SID_NAME = ORCL)
    )
  )
```

```sql
-- 配置共享服务器
ALTER SYSTEM SET shared_servers=5;
ALTER SYSTEM SET max_shared_servers=20;
ALTER SYSTEM SET dispatchers='(PROTOCOL=TCP)(DISPATCHERS=5)';
```

### 9.2 连接负载均衡

```ini
# tnsnames.ora
ORCL =
  (DESCRIPTION =
    (ADDRESS_LIST =
      (LOAD_BALANCE = ON)
      (ADDRESS = (PROTOCOL = TCP)(HOST = node1)(PORT = 1521))
      (ADDRESS = (PROTOCOL = TCP)(HOST = node2)(PORT = 1521))
    )
    (CONNECT_DATA =
      (SERVICE_NAME = ORCL)
    )
  )
```

### 9.3 透明应用故障转移（TAF）

```ini
ORCL =
  (DESCRIPTION =
    (ADDRESS = (PROTOCOL = TCP)(HOST = dbserver)(PORT = 1521))
    (CONNECT_DATA =
      (SERVICE_NAME = ORCL)
      (FAILOVER_MODE =
        (TYPE = SELECT)
        (METHOD = BASIC)
        (RETRIES = 180)
        (DELAY = 5)
      )
    )
  )
```

## 10. 最佳实践

### 10.1 配置建议

✅ **端口**：使用标准端口 1521  
✅ **注册**：优先使用动态注册  
✅ **日志**：启用并定期清理  
✅ **安全**：生产环境启用密码保护  
✅ **监控**：定期检查监听器状态  

### 10.2 性能优化

📌 合理设置连接超时  
📌 使用连接池减少连接开销  
📌 监控活动连接数  
📌 定期清理无效连接  

### 10.3 安全加固

📌 限制监听器管理权限  
📌 启用日志审计  
📌 配置防火墙规则  
📌 定期更新补丁
