## 1. Oracle 网络架构

### 1.1 组件说明

| 组件 | 文件 | 作用 |
|------|------|------|
| **监听器** | `listener.ora` | 接收连接请求 |

```
客户端应用
    ↓

    ↓

    ↓
监听器验证并转发请求
    ↓
创建服务器进程
    ↓
建立会话
```

## 2. tnsnames.ora 详解

位置：`$ORACLE_HOME/network/admin/tnsnames.ora`

### 2.1 基本语法

```ini
<alias> =
  (DESCRIPTION =
    (ADDRESS = (PROTOCOL = <protocol>)(HOST = <host>)(PORT = <port>))
    (CONNECT_DATA =
      (SERVER = <server_type>)
      (SERVICE_NAME = <service_name>)
    )
  )
```

### 2.2 标准配置示例

```ini
# 基本连接
ORCL =
  (DESCRIPTION =
    (ADDRESS = (PROTOCOL = TCP)(HOST = dbserver.company.com)(PORT = 1521))
    (CONNECT_DATA =
      (SERVER = DEDICATED)
      (SERVICE_NAME = ORCL)
    )
  )

# 使用 SID
ORCL_SID =
  (DESCRIPTION =
    (ADDRESS = (PROTOCOL = TCP)(HOST = dbserver)(PORT = 1521))
    (CONNECT_DATA =
      (SID = ORCL)
    )
  )
```

### 2.3 多地址配置

#### 故障转移

```ini
ORCL_FAILOVER =
  (DESCRIPTION =
    (ADDRESS_LIST =
      (FAILOVER = ON)
      (LOAD_BALANCE = OFF)
      (ADDRESS = (PROTOCOL = TCP)(HOST = primary)(PORT = 1521))
      (ADDRESS = (PROTOCOL = TCP)(HOST = standby)(PORT = 1521))
    )
    (CONNECT_DATA =
      (SERVICE_NAME = ORCL)
    )
  )
```

#### 负载均衡

```ini
ORCL_LB =
  (DESCRIPTION =
    (ADDRESS_LIST =
      (LOAD_BALANCE = ON)
      (FAILOVER = ON)
      (ADDRESS = (PROTOCOL = TCP)(HOST = node1)(PORT = 1521))
      (ADDRESS = (PROTOCOL = TCP)(HOST = node2)(PORT = 1521))
      (ADDRESS = (PROTOCOL = TCP)(HOST = node3)(PORT = 1521))
    )
    (CONNECT_DATA =
      (SERVICE_NAME = ORCL)
    )
  )
```

### 2.4 连接超时和重试

```ini
ORCL_RETRY =
  (DESCRIPTION =
    (ADDRESS = (PROTOCOL = TCP)(HOST = dbserver)(PORT = 1521))
    (CONNECT_DATA =
      (SERVICE_NAME = ORCL)
    )
    (CONNECT_TIMEOUT = 10)
    (RETRY_COUNT = 3)
    (RETRY_DELAY = 5)
  )
```

### 2.5 透明应用故障转移（TAF）

```ini
ORCL_TAF =
  (DESCRIPTION =
    (ADDRESS = (PROTOCOL = TCP)(HOST = dbserver)(PORT = 1521))
    (CONNECT_DATA =
      (SERVICE_NAME = ORCL)
      (FAILOVER_MODE =
        (TYPE = SELECT)
        (METHOD = BASIC)
        (RETRIES = 180)
        (DELAY = 5)
      )
    )
  )
```

**FAILOVER_MODE 参数**：
- `TYPE`：
  - `SESSION`：故障后重新连接
  - `SELECT`：重放未完成的查询
  - `NONE`：不故障转移
- `METHOD`：
  - `BASIC`：故障时建立新连接
  - `PRECONNECT`：预先建立备份连接

### 2.6 多租户配置

```ini

# 连接到 PDB
```

## 3. sqlnet.ora 详解

位置：`$ORACLE_HOME/network/admin/sqlnet.ora`

### 3.1 命名方法

```ini
# 名称解析顺序
NAMES.DIRECTORY_PATH = (TNSNAMES, EZCONNECT, LDAP, HOSTNAME)
```

**解析方法**：
- `TNSNAMES`：使用 tnsnames.ora
- `EZCONNECT`：使用简易连接语法
- `LDAP`：使用目录服务
- `HOSTNAME`：使用主机名解析

### 3.2 默认域名

```ini
# 自动附加域名
NAMES.DEFAULT_DOMAIN = company.com

# 配置后，只需使用：
# sqlplus user/password@ORCL
# 实际解析为：ORCL.company.com
```

### 3.3 连接超时

```ini
# 客户端空闲超时（分钟）
SQLNET.EXPIRE_TIME = 10

# TCP 连接超时（秒）
TCP.CONNECT_TIMEOUT = 10

# 出站连接超时
SQLNET.OUTBOUND_CONNECT_TIMEOUT = 10
```

### 3.4 认证配置

```ini
# Windows 认证
SQLNET.AUTHENTICATION_SERVICES = (NTS)

# 所有认证方式
SQLNET.AUTHENTICATION_SERVICES = (ALL)

# 无认证
SQLNET.AUTHENTICATION_SERVICES = (NONE)
```

### 3.5 加密配置

```ini
# 服务器端加密
SQLNET.ENCRYPTION_SERVER = REQUIRED
SQLNET.ENCRYPTION_TYPES_SERVER = (AES256, AES192, AES128)

# 客户端加密
SQLNET.ENCRYPTION_CLIENT = REQUIRED
SQLNET.ENCRYPTION_TYPES_CLIENT = (AES256)

# 数据完整性
SQLNET.CRYPTO_CHECKSUM_SERVER = REQUIRED
SQLNET.CRYPTO_CHECKSUM_TYPES_SERVER = (SHA256)
```

### 3.6 日志和跟踪

```ini
# 日志配置
LOG_DIRECTORY_CLIENT = /u01/app/oracle/diag/clients
LOG_FILE_CLIENT = sqlnet.log
LOGGING_CLIENT = ON

# 跟踪配置（调试用）
TRACE_DIRECTORY_CLIENT = /u01/app/oracle/diag/clients
TRACE_FILE_CLIENT = client.trc
TRACE_LEVEL_CLIENT = SUPPORT
TRACE_UNIQUE_CLIENT = ON
TRACE_TIMESTAMP_CLIENT = ON
```

**跟踪级别**：
- `OFF`：关闭
- `USER`：用户级
- `ADMIN`：管理员级
- `SUPPORT`：支持级（最详细）

## 5. 网络诊断工具

### 5.1 tnsping

测试网络连通性：

```bash
# 测试 TNS 连接
tnsping ORCL

# 输出示例：
# Used TNSNAMES adapter to resolve the alias
# Attempting to contact (DESCRIPTION = ...)
# OK (10 msec)
```

### 5.2 trcroute

跟踪路由（已废弃，使用 trace 替代）

### 5.3 网络跟踪

```bash
# 启用客户端跟踪
export TNS_TRACE=ON
export TNS_TRACE_LEVEL=SUPPORT

# 运行命令
sqlplus user/password@ORCL

# 查看跟踪文件
ls $ORACLE_HOME/network/trace/*.trc
```

### 5.4 监听器日志分析

```bash
# 查看最新连接
tail -f $ORACLE_BASE/diag/tnslsnr/hostname/listener/alert/log.xml

grep "establish" listener.log | wc -l

# 查找错误
grep -i "error" listener.log
```

## 6. 常见问题排查

### 6.1 TNS-12154：无法解析服务名

**原因**：
- tnsnames.ora 配置错误
- 文件权限问题
- 语法错误

**解决**：
```bash
# 检查文件是否存在
ls -l $ORACLE_HOME/network/admin/tnsnames.ora

# 验证语法
tnsping ORCL

# 检查 TNS_ADMIN 环境变量
echo $TNS_ADMIN
```

### 6.2 TNS-12541：无监听器

**原因**：监听器未启动

**解决**：
```bash
lsnrctl status
lsnrctl start
```

### 6.3 TNS-12560：协议适配器错误

**原因**：
- ORACLE_SID 未设置
- 环境变量问题

**解决**：
```bash
export ORACLE_SID=ORCL
export ORACLE_HOME=/u01/app/oracle/product/19.0.0/dbhome_1
```

### 6.4 ORA-12170：连接超时

**原因**：
- 网络不通
- 防火墙阻止
- 主机不可达

**解决**：
```bash
# 测试网络连通性
ping dbserver

# 测试端口
telnet dbserver 1521

# 检查防火墙
firewall-cmd --list-ports
```

### 6.5 ORA-12514：监听器不知道服务

**原因**：服务未注册

**解决**：
```sql
-- 检查数据库状态
SELECT status FROM v$instance;

-- 手动注册
ALTER SYSTEM REGISTER;

-- 检查服务名
SHOW PARAMETER service_names;
```

## 8. 实用脚本

### 8.1 批量测试连接

```bash
#!/bin/bash
# test_connections.sh

SERVICES="ORCL DEV_DB TEST_DB PROD_DB"

for service in $SERVICES; do
    echo "Testing $service..."
    tnsping $service
    echo "---"
done
```

### 8.2 监控活动连接

```sql
-- 查看当前会话
SELECT 
    s.sid,
    s.serial#,
    s.username,
    s.program,
    s.machine,
    s.status,
    TO_CHAR(s.logon_time, 'YYYY-MM-DD HH24:MI:SS') as logon_time
FROM v$session s
WHERE s.type = 'USER'
ORDER BY s.logon_time;

-- 统计各程序连接数
SELECT 
    program,
    COUNT(*) as connection_count
FROM v$session
WHERE type = 'USER'
GROUP BY program
ORDER BY connection_count DESC;
```

### 8.3 清理无效连接

```sql
-- 查找空闲超过 1 小时的会话
SELECT 
    sid,
    serial#,
    username,
    last_call_et/3600 as idle_hours
FROM v$session
WHERE status = 'INACTIVE'
AND last_call_et > 3600
AND username IS NOT NULL;

-- 终止会话（谨慎操作）
ALTER SYSTEM KILL SESSION 'sid,serial#' IMMEDIATE;
```
