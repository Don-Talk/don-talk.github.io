```
   ↓

```

## 2. SQL 优化原则

### 2.1 减少数据访问量

**原则**：只查询需要的数据

```sql
-- ❌ 差：查询所有列
SELECT * FROM employees;

-- ✅ 好：只查询需要的列
SELECT employee_id, first_name, last_name FROM employees;

-- ❌ 差：返回过多行
SELECT * FROM orders WHERE order_date > '2020-01-01';

-- ✅ 好：限制返回行数
SELECT * FROM orders 
WHERE order_date > SYSDATE - 30
FETCH FIRST 100 ROWS ONLY;
```

### 2.2 避免全表扫描

```sql
-- ❌ 差：全表扫描
SELECT * FROM employees WHERE UPPER(last_name) = 'SMITH';

-- ✅ 好：使用函数索引
CREATE INDEX idx_emp_upper_name ON employees(UPPER(last_name));

-- 或使用基于函数的查询
SELECT * FROM employees 
WHERE last_name = 'SMITH' OR last_name = 'smith';
```

### 2.3 减少 JOIN 数量

```sql
-- ❌ 差：过多 JOIN
SELECT e.name, d.dept_name, c.city_name, r.region_name
FROM employees e
JOIN departments d ON e.dept_id = d.dept_id
JOIN cities c ON d.city_id = c.city_id
JOIN regions r ON c.region_id = r.region_id;

-- ✅ 好：适当冗余或物化视图
CREATE MATERIALIZED VIEW mv_emp_dept AS
SELECT e.employee_id, e.name, d.dept_name, d.city_name, d.region_name
FROM employees e
JOIN departments d ON e.dept_id = d.dept_id;
```

### 2.4 避免子查询，使用 JOIN

```sql
-- ❌ 差：相关子查询
SELECT e.name, e.salary
FROM employees e
WHERE e.salary > (
    SELECT AVG(salary) 
    FROM employees 
    WHERE department_id = e.department_id
);

-- ✅ 好：使用 JOIN
SELECT e.name, e.salary
FROM employees e
JOIN (
    SELECT department_id, AVG(salary) as avg_sal
    FROM employees
    GROUP BY department_id
) dept_avg ON e.department_id = dept_avg.department_id
WHERE e.salary > dept_avg.avg_sal;
```

## 3. WHERE 条件优化

### 3.1 索引列使用原则

```sql
-- ❌ 差：对索引列进行计算
SELECT * FROM orders WHERE order_date + 7 = SYSDATE;

-- ✅ 好：计算移到右边
SELECT * FROM orders WHERE order_date = SYSDATE - 7;

-- ❌ 差：隐式类型转换
SELECT * FROM employees WHERE employee_id = '100';

-- ✅ 好：类型匹配
SELECT * FROM employees WHERE employee_id = 100;

-- ❌ 差：LIKE 左模糊
SELECT * FROM customers WHERE phone LIKE '%1234';

-- ✅ 好：右模糊可以使用索引
SELECT * FROM customers WHERE phone LIKE '138%';
```

### 3.2 谓词推进

```sql
-- ❌ 差：过滤条件在外层
SELECT * FROM (
    SELECT e.*, d.dept_name
    FROM employees e
    JOIN departments d ON e.dept_id = d.dept_id
) WHERE salary > 5000;

-- ✅ 好：过滤条件内推
SELECT e.*, d.dept_name
FROM employees e
JOIN departments d ON e.dept_id = d.dept_id
WHERE e.salary > 5000;
```

### 3.3 OR 条件优化

```sql
-- ❌ 差：OR 条件可能导致全表扫描
SELECT * FROM employees 
WHERE department_id = 10 OR department_id = 20;

-- ✅ 好：使用 IN
SELECT * FROM employees 
WHERE department_id IN (10, 20);

-- ✅ 更好：UNION ALL（如果两个条件都有索引）
SELECT * FROM employees WHERE department_id = 10
UNION ALL
SELECT * FROM employees WHERE department_id = 20;
```

### 3.4 EXISTS vs IN

```sql
-- 小表驱动大表时用 IN
SELECT * FROM employees 
WHERE department_id IN (
    SELECT department_id FROM departments WHERE location_id = 1700
);

-- 大表驱动小表时用 EXISTS
SELECT * FROM departments d
WHERE EXISTS (
    SELECT 1 FROM employees e 
    WHERE e.department_id = d.department_id
    AND e.salary > 10000
);
```

## 4. JOIN 优化

### 4.1 JOIN 类型选择

| JOIN 类型 | 适用场景 |
|----------|---------|
| **Nested Loops** | 小结果集、有索引 |
| **Hash Join** | 大结果集、无合适索引 |
| **Merge Join** | 已排序的大结果集 |

### 4.2 驱动表选择

```sql
-- Oracle CBO 会自动选择，但可以提示

-- 小表驱动大表（推荐）
SELECT /*+ LEADING(d e) USE_NL(e) */ 
    e.name, d.dept_name
FROM departments d, employees e
WHERE d.dept_id = e.dept_id
AND d.location_id = 1700;  -- 小结果集作为驱动
```

### 4.3 避免笛卡尔积

```sql
-- ❌ 差：缺少 JOIN 条件
SELECT * FROM employees, departments;

-- ✅ 好：明确的 JOIN 条件
SELECT * FROM employees e
JOIN departments d ON e.dept_id = d.dept_id;

-- 检查执行计划中是否有 MERGE JOIN CARTESIAN
```

### 4.4 外连接优化

```sql
-- 旧语法（不推荐）
SELECT e.name, d.dept_name
FROM employees e, departments d
WHERE e.dept_id = d.dept_id(+);

-- 新语法（推荐）
SELECT e.name, d.dept_name
FROM employees e
LEFT JOIN departments d ON e.dept_id = d.dept_id;
```

## 5. Hint 使用指南

### 5.1 访问路径 Hint

```sql
-- 强制使用索引
SELECT /*+ INDEX(e idx_emp_dept) */ *
FROM employees e
WHERE department_id = 10;

-- 强制全表扫描
SELECT /*+ FULL(e) */ *
FROM employees e;

-- 强制索引快速全扫描
SELECT /*+ INDEX_FFS(e idx_emp_dept) */ employee_id, department_id
FROM employees e;

-- 强制使用 ROWID
SELECT /*+ ROWID(e) */ *
FROM employees e
WHERE ROWID BETWEEN 'AAA' AND 'ZZZ';
```

### 5.2 JOIN 方法 Hint

```sql
-- Nested Loops
SELECT /*+ USE_NL(e d) */ *
FROM employees e, departments d
WHERE e.dept_id = d.dept_id;

-- Hash Join
SELECT /*+ USE_HASH(e d) */ *
FROM employees e, departments d
WHERE e.dept_id = d.dept_id;

-- Merge Join
SELECT /*+ USE_MERGE(e d) */ *
FROM employees e, departments d
WHERE e.dept_id = d.dept_id;
```

### 5.3 JOIN 顺序 Hint

```sql
-- 指定驱动表
SELECT /*+ LEADING(d e l) */ *
FROM employees e, departments d, locations l
WHERE e.dept_id = d.dept_id
AND d.location_id = l.location_id;
-- d → e → l 的顺序
```

### 5.4 并行 Hint

```sql
-- 并行查询
SELECT /*+ PARALLEL(e, 4) */ *
FROM employees e
WHERE salary > 10000;

-- 并行 DML
INSERT /*+ PARALLEL(target_table, 4) */ INTO target_table
SELECT * FROM source_table;
```

### 5.5 其他常用 Hint

```sql
-- 第一行优化（快速返回首行）
SELECT /*+ FIRST_ROWS(10) */ *
FROM employees
ORDER BY hire_date DESC;

-- 所有行优化（总体吞吐最优）
SELECT /*+ ALL_ROWS */ *
FROM employees;

-- 追加插入（直接路径插入）
INSERT /*+ APPEND */ INTO target_table
SELECT * FROM source_table;

-- 缓存结果
SELECT /*+ CACHE(e) */ *
FROM employees e;

-- 不使用某个索引
SELECT /*+ NO_INDEX(e idx_emp_dept) */ *
FROM employees e
WHERE department_id = 10;
```

### 5.6 Hint 使用注意事项

⚠️ **Hint 是最后手段**
- 优先优化 SQL 和索引
- 统计信息准确时 CBO 通常表现良好
- Hint 可能在版本升级后失效

⚠️ **Hint 优先级**
- Statement-level > Object-level
- 多个 Hint 可能冲突

⚠️ **测试验证**
- 始终验证执行计划
- 测试不同数据量下的表现

## 6. 分页查询优化

### 6.1 传统分页（性能差）

```sql
-- ❌ 差：OFFSET 需要扫描前面所有行
SELECT * FROM employees
ORDER BY employee_id
OFFSET 10000 ROWS FETCH NEXT 10 ROWS ONLY;
```

### 6.2 优化分页方案

**方案 1：游标分页（推荐）**
```sql
-- ✅ 好：基于上次查询的最后一条记录
SELECT * FROM employees
WHERE employee_id > :last_id
ORDER BY employee_id
FETCH FIRST 10 ROWS ONLY;
```

**方案 2：ROWNUM 分页（Oracle 传统方式）**
```sql
-- ✅ 较好：三层嵌套
SELECT * FROM (
    SELECT /*+ FIRST_ROWS(10) */ a.*, ROWNUM rnum
    FROM (
        SELECT * FROM employees ORDER BY employee_id
    ) a
    WHERE ROWNUM <= 10010
) WHERE rnum >= 10001;
```

**方案 3：覆盖索引 + 回表**
```sql
-- ✅ 好：先用索引定位，再回表
SELECT e.* 
FROM employees e
JOIN (
    SELECT employee_id
    FROM employees
    ORDER BY employee_id
    OFFSET 10000 ROWS FETCH NEXT 10 ROWS ONLY
) ids ON e.employee_id = ids.employee_id;
```

## 7. 聚合查询优化

### 7.1 使用物化视图

```sql
-- 创建物化视图预聚合
CREATE MATERIALIZED VIEW mv_dept_salary
BUILD IMMEDIATE
REFRESH FAST ON COMMIT
AS
SELECT 
    department_id,
    COUNT(*) as emp_count,
    SUM(salary) as total_salary,
    AVG(salary) as avg_salary
FROM employees
GROUP BY department_id;

-- 查询自动使用物化视图
SELECT department_id, avg_salary
FROM mv_dept_salary
WHERE avg_salary > 5000;
```

### 7.2 分析函数优化

```sql
-- ❌ 差：自 JOIN
SELECT e1.name, e1.salary, e2.avg_salary
FROM employees e1,
     (SELECT department_id, AVG(salary) as avg_salary
      FROM employees GROUP BY department_id) e2
WHERE e1.department_id = e2.department_id;

-- ✅ 好：使用分析函数
SELECT 
    name, 
    salary,
    AVG(salary) OVER (PARTITION BY department_id) as avg_salary
FROM employees;
```

### 7.3 GROUP BY 优化

```sql
-- 确保 GROUP BY 列有索引
CREATE INDEX idx_emp_dept ON employees(department_id);

-- 避免在 GROUP BY 中使用函数
-- ❌ 差
SELECT UPPER(department_id), COUNT(*) 
FROM employees GROUP BY UPPER(department_id);

-- ✅ 好
SELECT department_id, COUNT(*) 
FROM employees GROUP BY department_id;
```

## 8. 批量操作优化

### 8.1 批量插入

```sql
-- ❌ 差：逐行插入
FOR i IN 1..10000 LOOP
    INSERT INTO employees VALUES (...);
END LOOP;
COMMIT;

-- ✅ 好：批量插入
INSERT INTO employees
SELECT * FROM temp_employees;

-- 或使用 FORALL
FORALL i IN 1..10000
    INSERT INTO employees VALUES (v_data(i));
COMMIT;
```

### 8.2 批量更新

```sql
-- ❌ 差：逐行更新
FOR rec IN cursor LOOP
    UPDATE employees SET salary = salary * 1.1 
    WHERE employee_id = rec.employee_id;
END LOOP;

-- ✅ 好：集合更新
UPDATE employees e
SET salary = salary * 1.1
WHERE EXISTS (
    SELECT 1 FROM promotion_list p
    WHERE p.employee_id = e.employee_id
);
```

### 8.3 批量删除

```sql
-- ❌ 差：一次性删除大量数据
DELETE FROM logs WHERE create_date < SYSDATE - 365;

-- ✅ 好：分批删除
BEGIN
    LOOP
        DELETE FROM logs 
        WHERE create_date < SYSDATE - 365
        AND ROWNUM <= 10000;
        
        IF SQL%ROWCOUNT = 0 THEN
            EXIT;
        END IF;
        
        COMMIT;
        DBMS_LOCK.SLEEP(1);
    END LOOP;
END;
/
```

## 9. 动态 SQL 优化

### 9.1 使用绑定变量

```plsql
-- ❌ 差：字符串拼接（硬解析）
sql_stmt := 'SELECT * FROM employees WHERE employee_id = ' || v_id;
EXECUTE IMMEDIATE sql_stmt;

-- ✅ 好：绑定变量（软解析）
sql_stmt := 'SELECT * FROM employees WHERE employee_id = :id';
EXECUTE IMMEDIATE sql_stmt INTO v_emp USING v_id;
```

### 9.2 批量动态 SQL

```plsql
-- ✅ 好：批量绑定
TYPE t_emp_ids IS TABLE OF NUMBER;
v_ids t_emp_ids := t_emp_ids(1, 2, 3, 4, 5);

FORALL i IN 1..v_ids.COUNT
    UPDATE employees 
    SET salary = salary * 1.1 
    WHERE employee_id = v_ids(i);
```

## 10. 实战案例

### 案例 1：慢查询优化

**原始 SQL**（执行时间 30 秒）：
```sql
SELECT o.order_id, o.order_date, c.customer_name, p.product_name
FROM orders o
JOIN customers c ON o.customer_id = c.customer_id
JOIN order_items oi ON o.order_id = oi.order_id
JOIN products p ON oi.product_id = p.product_id
WHERE o.order_date >= '2025-01-01'
AND c.region = 'North'
ORDER BY o.order_date DESC;
```

**优化步骤**：

1. **查看执行计划**
```sql
EXPLAIN PLAN FOR <above SQL>;
SELECT * FROM TABLE(DBMS_XPLAN.DISPLAY);
```

2. **发现问题**
- orders 表全表扫描
- 缺少有效索引

3. **添加索引**
```sql
CREATE INDEX idx_orders_date ON orders(order_date DESC);
CREATE INDEX idx_customers_region ON customers(region);
```

4. **优化后 SQL**
```sql
SELECT /*+ LEADING(o c) USE_NL(c) */ 
    o.order_id, o.order_date, c.customer_name, p.product_name
FROM orders o
JOIN customers c ON o.customer_id = c.customer_id
JOIN order_items oi ON o.order_id = oi.order_id
JOIN products p ON oi.product_id = p.product_id
WHERE o.order_date >= DATE '2025-01-01'
AND c.region = 'North'
ORDER BY o.order_date DESC;
```

**结果**：执行时间从 30 秒降到 0.5 秒

### 案例 2：分页优化

**原始 SQL**（深分页很慢）：
```sql
SELECT * FROM products
ORDER BY product_id
OFFSET 100000 ROWS FETCH NEXT 10 ROWS ONLY;
```

**优化后**：
```sql
-- 使用游标分页
SELECT * FROM products
WHERE product_id > :last_product_id
ORDER BY product_id
FETCH FIRST 10 ROWS ONLY;
```

**结果**：从 5 秒降到 10 毫秒

## 11. 最佳实践总结

### 11.1 SQL 编写规范

✅ **SELECT**
- 只选择需要的列
- 避免 SELECT *
- 使用合适的分页

✅ **WHERE**
- 充分利用索引
- 避免函数转换
- 使用绑定变量

✅ **JOIN**
- 控制 JOIN 数量
- 选择合适的 JOIN 类型
- 确保 JOIN 条件有索引

✅ **DML**
- 批量操作代替逐行
- 合理控制事务大小
- 避免频繁 COMMIT

### 11.2 优化流程

1. **收集信息**：AWR、ASH、执行计划
2. **识别瓶颈**：Top SQL、等待事件
3. **分析原因**：执行计划、统计信息
4. **制定方案**：索引、SQL 改写、Hint
5. **测试验证**：对比优化前后
6. **实施监控**：上线后持续观察

### 11.3 常见误区

❌ 过度依赖 Hint  
❌ 忽视统计信息  
❌ 盲目添加索引  
❌ 不考虑数据分布  
❌ 忽略业务场景
