## 1. ASH 概述

### 1.1 什么是 ASH？

ASH（Active Session History，主动会话历史）是 Oracle 提供的秒级性能数据采集机制。

**特点**：
- ✅ 每秒采样一次
- ✅ 只记录活动会话
- ✅ 内存中保留最近 1 小时
- ✅ 持久化到 AWR（每小时）
- ✅ 实时性能诊断

<!-- TODO: 插入图片：ASH 工作原理图 -->

### 1.2 ASH vs AWR

| 特性 | ASH | AWR |
|------|-----|-----|
| **采样频率** | 每秒 | 每小时 |
| **数据粒度** | 会话级 | 聚合统计 |
| **保留时间** | 内存 1 小时 | 磁盘 8 天+ |
| **用途** | 实时诊断 | 趋势分析 |
| **存储** | V$ACTIVE_SESSION_HISTORY | DBA_HIST_* |

### 1.3 ASH 架构

```
活动会话（每秒采样）
    ↓
SGA 中的 Circular Buffer（环形缓冲区）
    ↓
保留最近 1 小时
    ↓
每小时写入 AWR（DBA_HIST_ACTIVE_SESS_HISTORY）
```

## 2. ASH 数据视图

### 2.1 主要视图

| 视图 | 说明 | 数据来源 |
|------|------|---------|
| **V$ACTIVE_SESSION_HISTORY** | 当前 ASH 数据 | SGA（1 小时） |
| **DBA_HIST_ACTIVE_SESS_HISTORY** | 历史 ASH 数据 | AWR（持久化） |
| **V$SESSION** | 当前会话 | 实时 |

### 2.2 关键字段

```sql
SELECT 
    sample_time,        -- 采样时间
    session_id,         -- 会话 ID
    session_serial#,    -- 会话序列号
    user_id,            -- 用户 ID
    sql_id,             -- SQL ID
    sql_opname,         -- SQL 操作类型
    event,              -- 等待事件
    wait_class,         -- 等待类别
    time_waited,        -- 等待时间（微秒）
    current_obj#,       -- 当前对象 ID
    current_file#,      -- 当前文件 ID
    current_block#,     -- 当前块 ID
    program,            -- 程序名
    module,             -- 模块名
    action              -- 动作名
FROM v$active_session_history
WHERE ROWNUM <= 10;
```

## 3. 实时性能诊断

### 3.1 当前最耗时的会话

```sql
-- 过去 15 分钟最活跃的会话
SELECT 
    session_id,
    session_serial#,
    COUNT(*) as samples,
    ROUND(COUNT(*) * 10 / 60, 2) as active_seconds_avg,
    sql_id,
    event,
    wait_class
FROM v$active_session_history
WHERE sample_time > SYSDATE - 15/1440
GROUP BY session_id, session_serial#, sql_id, event, wait_class
ORDER BY samples DESC
FETCH FIRST 10 ROWS ONLY;
```

### 3.2 当前最耗时的 SQL

```sql
-- 过去 30 分钟最耗时的 SQL
SELECT 
    sql_id,
    COUNT(*) as samples,
    ROUND(COUNT(*) * 10 / 60, 2) as active_seconds,
    MIN(sample_time) as first_seen,
    MAX(sample_time) as last_seen
FROM v$active_session_history
WHERE sample_time > SYSDATE - 30/1440
AND sql_id IS NOT NULL
GROUP BY sql_id
ORDER BY samples DESC
FETCH FIRST 10 ROWS ONLY;

-- 获取 SQL 文本
SELECT sql_text FROM v$sql WHERE sql_id = '<sql_id>';
```

### 3.3 当前最严重的等待事件

```sql
-- 过去 10 分钟的等待事件分布
SELECT 
    event,
    wait_class,
    COUNT(*) as samples,
    ROUND(COUNT(*) * 100 / SUM(COUNT(*)) OVER (), 2) as pct
FROM v$active_session_history
WHERE sample_time > SYSDATE - 10/1440
AND event IS NOT NULL
GROUP BY event, wait_class
ORDER BY samples DESC
FETCH FIRST 15 ROWS ONLY;
```

## 4. 历史性能分析

### 4.1 特定时间段的性能

```sql
-- 查询昨天 10:00-11:00 的性能
SELECT 
    TO_CHAR(sample_time, 'HH24:MI') as minute,
    COUNT(*) as samples,
    COUNT(DISTINCT session_id) as active_sessions
FROM dba_hist_active_sess_history
WHERE sample_time BETWEEN 
    TO_DATE('2026-05-18 10:00', 'YYYY-MM-DD HH24:MI')
    AND TO_DATE('2026-05-18 11:00', 'YYYY-MM-DD HH24:MI')
GROUP BY TO_CHAR(sample_time, 'HH24:MI')
ORDER BY minute;
```

### 4.2 查找性能峰值

```sql
-- 找出过去 24 小时的性能峰值
SELECT 
    TO_CHAR(sample_time, 'YYYY-MM-DD HH24:MI') as time_slot,
    COUNT(*) as samples
FROM dba_hist_active_sess_history
WHERE sample_time > SYSDATE - 1
GROUP BY TO_CHAR(sample_time, 'YYYY-MM-DD HH24:MI')
ORDER BY samples DESC
FETCH FIRST 10 ROWS ONLY;
```

### 4.3 对象级热点分析

```sql
-- 最频繁访问的对象
SELECT 
    o.owner,
    o.object_name,
    o.object_type,
    COUNT(*) as samples,
    ash.event
FROM dba_hist_active_sess_history ash, dba_objects o
WHERE ash.current_obj# = o.object_id
AND ash.sample_time > SYSDATE - 1
GROUP BY o.owner, o.object_name, o.object_type, ash.event
ORDER BY samples DESC
FETCH FIRST 10 ROWS ONLY;
```

## 5. 等待事件分析

### 5.1 等待事件时间线

```sql
-- 查看特定会话的等待事件时间线
SELECT 
    sample_time,
    event,
    wait_class,
    time_waited,
    sql_id,
    current_obj#
FROM v$active_session_history
WHERE session_id = 123
AND session_serial# = 456
AND sample_time > SYSDATE - 1/24
ORDER BY sample_time;
```

### 5.2 I/O 等待分析

```sql
-- I/O 相关等待事件
SELECT 
    event,
    COUNT(*) as samples,
    ROUND(AVG(time_waited)/1000, 2) as avg_wait_ms,
    current_file#,
    current_block#
FROM v$active_session_history
WHERE sample_time > SYSDATE - 30/1440
AND wait_class = 'User I/O'
GROUP BY event, current_file#, current_block#
ORDER BY samples DESC;

-- 转换为文件名
SELECT 
    file_name,
    tablespace_name
FROM dba_data_files
WHERE file_id = <file#>;
```

### 5.3 锁等待分析

```sql
-- 锁等待会话
SELECT 
    ash.session_id,
    ash.event,
    ash.sql_id,
    ash.blocking_session,
    ash.blocking_session_status,
    ash.wait_time,
    ash.time_waited
FROM v$active_session_history ash
WHERE ash.event LIKE 'enq: TX%'
AND ash.sample_time > SYSDATE - 15/1440
ORDER BY ash.sample_time DESC;
```

## 6. SQL 执行分析

### 6.1 SQL 执行计划变化

```sql
-- 查看 SQL 的执行计划历史
SELECT 
    sql_id,
    plan_hash_value,
    COUNT(*) as executions,
    MIN(sample_time) as first_seen,
    MAX(sample_time) as last_seen
FROM dba_hist_active_sess_history
WHERE sql_id = 'abc123def456'
GROUP BY sql_id, plan_hash_value
ORDER BY first_seen;
```

### 6.2 SQL 绑定变量

```sql
-- 查看 SQL 的绑定变量值
SELECT 
    sql_id,
    name,
    datatype_string,
    value_string,
    last_captured
FROM v$sql_bind_capture
WHERE sql_id = 'abc123def456';
```

### 6.3 SQL 执行时间分布

```sql
-- SQL 执行时间直方图
SELECT 
    sql_id,
    CASE 
        WHEN elapsed_time < 1000000 THEN '< 1s'
        WHEN elapsed_time < 10000000 THEN '1-10s'
        WHEN elapsed_time < 60000000 THEN '10-60s'
        ELSE '> 60s'
    END as duration_bucket,
    COUNT(*) as count
FROM v$active_session_history
WHERE sql_id IS NOT NULL
AND sample_time > SYSDATE - 1/24
GROUP BY sql_id, 
    CASE 
        WHEN elapsed_time < 1000000 THEN '< 1s'
        WHEN elapsed_time < 10000000 THEN '1-10s'
        WHEN elapsed_time < 60000000 THEN '10-60s'
        ELSE '> 60s'
    END
ORDER BY sql_id, duration_bucket;
```

## 7. ASH 报告

### 7.1 生成 ASH 报告

```bash
# 命令行生成
@?/rdbms/admin/ashrpt.sql

# 或使用 SQL*Plus
sqlplus / as sysdba
@?/rdbms/admin/ashrpt.sql
```

**交互式步骤**：
1. 选择报告类型（HTML/TXT）
2. 选择时间范围（开始和结束时间）
3. 输入报告文件名

### 7.2 ASH 报告内容

- **Top User Events**：顶级用户等待事件
- **Top Background Events**：顶级后台等待事件
- **Top Event P1/P2/P3 Values**：等待事件参数
- **Top Service/Module**：顶级服务/模块
- **Top Client IDs**：顶级客户端 ID
- **Top SQL Command Types**：顶级 SQL 命令类型
- **Top Phases of Execution**：顶级执行阶段
- **Top PL/SQL Procedures**：顶级 PL/SQL 过程
- **Top Java Workload**：顶级 Java 工作负载
- **Top Call Types**：顶级调用类型
- **Top Sessions**：顶级会话
- **Top Blocking Sessions**：顶级阻塞会话
- **Top SQL with Top Events**：带顶级事件的 SQL
- **Top SQL with Top Row Sources**：带顶级行源的 SQL
- **Top SQL Statements**：顶级 SQL 语句
- **Top SQL using literals**：使用字面量的 SQL
- **Top Complete SQL**：完整 SQL 文本

## 8. 实用查询脚本

### 8.1 实时监控脚本

```sql
-- 保存为 monitor_ash.sql
SET LINESIZE 200
SET PAGESIZE 100

PROMPT === Top Active Sessions (Last 5 Minutes) ===

SELECT 
    session_id as sid,
    sql_id,
    event,
    wait_class,
    COUNT(*) as samples
FROM v$active_session_history
WHERE sample_time > SYSDATE - 5/1440
GROUP BY session_id, sql_id, event, wait_class
ORDER BY samples DESC
FETCH FIRST 10 ROWS ONLY;

PROMPT === Top Waiting Events (Last 5 Minutes) ===

SELECT 
    event,
    wait_class,
    COUNT(*) as samples,
    ROUND(AVG(time_waited)/1000, 2) as avg_wait_ms
FROM v$active_session_history
WHERE sample_time > SYSDATE - 5/1440
AND event IS NOT NULL
GROUP BY event, wait_class
ORDER BY samples DESC
FETCH FIRST 10 ROWS ONLY;
```

### 8.2 性能问题诊断脚本

```sql
-- 诊断特定时间段的性能问题
DEFINE start_time = '&1'
DEFINE end_time = '&2'

SELECT 
    'Top Events' as section,
    event,
    COUNT(*) as cnt
FROM dba_hist_active_sess_history
WHERE sample_time BETWEEN TO_DATE('&start_time', 'YYYY-MM-DD HH24:MI')
                      AND TO_DATE('&end_time', 'YYYY-MM-DD HH24:MI')
AND event IS NOT NULL
GROUP BY event
ORDER BY cnt DESC
FETCH FIRST 5 ROWS ONLY

UNION ALL

SELECT 
    'Top SQL' as section,
    sql_id,
    COUNT(*) as cnt
FROM dba_hist_active_sess_history
WHERE sample_time BETWEEN TO_DATE('&start_time', 'YYYY-MM-DD HH24:MI')
                      AND TO_DATE('&end_time', 'YYYY-MM-DD HH24:MI')
AND sql_id IS NOT NULL
GROUP BY sql_id
ORDER BY cnt DESC
FETCH FIRST 5 ROWS ONLY;
```

## 9. 最佳实践

### 9.1 使用建议

✅ **实时监控使用 V$ACTIVE_SESSION_HISTORY**  
✅ **历史分析使用 DBA_HIST_ACTIVE_SESS_HISTORY**  
✅ **结合 AWR 报告进行综合分析**  
✅ **定期审查 Top SQL**  
✅ **建立性能基线**  

### 9.2 性能优化

📌 **关注等待事件占比**  
📌 **识别长时间运行的会话**  
📌 **监控锁等待**  
📌 **分析 I/O 热点**  
📌 **跟踪 SQL 执行计划变化**  

### 9.3 注意事项

⚠️ **ASH 数据量很大，查询时加时间过滤**  
⚠️ **生产环境避免频繁查询 ASH**  
⚠️ **结合其他视图（V$SESSION、V$SQL）一起分析**  
⚠️ **注意时区和时间格式**
