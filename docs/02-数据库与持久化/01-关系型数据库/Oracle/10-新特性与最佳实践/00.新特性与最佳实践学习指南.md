### 01. Oracle 12c多租户架构

**核心内容**：
- CDB（Container Database）概念
- PDB（Pluggable Database）概念
- 根容器（CDB$ROOT）
- 种子容器（PDB$SEED）
- CDB 和 PDB 管理
- 资源管理
- 克隆和插拔操作

**关键知识点**：
```sql
-- 查看 CDB 信息
SELECT name, cdb FROM v$database;

-- 查看所有 PDB
SELECT name, open_mode, restricted
FROM v$pdbs;

-- 创建 PDB
CREATE PLUGGABLE DATABASE pdb_sales
ADMIN USER sales_admin IDENTIFIED BY password
FILE_NAME_CONVERT = ('/u01/app/oracle/oradata/CDB/pdbseed/',
                     '/u01/app/oracle/oradata/CDB/pdb_sales/');

-- 打开 PDB
ALTER PLUGGABLE DATABASE pdb_sales OPEN;

-- 切换到 PDB
ALTER SESSION SET CONTAINER = pdb_sales;

-- 关闭 PDB
ALTER PLUGGABLE DATABASE pdb_sales CLOSE IMMEDIATE;

-- 拔出 PDB（生成 XML 元数据）
ALTER PLUGGABLE DATABASE pdb_sales UNPLUG INTO '/tmp/pdb_sales.xml';

-- 插入 PDB
CREATE PLUGGABLE DATABASE pdb_sales
USING '/tmp/pdb_sales.xml'
NOCOPY
TEMPFILE REUSE;

-- 克隆 PDB
CREATE PLUGGABLE DATABASE pdb_clone FROM pdb_sales
FILE_NAME_CONVERT = ('/pdb_sales/', '/pdb_clone/');

-- 查看资源使用情况
SELECT con_id, resource_name, current_utilization, limit_value
FROM v$resource_limit
WHERE con_id > 1;
```

**应用场景**：
- 多租户 SaaS 应用
- 开发/测试环境快速克隆
- 数据库整合
- 版本隔离

### 02. Oracle 19c长期支持版特性

**核心内容**：
- Oracle 19c 新特性概览
- 自动索引（Automatic Indexing）
- 实时统计信息收集
- 混合分区表
- SQL 宏（SQL Macros）
- 多态表函数
- 长期支持优势

**关键知识点**：

#### 自动索引
```sql
-- 启用自动索引
ALTER SYSTEM SET AUTO_INDEXING=TRUE SCOPE=BOTH;

-- 配置自动索引表空间
BEGIN
    DBMS_AUTO_INDEX.CONFIGURE(
        'AUTO_INDEX_DEFAULT_TABLESPACE',
        'AUTO_INDEX_TS'
    );
END;

-- 查看自动索引建议
SELECT * FROM DBA_AUTO_INDEX_CONFIG;
SELECT * FROM DBA_AUTO_INDEX_IND_ACTIONS;
SELECT * FROM DBA_AUTO_INDEX_VERIFICATIONS;

-- 手动执行自动索引任务
BEGIN
    DBMS_AUTO_INDEX.EXECUTE_AUTO_INDEX_TASK(
        task_name => 'my_auto_index_task'
    );
END;
```

#### 实时统计信息收集
```sql
-- 启用实时统计
EXEC DBMS_STATS.SET_TABLE_PREFS('HR', 'EMPLOYEES', 
    'INCREMENTAL', 'TRUE');

-- 查看统计信息偏好
SELECT * FROM DBA_TAB_STAT_PREFS;

-- 增量统计收集
EXEC DBMS_STATS.GATHER_TABLE_STATS('HR', 'EMPLOYEES', 
    GRANULARITY => 'AUTO');
```

#### 混合分区表
```sql
-- 创建混合分区表（部分分区外部存储）
CREATE TABLE hybrid_sales (
    sale_id NUMBER,
    sale_date DATE,
    amount NUMBER
)
PARTITION BY RANGE (sale_date)
(
    PARTITION p_2024 VALUES LESS THAN (DATE '2025-01-01'),
    PARTITION p_2025 VALUES LESS THAN (DATE '2026-01-01'),
    PARTITION p_external VALUES LESS THAN (MAXVALUE)
    EXTERNAL PARTITION ATTRIBUTES (
        TYPE ORACLE_LOADER
        DEFAULT DIRECTORY data_dir
        LOCATION ('sales_2026.csv')
    )
);
```

### 03. Oracle 21c创新特性

**核心内容**：
- Native JSON 类型
- SQL 宏增强
- 区块链表（Blockchain Tables）
- 内存优化增强
- 自治数据库特性
- 云原生特性

**关键知识点**：

#### Native JSON 类型
```sql
-- 创建包含 JSON 列的表
CREATE TABLE orders_json (
    order_id NUMBER PRIMARY KEY,
    order_data JSON,
    created_at DATE DEFAULT SYSDATE
);

-- 插入 JSON 数据
INSERT INTO orders_json (order_id, order_data)
VALUES (1, JSON('{
    "customer": "John Doe",
    "items": [
        {"product": "Laptop", "qty": 1, "price": 999.99},
        {"product": "Mouse", "qty": 2, "price": 25.50}
    ],
    "total": 1050.99
}'));

-- 查询 JSON 数据
SELECT 
    o.order_id,
    JSON_VALUE(o.order_data, '$.customer') as customer,
    JSON_VALUE(o.order_data, '$.total') as total_amount
FROM orders_json o;

-- JSON 数组展开
SELECT 
    o.order_id,
    j.product,
    j.qty,
    j.price
FROM orders_json o,
JSON_TABLE(o.order_data, '$.items[*]'
    COLUMNS (
        product VARCHAR2(50) PATH '$.product',
        qty NUMBER PATH '$.qty',
        price NUMBER PATH '$.price'
    )
) j;

-- JSON 检查约束
ALTER TABLE orders_json
ADD CONSTRAINT valid_json CHECK (order_data IS JSON);
```

#### 区块链表
```sql
-- 创建区块链表
CREATE BLOCKCHAIN TABLE bc_transactions (
    tx_id NUMBER,
    tx_data VARCHAR2(4000),
    tx_timestamp TIMESTAMP DEFAULT SYSTIMESTAMP
)
NO DROP UNTIL 31 DAYS IDLE
NO DELETE LOCKED
HASHING USING "SHA2_512" VERSION "V1";

-- 插入数据（不能更新或删除）
INSERT INTO bc_transactions (tx_id, tx_data)
VALUES (1, 'Transaction data');

-- 验证区块链完整性
SELECT * FROM DBA_BLOCKCHAIN_TABLES;
```

### 04. JSON在Oracle中的应用

**核心内容**：
- JSON 数据类型（21c）vs JSON 文本（12c/19c）
- JSON 函数详解
- JSON 索引
- JSON 与关系数据混合查询
- JSON 性能优化
- 实际应用场景

**关键知识点**：

#### JSON 函数
```sql
-- JSON_VALUE：提取标量值
SELECT JSON_VALUE(json_col, '$.name') as name
FROM my_table;

-- JSON_QUERY：提取对象或数组
SELECT JSON_QUERY(json_col, '$.address') as address
FROM my_table;

-- JSON_EXISTS：检查字段是否存在
SELECT * FROM my_table
WHERE JSON_EXISTS(json_col, '$.email');

-- JSON_TABLE：将 JSON 转换为关系表
SELECT jt.*
FROM my_table,
JSON_TABLE(json_col, '$.items[*]'
    COLUMNS (
        item_id NUMBER PATH '$.id',
        item_name VARCHAR2(100) PATH '$.name',
        item_price NUMBER PATH '$.price'
    )
) jt;

-- JSON_OBJECT：构建 JSON 对象
SELECT JSON_OBJECT(
    'id' VALUE employee_id,
    'name' VALUE name,
    'dept' VALUE department_name
) as emp_json
FROM employees;

-- JSON_ARRAY：构建 JSON 数组
SELECT JSON_ARRAY(1, 2, 3, 4) as num_array FROM dual;
```

#### JSON 索引
```sql
-- 函数索引（12c/19c）
CREATE INDEX idx_json_name ON my_table(
    JSON_VALUE(json_col, '$.name')
);

-- JSON Search Index（21c）
CREATE SEARCH INDEX idx_json_search 
ON my_table(json_col)
FOR JSON;

-- 点记法索引（21c）
CREATE INDEX idx_json_dot ON my_table(
    json_col.name
);
```

#### 实际应用示例
```sql
-- 产品目录
CREATE TABLE products (
    product_id NUMBER PRIMARY KEY,
    attributes JSON
);

INSERT INTO products VALUES (1, JSON('{
    "color": "red",
    "size": "XL",
    "material": "cotton",
    "tags": ["casual", "summer"]
}'));

-- 查询特定属性
SELECT * FROM products
WHERE JSON_VALUE(attributes, '$.color') = 'red';

-- 数组包含查询
SELECT * FROM products
WHERE JSON_EXISTS(attributes, '$.tags[*]?(@ == "summer")');
```

### 05. 分区表最佳实践

**核心内容**：
- 分区策略选择
- 范围分区（Range）
- 列表分区（List）
- 哈希分区（Hash）
- 组合分区
- 间隔分区（Interval）
- 分区维护操作
- 分区裁剪优化

**关键知识点**：

#### 范围分区
```sql
CREATE TABLE sales_range (
    sale_id NUMBER,
    sale_date DATE,
    amount NUMBER
)
PARTITION BY RANGE (sale_date)
(
    PARTITION p_2023 VALUES LESS THAN (DATE '2024-01-01'),
    PARTITION p_2024_q1 VALUES LESS THAN (DATE '2024-04-01'),
    PARTITION p_2024_q2 VALUES LESS THAN (DATE '2024-07-01'),
    PARTITION p_2024_q3 VALUES LESS THAN (DATE '2024-10-01'),
    PARTITION p_2024_q4 VALUES LESS THAN (DATE '2025-01-01'),
    PARTITION p_max VALUES LESS THAN (MAXVALUE)
);
```

#### 间隔分区（自动创建分区）
```sql
CREATE TABLE sales_interval (
    sale_id NUMBER,
    sale_date DATE,
    amount NUMBER
)
PARTITION BY RANGE (sale_date)
INTERVAL (NUMTOYMINTERVAL(1, 'MONTH'))
(
    PARTITION p_start VALUES LESS THAN (DATE '2024-01-01')
);
-- 自动按月创建新分区
```

#### 列表分区
```sql
CREATE TABLE customers_region (
    customer_id NUMBER,
    region VARCHAR2(20),
    name VARCHAR2(100)
)
PARTITION BY LIST (region)
(
    PARTITION p_north VALUES ('BEIJING', 'TIANJIN'),
    PARTITION p_south VALUES ('SHANGHAI', 'GUANGZHOU'),
    PARTITION p_west VALUES ('CHENGDU', 'XIAN'),
    PARTITION p_east VALUES ('HANGZHOU', 'NANJING')
);
```

#### 哈希分区
```sql
-- 均匀分布数据
CREATE TABLE orders_hash (
    order_id NUMBER,
    customer_id NUMBER,
    order_date DATE
)
PARTITION BY HASH (order_id)
PARTITIONS 8;
```

#### 组合分区
```sql
CREATE TABLE sales_composite (
    sale_id NUMBER,
    sale_date DATE,
    region VARCHAR2(20),
    amount NUMBER
)
PARTITION BY RANGE (sale_date)
SUBPARTITION BY LIST (region)
SUBPARTITION TEMPLATE (
    SUBPARTITION sp_north VALUES ('NORTH'),
    SUBPARTITION sp_south VALUES ('SOUTH'),
    SUBPARTITION sp_east VALUES ('EAST'),
    SUBPARTITION sp_west VALUES ('WEST')
)
(
    PARTITION p_2024 VALUES LESS THAN (DATE '2025-01-01'),
    PARTITION p_2025 VALUES LESS THAN (DATE '2026-01-01')
);
```

#### 分区维护
```sql
-- 添加分区
ALTER TABLE sales_range ADD PARTITION p_2025_q1 
    VALUES LESS THAN (DATE '2025-04-01');

-- 删除分区
ALTER TABLE sales_range DROP PARTITION p_2023;

-- 截断分区
ALTER TABLE sales_range TRUNCATE PARTITION p_2023;

-- 拆分分区
ALTER TABLE sales_range SPLIT PARTITION p_max AT (DATE '2025-01-01')
INTO (PARTITION p_2025, PARTITION p_max);

-- 合并分区
ALTER TABLE sales_range MERGE PARTITIONS p_2024_q1, p_2024_q2 
INTO PARTITION p_2024_h1;

-- 移动分区（压缩、迁移表空间）
ALTER TABLE sales_range MOVE PARTITION p_2024 
    TABLESPACE new_ts
    COMPRESS FOR OLTP;

-- 交换分区（快速加载数据）
ALTER TABLE sales_range EXCHANGE PARTITION p_2024 
WITH TABLE sales_staging;
```

#### 分区裁剪验证
```sql
-- 查看执行计划，确认使用了分区裁剪
EXPLAIN PLAN FOR
SELECT * FROM sales_range
WHERE sale_date >= DATE '2024-06-01'
AND sale_date < DATE '2024-07-01';

SELECT * FROM TABLE(DBMS_XPLAN.DISPLAY);
-- 应该看到 PARTITION RANGE SINGLE 或类似操作
```

### 06. 物化视图与查询重写

**核心内容**：
- 物化视图类型
- 刷新策略（COMPLETE、FAST、FORCE）
- 查询重写机制
- 物化视图日志
- 性能优化
- 维护策略

**关键知识点**：

#### 创建物化视图
```sql
-- 基本物化视图
CREATE MATERIALIZED VIEW mv_dept_stats
BUILD IMMEDIATE
REFRESH COMPLETE ON DEMAND
AS
SELECT 
    d.dept_id,
    d.department_name,
    COUNT(e.employee_id) as emp_count,
    AVG(e.salary) as avg_salary,
    SUM(e.salary) as total_salary
FROM departments d
LEFT JOIN employees e ON d.dept_id = e.dept_id
GROUP BY d.dept_id, d.department_name;

-- 快速刷新的物化视图
CREATE MATERIALIZED VIEW LOG ON employees
WITH ROWID, SEQUENCE (dept_id, salary)
INCLUDING NEW VALUES;

CREATE MATERIALIZED VIEW mv_dept_fast
BUILD IMMEDIATE
REFRESH FAST ON COMMIT
AS
SELECT 
    dept_id,
    COUNT(*) as emp_count,
    SUM(salary) as total_salary
FROM employees
GROUP BY dept_id;

-- 定时刷新
CREATE MATERIALIZED VIEW mv_daily_stats
BUILD IMMEDIATE
REFRESH FORCE START WITH SYSDATE NEXT SYSDATE + 1
AS
SELECT 
    TRUNC(order_date) as order_day,
    COUNT(*) as order_count,
    SUM(amount) as daily_revenue
FROM orders
GROUP BY TRUNC(order_date);
```

#### 查询重写
```sql
-- 启用查询重写
ALTER MATERIALIZED VIEW mv_dept_stats ENABLE QUERY REWRITE;

-- 会话级启用
ALTER SESSION SET QUERY_REWRITE_ENABLED = TRUE;

-- 系统级启用
ALTER SYSTEM SET QUERY_REWRITE_ENABLED = TRUE SCOPE=BOTH;

-- 强制使用物化视图
SELECT /*+ REWRITE */ * FROM mv_dept_stats;

-- 验证查询重写
EXPLAIN PLAN FOR
SELECT dept_id, AVG(salary)
FROM employees
GROUP BY dept_id;

SELECT * FROM TABLE(DBMS_XPLAN.DISPLAY);
-- 应该看到使用了物化视图
```

#### 刷新策略
```sql
-- 手动刷新
EXEC DBMS_MVIEW.REFRESH('MV_DEPT_STATS', 'C');  -- Complete
EXEC DBMS_MVIEW.REFRESH('MV_DEPT_FAST', 'F');   -- Fast
EXEC DBMS_MVIEW.REFRESH('MV_DEPT_STATS', '?');  -- Force

-- 刷新所有依赖的物化视图
EXEC DBMS_MVIEW.REFRESH_DEPENDENT(
    number_of_failures => 100,
    list => 'MV_DEPT_STATS,MV_DEPT_FAST',
    method => '?'
);

-- 查看刷新状态
SELECT 
    mview_name,
    last_refresh_type,
    last_refresh_date,
    staleness
FROM user_mviews;
```

#### 监控和优化
```sql
-- 查看物化视图日志
SELECT * FROM user_mview_logs;

-- 查看可以快刷的物化视图
SELECT 
    owner,
    mview_name,
    can_use_parallel,
    can_use_incr_refresh,
    rewrite_enabled
FROM user_mviews;

-- 调优建议工具
BEGIN
    DBMS_MVIEW.EXPLAIN_MVIEW('MV_DEPT_STATS');
END;

SELECT * FROM MV_CAPABILITIES_TABLE;
```
