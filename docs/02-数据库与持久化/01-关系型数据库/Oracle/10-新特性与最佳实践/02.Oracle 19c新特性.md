## 2. 自动索引（Auto Indexing）

### 2.1 什么是自动索引？

自动索引是 Oracle 19c 引入的智能索引管理功能，自动分析工作负载并创建/删除索引。

**优势**：
- ✅ 减少人工干预
- ✅ 自动优化性能
- ✅ 避免无效索引
- ✅ 持续调优

### 2.2 启用自动索引

```sql
-- 检查当前状态
SELECT parameter_name, parameter_value 
FROM dba_auto_index_config;

-- 启用自动索引
EXEC DBMS_AUTO_INDEX.CONFIGURE('AUTO_INDEX_MODE', 'IMPLEMENT');

-- 配置选项：
-- IMPLEMENT：自动创建并使用索引
-- REPORT ONLY：只生成报告，不创建索引
-- OFF：禁用

-- 设置保留时间（天）
EXEC DBMS_AUTO_INDEX.CONFIGURE('AUTO_INDEX_RETENTION_FOR_AUTO', '30');

-- 设置表空间
EXEC DBMS_AUTO_INDEX.CONFIGURE('AUTO_INDEX_DEFAULT_TABLESPACE', 'AUTO_IDX_TS');
```

### 2.3 监控自动索引

```sql
-- 查看自动索引配置
SELECT * FROM dba_auto_index_config;

-- 查看自动索引活动
SELECT 
    task_id,
    execution_start,
    execution_end,
    status,
    indexes_created,
    indexes_dropped,
    space_reclaimed
FROM dba_auto_index_executions
ORDER BY execution_start DESC;

-- 查看自动创建的索引
SELECT 
    owner,
    index_name,
    table_name,
    auto,
    created
FROM dba_indexes
WHERE auto = 'YES'
ORDER BY created DESC;

-- 查看自动索引报告
SELECT DBMS_AUTO_INDEX.REPORT_ACTIVITY(
    activity_start => SYSDATE - 7,
    activity_end => SYSDATE,
    type => 'TEXT',
    section => 'ALL',
    level => 'TYPICAL'
) FROM dual;
```

### 2.4 控制自动索引

```sql
-- 排除特定 schema
EXEC DBMS_AUTO_INDEX.CONFIGURE(
    'AUTO_INDEX_SCHEMA', 
    'HR', 
    allow => FALSE
);

-- 包含特定 schema
EXEC DBMS_AUTO_INDEX.CONFIGURE(
    'AUTO_INDEX_SCHEMA', 
    'SALES', 
    allow => TRUE
);

-- 排除特定表
EXEC DBMS_AUTO_INDEX.CONFIGURE(
    'AUTO_INDEX_TABLE', 
    'HR.EMPLOYEES', 
    allow => FALSE
);
```

### 2.5 最佳实践

✅ **先在测试环境启用**  
✅ **使用 REPORT ONLY 模式评估**  
✅ **定期审查自动索引**  
✅ **结合手动索引策略**  
✅ **监控性能影响**

## 3. 混合分区表（Hybrid Partitioned Tables）

### 3.1 什么是混合分区？

混合分区表允许将不同分区存储在不同的存储层（SSD、HDD、对象存储）。

**优势**：
- ✅ 成本优化（热数据 SSD，冷数据廉价存储）
- ✅ 性能优化（热点数据快速访问）
- ✅ 透明访问（应用无需关心存储位置）

<!-- TODO: 插入图片：混合分区架构图 -->

### 3.2 创建混合分区表

```sql
-- 创建外部分区（存储在文件系统）
CREATE TABLE sales_hybrid (
    sale_id NUMBER,
    sale_date DATE,
    amount NUMBER,
    region VARCHAR2(50)
)
PARTITION BY RANGE (sale_date)
(
    -- 热数据：内部存储（高性能）
    PARTITION p_2026_q1 VALUES LESS THAN (DATE '2026-04-01')
        TABLESPACE ssd_ts,
    
    PARTITION p_2026_q2 VALUES LESS THAN (DATE '2026-07-01')
        TABLESPACE ssd_ts,
    
    -- 温数据：内部存储（标准性能）
    PARTITION p_2025_q4 VALUES LESS THAN (DATE '2026-01-01')
        TABLESPACE hdd_ts,
    
    -- 冷数据：外部存储（低成本）
    PARTITION p_2025_q3 VALUES LESS THAN (DATE '2025-10-01')
        EXTERNAL PARTITION ATTRIBUTES (
            TYPE ORACLE_LOADER
            DEFAULT DIRECTORY ext_dir
            ACCESS PARAMETERS ('FIELDS TERMINATED BY ,')
            LOCATION ('sales_2025_q3.csv')
        ),
    
    PARTITION p_2025_q2 VALUES LESS THAN (DATE '2025-07-01')
        EXTERNAL PARTITION ATTRIBUTES (
            TYPE ORACLE_LOADER
            DEFAULT DIRECTORY ext_dir
            LOCATION ('sales_2025_q2.csv')
        )
);
```

### 3.3 管理混合分区

```sql
-- 移动分区到不同存储
ALTER TABLE sales_hybrid MOVE PARTITION p_2026_q1 
TABLESPACE hdd_ts;

-- 将内部分区转换为外部
ALTER TABLE sales_hybrid CONVERT TO EXTERNAL PARTITION 
p_2025_q1
ATTRIBUTES (
    TYPE ORACLE_LOADER
    DEFAULT DIRECTORY ext_dir
    LOCATION ('sales_2025_q1.csv')
);

-- 查询混合分区表（透明访问）
SELECT * FROM sales_hybrid 
WHERE sale_date >= DATE '2025-01-01';
```

### 3.4 适用场景

📊 **时序数据**：日志、交易记录  
📊 **历史归档**：旧数据低频访问  
📊 **大数据分析**：冷热数据分离  
📊 **成本敏感**：优化存储成本

## 4. 实时统计（Real-Time Statistics）

### 4.1 什么是实时统计？

Oracle 19c 在 DML 操作时自动维护基本统计信息，减少统计信息过时的影响。

**优势**：
- ✅ CBO 始终有最新统计
- ✅ 减少硬解析
- ✅ 避免执行计划退化
- ✅ 无需频繁收集统计

### 4.2 工作原理

```
DML 操作（INSERT/UPDATE/DELETE）
    ↓
更新内存中的实时统计
    ↓
CBO 使用实时统计优化查询
    ↓
定期合并到持久化统计
```

### 4.3 查看实时统计

```sql
-- 查看表的实时统计
SELECT 
    table_name,
    num_rows,
    blocks,
    stale_stats,
    notes
FROM user_tab_statistics
WHERE table_name = 'SALES';

-- NOTES 列显示是否有实时统计
-- 'STATS_ON_CONVENTIONAL_DML' 表示有实时统计

-- 对比实时统计和持久化统计
SELECT 
    table_name,
    num_rows as persistent_rows,
    pending_row_count as realtime_rows
FROM user_tab_pending_stats;
```

### 4.4 管理实时统计

```sql
-- 禁用实时统计（不推荐）
ALTER SYSTEM SET "_optimizer_use_feedback" = FALSE;

-- 强制收集统计（覆盖实时统计）
EXEC DBMS_STATS.GATHER_TABLE_STATS('HR', 'EMPLOYEES');

-- 查看统计信息偏好
SELECT 
    preference_name,
    preference_value
FROM dba_tab_stat_prefs
WHERE table_name = 'SALES';
```

### 4.5 最佳实践

✅ **保持默认启用**（不需要额外配置）  
✅ **定期收集完整统计**（夜间作业）  
✅ **监控统计质量**  
✅ **结合 SQL Plan Management**

## 5. SQL 宏（SQL Macros）

### 5.1 什么是 SQL 宏？

SQL 宏允许创建参数化的视图，提高代码复用性。

**类型**：
- **SCALAR**：返回单个值
- **TABLE**：返回结果集

### 5.2 创建 SQL 宏

#### 示例 1：TABLE 宏

```sql
-- 创建宏函数
CREATE OR REPLACE FUNCTION get_top_employees(
    p_dept_id NUMBER,
    p_limit NUMBER
) RETURN VARCHAR2
SQL_MACRO(TABLE) IS
BEGIN
    RETURN q'[
        SELECT employee_id, first_name, last_name, salary
        FROM employees
        WHERE department_id = p_dept_id
        ORDER BY salary DESC
        FETCH FIRST p_limit ROWS ONLY
    ]';
END;
/

-- 使用宏
SELECT * FROM get_top_employees(10, 5);
SELECT * FROM get_top_employees(20, 10);
```

#### 示例 2：SCALAR 宏

```sql
CREATE OR REPLACE FUNCTION calculate_bonus(
    p_salary NUMBER,
    p_performance NUMBER
) RETURN NUMBER
SQL_MACRO(SCALAR) IS
BEGIN
    RETURN 'p_salary * p_performance * 0.1';
END;
/

-- 使用宏
SELECT 
    employee_id,
    salary,
    calculate_bonus(salary, 0.8) as bonus
FROM employees;
```

### 5.3 优势

✅ **代码复用**：避免重复 SQL  
✅ **性能优化**：内联展开，无函数调用开销  
✅ **维护简单**：集中管理逻辑  
✅ **类型安全**：编译时检查

## 6. In-Memory 增强

### 6.1 In-Memory 概述

In-Memory Option 将数据以列式格式缓存在内存中，加速分析查询。

**19c 增强**：
- ✅ 自动 In-Memory 管理
- ✅ 更好的压缩算法
- ✅ 支持更多数据类型
- ✅ 改进的并行处理

### 6.2 启用 In-Memory

```sql
-- 检查 In-Memory 状态
SHOW PARAMETER inmemory_size;

-- 启用 In-Memory（需要重启）
ALTER SYSTEM SET inmemory_size = 4G SCOPE=SPFILE;
SHUTDOWN IMMEDIATE;
STARTUP;

-- 将表设置为 In-Memory
ALTER TABLE sales INMEMORY;

-- 指定优先级
ALTER TABLE sales INMEMORY PRIORITY HIGH;

-- 指定压缩
ALTER TABLE sales INMEMORY MEMCOMPRESS FOR CAPACITY HIGH;

-- 排除某些列
ALTER TABLE sales INMEMORY NO INMEMORY (comments, description);
```

### 6.3 监控 In-Memory

```sql
-- 查看 In-Memory 对象
SELECT 
    owner,
    segment_name,
    inmemory_priority,
    inmemory_compression,
    bytes_not_populated,
    bytes_inmemory
FROM v$im_segments;

-- 查看 In-Memory 使用情况
SELECT 
    pool,
    alloc_bytes/1024/1024 as alloc_mb,
    used_bytes/1024/1024 as used_mb,
    populate_status
FROM v$inmemory_area;
```

## 7. 其他重要特性

### 7.1 区块链表（Blockchain Tables）

```sql
-- 创建区块链表（防篡改）
CREATE BLOCKCHAIN TABLE ledger_transactions (
    transaction_id NUMBER,
    amount NUMBER,
    transaction_date DATE,
    signature RAW(256)
)
NO DROP UNTIL 30 DAYS IDLE
NO DELETE LOCKED
HASHING USING "SHA2_512" VERSION V1;

-- 插入数据（不可修改）
INSERT INTO ledger_transactions VALUES (1, 1000, SYSDATE, NULL);

-- 无法 UPDATE 或 DELETE
```

### 7.2 近似查询优化

```sql
-- 近似 COUNT DISTINCT（更快）
SELECT APPROX_COUNT_DISTINCT(customer_id) 
FROM orders;

-- 近似百分位数
SELECT APPROX_PERCENTILE(0.95) WITHIN GROUP (ORDER BY amount)
FROM transactions;
```

### 7.3 自动 SQL 调优

```sql
-- 启用自动 SQL 调优
EXEC DBMS_AUTO_TASK_ADMIN.ENABLE(
    client_name => 'sql tuning advisor',
    operation => NULL,
    window_name => NULL
);

-- 查看调优建议
SELECT * FROM dba_advisor_findings 
WHERE task_name LIKE 'SYS_AUTO_SQL_TUNING_TASK%';
```
