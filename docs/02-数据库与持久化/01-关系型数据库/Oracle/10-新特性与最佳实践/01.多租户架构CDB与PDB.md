## 1. 多租户概述

### 1.1 什么是多租户？

Oracle 多租户（Multitenant）是 Oracle 12c 引入的架构，允许一个容器数据库（CDB）包含多个可插拔数据库（PDB）。

**特点**：
- ✅  consolidating 多个数据库
- ✅ 简化管理和维护
- ✅ 资源隔离和共享
- ✅ 快速克隆和部署
- ✅ 降低成本

### 1.2 架构组件

```
CDB（Container Database，容器数据库）
    ├── ROOT（CDB$ROOT）
    │   ├── 系统元数据
    │   ├── 公共用户
    │   └── Oracle 提供的对象
    ├── SEED（PDB$SEED）
    │   └── PDB 模板（只读）
    ├── PDB1（可插拔数据库 1）
    │   ├── 应用数据
    │   ├── 本地用户
    │   └── 应用对象
    ├── PDB2（可插拔数据库 2）
    └── PDB3（可插拔数据库 3）
```

### 1.3 CDB vs PDB

| 特性 | CDB | PDB |
|------|-----|-----|
| **角色** | 容器 | 应用数据库 |
| **数量** | 1 个 | 多个（最多 4096） |
| **数据存储** | 元数据 | 应用数据 |
| **用户** | 公共用户（C##） | 本地用户 |
| **备份** | 整体备份 | 单独备份 |
| **升级** | 一次升级所有 PDB | 可单独升级 |

### 1.4 优势

✅ **整合**：多个数据库合并到一个实例  
✅ **简化**：统一管理、备份、补丁  
✅ **隔离**：PDB 之间资源隔离  
✅ **灵活**：快速克隆、插拔、移动  
✅ **成本**：减少硬件和许可成本

## 2. CDB 管理

### 2.1 检查多租户状态

```sql
-- 检查是否是多租户架构
SELECT name, cdb, con_id FROM v$database;
-- CDB = YES 表示是多租户

-- 查看容器信息
SELECT 
    con_id,
    name,
    open_mode,
    restricted
FROM v$containers
ORDER BY con_id;

-- 查看当前容器
SHOW CON_NAME;
SELECT SYS_CONTEXT('USERENV', 'CON_NAME') FROM dual;
```

### 2.2 切换容器

```sql
-- 切换到 ROOT
ALTER SESSION SET CONTAINER = CDB$ROOT;

-- 切换到特定 PDB
ALTER SESSION SET CONTAINER = pdb1;

-- 查看所有 PDB
SELECT name, open_mode FROM v$pdbs;
```

### 2.3 CDB 参数管理

```sql
-- 在 ROOT 中设置参数（影响所有 PDB）
ALTER SYSTEM SET processes = 500 SCOPE=SPFILE;

-- 在 PDB 中设置参数（仅影响该 PDB）
ALTER SESSION SET CONTAINER = pdb1;
ALTER SYSTEM SET processes = 200 SCOPE=SPFILE;

-- 查看参数在不同容器的值
SELECT 
    con_id,
    name,
    value
FROM v$system_parameter
WHERE name = 'processes';
```

## 3. PDB 管理

### 3.1 创建 PDB

#### 方法 1：从 SEED 创建

```sql
-- 基本创建
CREATE PLUGGABLE DATABASE pdb_sales
ADMIN USER sales_admin IDENTIFIED BY "SalesPass123!"
FILE_NAME_CONVERT = (
    '/u01/oradata/cdb/pdbseed/',
    '/u01/oradata/cdb/pdb_sales/'
);

-- 指定存储
CREATE PLUGGABLE DATABASE pdb_hr
ADMIN USER hr_admin IDENTIFIED BY "HrPass123!"
DEFAULT TABLESPACE users
DATAFILE '/u01/oradata/cdb/pdb_hr/users01.dbf' SIZE 100M AUTOEXTEND ON
FILE_NAME_CONVERT = (
    '/u01/oradata/cdb/pdbseed/',
    '/u01/oradata/cdb/pdb_hr/'
);

-- 使用 OMF（Oracle Managed Files）
CREATE PLUGGABLE DATABASE pdb_test
ADMIN USER test_admin IDENTIFIED BY "TestPass123!";
```

#### 方法 2：克隆 PDB

```sql
-- 本地克隆
CREATE PLUGGABLE DATABASE pdb_clone FROM pdb_sales;

-- 远程克隆（从另一个 CDB）
CREATE PLUGGABLE DATABASE pdb_remote 
FROM pdb_source@dblink_source
FILE_NAME_CONVERT = ('/source/', '/target/');

-- 快照克隆（快速克隆，共享数据文件）
CREATE PLUGGABLE DATABASE pdb_snapshot 
FROM pdb_sales 
SNAPSHOT COPY;
```

#### 方法 3：插入 PDB

```sql
-- 从 XML 文件插入（12c R1）
CREATE PLUGGABLE DATABASE pdb_insert
USING '/tmp/pdb_metadata.xml'
NOCOPY
TEMPFILE REUSE;

-- 从归档的 PDB 插入（12c R2+）
CREATE PLUGGABLE DATABASE pdb_unplugged
USING '/backup/pdb_archive.pdb';
```

### 3.2 打开和关闭 PDB

```sql
-- 打开单个 PDB
ALTER PLUGGABLE DATABASE pdb_sales OPEN;

-- 打开所有 PDB
ALTER PLUGGABLE DATABASE ALL OPEN;

-- 以读写模式打开
ALTER PLUGGABLE DATABASE pdb_sales OPEN READ WRITE;

-- 以只读模式打开
ALTER PLUGGABLE DATABASE pdb_sales OPEN READ ONLY;

-- 关闭 PDB
ALTER PLUGGABLE DATABASE pdb_sales CLOSE IMMEDIATE;

-- 关闭所有 PDB
ALTER PLUGGABLE DATABASE ALL CLOSE IMMEDIATE;
```

### 3.3 删除 PDB

```sql
-- 删除 PDB（保留数据文件）
DROP PLUGGABLE DATABASE pdb_test;

-- 删除 PDB 及其数据文件
DROP PLUGGABLE DATABASE pdb_test INCLUDING DATAFILES;

-- 强制删除
DROP PLUGGABLE DATABASE pdb_test INCLUDING DATAFILES FORCE;
```

### 3.4 重命名 PDB

```sql
-- 关闭 PDB
ALTER PLUGGABLE DATABASE pdb_old CLOSE IMMEDIATE;

-- 以受限模式打开
ALTER PLUGGABLE DATABASE pdb_old OPEN RESTRICTED;

-- 重命名
ALTER PLUGGABLE DATABASE pdb_old RENAME GLOBAL_NAME TO pdb_new;

-- 关闭并重新打开
ALTER PLUGGABLE DATABASE pdb_new CLOSE IMMEDIATE;
ALTER PLUGGABLE DATABASE pdb_new OPEN;
```

## 4. PDB 插拔操作

### 4.1 拔出 PDB

```sql
-- 关闭 PDB
ALTER PLUGGABLE DATABASE pdb_sales CLOSE IMMEDIATE;

-- 拔出为 XML（12c R1）
ALTER PLUGGABLE DATABASE pdb_sales UNPLUG INTO '/tmp/pdb_sales.xml';

-- 拔出为归档文件（12c R2+，推荐）
ALTER PLUGGABLE DATABASE pdb_sales UNPLUG INTO '/backup/pdb_sales.pdb';

-- 删除拔出的 PDB（保留数据文件）
DROP PLUGGABLE DATABASE pdb_sales KEEP DATAFILES;
```

### 4.2 插入 PDB

```sql
-- 从 XML 插入
CREATE PLUGGABLE DATABASE pdb_sales
USING '/tmp/pdb_sales.xml'
NOCOPY
TEMPFILE REUSE;

-- 从归档文件插入
CREATE PLUGGABLE DATABASE pdb_sales
USING '/backup/pdb_sales.pdb'
SOURCE_FILE_NAME_CONVERT = ('/old_path/', '/new_path/');

-- 打开插入的 PDB
ALTER PLUGGABLE DATABASE pdb_sales OPEN;
```

### 4.3 跨 CDB 移动 PDB

```sql
-- 源 CDB：拔出
ALTER PLUGGABLE DATABASE pdb_sales CLOSE IMMEDIATE;
ALTER PLUGGABLE DATABASE pdb_sales UNPLUG INTO '/tmp/pdb_sales.pdb';

-- 复制文件到目标服务器
scp /tmp/pdb_sales.pdb target_host:/tmp/
scp -r /u01/oradata/cdb/pdb_sales/* target_host:/u01/oradata/cdb/

-- 目标 CDB：插入
CREATE PLUGGABLE DATABASE pdb_sales
USING '/tmp/pdb_sales.pdb'
SOURCE_FILE_NAME_CONVERT = ('/source_path/', '/target_path/');

ALTER PLUGGABLE DATABASE pdb_sales OPEN;
```

## 5. 用户管理

### 5.1 公共用户（Common User）

```sql
-- 创建公共用户（必须在 ROOT 中）
ALTER SESSION SET CONTAINER = CDB$ROOT;

CREATE USER c##app_user IDENTIFIED BY "AppPass123!"
CONTAINER = ALL;  -- 在所有容器中创建

-- 授予公共权限
GRANT CREATE SESSION TO c##app_user CONTAINER = ALL;
GRANT SELECT ANY TABLE TO c##app_user CONTAINER = ALL;

-- 公共用户可以在所有 PDB 中使用
ALTER SESSION SET CONTAINER = pdb1;
CONNECT c##app_user/AppPass123!@pdb1
```

### 5.2 本地用户（Local User）

```sql
-- 创建本地用户（在特定 PDB 中）
ALTER SESSION SET CONTAINER = pdb_sales;

CREATE USER sales_user IDENTIFIED BY "SalesUser123!";
GRANT CREATE SESSION, CREATE TABLE TO sales_user;

-- 本地用户只能在该 PDB 中使用
```

### 5.3 权限管理

```sql
-- 在 ROOT 中授予公共角色
GRANT DBA TO c##admin_user CONTAINER = ALL;

-- 在 PDB 中授予本地角色
ALTER SESSION SET CONTAINER = pdb_sales;
GRANT RESOURCE TO sales_user;

-- 查看用户类型
SELECT 
    username,
    common,
    con_id
FROM cdb_users
WHERE username LIKE 'C##%' OR username = 'SALES_USER';
```

## 6. 资源管理

### 6.1 CPU 和资源隔离

```sql
-- 创建资源计划
ALTER SYSTEM SET resource_manager_plan = 'default_plan';

-- 为 PDB 分配资源
BEGIN
    DBMS_RESOURCE_MANAGER.CREATE_PENDING_AREA();
    
    DBMS_RESOURCE_MANAGER.CREATE_CDB_PLAN(
        plan => 'sales_plan',
        comment => 'Resource plan for PDB_SALES'
    );
    
    DBMS_RESOURCE_MANAGER.CREATE_CDB_PLAN_DIRECTIVE(
        plan => 'sales_plan',
        pluggable_database => 'PDB_SALES',
        shares => 3,
        utilization_limit => 50,  -- 最多使用 50% CPU
        parallel_server_limit => 25
    );
    
    DBMS_RESOURCE_MANAGER.SUBMIT_PENDING_AREA();
END;
/

-- 启用资源计划
ALTER SYSTEM SET resource_manager_plan = 'sales_plan';
```

### 6.2 存储限制

```sql
-- 设置 PDB 最大存储空间
ALTER PLUGGABLE DATABASE pdb_sales STORAGE (MAXSIZE 10G);

-- 查看存储使用
SELECT 
    con_id,
    name,
    total_size/1024/1024 as total_mb,
    used_size/1024/1024 as used_mb
FROM v$pdbs;
```

## 7. 备份和恢复

### 7.1 备份整个 CDB

```bash
# RMAN 备份整个 CDB
rman target /

RMAN> BACKUP DATABASE PLUS ARCHIVELOG;
```

### 7.2 备份单个 PDB

```bash
# 连接到 PDB
rman target sys/password@pdb_sales

# 备份 PDB
RMAN> BACKUP DATABASE;

# 或在 CDB 中备份特定 PDB
RMAN> BACKUP PLUGGABLE DATABASE pdb_sales;
```

### 7.3 恢复 PDB

```bash
# 恢复 PDB
RMAN> RESTORE PLUGGABLE DATABASE pdb_sales;
RMAN> RECOVER PLUGGABLE DATABASE pdb_sales;
RMAN> ALTER PLUGGABLE DATABASE pdb_sales OPEN;
```

## 8. 监控和维护

### 8.1 监控 PDB

```sql
-- 查看所有 PDB 状态
SELECT 
    con_id,
    name,
    open_mode,
    restricted,
    TO_CHAR(open_time, 'YYYY-MM-DD HH24:MI') as open_time
FROM v$pdbs;

-- 查看 PDB 资源使用
SELECT 
    con_id,
    name,
    sessions_current,
    sessions_highwater
FROM v$containers;

-- 查看 PDB 历史
SELECT * FROM dba_pdb_history;
```

### 8.2 性能监控

```sql
-- AWR 报告（针对特定 PDB）
ALTER SESSION SET CONTAINER = pdb_sales;
@?/rdbms/admin/awrrpt.sql

-- ASH 查询
SELECT 
    con_id,
    COUNT(*) as samples
FROM v$active_session_history
GROUP BY con_id
ORDER BY samples DESC;
```

## 9. 最佳实践

### 9.1 设计原则

✅ **合理规划 PDB 数量**
- 开发环境：少量 PDB
- 生产环境：根据业务划分
- 避免过度整合

✅ **命名规范**
```sql
-- PDB 命名
pdb_<business_unit>_<environment>
-- 示例：pdb_sales_prod, pdb_hr_dev

-- 用户命名
C##<role>_<app>  -- 公共用户
<app>_<role>     -- 本地用户
```

✅ **资源隔离**
- 为关键 PDB 分配足够资源
- 设置合理的限制
- 监控资源使用

✅ **备份策略**
- 定期备份整个 CDB
- 重要 PDB 单独备份
- 测试恢复流程

### 9.2 运维建议

📌 **定期更新 SEED**
```sql
-- 保持 PDB$SEED 最新
ALTER PLUGGABLE DATABASE pdb$seed OPEN READ ONLY;
-- 应用补丁后
ALTER PLUGGABLE DATABASE pdb$seed CLOSE;
```

📌 **监控空间使用**
📌 **定期清理无用 PDB**
📌 **文档化 PDB 用途**
📌 **建立变更管理流程**

### 9.3 安全考虑

⚠️ **隔离敏感数据**：不同安全级别的 PDB 分开  
⚠️ **控制公共用户权限**：最小权限原则  
⚠️ **审计 PDB 操作**：记录插拔操作  
⚠️ **加密敏感 PDB**：使用 TDE

## 10. 实战案例

### 案例 1：SaaS 多租户架构

**场景**：为多个客户部署 SaaS 应用。

```sql
-- 为每个客户创建独立 PDB
CREATE PLUGGABLE DATABASE pdb_customer1
ADMIN USER c1_admin IDENTIFIED BY "Pass123!"
FILE_NAME_CONVERT = ('/pdbseed/', '/customer1/');

CREATE PLUGGABLE DATABASE pdb_customer2
ADMIN USER c2_admin IDENTIFIED BY "Pass123!"
FILE_NAME_CONVERT = ('/pdbseed/', '/customer2/');

-- 设置资源限制
ALTER PLUGGABLE DATABASE pdb_customer1 STORAGE (MAXSIZE 50G);
ALTER PLUGGABLE DATABASE pdb_customer2 STORAGE (MAXSIZE 50G);

-- 快速克隆新客户
CREATE PLUGGABLE DATABASE pdb_customer3 FROM pdb_customer1;
```

### 案例 2：开发测试环境

**场景**：快速创建测试环境。

```sql
-- 从生产 PDB 克隆测试环境
CREATE PLUGGABLE DATABASE pdb_test FROM pdb_prod;

-- 刷新测试数据
ALTER SESSION SET CONTAINER = pdb_test;
EXEC refresh_test_data();

-- 测试完成后删除
DROP PLUGGABLE DATABASE pdb_test INCLUDING DATAFILES;
```

## 11. 总结

### 11.1 核心知识点

✅ **多租户架构**：CDB、PDB、ROOT、SEED  
✅ **PDB 管理**：创建、克隆、插拔、删除  
✅ **用户管理**：公共用户、本地用户  
✅ **资源管理**：CPU、存储隔离

📌 `CREATE PLUGGABLE DATABASE` - 创建 PDB
