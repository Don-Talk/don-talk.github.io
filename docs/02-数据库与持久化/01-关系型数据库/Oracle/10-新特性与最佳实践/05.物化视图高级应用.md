## 1. 物化视图概述

### 1.1 什么是物化视图？

物化视图（Materialized View，MV）是预计算并存储查询结果的对象。

**特点**：
- ✅ 物理存储数据
- ✅ 提高查询性能
- ✅ 支持查询重写
- ✅ 定期刷新保持同步
- ✅ 适用于数据仓库

<!-- TODO: 插入图片：物化视图架构图 -->

### 1.2 物化视图 vs 普通视图

| 特性 | 普通视图 | 物化视图 |
|------|---------|---------|
| **存储** | 不存储数据 | 存储数据 |
| **性能** | 每次执行查询 | 预计算，快速 |
| **实时性** | 实时 | 延迟（取决于刷新） |
| **空间** | 无额外空间 | 需要存储空间 |
| **维护** | 无需维护 | 需要刷新 |

### 1.3 应用场景

📊 **数据仓库**：预聚合报表数据  
📊 **分布式系统**：远程数据本地缓存  
📊 **复杂查询**：加速 JOIN 和聚合  
📊 **离线分析**：历史数据快照

## 2. 创建物化视图

### 2.1 基本语法

```sql
CREATE MATERIALIZED VIEW mv_name
[BUILD IMMEDIATE|BUILD DEFERRED]
[REFRESH FAST|COMPLETE|FORCE]
[ON COMMIT|ON DEMAND]
[ENABLE QUERY REWRITE]
AS
SELECT ...;
```

### 2.2 创建示例

#### 示例 1：简单聚合

```sql
-- 创建销售汇总物化视图
CREATE MATERIALIZED VIEW mv_sales_summary
BUILD IMMEDIATE
REFRESH COMPLETE
START WITH SYSDATE
NEXT TRUNC(SYSDATE + 1) + 1/24  -- 每天凌晨 1 点刷新
ENABLE QUERY REWRITE
AS
SELECT 
    department_id,
    TO_CHAR(order_date, 'YYYY-MM') as month,
    COUNT(*) as order_count,
    SUM(amount) as total_amount,
    AVG(amount) as avg_amount
FROM orders
GROUP BY department_id, TO_CHAR(order_date, 'YYYY-MM');
```

#### 示例 2：多表 JOIN

```sql
-- 创建订单详情物化视图
CREATE MATERIALIZED VIEW mv_order_details
BUILD IMMEDIATE
REFRESH FORCE
ON DEMAND
ENABLE QUERY REWRITE
AS
SELECT 
    o.order_id,
    o.order_date,
    c.customer_name,
    p.product_name,
    oi.quantity,
    oi.unit_price,
    oi.quantity * oi.unit_price as line_total
FROM orders o
JOIN customers c ON o.customer_id = c.customer_id
JOIN order_items oi ON o.order_id = oi.order_id
JOIN products p ON oi.product_id = p.product_id;
```

#### 示例 3：延迟构建

```sql
-- 延迟构建（先定义，后手动刷新）
CREATE MATERIALIZED VIEW mv_large_report
BUILD DEFERRED
REFRESH COMPLETE
ON DEMAND
AS
SELECT /* 复杂查询 */ ...;

-- 手动构建
EXEC DBMS_MVIEW.REFRESH('MV_LARGE_REPORT', 'C');
```

## 3. 刷新策略

### 3.1 刷新类型

| 类型 | 说明 | 适用场景 |
|------|------|---------|
| **COMPLETE** | 完全重建 | 小表、首次刷新 |
| **FAST** | 增量刷新 | 大表、频繁刷新 |
| **FORCE** | 尝试 FAST，失败则 COMPLETE | 通用 |

### 3.2 刷新时机

| 时机 | 说明 |
|------|------|
| **ON DEMAND** | 手动或定时刷新 |
| **ON COMMIT** | 事务提交时刷新 |

### 3.3 配置刷新

```sql
-- 修改刷新策略
ALTER MATERIALIZED VIEW mv_sales_summary
REFRESH FAST
NEXT SYSDATE + INTERVAL '1' HOUR;

-- 禁用自动刷新
ALTER MATERIALIZED VIEW mv_sales_summary
REFRESH ON DEMAND;

-- 启用自动刷新
ALTER MATERIALIZED VIEW mv_sales_summary
REFRESH START WITH SYSDATE NEXT SYSDATE + 1/24;
```

### 3.4 手动刷新

```sql
-- 刷新单个 MV
EXEC DBMS_MVIEW.REFRESH('MV_SALES_SUMMARY', 'C');  -- C=Complete
EXEC DBMS_MVIEW.REFRESH('MV_SALES_SUMMARY', 'F');  -- F=Fast

-- 刷新多个 MV
EXEC DBMS_MVIEW.REFRESH('MV1,MV2,MV3', 'F');

-- 使用列表
BEGIN
    DBMS_MVIEW.REFRESH(
        list => 'MV_SALES_SUMMARY, MV_ORDER_DETAILS',
        method => 'F',
        parallelism => 4
    );
END;
/
```

### 3.5 刷新组

```sql
-- 创建刷新组（保证一致性）
BEGIN
    DBMS_REFRESH.MAKE(
        name => 'RG_DAILY_REPORTS',
        list => 'MV_SALES_SUMMARY, MV_INVENTORY_STATUS',
        next_date => SYSDATE,
        interval => 'SYSDATE + 1'
    );
END;
/

-- 刷新组
EXEC DBMS_REFRESH.REFRESH('RG_DAILY_REPORTS');
```

## 4. 快速刷新

### 4.1 物化视图日志

快速刷新需要创建物化视图日志（MLOG）。

```sql
-- 为基表创建 MV 日志
CREATE MATERIALIZED VIEW LOG ON orders
WITH PRIMARY KEY, ROWID, SEQUENCE (order_date, amount)
INCLUDING NEW VALUES;

-- 为多表创建日志
CREATE MATERIALIZED VIEW LOG ON customers
WITH PRIMARY KEY, ROWID;

CREATE MATERIALIZED VIEW LOG ON order_items
WITH PRIMARY KEY, ROWID, SEQUENCE (quantity, unit_price)
INCLUDING NEW VALUES;
```

**日志选项**：
- `PRIMARY KEY` - 记录主键
- `ROWID` - 记录行 ID
- `SEQUENCE` - 记录操作顺序（支持 DELETE）
- `INCLUDING NEW VALUES` - 记录新值（UPDATE 时需要）

### 4.2 快速刷新限制

✅ **支持的查询**：
- 单表聚合
- 多表 JOIN（有 MV 日志）
- WHERE 条件简单

❌ **不支持的情况**：
- 包含子查询
- 使用 ANALYTIC 函数
- DISTINCT 在某些情况下
- UNION/INTERSECT/MINUS

### 4.3 检查快速刷新能力

```sql
-- 检查是否可以快速刷新
BEGIN
    DBMS_MVIEW.EXPLAIN_MVIEW(
        'SELECT department_id, COUNT(*) FROM orders GROUP BY department_id'
    );
END;
/

-- 查看结果
SELECT * FROM MV_CAPABILITIES_TABLE;

-- CAPABILITY_NAME              POSSIBLE
-- ------------------------------ --------
-- REFRESH_COMPLETE               Y
-- REFRESH_FAST                   Y
-- REWRITE                        Y
```

## 5. 查询重写

### 5.1 什么是查询重写？

优化器自动将查询重定向到物化视图，无需修改 SQL。

```sql
-- 原始查询
SELECT department_id, SUM(amount)
FROM orders
WHERE order_date >= DATE '2026-01-01'
GROUP BY department_id;

-- 优化器自动使用 MV_SALES_SUMMARY
-- 无需修改 SQL！
```

### 5.2 启用查询重写

```sql
-- 会话级
ALTER SESSION SET query_rewrite_enabled = TRUE;

-- 系统级
ALTER SYSTEM SET query_rewrite_enabled = TRUE;

-- MV 级（创建时指定）
CREATE MATERIALIZED VIEW mv_name
ENABLE QUERY REWRITE
AS ...;

-- 修改现有 MV
ALTER MATERIALIZED VIEW mv_name ENABLE QUERY REWRITE;
```

### 5.3 控制查询重写

```sql
-- 提示：强制使用 MV
SELECT /*+ REWRITE */ department_id, SUM(amount)
FROM orders
GROUP BY department_id;

-- 提示：不使用 MV
SELECT /*+ NO_REWRITE */ department_id, SUM(amount)
FROM orders
GROUP BY department_id;

-- 设置重写级别
ALTER SESSION SET query_rewrite_integrity = TRUSTED;
-- ENFORCED：严格模式
-- TRUSTED：信任 MV 数据
-- STALE_TOLERATED：允许过期数据
```

### 5.4 监控查询重写

```sql
-- 查看是否使用了 MV
EXPLAIN PLAN FOR
SELECT department_id, SUM(amount)
FROM orders
GROUP BY department_id;

SELECT * FROM TABLE(DBMS_XPLAN.DISPLAY);

-- 执行计划显示：
-- MAT_VIEW REWRITE ACCESS FULL MV_SALES_SUMMARY
```

## 6. 物化视图管理

### 6.1 查看物化视图

```sql
-- 查看所有 MV
SELECT 
    mview_name,
    refresh_mode,
    refresh_method,
    build_mode,
    last_refresh_type,
    last_refresh_date,
    staleness
FROM user_mviews
ORDER BY mview_name;

-- 查看 MV 依赖
SELECT 
    name,
    referenced_owner,
    referenced_name,
    referenced_type
FROM user_dependencies
WHERE type = 'MATERIALIZED VIEW';

-- 查看 MV 日志
SELECT 
    log_owner,
    master,
    log_table,
    rowids,
    primary_key
FROM user_mview_logs;
```

### 6.2 删除物化视图

```sql
-- 删除 MV
DROP MATERIALIZED VIEW mv_sales_summary;

-- 删除 MV 及其日志
DROP MATERIALIZED VIEW mv_sales_summary;
DROP MATERIALIZED VIEW LOG ON orders;
```

### 6.3 重新编译

```sql
-- 重新编译 MV
ALTER MATERIALIZED VIEW mv_sales_summary COMPILE;

-- 检查状态
SELECT mview_name, compile_state
FROM user_mviews;
```

## 7. 性能优化

### 7.1 索引物化视图

```sql
-- 为 MV 创建索引
CREATE INDEX idx_mv_sales_dept 
ON mv_sales_summary (department_id);

CREATE INDEX idx_mv_sales_month 
ON mv_sales_summary (month);

-- 位图索引（低基数列）
CREATE BITMAP INDEX idx_mv_sales_dept_bmx 
ON mv_sales_summary (department_id);
```

### 7.2 分区物化视图

```sql
-- 创建分区 MV
CREATE MATERIALIZED VIEW mv_sales_partitioned
PARTITION BY RANGE (month)
(
    PARTITION p_2025_q1 VALUES LESS THAN ('2025-04'),
    PARTITION p_2025_q2 VALUES LESS THAN ('2025-07'),
    PARTITION p_2025_q3 VALUES LESS THAN ('2025-10'),
    PARTITION p_2025_q4 VALUES LESS THAN ('2026-01')
)
REFRESH COMPLETE
AS
SELECT 
    department_id,
    TO_CHAR(order_date, 'YYYY-MM') as month,
    SUM(amount) as total_amount
FROM orders
GROUP BY department_id, TO_CHAR(order_date, 'YYYY-MM');
```

### 7.3 并行刷新

```sql
-- 并行刷新 MV
ALTER MATERIALIZED VIEW mv_sales_summary
PARALLEL 4;

-- 刷新时使用并行
EXEC DBMS_MVIEW.REFRESH('MV_SALES_SUMMARY', 'C', parallelism => 4);
```

### 7.4 压缩物化视图

```sql
-- 压缩 MV
ALTER MATERIALIZED VIEW mv_sales_summary
COMPRESS FOR QUERY HIGH;

-- 移动并压缩
ALTER MATERIALIZED VIEW mv_sales_summary
MOVE COMPRESS FOR ARCHIVE HIGH;
```

```sql

    segment_name,

BEGIN
    DBMS_MVIEW.PURGE_LOG(

SELECT * FROM dba_mview_analysis
WHERE last_refresh_status != 'SUCCESS';
```

## 9. 最佳实践

### 9.1 设计原则

✅ **选择合适的刷新策略**
- 小表：COMPLETE
- 大表：FAST（需要 MV 日志）
- 实时要求高：ON COMMIT

✅ **优化查询重写**
- 确保统计信息最新
- 设置合适的重写完整性级别
- 监控重写效果

✅ **定期维护**
- 收集 MV 统计信息
- 清理 MV 日志
- 重建碎片化 MV

✅ **监控性能**
- 跟踪刷新时间
- 监控 MV 日志增长
- 评估存储空间

### 9.2 性能优化

⚡ **为 MV 创建适当索引**  
⚡ **使用分区处理大数据**  
⚡ **并行刷新大 MV**  
⚡ **压缩历史数据**  
⚡ **避免过度频繁的刷新**

## 10. 实战案例

### 案例 1：数据仓库报表加速

**需求**：加速每日销售报表查询。

```sql
-- 创建 MV 日志
CREATE MATERIALIZED VIEW LOG ON sales
WITH PRIMARY KEY, SEQUENCE (sale_date, product_id, amount)
INCLUDING NEW VALUES;

CREATE MATERIALIZED VIEW LOG ON products
WITH PRIMARY KEY;

-- 创建汇总 MV
CREATE MATERIALIZED VIEW mv_daily_sales
BUILD IMMEDIATE
REFRESH FAST
START WITH SYSDATE
NEXT TRUNC(SYSDATE + 1) + 2/24  -- 每天凌晨 2 点
ENABLE QUERY REWRITE
AS
SELECT 
    s.sale_date,
    p.category,
    p.brand,
    COUNT(*) as transaction_count,
    SUM(s.amount) as total_sales,
    AVG(s.amount) as avg_sale
FROM sales s
JOIN products p ON s.product_id = p.product_id
GROUP BY s.sale_date, p.category, p.brand;

-- 创建索引
CREATE INDEX idx_mv_daily_date ON mv_daily_sales (sale_date);
CREATE INDEX idx_mv_daily_category ON mv_daily_sales (category);

-- 报表查询（自动使用 MV）
SELECT 
    category,
    SUM(total_sales) as monthly_sales
FROM mv_daily_sales
WHERE sale_date >= DATE '2026-01-01'
GROUP BY category;
```

### 案例 2：分布式数据同步

**需求**：从生产库同步数据到报表库。

```sql
-- 在报表库创建 DB Link
CREATE DATABASE LINK prod_link
CONNECT TO report_user IDENTIFIED BY "password"
USING 'PROD_DB';

-- 创建远程 MV
CREATE MATERIALIZED VIEW mv_remote_customers
BUILD IMMEDIATE
REFRESH FAST
START WITH SYSDATE
NEXT SYSDATE + INTERVAL '1' HOUR  -- 每小时刷新
AS
SELECT * FROM customers@prod_link;

-- 本地查询（无需访问远程）
SELECT * FROM mv_remote_customers
WHERE city = 'Beijing';
```
