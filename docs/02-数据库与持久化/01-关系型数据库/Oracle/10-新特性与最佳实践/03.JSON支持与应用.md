### 1.2 JSON vs 关系数据

| 特性 | 关系数据 | JSON |
|------|---------|------|
| **结构** | 固定 schema | 灵活 schema |
| **嵌套** | 困难 | 天然支持 |
| **查询** | SQL | SQL/JSON |
| **性能** | 高 | 中等（可优化） |
| **适用场景** | 结构化数据 | 半结构化数据 |

## 2. JSON 数据类型

### 2.1 存储方式

#### 方式 1：VARCHAR2/CLOB（传统方式）

```sql
-- 使用 VARCHAR2 存储 JSON
CREATE TABLE customers (
    customer_id NUMBER PRIMARY KEY,
    customer_data VARCHAR2(4000)
    CONSTRAINT ensure_json CHECK (customer_data IS JSON)
);

-- 使用 CLOB 存储大 JSON
CREATE TABLE documents (
    doc_id NUMBER PRIMARY KEY,
    doc_content CLOB
    CONSTRAINT ensure_json CHECK (doc_content IS JSON)
);
```

#### 方式 2：原生 JSON 类型（21c+）

```sql
-- 使用原生 JSON 类型
CREATE TABLE customers_v2 (
    customer_id NUMBER PRIMARY KEY,
    customer_data JSON
);

-- 优势：
-- - 自动验证
-- - 二进制存储（更高效）
-- - 更好的性能
```

### 2.2 插入 JSON 数据

```sql
-- 插入简单 JSON
INSERT INTO customers VALUES (1, '{
    "name": "张三",
    "age": 30,
    "email": "zhangsan@example.com"
}');

-- 插入嵌套 JSON
INSERT INTO customers VALUES (2, '{
    "name": "李四",
    "age": 25,
    "address": {
        "city": "北京",
        "street": "长安街100号",
        "zipcode": "100000"
    },
    "phones": [
        {"type": "mobile", "number": "13800138000"},
        {"type": "home", "number": "010-12345678"}
    ]
}');

-- 使用 JSON_OBJECT 构建
INSERT INTO customers VALUES (3, 
    JSON_OBJECT(
        'name' VALUE '王五',
        'age' VALUE 28,
        'email' VALUE 'wangwu@example.com'
    )
);
```

### 2.3 验证 JSON

```sql
-- 检查是否是有效 JSON
SELECT 
    customer_id,
    customer_data IS JSON as is_valid
FROM customers;

-- 严格模式（检查重复键等）
SELECT customer_data IS JSON STRICT as is_strict
FROM customers;

-- 带错误信息
SELECT 
    customer_id,
    CASE 
        WHEN customer_data IS JSON THEN 'Valid'
        ELSE 'Invalid'
    END as status
FROM customers;
```

## 3. JSON 查询

### 3.1 JSON_VALUE - 提取标量值

```sql
-- 提取单个值
SELECT 
    customer_id,
    JSON_VALUE(customer_data, '$.name') as name,
    JSON_VALUE(customer_data, '$.age') as age,
    JSON_VALUE(customer_data, '$.email') as email
FROM customers;

-- 处理缺失值
SELECT 
    customer_id,
    JSON_VALUE(customer_data, '$.phone' DEFAULT 'N/A' ON EMPTY) as phone
FROM customers;

-- 错误处理
SELECT 
    customer_id,
    JSON_VALUE(customer_data, '$.name' ERROR ON ERROR) as name
FROM customers;
```

### 3.2 JSON_QUERY - 提取对象或数组

```sql
-- 提取对象
SELECT 
    customer_id,
    JSON_QUERY(customer_data, '$.address') as address
FROM customers;

-- 提取数组
SELECT 
    customer_id,
    JSON_QUERY(customer_data, '$.phones') as phones
FROM customers;

-- 美化输出
SELECT 
    customer_id,
    JSON_QUERY(customer_data, '$.address' PRETTY) as address
FROM customers;

-- 包含包装器
SELECT 
    customer_id,
    JSON_QUERY(customer_data, '$.phones' WITH WRAPPER) as phones
FROM customers;
```

### 3.3 JSON_TABLE - 将 JSON 转换为关系表

```sql
-- 基本用法
SELECT jt.*
FROM customers c,
JSON_TABLE(
    c.customer_data,
    '$'
    COLUMNS (
        name VARCHAR2(50) PATH '$.name',
        age NUMBER PATH '$.age',
        city VARCHAR2(50) PATH '$.address.city',
        street VARCHAR2(100) PATH '$.address.street'
    )
) jt
WHERE c.customer_id = 2;

-- 展开数组
SELECT jt.*
FROM customers c,
JSON_TABLE(
    c.customer_data,
    '$.phones[*]'
    COLUMNS (
        phone_type VARCHAR2(20) PATH '$.type',
        phone_number VARCHAR2(20) PATH '$.number'
    )
) jt
WHERE c.customer_id = 2;

-- 结果：
-- PHONE_TYPE   PHONE_NUMBER
-- mobile       13800138000
-- home         010-12345678
```

### 3.4 JSON_EXISTS - 检查路径是否存在

```sql
-- 检查字段是否存在
SELECT customer_id
FROM customers
WHERE JSON_EXISTS(customer_data, '$.address');

-- 检查嵌套字段
SELECT customer_id
FROM customers
WHERE JSON_EXISTS(customer_data, '$.address.city');

-- 结合其他条件
SELECT customer_id, name
FROM customers
WHERE JSON_EXISTS(customer_data, '$.phones')
AND JSON_VALUE(customer_data, '$.age') > 25;
```

## 4. JSON 修改

### 4.1 JSON_MERGE_PATCH - 合并 JSON

```sql
-- 合并两个 JSON 对象
SELECT JSON_MERGE_PATCH(
    '{"name": "张三", "age": 30}',
    '{"age": 31, "email": "zhangsan@example.com"}'
) as merged
FROM dual;

-- 结果：{"name":"张三","age":31,"email":"zhangsan@example.com"}

-- 更新记录
UPDATE customers
SET customer_data = JSON_MERGE_PATCH(
    customer_data,
    '{"age": 31}'
)
WHERE customer_id = 1;
```

### 4.2 JSON_TRANSFORM - 转换 JSON（21c+）

```sql
-- 添加字段
UPDATE customers
SET customer_data = JSON_TRANSFORM(
    customer_data,
    SET '$.vip' = TRUE
)
WHERE customer_id = 1;

-- 删除字段
UPDATE customers
SET customer_data = JSON_TRANSFORM(
    customer_data,
    REMOVE '$.old_field'
)
WHERE customer_id = 1;

-- 重命名字段
UPDATE customers
SET customer_data = JSON_TRANSFORM(
    customer_data,
    RENAME '$.name' TO 'full_name'
)
WHERE customer_id = 1;
```

### 4.3 完整更新示例

```sql
-- 复杂更新
UPDATE customers
SET customer_data = JSON_MERGE_PATCH(
    customer_data,
    JSON_OBJECT(
        'last_login' VALUE SYSDATE,
        'status' VALUE 'active'
    )
)
WHERE customer_id = 1;
```

## 5. JSON 索引

### 5.1 函数索引

```sql
-- 为常用查询字段创建索引
CREATE INDEX idx_cust_name 
ON customers (JSON_VALUE(customer_data, '$.name'));

CREATE INDEX idx_cust_age 
ON customers (JSON_VALUE(customer_data, '$.age'));

-- 复合索引
CREATE INDEX idx_cust_city_age 
ON customers (
    JSON_VALUE(customer_data, '$.address.city'),
    JSON_VALUE(customer_data, '$.age')
);
```

### 5.2 JSON Search Index（全文索引）

```sql
-- 创建 JSON Search Index（自动索引所有字段）
CREATE SEARCH INDEX idx_cust_search 
ON customers (customer_data) 
FOR JSON;

-- 使用全文搜索
SELECT customer_id
FROM customers
WHERE JSON_TEXTCONTAINS(customer_data, '$', '北京');

-- 搜索特定字段
SELECT customer_id
FROM customers
WHERE JSON_TEXTCONTAINS(customer_data, '$.address.city', '北京');
```

### 5.3 索引优化建议

✅ **为频繁查询的字段创建索引**  
✅ **避免过度索引**（影响写入性能）  
✅ **使用 Search Index 进行全文搜索**  
✅ **定期监控索引使用情况**

## 6. JSON 生成

### 6.1 JSON_OBJECT - 构建 JSON 对象

```sql
-- 从关系数据生成 JSON
SELECT 
    JSON_OBJECT(
        'id' VALUE employee_id,
        'name' VALUE first_name || ' ' || last_name,
        'email' VALUE email,
        'salary' VALUE salary
    ) as emp_json
FROM employees
WHERE department_id = 10;

-- 嵌套对象
SELECT 
    JSON_OBJECT(
        'emp_id' VALUE e.employee_id,
        'name' VALUE e.first_name,
        'dept' VALUE (
            SELECT JSON_OBJECT(
                'dept_id' VALUE d.department_id,
                'dept_name' VALUE d.department_name
            )
            FROM departments d
            WHERE d.department_id = e.department_id
        )
    ) as emp_with_dept
FROM employees e
WHERE e.employee_id = 100;
```

### 6.2 JSON_ARRAY - 构建 JSON 数组

```sql
-- 创建数组
SELECT 
    JSON_ARRAY(1, 2, 3, 4, 5) as numbers
FROM dual;

-- 从查询结果创建数组
SELECT 
    department_id,
    JSON_ARRAYAGG(employee_id ORDER BY employee_id) as emp_ids
FROM employees
GROUP BY department_id;

-- 结果：
-- DEPARTMENT_ID   EMP_IDS
-- 10              [100, 101, 102]
-- 20              [200, 201, 202]
```

### 6.3 JSON_ARRAYAGG - 聚合成数组

```sql
-- 聚合员工列表
SELECT 
    d.department_name,
    JSON_ARRAYAGG(
        JSON_OBJECT(
            'id' VALUE e.employee_id,
            'name' VALUE e.first_name || ' ' || e.last_name,
            'salary' VALUE e.salary
        )
        ORDER BY e.salary DESC
    ) as employees
FROM departments d
JOIN employees e ON d.department_id = e.department_id
GROUP BY d.department_name;
```

## 7. JSON 与关系数据集成

### 7.1 混合模型

```sql
-- 关系列 + JSON 列
CREATE TABLE orders (
    order_id NUMBER PRIMARY KEY,
    order_date DATE NOT NULL,
    customer_id NUMBER NOT NULL,
    total_amount NUMBER,
    -- JSON 存储灵活的订单详情
    order_details JSON
);

-- 插入数据
INSERT INTO orders VALUES (
    1,
    SYSDATE,
    100,
    500,
    '{
        "items": [
            {"product": "笔记本", "qty": 1, "price": 5000},
            {"product": "鼠标", "qty": 2, "price": 100}
        ],
        "shipping": {
            "method": "express",
            "address": "北京市朝阳区xxx"
        }
    }'
);

-- 查询：结合关系列和 JSON 列
SELECT 
    o.order_id,
    o.order_date,
    o.total_amount,
    JSON_VALUE(o.order_details, '$.shipping.method') as shipping_method
FROM orders o
WHERE o.customer_id = 100
AND JSON_VALUE(o.order_details, '$.shipping.method') = 'express';
```

### 7.2 视图封装

```sql
-- 创建视图简化 JSON 访问
CREATE VIEW vw_order_details AS
SELECT 
    o.order_id,
    o.order_date,
    o.customer_id,
    o.total_amount,
    JSON_VALUE(o.order_details, '$.shipping.method') as shipping_method,
    JSON_VALUE(o.order_details, '$.shipping.address') as shipping_address
FROM orders o;

-- 使用视图
SELECT * FROM vw_order_details
WHERE shipping_method = 'express';
```

## 8. 性能优化

### 8.1 最佳实践

✅ **使用原生 JSON 类型**（21c+）  
✅ **为常用查询字段创建索引**  
✅ **避免在大型 JSON 上频繁更新**  
✅ **使用 JSON_TABLE 代替多次 JSON_VALUE**  
✅ **限制 JSON 文档大小**  

### 8.2 性能监控

```sql
-- 查看 JSON 相关等待事件
SELECT event, total_waits, time_waited
FROM v$system_event
WHERE event LIKE '%json%';

-- 监控 JSON 索引使用
SELECT 
    index_name,
    table_name,
    status
FROM user_indexes
WHERE index_type LIKE '%FUNCTION-BASED%';
```

### 8.3 常见问题

⚠️ **JSON 过大影响性能** → 拆分或压缩  
⚠️ **缺少索引导致全表扫描** → 添加适当索引  
⚠️ **频繁解析 JSON** → 缓存解析结果  
⚠️ **深度嵌套查询慢** → 扁平化设计

## 9. 实战案例

### 案例 1：产品目录系统

**需求**：不同产品有不同属性，使用 JSON 存储。

```sql
-- 创建产品表
CREATE TABLE products (
    product_id NUMBER PRIMARY KEY,
    category VARCHAR2(50),
    base_price NUMBER,
    attributes JSON
);

-- 插入电子产品
INSERT INTO products VALUES (
    1,
    'electronics',
    5000,
    '{
        "brand": "Apple",
        "model": "MacBook Pro",
        "specs": {
            "cpu": "M2",
            "ram": "16GB",
            "storage": "512GB SSD"
        },
        "warranty": "2 years"
    }'
);

-- 插入服装
INSERT INTO products VALUES (
    2,
    'clothing',
    500,
    '{
        "brand": "Nike",
        "size": ["S", "M", "L", "XL"],
        "color": ["black", "white", "red"],
        "material": "cotton"
    }'
);

-- 查询电子产品的 CPU
SELECT 
    product_id,
    JSON_VALUE(attributes, '$.specs.cpu') as cpu
FROM products
WHERE category = 'electronics';

-- 创建索引
CREATE INDEX idx_prod_category 
ON products (category);

CREATE INDEX idx_prod_brand 
ON products (JSON_VALUE(attributes, '$.brand'));
```

### 案例 2：用户行为日志

**需求**：存储灵活的用户行为数据。

```sql
CREATE TABLE user_events (
    event_id NUMBER GENERATED BY DEFAULT AS IDENTITY,
    user_id NUMBER,
    event_time TIMESTAMP DEFAULT SYSTIMESTAMP,
    event_data JSON
);

-- 插入不同类型的事件
INSERT INTO user_events (user_id, event_data) VALUES (
    100,
    '{
        "type": "page_view",
        "page": "/products",
        "duration": 30
    }'
);

INSERT INTO user_events (user_id, event_data) VALUES (
    100,
    '{
        "type": "add_to_cart",
        "product_id": 123,
        "quantity": 2
    }'
);

-- 分析用户行为
SELECT 
    JSON_VALUE(event_data, '$.type') as event_type,
    COUNT(*) as count
FROM user_events
WHERE user_id = 100
GROUP BY JSON_VALUE(event_data, '$.type');
```
