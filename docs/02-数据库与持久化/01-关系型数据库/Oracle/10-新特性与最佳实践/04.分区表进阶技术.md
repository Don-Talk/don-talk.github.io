## 1. 分区技术回顾

### 1.1 分区类型

| 类型 | 说明 | 适用场景 |
|------|------|---------|
| **范围分区** | 按值范围分区 | 日期、数字范围 |
| **列表分区** | 按离散值分区 | 地区、状态 |
| **哈希分区** | 按哈希值分区 | 均匀分布 |
| **复合分区** | 多级分区 | 复杂查询模式 |
| **间隔分区** | 自动创建分区 | 时间序列数据 |
| **系统分区** | 应用控制分区 | 自定义逻辑 |

## 2. 高级分区策略

### 2.1 间隔分区（Interval Partitioning）

自动为新数据创建分区。

```sql
-- 创建间隔分区表
CREATE TABLE sales_interval (
    sale_id NUMBER,
    sale_date DATE,
    amount NUMBER,
    region VARCHAR2(50)
)
PARTITION BY RANGE (sale_date)
INTERVAL (NUMTOYMINTERVAL(1, 'MONTH'))
(
    PARTITION p_initial VALUES LESS THAN (DATE '2026-01-01')
);

-- 插入数据时自动创建分区
INSERT INTO sales_interval VALUES (1, DATE '2026-01-15', 1000, 'North');
INSERT INTO sales_interval VALUES (2, DATE '2026-02-20', 2000, 'South');

-- 查看自动创建的分区
SELECT 
    partition_name,
    high_value,
    num_rows
FROM user_tab_partitions
WHERE table_name = 'SALES_INTERVAL';
```

**间隔类型**：
- `NUMTOYMINTERVAL(n, 'MONTH')` - 按月
- `NUMTOYMINTERVAL(n, 'YEAR')` - 按年
- `NUMTODSINTERVAL(n, 'DAY')` - 按天

### 2.2 引用分区（Reference Partitioning）

子表继承父表的分区策略。

```sql
-- 父表
CREATE TABLE orders (
    order_id NUMBER PRIMARY KEY,
    order_date DATE,
    customer_id NUMBER
)
PARTITION BY RANGE (order_date)
(
    PARTITION p_2025_q1 VALUES LESS THAN (DATE '2025-04-01'),
    PARTITION p_2025_q2 VALUES LESS THAN (DATE '2025-07-01'),
    PARTITION p_2025_q3 VALUES LESS THAN (DATE '2025-10-01'),
    PARTITION p_2025_q4 VALUES LESS THAN (DATE '2026-01-01')
);

-- 子表（引用分区）
CREATE TABLE order_items (
    item_id NUMBER PRIMARY KEY,
    order_id NUMBER NOT NULL,
    product_id NUMBER,
    quantity NUMBER,
    CONSTRAINT fk_order FOREIGN KEY (order_id) REFERENCES orders(order_id)
)
PARTITION BY REFERENCE (fk_order);

-- 优势：
-- - 自动对齐分区
-- - 分区操作同步
-- - 连接查询性能优化
```

### 2.3 虚拟列分区（Virtual Column Partitioning）

基于计算表达式分区。

```sql
-- 使用虚拟列分区
CREATE TABLE employees_partitioned (
    employee_id NUMBER,
    first_name VARCHAR2(50),
    last_name VARCHAR2(50),
    salary NUMBER,
    hire_date DATE,
    -- 虚拟列
    salary_grade GENERATED ALWAYS AS (
        CASE 
            WHEN salary < 5000 THEN 'LOW'
            WHEN salary < 10000 THEN 'MEDIUM'
            ELSE 'HIGH'
        END
    ) VIRTUAL
)
PARTITION BY LIST (salary_grade)
(
    PARTITION p_low VALUES ('LOW'),
    PARTITION p_medium VALUES ('MEDIUM'),
    PARTITION p_high VALUES ('HIGH')
);

-- 插入数据（自动计算分区）
INSERT INTO employees_partitioned (employee_id, first_name, last_name, salary, hire_date)
VALUES (1, 'John', 'Doe', 8000, SYSDATE);  -- 进入 p_medium
```

### 2.4 复合分区（Composite Partitioning）

多级分区策略。

```sql
-- 范围-哈希复合分区
CREATE TABLE sales_composite (
    sale_id NUMBER,
    sale_date DATE,
    region VARCHAR2(50),
    amount NUMBER
)
PARTITION BY RANGE (sale_date)
SUBPARTITION BY HASH (region)
SUBPARTITIONS 4
(
    PARTITION p_2025_q1 VALUES LESS THAN (DATE '2025-04-01'),
    PARTITION p_2025_q2 VALUES LESS THAN (DATE '2025-07-01'),
    PARTITION p_2025_q3 VALUES LESS THAN (DATE '2025-10-01'),
    PARTITION p_2025_q4 VALUES LESS THAN (DATE '2026-01-01')
);

-- 范围-列表复合分区
CREATE TABLE sales_range_list (
    sale_id NUMBER,
    sale_date DATE,
    region VARCHAR2(50),
    amount NUMBER
)
PARTITION BY RANGE (sale_date)
SUBPARTITION BY LIST (region)
SUBPARTITION TEMPLATE (
    SUBPARTITION sp_north VALUES ('North'),
    SUBPARTITION sp_south VALUES ('South'),
    SUBPARTITION sp_east VALUES ('East'),
    SUBPARTITION sp_west VALUES ('West')
)
(
    PARTITION p_2025_q1 VALUES LESS THAN (DATE '2025-04-01'),
    PARTITION p_2025_q2 VALUES LESS THAN (DATE '2025-07-01')
);

-- 查看子分区
SELECT 
    partition_name,
    subpartition_name,
    high_value
FROM user_tab_subpartitions
WHERE table_name = 'SALES_RANGE_LIST';
```

## 3. 分区维护操作

### 3.1 添加分区

```sql
-- 添加范围分区
ALTER TABLE sales ADD PARTITION p_2026_q1 
VALUES LESS THAN (DATE '2026-04-01');

-- 添加列表分区
ALTER TABLE customers ADD PARTITION p_west 
VALUES ('California', 'Oregon', 'Washington');

-- 添加哈希分区
ALTER TABLE orders ADD PARTITION;

-- 添加子分区
ALTER TABLE sales_composite MODIFY PARTITION p_2026_q1 
ADD SUBPARTITION;
```

### 3.2 删除分区

```sql
-- 删除分区（同时删除数据）
ALTER TABLE sales DROP PARTITION p_2025_q1;

-- 删除分区但保留全局索引
ALTER TABLE sales DROP PARTITION p_2025_q1 UPDATE GLOBAL INDEXES;

-- 截断分区（清空数据，保留分区结构）
ALTER TABLE sales TRUNCATE PARTITION p_2025_q1;

-- 截断所有分区
ALTER TABLE sales TRUNCATE ALL PARTITIONS;
```

### 3.3 合并分区

```sql
-- 合并多个分区
ALTER TABLE sales MERGE PARTITIONS p_2025_q1, p_2025_q2 
INTO PARTITION p_2025_h1;

-- 合并相邻分区
ALTER TABLE sales MERGE PARTITIONS FOR (DATE '2025-01-15'), 
FOR (DATE '2025-04-15')
INTO PARTITION p_2025_merged;
```

### 3.4 拆分分区

```sql
-- 拆分分区
ALTER TABLE sales SPLIT PARTITION p_2025_q2 AT (DATE '2025-05-01')
INTO (
    PARTITION p_2025_april,
    PARTITION p_2025_may_june
);

-- 拆分子分区
ALTER TABLE sales_composite SPLIT SUBPARTITION sys_subp1 AT ('North')
INTO (
    SUBPARTITION sp_north,
    SUBPARTITION sp_other
);
```

### 3.5 移动分区

```sql
-- 移动分区到不同表空间
ALTER TABLE sales MOVE PARTITION p_2025_q1 
TABLESPACE new_ts;

-- 移动时更新索引
ALTER TABLE sales MOVE PARTITION p_2025_q1 
TABLESPACE new_ts 
UPDATE GLOBAL INDEXES;

-- 在线移动（不阻塞 DML）
ALTER TABLE sales MOVE PARTITION p_2025_q1 
TABLESPACE new_ts 
ONLINE;
```

### 3.6 重命名分区

```sql
-- 重命名分区
ALTER TABLE sales RENAME PARTITION p_2025_q1 TO p_2025_jan_mar;
```

## 4. 分区索引

### 4.1 本地索引（Local Index）

每个分区有独立的索引段。

```sql
-- 创建本地索引
CREATE INDEX idx_sales_date_local 
ON sales (sale_date) LOCAL;

-- 本地索引与分区对齐
CREATE INDEX idx_sales_region_local 
ON sales (region) LOCAL (
    PARTITION p_2025_q1 TABLESPACE idx_ts1,
    PARTITION p_2025_q2 TABLESPACE idx_ts2,
    PARTITION p_2025_q3 TABLESPACE idx_ts3,
    PARTITION p_2025_q4 TABLESPACE idx_ts4
);

-- 优势：
-- - 分区维护不影响其他分区索引
-- - 更好的可用性
-- - 支持分区级重建
```

### 4.2 全局索引（Global Index）

索引跨越所有分区。

```sql
-- 创建全局索引
CREATE INDEX idx_sales_id_global 
ON sales (sale_id) GLOBAL;

-- 指定分区
CREATE INDEX idx_sales_amount_global 
ON sales (amount) GLOBAL (
    PARTITION idx_p1 TABLESPACE idx_ts1,
    PARTITION idx_p2 TABLESPACE idx_ts2
);

-- 优势：
-- - 唯一约束必须使用全局索引
-- - 跨分区查询性能好
```

### 4.3 索引维护

```sql
-- 重建本地索引分区
ALTER INDEX idx_sales_date_local REBUILD PARTITION p_2025_q1;

-- 重建全局索引
ALTER INDEX idx_sales_id_global REBUILD;

-- 重建所有分区索引
ALTER INDEX idx_sales_date_local REBUILD SUBPARTITION ALL;

-- 禁用/启用索引
ALTER INDEX idx_sales_date_local UNUSABLE;
ALTER INDEX idx_sales_date_local REBUILD;
```

## 5. 分区裁剪

### 5.1 什么是分区裁剪？

优化器自动排除不需要的分区，减少 I/O。

```sql
-- 示例：查询只访问一个分区
EXPLAIN PLAN FOR
SELECT * FROM sales
WHERE sale_date >= DATE '2025-01-01'
AND sale_date < DATE '2025-04-01';

SELECT * FROM TABLE(DBMS_XPLAN.DISPLAY);

-- 执行计划显示：
-- PARTITION RANGE SINGLE
--   TABLE ACCESS FULL SALES [P_2025_Q1]
```

### 5.2 裁剪类型

| 类型 | 说明 |
|------|------|
| **静态裁剪** | 编译时确定分区 |
| **动态裁剪** | 运行时确定分区 |
| **多列裁剪** | 基于多个分区键 |

### 5.3 优化分区裁剪

```sql
-- ✅ 好：使用分区键
SELECT * FROM sales
WHERE sale_date = DATE '2025-01-15';

-- ❌ 差：对分区键使用函数
SELECT * FROM sales
WHERE TO_CHAR(sale_date, 'YYYY-MM') = '2025-01';

-- ✅ 好：范围查询
SELECT * FROM sales
WHERE sale_date BETWEEN DATE '2025-01-01' AND DATE '2025-03-31';

-- 验证裁剪效果
SET AUTOTRACE ON
SELECT * FROM sales WHERE sale_date >= DATE '2025-01-01';
-- 查看 "PARTITION RANGE" 部分
```

## 6. 分区并行处理

### 6.1 并行查询

```sql
-- 启用表级并行
ALTER TABLE sales PARALLEL 4;

-- 会话级并行
ALTER SESSION FORCE PARALLEL QUERY PARALLEL 4;

-- 语句级并行提示
SELECT /*+ PARALLEL(s, 4) */ *
FROM sales s
WHERE sale_date >= DATE '2025-01-01';

-- 并行分区扫描
SELECT /*+ PARALLEL */ *
FROM sales
WHERE sale_date >= DATE '2025-01-01';
```

### 6.2 并行 DDL

```sql
-- 并行创建索引
CREATE INDEX idx_sales_date 
ON sales (sale_date) 
PARALLEL 4;

-- 并行重建分区
ALTER TABLE sales MOVE PARTITION p_2025_q1 
PARALLEL 4;

-- 完成后恢复串行
ALTER INDEX idx_sales_date NOPARALLEL;
```

### 6.3 并行 DML

```sql
-- 启用并行 DML
ALTER SESSION ENABLE PARALLEL DML;

-- 并行插入
INSERT /*+ PARALLEL */ INTO sales_archive
SELECT * FROM sales
WHERE sale_date < DATE '2025-01-01';

-- 并行更新
UPDATE /*+ PARALLEL(s, 4) */ sales s
SET s.status = 'ARCHIVED'
WHERE s.sale_date < DATE '2025-01-01';
```

## 8. 最佳实践

### 8.1 设计原则

✅ **选择合适的分区键**
- 频繁用于查询过滤
- 数据分布均匀
- 避免热点分区

✅ **合理的分区粒度**
- 不要过多分区（< 1000）
- 每个分区大小适中（GB 级别）
- 考虑维护成本

✅ **使用本地索引**
- 除非需要唯一约束
- 便于分区维护

✅ **定期维护**
- 收集分区统计
- 清理旧分区
- 重建碎片化分区

### 8.2 性能优化

⚡ **利用分区裁剪**
- 查询包含分区键
- 避免对分区键使用函数

⚡ **并行处理**
- 大表操作使用并行
- 合理设置并行度

⚡ **分区压缩**
```sql
-- 压缩旧分区
ALTER TABLE sales MOVE PARTITION p_2024 
COMPRESS FOR ARCHIVE HIGH;
```

### 8.3 运维建议

📌 **自动化分区管理**
```sql
-- 定时任务清理旧分区
BEGIN

            ALTER TABLE sales DROP PARTITION FOR (DATE ''2024-01-01'');
        ]',

        enabled => TRUE
```

## 9. 实战案例

### 案例 1：日志表分区策略

**需求**：存储海量日志，快速查询近期数据。

```sql
-- 创建间隔分区日志表
CREATE TABLE app_logs (
    log_id NUMBER GENERATED BY DEFAULT AS IDENTITY,
    log_time TIMESTAMP DEFAULT SYSTIMESTAMP,
    level VARCHAR2(10),
    message CLOB,
    source VARCHAR2(100)
)
PARTITION BY RANGE (log_time)
INTERVAL (NUMTODSINTERVAL(1, 'DAY'))
(
    PARTITION p_initial VALUES LESS THAN (TIMESTAMP '2026-01-01 00:00:00')
)
ENABLE ROW MOVEMENT;

-- 创建本地索引
CREATE INDEX idx_logs_time ON app_logs (log_time) LOCAL;
CREATE INDEX idx_logs_level ON app_logs (level) LOCAL;

-- 查询近期日志（自动裁剪）
SELECT * FROM app_logs
WHERE log_time >= SYSTIMESTAMP - INTERVAL '1' HOUR;

-- 清理 30 天前的分区
ALTER TABLE app_logs DROP PARTITION 
FOR (TIMESTAMP '2025-12-01 00:00:00');
```

### 案例 2：订单历史归档

**需求**：将旧订单归档到廉价存储。

```sql
-- 主表（近期订单）
CREATE TABLE orders_current (
    order_id NUMBER,
    order_date DATE,
    amount NUMBER
)
PARTITION BY RANGE (order_date)
INTERVAL (NUMTOYMINTERVAL(1, 'MONTH'));

-- 归档表（历史订单）
CREATE TABLE orders_archive (
    order_id NUMBER,
    order_date DATE,
    amount NUMBER
)
PARTITION BY RANGE (order_date)
(
    PARTITION p_2024 VALUES LESS THAN (DATE '2025-01-01')
        COMPRESS FOR ARCHIVE HIGH
);

-- 定期归档
INSERT INTO orders_archive
SELECT * FROM orders_current
WHERE order_date < ADD_MONTHS(SYSDATE, -12);

DELETE FROM orders_current
WHERE order_date < ADD_MONTHS(SYSDATE, -12);

COMMIT;
```
