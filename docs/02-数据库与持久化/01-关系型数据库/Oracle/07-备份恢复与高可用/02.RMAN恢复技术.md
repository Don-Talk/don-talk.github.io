# 02RMAN恢复技术

**学习目标：**
> * 掌握 RMAN 完全恢复和不完全恢复
> * 学会表空间和数据文件级恢复
> * 了解块介质恢复
> * 能够诊断和解决恢复问题
> * 掌握恢复的最佳实践

---

## 1. 恢复概述

### 1.1 恢复类型

| 类型 | 说明 | 数据丢失 |
|------|------|---------|
| **完全恢复** | 恢复到最新状态 | 无 |
| **不完全恢复** | 恢复到指定时间点 | 有 |
| **表空间恢复** | 恢复特定表空间 | 取决于范围 |
| **数据文件恢复** | 恢复特定数据文件 | 取决于范围 |
| **块介质恢复** | 恢复损坏的数据块 | 无 |

### 1.2 恢复前提条件

✅ **ARCHIVELOG 模式**（不完全恢复必须）  
✅ **有效的备份**  
✅ **完整的归档日志**  
✅ **控制文件可用**  

```sql
-- 检查归档模式
ARCHIVE LOG LIST;

-- 或
SELECT log_mode FROM v$database;
```

## 2. 完全恢复

### 2.1 数据库级完全恢复

**场景**：整个数据库需要恢复。

```sql
-- 步骤 1：启动到 MOUNT 状态
STARTUP MOUNT;

-- 步骤 2：恢复数据库
RESTORE DATABASE;
RECOVER DATABASE;

-- 步骤 3：打开数据库
ALTER DATABASE OPEN;
```

**或使用 RUN 块**：
```sql
RUN {
    STARTUP MOUNT;
    RESTORE DATABASE;
    RECOVER DATABASE;
    ALTER DATABASE OPEN;
}
```

### 2.2 表空间级恢复

**场景**：单个表空间损坏，其他部分正常。

```sql
-- 在线恢复（数据库打开状态）
RUN {
    SQL 'ALTER TABLESPACE users OFFLINE IMMEDIATE';
    RESTORE TABLESPACE users;
    RECOVER TABLESPACE users;
    SQL 'ALTER TABLESPACE users ONLINE';
}

-- 离线恢复（数据库 MOUNT 状态）
STARTUP MOUNT;
RESTORE TABLESPACE users;
RECOVER TABLESPACE users;
ALTER DATABASE OPEN;
```

### 2.3 数据文件级恢复

**场景**：单个数据文件损坏。

```sql
-- 按文件号
RUN {
    SQL 'ALTER DATABASE DATAFILE 5 OFFLINE';
    RESTORE DATAFILE 5;
    RECOVER DATAFILE 5;
    SQL 'ALTER DATABASE DATAFILE 5 ONLINE';
}

-- 按文件名
RUN {
    SQL "ALTER DATABASE DATAFILE '/u01/oradata/users01.dbf' OFFLINE";
    RESTORE DATAFILE '/u01/oradata/users01.dbf';
    RECOVER DATAFILE '/u01/oradata/users01.dbf';
    SQL "ALTER DATABASE DATAFILE '/u01/oradata/users01.dbf' ONLINE";
}
```

## 3. 不完全恢复

### 3.1 基于时间的恢复

**场景**：误删除数据，需要恢复到删除前的时间点。

```sql
RUN {
    STARTUP MOUNT;
    SET UNTIL TIME "TO_DATE('2026-05-19 10:00:00', 'YYYY-MM-DD HH24:MI:SS')";
    RESTORE DATABASE;
    RECOVER DATABASE;
    ALTER DATABASE OPEN RESETLOGS;
}
```

**注意**：`RESETLOGS` 会重置日志序列号，之前的备份失效。

### 3.2 基于 SCN 的恢复

```sql
-- 先找到目标 SCN
SELECT current_scn FROM v$database;

-- 恢复
RUN {
    STARTUP MOUNT;
    SET UNTIL SCN 1234567;
    RESTORE DATABASE;
    RECOVER DATABASE;
    ALTER DATABASE OPEN RESETLOGS;
}
```

### 3.3 基于日志序列号的恢复

```sql
RUN {
    STARTUP MOUNT;
    SET UNTIL SEQUENCE 100 THREAD 1;
    RESTORE DATABASE;
    RECOVER DATABASE;
    ALTER DATABASE OPEN RESETLOGS;
}
```

### 3.4 取消式恢复

手动控制恢复到哪个归档日志。

```sql
RUN {
    STARTUP MOUNT;
    RESTORE DATABASE;
    RECOVER DATABASE UNTIL CANCEL;
    -- RMAN 会提示应用哪个归档日志
    -- 输入 CANCEL 停止
    ALTER DATABASE OPEN RESETLOGS;
}
```

## 4. 控制文件恢复

### 4.1 所有控制文件丢失

**场景**：所有控制文件损坏或丢失。

```sql
-- 步骤 1：启动到 NOMOUNT
STARTUP NOMOUNT;

-- 步骤 2：从自动备份恢复控制文件
RESTORE CONTROLFILE FROM AUTOBACKUP;

-- 步骤 3：挂载数据库
ALTER DATABASE MOUNT;

-- 步骤 4：恢复数据库
RESTORE DATABASE;
RECOVER DATABASE;

-- 步骤 5：打开数据库
ALTER DATABASE OPEN RESETLOGS;
```

### 4.2 部分控制文件丢失

```sql
-- 复制完好的控制文件
cp /u01/oradata/control01.ctl /u01/oradata/control02.ctl

-- 或从备份恢复
RESTORE CONTROLFILE FROM '/backup/controlfile_backup';
```

## 5. 重做日志恢复

### 5.1 当前日志组成员丢失

```sql
-- 清除损坏的日志组
ALTER DATABASE CLEAR LOGFILE GROUP 1;

-- 如果未归档
ALTER DATABASE CLEAR UNARCHIVED LOGFILE GROUP 1;
```

### 5.2 所有日志组成员丢失

**严重情况**：可能需要不完全恢复。

```sql
-- 尝试恢复
RECOVER DATABASE UNTIL CANCEL;

-- 如果无法恢复，可能需要从备份恢复
```

## 6. 块介质恢复（BMR）

### 6.1 什么是 BMR？

Block Media Recovery：只恢复损坏的数据块，不需要恢复整个数据文件。

**优势**：
- ✅ 恢复速度快
- ✅ 数据文件保持在线
- ✅ 最小化停机时间

### 6.2 检测块损坏

```sql
-- 查看块损坏
SELECT * FROM v$database_block_corruption;

-- 使用 RMAN 检查
BACKUP VALIDATE CHECK LOGICAL DATABASE;
```

### 6.3 执行块恢复

```sql
-- 恢复特定块
RUN {
    BLOCKRECOVER DATAFILE 5 BLOCK 10, 20, 30;
}

-- 恢复所有标记为损坏的块
BLOCKRECOVER CORRUPTION LIST;

-- 从特定备份恢复
BLOCKRECOVER DATAFILE 5 BLOCK 10 FROM BACKUPSET 123;
```

### 6.4 自动块恢复

Oracle 11g+ 支持自动检测和修复。

```sql
-- 启用自动块恢复
CONFIGURE DEVICE TYPE DISK BACKUP TYPE TO BACKUPSET;

-- 当检测到损坏时，自动从镜像副本或备份恢复
```

## 7. SPFILE 恢复

### 7.1 从自动备份恢复

```sql
-- NOMOUNT 状态下
STARTUP FORCE NOMOUNT;

-- 恢复 SPFILE
RESTORE SPFILE FROM AUTOBACKUP;

-- 重启
SHUTDOWN IMMEDIATE;
STARTUP;
```

### 7.2 恢复到指定位置

```sql
RESTORE SPFILE TO '/tmp/spfile.ora' FROM AUTOBACKUP;
```

### 7.3 从备份集恢复

```sql
RESTORE SPFILE FROM '/backup/backup_piece_name';
```

## 8. 恢复目录使用

### 8.1 连接到恢复目录

```bash
rman target / catalog rman/rman@catdb
```

### 8.2 注册数据库

```sql
-- 在 RMAN 中
REGISTER DATABASE;
```

### 8.3 同步目录

```sql
-- 同步控制文件和目录
RESYNC CATALOG;
```

### 8.4 优势

✅ **集中管理多个数据库**  
✅ **保留更长的备份历史**  
✅ **存储 RMAN 脚本**  
✅ **不依赖控制文件**

## 9. 恢复监控和诊断

### 9.1 监控恢复进度

```sql
-- 查看恢复进度
SELECT 
    sid,
    serial#,
    context,
    sofar,
    totalwork,
    ROUND(sofar/totalwork*100, 2) as pct_complete,
    elapsed_seconds,
    time_remaining
FROM v$session_longops
WHERE opname LIKE 'RMAN%'
AND totalwork != 0;
```

### 9.2 查看恢复历史

```sql
-- 恢复作业详情
SELECT 
    TO_CHAR(start_time, 'YYYY-MM-DD HH24:MI') as start_time,
    TO_CHAR(end_time, 'YYYY-MM-DD HH24:MI') as end_time,
    operation,
    status,
    input_type,
    ROUND(elapsed_seconds/60, 2) as elapsed_minutes
FROM v$rman_backup_job_details
WHERE operation = 'RESTORE' OR operation = 'RECOVER'
ORDER BY start_time DESC;
```

### 9.3 常见问题诊断

#### 问题 1：找不到备份

```sql
-- 检查备份是否存在
LIST BACKUP OF DATABASE;

-- 交叉检查
CROSSCHECK BACKUP;

-- 删除过期记录
DELETE EXPIRED BACKUP;
```

#### 问题 2：归档日志缺失

```sql
-- 查看需要的归档日志
LIST ARCHIVELOG ALL;

-- 检查是否有缺口
SELECT 
    thread#,
    sequence#,
    applied
FROM v$archived_log
WHERE sequence# BETWEEN 100 AND 200
ORDER BY sequence#;
```

#### 问题 3：恢复失败

**检查**：
- Alert log
- RMAN 输出
- 跟踪文件

**常见原因**：
- 备份不完整
- 归档日志缺失
- 空间不足
- 权限问题

## 10. 最佳实践

### 10.1 恢复准备

✅ **定期测试恢复**
- 每月至少一次
- 在测试环境验证
- 记录恢复时间

✅ **文档化恢复流程**
- 编写恢复手册
- 明确责任人
- 定期演练

✅ **监控备份状态**
- 每日检查备份日志
- 设置告警
- 验证备份可用性

### 10.2 恢复策略

📌 **优先使用块恢复**（最小影响）  
📌 **表空间级恢复优于全库恢复**  
📌 **不完全恢复前确认时间点**  
📌 **RESETLOGS 后备份立即失效**  
📌 **保留足够的归档日志**  

### 10.3 性能优化

⚡ **并行恢复**
```sql
CONFIGURE DEVICE TYPE DISK PARALLELISM 4;
```

⚡ **使用快速恢复区**  
⚡ **预分配通道**  
⚡ **优化 I/O 子系统**

## 11. 总结

### 11.1 核心知识点

✅ **完全恢复**：恢复到最新状态  
✅ **不完全恢复**：基于时间、SCN、日志序列号  
✅ **块介质恢复**：最小化停机时间  
✅ **控制文件恢复**：从自动备份  
✅ **监控诊断**：进度跟踪、问题排查  

### 11.2 关键命令

📌 `RESTORE DATABASE` - 恢复数据库  
📌 `RECOVER DATABASE` - 应用归档日志  
📌 `SET UNTIL TIME` - 设置恢复终点  
📌 `BLOCKRECOVER` - 块介质恢复  
📌 `ALTER DATABASE OPEN RESETLOGS` - 打开数据库
