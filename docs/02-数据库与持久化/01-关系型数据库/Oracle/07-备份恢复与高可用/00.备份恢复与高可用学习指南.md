### 01. RMAN基础与完全备份

**核心内容**：
- RMAN 架构和组件
- 连接到目标数据库
- 备份类型（全备、增量、差异）
- 备份集和镜像副本
- 保留策略和过期管理

**关键命令**：
```bash
# 连接 RMAN
rman target /

# 完全备份
BACKUP DATABASE PLUS ARCHIVELOG;

# 备份到指定位置
BACKUP DATABASE FORMAT '/backup/db_%U';

# 查看备份信息
LIST BACKUP SUMMARY;
LIST COPY;

# 验证备份
RESTORE DATABASE VALIDATE;
VALIDATE BACKUPSET 123;

# 删除过期备份
DELETE OBSOLETE;
DELETE EXPIRED BACKUP;
```

```sql
-- 配置保留策略
CONFIGURE RETENTION POLICY TO REDUNDANCY 2;
CONFIGURE RETENTION POLICY TO RECOVERY WINDOW OF 7 DAYS;

-- 配置并行度
CONFIGURE DEVICE TYPE DISK PARALLELISM 4;

-- 配置压缩
CONFIGURE DEVICE TYPE DISK BACKUP TYPE TO COMPRESSED BACKUPSET;
```

### 02. RMAN增量备份

**核心内容**：
- 增量备份级别（Level 0/1）
- 差异增量 vs 累积增量
- 块变化跟踪（Block Change Tracking）
- 增量备份策略设计
- 增量备份恢复

**关键命令**：
```bash
# Level 0 备份（相当于全备）
BACKUP INCREMENTAL LEVEL 0 DATABASE;

# 差异增量备份（默认）
BACKUP INCREMENTAL LEVEL 1 DATABASE;

# 累积增量备份
BACKUP INCREMENTAL LEVEL 1 CUMULATIVE DATABASE;

# 启用块变化跟踪
ALTER DATABASE ENABLE BLOCK CHANGE TRACKING 
USING FILE '/u01/app/oracle/oradata/ORCL/block_change.log';

# 禁用块变化跟踪
ALTER DATABASE DISABLE BLOCK CHANGE TRACKING;
```

**备份策略示例**：
```bash
# 周日：Level 0
RUN {
    BACKUP INCREMENTAL LEVEL 0 DATABASE;
    DELETE OBSOLETE;
}

# 周一到周六：Level 1
RUN {
    BACKUP INCREMENTAL LEVEL 1 DATABASE;
    DELETE OBSOLETE;
}
```

### 03. 时间点恢复PITR

**核心内容**：
- 不完全恢复概念
- 基于时间的恢复
- 基于 SCN 的恢复
- 基于日志序列的恢复
- 表空间时间点恢复（TSPITR）

**关键命令**：
```bash
# 基于时间的恢复
RUN {
    SET UNTIL TIME "TO_DATE('2026-05-19 14:30:00', 'YYYY-MM-DD HH24:MI:SS')";
    RESTORE DATABASE;
    RECOVER DATABASE;
    ALTER DATABASE OPEN RESETLOGS;
}

# 基于 SCN 的恢复
RUN {
    SET UNTIL SCN 1234567;
    RESTORE DATABASE;
    RECOVER DATABASE;
    ALTER DATABASE OPEN RESETLOGS;
}

# 基于日志序列的恢复
RUN {
    SET UNTIL SEQUENCE 100 THREAD 1;
    RESTORE DATABASE;
    RECOVER DATABASE;
    ALTER DATABASE OPEN RESETLOGS;
}

# 表空间时间点恢复
RECOVER TABLESPACE users UNTIL TIME "TO_DATE('2026-05-19 14:30:00', 'YYYY-MM-DD HH24:MI:SS')";
```

```sql
-- 查找恢复点
SELECT FIRST_CHANGE#, NEXT_CHANGE#, FIRST_TIME, SEQUENCE#
FROM v$archived_log
ORDER BY FIRST_TIME DESC;

-- 查看当前 SCN
SELECT CURRENT_SCN FROM v$database;
```

### 04. Data Guard物理备库

**核心内容**：
- Data Guard 架构
- 物理备库创建
- Redo 传输模式（LGWR/LNS、ARCH）
- 应用模式（实时应用、延迟应用）
- 角色转换（Switchover/Failover）

**关键配置**：
```sql
-- 主库配置
ALTER DATABASE FORCE LOGGING;
ALTER DATABASE ADD STANDBY LOGFILE GROUP 4 
    ('/u01/app/oracle/oradata/ORCL/standby_redo04.log') SIZE 50M;

-- 参数配置
ALTER SYSTEM SET LOG_ARCHIVE_CONFIG='DG_CONFIG=(primary,standby)';
ALTER SYSTEM SET LOG_ARCHIVE_DEST_2='SERVICE=standby LGWR SYNC AFFIRM VALID_FOR=(ONLINE_LOGFILES,PRIMARY_ROLE) DB_UNIQUE_NAME=standby';
ALTER SYSTEM SET FAL_SERVER=standby;
ALTER SYSTEM SET DB_FILE_NAME_CONVERT='/primary/','/standby/' SCOPE=SPFILE;
ALTER SYSTEM SET LOG_FILE_NAME_CONVERT='/primary/','/standby/' SCOPE=SPFILE;
ALTER SYSTEM SET STANDBY_FILE_MANAGEMENT=AUTO;
```

**管理命令**：
```sql
-- 启动备库
STARTUP MOUNT;
ALTER DATABASE RECOVER MANAGED STANDBY DATABASE DISCONNECT;

-- 实时应用
ALTER DATABASE RECOVER MANAGED STANDBY DATABASE USING CURRENT LOGFILE DISCONNECT;

-- 停止应用
ALTER DATABASE RECOVER MANAGED STANDBY DATABASE CANCEL;

-- 打开备库为只读
ALTER DATABASE RECOVER MANAGED STANDBY DATABASE CANCEL;
ALTER DATABASE OPEN READ ONLY;

-- Switchover（计划内切换）
-- 主库执行
ALTER DATABASE COMMIT TO SWITCHOVER TO PHYSICAL STANDBY WITH SESSION SHUTDOWN;
-- 原备库执行
ALTER DATABASE COMMIT TO SWITCHOVER TO PRIMARY;

-- Failover（故障切换）
ALTER DATABASE RECOVER MANAGED STANDBY DATABASE FINISH;
ALTER DATABASE ACTIVATE STANDBY DATABASE;
```

### 05. Data Guard逻辑备库

**核心内容**：
- 逻辑备库 vs 物理备库
- SQL Apply 机制
- 逻辑备库创建
- 跳过对象配置
- 应用场景

**关键配置**：
```sql
-- 转换为逻辑备库
ALTER DATABASE RECOVER TO LOGICAL STANDBY db_name;

-- 打开逻辑备库
ALTER DATABASE OPEN;
ALTER DATABASE START LOGICAL STANDBY APPLY IMMEDIATE;

-- 查看应用状态
SELECT * FROM V$LOGSTDBY_PROCESS;
SELECT * FROM V$LOGSTDBY_STATS;

-- 跳过特定对象
EXEC DBMS_LOGSTDBY.SKIP('DML', 'HR', 'TEMP_TABLE');
```

**优势**：
- 可以打开进行读写操作
- 可以创建额外的索引和物化视图
- 用于报表查询，减轻主库压力

### 06. GoldenGate数据同步

**核心内容**：
- GoldenGate 架构
- Extract 进程配置
- Replicat 进程配置
- Trail 文件管理
- 双向复制和冲突解决

**关键配置**：
```
-- Manager 配置（mgr.prm）
PORT 7809
AUTOSTART ER *
AUTORESTART ER *, RETRIES 3, WAITMINUTES 5

-- Extract 配置（ext1.prm）
EXTRACT ext1
USERID ggs_owner, PASSWORD password
EXTTRAIL ./dirdat/et
TABLE hr.employees;
TABLE hr.departments;

-- Pump 配置（pump1.prm）
EXTRACT pump1
USERID ggs_owner, PASSWORD password
RMTHOST standby_server, MGRPORT 7809
RMTTRAIL ./dirdat/rt
TABLE hr.employees;
TABLE hr.departments;

-- Replicat 配置（rep1.prm）
REPLICAT rep1
USERID ggs_owner, PASSWORD password
HANDLECOLLISIONS
MAP hr.employees, TARGET hr.employees;
MAP hr.departments, TARGET hr.departments;
```

**管理命令**：
```bash
# GGSCI 命令行
ggsci

GGSCI> INFO ALL
GGSCI> START EXTRACT ext1
GGSCI> START REPLICAT rep1
GGSCI> LAG EXTRACT ext1
GGSCI> LAG REPLICAT rep1
```

### 07. 导出导入高级用法

**核心内容**：
- Data Pump 架构
- expdp/impdp 参数详解
- 并行导出导入
- 网络链接导入
- 表空间传输
- 跨版本迁移

**关键命令**：
```bash
# 基本导出
expdp system/password DIRECTORY=dp_dir DUMPFILE=export.dmp LOGFILE=export.log SCHEMAS=hr

# 并行导出
expdp system/password DIRECTORY=dp_dir DUMPFILE=export_%U.dmp LOGFILE=export.log \
    SCHEMAS=hr PARALLEL=4

# 按表导出
expdp system/password DIRECTORY=dp_dir DUMPFILE=tables.dmp \
    TABLES=hr.employees,hr.departments

# 按条件导出
expdp system/password DIRECTORY=dp_dir DUMPFILE=filtered.dmp \
    TABLES=hr.employees QUERY="WHERE department_id = 10"

# 网络链接导入（无需 dump 文件）
impdp system/password NETWORK_LINK=db_link REMAP_SCHEMA=hr:hr_new

# 表空间传输
expdp system/password TRANSPORT_TABLESPACES=users_ts \
    DIRECTORY=dp_dir DUMPFILE=tts.dmp

# 导入时转换
impdp system/password DIRECTORY=dp_dir DUMPFILE=export.dmp \
    REMAP_SCHEMA=hr:hr_new \
    REMAP_TABLESPACE=users:new_users \
    TRANSFORM=SEGMENT_ATTRIBUTES:N
```

```sql
-- 创建 Directory
CREATE DIRECTORY dp_dir AS '/u01/app/oracle/dpump';
GRANT READ, WRITE ON DIRECTORY dp_dir TO system;

-- 监控 Data Pump 作业
SELECT * FROM DBA_DATAPUMP_JOBS;
SELECT * FROM DBA_DATAPUMP_SESSIONS;
```

### 08. 闪回技术全家桶

**核心内容**：
- Flashback Query
- Flashback Version Query
- Flashback Transaction Query
- Flashback Table
- Flashback Drop（回收站）
- Flashback Database

**关键知识点**：
```sql
-- 1. Flashback Query（查询历史数据）
SELECT * FROM employees AS OF TIMESTAMP 
    TO_TIMESTAMP('2026-05-19 14:30:00', 'YYYY-MM-DD HH24:MI:SS');

SELECT * FROM employees AS OF SCN 1234567;

-- 2. Flashback Version Query（查看行版本历史）
SELECT 
    versions_startscn,
    versions_endscn,
    versions_operation,
    employee_id,
    name,
    salary
FROM employees VERSIONS BETWEEN SCN MINVALUE AND MAXVALUE
WHERE employee_id = 100;

-- 3. Flashback Transaction Query（查看事务历史）
SELECT 
    xid,
    operation,
    table_name,
    undo_sql
FROM flashback_transaction_query
WHERE table_name = 'EMPLOYEES';

-- 4. Flashback Table（恢复表到过去状态）
FLASHBACK TABLE employees TO TIMESTAMP 
    TO_TIMESTAMP('2026-05-19 14:30:00', 'YYYY-MM-DD HH24:MI:SS');

FLASHBACK TABLE employees TO SCN 1234567;

-- 启用行移动（Flashback Table 前提）
ALTER TABLE employees ENABLE ROW MOVEMENT;

-- 5. Flashback Drop（恢复删除的表）
DROP TABLE employees;
FLASHBACK TABLE employees TO BEFORE DROP;
FLASHBACK TABLE employees TO BEFORE DROP RENAME TO employees_old;

-- 查看回收站
SHOW RECYCLEBIN;
SELECT * FROM RECYCLEBIN;

-- 清空回收站
PURGE RECYCLEBIN;
PURGE TABLE employees;

-- 6. Flashback Database（整个数据库回退）
-- 需要启用闪回功能
ALTER DATABASE FLASHBACK ON;

-- 闪回数据库
SHUTDOWN IMMEDIATE;
STARTUP MOUNT;
FLASHBACK DATABASE TO TIMESTAMP 
    TO_TIMESTAMP('2026-05-19 14:30:00', 'YYYY-MM-DD HH24:MI:SS');
ALTER DATABASE OPEN RESETLOGS;

-- 闪回到还原点
CREATE RESTORE POINT before_upgrade GUARANTEE FLASHBACK DATABASE;
FLASHBACK DATABASE TO RESTORE POINT before_upgrade;
```

**配置闪回**：
```sql
-- 启用闪回数据库
SHUTDOWN IMMEDIATE;
STARTUP MOUNT;
ALTER DATABASE ARCHIVELOG;
ALTER DATABASE FLASHBACK ON;
ALTER SYSTEM SET db_recovery_file_dest_size=10G;
ALTER SYSTEM SET db_recovery_file_dest='/u01/app/oracle/fast_recovery_area';
ALTER DATABASE OPEN;

-- 配置闪回保留时间
ALTER SYSTEM SET db_flashback_retention_target=1440;  -- 1440分钟 = 24小时

-- 监控闪回区使用
SELECT * FROM V$FLASH_RECOVERY_AREA_USAGE;
```
