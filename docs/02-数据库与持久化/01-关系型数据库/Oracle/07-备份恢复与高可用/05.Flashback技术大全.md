## 1. Flashback 技术概述

### 1.1 什么是 Flashback？

Flashback 是 Oracle 提供的一组技术，允许快速查看和恢复到过去的状态。

**优势**：
- ✅ 速度快（秒级恢复）
- ✅ 操作简单
- ✅ 无需备份
- ✅ 最小化停机时间

### 1.2 Flashback 技术家族

| 技术 | 粒度 | 依赖 | 用途 |
|------|------|------|------|
| **Flashback Query** | 行级 | UNDO | 查询历史数据 |
| **Flashback Version Query** | 行级 | UNDO | 查看行变更历史 |
| **Flashback Transaction Query** | 事务级 | UNDO | 查看事务变更 |
| **Flashback Table** | 表级 | UNDO | 恢复表到过去状态 |
| **Flashback Drop** | 表级 | 回收站 | 恢复删除的表 |
| **Flashback Database** | 数据库级 | Flashback Logs | 恢复整个数据库 |

## 2. Flashback Query

### 2.1 基本语法

```sql
-- 查询过去某个时间点的数据
SELECT * FROM employees AS OF TIMESTAMP 
    TO_TIMESTAMP('2026-05-19 10:00:00', 'YYYY-MM-DD HH24:MI:SS');

-- 或使用 SCN
SELECT * FROM employees AS OF SCN 1234567;

-- 计算相对时间
SELECT * FROM employees AS OF TIMESTAMP (SYSTIMESTAMP - INTERVAL '1' HOUR);
```

### 2.2 应用场景

#### 场景 1：恢复误删除的数据

```sql
-- 当前数据（已删除部分行）
SELECT COUNT(*) FROM employees;
-- 返回 95

-- 查看 1 小时前的数据
SELECT COUNT(*) FROM employees AS OF TIMESTAMP (SYSTIMESTAMP - INTERVAL '1' HOUR);
-- 返回 100

-- 恢复缺失的行
INSERT INTO employees
SELECT * FROM employees AS OF TIMESTAMP (SYSTIMESTAMP - INTERVAL '1' HOUR)
WHERE employee_id NOT IN (SELECT employee_id FROM employees);
```

#### 场景 2：数据对比

```sql
-- 对比当前和昨天的数据
SELECT 
    curr.employee_id,
    curr.salary as current_salary,
    prev.salary as yesterday_salary,
    curr.salary - prev.salary as difference
FROM employees curr,
     employees AS OF TIMESTAMP (SYSTIMESTAMP - INTERVAL '1' DAY) prev
WHERE curr.employee_id = prev.employee_id
AND curr.salary != prev.salary;
```

### 2.3 限制

⚠️ **依赖 UNDO 保留时间**  
⚠️ **不能跨越 DDL 操作**  
⚠️ **系统表不支持**

## 3. Flashback Version Query

### 3.1 查看行的变更历史

```sql
SELECT 
    versions_startscn,
    versions_endscn,
    versions_starttime,
    versions_endtime,
    versions_xid,
    versions_operation,
    employee_id,
    salary
FROM employees VERSIONS BETWEEN TIMESTAMP 
    TO_TIMESTAMP('2026-05-19 08:00:00', 'YYYY-MM-DD HH24:MI:SS')
    AND TO_TIMESTAMP('2026-05-19 12:00:00', 'YYYY-MM-DD HH24:MI:SS')
WHERE employee_id = 100
ORDER BY versions_starttime;
```

**输出示例**：
```
VERSIONS_OPERATION: I (Insert), U (Update), D (Delete)
```

### 3.2 结合 Flashback Transaction Query

```sql
-- 查找事务的详细信息
SELECT 
    xid,
    operation,
    table_name,
    row_id,
    undo_sql
FROM flashback_transaction_query
WHERE xid = HEXTORAW('0A0B000C0D0E0F10');
```

**UNDO_SQL**：可以生成回滚该操作的 SQL。

## 4. Flashback Table

### 4.1 恢复表到过去状态

```sql
-- 启用行移动（必须）
ALTER TABLE employees ENABLE ROW MOVEMENT;

-- 恢复表到 1 小时前
FLASHBACK TABLE employees TO TIMESTAMP (SYSTIMESTAMP - INTERVAL '1' HOUR);

-- 或恢复到特定 SCN
FLASHBACK TABLE employees TO SCN 1234567;

-- 或多个表
FLASHBACK TABLE employees, departments 
TO TIMESTAMP TO_TIMESTAMP('2026-05-19 10:00:00', 'YYYY-MM-DD HH24:MI:SS');
```

### 4.2 注意事项

✅ **自动恢复相关索引、触发器**  
✅ **保持表在线**  
❌ **不能跨越 DDL（如 TRUNCATE）**  
❌ **需要足够的 UNDO**

## 5. Flashback Drop

### 5.1 回收站机制

删除的表进入回收站，可以快速恢复。

```sql
-- 删除表（进入回收站）
DROP TABLE employees;

-- 查看回收站
SHOW RECYCLEBIN;

-- 或
SELECT * FROM user_recyclebin;

-- 恢复表
FLASHBACK TABLE employees TO BEFORE DROP;

-- 恢复并重命名
FLASHBACK TABLE employees TO BEFORE DROP RENAME TO employees_old;

-- 永久删除（不进入回收站）
DROP TABLE employees PURGE;

-- 清空回收站
PURGE RECYCLEBIN;
PURGE TABLE employees;  -- 清空特定表
```

### 5.2 回收站管理

```sql
-- 禁用回收站
ALTER SYSTEM SET recyclebin = OFF SCOPE=SPFILE;

-- 启用回收站
ALTER SYSTEM SET recyclebin = ON SCOPE=SPFILE;
```

## 6. Flashback Database

### 6.1 启用 Flashback Database

```sql
-- 步骤 1：设置 FRA
ALTER SYSTEM SET db_recovery_file_dest = '/u01/fra';
ALTER SYSTEM SET db_recovery_file_dest_size = 100G;

-- 步骤 2：设置保留时间（分钟）
ALTER SYSTEM SET db_flashback_retention_target = 1440;  -- 24 小时

-- 步骤 3：启用 Flashback
SHUTDOWN IMMEDIATE;
STARTUP MOUNT;
ALTER DATABASE FLASHBACK ON;
ALTER DATABASE OPEN;

-- 验证
SELECT flashback_on FROM v$database;
-- 应该返回：YES
```

### 6.2 执行 Flashback Database

```sql
-- 步骤 1：关闭数据库
SHUTDOWN IMMEDIATE;
STARTUP MOUNT;

-- 步骤 2：Flashback
FLASHBACK DATABASE TO TIMESTAMP 
    TO_TIMESTAMP('2026-05-19 10:00:00', 'YYYY-MM-DD HH24:MI:SS');

-- 或 SCN
FLASHBACK DATABASE TO SCN 1234567;

-- 步骤 3：打开数据库
ALTER DATABASE OPEN RESETLOGS;
```

**注意**：`RESETLOGS` 会重置日志序列号。

### 6.3 监控 Flashback

```sql
-- 查看 Flashback 日志信息
SELECT 
    estimated_flashback_size,
    flashback_size,
    oldest_flashback_scn,
    oldest_flashback_time
FROM v$flashback_database_log;

-- 查看 Flashback 统计
SELECT * FROM v$flashback_database_stat;
```

## 7. Flashback 配置和管理

### 7.1 UNDO 配置（用于 Flashback Query/Table）

```sql
-- 设置 UNDO 保留时间
ALTER SYSTEM SET undo_retention = 86400;  -- 24 小时

-- 确保 UNDO 表空间足够大
ALTER DATABASE DATAFILE '/u01/oradata/undotbs01.dbf' RESIZE 10G;
AUTOEXTEND ON NEXT 1G MAXSIZE 50G;

-- 使用 GUARANTEE（可选）
ALTER TABLESPACE undotbs1 RETENTION GUARANTEE;
```

### 7.2 Flashback Log 配置

```sql
-- 查看 Flashback 日志大小
SELECT 
    ROUND(SUM(bytes)/1024/1024/1024, 2) as size_gb
FROM v$flashback_database_logfile;

-- Flashback 日志自动管理，无需手动干预
```

## 8. 实战案例

### 案例 1：误更新数据

```sql
-- 场景：错误更新了所有员工的工资
UPDATE employees SET salary = salary * 10;
COMMIT;

-- 解决方案 1：Flashback Query
INSERT INTO employees_backup
SELECT * FROM employees AS OF TIMESTAMP (SYSTIMESTAMP - INTERVAL '5' MINUTE);

-- 然后手动恢复

-- 解决方案 2：Flashback Table
ALTER TABLE employees ENABLE ROW MOVEMENT;
FLASHBACK TABLE employees TO TIMESTAMP (SYSTIMESTAMP - INTERVAL '5' MINUTE);
```

### 案例 2：误删除表

```sql
-- 场景：DROP TABLE employees;

-- 解决方案：Flashback Drop
FLASHBACK TABLE employees TO BEFORE DROP;

-- 如果回收站已清空，使用 TSPITR 或从备份恢复
```

### 案例 3：应用升级失败

```sql
-- 场景：升级后数据混乱，需要回滚到升级前

-- 解决方案：Flashback Database
SHUTDOWN IMMEDIATE;
STARTUP MOUNT;
FLASHBACK DATABASE TO TIMESTAMP 
    TO_TIMESTAMP('2026-05-19 07:00:00', 'YYYY-MM-DD HH24:MI:SS');
ALTER DATABASE OPEN RESETLOGS;
```

## 9. Flashback vs 其他技术

### 9.1 对比表

| 技术 | 速度 | 粒度 | 依赖 | 适用场景 |
|------|------|------|------|---------|
| **Flashback Query** | 秒级 | 行级 | UNDO | 查看历史数据 |
| **Flashback Table** | 分钟级 | 表级 | UNDO | 恢复单个表 |
| **Flashback Database** | 分钟级 | 数据库级 | Flashback Logs | 快速回滚 |
| **RMAN PITR** | 小时级 | 表空间级 | 备份+归档 | 大范围恢复 |
| **TSPITR** | 小时级 | 表空间级 | 备份+归档 | 表空间恢复 |

### 9.2 选择指南

**使用 Flashback**：
- 短时间内的错误（UNDO/Flashback Logs 可用）
- 需要快速恢复
- 小范围错误

**使用 RMAN/TSPITR**：
- 长时间之前的错误
- Flashback 不可用
- 大范围损坏

## 10. 最佳实践

### 10.1 配置建议

✅ **启用 Flashback Database**（生产环境）  
✅ **设置合理的 UNDO_RETENTION**  
✅ **监控 FRA 空间**  
✅ **定期测试 Flashback**  
✅ **记录 Flashback 操作**  

### 10.2 使用建议

📌 **优先使用 Flashback**（速度快）  
📌 **Flashback 失败再用 RMAN**  
📌 **重要操作前创建 Restore Point**  
📌 **监控 UNDO 和 Flashback Logs**  
📌 **培训开发人员使用 Flashback Query**  

### 10.3 Restore Point

```sql
-- 创建正常 Restore Point
CREATE RESTORE POINT before_upgrade;

-- 创建保证型 Restore Point（即使 Flashback 关闭也可用）
CREATE RESTORE POINT before_upgrade GUARANTEE FLASHBACK DATABASE;

-- 查看所有 Restore Points
SELECT * FROM v$restore_point;

-- Flashback 到 Restore Point
FLASHBACK DATABASE TO RESTORE POINT before_upgrade;

-- 删除 Restore Point
DROP RESTORE POINT before_upgrade;
```
