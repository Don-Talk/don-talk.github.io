## 1. RMAN 概述

### 1.1 什么是 RMAN？

RMAN（Recovery Manager）是 Oracle 提供的企业级备份和恢复工具。

**特点**：
- ✅ 集成在 Oracle 数据库中
- ✅ 支持增量备份
- ✅ 自动管理备份文件
- ✅ 支持压缩和加密
- ✅ 块级别损坏检测

### 1.2 RMAN vs 用户管理备份

| 特性 | RMAN | 用户管理备份 |
|------|------|------------|
| **备份类型** | 物理备份 | 物理备份 |
| **增量备份** | ✅ 支持 | ❌ 不支持 |
| **块级别恢复** | ✅ 支持 | ❌ 不支持 |
| **自动化** | ✅ 高 | ❌ 手动 |
| **元数据管理** | ✅ 控制文件/目录 | ❌ 手动记录 |
| **压缩** | ✅ 内置 | ❌ 需外部工具 |
| **并行化** | ✅ 自动 | ❌ 手动配置 |

## 2. RMAN 架构

### 2.1 组件

```
RMAN Client
    ↓
Channel（通道）
    ↓
Server Session（服务器会话）
    ↓
Database（数据库）
```

**关键组件**：
- **RMAN Client**：命令行或图形界面
- **Channel**：到数据库的连接，执行 I/O
- **Target Database**：要备份的数据库
- **Recovery Catalog**（可选）：存储备份元数据

### 2.2 连接 RMAN

```bash
# 连接到目标数据库
rman target /

# 连接到目标数据库和恢复目录
rman target / catalog rman/rman@catdb

# 连接到辅助数据库（用于复制）
rman target / auxiliary sys/password@auxdb
```

## 3. 备份类型

### 3.1 全量备份（Full Backup）

备份所有使用的数据块。

```sql
-- 全库备份
BACKUP DATABASE;

-- 备份到指定位置
BACKUP DATABASE FORMAT '/backup/full_%U';

-- 带标签
BACKUP DATABASE TAG 'weekly_full';
```

**特点**：
- ✅ 完整的备份
- ✅ 恢复简单
- ❌ 耗时较长
- ❌ 占用空间大

### 3.2 增量备份（Incremental Backup）

只备份自上次备份以来变化的块。

#### （1）累积增量备份（Cumulative）

备份自上次 0 级备份以来的所有变化。

```sql
-- 0 级备份（相当于全量）
BACKUP INCREMENTAL LEVEL 0 DATABASE;

-- 1 级累积备份
BACKUP INCREMENTAL LEVEL 1 CUMULATIVE DATABASE;
```

#### （2）差异增量备份（Differential）

备份自上次任何级别备份以来的变化。

```sql
-- 1 级差异备份（默认）
BACKUP INCREMENTAL LEVEL 1 DATABASE;
```

**对比**：

| 类型 | 备份内容 | 恢复速度 | 备份时间 |
|------|---------|---------|---------|
| **Level 0** | 所有块 | 快 | 长 |
| **Level 1 Cumulative** | 自 L0 以来的变化 | 较快 | 中等 |
| **Level 1 Differential** | 自上次备份以来的变化 | 较慢 | 短 |

### 3.3 表空间备份

```sql
-- 备份特定表空间
BACKUP TABLESPACE users, app_data;

-- 备份系统表空间
BACKUP TABLESPACE system, sysaux;
```

### 3.4 数据文件备份

```sql
-- 备份特定数据文件
BACKUP DATAFILE 1, 2, 3;

-- 按文件名
BACKUP DATAFILE '/u01/oradata/users01.dbf';
```

### 3.5 归档日志备份

```sql
-- 备份所有归档日志
BACKUP ARCHIVELOG ALL;

-- 备份后删除
BACKUP ARCHIVELOG ALL DELETE INPUT;

-- 备份最近 7 天的归档日志
BACKUP ARCHIVELOG FROM TIME 'SYSDATE-7';
```

### 3.6 控制文件和 SPFILE 备份

```sql
-- 自动备份（推荐）
CONFIGURE CONTROLFILE AUTOBACKUP ON;

-- 手动备份
BACKUP CURRENT CONTROLFILE;
BACKUP SPFILE;
```

## 4. 备份格式和位置

### 4.1 备份集（Backup Set）

RMAN 默认的备份格式，多个文件打包成一个备份集。

```sql
-- 备份到快速恢复区（FRA）
BACKUP DATABASE;

-- 备份到指定位置
BACKUP DATABASE FORMAT '/backup/db_%U';
```

**格式变量**：
- `%U`：唯一文件名
- `%d`：数据库名
- `%t`：时间戳
- `%s`：备份集号

### 4.2 镜像副本（Image Copy）

数据文件的精确副本。

```sql
-- 创建镜像副本
BACKUP AS COPY DATABASE;

-- 创建特定文件的副本
BACKUP AS COPY DATAFILE 1 FORMAT '/backup/users_copy.dbf';
```

**特点**：
- ✅ 可以直接使用
- ✅ 恢复速度快
- ❌ 占用空间大（无压缩）

### 4.3 快速恢复区（FRA）

```sql
-- 配置 FRA
ALTER SYSTEM SET db_recovery_file_dest = '/u01/fra';
ALTER SYSTEM SET db_recovery_file_dest_size = 100G;

-- 查看 FRA 使用
SELECT 
    name,
    ROUND(space_limit/1024/1024/1024, 2) as limit_gb,
    ROUND(space_used/1024/1024/1024, 2) as used_gb,
    ROUND(space_reclaimable/1024/1024/1024, 2) as reclaimable_gb
FROM v$recovery_file_dest;
```

## 5. 备份优化

### 5.1 压缩备份

```sql
-- 启用压缩
CONFIGURE DEVICE TYPE DISK BACKUP TYPE TO COMPRESSED BACKUPSET;

-- 或单次备份使用压缩
BACKUP AS COMPRESSED BACKUPSET DATABASE;
```

**压缩级别**：
- **BASIC**：免费，中等压缩率
- **LOW/MEDIUM/HIGH**：需要 Advanced Compression Option

### 5.2 并行备份

```sql
-- 配置并行通道
CONFIGURE DEVICE TYPE DISK PARALLELISM 4;

-- 或使用 RUN 块
RUN {
    ALLOCATE CHANNEL c1 DEVICE TYPE DISK;
    ALLOCATE CHANNEL c2 DEVICE TYPE DISK;
    ALLOCATE CHANNEL c3 DEVICE TYPE DISK;
    ALLOCATE CHANNEL c4 DEVICE TYPE DISK;
    BACKUP DATABASE;
}
```

### 5.3 备份优化

```sql
-- 跳过已备份且未变化的文件
CONFIGURE BACKUP OPTIMIZATION ON;

-- 跳过只读和离线表空间
CONFIGURE SKIP READONLY ON;
CONFIGURE SKIP OFFLINE ON;
```

### 5.4 块变化跟踪（Block Change Tracking）

加速增量备份。

```sql
-- 启用块变化跟踪
ALTER DATABASE ENABLE BLOCK CHANGE TRACKING 
USING FILE '/u01/oradata/bct.dbf';

-- 查看状态
SELECT status, filename FROM v$block_change_tracking;

-- 禁用
ALTER DATABASE DISABLE BLOCK CHANGE TRACKING;
```

**效果**：
- 增量备份速度提升 5-10 倍
- 额外空间：约数据库大小的 1/30000

## 6. 备份策略

### 6.1 典型备份策略

#### 策略 1：每周全量 + 每日增量

```sql
-- 周日：0 级备份
BACKUP INCREMENTAL LEVEL 0 DATABASE TAG 'weekly_full';

-- 周一至周六：1 级差异备份
BACKUP INCREMENTAL LEVEL 1 DATABASE TAG 'daily_incr';

-- 每天：归档日志备份
BACKUP ARCHIVELOG ALL DELETE INPUT;
```

#### 策略 2：Oracle 建议策略

```sql
-- 配置保留策略
CONFIGURE RETENTION POLICY TO RECOVERY WINDOW OF 7 DAYS;

-- 每周日：全量
BACKUP DATABASE TAG 'weekly_full';

-- 每天：增量
BACKUP INCREMENTAL LEVEL 1 DATABASE TAG 'daily_incr';

-- 每小时：归档日志
BACKUP ARCHIVELOG ALL NOT BACKED UP DELETE INPUT;
```

### 6.2 保留策略

```sql
-- 基于时间的保留
CONFIGURE RETENTION POLICY TO RECOVERY WINDOW OF 14 DAYS;

-- 基于冗余度的保留
CONFIGURE RETENTION POLICY TO REDUNDANCY 2;

-- 查看过期的备份
REPORT OBSOLETE;

-- 删除过期的备份
DELETE OBSOLETE;
```

## 7. 备份监控和维护

### 7.1 查看备份信息

```sql
-- 查看备份摘要
LIST BACKUP SUMMARY;

-- 查看数据库备份
LIST BACKUP OF DATABASE;

-- 查看归档日志备份
LIST BACKUP OF ARCHIVELOG ALL;

-- 查看过期备份
LIST EXPIRED BACKUP;
```

### 7.2 验证备份

```sql
-- 验证备份完整性
RESTORE DATABASE VALIDATE;

-- 验证特定备份集
VALIDATE BACKUPSET <backup_set_id>;

-- 检查物理和逻辑损坏
BACKUP VALIDATE CHECK LOGICAL DATABASE;
```

### 7.3 交叉检查

```sql
-- 检查备份文件是否存在
CROSSCHECK BACKUP;

-- 删除不存在的备份记录
DELETE EXPIRED BACKUP;
```

### 7.4 监控脚本

```sql
-- 备份历史
SELECT 
    TO_CHAR(start_time, 'YYYY-MM-DD HH24:MI') as start_time,
    TO_CHAR(end_time, 'YYYY-MM-DD HH24:MI') as end_time,
    input_type,
    status,
    ROUND(elapsed_seconds/60, 2) as elapsed_minutes,
    ROUND(output_bytes/1024/1024/1024, 2) as output_gb
FROM v$rman_backup_job_details
ORDER BY start_time DESC;

-- 备份进度
SELECT 
    sid,
    serial#,
    context,
    sofar,
    totalwork,
    ROUND(sofar/totalwork*100, 2) as pct_complete
FROM v$session_longops
WHERE opname LIKE 'RMAN%'
AND totalwork != 0;
```

## 8. 最佳实践

### 8.1 备份设计原则

✅ **3-2-1 规则**
- 至少 3 份备份
- 至少 2 种不同介质
- 至少 1 份异地备份

✅ **定期测试恢复**
- 每月测试一次
- 验证备份可用性
- 记录恢复时间

✅ **监控备份状态**
- 每日检查备份日志
- 设置告警机制
- 监控 FRA 空间

### 8.2 性能优化

📌 **使用块变化跟踪**  
📌 **配置适当的并行度**  
📌 **使用压缩节省空间**  
📌 **避开业务高峰期**  
📌 **定期清理过期备份**  

### 8.3 安全考虑

⚠️ **加密敏感备份**
```sql
CONFIGURE ENCRYPTION FOR DATABASE ON;
CONFIGURE ENCRYPTION ALGORITHM 'AES256';
```

⚠️ **保护恢复目录**
⚠️ **限制 RMAN 访问权限**
⚠️ **审计备份操作**

## 9. 常见问题

### 9.1 备份失败

**检查**：
```sql
-- 查看错误信息
LIST BACKUP FAILED;

-- 检查 alert log
-- 检查 RMAN 输出
```

**常见原因**：
- 空间不足
- 权限问题
- 网络中断
- 数据库状态异常

### 9.2 FRA 空间不足

**解决**：
```sql
-- 增加 FRA 大小
ALTER SYSTEM SET db_recovery_file_dest_size = 200G;

-- 清理过期备份
DELETE OBSOLETE;
DELETE EXPIRED BACKUP;

-- 备份到其他地方
BACKUP RECOVERY AREA TO DESTINATION '/backup';
```

### 9.3 备份速度慢

**优化**：
```sql
-- 增加并行度
CONFIGURE DEVICE TYPE DISK PARALLELISM 8;

-- 启用块变化跟踪
ALTER DATABASE ENABLE BLOCK CHANGE TRACKING;

-- 使用压缩
CONFIGURE DEVICE TYPE DISK BACKUP TYPE TO COMPRESSED BACKUPSET;
```
