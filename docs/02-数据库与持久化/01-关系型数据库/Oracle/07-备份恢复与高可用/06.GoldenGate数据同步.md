## 1. GoldenGate 概述

### 1.1 什么是 GoldenGate？

Oracle GoldenGate（OGG）是业界领先的数据复制和集成平台。

**核心功能**：
- ✅ 实时数据复制
- ✅ 异构数据库同步
- ✅ 零停机迁移
- ✅ 读写分离
- ✅ 数据集成

### 1.2 GoldenGate vs Data Guard

| 特性 | GoldenGate | Data Guard |
|------|-----------|------------|
| **复制类型** | 逻辑复制 | 物理复制 |
| **数据库类型** | 异构支持 | 仅 Oracle |
| **粒度** | 表/列级 | 数据库级 |
| **延迟** | 秒级 | 毫秒级 |
| **备库可写** | ✅ 支持 | ❌ 只读（Active DG） |
| **过滤转换** | ✅ 支持 | ❌ 不支持 |
| **复杂度** | 高 | 低 |

### 1.3 应用场景

📊 **数据仓库同步**  
📊 **读写分离**  
📊 **零停机升级/迁移**  
📊 **多活数据中心**  
📊 **数据集成**

## 2. GoldenGate 架构

### 2.1 核心组件

```
Source Database
    ↓
Extract Process（捕获进程）
    ↓
Trail Files（队列文件）
    ↓
Data Pump（可选，传输进程）
    ↓
Network Transfer
    ↓
Replicat Process（应用进程）
    ↓
Target Database
```

### 2.2 进程类型

#### （1）Extract（捕获进程）

- 从源数据库读取 redo/archivelog
- 提取 DML/DDL 操作
- 写入本地 Trail 文件

#### （2）Data Pump（传输进程）

- 读取本地 Trail 文件
- 通过网络传输到目标端
- 可选组件

#### （3）Replicat（应用进程）

- 读取远程 Trail 文件
- 在目标数据库重放操作
- 保持数据一致性

### 2.3 Trail 文件

- 二进制文件，存储捕获的操作
- 按顺序处理，保证事务一致性
- 自动管理，可配置保留策略

## 3. 安装和配置

### 3.1 安装步骤

```bash
# 1. 下载 GoldenGate 软件
# 2. 解压
unzip ggs_Linux_x64.zip -d /u01/goldengate

# 3. 设置环境变量
export GG_HOME=/u01/goldengate
export LD_LIBRARY_PATH=$GG_HOME:$LD_LIBRARY_PATH
export PATH=$GG_HOME:$PATH

# 4. 创建子目录
cd $GG_HOME
./ggsci
GGSCI> CREATE SUBDIRS
```

### 3.2 源数据库配置

```sql
-- 启用归档模式
ARCHIVE LOG LIST;
ALTER DATABASE ARCHIVELOG;

-- 启用补充日志
ALTER DATABASE ADD SUPPLEMENTAL LOG DATA;
ALTER DATABASE ADD SUPPLEMENTAL LOG DATA (PRIMARY KEY) COLUMNS;

-- 创建 GoldenGate 用户
CREATE USER ggate IDENTIFIED BY password;
GRANT CONNECT, RESOURCE TO ggate;
GRANT SELECT ANY DICTIONARY, SELECT ANY TABLE TO ggate;
GRANT ALTER ANY TABLE TO ggate;
GRANT FLASHBACK ANY TABLE TO ggate;
GRANT EXECUTE ON DBMS_FLASHBACK TO ggate;
```

### 3.3 目标数据库配置

```sql
-- 创建 GoldenGate 用户
CREATE USER ggate IDENTIFIED BY password;
GRANT CONNECT, RESOURCE TO ggate;
GRANT ALTER ANY TABLE, INSERT ANY TABLE, UPDATE ANY TABLE, DELETE ANY TABLE TO ggate;
```

## 4. 单向复制配置

### 4.1 配置 Extract

```bash
# 进入 GGSCI
cd $GG_HOME
./ggsci

# 添加 Extract 组
GGSCI> ADD EXTRACT ext1, TRANLOG, BEGIN NOW

# 添加本地 Trail
GGSCI> ADD EXTTRAIL ./dirdat/lt, EXTRACT ext1, MEGABYTES 100

# 编辑 Extract 参数
GGSCI> EDIT PARAMS ext1
```

**Extract 参数文件**：
```
EXTRACT ext1
USERID ggate@source_db, PASSWORD password
TRANLOGOPTIONS INTEGRATEDPARAMS (max_sga_size 256)
EXTTRAIL ./dirdat/lt

-- 指定要复制的表
TABLE hr.employees;
TABLE hr.departments;
TABLE sales.orders;
```

### 4.2 配置 Data Pump（可选）

```bash
# 添加 Data Pump
GGSCI> ADD EXTRACT dpump1, EXTTRAILSOURCE ./dirdat/lt

# 添加远程 Trail
GGSCI> ADD RMTTRAIL ./dirdat/rt, EXTRACT dpump1, MEGABYTES 100

# 编辑参数
GGSCI> EDIT PARAMS dpump1
```

**Data Pump 参数文件**：
```
EXTRACT dpump1
USERID ggate@source_db, PASSWORD password
RMTHOST target_host, MGRPORT 7809
RMTTRAIL ./dirdat/rt
PASSTHRU

TABLE hr.*;
TABLE sales.*;
```

### 4.3 配置 Replicat

```bash
# 在目标端
GGSCI> ADD REPLICAT rep1, EXTTRAIL ./dirdat/rt

# 编辑参数
GGSCI> EDIT PARAMS rep1
```

**Replicat 参数文件**：
```
REPLICAT rep1
USERID ggate@target_db, PASSWORD password
ASSUMETARGETDEFS
HANDLECOLLISIONS

-- 映射表
MAP hr.*, TARGET hr.*;
MAP sales.*, TARGET sales.*;
```

### 4.4 启动进程

```bash
# 源端
GGSCI> START EXTRACT ext1
GGSCI> START EXTRACT dpump1

# 目标端
GGSCI> START REPLICAT rep1

# 查看状态
GGSCI> INFO ALL
```

## 6. 监控和维护

### 6.1 监控命令

```bash
# 查看所有进程状态
GGSCI> INFO ALL

# 查看特定进程
GGSCI> INFO EXTRACT ext1, DETAIL
GGSCI> INFO REPLICAT rep1, DETAIL

# 查看延迟
GGSCI> LAG EXTRACT ext1
GGSCI> LAG REPLICAT rep1

# 查看统计信息
GGSCI> STATS EXTRACT ext1
GGSCI> STATS REPLICAT rep1
```

### 6.2 监控视图

```sql
-- 源端
SELECT * FROM ggate.v$gg_extract;

-- 目标端
SELECT * FROM ggate.v$gg_replicat;
```

### 6.3 告警脚本

```bash
#!/bin/bash
# 检查 GoldenGate 进程状态

cd $GG_HOME
./ggsci << EOF
INFO ALL
EXIT
EOF | grep -E "STOPPED|ABENDED"

if [ $? -eq 0 ]; then
    echo "GoldenGate 进程异常！" | mail -s "OGG Alert" admin@example.com
fi
```

## 7. 故障排查

### 7.1 常见问题

#### 问题 1：Extract 停止

**原因**：
- 归档日志缺失
- 权限不足
- 表结构变更

**解决**：
```bash
# 查看错误日志
GGSCI> VIEW REPORT ext1

# 检查归档日志
ls -l $ORACLE_BASE/archive/

# 重新启动
GGSCI> STOP EXTRACT ext1
GGSCI> START EXTRACT ext1
```

#### 问题 2：Replicat 延迟

**原因**：
- 网络慢
- 目标库负载高
- 大事务

**解决**：
```bash
# 增加并行度
GGSCI> ALTER REPLICAT rep1, MAXPARALLEL 4

# 优化目标库性能
# 检查网络带宽
```

#### 问题 3：数据不一致

**检查**：
```bash
# 使用 Veridata（单独产品）
# 或手动对比
```

**解决**：
```bash
# 重新初始化
GGSCI> STOP REPLICAT rep1
# 重新导出数据
# 导入后重新启动
GGSCI> START REPLICAT rep1
```

### 7.2 日志分析

```bash
# Extract 日志
tail -f $GG_HOME/ggserr.log

# Replicat 报告
cat $GG_HOME/dirrpt/rep1.rpt

# Trail 文件信息
ls -lh $GG_HOME/dirdat/
```

## 8. 最佳实践

### 8.1 配置建议

✅ **使用 Integrated Extract**（12c+）  
✅ **配置 Data Pump**（网络隔离）  
✅ **启用加密**（敏感数据）  
✅ **设置告警机制**

⚡ **调整 Extract SGA**

## 9. 高级特性

### 9.1 DDL 复制

```
-- 自动复制表结构变更
DDLOPTIONS ADDTRANDATA

-- 过滤 DDL
DDL INCLUDE MAPPED OBJTYPE 'TABLE'
DDL EXCLUDE OBJNAME 'hr.temp_*'
```

### 9.2 数据转换

```
-- 列映射
MAP hr.employees, TARGET hr.emp_target,
    COLMAP (USEDEFAULTS, 
            full_name = first_name || ' ' || last_name);

-- 条件过滤
TABLE hr.employees, FILTER (salary > 5000);
```

### 9.3 异构复制

```
-- Oracle → MySQL
MAP hr.employees, TARGET test.employees;

-- Oracle → Kafka
TARGETDB LIBFILE libggjava.so SET property=dirprm/kafka.props
```
