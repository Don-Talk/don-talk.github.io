## 1. TDE 概述

### 1.1 什么是 TDE？

TDE（Transparent Data Encryption，透明数据加密）是 Oracle 提供的数据加密技术。

**特点**：
- ✅ 对应用透明
- ✅ 自动加密/解密
- ✅ 保护静态数据（Data at Rest）
- ✅ 符合合规要求
- ✅ 防止存储介质被盗

### 1.2 TDE vs 其他加密方式

| 特性 | TDE | 应用层加密 | 网络加密 |
|------|-----|-----------|---------|
| **加密位置** | 数据库层 | 应用层 | 传输层 |
| **透明度** | 完全透明 | 需修改代码 | 透明 |
| **性能影响** | 小 | 大 | 小 |
| **保护范围** | 静态数据 | 端到端 | 传输中数据 |
| **密钥管理** | 数据库管理 | 应用管理 | SSL/TLS |

### 1.3 加密层次

```
应用层（无需修改）
    ↓
SQL 层（透明加解密）
    ↓
存储引擎层
    ↓
加密的数据文件
```

## 2. TDE 架构

### 2.1 密钥层次结构

```
主密钥（Master Key）
    ↓
表密钥/表空间密钥（Table/Tablespace Key）
    ↓
加密的数据
```

**说明**：
- **主密钥**：存储在钱包中，保护表密钥
- **表密钥**：加密实际数据，被主密钥加密后存储在数据字典中
- **双层密钥**：便于密钥轮换，无需重新加密所有数据

### 2.2 Oracle Wallet

Wallet 是存储加密密钥的安全容器。

**类型**：
- **软件钱包**：文件系统存储
- **硬件安全模块（HSM）**：硬件设备存储
- **Oracle Key Vault**：集中密钥管理

## 3. 配置 TDE

### 3.1 准备工作

#### （1）创建钱包目录

```bash
# 创建钱包目录
mkdir -p /u01/app/oracle/wallet

# 设置权限
chown oracle:oinstall /u01/app/oracle/wallet
chmod 700 /u01/app/oracle/wallet
```

#### （2）配置 sqlnet.ora

```ini
# $ORACLE_HOME/network/admin/sqlnet.ora

ENCRYPTION_WALLET_LOCATION =
  (SOURCE =
    (METHOD = FILE)
    (METHOD_DATA =
      (DIRECTORY = /u01/app/oracle/wallet)
    )
  )
```

### 3.2 创建钱包和主密钥

```sql
-- 创建钱包并打开
ADMINISTER KEY MANAGEMENT CREATE KEYSTORE 
'/u01/app/oracle/wallet' 
IDENTIFIED BY "WalletPass123!";

-- 打开钱包
ADMINISTER KEY MANAGEMENT SET KEYSTORE OPEN 
IDENTIFIED BY "WalletPass123!";

-- 设置主密钥
ADMINISTER KEY MANAGEMENT SET KEY 
IDENTIFIED BY "WalletPass123!" 
WITH BACKUP USING 'backup_key_001';

-- 验证钱包状态
SELECT * FROM v$encryption_wallet;
```

**钱包状态**：
- **OPEN_NO_MASTER_KEY**：钱包打开但无主密钥
- **OPEN**：钱包打开且有主密钥
- **CLOSED**：钱包关闭
- **NOT_AVAILABLE**：钱包不可用

### 3.3 自动打开钱包

```sql
-- 配置钱包自动打开（需要额外配置）
-- 在 sqlnet.ora 中添加
WALLET_ROOT = /u01/app/oracle/wallet
TDE_CONFIGURATION = KEYSTORE_CONFIGURATION=FILE

-- 或使用 Oracle Key Vault 集中管理
```

## 4. 表空间加密

### 4.1 创建加密表空间

```sql
-- 创建加密表空间
CREATE TABLESPACE encrypted_ts
DATAFILE '/u01/oradata/encrypted_ts01.dbf'
SIZE 1G
ENCRYPTION USING 'AES256'
DEFAULT STORAGE(ENCRYPT);

-- 验证
SELECT 
    tablespace_name,
    encrypted,
    encryption_alg
FROM dba_tablespaces
WHERE tablespace_name = 'ENCRYPTED_TS';
```

**加密算法**：
- **AES128**：128 位密钥
- **AES192**：192 位密钥
- **AES256**：256 位密钥（推荐）
- **3DES168**：三重 DES（已过时）

### 4.2 移动表到加密表空间

```sql
-- 移动现有表
ALTER TABLE hr.employees MOVE TABLESPACE encrypted_ts;

-- 重建索引
ALTER INDEX hr.emp_pk REBUILD TABLESPACE encrypted_ts;

-- 验证
SELECT 
    segment_name,
    tablespace_name,
    encrypted
FROM dba_segments
WHERE segment_name = 'EMPLOYEES';
```

### 4.3 加密整个数据库

```sql
-- 步骤 1：创建加密表空间
CREATE TABLESPACE enc_system ENCRYPTION USING 'AES256' DEFAULT STORAGE(ENCRYPT);
CREATE TABLESPACE enc_sysaux ENCRYPTION USING 'AES256' DEFAULT STORAGE(ENCRYPT);
CREATE TABLESPACE enc_users ENCRYPTION USING 'AES256' DEFAULT STORAGE(ENCRYPT);

-- 步骤 2：移动所有对象
-- SYSTEM、SYSAUX 需要在 MOUNT 状态下操作
SHUTDOWN IMMEDIATE;
STARTUP MOUNT;

-- 使用 RMAN 转换
RMAN> CONVERT TABLESPACE SYSTEM TO ENCRYPTED TABLESPACE;
RMAN> CONVERT TABLESPACE SYSAUX TO ENCRYPTED TABLESPACE;

ALTER DATABASE OPEN;

-- 步骤 3：移动用户表空间
ALTER TABLESPACE users ENCRYPTION ONLINE USING 'AES256' ENCRYPT;
```

## 5. 列级加密

### 5.1 创建加密列

```sql
-- 创建包含加密列的表
CREATE TABLE hr.employee_sensitive (
    employee_id NUMBER PRIMARY KEY,
    first_name VARCHAR2(50),
    last_name VARCHAR2(50),
    ssn VARCHAR2(11) ENCRYPT USING 'AES256' NO SALT,
    bank_account VARCHAR2(20) ENCRYPT USING 'AES256' SALT,
    salary NUMBER(10,2) ENCRYPT
);
```

**加密选项**：
- **NO SALT**：不使用盐值（可索引，相同值加密结果相同）
- **SALT**：使用盐值（更安全，相同值加密结果不同，默认）
- **NOMAC**：不使用消息认证码（性能更好，安全性略低）

### 5.2 添加加密列

```sql
-- 为现有表添加加密列
ALTER TABLE hr.employees 
ADD (ssn VARCHAR2(11) ENCRYPT USING 'AES256');

-- 修改列为加密
ALTER TABLE hr.employees 
MODIFY (phone_number ENCRYPT USING 'AES256');
```

### 5.3 加密列的限制

⚠️ **不能创建外键约束**  
⚠️ **不能使用物化视图日志**  
⚠️ **索引限制**：只能对 NO SALT 列创建索引  
⚠️ **数据类型限制**：不支持 LOB、LONG 等  

### 5.4 查看加密列

```sql
-- 查看加密列
SELECT 
    owner,
    table_name,
    column_name,
    encryption_alg,
    salt,
    mac
FROM dba_encrypted_columns
WHERE owner = 'HR';
```

## 6. 密钥管理

### 6.1 密钥轮换

```sql
-- 轮换主密钥（重新加密所有表密钥）
ADMINISTER KEY MANAGEMENT SET KEY 
IDENTIFIED BY "WalletPass123!" 
FORCE KEYSTORE 
WITH BACKUP USING 'backup_key_002';

-- 验证
SELECT key_id, creation_time, activating_time
FROM v$encryption_keys
ORDER BY creation_time DESC;
```

**注意**：密钥轮换会重新加密所有表密钥，可能耗时较长。

### 6.2 导出和导入钱包

```bash
# 导出钱包（备份）
cp /u01/app/oracle/wallet/ewallet.p12 /backup/wallet_backup.p12

# 导入钱包（恢复）
cp /backup/wallet_backup.p12 /u01/app/oracle/wallet/ewallet.p12
```

### 6.3 钱包维护

```sql
-- 关闭钱包
ADMINISTER KEY MANAGEMENT SET KEYSTORE CLOSE 
IDENTIFIED BY "WalletPass123!";

-- 打开钱包
ADMINISTER KEY MANAGEMENT SET KEYSTORE OPEN 
IDENTIFIED BY "WalletPass123!";

-- 更改钱包密码
ADMINISTER KEY MANAGEMENT ALTER KEYSTORE PASSWORD 
FROM "OldPass123!" TO "NewPass456!" 
WITH BACKUP USING 'backup_password_change';

-- 合并钱包（多个实例共享）
ADMINISTER KEY MANAGEMENT MERGE KEYSTORE 
'/path/to/source_wallet' 
IDENTIFIED BY "SourcePass" 
INTO KEYSTORE 
'/path/to/target_wallet' 
IDENTIFIED BY "TargetPass" 
WITH BACKUP;
```

### 6.4 监控钱包状态

```sql
-- 查看钱包状态
SELECT 
    wrl_parameter,
    status,
    wallet_type,
    con_id
FROM v$encryption_wallet;

-- 查看密钥信息
SELECT 
    key_id,
    keystore_type,
    creation_time,
    activating_time
FROM v$encryption_keys;
```

## 7. 性能影响

### 7.1 性能测试

```sql
-- 测试加密性能
SET TIMING ON

-- 未加密表
INSERT INTO unencrypted_table SELECT * FROM source_table;
-- 耗时：10 秒

-- 加密表空间
INSERT INTO encrypted_table SELECT * FROM source_table;
-- 耗时：11 秒（约 10% 开销）

-- 加密列
INSERT INTO table_with_encrypted_cols SELECT * FROM source_table;
-- 耗时：12 秒（约 20% 开销）
```

### 7.2 性能优化建议

✅ **使用 AES-NI 指令集**（现代 CPU 支持）  
✅ **选择适当的加密算法**（AES128 比 AES256 快）  
✅ **批量操作优于单行操作**  
✅ **避免过度使用列加密**  
✅ **监控 I/O 性能**  

### 7.3 性能监控

```sql
-- 查看加密相关的等待事件
SELECT event, total_waits, time_waited
FROM v$system_event
WHERE event LIKE '%encrypt%';

-- 监控加密表空间的 I/O
SELECT 
    ts.name as tablespace_name,
    df.phyrds,
    df.phywrts,
    df.readtim,
    df.writetim
FROM v$datafile df, v$tablespace ts
WHERE df.ts# = ts.ts#
AND ts.encrypted = 'YES';
```

## 8. 实战案例

### 案例 1：金融系统数据加密

**需求**：加密所有敏感客户数据。

**实施方案**：

```sql
-- 1. 创建加密表空间
CREATE TABLESPACE secure_data
DATAFILE '/u01/oradata/secure_data01.dbf' SIZE 10G
ENCRYPTION USING 'AES256' DEFAULT STORAGE(ENCRYPT);

-- 2. 创建加密表
CREATE TABLE customers (
    customer_id NUMBER PRIMARY KEY,
    name VARCHAR2(100),
    ssn VARCHAR2(11) ENCRYPT NO SALT,  -- 可索引
    credit_card VARCHAR2(20) ENCRYPT,
    email VARCHAR2(100),
    phone VARCHAR2(20)
) TABLESPACE secure_data;

-- 3. 为 SSN 创建索引（需要 NO SALT）
CREATE INDEX idx_cust_ssn ON customers(ssn);

-- 4. 迁移现有数据
INSERT INTO customers SELECT * FROM old_customers;

-- 5. 验证加密
SELECT column_name, encryption_alg, salt
FROM dba_encrypted_columns
WHERE table_name = 'CUSTOMERS';
```

### 案例 2：医疗记录加密

**需求**：加密患者病历，符合 HIPAA 要求。

```sql
-- 加密表空间
CREATE TABLESPACE medical_records
ENCRYPTION USING 'AES256' DEFAULT STORAGE(ENCRYPT);

-- 加密敏感列
CREATE TABLE patient_records (
    record_id NUMBER PRIMARY KEY,
    patient_id NUMBER,
    diagnosis VARCHAR2(500) ENCRYPT,
    treatment VARCHAR2(1000) ENCRYPT,
    prescription VARCHAR2(500) ENCRYPT,
    doctor_notes CLOB  -- CLOB 不能直接加密，考虑表空间加密
) TABLESPACE medical_records;
```

## 9. 最佳实践

### 9.1 安全建议

✅ **使用 AES256 算法**  
✅ **定期轮换密钥**（至少每年一次）  
✅ **备份钱包**（安全存储）  
✅ **分离密钥和数据**（不同存储位置）  
✅ **使用 HSM 或 Key Vault**（生产环境）  

### 9.2 运维建议

📌 **监控钱包状态**  
📌 **测试密钥恢复流程**  
📌 **文档化密钥管理流程**  
📌 **培训 DBA 团队**  
📌 **定期审计加密配置**  

### 9.3 合规要求

⚠️ **PCI DSS**：信用卡数据必须加密  
⚠️ **HIPAA**：医疗记录必须加密  
⚠️ **GDPR**：个人数据保护  
⚠️ **SOX**：财务数据保护

## 10. 常见问题

### 10.1 钱包无法打开

**原因**：
- 密码错误
- 文件权限问题
- 路径配置错误

**解决**：
```sql
-- 检查钱包状态
SELECT * FROM v$encryption_wallet;

-- 确认路径正确
SHOW PARAMETER encryption_wallet_location;

-- 检查文件权限
ls -l /u01/app/oracle/wallet/
```

### 10.2 性能下降明显

**原因**：
- CPU 不支持 AES-NI
- 大量加密列
- I/O 瓶颈

**解决**：
```sql
-- 检查 CPU 支持
grep aes /proc/cpuinfo

-- 考虑使用表空间加密代替列加密
-- 或使用 AES128 代替 AES256
```

### 10.3 备份恢复问题

**重要**：备份时必须包含钱包文件！

```bash
# 完整备份包括
tar czf oracle_backup.tar.gz \
    /u01/oradata/ \           # 数据文件
    /u01/app/oracle/wallet/   # 钱包文件
```

## 11. 总结

### 11.1 核心知识点

✅ **TDE 原理**：透明加密，保护静态数据  
✅ **密钥层次**：主密钥 → 表密钥 → 数据  
✅ **表空间加密**：推荐方式，性能好  
✅ **列级加密**：细粒度控制，有限制  
✅ **钱包管理**：创建、打开、轮换、备份  

### 11.2 关键命令

📌 `ADMINISTER KEY MANAGEMENT` - 密钥管理  
📌 `CREATE TABLESPACE ... ENCRYPTION` - 加密表空间  
📌 `column_name ENCRYPT` - 加密列  
📌 `v$encryption_wallet` - 钱包状态  
📌 `dba_encrypted_columns` - 加密列信息
