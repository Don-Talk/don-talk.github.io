## 2. 用户管理

### 2.1 创建用户

```sql
-- 基本语法
CREATE USER username 
IDENTIFIED BY password
[DEFAULT TABLESPACE tablespace_name]
[TEMPORARY TABLESPACE temp_tablespace]
[QUOTA size ON tablespace_name]
[PASSWORD EXPIRE]
[ACCOUNT LOCK|UNLOCK];

-- 示例：创建应用用户
CREATE USER app_user 
IDENTIFIED BY "SecurePass123!"
DEFAULT TABLESPACE users
TEMPORARY TABLESPACE temp
QUOTA 100M ON users
QUOTA UNLIMITED ON temp;

-- 示例：创建只读用户
CREATE USER report_user 
IDENTIFIED BY "Report2026!"
DEFAULT TABLESPACE users
TEMPORARY TABLESPACE temp
QUOTA 0 ON users;  -- 不允许存储数据
```

### 2.2 修改用户

```sql
-- 修改密码
ALTER USER app_user IDENTIFIED BY "NewPass456!";

-- 解锁账户
ALTER USER app_user ACCOUNT UNLOCK;

-- 锁定账户
ALTER USER app_user ACCOUNT LOCK;

-- 强制密码过期
ALTER USER app_user PASSWORD EXPIRE;

-- 修改表空间配额
ALTER USER app_user QUOTA 200M ON users;

-- 修改默认表空间
ALTER USER app_user DEFAULT TABLESPACE app_data;
```

### 2.3 删除用户

```sql
-- 删除空用户
DROP USER username;

-- 删除用户及其所有对象
DROP USER username CASCADE;

-- 注意：CASCADE 会删除用户拥有的所有对象，谨慎使用
```

### 2.4 查看用户信息

```sql
-- 查看所有用户
SELECT 
    username,
    user_id,
    account_status,
    default_tablespace,
    temporary_tablespace,
    created,
    profile
FROM dba_users
ORDER BY created DESC;

-- 查看用户配额
SELECT 
    username,
    tablespace_name,
    bytes/1024/1024 as used_mb,
    max_bytes/1024/1024 as quota_mb
FROM dba_ts_quotas
WHERE username = 'APP_USER';

-- 查看用户会话
SELECT 
    username,
    sid,
    serial#,
    program,
    logon_time
FROM v$session
WHERE username IS NOT NULL;
```

## 3. 权限管理

### 3.1 系统权限

**定义**：允许用户在数据库中执行特定操作的权限。

#### 常用系统权限

| 权限 | 说明 |
|------|------|
| **CREATE SESSION** | 连接数据库 |
| **CREATE TABLE** | 创建表 |
| **CREATE VIEW** | 创建视图 |
| **CREATE PROCEDURE** | 创建存储过程 |
| **CREATE SEQUENCE** | 创建序列 |
| **CREATE TRIGGER** | 创建触发器 |
| **CREATE INDEX** | 创建索引 |
| **CREATE SYNONYM** | 创建同义词 |
| **ALTER ANY TABLE** | 修改任何表 |
| **DROP ANY TABLE** | 删除任何表 |
| **SELECT ANY TABLE** | 查询任何表 |
| **INSERT ANY TABLE** | 插入任何表 |
| **UPDATE ANY TABLE** | 更新任何表 |
| **DELETE ANY TABLE** | 删除任何表 |
| **DBA** | 数据库管理员（所有权限） |

#### 授予系统权限

```sql
-- 授予单个权限
GRANT CREATE SESSION TO app_user;
GRANT CREATE TABLE TO app_user;
GRANT CREATE VIEW TO app_user;

-- 授予多个权限
GRANT CREATE SESSION, CREATE TABLE, CREATE VIEW TO app_user;

-- 授予权限并允许转授
GRANT CREATE TABLE TO app_user WITH ADMIN OPTION;

-- 授予所有系统权限（不推荐）
GRANT DBA TO app_user;
```

#### 撤销系统权限

```sql
-- 撤销权限
REVOKE CREATE TABLE FROM app_user;
REVOKE CREATE SESSION FROM app_user;

-- 注意：WITH ADMIN OPTION 授予的权限撤销时不会级联
```

### 3.2 对象权限

**定义**：对特定数据库对象（表、视图、过程等）的操作权限。

#### 常用对象权限

| 权限 | 适用对象 | 说明 |
|------|---------|------|
| **SELECT** | 表、视图、序列 | 查询 |
| **INSERT** | 表、视图 | 插入 |
| **UPDATE** | 表、视图 | 更新 |
| **DELETE** | 表、视图 | 删除 |
| **EXECUTE** | 过程、函数、包 | 执行 |
| **INDEX** | 表 | 创建索引 |
| **REFERENCES** | 表 | 创建外键 |
| **ALTER** | 表、序列 | 修改 |
| **ALL** | 所有对象 | 所有权限 |

#### 授予对象权限

```sql
-- 授予表的 SELECT 权限
GRANT SELECT ON hr.employees TO app_user;

-- 授予多个权限
GRANT SELECT, INSERT, UPDATE ON hr.employees TO app_user;

-- 授予所有权限
GRANT ALL ON hr.employees TO app_user;

-- 授予列级权限
GRANT UPDATE (salary, commission_pct) ON hr.employees TO app_user;

-- 授予 EXECUTE 权限
GRANT EXECUTE ON hr.calculate_bonus TO app_user;

-- 允许转授
GRANT SELECT ON hr.employees TO app_user WITH GRANT OPTION;
```

#### 撤销对象权限

```sql
-- 撤销对象权限
REVOKE SELECT ON hr.employees FROM app_user;
REVOKE ALL ON hr.employees FROM app_user;

-- 注意：WITH GRANT OPTION 授予的权限撤销时会级联
```

#### 查看对象权限

```sql
-- 查看用户拥有的对象权限
SELECT 
    grantee,
    owner,
    table_name,
    privilege,
    grantable
FROM dba_tab_privs
WHERE grantee = 'APP_USER';

-- 查看列级权限
SELECT 
    grantee,
    owner,
    table_name,
    column_name,
    privilege
FROM dba_col_privs
WHERE grantee = 'APP_USER';
```

### 3.3 PUBLIC 权限

```sql
-- 授予 PUBLIC（所有用户）
GRANT SELECT ON hr.departments TO PUBLIC;

-- 撤销 PUBLIC 权限
REVOKE SELECT ON hr.departments FROM PUBLIC;

-- 注意：谨慎使用 PUBLIC，避免安全风险
```

## 4. 角色管理

### 4.1 预定义角色

| 角色 | 说明 |
|------|------|
| **CONNECT** | 基本连接权限 |
| **RESOURCE** | 创建对象权限 |
| **DBA** | 数据库管理员 |
| **EXP_FULL_DATABASE** | 导出权限 |
| **IMP_FULL_DATABASE** | 导入权限 |
| **SELECT_CATALOG_ROLE** | 查询数据字典 |

#### 查看预定义角色的权限

```sql
-- 查看角色包含的系统权限
SELECT 
    role,
    privilege,
    admin_option
FROM dba_sys_privs
WHERE role = 'RESOURCE'
ORDER BY privilege;

-- 查看角色包含的对象权限
SELECT 
    role,
    owner,
    table_name,
    privilege
FROM dba_tab_privs
WHERE role = 'SELECT_CATALOG_ROLE';
```

### 4.2 创建自定义角色

```sql
-- 创建角色
CREATE ROLE app_readonly;
CREATE ROLE app_readwrite;
CREATE ROLE app_admin;

-- 授予权限到角色
GRANT CREATE SESSION TO app_readonly;
GRANT SELECT ANY TABLE TO app_readonly;

GRANT CREATE SESSION, CREATE TABLE, CREATE VIEW TO app_readwrite;
GRANT SELECT, INSERT, UPDATE, DELETE ON hr.employees TO app_readwrite;

GRANT DBA TO app_admin;

-- 将角色授予用户
GRANT app_readonly TO report_user;
GRANT app_readwrite TO app_user;
GRANT app_admin TO dba_user;
```

### 4.3 角色嵌套

```sql
-- 角色可以包含其他角色
GRANT app_readonly TO app_readwrite;
-- app_readwrite 现在包含 app_readonly 的所有权限
```

### 4.4 启用和禁用角色

```sql
-- 查看当前会话的角色
SELECT * FROM session_roles;

-- 设置默认角色
ALTER USER app_user DEFAULT ROLE app_readwrite;

-- 禁用所有角色
ALTER USER app_user DEFAULT ROLE NONE;

-- 会话中启用角色
SET ROLE app_readonly;
SET ROLE app_readwrite IDENTIFIED BY password;
SET ROLE ALL;  -- 启用所有角色
SET ROLE NONE;  -- 禁用所有角色
```

### 4.5 带密码的角色

```sql
-- 创建带密码的角色
CREATE ROLE sensitive_role IDENTIFIED BY "RolePass123!";

-- 授予权限
GRANT SELECT ON hr.salaries TO sensitive_role;

-- 启用时需要密码
SET ROLE sensitive_role IDENTIFIED BY "RolePass123!";
```

## 5. 密码策略

### 5.1 Profile 管理

Profile 用于管理用户的密码和资源限制。

```sql
-- 查看默认 Profile
SELECT * FROM dba_profiles WHERE profile = 'DEFAULT';

-- 创建自定义 Profile
CREATE PROFILE app_profile LIMIT
    FAILED_LOGIN_ATTEMPTS 5
    PASSWORD_LOCK_TIME 1
    PASSWORD_LIFE_TIME 90
    PASSWORD_GRACE_TIME 7
    PASSWORD_REUSE_TIME 365
    PASSWORD_REUSE_MAX 5
    PASSWORD_VERIFY_FUNCTION verify_function_11g
    INACTIVE_ACCOUNT_TIME 30;

-- 将 Profile 分配给用户
ALTER USER app_user PROFILE app_profile;
```

### 5.2 密码策略参数

| 参数 | 说明 | 示例值 |
|------|------|--------|
| **FAILED_LOGIN_ATTEMPTS** | 失败登录尝试次数 | 5 |
| **PASSWORD_LOCK_TIME** | 账户锁定时间（天） | 1 |
| **PASSWORD_LIFE_TIME** | 密码有效期（天） | 90 |
| **PASSWORD_GRACE_TIME** | 密码过期宽限期（天） | 7 |
| **PASSWORD_REUSE_TIME** | 密码重用间隔（天） | 365 |
| **PASSWORD_REUSE_MAX** | 密码重用前需更改次数 | 5 |
| **PASSWORD_VERIFY_FUNCTION** | 密码验证函数 | verify_function |
| **INACTIVE_ACCOUNT_TIME** | 非活动账户锁定时间（天） | 30 |

### 5.3 密码验证函数

```sql
-- Oracle 提供的验证函数
-- $ORACLE_HOME/rdbms/admin/utlpwdmg.sql

-- 运行脚本创建验证函数
@?/rdbms/admin/utlpwdmg.sql

-- 自定义密码验证函数
CREATE OR REPLACE FUNCTION my_password_verify(
    username VARCHAR2,
    password VARCHAR2,
    old_password VARCHAR2
) RETURN BOOLEAN IS
BEGIN
    -- 密码长度至少 8 位
    IF LENGTH(password) < 8 THEN
        RAISE_APPLICATION_ERROR(-20001, 'Password must be at least 8 characters');
    END IF;
    
    -- 必须包含数字
    IF NOT REGEXP_LIKE(password, '[0-9]') THEN
        RAISE_APPLICATION_ERROR(-20002, 'Password must contain at least one digit');
    END IF;
    
    -- 不能与用户名相同
    IF LOWER(password) = LOWER(username) THEN
        RAISE_APPLICATION_ERROR(-20003, 'Password cannot be same as username');
    END IF;
    
    RETURN TRUE;
END;
/

-- 应用到 Profile
ALTER PROFILE app_profile LIMIT PASSWORD_VERIFY_FUNCTION my_password_verify;
```

## 6. 权限诊断

### 6.1 查看用户权限

```sql
-- 查看用户的系统权限
SELECT 
    grantee,
    privilege,
    admin_option
FROM dba_sys_privs
WHERE grantee = 'APP_USER';

-- 查看用户的角色
SELECT 
    grantee,
    granted_role,
    admin_option,
    default_role
FROM dba_role_privs
WHERE grantee = 'APP_USER';

-- 查看用户的所有权限（通过角色）
SELECT 
    role,
    privilege
FROM role_sys_privs
WHERE role IN (
    SELECT granted_role 
    FROM dba_role_privs 
    WHERE grantee = 'APP_USER'
);
```

### 6.2 权限继承关系

```sql
-- 递归查询用户的所有权限
WITH RECURSIVE user_roles AS (
    SELECT grantee, granted_role
    FROM dba_role_privs
    WHERE grantee = 'APP_USER'
    
    UNION ALL
    
    SELECT drp.grantee, drp.granted_role
    FROM dba_role_privs drp
    JOIN user_roles ur ON drp.grantee = ur.granted_role
)
SELECT DISTINCT 
    privilege
FROM role_sys_privs rsp
JOIN user_roles ur ON rsp.role = ur.granted_role;
```

### 6.3 权限问题排查

**问题 1：权限不足**
```sql
-- 错误：ORA-01031: insufficient privileges

-- 检查用户是否有相应权限
SELECT * FROM dba_sys_privs WHERE grantee = 'USERNAME';
SELECT * FROM dba_tab_privs WHERE grantee = 'USERNAME';
SELECT * FROM dba_role_privs WHERE grantee = 'USERNAME';
```

**问题 2：权限过多**
```sql
-- 审计用户权限
SELECT 
    grantee,
    COUNT(*) as priv_count
FROM dba_sys_privs
GROUP BY grantee
ORDER BY priv_count DESC;

-- 遵循最小权限原则，撤销不必要的权限
REVOKE DBA FROM app_user;
GRANT CREATE SESSION, CREATE TABLE TO app_user;
```

## 7. 最佳实践

### 7.1 安全原则

✅ **最小权限原则**
- 只授予必要的权限
- 定期审查和清理权限
- 避免使用 DBA 角色

✅ **使用角色管理**
- 通过角色分配权限
- 便于管理和维护
- 减少直接授予用户权限

✅ **强密码策略**
- 使用复杂的密码验证函数
- 定期更换密码
- 限制失败登录尝试

✅ **分离职责**
- DBA、开发、应用、报表用户分离
- 不同环境使用不同用户
- 生产环境严格控制

### 7.2 权限模型设计

**典型三层模型**：

```
应用用户（app_user）
    ├── 角色：app_readwrite
    ├── 权限：CRUD 业务表
    └── 限制：不能访问系统表

报表用户（report_user）
    ├── 角色：app_readonly
    ├── 权限：SELECT 业务表
    └── 限制：不能修改数据

DBA 用户（dba_user）
    ├── 角色：DBA
    ├── 权限：管理数据库
    └── 限制：仅运维使用
```

### 7.3 定期审计

```sql
-- 每月审查权限变更
SELECT 
    username,
    action_name,
    obj_name,
    timestamp
FROM dba_audit_trail
WHERE action_name IN ('GRANT', 'REVOKE')
AND timestamp > SYSDATE - 30
ORDER BY timestamp DESC;

-- 检查未使用的权限
-- 结合审计日志分析
```
