## 1. 统一审计概述

### 1.1 什么是统一审计？

统一审计（Unified Auditing）是 Oracle 12c 引入的新审计框架，整合了所有审计功能。

**特点**：
- ✅ 统一的审计基础设施
- ✅ 更好的性能
- ✅ 更灵活的策略
- ✅ 集中管理
- ✅ 符合最新合规要求

<!-- TODO: 插入图片：统一审计架构图 -->

### 1.2 统一审计 vs 传统审计

| 特性 | 传统审计 | 统一审计 |
|------|---------|---------|
| **存储位置** | AUD$ 表 | 专用只读表 |
| **性能** | 较差 | 更好 |
| **管理** | 分散 | 集中 |
| **策略** | 简单 | 灵活 |
| **版本** | 11g 及之前 | 12c+ |
| **默认状态** | 启用 | 可选启用 |

### 1.3 审计类型

| 类型 | 说明 |
|------|------|
| **标准审计** | 系统级操作、对象访问 |
| **FGA** | 细粒度审计 |
| **RMAN 审计** | 备份恢复操作 |
| **Data Pump 审计** | 数据泵操作 |
| **OLS 审计** | Label Security |
| **Database Vault 审计** | Database Vault 操作 |

## 2. 启用统一审计

### 2.1 检查当前状态

```sql
-- 检查是否启用统一审计
SELECT parameter, value 
FROM v$option 
WHERE parameter = 'Unified Auditing';

-- VALUE = TRUE 表示已启用
```

### 2.2 启用统一审计

**注意**：需要在数据库关闭状态下操作。

```bash
# 1. 关闭数据库
sqlplus / as sysdba
SHUTDOWN IMMEDIATE;

# 2. 重新链接 Oracle 二进制文件（Linux/Unix）
cd $ORACLE_HOME/rdbms/lib
make -f ins_rdbms.mk uniaud_on ioracle

# Windows：重新安装时选择统一审计选项

# 3. 启动数据库
STARTUP;

# 4. 验证
SELECT parameter, value FROM v$option WHERE parameter = 'Unified Auditing';
```

### 2.3 混合模式

Oracle 支持同时运行传统审计和统一审计（过渡期）。

```sql
-- 查看审计配置
SHOW PARAMETER audit_trail;

-- audit_trail = DB,UNIFIED 表示混合模式
```

## 3. 统一审计策略

### 3.1 预定义策略

Oracle 提供了一些预定义策略：

```sql
-- 查看所有预定义策略
SELECT policy_name, enabled_opt, success, failure
FROM audit_unified_enabled_policies
ORDER BY policy_name;

-- 常见预定义策略：
-- ORA_LOGON_FAILURES：登录失败
-- ORA_SECURECONFIG：安全配置（默认启用）
```

### 3.2 创建自定义策略

#### 示例 1：审计表访问

```sql
-- 创建审计策略
CREATE AUDIT POLICY audit_employee_access
    ACTIONS SELECT ON hr.employees,
            INSERT ON hr.employees,
            UPDATE ON hr.employees,
            DELETE ON hr.employees;

-- 启用策略（对所有用户）
AUDIT POLICY audit_employee_access;

-- 启用策略（对特定用户）
AUDIT POLICY audit_employee_access BY hr_user, app_user;

-- 启用策略（排除特定用户）
AUDIT POLICY audit_employee_access EXCEPT dba_user;
```

#### 示例 2：审计系统权限使用

```sql
-- 审计系统权限的使用
CREATE AUDIT POLICY audit_admin_actions
    PRIVILEGES CREATE ANY TABLE,
               DROP ANY TABLE,
               ALTER ANY TABLE,
               CREATE USER,
               DROP USER,
               GRANT ANY PRIVILEGE;

-- 启用策略
AUDIT POLICY audit_admin_actions;
```

#### 示例 3：审计角色使用

```sql
-- 审计角色的授予和使用
CREATE AUDIT POLICY audit_dba_role
    ROLES DBA,
          IMP_FULL_DATABASE,
          EXP_FULL_DATABASE;

AUDIT POLICY audit_dba_role;
```

#### 示例 4：审计组件级操作

```sql
-- 审计 Data Pump 操作
CREATE AUDIT POLICY audit_datapump
    ACTIONS COMPONENT = DATAPUMP ALL;

-- 审计 RMAN 操作
CREATE AUDIT POLICY audit_rman
    ACTIONS COMPONENT = RMAN ALL;

-- 审计 SQL*Loader
CREATE AUDIT POLICY audit_sqlloader
    ACTIONS COMPONENT = SQLLOADER ALL;

AUDIT POLICY audit_datapump;
AUDIT POLICY audit_rman;
AUDIT POLICY audit_sqlloader;
```

#### 示例 5：带条件的审计

```sql
-- 只在特定条件下审计
CREATE AUDIT POLICY audit_high_value_transactions
    ACTIONS INSERT, UPDATE, DELETE ON hr.salaries
    WHEN 'NEW.salary > 100000 OR OLD.salary > 100000'
    EVALUATE PER STATEMENT;

AUDIT POLICY audit_high_value_transactions;
```

### 3.3 复杂策略示例

```sql
-- 综合审计策略
CREATE AUDIT POLICY comprehensive_audit
    -- 对象审计
    ACTIONS SELECT, INSERT, UPDATE, DELETE ON hr.employees,
            SELECT ON hr.salaries,
    -- 系统权限审计
    PRIVILEGES CREATE USER, DROP USER, ALTER DATABASE,
    -- 角色审计
    ROLES DBA,
    -- 组件审计
    ACTIONS COMPONENT = DATAPUMP EXPORT, IMPORT;

AUDIT POLICY comprehensive_audit;
```

## 4. 管理审计策略

### 4.1 查看策略

```sql
-- 查看所有审计策略
SELECT 
    policy_name,
    audit_option_type,
    audit_option,
    common,
    created
FROM audit_unified_policies
ORDER BY policy_name;

-- 查看启用的策略
SELECT 
    policy_name,
    enabled_opt,
    user_name,
    success,
    failure
FROM audit_unified_enabled_policies
ORDER BY policy_name;

-- 查看策略详情
SELECT * 
FROM audit_unified_policy_statements
WHERE policy_name = 'AUDIT_EMPLOYEE_ACCESS';
```

### 4.2 启用/禁用策略

```sql
-- 启用策略
AUDIT POLICY audit_employee_access;

-- 对特定用户启用
AUDIT POLICY audit_employee_access BY hr_user;

-- 排除特定用户
AUDIT POLICY audit_employee_access EXCEPT dba_user;

-- 只审计成功操作
AUDIT POLICY audit_employee_access WHENEVER SUCCESSFUL;

-- 只审计失败操作
AUDIT POLICY audit_employee_access WHENEVER NOT SUCCESSFUL;

-- 禁用策略
NOAUDIT POLICY audit_employee_access;
```

### 4.3 删除策略

```sql
-- 先禁用
NOAUDIT POLICY audit_employee_access;

-- 再删除
DROP AUDIT POLICY audit_employee_access;
```

## 5. 查看审计记录

### 5.1 统一审计视图

| 视图 | 说明 |
|------|------|
| **UNIFIED_AUDIT_TRAIL** | 主要审计视图 |
| **AUDIT_UNIFIED_ENABLED_POLICIES** | 启用的策略 |
| **AUDIT_UNIFIED_POLICIES** | 所有策略 |

### 5.2 查询审计记录

```sql
-- 查看最近的审计记录
SELECT 
    event_timestamp,
    dbusername,
    userhost,
    action_name,
    object_schema,
    object_name,
    sql_text,
    return_code
FROM unified_audit_trail
ORDER BY event_timestamp DESC
FETCH FIRST 20 ROWS ONLY;

-- 查看特定策略的审计记录
SELECT 
    event_timestamp,
    dbusername,
    action_name,
    object_name,
    sql_text
FROM unified_audit_trail
WHERE unified_audit_policies = 'AUDIT_EMPLOYEE_ACCESS'
ORDER BY event_timestamp DESC;

-- 查看失败的登录尝试
SELECT 
    event_timestamp,
    dbusername,
    userhost,
    action_name,
    return_code
FROM unified_audit_trail
WHERE action_name = 'LOGON'
AND return_code != 0
ORDER BY event_timestamp DESC;
```

### 5.3 分析审计数据

```sql
-- 按用户统计
SELECT 
    dbusername,
    COUNT(*) as action_count,
    COUNT(DISTINCT action_name) as unique_actions
FROM unified_audit_trail
WHERE event_timestamp > SYSDATE - 7
GROUP BY dbusername
ORDER BY action_count DESC;

-- 按操作类型统计
SELECT 
    action_name,
    COUNT(*) as count
FROM unified_audit_trail
WHERE event_timestamp > SYSDATE - 1
GROUP BY action_name
ORDER BY count DESC;

-- 查找异常活动
SELECT 
    dbusername,
    object_name,
    COUNT(*) as access_count
FROM unified_audit_trail
WHERE event_timestamp > SYSDATE - 1
AND action_name = 'SELECT'
GROUP BY dbusername, object_name
HAVING COUNT(*) > 100;

-- 按时间分布
SELECT 
    TO_CHAR(event_timestamp, 'YYYY-MM-DD HH24') as hour,
    COUNT(*) as audit_count
FROM unified_audit_trail
WHERE event_timestamp > SYSDATE - 1
GROUP BY TO_CHAR(event_timestamp, 'YYYY-MM-DD HH24')
ORDER BY hour;
```

### 5.4 导出审计记录

```sql
-- 导出到表
CREATE TABLE audit_archive AS
SELECT * FROM unified_audit_trail
WHERE event_timestamp < SYSDATE - 90;

-- 导出到文件（使用 Data Pump）
-- 或使用外部表
```

## 6. 审计管理

### 6.1 清理审计记录

```sql
-- 清理旧的审计记录
BEGIN
    DBMS_AUDIT_MGMT.CLEAN_AUDIT_TRAIL(
        audit_trail_type => DBMS_AUDIT_MGMT.AUDIT_TRAIL_UNIFIED,
        use_last_arch_timestamp => FALSE,
        container => DBMS_AUDIT_MGMT.CONTAINER_CURRENT
    );
END;
/

-- 设置清理时间戳
BEGIN
    DBMS_AUDIT_MGMT.SET_LAST_ARCHIVE_TIMESTAMP(
        audit_trail_type => DBMS_AUDIT_MGMT.AUDIT_TRAIL_UNIFIED,
        last_archive_time => SYSTIMESTAMP - 90,
        container => DBMS_AUDIT_MGMT.CONTAINER_CURRENT
    );
END;
/
```

### 6.2 自动清理作业

```sql
-- 创建定期清理作业
BEGIN
    DBMS_AUDIT_MGMT.CREATE_PURGE_JOB(
        audit_trail_type => DBMS_AUDIT_MGMT.AUDIT_TRAIL_UNIFIED,
        audit_trail_purge_interval => 24,  -- 小时
        audit_trail_purge_name => 'UNIFIED_AUDIT_PURGE',
        use_last_arch_timestamp => TRUE,
        container => DBMS_AUDIT_MGMT.CONTAINER_CURRENT
    );
END;
/
```

### 6.3 移动审计表空间

```sql
-- 将审计记录移动到专用表空间
BEGIN
    DBMS_AUDIT_MGMT.SET_AUDIT_TRAIL_LOCATION(
        audit_trail_type => DBMS_AUDIT_MGMT.AUDIT_TRAIL_UNIFIED,
        audit_trail_location_value => 'AUDIT_TS'
    );
END;
/
```

## 7. 实战案例

### 案例 1：金融系统合规审计

**需求**：满足 PCI DSS 要求，审计所有敏感数据访问。

```sql
-- 创建综合审计策略
CREATE AUDIT POLICY pci_compliance_audit
    ACTIONS 
        SELECT, INSERT, UPDATE, DELETE ON app.credit_cards,
        SELECT, INSERT, UPDATE, DELETE ON app.customer_data,
    PRIVILEGES 
        ALTER SYSTEM,
        CREATE USER,
        DROP USER
    ROLES 
        DBA;

AUDIT POLICY pci_compliance_audit;

-- 定期生成合规报告
SELECT 
    TO_CHAR(event_timestamp, 'YYYY-MM-DD') as audit_date,
    dbusername,
    action_name,
    object_name,
    COUNT(*) as access_count
FROM unified_audit_trail
WHERE unified_audit_policies = 'PCI_COMPLIANCE_AUDIT'
AND event_timestamp > SYSDATE - 30
GROUP BY TO_CHAR(event_timestamp, 'YYYY-MM-DD'), 
         dbusername, action_name, object_name
ORDER BY audit_date;
```

### 案例 2：医疗系统 HIPAA 审计

**需求**：符合 HIPAA 要求，审计患者记录访问。

```sql
-- 审计所有患者记录访问
CREATE AUDIT POLICY hipaa_patient_audit
    ACTIONS 
        SELECT, INSERT, UPDATE, DELETE ON medical.patient_records,
        SELECT ON medical.prescriptions
    WHEN 
        'SYS_CONTEXT(''USERENV'', ''SESSION_USER'') != ''MEDICAL_ADMIN'''
    EVALUATE PER STATEMENT;

AUDIT POLICY hipaa_patient_audit;

-- 监控异常访问
SELECT 
    dbusername,
    COUNT(*) as access_count,
    MIN(event_timestamp) as first_access,
    MAX(event_timestamp) as last_access
FROM unified_audit_trail
WHERE unified_audit_policies = 'HIPAA_PATIENT_AUDIT'
AND event_timestamp > SYSDATE - 7
GROUP BY dbusername
HAVING COUNT(*) > 50;  -- 频繁访问告警
```

## 8. 最佳实践

### 8.1 设计原则

✅ **最小化审计范围**：只审计必要的操作  
✅ **分层审计策略**：不同级别不同策略  
✅ **定期审查**：更新策略以适应变化  
✅ **保护审计数据**：防止篡改  
✅ **设置保留策略**：平衡存储和合规  

### 8.2 性能优化

⚡ **避免过度审计**  
⚡ **使用条件过滤**  
⚡ **定期清理审计日志**  
⚡ **使用专用表空间**  
⚡ **监控性能影响**  

### 8.3 合规建议

📌 **记录完整上下文**  
📌 **保留足够长的历史**  
📌 **定期导出和归档**  
📌 **实施访问控制**  
📌 **生成定期报告**

## 9. 常见问题

### 9.1 审计记录未生成

**原因**：
- 策略未启用
- 条件不匹配
- 统一审计未启用

**解决**：
```sql
-- 检查策略状态
SELECT * FROM audit_unified_enabled_policies;

-- 检查统一审计是否启用
SELECT * FROM v$option WHERE parameter = 'Unified Auditing';
```

### 9.2 审计表空间已满

**解决**：
```sql
-- 增加表空间
ALTER TABLESPACE audit_ts ADD DATAFILE '...' SIZE 10G;

-- 或清理旧记录
EXEC DBMS_AUDIT_MGMT.CLEAN_AUDIT_TRAIL(...);
```

### 9.3 性能影响

**优化**：
- 减少审计策略数量
- 使用精确的条件
- 避免审计高频操作
- 监控性能指标
