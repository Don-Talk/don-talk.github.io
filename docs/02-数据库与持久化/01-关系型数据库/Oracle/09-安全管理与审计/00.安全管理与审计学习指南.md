### 01. 用户与角色管理

**核心内容**：
- 用户创建和配置
- 密码策略
- Profile 管理
- 预定义角色
- 自定义角色
- 角色继承

**关键知识点**：
```sql
-- 创建用户
CREATE USER john IDENTIFIED BY password123
DEFAULT TABLESPACE users
TEMPORARY TABLESPACE temp
QUOTA 100M ON users;

-- 创建 Profile
CREATE PROFILE app_profile LIMIT
    PASSWORD_LIFE_TIME 90
    PASSWORD_GRACE_TIME 7
    FAILED_LOGIN_ATTEMPTS 5
    PASSWORD_LOCK_TIME 1
    SESSIONS_PER_USER 3
    CPU_PER_SESSION 10000
    CONNECT_TIME 60;

-- 应用 Profile
ALTER USER john PROFILE app_profile;

-- 创建角色
CREATE ROLE app_user;
CREATE ROLE app_admin;

-- 授权给角色
GRANT CREATE SESSION, CREATE TABLE TO app_user;
GRANT app_user TO app_admin;
GRANT DBA TO app_admin;

-- 授予角色给用户
GRANT app_user TO john;

-- 查看用户信息
SELECT username, default_tablespace, temporary_tablespace, profile
FROM dba_users;

-- 查看角色权限
SELECT * FROM role_sys_privs WHERE role = 'APP_USER';
SELECT * FROM role_tab_privs WHERE role = 'APP_USER';

-- 查看用户角色
SELECT * FROM dba_role_privs WHERE grantee = 'JOHN';
```

### 02. 权限授予与回收

**核心内容**：
- 系统权限 vs 对象权限
- 权限授予语法
- WITH ADMIN OPTION
- WITH GRANT OPTION
- 权限回收
- 权限传递链

**关键知识点**：
```sql
-- 系统权限
GRANT CREATE SESSION TO john;
GRANT CREATE TABLE, CREATE VIEW, CREATE PROCEDURE TO john;
GRANT UNLIMITED TABLESPACE TO john;

-- WITH ADMIN OPTION（可以转授）
GRANT CREATE SESSION TO john WITH ADMIN OPTION;

-- 对象权限
GRANT SELECT, INSERT, UPDATE ON hr.employees TO john;
GRANT ALL ON hr.departments TO john;

-- WITH GRANT OPTION（可以转授对象权限）
GRANT SELECT ON hr.employees TO john WITH GRANT OPTION;

-- 回收权限
REVOKE CREATE TABLE FROM john;
REVOKE SELECT ON hr.employees FROM john;

-- 回收角色
REVOKE app_user FROM john;

-- 查看当前用户权限
SELECT * FROM user_sys_privs;
SELECT * FROM user_tab_privs;
SELECT * FROM user_role_privs;

-- 查看所有权限（DBA）
SELECT * FROM dba_sys_privs WHERE grantee = 'JOHN';
SELECT * FROM dba_tab_privs WHERE grantee = 'JOHN';
```

**最小权限原则**：
- 只授予必要的权限
- 优先使用角色而非直接授权
- 定期审查和清理权限
- 避免授予 DBA 权限

### 03. 虚拟私有数据库VPD

**核心内容**：
- VPD/RLS（行级安全）概念
- 安全策略函数
- 动态谓词添加
- 列级 VPD
- VPD 性能考虑

**关键知识点**：
```sql
-- 创建策略函数
CREATE OR REPLACE FUNCTION emp_security_policy(
    p_schema IN VARCHAR2,
    p_object IN VARCHAR2
) RETURN VARCHAR2 AS
    v_dept_id NUMBER;
BEGIN
    -- 获取当前用户的部门 ID
    SELECT dept_id INTO v_dept_id
    FROM emp_access
    WHERE username = SYS_CONTEXT('USERENV', 'SESSION_USER');
    
    -- 返回谓词
    RETURN 'dept_id = ' || v_dept_id;
EXCEPTION
    WHEN NO_DATA_FOUND THEN
        RETURN '1=0';  -- 无访问权限
END;

-- 添加策略
BEGIN
    DBMS_RLS.ADD_POLICY(
        object_schema => 'HR',
        object_name => 'EMPLOYEES',
        policy_name => 'EMP_DEPT_POLICY',
        function_schema => 'HR',
        policy_function => 'emp_security_policy',
        statement_types => 'SELECT, INSERT, UPDATE, DELETE',
        update_check => TRUE
    );
END;

-- 删除策略
BEGIN
    DBMS_RLS.DROP_POLICY(
        object_schema => 'HR',
        object_name => 'EMPLOYEES',
        policy_name => 'EMP_DEPT_POLICY'
    );
END;

-- 查看策略
SELECT * FROM dba_policies WHERE object_owner = 'HR';
```

**应用场景**：
- 多租户数据隔离
- 部门数据隔离
- 敏感数据访问控制

### 04. 透明数据加密TDE

**核心内容**：
- TDE 架构和原理
- Wallet 配置和管理
- 表空间加密
- 列级加密
- 密钥管理
- 性能影响

**关键知识点**：
```sql
-- 配置 Wallet
-- sqlnet.ora 中添加：
-- ENCRYPTION_WALLET_LOCATION =
--   (SOURCE = (METHOD = FILE)
--     (METHOD_DATA = (DIRECTORY = /u01/app/oracle/wallet)))

-- 创建 Wallet
ADMINISTER KEY MANAGEMENT CREATE KEYSTORE '/u01/app/oracle/wallet' 
IDENTIFIED BY wallet_password;

-- 打开 Wallet
ADMINISTER KEY MANAGEMENT SET KEYSTORE OPEN 
IDENTIFIED BY wallet_password;

-- 设置主密钥
ADMINISTER KEY MANAGEMENT SET KEY 
IDENTIFIED BY wallet_password WITH BACKUP;

-- 加密表空间
CREATE TABLESPACE encrypted_ts
DATAFILE '/u01/app/oracle/oradata/ORCL/encrypted01.dbf'
SIZE 1G
ENCRYPTION USING 'AES256'
DEFAULT STORAGE(ENCRYPT);

-- 加密列
CREATE TABLE sensitive_data (
    id NUMBER PRIMARY KEY,
    ssn VARCHAR2(11) ENCRYPT USING 'AES256',
    credit_card VARCHAR2(16) ENCRYPT,
    name VARCHAR2(100)
);

-- 查看加密状态
SELECT tablespace_name, encrypted FROM dba_tablespaces;
SELECT table_name, column_name, encryption_alg FROM dba_encrypted_columns;
```

**注意事项**：
- ⚠️ 备份 Wallet 文件
- ⚠️ 加密会影响性能（5-10%）
- ⚠️ 索引列加密后无法范围扫描
- ⚠️ 不能加密 SYS 用户的表

### 05. 细粒度审计FGA

**核心内容**：
- FGA 与传统审计对比
- 审计策略创建
- 审计条件定义
- 审计记录查询
- 审计策略管理
- 性能优化

**关键知识点**：
```sql
-- 创建审计策略
BEGIN
    DBMS_FGA.ADD_POLICY(
        object_schema => 'HR',
        object_name => 'EMPLOYEES',
        policy_name => 'SALARY_AUDIT',
        audit_condition => 'salary > 10000',
        audit_column => 'SALARY,COMMISSION_PCT',
        handler_schema => NULL,
        handler_module => NULL,
        enable => TRUE,
        statement_types => 'SELECT,INSERT,UPDATE,DELETE',
        audit_trail => DBMS_FGA.DB + DBMS_FGA.EXTENDED,
        audit_column_opts => DBMS_FGA.ANY_COLUMNS
    );
END;

-- 查看审计记录
SELECT 
    timestamp,
    db_user,
    object_schema,
    object_name,
    policy_name,
    sql_text,
    scn
FROM dba_fga_audit_trail
ORDER BY timestamp DESC;

-- 禁用策略
BEGIN
    DBMS_FGA.DISABLE_POLICY(
        object_schema => 'HR',
        object_name => 'EMPLOYEES',
        policy_name => 'SALARY_AUDIT'
    );
END;

-- 删除策略
BEGIN
    DBMS_FGA.DROP_POLICY(
        object_schema => 'HR',
        object_name => 'EMPLOYEES',
        policy_name => 'SALARY_AUDIT'
    );
END;

-- 清理审计记录
DELETE FROM dba_fga_audit_trail
WHERE timestamp < SYSDATE - 90;
```

**审计场景**：
- 敏感数据访问监控
- 合规性要求
- 异常操作检测
- 数据泄露追踪

## 🔍 实战案例预告

### 案例 1：多租户数据隔离
```sql
-- 使用 VPD 实现租户隔离
CREATE OR REPLACE FUNCTION tenant_policy(
    p_schema VARCHAR2,
    p_object VARCHAR2
) RETURN VARCHAR2 AS
BEGIN
    RETURN 'tenant_id = SYS_CONTEXT(''USERENV'', ''CLIENT_IDENTIFIER'')';
END;

BEGIN
    DBMS_RLS.ADD_POLICY(
        object_schema => 'APP',
        object_name => 'CUSTOMERS',
        policy_name => 'TENANT_POLICY',
        function_schema => 'APP',
        policy_function => 'tenant_policy'
    );
END;
```

### 案例 2：完整审计方案
```sql
-- 结合传统审计和 FGA
-- 1. 启用登录审计
AUDIT SESSION;

-- 2. 启用 DDL 审计
AUDIT CREATE TABLE, DROP TABLE, ALTER TABLE;

-- 3. 敏感表 FGA
BEGIN
    DBMS_FGA.ADD_POLICY(
        object_schema => 'HR',
        object_name => 'SALARIES',
        policy_name => 'SALARY_ACCESS',
        audit_condition => NULL,  -- 所有访问
        statement_types => 'SELECT'
    );
END;
```
