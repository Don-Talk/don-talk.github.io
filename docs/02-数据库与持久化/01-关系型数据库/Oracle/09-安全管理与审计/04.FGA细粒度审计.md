## 1. FGA 概述

### 1.1 什么是 FGA？

FGA（Fine-Grained Auditing，细粒度审计）是 Oracle 提供的基于内容的审计功能。

**特点**：
- ✅ 基于条件的审计
- ✅ 记录 SQL 语句和绑定变量
- ✅ 支持 SELECT、DML、DDL
- ✅ 可自定义审计条件
- ✅ 满足合规要求

### 1.2 FGA vs 标准审计

| 特性 | 标准审计 | FGA |
|------|---------|-----|
| **审计粒度** | 对象级 | 行级/列级 |
| **条件过滤** | 无 | 支持 |
| **记录内容** | 操作类型 | SQL 文本、绑定变量 |
| **性能影响** | 较小 | 较大 |
| **适用场景** | 通用审计 | 敏感数据审计 |

### 1.3 应用场景

📊 **敏感数据访问**：谁查看了薪资信息  
📊 **合规审计**：满足 SOX、HIPAA 等要求  
📊 **安全监控**：检测异常访问模式  
📊 **数据泄露调查**：追踪数据访问历史

## 2. FGA 架构

### 2.1 工作流程

```
用户执行 SQL
    ↓
检查 FGA 策略条件
    ↓
条件匹配 → 生成审计记录
    ↓
写入审计日志（DBA_FGA_AUDIT_TRAIL）
```

### 2.2 关键组件

| 组件 | 说明 |
|------|------|
| **策略** | 定义审计条件和动作 |
| **审计条件** | WHERE 子句，决定何时审计 |
| **审计列** | 指定要审计的列 |
| **处理程序** | 可选，触发时执行的存储过程 |
| **审计日志** | 存储审计记录 |

## 3. 创建 FGA 策略

### 3.1 基本语法

```sql
DBMS_FGA.ADD_POLICY(
    object_schema      => 'HR',
    object_name        => 'EMPLOYEES',
    policy_name        => 'audit_salary_access',
    audit_condition    => 'salary > 10000',
    audit_column       => 'SALARY, COMMISSION_PCT',
    handler_schema     => NULL,
    handler_module     => NULL,
    enable             => TRUE,
    statement_types    => 'SELECT, INSERT, UPDATE, DELETE',
    audit_trail        => DBMS_FGA.DB + DBMS_FGA.EXTENDED,
    audit_column_opts  => DBMS_FGA.ANY_COLUMNS
);
```

### 3.2 参数说明

| 参数 | 说明 | 示例值 |
|------|------|--------|
| **object_schema** | 对象所有者 | HR |
| **object_name** | 表或视图名 | EMPLOYEES |
| **policy_name** | 策略名称 | audit_salary |
| **audit_condition** | 审计条件 | salary > 10000 |
| **audit_column** | 审计列 | SALARY, EMAIL |
| **statement_types** | 审计的操作类型 | SELECT, UPDATE |
| **audit_trail** | 审计日志类型 | DB, DB+EXTENDED, XML |
| **handler_module** | 处理程序 | schema.proc_name |

### 3.3 审计条件示例

#### 示例 1：审计高薪资查询

```sql
BEGIN
    DBMS_FGA.ADD_POLICY(
        object_schema   => 'HR',
        object_name     => 'EMPLOYEES',
        policy_name     => 'audit_high_salary',
        audit_condition => 'salary > 50000',
        audit_column    => 'SALARY',
        statement_types => 'SELECT',
        audit_trail     => DBMS_FGA.DB + DBMS_FGA.EXTENDED
    );
END;
/
```

**效果**：只有当查询涉及 salary > 50000 的记录时才审计。

#### 示例 2：审计特定列访问

```sql
BEGIN
    DBMS_FGA.ADD_POLICY(
        object_schema   => 'HR',
        object_name     => 'EMPLOYEES',
        policy_name     => 'audit_ssn_access',
        audit_condition => NULL,  -- 所有访问都审计
        audit_column    => 'SSN, BANK_ACCOUNT',
        statement_types => 'SELECT',
        audit_trail     => DBMS_FGA.DB + DBMS_FGA.EXTENDED
    );
END;
/
```

**效果**：任何访问 SSN 或 BANK_ACCOUNT 列的操作都被审计。

#### 示例 3：审计非工作时间访问

```sql
BEGIN
    DBMS_FGA.ADD_POLICY(
        object_schema   => 'HR',
        object_name     => 'EMPLOYEES',
        policy_name     => 'audit_offhours',
        audit_condition => q'[
            TO_CHAR(SYSDATE, 'HH24') NOT BETWEEN '09' AND '18'
            OR TO_CHAR(SYSDATE, 'DY') IN ('SAT', 'SUN')
        ]',
        statement_types => 'SELECT, UPDATE, DELETE',
        audit_trail     => DBMS_FGA.DB + DBMS_FGA.EXTENDED
    );
END;
/
```

**效果**：非工作时间的访问被审计。

#### 示例 4：审计特定用户

```sql
BEGIN
    DBMS_FGA.ADD_POLICY(
        object_schema   => 'HR',
        object_name     => 'EMPLOYEES',
        policy_name     => 'audit_specific_user',
        audit_condition => q'[
            SYS_CONTEXT('USERENV', 'SESSION_USER') = 'REPORT_USER'
        ]',
        statement_types => 'SELECT',
        audit_trail     => DBMS_FGA.DB + DBMS_FGA.EXTENDED
    );
END;
/
```

## 4. 管理 FGA 策略

### 4.1 查看策略

```sql
-- 查看所有 FGA 策略
SELECT 
    object_schema,
    object_name,
    policy_name,
    audit_condition,
    audit_column,
    enabled,
    statement_types
FROM dba_audit_policies
ORDER BY object_schema, object_name;

-- 查看特定表的策略
SELECT * FROM dba_audit_policies 
WHERE object_name = 'EMPLOYEES';
```

### 4.2 启用/禁用策略

```sql
-- 禁用策略
BEGIN
    DBMS_FGA.ENABLE_POLICY(
        object_schema => 'HR',
        object_name   => 'EMPLOYEES',
        policy_name   => 'audit_salary_access',
        enable        => FALSE
    );
END;
/

-- 启用策略
BEGIN
    DBMS_FGA.ENABLE_POLICY(
        object_schema => 'HR',
        object_name   => 'EMPLOYEES',
        policy_name   => 'audit_salary_access',
        enable        => TRUE
    );
END;
/
```

### 4.3 删除策略

```sql
BEGIN
    DBMS_FGA.DROP_POLICY(
        object_schema => 'HR',
        object_name   => 'EMPLOYEES',
        policy_name   => 'audit_salary_access'
    );
END;
/
```

## 5. 查看审计记录

### 5.1 审计日志视图

| 视图 | 说明 |
|------|------|
| **DBA_FGA_AUDIT_TRAIL** | FGA 审计记录 |
| **DBA_COMMON_AUDIT_TRAIL** | 统一审计视图（包含 FGA） |

### 5.2 查询审计记录

```sql
-- 查看最近的 FGA 审计记录
SELECT 
    timestamp,
    db_user,
    os_user,
    userhost,
    object_schema,
    object_name,
    policy_name,
    sql_text,
    sql_bind
FROM dba_fga_audit_trail
ORDER BY timestamp DESC
FETCH FIRST 20 ROWS ONLY;

-- 查看特定策略的审计记录
SELECT 
    timestamp,
    db_user,
    sql_text,
    sql_bind,
    statement_type
FROM dba_fga_audit_trail
WHERE policy_name = 'AUDIT_SALARY_ACCESS'
ORDER BY timestamp DESC;

-- 统计审计记录
SELECT 
    policy_name,
    COUNT(*) as audit_count,
    MIN(timestamp) as first_access,
    MAX(timestamp) as last_access
FROM dba_fga_audit_trail
GROUP BY policy_name
ORDER BY audit_count DESC;
```

### 5.3 分析审计数据

```sql
-- 按用户统计
SELECT 
    db_user,
    COUNT(*) as access_count,
    COUNT(DISTINCT sql_text) as unique_queries
FROM dba_fga_audit_trail
WHERE timestamp > SYSDATE - 7
GROUP BY db_user
ORDER BY access_count DESC;

-- 按时间分布
SELECT 
    TO_CHAR(timestamp, 'YYYY-MM-DD HH24') as hour,
    COUNT(*) as access_count
FROM dba_fga_audit_trail
WHERE timestamp > SYSDATE - 1
GROUP BY TO_CHAR(timestamp, 'YYYY-MM-DD HH24')
ORDER BY hour;

-- 查找异常访问模式
SELECT 
    db_user,
    object_name,
    COUNT(*) as access_count
FROM dba_fga_audit_trail
WHERE timestamp > SYSDATE - 1
GROUP BY db_user, object_name
HAVING COUNT(*) > 100;  -- 频繁访问
```

## 6. 审计处理程序

### 6.1 创建处理程序

处理程序是在审计事件发生时自动执行的存储过程。

```sql
-- 创建处理程序
CREATE OR REPLACE PROCEDURE security.fga_handler(
    p_schema VARCHAR2,
    p_object VARCHAR2,
    p_policy VARCHAR2
) IS
    PRAGMA AUTONOMOUS_TRANSACTION;
BEGIN
    -- 记录到自定义审计表
    INSERT INTO custom_audit_log (
        audit_time,
        db_user,
        object_name,
        policy_name,
        action
    ) VALUES (
        SYSTIMESTAMP,
        SYS_CONTEXT('USERENV', 'SESSION_USER'),
        p_object,
        p_policy,
        'FGA_TRIGGERED'
    );
    
    COMMIT;
    
    -- 可以发送邮件告警
    -- UTL_MAIL.SEND(...);
END;
/

-- 添加策略时指定处理程序
BEGIN
    DBMS_FGA.ADD_POLICY(
        object_schema   => 'HR',
        object_name     => 'EMPLOYEES',
        policy_name     => 'audit_with_handler',
        audit_condition => 'salary > 100000',
        handler_schema  => 'SECURITY',
        handler_module  => 'fga_handler',
        statement_types => 'SELECT'
    );
END;
/
```

## 7. 清理审计日志

### 7.1 手动清理

```sql
-- 删除旧的审计记录
DELETE FROM dba_fga_audit_trail
WHERE timestamp < SYSDATE - 90;

COMMIT;

-- 注意：需要适当权限
```

### 7.2 自动清理

```sql
-- 创建定期清理作业
BEGIN
    DBMS_SCHEDULER.CREATE_JOB(
        job_name        => 'CLEAN_FGA_AUDIT',
        job_type        => 'PLSQL_BLOCK',
        job_action      => q'[
            DELETE FROM sys.fga_log$
            WHERE ntimestamp# < SYSDATE - 90;
            COMMIT;
        ]',
        start_date      => SYSTIMESTAMP,
        repeat_interval => 'FREQ=DAILY; BYHOUR=2; BYMINUTE=0',
        enabled         => TRUE,
        comments        => 'Clean FGA audit trail older than 90 days'
    );
END;
/
```

## 8. 实战案例

### 案例 1：薪资数据审计

**需求**：审计所有对高薪员工数据的访问。

```sql
-- 创建审计策略
BEGIN
    DBMS_FGA.ADD_POLICY(
        object_schema   => 'HR',
        object_name     => 'EMPLOYEES',
        policy_name     => 'audit_salary_changes',
        audit_condition => 'salary > 50000 OR commission_pct > 0.3',
        audit_column    => 'SALARY, COMMISSION_PCT, BONUS',
        statement_types => 'SELECT, UPDATE',
        audit_trail     => DBMS_FGA.DB + DBMS_FGA.EXTENDED
    );
END;
/

-- 测试
CONNECT hr_user/password
SELECT employee_id, salary FROM hr.employees WHERE salary > 60000;

-- 查看审计记录
SELECT 
    timestamp,
    db_user,
    sql_text,
    sql_bind
FROM dba_fga_audit_trail
WHERE policy_name = 'AUDIT_SALARY_CHANGES'
ORDER BY timestamp DESC;
```

### 案例 2：患者记录访问审计

**需求**：符合 HIPAA 要求，审计所有患者记录访问。

```sql
-- 审计所有访问
BEGIN
    DBMS_FGA.ADD_POLICY(
        object_schema   => 'MEDICAL',
        object_name     => 'PATIENT_RECORDS',
        policy_name     => 'hipaa_audit',
        audit_condition => NULL,  -- 所有访问
        audit_column    => 'DIAGNOSIS, TREATMENT, PRESCRIPTION',
        statement_types => 'SELECT, INSERT, UPDATE, DELETE',
        audit_trail     => DBMS_FGA.DB + DBMS_FGA.EXTENDED
    );
END;
/

-- 定期生成审计报告
SELECT 
    TO_CHAR(timestamp, 'YYYY-MM-DD') as access_date,
    db_user,
    COUNT(*) as access_count
FROM dba_fga_audit_trail
WHERE policy_name = 'HIPAA_AUDIT'
AND timestamp > SYSDATE - 30
GROUP BY TO_CHAR(timestamp, 'YYYY-MM-DD'), db_user
ORDER BY access_date, access_count DESC;
```

## 9. 最佳实践

### 9.1 设计原则

✅ **最小化审计范围**：只审计必要的操作  
✅ **使用精确条件**：避免过度审计  
✅ **定期审查策略**：更新以适应变化  
✅ **保护审计日志**：防止篡改  
✅ **设置保留策略**：平衡存储和合规  

### 9.2 性能优化

⚡ **避免审计高频操作**  
⚡ **使用条件过滤**  
⚡ **定期清理审计日志**  
⚡ **监控审计表大小**  
⚡ **考虑分区审计表**  

### 9.3 合规建议

📌 **记录完整的 SQL 文本**  
📌 **保存绑定变量值**  
📌 **保留足够长的历史**（通常 1-7 年）  
📌 **定期导出审计日志**  
📌 **实施访问控制**

## 10. 常见问题

### 10.1 审计记录未生成

**原因**：
- 策略未启用
- 条件不匹配
- 权限不足

**解决**：
```sql
-- 检查策略状态
SELECT policy_name, enabled FROM dba_audit_policies;

-- 检查审计条件是否匹配
-- 手动测试条件
```

### 10.2 审计日志过大

**解决**：
```sql
-- 定期清理
DELETE FROM sys.fga_log$ WHERE ntimestamp# < SYSDATE - 90;

-- 或使用分区表
-- 或导出后归档
```

### 10.3 性能影响

**优化**：
- 减少审计的策略数量
- 使用更精确的条件
- 避免审计高频表
- 监控性能影响
