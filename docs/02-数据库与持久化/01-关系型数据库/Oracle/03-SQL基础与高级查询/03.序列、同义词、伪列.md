## 1. SEQUENCE（序列）

### 1.1 基本概念

SEQUENCE 是 Oracle 提供的唯一整数生成器，常用于主键自增。

**特点**：
- 独立于表的数据库对象
- 线程安全，高并发性能好
- 可共享，多个表可使用同一序列

### 1.2 创建序列

```sql
CREATE SEQUENCE emp_seq
    START WITH 1          -- 起始值
    INCREMENT BY 1        -- 步长
    MINVALUE 1            -- 最小值
    MAXVALUE 999999       -- 最大值
    CACHE 20              -- 缓存数量（提高性能）
    NOCYCLE;              -- 不循环
```

**参数说明**：
- `START WITH`：起始值（默认 1）
- `INCREMENT BY`：步长（默认 1，可为负数）
- `MINVALUE` / `NOMINVALUE`：最小值
- `MAXVALUE` / `NOMAXVALUE`：最大值
- `CACHE n`：预分配 n 个值到内存（默认 20）
- `NOCACHE`：不缓存（保证连续性，但性能较差）
- `CYCLE` / `NOCYCLE`：达到最大值后是否循环

### 1.3 使用序列

```sql
-- 获取下一个值
SELECT emp_seq.NEXTVAL FROM dual;

-- 获取当前值（必须先调用过 NEXTVAL）
SELECT emp_seq.CURRVAL FROM dual;

-- 插入数据时使用
INSERT INTO employees (emp_id, name, email)
VALUES (emp_seq.NEXTVAL, 'John Doe', 'john@test.com');

-- 批量插入
INSERT INTO employees (emp_id, name, email)
SELECT emp_seq.NEXTVAL, username, email
FROM temp_users;
```

### 1.4 序列缓存与性能

```sql
-- 高并发场景：增大缓存
CREATE SEQUENCE high_perf_seq
    START WITH 1
    INCREMENT BY 1
    CACHE 1000;  -- 大缓存，性能更好

-- 要求连续：不使用缓存
CREATE SEQUENCE continuous_seq
    START WITH 1
    INCREMENT BY 1
    NOCACHE;     -- 保证连续，但性能较差
```

**注意**：
- 缓存会导致序列跳号（实例重启时丢失缓存值）
- RAC 环境下建议使用较大缓存或 NOCACHE + ORDER

### 1.5 修改和删除序列

```sql
-- 修改序列（不能修改 START WITH）
ALTER SEQUENCE emp_seq
    INCREMENT BY 2
    MAXVALUE 9999999
    CACHE 50;

-- 重置序列（需要重建）
DROP SEQUENCE emp_seq;
CREATE SEQUENCE emp_seq START WITH 1 INCREMENT BY 1;

-- 或者使用技巧重置
ALTER SEQUENCE emp_seq INCREMENT BY -100;  -- 回退
SELECT emp_seq.NEXTVAL FROM dual;
ALTER SEQUENCE emp_seq INCREMENT BY 1;     -- 恢复正常
```

### 1.6 常见应用场景

#### 主键自增

```sql
CREATE TABLE orders (
    order_id NUMBER PRIMARY KEY,
    order_date DATE DEFAULT SYSDATE,
    customer_id NUMBER
);

CREATE SEQUENCE order_seq START WITH 1 INCREMENT BY 1;

INSERT INTO orders (order_id, customer_id)
VALUES (order_seq.NEXTVAL, 100);
```

#### Oracle 12c+ 自增列（IDENTITY）

```sql
CREATE TABLE products (
    product_id NUMBER GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    product_name VARCHAR2(100),
    price NUMBER(10,2)
);

-- 自动使用内部序列
INSERT INTO products (product_name, price) VALUES ('Product A', 99.99);
INSERT INTO products (product_name, price) VALUES ('Product B', 149.99);
```

## 2. 同义词（SYNONYM）

### 2.1 概念和作用

同义词是数据库对象的别名，用于：
- 简化对象访问
- 隐藏对象所有者
- 提供位置透明性

### 2.2 私有同义词

只对当前用户可见。

```sql
-- 创建同义词
CREATE SYNONYM emp FOR hr.employees;
CREATE SYNONYM dept FOR hr.departments;

-- 使用同义词
SELECT * FROM emp;  -- 等价于 SELECT * FROM hr.employees

-- 删除同义词
DROP SYNONYM emp;
```

### 2.3 公共同义词

所有用户都可访问。

```sql
-- 需要 DBA 权限
CREATE PUBLIC SYNONYM employees FOR hr.employees;
CREATE PUBLIC SYNONYM departments FOR hr.departments;

-- 任何用户都可以直接查询
SELECT * FROM employees;

-- 删除公共同义词
DROP PUBLIC SYNONYM employees;
```

### 2.4 同义词链

```sql
-- 同义词可以指向另一个同义词
CREATE SYNONYM e FOR hr.employees;
CREATE SYNONYM emp FOR e;

SELECT * FROM emp;  -- 最终指向 hr.employees
```

**注意**：避免过长的同义词链，影响性能和维护。

### 2.5 查看同义词

```sql
-- 查看用户的同义词
SELECT synonym_name, table_owner, table_name
FROM user_synonyms;

-- 查看公共同义词
SELECT synonym_name, table_owner, table_name
FROM dba_synonyms
WHERE owner = 'PUBLIC';
```

## 3. ROWNUM 伪列

### 3.1 基本概念

ROWNUM 是查询结果集中行的序号，从 1 开始。

```sql
SELECT rownum, employee_id, name
FROM employees
WHERE rownum <= 10;
```

### 3.2 ROWNUM 的执行顺序

**重要**：ROWNUM 在 WHERE 条件之后、ORDER BY 之前分配！

```sql
-- ❌ 错误：这不是取前 10 条薪资最高的员工
SELECT rownum, employee_id, name, salary
FROM employees
WHERE rownum <= 10
ORDER BY salary DESC;

-- ✅ 正确：使用子查询
SELECT rownum, employee_id, name, salary
FROM (
    SELECT employee_id, name, salary
    FROM employees
    ORDER BY salary DESC
)
WHERE rownum <= 10;
```

### 3.3 ROWNUM 分页

#### Oracle 11g 及之前

```sql
-- 第 2 页，每页 10 条（记录 11-20）
SELECT *
FROM (
    SELECT t.*, rownum as rn
    FROM (
        SELECT employee_id, name, salary
        FROM employees
        ORDER BY salary DESC
    ) t
    WHERE rownum <= 20  -- 上限
)
WHERE rn > 10;  -- 下限
```

**通用分页模板**：
```sql
-- 第 N 页，每页 M 条
SELECT *
FROM (
    SELECT t.*, rownum as rn
    FROM (
        -- 你的查询
        SELECT * FROM your_table ORDER BY some_column
    ) t
    WHERE rownum <= :N * :M
)
WHERE rn > (:N - 1) * :M;
```

#### Oracle 12c+（推荐）

```sql
--  OFFSET-FETCH 语法（更符合 SQL 标准）
SELECT employee_id, name, salary
FROM employees
ORDER BY salary DESC
OFFSET 10 ROWS FETCH NEXT 10 ROWS ONLY;  -- 第 2 页

-- 更简洁的写法
SELECT employee_id, name, salary
FROM employees
ORDER BY salary DESC
FETCH FIRST 10 ROWS ONLY;  -- 前 10 条
```

### 3.4 ROWNUM 陷阱

#### 陷阱 1：ROWNUM > n 永远返回空

```sql
-- ❌ 这个查询不会返回任何结果
SELECT * FROM employees WHERE rownum > 10;

-- ✅ 正确做法
SELECT *
FROM (
    SELECT t.*, rownum as rn
    FROM employees t
)
WHERE rn > 10;
```

**原因**：ROWNUM 从 1 开始分配，第一行不满足 `> 10`，被过滤；第二行 ROWNUM 还是 1，也被过滤……以此类推。

#### 陷阱 2：ROWNUM 与 ORDER BY

```sql
-- ❌ ROWNUM 在 ORDER BY 之前分配
SELECT rownum, employee_id, salary
FROM employees
WHERE rownum <= 5
ORDER BY salary DESC;

-- ✅ 先排序再分配 ROWNUM
SELECT rownum, employee_id, salary
FROM (
    SELECT employee_id, salary
    FROM employees
    ORDER BY salary DESC
)
WHERE rownum <= 5;
```

### 3.5 ROWNUM 实用技巧

```sql
-- 随机抽取 N 条记录
SELECT *
FROM (
    SELECT * FROM employees ORDER BY DBMS_RANDOM.VALUE
)
WHERE rownum <= 5;

-- 每组取前 N 条（使用分析函数更好）
SELECT *
FROM (
    SELECT 
        t.*,
        row_number() OVER (PARTITION BY dept_id ORDER BY salary DESC) as rn
    FROM employees t
)
WHERE rn <= 3;  -- 每个部门薪资最高的 3 人
```

## 4. ROWID 伪列

### 4.1 基本概念

ROWID 是行的物理地址，是访问单行最快的方式。

```sql
SELECT rowid, employee_id, name
FROM employees
WHERE employee_id = 100;
```

**ROWID 格式**：`OOOOOOFFFBBBBBBRRR`
- `OOOOOO`：数据对象号（6位）
- `FFF`：文件号（3位）
- `BBBBBB`：块号（6位）
- `RRR`：行号（3位）

### 4.2 ROWID 的特点

✅ **唯一标识**：每行有唯一的 ROWID  
✅ **永久不变**：除非行被移动或删除  
✅ **最快访问**：直接定位物理位置  
❌ **不建议存储**：行迁移后会改变  

### 4.3 使用场景

#### 快速删除重复记录

```sql
-- 保留 ROWID 最小的记录
DELETE FROM employees
WHERE rowid NOT IN (
    SELECT MIN(rowid)
    FROM employees
    GROUP BY employee_id  -- 假设 employee_id 应该唯一
);

-- 更高效的方式
DELETE FROM employees e1
WHERE rowid > (
    SELECT MIN(rowid)
    FROM employees e2
    WHERE e2.employee_id = e1.employee_id
);
```

#### 快速更新

```sql
UPDATE employees
SET salary = salary * 1.1
WHERE rowid = 'AAAR3qAAEAAAACHAAA';
```

#### 行链接检测

```sql
-- 查找行链接/行迁移
SELECT rowid, employee_id
FROM employees
WHERE rowid IN (
    SELECT head_rowid
    FROM chained_rows
    WHERE table_name = 'EMPLOYEES'
);
```

### 4.4 UROWID（通用 ROWID）

支持索引组织表（IOT）和外部表。

```sql
DECLARE
    v_rowid UROWID;
BEGIN
    SELECT rowid INTO v_rowid
    FROM employees
    WHERE employee_id = 100;
    
    DBMS_OUTPUT.PUT_LINE('ROWID: ' || v_rowid);
END;
```

## 5. LEVEL 伪列

### 5.1 层次查询

LEVEL 用于树形结构查询，表示节点层级。

```sql
-- 组织架构查询
SELECT 
    employee_id,
    manager_id,
    name,
    LEVEL,
    LPAD(' ', (LEVEL - 1) * 2) || name as org_chart
FROM employees
START WITH manager_id IS NULL  -- 根节点
CONNECT BY PRIOR employee_id = manager_id;
```

**输出示例**：
```
EMPLOYEE_ID MANAGER_ID NAME      LEVEL ORG_CHART
----------- ---------- --------- ----- -------------------------
          1            CEO           1 CEO
          2          1 Manager       2   Manager
          3          2 Staff         3     Staff
```

### 5.2 CONNECT BY 语法

```sql
SELECT column_list
FROM table_name
START WITH start_condition
CONNECT BY PRIOR parent_column = child_column;
```

**关键要素**：
- `START WITH`：定义根节点
- `CONNECT BY PRIOR`：定义父子关系
- `LEVEL`：当前层级（从 1 开始）

### 5.3 层次查询函数

```sql
SELECT 
    employee_id,
    manager_id,
    name,
    LEVEL,
    SYS_CONNECT_BY_PATH(name, ' -> ') as path,  -- 完整路径
    CONNECT_BY_ROOT name as root_employee,       -- 根节点
    CONNECT_BY_ISLEAF as is_leaf,                -- 是否叶子节点
    CONNECT_BY_ISCYCLE as is_cycle               -- 是否有循环
FROM employees
START WITH manager_id IS NULL
CONNECT BY NOCYCLE PRIOR employee_id = manager_id;
```

### 5.4 实际应用

#### 生成连续数字

```sql
-- 生成 1-100 的数字
SELECT LEVEL as num
FROM dual
CONNECT BY LEVEL <= 100;

-- 生成日期序列
SELECT TRUNC(SYSDATE) + LEVEL - 1 as date_value
FROM dual
CONNECT BY LEVEL <= 365;
```

#### 查找层级深度

```sql
-- 查找最深的组织层级
SELECT MAX(LEVEL) as max_depth
FROM employees
START WITH manager_id IS NULL
CONNECT BY PRIOR employee_id = manager_id;
```

## 6. 其他伪列

### 6.1 CURRVAL 和 NEXTVAL

已在序列部分介绍。

### 6.2 ORA_ROWSCN

返回行最后修改的系统变更号（SCN）。

```sql
SELECT 
    employee_id,
    name,
    ORA_ROWSCN,
    SCN_TO_TIMESTAMP(ORA_ROWSCN) as last_modified
FROM employees;
```

### 6.3 XML 相关伪列

```sql
-- OBJECT_VALUE（对象表）
SELECT o.OBJECT_VALUE
FROM customers o
WHERE o.OBJECT_VALUE.extract('/customer/name/text()').getStringVal() = 'John';
```

## 7. 综合实战

### 7.1 完整示例：订单系统

```sql
-- 1. 创建序列
CREATE SEQUENCE order_seq
    START WITH 1
    INCREMENT BY 1
    CACHE 100;

-- 2. 创建表
CREATE TABLE orders (
    order_id NUMBER PRIMARY KEY,
    customer_id NUMBER NOT NULL,
    order_date DATE DEFAULT SYSDATE,
    total_amount NUMBER(12,2),
    status VARCHAR2(20) DEFAULT 'PENDING'
);

-- 3. 插入数据（使用序列）
INSERT INTO orders (order_id, customer_id, total_amount)
VALUES (order_seq.NEXTVAL, 100, 599.99);

INSERT INTO orders (order_id, customer_id, total_amount)
VALUES (order_seq.NEXTVAL, 101, 1299.99);

-- 4. 分页查询（最新 10 个订单）
SELECT *
FROM (
    SELECT t.*, rownum as rn
    FROM (
        SELECT * FROM orders ORDER BY order_date DESC
    ) t
    WHERE rownum <= 10
)
WHERE rn > 0;

-- 5. 创建同义词简化访问
CREATE SYNONYM ord FOR orders;
SELECT * FROM ord WHERE rownum <= 5;
```

### 7.2 去重示例

```sql
-- 查找并删除重复邮箱（保留最早的记录）
DELETE FROM employees
WHERE rowid NOT IN (
    SELECT MIN(rowid)
    FROM employees
    GROUP BY email
);
```

## 8. 最佳实践

### 8.1 序列

✅ 高并发场景使用较大 CACHE 值  
✅ 主键优先使用 SEQUENCE 而非 MAX+1  
✅ Oracle 12c+ 可使用 IDENTITY 列  
❌ 不要依赖序列的连续性  

### 8.2 同义词

✅ 为常用表创建同义词简化代码  
✅ 使用公共同义词简化跨 schema 访问  
❌ 避免过长的同义词链  

### 8.3 ROWNUM

✅ 分页时使用子查询确保正确排序  
✅ Oracle 12c+ 优先使用 OFFSET-FETCH  
❌ 避免 `ROWNUM > n` 的错误用法  

### 8.4 ROWID

✅ 用于快速删除重复记录  
✅ 用于快速单行更新  
❌ 不要将 ROWID 存储在表中

## 9. 总结

### 9.1 核心知识点

✅ **SEQUENCE**：唯一整数生成器，用于主键自增  
✅ **同义词**：对象别名，简化访问  
✅ **ROWNUM**：结果集行号，注意执行顺序  
✅ **ROWID**：物理地址，最快访问方式  
✅ **LEVEL**：层次查询层级  

### 9.2 常用场景

📌 主键自增 → SEQUENCE  
📌 简化表名 → 同义词  
📌 分页查询 → ROWNUM 或 OFFSET-FETCH  
📌 删除重复 → ROWID  
📌 树形结构 → LEVEL + CONNECT BY
