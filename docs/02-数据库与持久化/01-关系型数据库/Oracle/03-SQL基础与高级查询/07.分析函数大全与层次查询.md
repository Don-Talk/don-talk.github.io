## 1. 分析函数概述

### 1.1 什么是分析函数？

分析函数（Analytic Functions）在行组上执行计算，返回每行的聚合结果。

**与传统聚合函数的区别**：
- 聚合函数：多行 → 一行（GROUP BY）
- 分析函数：多行 → 多行（保留原始行）

```sql
-- 传统聚合
SELECT dept_id, AVG(salary) FROM employees GROUP BY dept_id;

-- 分析函数（保留每行）
SELECT 
    name,
    dept_id,
    salary,
    AVG(salary) OVER (PARTITION BY dept_id) as dept_avg_salary
FROM employees;
```

### 1.2 基本语法

```sql
FUNCTION() OVER (
    [PARTITION BY column]
    [ORDER BY column]
    [window_clause]
)
```

**关键要素**：
- `PARTITION BY`：分组（类似 GROUP BY）
- `ORDER BY`：排序
- `window_clause`：窗口范围

## 2. 排名函数

### 2.1 ROW_NUMBER()

为每行分配唯一的连续序号。

```sql
SELECT 
    name,
    dept_id,
    salary,
    ROW_NUMBER() OVER (ORDER BY salary DESC) as rank
FROM employees;
```

**特点**：
- 唯一序号（1, 2, 3, 4...）
- 无并列
- 适合分页、Top N

**应用：每个部门薪资最高的员工**
```sql
SELECT *
FROM (
    SELECT 
        name,
        dept_id,
        salary,
        ROW_NUMBER() OVER (PARTITION BY dept_id ORDER BY salary DESC) as rn
    FROM employees
)
WHERE rn = 1;
```

### 2.2 RANK()

跳跃排名，相同值排名相同。

```sql
SELECT 
    name,
    salary,
    RANK() OVER (ORDER BY salary DESC) as rank
FROM employees;
```

**示例**：
```
NAME   | SALARY | RANK
-------|--------|-----
John   | 10000  | 1
Jane   | 10000  | 1  ← 并列
Bob    | 9000   | 3  ← 跳过 2
```

### 2.3 DENSE_RANK()

密集排名，相同值排名相同，不跳号。

```sql
SELECT 
    name,
    salary,
    DENSE_RANK() OVER (ORDER BY salary DESC) as rank
FROM employees;
```

**示例**：
```
NAME   | SALARY | DENSE_RANK
-------|--------|-----------
John   | 10000  | 1
Jane   | 10000  | 1  ← 并列
Bob    | 9000   | 2  ← 不跳号
```

### 2.4 三种排名对比

```sql
SELECT 
    name,
    salary,
    ROW_NUMBER() OVER (ORDER BY salary DESC) as row_num,
    RANK() OVER (ORDER BY salary DESC) as rank,
    DENSE_RANK() OVER (ORDER BY salary DESC) as dense_rank
FROM employees
ORDER BY salary DESC;
```

**输出**：
```
NAME   | SALARY | ROW_NUM | RANK | DENSE_RANK
-------|--------|---------|------|-----------
John   | 10000  | 1       | 1    | 1
Jane   | 10000  | 2       | 1    | 1
Bob    | 9000   | 3       | 3    | 2
Alice  | 8000   | 4       | 4    | 3
```

### 2.5 NTILE()

将数据均匀分成 N 组。

```sql
-- 分为 4 个等级（四分位数）
SELECT 
    name,
    salary,
    NTILE(4) OVER (ORDER BY salary DESC) as quartile
FROM employees;

-- 每个部门内部分为 3 组
SELECT 
    name,
    dept_id,
    salary,
    NTILE(3) OVER (PARTITION BY dept_id ORDER BY salary DESC) as group_num
FROM employees;
```

**应用**：
- 绩效评级（前 25% 为 A，25%-50% 为 B...）
- 客户分群

## 3. 偏移函数

### 3.1 LAG()

访问前 N 行的值。

```sql
SELECT 
    order_date,
    amount,
    LAG(amount, 1) OVER (ORDER BY order_date) as prev_amount,
    amount - LAG(amount, 1) OVER (ORDER BY order_date) as diff
FROM orders;
```

**参数**：
- `LAG(column, offset, default)`
- `offset`：向前几行（默认 1）
- `default`：无数据时的默认值

### 3.2 LEAD()

访问后 N 行的值。

```sql
SELECT 
    order_date,
    amount,
    LEAD(amount, 1) OVER (ORDER BY order_date) as next_amount,
    LEAD(amount, 1, 0) OVER (ORDER BY order_date) as next_with_default
FROM orders;
```

### 3.3 实际应用：环比增长

```sql
SELECT 
    month,
    revenue,
    LAG(revenue) OVER (ORDER BY month) as prev_revenue,
    ROUND((revenue - LAG(revenue) OVER (ORDER BY month)) / 
          LAG(revenue) OVER (ORDER BY month) * 100, 2) as growth_rate
FROM monthly_sales;
```

## 4. 首尾值函数

### 4.1 FIRST_VALUE()

获取窗口中的第一个值。

```sql
SELECT 
    name,
    dept_id,
    salary,
    FIRST_VALUE(name) OVER (
        PARTITION BY dept_id ORDER BY salary DESC
    ) as highest_earner
FROM employees;
```

### 4.2 LAST_VALUE()

获取窗口中的最后一个值。

```sql
SELECT 
    name,
    dept_id,
    salary,
    LAST_VALUE(name) OVER (
        PARTITION BY dept_id ORDER BY salary DESC
        ROWS BETWEEN UNBOUNDED PRECEDING AND UNBOUNDED FOLLOWING
    ) as lowest_earner
FROM employees;
```

**重要**：LAST_VALUE 必须指定完整窗口范围！

## 5. 聚合分析函数

### 5.1 SUM/AVG/COUNT/MAX/MIN OVER()

```sql
SELECT 
    order_date,
    amount,
    SUM(amount) OVER (ORDER BY order_date) as running_total,
    AVG(amount) OVER (ORDER BY order_date) as running_avg,
    COUNT(*) OVER (ORDER BY order_date) as running_count,
    MAX(amount) OVER (ORDER BY order_date) as running_max,
    MIN(amount) OVER (ORDER BY order_date) as running_min
FROM orders;
```

### 5.2 分区聚合

```sql
SELECT 
    name,
    dept_id,
    salary,
    SUM(salary) OVER (PARTITION BY dept_id) as dept_total,
    AVG(salary) OVER (PARTITION BY dept_id) as dept_avg,
    COUNT(*) OVER (PARTITION BY dept_id) as dept_count,
    ROUND(salary / SUM(salary) OVER (PARTITION BY dept_id) * 100, 2) as pct_of_dept
FROM employees;
```

### 5.3 移动平均

```sql
-- 7天移动平均
SELECT 
    trade_date,
    price,
    AVG(price) OVER (
        ORDER BY trade_date
        ROWS BETWEEN 6 PRECEDING AND CURRENT ROW
    ) as ma_7d,
    AVG(price) OVER (
        ORDER BY trade_date
        ROWS BETWEEN 29 PRECEDING AND CURRENT ROW
    ) as ma_30d
FROM stock_prices;
```

### 5.4 年初至今累计

```sql
SELECT 
    month,
    revenue,
    SUM(revenue) OVER (
        PARTITION BY EXTRACT(YEAR FROM month)
        ORDER BY month
    ) as ytd_revenue
FROM monthly_sales;
```

## 6. 窗口子句详解

### 6.1 ROWS vs RANGE

```sql
-- ROWS：物理行数
SUM(amount) OVER (
    ORDER BY order_date
    ROWS BETWEEN 2 PRECEDING AND CURRENT ROW
)

-- RANGE：逻辑范围（基于值）
SUM(amount) OVER (
    ORDER BY order_date
    RANGE BETWEEN INTERVAL '2' DAY PRECEDING AND CURRENT ROW
)
```

### 6.2 窗口边界

| 边界 | 说明 |
|------|------|
| `UNBOUNDED PRECEDING` | 分区第一行 |
| `n PRECEDING` | 前 n 行 |
| `CURRENT ROW` | 当前行 |
| `n FOLLOWING` | 后 n 行 |
| `UNBOUNDED FOLLOWING` | 分区最后一行 |

### 6.3 常用窗口定义

```sql
-- 从开头到当前行
ROWS BETWEEN UNBOUNDED PRECEDING AND CURRENT ROW

-- 前后各 3 行
ROWS BETWEEN 3 PRECEDING AND 3 FOLLOWING

-- 当前行到结尾
ROWS BETWEEN CURRENT ROW AND UNBOUNDED FOLLOWING

-- 整个分区
ROWS BETWEEN UNBOUNDED PRECEDING AND UNBOUNDED FOLLOWING
```

## 7. 层次查询（CONNECT BY）

### 7.1 基本语法

```sql
SELECT column_list
FROM table_name
START WITH start_condition
CONNECT BY PRIOR parent_column = child_column;
```

### 7.2 组织架构示例

```sql
SELECT 
    employee_id,
    manager_id,
    name,
    LEVEL,
    LPAD(' ', (LEVEL - 1) * 2) || name as org_chart,
    SYS_CONNECT_BY_PATH(name, ' -> ') as path
FROM employees
START WITH manager_id IS NULL
CONNECT BY PRIOR employee_id = manager_id
ORDER SIBLINGS BY name;
```

**输出**：
```
EMPLOYEE_ID | MANAGER_ID | NAME     | LEVEL | ORG_CHART
------------|------------|----------|-------|------------------
1           |            | CEO      | 1     | CEO
2           | 1          | VP1      | 2     | CEO -> VP1
4           | 2          | Mgr1     | 3     | CEO -> VP1 -> Mgr1
3           | 1          | VP2      | 2     | CEO -> VP2
```

### 7.3 层次查询函数

```sql
SELECT 
    employee_id,
    name,
    LEVEL,
    CONNECT_BY_ROOT name as root_employee,      -- 根节点
    CONNECT_BY_ISLEAF as is_leaf,                -- 是否叶子节点
    CONNECT_BY_ISCYCLE as is_cycle,              -- 是否有循环
    SYS_CONNECT_BY_PATH(name, '/') as full_path  -- 完整路径
FROM employees
START WITH manager_id IS NULL
CONNECT BY NOCYCLE PRIOR employee_id = manager_id;
```

### 7.4 避免循环

```sql
-- 使用 NOCYCLE 防止无限循环
CONNECT BY NOCYCLE PRIOR employee_id = manager_id

-- 检测循环
SELECT *
FROM employees
WHERE CONNECT_BY_ISCYCLE = 1
START WITH manager_id IS NULL
CONNECT BY NOCYCLE PRIOR employee_id = manager_id;
```

### 7.5 实际应用：分类目录

```sql
CREATE TABLE categories (
    category_id NUMBER PRIMARY KEY,
    parent_id NUMBER,
    category_name VARCHAR2(100)
);

-- 查询完整分类树
SELECT 
    LPAD(' ', (LEVEL - 1) * 2) || category_name as tree_view,
    category_id,
    parent_id,
    LEVEL,
    SYS_CONNECT_BY_PATH(category_name, ' > ') as full_path
FROM categories
START WITH parent_id IS NULL
CONNECT BY PRIOR category_id = parent_id
ORDER SIBLINGS BY category_name;
```

## 8. 综合实战案例

### 案例 1：销售排行榜

```sql
-- 每月销售冠军
SELECT *
FROM (
    SELECT 
        TO_CHAR(order_date, 'YYYY-MM') as month,
        salesperson,
        SUM(amount) as total_sales,
        RANK() OVER (
            PARTITION BY TO_CHAR(order_date, 'YYYY-MM')
            ORDER BY SUM(amount) DESC
        ) as rank
    FROM orders
    GROUP BY TO_CHAR(order_date, 'YYYY-MM'), salesperson
)
WHERE rank = 1
ORDER BY month;
```

### 案例 2：留存率分析

```sql
-- 用户次日留存
WITH user_activity AS (
    SELECT DISTINCT
        user_id,
        TRUNC(login_time) as login_date
    FROM login_logs
),
retention AS (
    SELECT 
        a.login_date,
        COUNT(DISTINCT a.user_id) as day0_users,
        COUNT(DISTINCT b.user_id) as day1_users
    FROM user_activity a
    LEFT JOIN user_activity b 
        ON a.user_id = b.user_id
        AND b.login_date = a.login_date + 1
    GROUP BY a.login_date
)
SELECT 
    login_date,
    day0_users,
    day1_users,
    ROUND(day1_users / NULLIF(day0_users, 0) * 100, 2) as retention_rate
FROM retention
ORDER BY login_date;
```

### 案例 3：股票技术分析

```sql
SELECT 
    trade_date,
    close_price,
    -- 5日均线
    AVG(close_price) OVER (
        ORDER BY trade_date
        ROWS BETWEEN 4 PRECEDING AND CURRENT ROW
    ) as ma5,
    -- 20日均线
    AVG(close_price) OVER (
        ORDER BY trade_date
        ROWS BETWEEN 19 PRECEDING AND CURRENT ROW
    ) as ma20,
    -- 涨跌幅
    ROUND((close_price - LAG(close_price) OVER (ORDER BY trade_date)) / 
          LAG(close_price) OVER (ORDER BY trade_date) * 100, 2) as change_pct,
    -- 成交量排名
    NTILE(10) OVER (ORDER BY volume DESC) as volume_decile
FROM stock_prices
WHERE stock_code = 'AAPL'
ORDER BY trade_date;
```

## 9. 性能优化

### 9.1 索引建议

```sql
-- 为 PARTITION BY 和 ORDER BY 列创建索引
CREATE INDEX idx_emp_dept_sal ON employees(dept_id, salary DESC);

-- 为窗口函数常用列创建索引
CREATE INDEX idx_orders_date_amount ON orders(order_date, amount);
```

### 9.2 避免过度使用

```sql
-- ❌ 多个复杂窗口函数
SELECT 
    SUM(...) OVER (...),
    AVG(...) OVER (...),
    RANK() OVER (...),
    LAG(...) OVER (...)
FROM large_table;

-- ✅ 分解为多个简单查询或使用物化视图
```

### 9.3 使用 Hint

```sql
SELECT /*+ WINDOW_SORT */ *
FROM (
    SELECT 
        name,
        RANK() OVER (ORDER BY salary DESC) as rank
    FROM employees
);
```

## 10. 最佳实践

### 10.1 函数选择

✅ 唯一排名 → ROW_NUMBER  
✅ 允许并列且跳号 → RANK  
✅ 允许并列不跳号 → DENSE_RANK  
✅ 均匀分组 → NTILE  
✅ 前后行访问 → LAG/LEAD  
✅ 累积计算 → SUM/AVG OVER  

### 10.2 代码规范

✅ 明确指定窗口范围  
✅ 为复杂查询添加注释  
✅ 测试边界情况  
✅ 监控执行计划  

### 10.3 常见陷阱

❌ LAST_VALUE 未指定完整窗口  
❌ 忽略 NULL 值的影响  
❌ 过度嵌套窗口函数  
❌ 忘记 PARTITION BY 导致全表计算

## 11. 总结

### 11.1 核心知识点

✅ **排名函数**：ROW_NUMBER、RANK、DENSE_RANK、NTILE  
✅ **偏移函数**：LAG、LEAD  
✅ **首尾值**：FIRST_VALUE、LAST_VALUE  
✅ **聚合分析**：SUM、AVG、COUNT OVER  
✅ **窗口子句**：ROWS/RANGE BETWEEN  
✅ **层次查询**：CONNECT BY、LEVEL  

### 11.2 应用场景

📌 Top N 查询 → ROW_NUMBER  
📌 排名分析 → RANK/DENSE_RANK  
📌 趋势分析 → LAG/LEAD  
📌 累积统计 → SUM OVER  
📌 移动平均 → AVG OVER + ROWS  
📌 树形结构 → CONNECT BY
