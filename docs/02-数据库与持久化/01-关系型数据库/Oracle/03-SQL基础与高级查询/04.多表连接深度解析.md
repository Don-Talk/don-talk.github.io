## 1. 连接类型总览

<!-- TODO: 插入图片：JOIN 类型对比图 -->

| 连接类型 | 说明 | 返回结果 |
|---------|------|---------|
| **INNER JOIN** | 内连接 | 只返回匹配的行 |
| **LEFT JOIN** | 左外连接 | 左表全部 + 右表匹配 |
| **RIGHT JOIN** | 右外连接 | 右表全部 + 左表匹配 |
| **FULL JOIN** | 全外连接 | 两表全部，无匹配填 NULL |
| **CROSS JOIN** | 笛卡尔积 | 两表行的所有组合 |
| **SELF JOIN** | 自连接 | 表与自身连接 |

## 2. INNER JOIN（内连接）

### 2.1 基本语法

```sql
-- ANSI 标准语法（推荐）
SELECT e.employee_id, e.name, d.department_name
FROM employees e
INNER JOIN departments d ON e.dept_id = d.dept_id;

-- Oracle 传统语法（旧式）
SELECT e.employee_id, e.name, d.department_name
FROM employees e, departments d
WHERE e.dept_id = d.dept_id;
```

### 2.2 多表连接

```sql
-- 三表连接
SELECT 
    e.employee_id,
    e.name,
    d.department_name,
    l.city
FROM employees e
INNER JOIN departments d ON e.dept_id = d.dept_id
INNER JOIN locations l ON d.location_id = l.location_id;

-- 四表连接
SELECT 
    e.name,
    d.department_name,
    l.city,
    c.country_name
FROM employees e
INNER JOIN departments d ON e.dept_id = d.dept_id
INNER JOIN locations l ON d.location_id = l.location_id
INNER JOIN countries c ON l.country_id = c.country_id;
```

### 2.3 连接条件

```sql
-- 等值连接
SELECT * FROM emp e INNER JOIN dept d ON e.dept_id = d.dept_id;

-- 非等值连接
SELECT 
    e.name,
    e.salary,
    s.grade_level
FROM employees e
INNER JOIN salary_grades s 
    ON e.salary BETWEEN s.min_salary AND s.max_salary;

-- 复合条件
SELECT * 
FROM orders o
INNER JOIN order_items oi 
    ON o.order_id = oi.order_id 
    AND oi.quantity > 10;
```

## 3. OUTER JOIN（外连接）

### 3.1 LEFT JOIN（左外连接）

返回左表所有行，右表无匹配则填 NULL。

```sql
-- 查询所有员工及其部门（包括没有部门的员工）
SELECT 
    e.employee_id,
    e.name,
    d.department_name
FROM employees e
LEFT JOIN departments d ON e.dept_id = d.dept_id;

-- Oracle 传统语法（+ 在右边）
SELECT e.employee_id, e.name, d.department_name
FROM employees e, departments d
WHERE e.dept_id = d.dept_id(+);
```

**应用场景**：
- 查找没有订单的客户
- 查找没有员工的部门
- 查找未分配的任务

```sql
-- 查找没有员工的部门
SELECT d.department_name
FROM departments d
LEFT JOIN employees e ON d.dept_id = e.dept_id
WHERE e.employee_id IS NULL;
```

### 3.2 RIGHT JOIN（右外连接）

返回右表所有行，左表无匹配则填 NULL。

```sql
-- 查询所有部门及其员工（包括没有员工的部门）
SELECT 
    e.employee_id,
    e.name,
    d.department_name
FROM employees e
RIGHT JOIN departments d ON e.dept_id = d.dept_id;

-- Oracle 传统语法（+ 在左边）
SELECT e.employee_id, e.name, d.department_name
FROM employees e, departments d
WHERE e.dept_id(+) = d.dept_id;
```

**提示**：LEFT JOIN 和 RIGHT JOIN 可以互换，通常推荐使用 LEFT JOIN。

### 3.3 FULL JOIN（全外连接）

返回两表所有行，无匹配则对应侧填 NULL。

```sql
-- 查询所有员工和所有部门
SELECT 
    e.employee_id,
    e.name,
    d.department_name
FROM employees e
FULL JOIN departments d ON e.dept_id = d.dept_id;

-- Oracle 传统语法不支持 FULL JOIN，需使用 UNION
SELECT e.employee_id, e.name, d.department_name
FROM employees e, departments d
WHERE e.dept_id = d.dept_id(+)
UNION ALL
SELECT e.employee_id, e.name, d.department_name
FROM employees e, departments d
WHERE e.dept_id(+) = d.dept_id
AND e.employee_id IS NULL;
```

**应用场景**：
- 对账系统（两边都可能有不匹配的记录）
- 数据比对

## 4. 特殊连接

### 4.1 CROSS JOIN（笛卡尔积）

返回两表行的所有可能组合。

```sql
-- 生成所有员工和所有部门的组合
SELECT e.name, d.department_name
FROM employees e
CROSS JOIN departments d;

-- 传统语法
SELECT e.name, d.department_name
FROM employees e, departments d;
```

**应用**：
- 生成测试数据
- 日期与产品组合
- 排列组合场景

```sql
-- 生成 2026 年所有日期和产品的组合
SELECT d.date_value, p.product_id
FROM (
    SELECT TRUNC(TO_DATE('2026-01-01', 'YYYY-MM-DD')) + LEVEL - 1 as date_value
    FROM dual
    CONNECT BY LEVEL <= 365
) d
CROSS JOIN products p;
```

### 4.2 SELF JOIN（自连接）

表与自身连接，用于层次结构或比较。

```sql
-- 员工和经理（同一张表）
SELECT 
    e.employee_id,
    e.name as employee_name,
    m.employee_id as manager_id,
    m.name as manager_name
FROM employees e
LEFT JOIN employees m ON e.manager_id = m.employee_id;

-- 查找薪资高于同部门平均水平的员工
SELECT 
    e1.name,
    e1.salary,
    e1.dept_id
FROM employees e1
INNER JOIN (
    SELECT dept_id, AVG(salary) as avg_salary
    FROM employees
    GROUP BY dept_id
) e2 ON e1.dept_id = e2.dept_id
WHERE e1.salary > e2.avg_salary;
```

## 5. NATURAL JOIN 和 USING

### 5.1 NATURAL JOIN

自动使用同名同类型列进行连接。

```sql
-- 自动使用 dept_id 连接（假设两表都有此列）
SELECT *
FROM employees
NATURAL JOIN departments;
```

**缺点**：
- 不够明确，容易出错
- 如果表结构变化（新增同名列），查询结果会改变
- **不推荐使用**

### 5.2 USING 子句

明确指定连接列名。

```sql
-- 使用 USING
SELECT e.name, d.department_name
FROM employees e
JOIN departments d USING (dept_id);

-- 等价于 ON
SELECT e.name, d.department_name
FROM employees e
JOIN departments d ON e.dept_id = d.dept_id;
```

**注意**：USING 中的列不能在 SELECT 中加表前缀。

```sql
-- ✅ 正确
SELECT dept_id, e.name  -- dept_id 不加前缀

-- ❌ 错误
SELECT e.dept_id, e.name
```

## 6. 连接执行顺序和优化

### 6.1 连接顺序

```sql
SELECT *
FROM A
JOIN B ON ...
JOIN C ON ...
JOIN D ON ...;
```

**执行顺序**（由优化器决定，通常是）：
1. 选择最小的表作为驱动表
2. 依次连接其他表
3. 应用过滤条件

### 6.2 影响连接性能的因素

#### 索引

```sql
-- 为外键列创建索引（非常重要！）
CREATE INDEX idx_emp_dept ON employees(dept_id);
CREATE INDEX idx_order_customer ON orders(customer_id);
```

#### 表大小

- 小表作为驱动表
- 大表建立索引

#### 连接方法

Oracle 使用三种连接方法：

**1. Nested Loops Join（嵌套循环）**
- 适合：小表驱动大表
- 特点：快速返回第一行

**2. Hash Join（哈希连接）**
- 适合：大表连接大表
- 特点：吞吐量高

**3. Sort Merge Join（排序合并）**
- 适合：已排序的数据
- 特点：需要额外排序

### 6.3 查看执行计划

```sql
-- 方法 1：EXPLAIN PLAN
EXPLAIN PLAN FOR
SELECT e.name, d.department_name
FROM employees e
JOIN departments d ON e.dept_id = d.dept_id;

SELECT * FROM TABLE(DBMS_XPLAN.DISPLAY);

-- 方法 2：AUTOTRACE
SET AUTOTRACE ON EXPLAIN;
SELECT e.name, d.department_name
FROM employees e
JOIN departments d ON e.dept_id = d.dept_id;
SET AUTOTRACE OFF;
```

## 7. 高级连接技巧

### 7.1 LATERAL JOIN（Oracle 12c+）

```sql
-- 为每个部门查找薪资最高的 3 名员工
SELECT d.department_name, e.name, e.salary
FROM departments d
CROSS APPLY (
    SELECT *
    FROM employees e
    WHERE e.dept_id = d.dept_id
    ORDER BY e.salary DESC
    FETCH FIRST 3 ROWS ONLY
) e;
```

### 7.2 PARTITIONED OUTER JOIN

用于稀疏数据填充。

```sql
-- 为每个部门填充缺失的月份数据
SELECT 
    d.department_name,
    t.month,
    NVL(SUM(e.salary), 0) as total_salary
FROM departments d
LEFT OUTER JOIN employees e PARTITION BY (d.dept_id)
    ON e.dept_id = d.dept_id
CROSS JOIN (
    SELECT LEVEL as month
    FROM dual
    CONNECT BY LEVEL <= 12
) t
GROUP BY d.department_name, t.month
ORDER BY d.department_name, t.month;
```

### 7.3 半连接和反连接

```sql
-- 半连接（SEMI JOIN）- EXISTS
SELECT * FROM departments d
WHERE EXISTS (
    SELECT 1 FROM employees e
    WHERE e.dept_id = d.dept_id
);

-- 反连接（ANTI JOIN）- NOT EXISTS
SELECT * FROM departments d
WHERE NOT EXISTS (
    SELECT 1 FROM employees e
    WHERE e.dept_id = d.dept_id
);

-- 使用 IN
SELECT * FROM departments
WHERE dept_id IN (SELECT dept_id FROM employees);

-- 使用 NOT IN（注意 NULL 问题）
SELECT * FROM departments
WHERE dept_id NOT IN (
    SELECT dept_id FROM employees WHERE dept_id IS NOT NULL
);
```

## 8. 常见陷阱和解决方案

### 8.1 笛卡尔积陷阱

```sql
-- ❌ 忘记写连接条件，产生笛卡尔积
SELECT * FROM employees e, departments d;

-- ✅ 始终写明连接条件
SELECT * FROM employees e, departments d
WHERE e.dept_id = d.dept_id;
```

### 8.2 NULL 值问题

```sql
-- NOT IN 遇到 NULL 会返回空结果
SELECT * FROM departments
WHERE dept_id NOT IN (SELECT dept_id FROM employees);
-- 如果 employees 中有 dept_id 为 NULL，结果为空

-- ✅ 使用 NOT EXISTS 或过滤 NULL
SELECT * FROM departments d
WHERE NOT EXISTS (
    SELECT 1 FROM employees e WHERE e.dept_id = d.dept_id
);
```

### 8.3 重复列名

```sql
-- ❌ 歧义
SELECT dept_id FROM employees e JOIN departments d ON e.dept_id = d.dept_id;

-- ✅ 明确指定表别名
SELECT e.dept_id FROM employees e JOIN departments d ON e.dept_id = d.dept_id;
```

### 8.4 连接过多表

```sql
-- ❌ 连接太多表，性能差
SELECT * 
FROM a JOIN b ON ... 
JOIN c ON ... 
JOIN d ON ... 
JOIN e ON ... 
JOIN f ON ...;

-- ✅ 优化建议：
-- 1. 只选择需要的列
-- 2. 先过滤再连接
-- 3. 考虑创建物化视图
SELECT a.col1, b.col2, c.col3
FROM (SELECT * FROM a WHERE condition) a
JOIN b ON ... 
JOIN c ON ...;
```

## 9. 实战案例

### 案例 1：电商订单分析

```sql
-- 查询每个客户的订单总金额
SELECT 
    c.customer_id,
    c.name,
    COUNT(o.order_id) as order_count,
    SUM(oi.quantity * oi.unit_price) as total_amount
FROM customers c
LEFT JOIN orders o ON c.customer_id = o.customer_id
LEFT JOIN order_items oi ON o.order_id = oi.order_id
GROUP BY c.customer_id, c.name
ORDER BY total_amount DESC;

-- 查找从未下单的客户
SELECT c.customer_id, c.name
FROM customers c
LEFT JOIN orders o ON c.customer_id = o.customer_id
WHERE o.order_id IS NULL;
```

### 案例 2：员工层级查询

```sql
-- 员工及其完整管理层级
SELECT 
    e.employee_id,
    LPAD(' ', (LEVEL - 1) * 2) || e.name as name,
    LEVEL as hierarchy_level,
    SYS_CONNECT_BY_PATH(e.name, ' -> ') as management_chain
FROM employees e
START WITH e.manager_id IS NULL
CONNECT BY PRIOR e.employee_id = e.manager_id;
```

## 10. 最佳实践

### 10.1 编写规范

✅ 使用 ANSI 标准语法（可读性更好）  
✅ 始终使用表别名  
✅ 明确指定连接条件  
✅ 为外键列创建索引  
✅ 先过滤再连接  

### 10.2 性能优化

📌 小表驱动大表  
📌 确保连接列有索引  
📌 避免不必要的列  
📌 使用 EXISTS 代替 DISTINCT  
📌 监控执行计划  

### 10.3 代码示例

```sql
-- ✅ 推荐的写法
SELECT 
    e.employee_id,
    e.name,
    d.department_name
FROM employees e
INNER JOIN departments d ON e.dept_id = d.dept_id
WHERE e.salary > 5000
ORDER BY e.name;

-- 确保有索引
CREATE INDEX idx_emp_dept ON employees(dept_id);
CREATE INDEX idx_emp_salary ON employees(salary);
```

## 11. 总结

### 11.1 核心知识点

✅ **INNER JOIN**：只返回匹配的行  
✅ **LEFT/RIGHT/FULL JOIN**：外连接处理不匹配  
✅ **CROSS JOIN**：笛卡尔积  
✅ **SELF JOIN**：表与自身连接  
✅ **连接优化**：索引、驱动表、执行计划  

### 11.2 常用场景

📌 多表关联查询 → JOIN  
📌 查找不匹配记录 → LEFT JOIN + IS NULL  
📌 层级关系 → SELF JOIN  
📌 存在性检查 → EXISTS（半连接）
