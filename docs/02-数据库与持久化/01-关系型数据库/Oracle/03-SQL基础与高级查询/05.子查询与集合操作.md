# 05子查询与集合操作

**学习目标：**
> * 掌握各种类型的子查询
> * 理解相关子查询的执行机制
> * 熟练使用集合操作符
> * 能够优化复杂查询

---

## 1. 子查询概述

### 1.1 什么是子查询？

子查询是嵌套在其他 SQL 语句中的 SELECT 语句。

```sql
-- 子查询示例
SELECT name, salary
FROM employees
WHERE salary > (SELECT AVG(salary) FROM employees);
```

### 1.2 子查询分类

**按返回结果**：
- **单行子查询**：返回一行一列
- **多行子查询**：返回多行
- **多列子查询**：返回多列
- **表子查询**：返回多行多列

**按执行方式**：
- **非相关子查询**：独立执行一次
- **相关子查询**：依赖外部查询，多次执行

## 2. 单行子查询

### 2.1 WHERE 子句中的子查询

```sql
-- 查询薪资高于平均薪资的员工
SELECT name, salary
FROM employees
WHERE salary > (SELECT AVG(salary) FROM employees);

-- 查询薪资最高的员工
SELECT name, salary
FROM employees
WHERE salary = (SELECT MAX(salary) FROM employees);

-- 查询与某员工同部门的员工
SELECT name, dept_id
FROM employees
WHERE dept_id = (
    SELECT dept_id FROM employees WHERE name = 'John'
);
```

### 2.2 HAVING 子句中的子查询

```sql
-- 查询平均薪资高于公司平均的部门
SELECT dept_id, AVG(salary) as avg_salary
FROM employees
GROUP BY dept_id
HAVING AVG(salary) > (SELECT AVG(salary) FROM employees);
```

### 2.3 FROM 子句中的子查询（内联视图）

```sql
-- 查询每个部门薪资高于该部门平均的员工
SELECT e.name, e.dept_id, e.salary, dept_avg.avg_sal
FROM employees e
INNER JOIN (
    SELECT dept_id, AVG(salary) as avg_sal
    FROM employees
    GROUP BY dept_id
) dept_avg ON e.dept_id = dept_avg.dept_id
WHERE e.salary > dept_avg.avg_sal;
```

### 2.4 SELECT 子句中的子查询（标量子查询）

```sql
-- 查询员工及其部门平均薪资
SELECT 
    name,
    salary,
    dept_id,
    (SELECT AVG(salary) FROM employees e2 WHERE e2.dept_id = e1.dept_id) as dept_avg_salary
FROM employees e1;
```

**注意**：标量子查询必须只返回一行一列，否则报错。

## 3. 多行子查询

### 3.1 IN 操作符

```sql
-- 查询在特定部门工作的员工
SELECT name, dept_id
FROM employees
WHERE dept_id IN (10, 20, 30);

-- 使用子查询
SELECT name, dept_id
FROM employees
WHERE dept_id IN (
    SELECT dept_id FROM departments WHERE location = 'NEW YORK'
);
```

### 3.2 EXISTS 操作符

```sql
-- 查询有员工的部门
SELECT department_name
FROM departments d
WHERE EXISTS (
    SELECT 1 FROM employees e WHERE e.dept_id = d.dept_id
);

-- 查询没有员工的部门
SELECT department_name
FROM departments d
WHERE NOT EXISTS (
    SELECT 1 FROM employees e WHERE e.dept_id = d.dept_id
);
```

### 3.3 ANY/SOME 操作符

```sql
-- 查询薪资高于任意一个部门经理的员工
SELECT name, salary
FROM employees
WHERE salary > ANY (
    SELECT salary FROM employees WHERE job_title = 'MANAGER'
);

-- 等价于 > MIN
SELECT name, salary
FROM employees
WHERE salary > (
    SELECT MIN(salary) FROM employees WHERE job_title = 'MANAGER'
);
```

### 3.4 ALL 操作符

```sql
-- 查询薪资高于所有部门经理的员工
SELECT name, salary
FROM employees
WHERE salary > ALL (
    SELECT salary FROM employees WHERE job_title = 'MANAGER'
);

-- 等价于 > MAX
SELECT name, salary
FROM employees
WHERE salary > (
    SELECT MAX(salary) FROM employees WHERE job_title = 'MANAGER'
);
```

### 3.5 对比总结

| 操作符 | 含义 | 等价 |
|--------|------|------|
| `IN` | 在列表中 | `= ANY` |
| `NOT IN` | 不在列表中 | `<> ALL` |
| `ANY` / `SOME` | 任一值 | - |
| `ALL` | 所有值 | - |

## 4. 相关子查询

### 4.1 基本概念

相关子查询引用了外部查询的列，对外部查询的每一行都执行一次。

```sql
-- 查询薪资高于本部门平均的员工
SELECT name, dept_id, salary
FROM employees e1
WHERE salary > (
    SELECT AVG(salary)
    FROM employees e2
    WHERE e2.dept_id = e1.dept_id  -- 引用外部查询
);
```

**执行过程**：
1. 获取外部查询的第一行
2. 执行子查询（使用该行的值）
3. 根据子查询结果决定是否保留该行
4. 处理下一行，重复上述步骤

### 4.2 EXISTS 相关子查询

```sql
-- 查询有高薪员工（> 10000）的部门
SELECT department_name
FROM departments d
WHERE EXISTS (
    SELECT 1
    FROM employees e
    WHERE e.dept_id = d.dept_id
    AND e.salary > 10000
);
```

### 4.3 UPDATE 中的相关子查询

```sql
-- 更新员工薪资为部门平均薪资
UPDATE employees e1
SET salary = (
    SELECT AVG(salary)
    FROM employees e2
    WHERE e2.dept_id = e1.dept_id
);
```

### 4.4 DELETE 中的相关子查询

```sql
-- 删除薪资低于部门平均的员工
DELETE FROM employees e1
WHERE salary < (
    SELECT AVG(salary)
    FROM employees e2
    WHERE e2.dept_id = e1.dept_id
);
```

## 5. 子查询性能优化

### 5.1 EXISTS vs IN

```sql
-- IN（子查询结果集小时使用）
SELECT * FROM departments
WHERE dept_id IN (SELECT dept_id FROM employees);

-- EXISTS（子查询结果集大时使用）
SELECT * FROM departments d
WHERE EXISTS (SELECT 1 FROM employees e WHERE e.dept_id = d.dept_id);
```

**选择原则**：
- 子查询结果小 → 用 IN
- 子查询结果大 → 用 EXISTS
- EXISTS 找到第一个匹配就停止

### 5.2 避免不必要的子查询

```sql
-- ❌ 低效：子查询执行多次
SELECT name, 
    (SELECT department_name FROM departments d WHERE d.dept_id = e.dept_id) as dept_name
FROM employees e;

-- ✅ 高效：使用 JOIN
SELECT e.name, d.department_name
FROM employees e
JOIN departments d ON e.dept_id = d.dept_id;
```

### 5.3 使用 WITH 子句（CTE）

```sql
-- 可复用的子查询
WITH dept_avg AS (
    SELECT dept_id, AVG(salary) as avg_sal
    FROM employees
    GROUP BY dept_id
)
SELECT e.name, e.salary, da.avg_sal
FROM employees e
JOIN dept_avg da ON e.dept_id = da.dept_id
WHERE e.salary > da.avg_sal;
```

**优势**：
- 提高可读性
- 避免重复计算
- Oracle 可能物化 CTE 结果

## 6. 集合操作

### 6.1 UNION（并集，去重）

```sql
-- 合并两个查询结果，去除重复
SELECT name FROM employees
UNION
SELECT name FROM customers;
```

**特点**：
- 自动去重
- 按第一列排序
- 性能较慢（需要排序去重）

### 6.2 UNION ALL（并集，不去重）

```sql
-- 合并两个查询结果，保留重复
SELECT name FROM employees
UNION ALL
SELECT name FROM customers;
```

**特点**：
- 不去重
- 不排序
- 性能更好

**对比**：
```sql
-- UNION（慢）
SELECT dept_id FROM employees
UNION
SELECT dept_id FROM departments;

-- UNION ALL（快）
SELECT dept_id FROM employees
UNION ALL
SELECT dept_id FROM departments;
```

### 6.3 INTERSECT（交集）

```sql
-- 返回两个查询的共同结果
SELECT dept_id FROM employees
INTERSECT
SELECT dept_id FROM departments;
```

**应用**：查找同时满足两个条件的记录

```sql
-- 查找既有员工又有预算的部门
SELECT dept_id FROM employees
INTERSECT
SELECT dept_id FROM budgets;
```

### 6.4 MINUS（差集）

```sql
-- 返回在第一个查询中但不在第二个查询中的结果
SELECT dept_id FROM departments
MINUS
SELECT dept_id FROM employees;
```

**应用**：查找没有员工的部门

```sql
-- 查找从未下过订单的客户
SELECT customer_id FROM customers
MINUS
SELECT customer_id FROM orders;
```

### 6.5 集合操作规则

✅ **列数必须相同**  
✅ **数据类型必须兼容**  
✅ **结果集列名以第一个查询为准**  
✅ **ORDER BY 只能放在最后**  

```sql
-- ✅ 正确
SELECT id, name FROM employees
UNION
SELECT id, name FROM customers
ORDER BY name;

-- ❌ 错误
SELECT id, name FROM employees ORDER BY name
UNION
SELECT id, name FROM customers;
```

## 7. WITH 子句（CTE）

### 7.1 基本语法

```sql
WITH cte_name AS (
    SELECT ...
)
SELECT * FROM cte_name;
```

### 7.2 单个 CTE

```sql
-- 查询薪资高于部门平均的员工
WITH dept_avg AS (
    SELECT dept_id, AVG(salary) as avg_salary
    FROM employees
    GROUP BY dept_id
)
SELECT e.name, e.salary, da.avg_salary
FROM employees e
JOIN dept_avg da ON e.dept_id = da.dept_id
WHERE e.salary > da.avg_salary;
```

### 7.3 多个 CTE

```sql
WITH 
dept_stats AS (
    SELECT 
        dept_id,
        COUNT(*) as emp_count,
        AVG(salary) as avg_salary
    FROM employees
    GROUP BY dept_id
),
high_perf_depts AS (
    SELECT dept_id
    FROM dept_stats
    WHERE avg_salary > 8000
)
SELECT d.department_name, ds.emp_count, ds.avg_salary
FROM departments d
JOIN dept_stats ds ON d.dept_id = ds.dept_id
WHERE d.dept_id IN (SELECT dept_id FROM high_perf_depts);
```

### 7.4 递归 CTE（Oracle 11g R2+）

```sql
-- 生成 1-100 的数字
WITH numbers (n) AS (
    SELECT 1 FROM dual
    UNION ALL
    SELECT n + 1 FROM numbers WHERE n < 100
)
SELECT * FROM numbers;

-- 层级查询
WITH emp_hierarchy (employee_id, name, manager_id, level) AS (
    SELECT employee_id, name, manager_id, 1
    FROM employees
    WHERE manager_id IS NULL
    UNION ALL
    SELECT e.employee_id, e.name, e.manager_id, eh.level + 1
    FROM employees e
    JOIN emp_hierarchy eh ON e.manager_id = eh.employee_id
)
SELECT * FROM emp_hierarchy;
```

## 8. 实战案例

### 案例 1：复杂报表查询

```sql
-- 查询每个部门的薪资统计，并标记高于公司平均的部门
WITH company_avg AS (
    SELECT AVG(salary) as avg_sal FROM employees
),
dept_stats AS (
    SELECT 
        dept_id,
        COUNT(*) as emp_count,
        AVG(salary) as avg_salary,
        MAX(salary) as max_salary,
        MIN(salary) as min_salary
    FROM employees
    GROUP BY dept_id
)
SELECT 
    d.department_name,
    ds.emp_count,
    ds.avg_salary,
    ds.max_salary,
    ds.min_salary,
    CASE 
        WHEN ds.avg_salary > ca.avg_sal THEN '高于平均'
        ELSE '低于平均'
    END as status
FROM departments d
JOIN dept_stats ds ON d.dept_id = ds.dept_id
CROSS JOIN company_avg ca
ORDER BY ds.avg_salary DESC;
```

### 案例 2：分页查询

```sql
-- 第 2 页，每页 10 条
WITH ranked_employees AS (
    SELECT 
        employee_id,
        name,
        salary,
        ROW_NUMBER() OVER (ORDER BY salary DESC) as rn
    FROM employees
)
SELECT * FROM ranked_employees
WHERE rn BETWEEN 11 AND 20;
```

## 9. 最佳实践

### 9.1 性能优化

✅ 优先使用 JOIN 替代子查询  
✅ 小结果集用 IN，大结果集用 EXISTS  
✅ 使用 WITH 子句避免重复计算  
✅ 为子查询中的列创建索引  
✅ 避免在子查询中使用函数  

### 9.2 代码规范

✅ 缩进清晰，易于阅读  
✅ 为子查询起有意义的别名  
✅ 使用注释说明复杂逻辑  
✅ 避免过深的嵌套（不超过 3 层）  

### 9.3 常见陷阱

❌ NOT IN 遇到 NULL 返回空结果  
❌ 相关子查询性能差（逐行执行）  
❌ 标量子查询返回多行报错  
❌ 集合操作列数不匹配

## 10. 总结

### 10.1 核心知识点

✅ **单行子查询**：返回一行一列  
✅ **多行子查询**：IN、EXISTS、ANY、ALL  
✅ **相关子查询**：依赖外部查询  
✅ **集合操作**：UNION、INTERSECT、MINUS  
✅ **CTE**：提高可读性和性能  

### 10.2 选择指南

📌 简单条件 → 普通子查询  
📌 存在性检查 → EXISTS  
📌 列表匹配 → IN  
📌 复杂逻辑 → CTE  
📌 合并结果 → UNION ALL
