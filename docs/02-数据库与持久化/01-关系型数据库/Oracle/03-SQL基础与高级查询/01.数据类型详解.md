# 01数据类型详解

**学习目标：**
> * 掌握 Oracle 所有常用数据类型
> * 理解不同数据类型的存储特性和性能影响
> * 能够根据业务需求选择合适的数据类型
> * 避免常见的数据类型使用陷阱

---

## 1. 数据类型总览

Oracle 支持丰富的数据类型，主要分为以下几类：

| 类别 | 数据类型 | 用途 |
|------|---------|------|
| **字符型** | `CHAR`、`VARCHAR2`、`NCHAR`、`NVARCHAR2`、`CLOB` | 存储文本 |
| **数值型** | `NUMBER`、`FLOAT`、`BINARY_FLOAT` | 存储数字 |
| **日期时间** | `DATE`、`TIMESTAMP`、`INTERVAL` | 存储时间 |
| **二进制** | `RAW`、`LONG RAW`、`BLOB`、`BFILE` | 存储二进制数据 |
| **其他** | `ROWID`、`UROWID`、`XMLType` | 特殊用途 |

## 2. 字符数据类型

### 2.1 CHAR vs VARCHAR2

#### CHAR（定长字符串）

```sql
-- 创建测试表
CREATE TABLE test_char (
    id NUMBER,
    name CHAR(10)
);

-- 插入数据
INSERT INTO test_char VALUES (1, 'ABC');

-- 查询（注意尾部空格）
SELECT name, LENGTH(name), DUMP(name) FROM test_char;
```

**特点**：
- 固定长度，不足部分用空格填充
- 最大长度：2000 字节
- 适合存储固定长度的数据（如身份证号、电话号码）

**缺点**：
- 浪费存储空间
- 比较时需要注意空格

#### VARCHAR2（变长字符串）

```sql
CREATE TABLE test_varchar (
    id NUMBER,
    name VARCHAR2(100)
);

INSERT INTO test_varchar VALUES (1, 'ABC');

-- 查询
SELECT name, LENGTH(name), DUMP(name) FROM test_varchar;
```

**特点**：
- 可变长度，只存储实际数据
- 最大长度：4000 字节（SQL），32767 字节（PL/SQL）
- 节省存储空间
- **推荐使用**

**对比示例**：
```sql
-- CHAR 会填充空格
SELECT '"' || CAST('ABC' AS CHAR(10)) || '"' as char_value FROM dual;
-- 输出："ABC       "

-- VARCHAR2 不会填充
SELECT '"' || CAST('ABC' AS VARCHAR2(10)) || '"' as varchar_value FROM dual;
-- 输出："ABC"
```

### 2.2 NCHAR vs NVARCHAR2

用于存储 Unicode 字符（多语言支持）。

```sql
CREATE TABLE test_nchar (
    id NUMBER,
    chinese_name NCHAR(10),
    english_name NVARCHAR2(50)
);

INSERT INTO test_nchar VALUES (1, '张三', 'John Smith');
```

**特点**：
- 支持国家字符集
- 适合存储中文、日文等多字节字符
- 占用空间更大（通常 2-3 倍）

### 2.3 CLOB（字符大对象）

存储大量文本数据（如文章、日志）。

```sql
CREATE TABLE articles (
    id NUMBER PRIMARY KEY,
    title VARCHAR2(200),
    content CLOB,
    create_date DATE DEFAULT SYSDATE
);

-- 插入大文本
INSERT INTO articles (id, title, content)
VALUES (1, 'Oracle教程', RPAD('A', 10000, 'A'));

-- 查询
SELECT id, title, DBMS_LOB.GETLENGTH(content) as content_length
FROM articles;
```

**特点**：
- 最大容量：4 GB
- 适合存储超长文本
- 需要使用 DBMS_LOB 包操作

**常用操作**：
```sql
-- 追加内容
DECLARE
    v_clob CLOB;
BEGIN
    SELECT content INTO v_clob FROM articles WHERE id = 1 FOR UPDATE;
    DBMS_LOB.WRITEAPPEND(v_clob, LENGTH('新内容'), '新内容');
    COMMIT;
END;

-- 截取内容
SELECT DBMS_LOB.SUBSTR(content, 100, 1) as first_100_chars
FROM articles WHERE id = 1;
```

### 2.4 LONG（已废弃）

**不推荐使用**，Oracle 建议用 CLOB 替代。

**限制**：
- 每个表只能有一个 LONG 列
- 不能用于 WHERE 条件
- 不能建立索引

## 3. 数值数据类型

### 3.1 NUMBER

Oracle 最常用的数值类型。

```sql
CREATE TABLE test_number (
    id NUMBER,
    price NUMBER(10,2),      -- 总共10位，小数2位
    quantity NUMBER(5),       -- 整数5位
    salary NUMBER             -- 不限精度
);

INSERT INTO test_number VALUES (1, 99.99, 100, 5000.50);
```

**语法**：`NUMBER(p,s)`
- `p`（precision）：总位数（1-38）
- `s`（scale）：小数位数（-84 到 127）

**示例**：
```sql
-- 正 scale：小数位数
NUMBER(10,2)  -- 最大值：99999999.99

-- 零 scale：整数
NUMBER(5,0)   -- 最大值：99999

-- 负 scale：四舍五入到十位、百位等
NUMBER(10,-2) -- 1234 存储为 1200
```

**最佳实践**：
✅ 明确指定精度和标度  
✅ 金额使用 `NUMBER(10,2)` 或 `NUMBER(15,2)`  
✅ 计数使用 `NUMBER(10,0)`  
✅ 避免使用不带精度的 `NUMBER`（除非必要）  

### 3.2 FLOAT / BINARY_FLOAT / BINARY_DOUBLE

用于科学计算，支持 IEEE 754 标准。

```sql
CREATE TABLE scientific_data (
    id NUMBER,
    value_float FLOAT,              -- Oracle FLOAT（实际上是 NUMBER）
    value_binary_float BINARY_FLOAT, -- 单精度（4字节）
    value_binary_double BINARY_DOUBLE -- 双精度（8字节）
);

INSERT INTO scientific_data VALUES (
    1, 
    3.14159265358979,
    3.14159265358979,
    3.14159265358979
);

-- 查看精度差异
SELECT 
    value_float,
    value_binary_float,
    value_binary_double
FROM scientific_data;
```

**对比**：

| 类型 | 精度 | 存储 | 适用场景 |
|------|------|------|---------|
| `NUMBER` | 最高 | 可变 | 金融、精确计算 |
| `BINARY_FLOAT` | 7位有效数字 | 4字节 | 科学计算、图形 |
| `BINARY_DOUBLE` | 15位有效数字 | 8字节 | 高精度科学计算 |

**注意**：浮点数存在精度问题，不适合金融计算！

```sql
-- 浮点数精度问题
SELECT 0.1 + 0.2 as result FROM dual;
-- 可能返回 0.30000000000000004

-- 使用 NUMBER 避免此问题
SELECT CAST(0.1 AS NUMBER) + CAST(0.2 AS NUMBER) as result FROM dual;
-- 返回 0.3
```

## 4. 日期时间数据类型

### 4.1 DATE

存储日期和时间（年月日时分秒）。

```sql
CREATE TABLE events (
    id NUMBER,
    event_name VARCHAR2(100),
    event_date DATE
);

-- 插入日期
INSERT INTO events VALUES (1, '会议', SYSDATE);
INSERT INTO events VALUES (2, '生日', TO_DATE('2026-05-19', 'YYYY-MM-DD'));
INSERT INTO events VALUES (3, '截止', TO_DATE('2026-12-31 23:59:59', 'YYYY-MM-DD HH24:MI:SS'));

-- 查询
SELECT 
    event_name,
    event_date,
    TO_CHAR(event_date, 'YYYY-MM-DD HH24:MI:SS') as formatted_date
FROM events;
```

**常用函数**：
```sql
-- 当前日期时间
SELECT SYSDATE FROM dual;

-- 日期运算
SELECT 
    SYSDATE,
    SYSDATE + 7 as next_week,
    SYSDATE - 30 as last_month,
    ADD_MONTHS(SYSDATE, 3) as three_months_later
FROM dual;

-- 日期差值
SELECT 
    TO_DATE('2026-12-31', 'YYYY-MM-DD') - SYSDATE as days_remaining
FROM dual;

-- 提取部分
SELECT 
    EXTRACT(YEAR FROM SYSDATE) as year,
    EXTRACT(MONTH FROM SYSDATE) as month,
    EXTRACT(DAY FROM SYSDATE) as day
FROM dual;
```

### 4.2 TIMESTAMP

比 DATE 更高精度，支持小数秒和时区。

```sql
CREATE TABLE transactions (
    id NUMBER,
    amount NUMBER(10,2),
    transaction_time TIMESTAMP(6),  -- 6位小数秒
    transaction_tz TIMESTAMP(6) WITH TIME ZONE
);

INSERT INTO transactions VALUES (
    1,
    100.00,
    SYSTIMESTAMP,
    SYSTIMESTAMP
);

SELECT 
    transaction_time,
    transaction_tz,
    TO_CHAR(transaction_time, 'YYYY-MM-DD HH24:MI:SS.FF6') as formatted
FROM transactions;
```

**精度**：
- `TIMESTAMP(0)`：秒
- `TIMESTAMP(3)`：毫秒
- `TIMESTAMP(6)`：微秒（默认）
- `TIMESTAMP(9)`：纳秒

### 4.3 INTERVAL

存储时间间隔。

```sql
-- INTERVAL YEAR TO MONTH
SELECT 
    INTERVAL '5' YEAR as five_years,
    INTERVAL '3' MONTH as three_months,
    INTERVAL '2-6' YEAR TO MONTH as two_years_six_months
FROM dual;

-- INTERVAL DAY TO SECOND
SELECT 
    INTERVAL '10' DAY as ten_days,
    INTERVAL '5' HOUR as five_hours,
    INTERVAL '30' MINUTE as thirty_minutes,
    INTERVAL '45' SECOND as fortyfive_seconds,
    INTERVAL '1 2:30:45' DAY TO SECOND as one_day_two_hours
FROM dual;

-- 实际应用：计算到期日
SELECT 
    SYSDATE as today,
    SYSDATE + INTERVAL '30' DAY as due_date
FROM dual;
```

### 4.4 日期格式化

```sql
-- 常用格式模型
SELECT 
    TO_CHAR(SYSDATE, 'YYYY-MM-DD') as iso_date,
    TO_CHAR(SYSDATE, 'YYYY/MM/DD HH24:MI:SS') as datetime,
    TO_CHAR(SYSDATE, 'DD-MON-YYYY') as oracle_format,
    TO_CHAR(SYSDATE, 'Day, Month DD, YYYY') as long_format,
    TO_CHAR(SYSDATE, 'Q') as quarter,
    TO_CHAR(SYSDATE, 'WW') as week_of_year
FROM dual;

-- 字符串转日期
SELECT 
    TO_DATE('2026-05-19', 'YYYY-MM-DD') as date1,
    TO_DATE('05/19/2026', 'MM/DD/YYYY') as date2,
    TO_TIMESTAMP('2026-05-19 14:30:00.123456', 'YYYY-MM-DD HH24:MI:SS.FF6') as ts
FROM dual;
```

## 5. LOB 数据类型

### 5.1 BLOB（二进制大对象）

存储图片、音频、视频等二进制数据。

```sql
CREATE TABLE documents (
    id NUMBER PRIMARY KEY,
    filename VARCHAR2(200),
    file_data BLOB,
    upload_date DATE DEFAULT SYSDATE
);

-- 插入 BLOB（从文件加载需要 PL/SQL）
DECLARE
    v_blob BLOB;
    v_bfile BFILE;
BEGIN
    INSERT INTO documents (id, filename, file_data)
    VALUES (1, 'photo.jpg', EMPTY_BLOB())
    RETURNING file_data INTO v_blob;
    
    v_bfile := BFILENAME('MY_DIR', 'photo.jpg');
    DBMS_LOB.FILEOPEN(v_bfile, DBMS_LOB.FILE_READONLY);
    DBMS_LOB.LOADFROMFILE(v_blob, v_bfile, DBMS_LOB.GETLENGTH(v_bfile));
    DBMS_LOB.FILECLOSE(v_bfile);
    COMMIT;
END;
```

### 5.2 CLOB vs BLOB

| 特性 | CLOB | BLOB |
|------|------|------|
| **用途** | 文本 | 二进制 |
| **字符集** | 是 | 否 |
| **最大大小** | 4 GB | 4 GB |
| **典型应用** | 文章、日志 | 图片、文件 |

### 5.3 BFILE（外部文件引用）

存储指向操作系统文件的指针。

```sql
CREATE DIRECTORY MY_DIR AS '/u01/files';

CREATE TABLE external_files (
    id NUMBER,
    filename VARCHAR2(200),
    file_pointer BFILE
);

INSERT INTO external_files VALUES (
    1,
    'document.pdf',
    BFILENAME('MY_DIR', 'document.pdf')
);

-- 读取文件
DECLARE
    v_bfile BFILE;
    v_raw RAW(100);
BEGIN
    SELECT file_pointer INTO v_bfile FROM external_files WHERE id = 1;
    DBMS_LOB.FILEOPEN(v_bfile);
    DBMS_LOB.READ(v_bfile, 100, 1, v_raw);
    DBMS_LOB.FILECLOSE(v_bfile);
END;
```

## 6. 其他数据类型

### 6.1 RAW

存储原始二进制数据（如哈希值、加密数据）。

```sql
CREATE TABLE security_data (
    id NUMBER,
    password_hash RAW(32),
    session_id RAW(16)
);

-- 插入 RAW 数据
INSERT INTO security_data VALUES (
    1,
    HEXTORAW('5F4DCC3B5AA765D61D8327DEB882CF99'),
    SYS_GUID()
);

-- 查询
SELECT 
    id,
    RAWTOHEX(password_hash) as hash_hex
FROM security_data;
```

### 6.2 ROWID

每行数据的物理地址（伪列）。

```sql
-- 查看 ROWID
SELECT 
    rowid,
    employee_id,
    name
FROM employees
WHERE rownum <= 5;

-- ROWID 结构：OOOOOOFFFBBBBBBRRR
-- O: 对象号, F: 文件号, B: 块号, R: 行号

-- 快速定位行（最快访问方式）
SELECT * FROM employees
WHERE rowid = 'AAAR3qAAEAAAACHAAA';
```

### 6.3 XMLType

存储和查询 XML 数据。

```sql
CREATE TABLE xml_documents (
    id NUMBER,
    doc XMLType
);

INSERT INTO xml_documents VALUES (
    1,
    XMLType('<?xml version="1.0"?>
             <employee>
               <id>1</id>
               <name>John</name>
               <dept>IT</dept>
             </employee>')
);

-- 查询 XML
SELECT 
    e.doc.extract('/employee/name/text()').getStringVal() as name
FROM xml_documents e
WHERE e.doc.existsNode('/employee/dept[text()="IT"]') = 1;
```

## 7. 数据类型选择指南

### 7.1 字符数据

| 场景 | 推荐类型 | 示例 |
|------|---------|------|
| 固定长度代码 | `CHAR(n)` | 身份证号、邮编 |
| 可变文本 | `VARCHAR2(n)` | 姓名、地址 |
| 多语言文本 | `NVARCHAR2(n)` | 国际化应用 |
| 超长文本 | `CLOB` | 文章、评论 |

### 7.2 数值数据

| 场景 | 推荐类型 | 示例 |
|------|---------|------|
| 金额 | `NUMBER(10,2)` | 价格、工资 |
| 数量 | `NUMBER(10,0)` | 库存、计数 |
| 百分比 | `NUMBER(5,2)` | 利率、折扣 |
| 科学计算 | `BINARY_DOUBLE` | 物理量、坐标 |

### 7.3 日期时间

| 场景 | 推荐类型 | 示例 |
|------|---------|------|
| 一般日期 | `DATE` | 生日、订单日期 |
| 高精度时间 | `TIMESTAMP(6)` | 交易日志、审计 |
| 跨时区 | `TIMESTAMP WITH TIME ZONE` | 全球应用 |
| 时间间隔 | `INTERVAL` | 有效期、倒计时 |

## 8. 常见陷阱和最佳实践

### 8.1 隐式类型转换

```sql
-- 避免：字符串和数字比较
SELECT * FROM employees WHERE employee_id = '100';  -- 隐式转换

-- 推荐：显式转换
SELECT * FROM employees WHERE employee_id = TO_NUMBER('100');
```

### 8.2 字符集问题

```sql
-- 检查数据库字符集
SELECT parameter, value 
FROM nls_database_parameters 
WHERE parameter LIKE '%CHARACTERSET%';

-- 确保客户端和服务器字符集一致
```

### 8.3 存储空间优化

```sql
-- 查看表的存储信息
SELECT 
    segment_name,
    bytes/1024/1024 as size_mb
FROM user_segments
WHERE segment_name = 'YOUR_TABLE';

-- 选择合适的大小
VARCHAR2(100) 而不是 VARCHAR2(4000)（如果不需要）
```

### 8.4 NULL 处理

```sql
-- NULL 不等于空字符串
SELECT * FROM employees WHERE name = '';  -- 不会返回 NULL 值

-- 正确判断 NULL
SELECT * FROM employees WHERE name IS NULL;
SELECT * FROM employees WHERE name IS NOT NULL;
```

## 9. 实战练习

### 练习 1：设计员工表

```sql
CREATE TABLE employees (
    emp_id NUMBER(6) PRIMARY KEY,
    emp_no CHAR(8) NOT NULL,           -- 员工编号（固定8位）
    name VARCHAR2(50) NOT NULL,        -- 姓名
    email VARCHAR2(100) UNIQUE,        -- 邮箱
    phone CHAR(11),                     -- 手机号
    hire_date DATE DEFAULT SYSDATE,    -- 入职日期
    salary NUMBER(10,2),               -- 薪资
    commission_pct NUMBER(4,2),        -- 佣金比例
    resume CLOB,                        -- 简历
    photo BLOB,                         -- 照片
    created_at TIMESTAMP(6) DEFAULT SYSTIMESTAMP
);
```

### 练习 2：数据类型转换

```sql
-- 字符串转数字
SELECT TO_NUMBER('123.45') as num FROM dual;

-- 数字转字符串
SELECT TO_CHAR(123.45, '999.99') as str FROM dual;

-- 字符串转日期
SELECT TO_DATE('2026-05-19', 'YYYY-MM-DD') as dt FROM dual;

-- 日期转字符串
SELECT TO_CHAR(SYSDATE, 'YYYY-MM-DD HH24:MI:SS') as str FROM dual;
```

## 10. 总结

### 10.1 核心知识点

✅ **字符类型**：VARCHAR2（首选）、CHAR、CLOB  
✅ **数值类型**：NUMBER（精确）、BINARY_FLOAT/DOUBLE（科学计算）  
✅ **日期类型**：DATE（通用）、TIMESTAMP（高精度）  
✅ **LOB 类型**：CLOB（文本）、BLOB（二进制）  
✅ **特殊类型**：RAW、ROWID、XMLType  

### 10.2 最佳实践

📌 优先使用 VARCHAR2 而非 CHAR  
📌 金额使用 NUMBER，不用浮点数  
📌 明确指定 NUMBER 的精度和标度  
📌 日期使用 DATE，需要高精度用 TIMESTAMP  
📌 避免隐式类型转换  
📌 考虑字符集兼容性
