## 2. UNION（并集）

### 2.1 基本用法

```sql
-- 合并员工和客户姓名（去重）
SELECT name FROM employees
UNION
SELECT name FROM customers;
```

**特点**：
- 自动去除重复行
- 按第一列排序
- 性能较慢（需要排序去重）

### 2.2 实际应用

```sql
-- 查找所有涉及的人员（员工+客户+供应商）
SELECT 'Employee' as type, name, email
FROM employees
UNION
SELECT 'Customer', name, email
FROM customers
UNION
SELECT 'Supplier', contact_name, email
FROM suppliers
ORDER BY type, name;
```

### 2.3 注意事项

```sql
-- ✅ 列数必须相同
SELECT id, name FROM employees
UNION
SELECT id, name FROM customers;

-- ❌ 错误：列数不同
SELECT id, name FROM employees
UNION
SELECT id FROM customers;

-- ✅ 数据类型兼容即可
SELECT employee_id FROM employees
UNION
SELECT customer_id FROM customers;
```

## 3. UNION ALL（快速并集）

### 3.1 基本用法

```sql
-- 合并员工和客户姓名（保留重复）
SELECT name FROM employees
UNION ALL
SELECT name FROM customers;
```

**特点**：
- 不去重
- 不排序
- 性能最优

### 3.2 性能对比

```sql
-- 慢：需要排序去重
SELECT dept_id FROM employees
UNION
SELECT dept_id FROM departments;

-- 快：直接合并
SELECT dept_id FROM employees
UNION ALL
SELECT dept_id FROM departments;
```

**性能差异**：UNION ALL 比 UNION 快 50%-90%

### 3.3 实际应用场景

#### 分区表查询

```sql
-- 查询多个分区的数据
SELECT * FROM sales_2024_q1
UNION ALL
SELECT * FROM sales_2024_q2
UNION ALL
SELECT * FROM sales_2024_q3
UNION ALL
SELECT * FROM sales_2024_q4;
```

#### 汇总统计

```sql
-- 分别统计员工和客户的数量
SELECT 'Employees' as category, COUNT(*) as count FROM employees
UNION ALL
SELECT 'Customers', COUNT(*) FROM customers
UNION ALL
SELECT 'Suppliers', COUNT(*) FROM suppliers;
```

## 4. INTERSECT（交集）

### 4.1 基本用法

```sql
-- 查找同时是员工和客户的人
SELECT email FROM employees
INTERSECT
SELECT email FROM customers;
```

### 4.2 实际应用

```sql
-- 查找有订单又有退货的客户
SELECT customer_id FROM orders
INTERSECT
SELECT customer_id FROM returns;

-- 查找所有部门都有的职位
SELECT job_title FROM employees WHERE dept_id = 10
INTERSECT
SELECT job_title FROM employees WHERE dept_id = 20
INTERSECT
SELECT job_title FROM employees WHERE dept_id = 30;
```

### 4.3 等价写法

```sql
-- INTERSECT 等价于 EXISTS
SELECT DISTINCT e1.email
FROM employees e1
WHERE EXISTS (
    SELECT 1 FROM customers c WHERE c.email = e1.email
);

-- 或使用 JOIN
SELECT DISTINCT e.email
FROM employees e
INNER JOIN customers c ON e.email = c.email;
```

## 5. MINUS（差集）

### 5.1 基本用法

```sql
-- 查找是员工但不是客户的人
SELECT email FROM employees
MINUS
SELECT email FROM customers;
```

### 5.2 实际应用

```sql
-- 查找没有下过订单的客户
SELECT customer_id FROM customers
MINUS
SELECT customer_id FROM orders;

-- 查找没有员工的部门
SELECT dept_id FROM departments
MINUS
SELECT dept_id FROM employees;

-- 查找本月未登录的用户
SELECT user_id FROM users
MINUS
SELECT user_id FROM login_logs
WHERE login_date >= TRUNC(SYSDATE, 'MM');
```

### 5.3 等价写法

```sql
-- MINUS 等价于 NOT EXISTS
SELECT c.customer_id
FROM customers c
WHERE NOT EXISTS (
    SELECT 1 FROM orders o WHERE o.customer_id = c.customer_id
);

-- 或 NOT IN（注意 NULL）
SELECT customer_id
FROM customers
WHERE customer_id NOT IN (
    SELECT customer_id FROM orders WHERE customer_id IS NOT NULL
);
```

## 6. 集合操作的高级用法

### 6.1 组合使用

```sql
-- 查找只在 A 表或只在 B 表的记录（对称差集）
(SELECT id FROM table_a MINUS SELECT id FROM table_b)
UNION ALL
(SELECT id FROM table_b MINUS SELECT id FROM table_a);
```

### 6.2 带 ORDER BY

```sql
-- ORDER BY 只能放在最后
SELECT id, name FROM employees
UNION ALL
SELECT id, name FROM customers
ORDER BY name;  -- 对整个结果集排序

-- ❌ 错误：不能在每个查询中单独排序
SELECT id, name FROM employees ORDER BY name
UNION ALL
SELECT id, name FROM customers ORDER BY name;
```

### 6.3 分页处理

```sql
-- 对 UNION 结果分页
SELECT * FROM (
    SELECT rownum as rn, t.*
    FROM (
        SELECT id, name FROM employees
        UNION ALL
        SELECT id, name FROM customers
        ORDER BY name
    ) t
    WHERE rownum <= 20
)
WHERE rn > 10;
```

## 7. 高级查询技巧

### 7.1 PIVOT（行转列）

```sql
-- 原始数据
SELECT dept_id, job_title, COUNT(*) as cnt
FROM employees
GROUP BY dept_id, job_title;

-- 行转列
SELECT *
FROM (
    SELECT dept_id, job_title, COUNT(*) as cnt
    FROM employees
    GROUP BY dept_id, job_title
)
PIVOT (
    SUM(cnt)
    FOR job_title IN ('MANAGER' as MGR, 'CLERK' as CLR, 'ANALYST' as ANA)
)
ORDER BY dept_id;
```

**输出**：
```
DEPT_ID | MGR | CLR | ANA
--------|-----|-----|----
10      | 2   | 5   | 3
20      | 3   | 4   | 2
```

### 7.2 UNPIVOT（列转行）

```sql
-- 原始数据（宽表）
CREATE TABLE sales_summary (
    year NUMBER,
    q1 NUMBER,
    q2 NUMBER,
    q3 NUMBER,
    q4 NUMBER
);

-- 列转行
SELECT year, quarter, amount
FROM sales_summary
UNPIVOT (
    amount FOR quarter IN (
        q1 AS 'Q1',
        q2 AS 'Q2',
        q3 AS 'Q3',
        q4 AS 'Q4'
    )
);
```

### 7.3 MATCH_RECOGNIZE（模式匹配，Oracle 12c+）

```sql
-- 查找连续上涨的股票
SELECT *
FROM stock_prices
MATCH_RECOGNIZE (
    PARTITION BY stock_id
    ORDER BY trade_date
    MEASURES
        FIRST(trade_date) as start_date,
        LAST(trade_date) as end_date,
        COUNT(*) as days
    PATTERN (A+ B)
    DEFINE
        A AS price > PREV(price)
)
WHERE days >= 5;  -- 连续上涨至少 5 天
```

### 7.4 MODEL 子句

```sql
-- 复杂计算
SELECT *
FROM sales_data
MODEL
    PARTITION BY (region)
    DIMENSION BY (year, month)
    MEASURES (amount, 0 as forecast)
    RULES (
        forecast[2026, 1] = amount[2025, 12] * 1.1,
        forecast[ANY, ANY] = amount[CV(year), CV(month)] * 1.05
    );
```

## 8. 分层查询进阶

### 8.1 CONNECT BY 完整语法

```sql
SELECT 
    employee_id,
    manager_id,
    name,
    LEVEL,
    SYS_CONNECT_BY_PATH(name, '/') as path,
    CONNECT_BY_ROOT name as root_name,
    CONNECT_BY_ISLEAF as is_leaf,
    CONNECT_BY_ISCYCLE as is_cycle
FROM employees
START WITH manager_id IS NULL
CONNECT BY NOCYCLE PRIOR employee_id = manager_id
ORDER SIBLINGS BY name;
```

### 8.2 查找循环引用

```sql
-- 检测循环引用
SELECT 
    employee_id,
    manager_id,
    name,
    LEVEL,
    CONNECT_BY_ISCYCLE as has_cycle
FROM employees
CONNECT BY NOCYCLE PRIOR employee_id = manager_id
WHERE CONNECT_BY_ISCYCLE = 1;
```

### 8.3 树形结构应用

```sql
-- 生成目录树
SELECT 
    LPAD(' ', (LEVEL - 1) * 2) || folder_name as tree_view,
    folder_id,
    parent_id,
    LEVEL
FROM folders
START WITH parent_id IS NULL
CONNECT BY PRIOR folder_id = parent_id
ORDER SIBLINGS BY folder_name;
```

## 9. 窗口函数进阶

### 9.1 排名函数对比

```sql
SELECT 
    name,
    salary,
    ROW_NUMBER() OVER (ORDER BY salary DESC) as row_num,  -- 1,2,3,4
    RANK() OVER (ORDER BY salary DESC) as rank,           -- 1,1,3,4
    DENSE_RANK() OVER (ORDER BY salary DESC) as dense_rank -- 1,1,2,3
FROM employees;
```

### 9.2 NTILE 分组

```sql
-- 将员工分为 4 个薪资等级
SELECT 
    name,
    salary,
    NTILE(4) OVER (ORDER BY salary DESC) as quartile
FROM employees;

-- 每个部门内部分为 3 组
SELECT 
    name,
    dept_id,
    salary,
    NTILE(3) OVER (PARTITION BY dept_id ORDER BY salary DESC) as tertile
FROM employees;
```

### 9.3 前后行访问

```sql
-- LAG/LEAD
SELECT 
    order_date,
    amount,
    LAG(amount, 1) OVER (ORDER BY order_date) as prev_amount,
    LEAD(amount, 1) OVER (ORDER BY order_date) as next_amount,
    amount - LAG(amount, 1) OVER (ORDER BY order_date) as diff
FROM orders;

-- 计算环比增长
SELECT 
    month,
    revenue,
    ROUND((revenue - LAG(revenue) OVER (ORDER BY month)) / 
          LAG(revenue) OVER (ORDER BY month) * 100, 2) as growth_rate
FROM monthly_sales;
```

### 9.4 累积计算

```sql
-- 累积求和
SELECT 
    order_date,
    amount,
    SUM(amount) OVER (ORDER BY order_date) as running_total
FROM orders;

-- 移动平均（7天）
SELECT 
    trade_date,
    price,
    AVG(price) OVER (
        ORDER BY trade_date 
        ROWS BETWEEN 6 PRECEDING AND CURRENT ROW
    ) as moving_avg_7d
FROM stock_prices;

-- 年初至今累计
SELECT 
    month,
    revenue,
    SUM(revenue) OVER (
        PARTITION BY EXTRACT(YEAR FROM month)
        ORDER BY month
    ) as ytd_revenue
FROM monthly_sales;
```

### 9.5 FIRST_VALUE / LAST_VALUE

```sql
SELECT 
    name,
    dept_id,
    salary,
    FIRST_VALUE(name) OVER (
        PARTITION BY dept_id ORDER BY salary DESC
    ) as highest_earner,
    LAST_VALUE(name) OVER (
        PARTITION BY dept_id ORDER BY salary DESC
        ROWS BETWEEN UNBOUNDED PRECEDING AND UNBOUNDED FOLLOWING
    ) as lowest_earner
FROM employees;
```

**注意**：LAST_VALUE 需要指定 `ROWS BETWEEN UNBOUNDED PRECEDING AND UNBOUNDED FOLLOWING`

## 10. 查询优化技巧

### 10.1 避免全表扫描

```sql
-- ❌ 函数导致索引失效
SELECT * FROM employees WHERE UPPER(name) = 'JOHN';

-- ✅ 使用函数索引
CREATE INDEX idx_emp_name_upper ON employees(UPPER(name));

-- 或使用 LIKE
SELECT * FROM employees WHERE name LIKE 'John%';
```

### 10.2 减少子查询

```sql
-- ❌ 相关子查询（慢）
SELECT e.name,
    (SELECT d.department_name FROM departments d WHERE d.dept_id = e.dept_id)
FROM employees e;

-- ✅ JOIN（快）
SELECT e.name, d.department_name
FROM employees e
LEFT JOIN departments d ON e.dept_id = d.dept_id;
```

### 10.3 使用 Hint 提示

```sql
-- 强制使用索引
SELECT /*+ INDEX(e idx_emp_salary) */ *
FROM employees e
WHERE salary > 5000;

-- 强制使用哈希连接
SELECT /*+ USE_HASH(e d) */ *
FROM employees e
JOIN departments d ON e.dept_id = d.dept_id;

-- 并行查询
SELECT /*+ PARALLEL(4) */ *
FROM large_table;
```

### 10.4 物化视图

```sql
-- 创建物化视图
CREATE MATERIALIZED VIEW mv_dept_stats
BUILD IMMEDIATE
REFRESH COMPLETE ON DEMAND
AS
SELECT 
    dept_id,
    COUNT(*) as emp_count,
    AVG(salary) as avg_salary,
    SUM(salary) as total_salary
FROM employees
GROUP BY dept_id;

-- 查询物化视图（快速）
SELECT * FROM mv_dept_stats;

-- 刷新物化视图
EXEC DBMS_MVIEW.REFRESH('MV_DEPT_STATS');
```

## 11. 实战案例

### 案例 1：销售报表

```sql
-- 月度销售趋势（含环比、同比）
WITH monthly_sales AS (
    SELECT 
        TO_CHAR(order_date, 'YYYY-MM') as month,
        SUM(amount) as revenue,
        COUNT(DISTINCT customer_id) as customers,
        COUNT(*) as orders
    FROM orders
    GROUP BY TO_CHAR(order_date, 'YYYY-MM')
)
SELECT 
    month,
    revenue,
    customers,
    orders,
    ROUND(revenue / NULLIF(orders, 0), 2) as avg_order_value,
    ROUND((revenue - LAG(revenue) OVER (ORDER BY month)) / 
          LAG(revenue) OVER (ORDER BY month) * 100, 2) as mom_growth,
    ROUND((revenue - LAG(revenue, 12) OVER (ORDER BY month)) / 
          LAG(revenue, 12) OVER (ORDER BY month) * 100, 2) as yoy_growth
FROM monthly_sales
ORDER BY month;
```

### 案例 2：Top N 查询

```sql
-- 每个部门薪资最高的 3 人
SELECT *
FROM (
    SELECT 
        name,
        dept_id,
        salary,
        ROW_NUMBER() OVER (PARTITION BY dept_id ORDER BY salary DESC) as rn
    FROM employees
)
WHERE rn <= 3
ORDER BY dept_id, rn;
```

## 12. 最佳实践

### 12.1 集合操作

✅ 优先使用 UNION ALL（性能更好）  
✅ 确保列类型一致  
✅ 只在最后使用 ORDER BY  
✅ 注意 NULL 值的影响  

### 12.2 窗口函数

✅ 明确指定窗口范围  
✅ LAST_VALUE 使用完整窗口  
✅ 合理使用 PARTITION BY  
✅ 避免过度复杂的窗口定义  

### 12.3 性能优化

📌 使用 EXPLAIN PLAN 分析  
📌 为常用查询创建索引  
📌 考虑使用物化视图  
📌 避免不必要的排序  
📌 限制结果集大小

## 13. 总结

### 13.1 核心知识点

✅ **集合操作**：UNION、INTERSECT、MINUS  
✅ **行列转换**：PIVOT、UNPIVOT  
✅ **窗口函数**：排名、累积、移动平均  
✅ **分层查询**：CONNECT BY  
✅ **查询优化**：索引、Hint、物化视图  

### 13.2 选择指南

📌 合并数据 → UNION ALL  
📌 找共同记录 → INTERSECT  
📌 找差异记录 → MINUS  
📌 行转列 → PIVOT  
📌 排名分析 → 窗口函数  
📌 层级数据 → CONNECT BY
