# 02约束与完整性

**学习目标：**
> * 掌握 Oracle 五种约束类型
> * 能够创建和管理约束
> * 理解级联操作和延迟约束
> * 学会诊断约束相关错误

---

## 1. 约束概述

### 1.1 什么是约束？

约束（Constraint）是数据库级别的规则，用于保证数据的完整性和一致性。

**约束的作用**：
- 防止无效数据插入
- 维护表之间的关系
- 保证业务规则的执行
- 提高数据质量

### 1.2 约束类型

| 约束类型 | 关键字 | 作用 |
|---------|--------|------|
| **非空约束** | `NOT NULL` | 列不能为空 |
| **唯一约束** | `UNIQUE` | 值必须唯一 |
| **主键约束** | `PRIMARY KEY` | 唯一标识行 |
| **外键约束** | `FOREIGN KEY` | 引用其他表 |
| **检查约束** | `CHECK` | 满足条件表达式 |

## 2. NOT NULL 约束

### 2.1 创建时指定

```sql
CREATE TABLE employees (
    emp_id NUMBER(6) NOT NULL,
    name VARCHAR2(50) NOT NULL,
    email VARCHAR2(100),
    phone VARCHAR2(20)
);
```

### 2.2 修改表添加

```sql
-- 添加 NOT NULL 约束
ALTER TABLE employees MODIFY email NOT NULL;

-- 删除 NOT NULL 约束
ALTER TABLE employees MODIFY email NULL;
```

### 2.3 注意事项

```sql
-- 如果表中已有 NULL 值，无法添加 NOT NULL 约束
-- 先清理数据
UPDATE employees SET email = 'unknown@company.com' WHERE email IS NULL;

-- 再添加约束
ALTER TABLE employees MODIFY email NOT NULL;
```

## 3. UNIQUE 约束

### 3.1 列级约束

```sql
CREATE TABLE employees (
    emp_id NUMBER(6) PRIMARY KEY,
    email VARCHAR2(100) UNIQUE,
    phone VARCHAR2(20) UNIQUE
);
```

### 3.2 表级约束

```sql
CREATE TABLE employees (
    emp_id NUMBER(6),
    email VARCHAR2(100),
    phone VARCHAR2(20),
    CONSTRAINT uk_email UNIQUE (email),
    CONSTRAINT uk_phone UNIQUE (phone)
);
```

### 3.3 复合唯一约束

```sql
CREATE TABLE course_enrollments (
    student_id NUMBER,
    course_id NUMBER,
    enrollment_date DATE,
    CONSTRAINT uk_student_course UNIQUE (student_id, course_id)
);

-- 同一学生不能重复选修同一课程
INSERT INTO course_enrollments VALUES (1, 101, SYSDATE);  -- 成功
INSERT INTO course_enrollments VALUES (1, 101, SYSDATE+1); -- 失败：违反唯一约束
```

### 3.4 NULL 值处理

```sql
-- UNIQUE 允许多个 NULL 值
INSERT INTO employees (emp_id, email) VALUES (1, NULL);  -- 成功
INSERT INTO employees (emp_id, email) VALUES (2, NULL);  -- 成功

-- 但只允许一个非 NULL 的相同值
INSERT INTO employees (emp_id, email) VALUES (3, 'test@test.com');  -- 成功
INSERT INTO employees (emp_id, email) VALUES (4, 'test@test.com');  -- 失败
```

## 4. PRIMARY KEY 约束

### 4.1 单列主键

```sql
-- 方式1：列级
CREATE TABLE departments (
    dept_id NUMBER(4) PRIMARY KEY,
    dept_name VARCHAR2(50)
);

-- 方式2：表级
CREATE TABLE departments (
    dept_id NUMBER(4),
    dept_name VARCHAR2(50),
    CONSTRAINT pk_departments PRIMARY KEY (dept_id)
);
```

### 4.2 复合主键

```sql
CREATE TABLE order_items (
    order_id NUMBER,
    item_id NUMBER,
    product_name VARCHAR2(100),
    quantity NUMBER,
    CONSTRAINT pk_order_items PRIMARY KEY (order_id, item_id)
);
```

### 4.3 主键特性

✅ 自动创建唯一索引  
✅ 不允许 NULL 值  
✅ 每个表只能有一个主键  
✅ 通常作为外键引用的目标

## 5. FOREIGN KEY 约束

### 5.1 基本语法

```sql
CREATE TABLE employees (
    emp_id NUMBER(6) PRIMARY KEY,
    name VARCHAR2(50),
    dept_id NUMBER(4),
    CONSTRAINT fk_emp_dept 
        FOREIGN KEY (dept_id) 
        REFERENCES departments(dept_id)
);
```

### 5.2 级联操作

#### CASCADE（级联删除）

```sql
CREATE TABLE employees (
    emp_id NUMBER(6) PRIMARY KEY,
    name VARCHAR2(50),
    dept_id NUMBER(4),
    CONSTRAINT fk_emp_dept 
        FOREIGN KEY (dept_id) 
        REFERENCES departments(dept_id)
        ON DELETE CASCADE
);

-- 删除部门时，该部门所有员工也被删除
DELETE FROM departments WHERE dept_id = 10;
```

#### SET NULL（设置为 NULL）

```sql
CREATE TABLE employees (
    emp_id NUMBER(6) PRIMARY KEY,
    name VARCHAR2(50),
    dept_id NUMBER(4),
    CONSTRAINT fk_emp_dept 
        FOREIGN KEY (dept_id) 
        REFERENCES departments(dept_id)
        ON DELETE SET NULL
);

-- 删除部门时，员工的 dept_id 被设为 NULL
DELETE FROM departments WHERE dept_id = 10;
```

#### NO ACTION（默认，阻止删除）

```sql
-- 如果有员工引用该部门，删除会失败
DELETE FROM departments WHERE dept_id = 10;
-- ORA-02292: integrity constraint violated - child record found
```

### 5.3 自引用外键

```sql
CREATE TABLE employees (
    emp_id NUMBER(6) PRIMARY KEY,
    name VARCHAR2(50),
    manager_id NUMBER(6),
    CONSTRAINT fk_emp_manager 
        FOREIGN KEY (manager_id) 
        REFERENCES employees(emp_id)
);

-- 员工可以指向另一个员工作为经理
INSERT INTO employees VALUES (1, 'CEO', NULL);
INSERT INTO employees VALUES (2, 'Manager', 1);
INSERT INTO employees VALUES (3, 'Staff', 2);
```

### 5.4 禁用和启用外键

```sql
-- 禁用约束（批量导入数据时有用）
ALTER TABLE employees DISABLE CONSTRAINT fk_emp_dept;

-- 启用约束
ALTER TABLE employees ENABLE CONSTRAINT fk_emp_dept;

-- 启用并验证现有数据
ALTER TABLE employees ENABLE VALIDATE CONSTRAINT fk_emp_dept;

-- 启用但不验证现有数据（更快）
ALTER TABLE employees ENABLE NOVALIDATE CONSTRAINT fk_emp_dept;
```

## 6. CHECK 约束

### 6.1 基本用法

```sql
CREATE TABLE products (
    product_id NUMBER PRIMARY KEY,
    product_name VARCHAR2(100),
    price NUMBER(10,2) CHECK (price > 0),
    quantity NUMBER CHECK (quantity >= 0),
    category VARCHAR2(20) CHECK (category IN ('Electronics', 'Books', 'Clothing'))
);
```

### 6.2 复杂条件

```sql
CREATE TABLE employees (
    emp_id NUMBER(6) PRIMARY KEY,
    name VARCHAR2(50),
    salary NUMBER(10,2),
    commission_pct NUMBER(4,2),
    hire_date DATE,
    
    -- 薪资必须大于 0
    CONSTRAINT chk_salary CHECK (salary > 0),
    
    -- 佣金比例在 0-1 之间
    CONSTRAINT chk_commission CHECK (commission_pct BETWEEN 0 AND 1),
    
    -- 入职日期不能晚于今天
    CONSTRAINT chk_hire_date CHECK (hire_date <= SYSDATE),
    
    -- 多列检查
    CONSTRAINT chk_salary_comm CHECK (
        (commission_pct IS NULL AND salary >= 3000) OR
        (commission_pct IS NOT NULL AND salary >= 2000)
    )
);
```

### 6.3 命名约束

```sql
-- 推荐：显式命名约束
CREATE TABLE orders (
    order_id NUMBER PRIMARY KEY,
    order_date DATE NOT NULL,
    status VARCHAR2(20),
    total_amount NUMBER(12,2),
    
    CONSTRAINT chk_order_status CHECK (status IN ('PENDING', 'PROCESSING', 'SHIPPED', 'DELIVERED', 'CANCELLED')),
    CONSTRAINT chk_total_amount CHECK (total_amount >= 0)
);
```

## 7. 延迟约束（DEFERRABLE）

### 7.1 概念

延迟约束允许在事务提交时才检查约束，而不是每条语句执行时检查。

### 7.2 使用场景

```sql
-- 创建可延迟约束
CREATE TABLE accounts (
    account_id NUMBER PRIMARY KEY,
    balance NUMBER(12,2) CHECK (balance >= 0) DEFERRABLE INITIALLY IMMEDIATE
);

-- 转账操作（中间状态可能违反约束）
SET CONSTRAINTS ALL DEFERRED;

UPDATE accounts SET balance = balance - 1000 WHERE account_id = 1;
-- 此时 account_1 余额可能为负（暂时违反约束）

UPDATE accounts SET balance = balance + 1000 WHERE account_id = 2;
-- 此时 account_2 余额增加

COMMIT;  -- 提交时检查约束
```

### 7.3 切换约束模式

```sql
-- 当前会话设置为延迟检查
SET CONSTRAINTS ALL DEFERRED;

-- 恢复为立即检查
SET CONSTRAINTS ALL IMMEDIATE;

-- 针对特定约束
SET CONSTRAINTS chk_balance DEFERRED;
```

## 8. 约束管理

### 8.1 查看约束信息

```sql
-- 查看表的所有约束
SELECT 
    constraint_name,
    constraint_type,
    table_name,
    status,
    deferrable,
    deferred
FROM user_constraints
WHERE table_name = 'EMPLOYEES';

-- 约束类型：
-- P: Primary Key
-- U: Unique
-- R: Foreign Key (Reference)
-- C: Check
-- V: With Check Option
-- O: Read Only
```

### 8.2 查看约束列

```sql
SELECT 
    constraint_name,
    column_name,
    position
FROM user_cons_columns
WHERE table_name = 'EMPLOYEES'
ORDER BY constraint_name, position;
```

### 8.3 添加约束

```sql
-- 添加主键
ALTER TABLE employees ADD CONSTRAINT pk_employees PRIMARY KEY (emp_id);

-- 添加外键
ALTER TABLE employees ADD CONSTRAINT fk_emp_dept 
    FOREIGN KEY (dept_id) REFERENCES departments(dept_id);

-- 添加检查约束
ALTER TABLE employees ADD CONSTRAINT chk_salary CHECK (salary > 0);

-- 添加唯一约束
ALTER TABLE employees ADD CONSTRAINT uk_email UNIQUE (email);
```

### 8.4 删除约束

```sql
-- 删除约束
ALTER TABLE employees DROP CONSTRAINT chk_salary;

-- 删除主键（包括索引）
ALTER TABLE employees DROP PRIMARY KEY DROP INDEX;

-- 级联删除外键
ALTER TABLE departments DROP PRIMARY KEY CASCADE;
```

### 8.5 启用/禁用约束

```sql
-- 禁用约束
ALTER TABLE employees DISABLE CONSTRAINT fk_emp_dept;

-- 启用约束
ALTER TABLE employees ENABLE CONSTRAINT fk_emp_dept;

-- 禁用所有约束（慎用）
BEGIN
    FOR c IN (SELECT constraint_name FROM user_constraints WHERE table_name = 'EMPLOYEES') LOOP
        EXECUTE IMMEDIATE 'ALTER TABLE employees DISABLE CONSTRAINT ' || c.constraint_name;
    END LOOP;
END;
```

## 9. 约束冲突处理

### 9.1 常见错误

#### ORA-00001：违反唯一约束

```sql
INSERT INTO employees (emp_id, email) VALUES (1, 'test@test.com');
INSERT INTO employees (emp_id, email) VALUES (2, 'test@test.com');
-- ORA-00001: unique constraint (UK_EMAIL) violated
```

**解决**：
```sql
-- 查找重复值
SELECT email, COUNT(*)
FROM employees
GROUP BY email
HAVING COUNT(*) > 1;

-- 删除或修改重复数据
DELETE FROM employees WHERE emp_id = 2;
```

#### ORA-02291：违反外键约束

```sql
INSERT INTO employees (emp_id, dept_id) VALUES (1, 999);
-- ORA-02291: integrity constraint violated - parent key not found
```

**解决**：
```sql
-- 先插入父记录
INSERT INTO departments (dept_id) VALUES (999);

-- 再插入子记录
INSERT INTO employees (emp_id, dept_id) VALUES (1, 999);
```

#### ORA-02292：违反外键约束（子记录存在）

```sql
DELETE FROM departments WHERE dept_id = 10;
-- ORA-02292: integrity constraint violated - child record found
```

**解决**：
```sql
-- 先删除子记录
DELETE FROM employees WHERE dept_id = 10;

-- 再删除父记录
DELETE FROM departments WHERE dept_id = 10;

-- 或使用级联删除（建表时设置 ON DELETE CASCADE）
```

#### ORA-02290：违反检查约束

```sql
INSERT INTO products (product_id, price) VALUES (1, -10);
-- ORA-02290: check constraint (CHK_PRICE) violated
```

**解决**：确保数据符合检查条件。

## 10. 最佳实践

### 10.1 命名规范

```sql
-- 推荐的命名约定
pk_<table_name>          -- 主键
fk_<table>_<referenced>  -- 外键
uk_<table>_<column>      -- 唯一键
chk_<table>_<condition>  -- 检查约束
nn_<table>_<column>      -- 非空约束

-- 示例
CREATE TABLE employees (
    emp_id NUMBER CONSTRAINT pk_employees PRIMARY KEY,
    email VARCHAR2(100) CONSTRAINT uk_employees_email UNIQUE,
    dept_id NUMBER CONSTRAINT fk_employees_dept REFERENCES departments(dept_id),
    salary NUMBER CONSTRAINT chk_employees_salary CHECK (salary > 0)
);
```

### 10.2 设计建议

✅ **始终为主键命名**  
✅ **外键建立索引**（提高连接查询性能）  
✅ **使用有意义的约束名**  
✅ **优先使用声明式约束**（而非触发器）  
✅ **谨慎使用级联删除**  
✅ **批量导入时可临时禁用约束**  

### 10.3 性能考虑

```sql
-- 外键列创建索引
CREATE INDEX idx_emp_dept ON employees(dept_id);

-- 批量加载数据时禁用约束
ALTER TABLE employees DISABLE CONSTRAINT fk_emp_dept;
-- ... 加载数据 ...
ALTER TABLE employees ENABLE CONSTRAINT fk_emp_dept;
```

## 11. 实战练习

### 练习 1：创建完整的订单系统

```sql
-- 客户表
CREATE TABLE customers (
    customer_id NUMBER PRIMARY KEY,
    name VARCHAR2(100) NOT NULL,
    email VARCHAR2(100) UNIQUE NOT NULL,
    phone VARCHAR2(20),
    created_at DATE DEFAULT SYSDATE
);

-- 产品表
CREATE TABLE products (
    product_id NUMBER PRIMARY KEY,
    name VARCHAR2(200) NOT NULL,
    price NUMBER(10,2) CHECK (price > 0),
    stock_quantity NUMBER CHECK (stock_quantity >= 0),
    category VARCHAR2(50) CHECK (category IN ('Electronics', 'Books', 'Clothing', 'Food'))
);

-- 订单表
CREATE TABLE orders (
    order_id NUMBER PRIMARY KEY,
    customer_id NUMBER NOT NULL,
    order_date DATE DEFAULT SYSDATE,
    status VARCHAR2(20) DEFAULT 'PENDING' 
        CHECK (status IN ('PENDING', 'CONFIRMED', 'SHIPPED', 'DELIVERED', 'CANCELLED')),
    total_amount NUMBER(12,2) CHECK (total_amount >= 0),
    CONSTRAINT fk_orders_customer 
        FOREIGN KEY (customer_id) 
        REFERENCES customers(customer_id)
        ON DELETE CASCADE
);

-- 订单明细表
CREATE TABLE order_items (
    item_id NUMBER PRIMARY KEY,
    order_id NUMBER NOT NULL,
    product_id NUMBER NOT NULL,
    quantity NUMBER CHECK (quantity > 0),
    unit_price NUMBER(10,2) CHECK (unit_price > 0),
    CONSTRAINT fk_items_order 
        FOREIGN KEY (order_id) 
        REFERENCES orders(order_id)
        ON DELETE CASCADE,
    CONSTRAINT fk_items_product 
        FOREIGN KEY (product_id) 
        REFERENCES products(product_id)
);

-- 创建外键索引
CREATE INDEX idx_orders_customer ON orders(customer_id);
CREATE INDEX idx_items_order ON order_items(order_id);
CREATE INDEX idx_items_product ON order_items(product_id);
```

### 练习 2：测试约束

```sql
-- 测试非空约束
INSERT INTO customers (customer_id, email) VALUES (1, 'test@test.com');
-- ORA-01400: cannot insert NULL into ("NAME")

-- 测试唯一约束
INSERT INTO customers VALUES (1, 'John', 'john@test.com', '123456', SYSDATE);
INSERT INTO customers VALUES (2, 'Jane', 'john@test.com', '789012', SYSDATE);
-- ORA-00001: unique constraint violated

-- 测试检查约束
INSERT INTO products VALUES (1, 'Product A', -10, 100, 'Electronics');
-- ORA-02290: check constraint violated

-- 测试外键约束
INSERT INTO orders VALUES (1, 999, SYSDATE, 'PENDING', 100);
-- ORA-02291: parent key not found
```

## 12. 总结

### 12.1 核心知识点

✅ **NOT NULL**：确保列不为空  
✅ **UNIQUE**：确保值唯一（允许多个 NULL）  
✅ **PRIMARY KEY**：唯一标识（NOT NULL + UNIQUE）  
✅ **FOREIGN KEY**：维护参照完整性  
✅ **CHECK**：自定义条件检查  
✅ **DEFERRABLE**：延迟约束检查  

### 12.2 最佳实践

📌 始终为约束命名  
📌 外键列创建索引  
📌 谨慎使用级联删除  
📌 批量导入时临时禁用约束  
📌 使用有意义的约束名称
